            <h3 class="entry-title">Reversing Golang Developed Ransomware: SNAKE</h3>
        </header><!-- .entry-header -->

        <div class="entry-summary">

            <h2 class="wp-block-heading">Introduction</h2><p>Snake Ransomware (or EKANS Ransomware) is a Golang ransomware which in the past has affected several companies such as Enel and Honda. &nbsp;The MD5 hashing of the analyzed sample is&nbsp;<a href="https://app.any.run/tasks/ad3d809f-e197-4091-9b93-b6d68368c05b/" data-type="URL" data-id="https://app.any.run/tasks/ad3d809f-e197-4091-9b93-b6d68368c05b/" target="_blank" rel="noreferrer noopener">ED3C05BDE9F0EA0F1321355B03AC42D0</a>. This sample in particular is obfuscated with&nbsp;<a href="https://github.com/unixpickle/gobfuscate" target="_blank" rel="noreferrer noopener">Gobfuscate</a>, an open source obfuscation project available on Github.</p><p>Let’s start by quickly summarizing the functionality of the malware:</p><ul class="wp-block-list"><li>First, the sample checks the domain to which the infected host belongs to, before continuing execution</li>

<li>Next, it checks whether the computer is a Backup Domain Controller or Primary Domain Controller, and if so will only drop the ransom note, rather than encrypting the machine</li>

<li>SNAKE will then isolate the host machine from the network by leveraging the netsh tool</li>

<li>The shadow copies on the system are deleted using WMI</li>

<li>SNAKE attempts to terminate any running AV, EDR, and SIEM components</li>

<li>Finally, local files on the system are encrypted<ul class="wp-block-list"><li>For each file, a unique AES encryption key is generated, which is later encrypted with an RSA-2048 public key and stored within the encrypted file.</li></ul></li></ul><p>Let’s start reversing this sample with IDA!</p><h2 class="wp-block-heading">Static Analysis</h2><p>There are some differences of Go from other languages to keep in mind:</p><ul class="wp-block-list"><li>Functions can return multiple values.</li>

<li>Function parameters are passed on the stack.</li>

<li>Strings are typically a sequence of bytes with a fixed length, that are not null terminated; the string is represented by a structure formed by a pointer to a byte array, and the length of the string.</li>

<li>The constants are stored in one large buffer and sorted by length.</li>

<li>Many stack manipulations are present within the binaries, that can make the analysis complex.</li></ul><h3 class="wp-block-heading">Obfuscation</h3><p>Opening the sample with IDA we immediately notice that the function names are very obfuscated:</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image001.png" alt="" class="wp-image-6479"/><figcaption class="wp-element-caption">Gobfuscate obfuscates almost all function names within the binary</figcaption></figure></div><p>Performing a quick search, I found that the malware is obfuscated with&nbsp;<strong><a href="https://github.com/unixpickle/gobfuscate" target="_blank" rel="noreferrer noopener">Gobfuscate</a></strong>. This project performs obfuscation of several components: Global names, Package names, Struct methods, and Strings.</p><p>Each string in the binary is replaced by a function call. Each function contains two arrays that are xored to get the original string. When implementing the decryption function, keep in mind that there are different ways in which these arrays are passed to the function&nbsp;<strong>runtime.stringtoslicebyte</strong>&nbsp;– either via a variable, a pointer or a hardcoded value).</p><p>Analyzing the sample, you will usually find the call to a main function that contains a large number of other calls within it, where each subroutine performs decryption of only one string. Sometimes only the decryption operations are found within these subroutines, other times additional operations are performed on the decrypted string.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image002.png" alt="" class="wp-image-6481"/><figcaption class="wp-element-caption">String Decryption Functions</figcaption></figure></div><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image003.png" alt="" class="wp-image-6482"/><figcaption class="wp-element-caption">String Decryption Functions</figcaption></figure></div><p>As mentioned, the string encryption is fairly basic, XORing the contents of two arrays together to retrieve the final string.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image004.png" alt="" class="wp-image-6483"/><figcaption class="wp-element-caption">Loading the two arrays for decryption</figcaption></figure></div><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image005.png" alt="" class="wp-image-6484"/><figcaption class="wp-element-caption">XORing the arrays together to get the decrypted string</figcaption></figure></div><p>Due to the simplicity of the algorithm, we can develop a simple Python script utilising some regular expressions to locate and decrypt 90% of the encrypted strings within the binary! The script can be found at the end of this post.</p><pre class="wp-block-code"><code>startDecryptFunction = b"\x64\x8B\x0D\x14\x00\x00\x00" 
sliceStr = b"(" + b"\x8D.....\x89.\$\x04\xC7\x44\$\b...." + b")" 
xorLoop = b"\x0F\xB6&lt;\)1\xFE(\x83|\x81)\xFD" </code></pre><p>With the majority of the strings decrypted, let’s continue the analysis!</p><h3 class="wp-block-heading">Check Environment</h3><p>One of the first operations I perform when analyzing malware in GO is to jump to the&nbsp;<strong>main.init</strong>&nbsp;function.</p><p>The&nbsp;<strong>main.init</strong>&nbsp;is generated for each package by the compiler to initialise all other packages that this sample relies on, as well as the global variables; analysing this function is very important because it allows us to understand a large amount of the malware’s functionality and speed up subsequent analysis.</p><p>In the&nbsp;<strong>main.init</strong>&nbsp;function we can find, for example, references to encryption:&nbsp;<strong>AES, RSA, SHA1, X509</strong>. In addition, there are several functions for decryption of strings and function names.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image006.png" alt="" class="wp-image-6485"/><figcaption class="wp-element-caption">Initialisations performed within main.init function</figcaption></figure></div><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image007.png" alt="" class="wp-image-6488"/><figcaption class="wp-element-caption">API function decryption and loading through NewProc</figcaption></figure></div><p>Now we move on to analyze the&nbsp;<strong>main.main</strong>&nbsp;function. One of the first activities the malware performs is to check the environment before continuing with encryption.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image008.png" alt="" class="wp-image-6489"/><figcaption class="wp-element-caption">Call to CheckEnvironment within main.main</figcaption></figure></div><p>The function&nbsp;<strong>CheckEnvironment</strong>&nbsp;starts by attempting to resolve the hostname&nbsp;<strong>mds.honda.com</strong>&nbsp;and compare the returned value with&nbsp;<strong>172[.]108[.]71[.]153</strong>.<strong>&nbsp;</strong>This check is used to confirm that the infected machine is part of the correct domain. In fact, it is important to remember that this ransomware is deployed at the end of the infection chain by other loaders, and thus it is likely custom-built for the victim.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image009.png" alt="" class="wp-image-6491"/><figcaption class="wp-element-caption">Resolving hostname via DNS</figcaption></figure></div><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image010-1.png" alt="" class="wp-image-6493"/><figcaption class="wp-element-caption">Comparing resolved address to hardcoded address</figcaption></figure></div><p>After the first environment check, the malware executes the API calls&nbsp;<strong>CoInitializeEx</strong>,&nbsp;<strong>CoInitializeSecurity</strong>&nbsp;and&nbsp;<strong>CoCreateInstance</strong>&nbsp;to instantiate an object of the&nbsp;<strong>SWbemLocator</strong>&nbsp;interface. Using the&nbsp;<strong>SWbemLocator</strong>&nbsp;object, SNAKE then invokes the method&nbsp;<strong>SWbemLocator::ConnectServer</strong>&nbsp;and obtains a pointer to an&nbsp;<strong>SWbemServices</strong>&nbsp;object. Finally, with this object, it will execute&nbsp;<strong>ExecQuery&nbsp;</strong>with the following query:</p><pre class="wp-block-code"><code>select DomainRole from Win32_ComputerSystem</code></pre><p>In an attempt to determine whether the infected computer is a server or a workstation.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image011.png" alt="" class="wp-image-6494"/><figcaption class="wp-element-caption">Decryption of object used to perform WMI query</figcaption></figure></div><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image012.png" alt="" class="wp-image-6495"/><figcaption class="wp-element-caption">Decryption of WMI query to retrieve DomainRole</figcaption></figure></div><p>After making the query, the malware only continues execution if the&nbsp;<strong>DomainRole</strong>&nbsp;value is equal to or less than 3.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image013.png" alt="" class="wp-image-6496"/><figcaption class="wp-element-caption">Execution of WMI query and checking of result</figcaption></figure></div><p>According to&nbsp;<a href="https://docs.microsoft.com/en-us/dotnet/api/microsoft.powershell.commands.domainrole?view=powershellsdk-1.1.0" target="_blank" rel="noreferrer noopener">Microsoft documentation</a>, the integers returned by the call correspond to different values:</p><figure class="wp-block-table"><table><tbody><tr><td><strong>VALUE</strong></td><td><strong>MEANING</strong></td></tr><tr><td>0</td><td>Standalone Workstation</td></tr><tr><td>1</td><td>Member Workstation</td></tr><tr><td>2</td><td>Standalone Server</td></tr><tr><td>3</td><td>Member Server</td></tr><tr><td>4</td><td>Backup Domain Controller</td></tr><tr><td>5</td><td>Primary Domain Controller</td></tr></tbody></table></figure><p>Therefore, the malware performs the infection only if the role obtained of the computer is&nbsp;<strong>Standalone Workstation</strong>,&nbsp;<strong>Member Workstation</strong>,&nbsp;<strong>Standalone Server</strong>, or<strong>&nbsp;Member Server.</strong></p><p>If this check is successful, the mutex&nbsp;<strong>Global\EKANS</strong>&nbsp;is created, and presuming the mutex is created successfully, the sample continues executing.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image014.png" alt="" class="wp-image-6498"/><figcaption class="wp-element-caption">Execution flow depending on the result of the DomainRole</figcaption></figure></div><p>If, on the other hand, the computer role is either a backup domain controller or primary domain controller, a ransom note is dropped to C:\Users\Public\Desktop, and files are not encrypted. Within the ransom note is an email on how to contact the threat actors, with the email used in this sample being&nbsp;<strong><a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="b9fad8cbcbd6d5fbd0dddcd5d5f9cdcccdd8d7d6cdd897dad6d4">[email&#160;protected]</a></strong>.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image015-1024x521.png" alt="" class="wp-image-6500"/><figcaption class="wp-element-caption">Ransom note</figcaption></figure></div><p>When analyzing Go developed programs, two functions to pay attention to are&nbsp;<strong>NewLazyDll</strong>&nbsp;(essentially LoadLibrary), and&nbsp;<strong>NewProc</strong>&nbsp;(as you may have guessed, basically GetProcAddress). With the use of Gobfuscate to obfuscate this sample, the names of the libraries and API functions to be passed to the described functions. Pointers to the loaded libraries/functions are stored within DWORDs for later reference.</p><p>For example, we can see in the sample we have the function that performs the decryption of a function name before calling&nbsp;<strong>LazyDLL.NewProc</strong>:</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image016.png" alt="" class="wp-image-6501"/><figcaption class="wp-element-caption">API function decryption and loading via NewProc</figcaption></figure></div><p>A pointer to the function is saved in a DWORD, so that we can trace to see where the function is called within the binary.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image017-1024x201.png" alt="" class="wp-image-6502"/><figcaption class="wp-element-caption">Cross references for API functions</figcaption></figure></div><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image018.png" alt="" class="wp-image-6503"/><figcaption class="wp-element-caption">Execution of previously loaded API function</figcaption></figure></div><h3 class="wp-block-heading">Endpoint Isolation</h3><p>After the function&nbsp;<em>CheckEnvironment</em>&nbsp;has finished, the strings “<strong>netsh advfirewall set allprofiles state on</strong>” and “<strong>netsh advfirewall set allprofiles firewallpolicy blockinbound,blockoutbound</strong>” are decrypted and executed via&nbsp;<strong>cmd.run</strong>. The first command enables Windows Firewall for all network profiles, while the second blocks all incoming and outgoing connections. This is fairly unusual behaviour for ransomware, which typically performs lateral movement across a network to infect additional machines.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image019.png" alt="" class="wp-image-6504"/><figcaption class="wp-element-caption">Decryption of netsh commands to alter the firewall</figcaption></figure></div><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image020.png" alt="" class="wp-image-6506"/><figcaption class="wp-element-caption">Execution of above commands through os.exec</figcaption></figure></div><h3 class="wp-block-heading">Terminate Process And Services</h3><p>Prior to encryption, the ransomware terminates a number of processes, to reduce the amount of interference with the encryption (for example any open file handles), as well as disable any running EDR/SIEM software on the machine.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image021.png" alt="" class="wp-image-6507"/><figcaption class="wp-element-caption">Decryption of target processes to be terminated</figcaption></figure></div><p>Processes are terminated using&nbsp;<strong>syscall.OpenProcess</strong>&nbsp;and&nbsp;<strong>syscall.TerminateProcess</strong>&nbsp;calls. In order for SNAKE to retrieve the PIDs of the target processes, the usual calls of CreateToolhelp32Snapshot, Process32FirstW, and Process32NextW are performed.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image022.png" alt="" class="wp-image-6509"/><figcaption class="wp-element-caption">Terminating processes with OpenProcess and TerminateProcess</figcaption></figure></div><p>In addition to terminating processes, the ransomware stops more than 200 services related to EDR, SIEM, AV, etc.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image023.png" alt="" class="wp-image-6510"/><figcaption class="wp-element-caption">Decryption of target service names to be terminated</figcaption></figure></div><p>In order to terminate services, the following API function calls are made:</p><ul class="wp-block-list"><li><strong>OpenSCManagerA</strong>: gets a service control manager handle for subsequent calls.</li>

<li><strong>EnumServicesStatusEx</strong>: enumeration of services.</li>

<li><strong>OpenServiceW</strong>: gets a service handle for subsequent calls.</li>

<li><strong>QueryServiceStatusEx</strong>: check the status of services.</li>

<li><strong>ControlService:&nbsp;</strong>used to stop the service (flag SERVICE_CONTROL_STOP).</li></ul><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image002-1-1024x585.png" alt="" class="wp-image-6540"/><figcaption class="wp-element-caption">Termination of services using OpenService and ControlService</figcaption></figure></div><h3 class="wp-block-heading">Shadow Copy Deletion</h3><p>The ransomware executes the WMI query&nbsp;<strong>“SELECT * FROM Win32_ShadowCopy”</strong>&nbsp;to get the IDs of Shadow Copies and will always use WMI for deletion (remember that there are many ways to perform shadow copy deletion).</p><p>In addition to the&nbsp;<strong>Wbemscripting.SWbemLocator</strong>&nbsp;object,&nbsp;<strong>WbemScripting.SWbemNamedValueSet</strong>&nbsp;is also created.</p><p>For deletion, SNAKE uses the&nbsp;<strong>DeleteInstance</strong>&nbsp;method by passing the ID of previously obtained Shadow Copies.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image025.png" alt="" class="wp-image-6513"/><figcaption class="wp-element-caption">Decryption of WbemScripting.SWbemNamedValueSet</figcaption></figure></div><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image026.png" alt="" class="wp-image-6514"/><figcaption class="wp-element-caption">Decryption of WMI Query</figcaption></figure></div><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image027.png" alt="" class="wp-image-6515"/><figcaption class="wp-element-caption">Decryption of WMI namespace</figcaption></figure></div><h3 class="wp-block-heading">Encryption Process</h3><p>SNAKE first encrypts all the various files by initializing 8 go-routines (<strong>runtime.newproc</strong>), before beginning to rename the files.</p><p>The offset of the function that does the encryption is passed to&nbsp;<strong>runtime.newproc</strong>&nbsp;(<strong>OffsetStartEncryption</strong>).</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image028.png" alt="" class="wp-image-6516"/><figcaption class="wp-element-caption">Initialisation of go-routines prior to file renaming function</figcaption></figure></div><p>Before beginning encryption of the file, it’s checked that it has not already been encrypted by checking for the presence of the string&nbsp;<strong>EKANS</strong>&nbsp;at the end of the file.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image029.png" alt="" class="wp-image-6517"/><figcaption class="wp-element-caption">Checking if file is already encrypted</figcaption></figure></div><p>If the file hasn’t yet been encrypted and the files are among those to be encrypted (there is an allowlist and a denylist), encryption is initiated, which takes care of:</p><ul class="wp-block-list"><li>Generating AES key for each file; this key is encrypted with an RSA public key in OAEP Mode.</li>

<li>Encryption of file via AES in CTR mode, with Random Key (32 bytes) and Random IV (16 bytes).</li>

<li>A random 5 character is appended to the file extension of encrypted files.</li>

<li>Adds data to the end of the file: encrypted AES Key, IV and EKANS string.</li></ul><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image030.png" alt="" class="wp-image-6519"/><figcaption class="wp-element-caption">Key generation, encryption, and metadata being added to file</figcaption></figure></div><p>After instantiating the CTR cipher with&nbsp;<strong>cipher.NewCTR,</strong>&nbsp;encryption is performed with the&nbsp;<strong>XORKeyStream</strong>&nbsp;method of that class.</p><p>The function reads 0x19000 bytes at a time and after encryption the file is rewritten using&nbsp;<strong>WriteAt</strong>.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image031.png" alt="" class="wp-image-6520"/><figcaption class="wp-element-caption">Generating the buffer to hold bytes read from the file</figcaption></figure></div><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image032.png" alt="" class="wp-image-6521"/><figcaption class="wp-element-caption">Encrypting buffer data and overwriting file</figcaption></figure></div><p>After finishing the encryption, three more writes are performed on the file:</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image033.png" alt="" class="wp-image-6522"/><figcaption class="wp-element-caption">Adding metadata to the file</figcaption></figure></div><p>It’s easy to see that in the last one the string&nbsp;<strong>EKANS</strong>&nbsp;is written (which is used to determine if the file has already been encrypted), while it is much more complex to figure out what is written in the first two. As a result, let’s jump over to a debugger.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image034.png" alt="" class="wp-image-6523"/><figcaption class="wp-element-caption">Observing metadata being written to end of file using a debugger</figcaption></figure></div><p>The first write adds the following to the file:</p><ul class="wp-block-list"><li>The encrypted AES Key</li>

<li>The random IV</li>

<li>The path of encrypted file</li></ul><p>The AES Key for each file is encrypted with a public RSA key. After decryption, the public key is parsed with&nbsp;<strong>pem.decode&nbsp;</strong>and&nbsp;<strong>x509.ParsePKCS1PublicKey</strong>.</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image035-1024x184.png" alt="" class="wp-image-6525"/><figcaption class="wp-element-caption">Decryption of RSA public key</figcaption></figure></div><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image001-1.png" alt="" class="wp-image-6539"/><figcaption class="wp-element-caption">Parsing of the RSA key</figcaption></figure></div><p>The first parameter of the&nbsp;<strong>EncryptOAEP</strong>&nbsp;function must be the hash function, which in this case is&nbsp;<strong>sha1</strong>:</p><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image037-1024x203.png" alt="" class="wp-image-6527"/><figcaption class="wp-element-caption">EncryptOAEP Function</figcaption></figure></div><div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://web.archive.org/web/20220824130324im_/https://www.0ffset.net/wp-content/uploads/2022/08/image038.png" alt="" class="wp-image-6529"/><figcaption class="wp-element-caption">Call to EncryptOAEP function</figcaption></figure></div><p>Various extensions, file and folders are excluded for file encryption, also using a regex.</p><p><strong>Partially excluded Files</strong>:</p><figure
