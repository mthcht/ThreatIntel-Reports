            <h3 class="entry-title">BAZARLOADER: Unpacking an ISO File Infection</h3>
        </header><!-- .entry-header -->

        <div class="entry-summary">

            <p>BAZARLOADER (aka BAZARBACKDOOR) is a Windows-based loader that spreads through attachments in phishing emails. During an infection, the final loader payload typically downloads and executes a Cobalt Strike beacon to provide remote access for the threat actors, which, in a lot of cases, leads to ransomware being deployed to the victim’s machine.</p><p>In this initial post, we will unpack the different stages of a BAZARLOADER infection that comes in the form of an optical disk image (ISO) file. We will also dive into the obfuscation methods used by the main BAZARLOADER payload.</p><p>To follow along, you can grab the sample as well as the PCAP files for it on <a href="https://www.malware-traffic-analysis.net/2022/02/07/index.html" target="_blank" rel="noreferrer noopener">Malware-Traffic-Analysis.net</a>.</p><p>SHA256: 0900b4eb02bdcaefd21df169d21794c8c70bfbc68b2f0612861fcabc82f28149</p><h2 class="wp-block-heading"><strong>Step 1: Mounting ISO File &amp; Extracting Stage 1 Executable</strong></h2><p>Recent BAZARLOADER samples arrive in emails containing OneDrive links to download an ISO file to avoid detection since most AVs tend to ignore this particular file type. With Windows 7 and above integrating the mounting functionality into Windows Explorer, we can mount any ISO file as a virtual drive by double-clicking on it.</p><p>When we mount the malicious ISO file, we see that a drive is mounted on the system that contains a shortcut file named <strong>“Attachments.lnk”</strong> and a hidden file named <strong>“documents.log”</strong>.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img decoding="async" width="833" height="203" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_01.png" alt="" class="wp-image-5759" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_01.png 833w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_01-600x146.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_01-300x73.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_01-768x187.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_01-720x175.png 720w" sizes="(max-width: 833px) 100vw, 833px" /></figure></div><p>The shortcut file has to be run by the victim to begin the chain of infection. We can quickly extract the actual command being executed by this shortcut from its <strong>Properties </strong>window.</p><pre class="wp-block-code"><code>C:\Windows\System32\rundll32.exe documents.log,vspa</code></pre><div class="wp-block-image"><figure class="aligncenter size-full"><img decoding="async" width="362" height="507" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_02.png" alt="" class="wp-image-5761" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_02.png 362w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_02-214x300.png 214w" sizes="(max-width: 362px) 100vw, 362px" /></figure></div><p>Once the victim double-clicks on the shortcut file, the command executes the Windows <strong>rundll32.exe </strong>program to launch the <strong>“documents.log”</strong> file. This lets us know that the file being launched is a DLL file, and the entry point is its export function <strong>vspa</strong>.</p><h2 class="wp-block-heading"><strong>Step 2: Extracting Second Stage Shellcode</strong></h2><p>Taking a quick look in IDA, we can somewhat tell that the extracted DLL is packed since it has only a few functions and a really suspicious looking buffer of ASCII characters in its custom <strong>.odata</strong> section.</p><div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="623" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_03-1024x623.png" alt="" class="wp-image-5764" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_03-1024x623.png 1024w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_03-600x365.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_03-300x182.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_03-768x467.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_03-720x438.png 720w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_03.png 1039w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure></div><p>With that in mind, we will just perform some quick static analysis to determine where we can dump the next stage.</p><p>In the first function of the <strong>vspa </strong>export, we see <strong>sub_1800045D6</strong> takes a DWORD in as the parameter. This function returns a variable that contains an address to a function that is later called in the code.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="902" height="340" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_04.png" alt="" class="wp-image-5765" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_04.png 902w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_04-600x226.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_04-300x113.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_04-768x289.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_04-720x271.png 720w" sizes="(max-width: 902px) 100vw, 902px" /></figure></div><p>At this point, we can safely guess that <strong>sub_1800045D6 </strong>is an API resolving function, and the parameter it takes is the hash of the API’s name. Because this is still the unpacking phase, we won’t dive too deep into analyzing this function.</p><p>Instead, I’ll just use <strong>OALabs’s</strong> <a href="https://github.com/OALabs/hashdb-ida" target="_blank" rel="noreferrer noopener"><strong>HashDB</strong></a> IDA plugin to quickly reverse-lookup the hashing algorithm used from the hash. The result shows that the hash corresponds to an API name hashed with <a href="https://github.com/OALabs/hashdb/blob/b3cc544fad41c749d04616663ac13f02bf1aaf37/algorithms/metasploit.py" target="_blank" rel="noreferrer noopener">Metasploit’s hashing algorithm <strong>ROR13</strong></a>.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="663" height="307" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_05.png" alt="" class="wp-image-5766" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_05.png 663w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_05-600x278.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_05-300x139.png 300w" sizes="(max-width: 663px) 100vw, 663px" /></figure></div><p>After determining the hashing algorithm, we can use <strong>HashDB </strong>to quickly look up the APIs being resolved by this function. It becomes clear that this function resolves the <strong>RtlAllocateHeap</strong> API, calls that to allocate a heap buffer and writes the encoded ASCII data to it.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="923" height="444" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_06-1.png" alt="" class="wp-image-5772" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_06-1.png 923w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_06-1-600x289.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_06-1-300x144.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_06-1-768x369.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_06-1-720x346.png 720w" sizes="(max-width: 923px) 100vw, 923px" /></figure></div><p>From this point onward, we can guess that the packer will decode this buffer and launch it somewhere later in the code. If we skip toward the end of the <strong>vspa </strong>export, we see a <strong>call </strong>instruction<strong> </strong>on a variable that is not returned from the API resolving function, so it can potentially be our tail jump.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="885" height="302" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_07.png" alt="" class="wp-image-5773" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_07.png 885w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_07-600x205.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_07-300x102.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_07-768x262.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_07-720x246.png 720w" sizes="(max-width: 885px) 100vw, 885px" /></figure></div><p>The last function to modify that <strong>v19 </strong>variable is <strong>sub_180003FE6</strong>, so we can quickly take a look at that.</p><p>It turns out the <strong>sub_180003FE6</strong> function just resolves and calls <strong>NtMapViewOfSection</strong> to map a view of a section into the virtual address space and writes the base address of the view into the <strong>v19 </strong>variable. Then, it just executes <strong>qmemcpy</strong> to copy the data in the second variable to the returned virtual base address.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="885" height="570" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_08.png" alt="" class="wp-image-5774" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_08.png 885w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_08-600x386.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_08-300x193.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_08-768x495.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_08-720x464.png 720w" sizes="(max-width: 885px) 100vw, 885px" /></figure></div><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="617" height="193" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_09.png" alt="" class="wp-image-5775" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_09.png 617w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_09-600x188.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_09-300x94.png 300w" sizes="(max-width: 617px) 100vw, 617px" /></figure></div><p>This tells us two things. First, our guess that the <strong>v19 </strong>variable will contain the address to executable code is correct. Second, we know that the executable code is shellcode since the data is mapped and executed directly at offset 0 from where it is written.</p><p>From here, we can set up <strong>x64dbg</strong>, execute the DLL file at the <strong>vspa</strong> export, and break at the call instruction. After stepping into the function, we will be at the head of the shellcode.</p><div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="520" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_10-1024x520.png" alt="" class="wp-image-5777" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_10-1024x520.png 1024w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_10-600x305.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_10-300x152.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_10-768x390.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_10-720x366.png 720w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_10.png 1166w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure></div><p>We can now dump this virtual memory buffer to retrieve the second stage shellcode for the next unpacking step.</p><h2 class="wp-block-heading"><strong>Step 3: Extracting The Final BAZARLOADER Payload</strong></h2><p>When we examine the shellcode in IDA, we can quickly use the same trick with <strong>HashDB </strong>above to see that the shellcode also performs API hashing with Metasploit’s <strong>ROR13</strong>.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="794" height="489" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_11.png" alt="" class="wp-image-5780" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_11.png 794w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_11-600x370.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_11-300x185.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_11-768x473.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_11-720x443.png 720w" sizes="(max-width: 794px) 100vw, 794px" /></figure></div><p>At the entry point above, the shellcode resolves a set of functions that it will call, most notably <strong>VirtualAlloc</strong> and <strong>VirtualProtect</strong>. These two functions are typically used by packers to allocate virtual memory to decode and write the next stage executable in before launching it.</p><p>With this in mind, our next step should be debugging the shellcode and setting breakpoints at these two API calls. We can pick up where we are after dumping in <strong>x64dbg</strong> during <strong>Step 2</strong>, or we can launch the shellcode directly in our debugger using OALabs’s <a href="https://github.com/OALabs/BlobRunner">BlobRunner</a> or similar shellcode launcher.</p><p>Our first hit with <strong>VirtualAlloc</strong> is a call to allocate a virtual memory buffer at virtual address 0x204140000 with the size of 0x2A000 bytes.</p><div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="452" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_12-1024x452.png" alt="" class="wp-image-5784" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_12-1024x452.png 1024w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_12-600x265.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_12-300x132.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_12-768x339.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_12-720x318.png 720w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_12.png 1355w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure></div><p>We can run until <strong>VirtualAlloc</strong> returns and start monitoring the memory at address 0x204140000. After running until the next <strong>VirtualProtect</strong> call, we see that a valid PE executable has been written to this memory region.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="860" height="377" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_13.png" alt="" class="wp-image-5785" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_13.png 860w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_13-600x263.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_13-300x132.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_13-768x337.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_13-720x316.png 720w" sizes="(max-width: 860px) 100vw, 860px" /></figure></div><p>Finally, we can dump this memory region into a file to extract the BAZARLOADER payload.</p><h2 class="wp-block-heading"><strong>Step 4: BAZARLOADER’s String Obfuscation</strong></h2><p>As we begin performing static analysis on BAZARLOADER, it is crucial that we identify obfuscation methods that the malware uses.</p><p>One of those methods is string obfuscation, where the malware uses encoded stack strings to hide them from static analysis.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="730" height="219" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_14.png" alt="" class="wp-image-5787" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_14.png 730w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_14-600x180.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_14-300x90.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_14-720x216.png 720w" sizes="(max-width: 730px) 100vw, 730px" /></figure></div><p>As shown, a typical encoded string is pushed on the stack and decoded dynamically using some multiplication, subtraction, and modulus operations.</p><p>There are different ways to resolve these stack strings, such as writing IDAPython scripts, <a href="https://github.com/mandiant/flare-emu" target="_blank" rel="noreferrer noopener">emulation</a>, or just running the program in a debugger and dumping the stack strings when they are resolved.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="935" height="540" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_15.png" alt="" class="wp-image-5789" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_15.png 935w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_15-600x347.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_15-300x173.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_15-768x444.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_15-720x416.png 720w" sizes="(max-width: 935px) 100vw, 935px" /></figure></div><h2 class="wp-block-heading"><strong>Step 5: BAZARLOADER’s API Obfuscation</strong></h2><p>BAZARLOADER obfuscates most of its API calls through a few structures that it constructs in the <strong>DllEntryPoint </strong>function.</p><p>First, the malware populates the following structure that contains a handle to <strong>Kernel32.dll</strong> and addresses to API required to load libraries and get their API addresses.</p><pre class="wp-block-code"><code>struct API_IMPORT_STRUCT {
    HANDLE kernel32_handle;
    FARPROC mw_GetProcAddress;
    FARPROC mw_LoadLibraryW;
    FARPROC mw_LoadLibraryA;
    FARPROC mw_LoadLibraryA2;
    FARPROC mw_FreeLibrary;
    FARPROC mw_GetModuleHandleW;
    FARPROC mw_GetModuleHandleA;
};
</code></pre><p>It calls <strong>GetModuleHandle</strong> to retrieve the handle to <strong>Kernel32.dll</strong>, calls <strong>GetProcAddress</strong> to retrieve the address of the <strong>GetProcAddress</strong> API, and writes those in the structure.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="933" height="613" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_16.png" alt="" class="wp-image-5791" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_16.png 933w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_16-600x394.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_16-300x197.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_16-768x505.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_16-720x473.png 720w" sizes="(max-width: 933px) 100vw, 933px" /></figure></div><p>Using the structure’s <strong>GetProcAddress</strong> API field, BAZARLOADER retrieves the rest of the required APIs to populate other fields in the structure. This <strong>API_IMPORT_STRUCT</strong> structure will later be used to import other libraries’ APIs.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="947" height="362" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_17.png" alt="" class="wp-image-5794" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_17.png 947w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_17-600x229.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_17-300x115.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_17-768x294.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_17-720x275.png 720w" sizes="(max-width: 947px) 100vw, 947px" /></figure></div><p>Next, for each library to be imported, BAZARLOADER populates the following <strong>LIBRARY_STRUCT </strong>structure that contains a set of functions to interact with the library and the library handle.</p><pre class="wp-block-code"><code>struct LIB_FUNCS
{
    FARPROC free_lib;
    FARPROC w_free_lib;
    __int64 (__fastcall *get_API_addr)(API_IMPORT_STRUCT*, HANDLE, char*);
};

struct LIBRARY_STRUCT
{
    LIB_FUNCS *lib_funcs;
    HANDLE lib_handle;
};
</code></pre><p>The first 2 functions in the <strong>LIB_FUNCS </strong>structure just call the <strong>FreeLibrary</strong> API from the global <strong>API_IMPORT_STRUCT</strong> to free the library module.</p><p>The third function calls the <strong>GetProcAddress </strong>from the <strong>API_IMPORT_STRUCT</strong>’s field to retrieve the address of an API exported from that specific library.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="656" height="256" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_18.png" alt="" class="wp-image-5795" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_18.png 656w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_18-600x234.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_18-300x117.png 300w" sizes="(max-width: 656px) 100vw, 656px" /></figure></div><div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="81" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_19-1024x81.png" alt="" class="wp-image-5797" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_19-1024x81.png 1024w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_19-600x47.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_19-300x24.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_19-768x61.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_19-720x57.png 720w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_19.png 1079w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure></div><p>To begin populating each <strong>LIBRARY_STRUCT </strong>structure, BAZARLOADER decodes the library name from a stack string and populates it with the corresponding set of functions and the library handle retrieved from calling <strong>LoadLibraryA</strong>.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="774" height="398" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_20.png" alt="" class="wp-image-5798" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_20.png 774w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_20-600x309.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_20-300x154.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_20-768x395.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_20-720x370.png 720w" sizes="(max-width: 774px) 100vw, 774px" /></figure></div><p>Below is the list of all libraries used by the malware.</p><pre class="wp-block-code"><code>kernel32.dll, wininet.dll, advapi32.dll, ole32.dll, rpcrt4.dll, shell32.dll, bcrypt.dll, crypt32.dll, dnsapi.dll, netapi32.dll, shlwapi.dll, user32.dll, ktmw32.dll</code></pre><p>The <strong>LIBRARY_STRUCT </strong>structures corresponding to these are pushed into a global list in the order below.</p><pre class="wp-block-code"><code>struct LIBRARY_STRUCT_LIST
{
    LIBRARY_STRUCT *lib_struct_kernel32;
    LIBRARY_STRUCT *lib_struct_wininet;
    LIBRARY_STRUCT *lib_struct_advapi32;
    LIBRARY_STRUCT *lib_struct_ole32;
    LIBRARY_STRUCT *lib_struct_rpcrt4;
    LIBRARY_STRUCT *lib_struct_shell32;
    LIBRARY_STRUCT *lib_struct_bcrypt;
    LIBRARY_STRUCT *lib_struct_crypt32;
    LIBRARY_STRUCT *lib_struct_dnsapi;
    LIBRARY_STRUCT *lib_struct_netapi32;
    LIBRARY_STRUCT *lib_struct_shlwapi;
    LIBRARY_STRUCT *lib_struct_user32;
    LIBRARY_STRUCT *lib_struct_ktmw32;
};
</code></pre><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="767" height="673" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_21.png" alt="" class="wp-image-5800" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_21.png 767w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_21-600x526.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_21-300x263.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_21-720x632.png 720w" sizes="(max-width: 767px) 100vw, 767px" /></figure></div><p>After this global list of <strong>LIBRARY_STRUCT </strong>is populated, an API can be called from a function taking in its corresponding library’s <strong>LIBRARY_STRUCT </strong>structure and its parameters. </p><p>This function resolves the API name from a stack string, retrieves the API’s address using the <strong>get_API_addr</strong> function from the library structure, and calls the API with its parameters.</p><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="847" height="259" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_22.png" alt="" class="wp-image-5802" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_22.png 847w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_22-600x183.png 600w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_22-300x92.png 300w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_22-768x235.png 768w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_22-720x220.png 720w" sizes="(max-width: 847px) 100vw, 847px" /></figure></div><div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="570" height="220" src="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_23.png" alt="" class="wp-image-5803" srcset="https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_23.png 570w, https://www.0ffset.net/wp-content/uploads/2022/04/bazarloader_23-300x116.png 300w" sizes="(max-width: 570px) 100vw, 570px" /></figure></div><p>The way the wrapper function is setup to call the actual API is really intuitive, making the code simple to understand through static analysis. However, it’s a bit more difficult to automate the process since there is no API hashing involved.</p><p>For my analysis, I just manually decode the stack strings in my debugger and rename the wrapper function accordingly.</p><p>At this point, we have fully unpacked BAZARLOADER and understood how the malware obfuscates its strings and APIs to make analysis harder.</p><p>In the next blog post, we will fully analyze how the loader downloads and launches a Cobalt Strike beacon from its C2 servers!</p>
