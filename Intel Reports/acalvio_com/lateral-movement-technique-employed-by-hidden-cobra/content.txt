                        <h1>Lateral Movement Technique Employed by Hidden Cobra</h1>
                    </div>
                                        <div class="date">by&nbsp;Team Acalvio | June 13, 2018</div>

                <div class="social-content-wrap">

                  <div class="blog-innerpage-content">
                    <img width="600" height="330" src="/wp-content/uploads/2018/05/Acalvio_New_Web_Blog_600x330_02.jpg" class="attachment-post-thumbnail size-post-thumbnail wp-post-image" alt="Acalvio Autonomous Deception" loading="lazy" srcset="/wp-content/uploads/2018/05/Acalvio_New_Web_Blog_600x330_02.jpg 600w, /wp-content/uploads/2018/05/Acalvio_New_Web_Blog_600x330_02-300x165.jpg 300w" sizes="(max-width: 600px) 100vw, 600px" />                    <div class="content-wrap">
                      <p>US-Cert recently issued notification regarding malicious cyber activity by the North Korean government [1] Hidden Cobra. Â There are two families of malware used by the North Korean Government.</p>
<ul>
<li>Remote Access Tool (RAT) known as Jonap</li>
<li>A Server Message Block (SMB) worm called as Brambul worm.</li>
</ul>
<p>As per the US-Certreport, HddenÂ Cobra has been using this malware since 2009 to target multiple victims globally and in the United States, including media, aerospace, financial industries, and critical infrastructure sectors.</p>
<p>In this blog, we share the technical details and spreading techniques used by the Brambul worm. Thereafter, we discuss how it can be detected by <a href="/why-active-defense/">distributed deception platform</a>.</p>
<p><b>Brambul Worm </b></p>
<p>The worm invokes multiple threads which then randomly generates IP addresses for infection.</p>
<p><img loading="lazy" class="wp-image-4404 aligncenter" src="/wp-content/uploads/2018/06/Screen-Shot-2018-06-11-at-3.33.49-PM.png" sizes="(max-width: 476px) 100vw, 476px" srcset="/wp-content/uploads/2018/06/Screen-Shot-2018-06-11-at-3.33.49-PM.png 974w, /wp-content/uploads//wp-content/uploads/2018/06/Screen-Shot-2018-06-11-at-3.33.49-PM-300x188-1.png 300w, /wp-content/uploads/2018/06/Screen-Shot-2018-06-11-at-3.33.49-PM-768x481-1.png 768w, /wp-content/uploads/2018/06/Screen-Shot-2018-06-11-at-3.33.49-PM-400x250-1.png 400w" alt="" width="476" height="298" /></p>
<p>Figure 1.0 Showing the code for random generation of IP address.</p>
<p>Once the victimâs IP addresses have been generated it connects to \\IPC$ share, on the port 445 of the victim machine using Administrator as the username and fixed hardcoded passwords.</p>
<p>Thereafter, the malware code makes a call to the WNetAddConnection2 API Â to connect to a network resource and constructs the below command.</p>
<p>â<i>cmd.exe /q /c net share admin$=%%SystemRoot%% /GRANT:%s, FULL</i>â</p>
<p>It then makes calls to the service manager. <i>OpenSCManagerA()</i> with the victim machine machines on the network as the parameter. <i>Â StartSeviceA()</i> then executes the command which grants full permission on the remote machine. Â Once the command has been executed, the code makes aÂ call to <i>DeleteService</i>() which then deletes the service.</p>
<p><img loading="lazy" class="wp-image-4405 aligncenter" src="/wp-content/uploads/2018/06/Screen-Shot-2018-06-11-at-2.19.07-PM-1024x379.png" sizes="(max-width: 859px) 100vw, 859px" srcset="/wp-content/uploads/2018/06/Screen-Shot-2018-06-11-at-2.19.07-PM-1024x379.png 1024w, /wp-content/uploads/2018/06/Screen-Shot-2018-06-11-at-2.19.07-PM-300x111.png 300w, /wp-content/uploads/2018/06/Screen-Shot-2018-06-11-at-2.19.07-PM-768x284.png 768w, /wp-content/uploads//wp-content/uploads/2018/06/Screen-Shot-2018-06-11-at-2.19.07-PM-1080x400.png 1080w, /wp-content/uploads/2018/06/Screen-Shot-2018-06-11-at-2.19.07-PM.png 1600w" alt="" width="859" height="318" /></p>
<p>Once the full permission is granted on the remote machine, the worm is copied to the remote machine.</p>
<p><b>Detection by Distributed Deception Platform </b></p>
<p>As such, the worm is not quite sophisticated and primarily relies on brute force attempts. This will be successful only in weak environments. If a Distributed Deception Platform is deployed in a <a href="/resources/blog/deception-deployment-strategies-threat-agnostic-vs-service-agnostic/">threat agnostic </a>manner network, enumeration by the Brambul Worm will get detected with very high confidence. [deployment of a distributed deception platform is discussed inÂ a<a href="/resources/blog/deception-deployment-strategies-threat-agnostic-vs-service-agnostic/"> previous blog</a>]. Brute force attacks on the Distributed Deception Platform leads to isolation of the end-point, thereby containing damage in a timely manner.</p>
