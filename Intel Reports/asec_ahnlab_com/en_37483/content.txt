						<h1 class="post-title single_blog_inner__Title">
							Gwisin Ransomware Targeting Korean Companies						</h1>

												<div class="slider-post-meta-items tab-small-col-meta blog-details-meta-wrap">
							<div class="slider-meta-left">

<!-- 								<div class="slider-meta-left-author">
									<img alt='' src='https://secure.gravatar.com/avatar/b627a3f2c6e7bf34dc4ff9908f03f7c3?s=48&#038;r=g' srcset='https://secure.gravatar.com/avatar/b627a3f2c6e7bf34dc4ff9908f03f7c3?s=96&#038;r=g 2x' class='avatar avatar-48 photo' height='48' width='48' decoding='async'/>								</div> -->
								
								<div class="slider-meta-left-content">

<!-- 									<h4 class="post-author-name">
										ATCP									</h4> -->

									<ul class="slider-bottom-meta-list">
										<li class="slider-meta-date">Jul 27 2022</li>
									</ul>
								</div>
							</div>
							
						</div>
							

						</header>
					</div>  

				</div>
							
<!-- 				<div class="theme-blog-details"> -->
				<div class="ck-content">
				
								<div class="post-featured-image">
									<img class="img-fluid" src="https://asec.ahnlab.com/wp-content/uploads/2022/07/10_ransomware_02-803x490.png?crop=1" alt="Gwisin Ransomware Targeting Korean Companies">
									</div>
								
				<div class="post-body clearfix single-blog-header single-blog-inner blog-single-block blog-details-content">
					<!-- Article content -->
					<div class="entry-content clearfix">
						
						<p>The cases of Gwisin ransomware attacking Korean companies are recently on the rise. It is being distributed to target specific companies. It is similar to Magniber in that it operates in the MSI installer form. Yet unlike Magniber which targets random individuals, Gwisin does not perform malicious behaviors on its own, requiring a special value for the execution argument. The value is used as key information to run the DLL file included in the MSI.</p>
<p>As such, the file alone does not perform ransomware activities on security products of various sandbox environments, making it difficult to detect Gwisin. The ransomware&#8217;s internal DLL operates by being injected into a normal Windows process. The process is different for each infected company.</p>
<p>The following shows the characteristics of Gwisin that have been identified so far.</p>
<p>(1) Distributed in an MSI installer file form<br />(2) Uses the argument value used to run MSI to run internal DLL<br />(3) Performs ransomware behaviors by being injected into a Windows system process<br />(4) Contains the information of the infected company inside the DLL (displayed in the ransom note)<br />(5) Supports a feature to encrypt files in safe mode</p>
<p>When the MSI file is run, it calls the export function update() of the internal ransomware DLL. The function checks the execution argument. If it is abnormal, the function will not operate.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2022/07/image-103.png?resize=391%2C242&amp;ssl=1" alt=""></figure>
<p>&nbsp;</p>
<p>At the moment of the encryption process, the ransomware is executed with the following arguments (some parts of the arguments are hidden).</p>
<p>&nbsp;</p>
<p>&gt; msiexec /qn /i C:\ProgramData\*****.msi&nbsp;SERIAL=463f********7ce7&nbsp;LICENSE=7f21********5071&nbsp;SMM=0&nbsp;ORG=***</p>
<p>Among arguments that are needed to run Gwisin, SMM can have a value of 0 or 1. Normally, the routine for encrypting files is processed if the value is 0. If SMM is 1, the ransomware is installed to operate on safe mode. It first copies itself to a certain path of ProgramData and is registered as a service. It then uses bcdedit to set the boot option as safe mode. The computer is forcibly rebooted after 5 seconds. After the system is rebooted in safe mode, the registered service starts encrypting files.</p>
<figure class="table">
<table>
<tbody>
<tr>
<td><strong>Service Name</strong></td>
<td><strong>Command</strong></td>
</tr>
<tr>
<td>a35f23725b5feab2</td>
<td>&gt; msiexec /qn /i C:\ProgramData\*****.msi&nbsp;SERIAL=463f********7ce7&nbsp;LICENSE=7f21********5071&nbsp;SMM=0&nbsp;ORG=***</td>
</tr>
</tbody>
</table>
</figure>
<p style="text-align:center;"><span style="color:#6B7684;font-size:14px;">Registered service</span></p>
<p>&nbsp;</p>
<p>When the process for verifying the argument ends, the ransomware decrypts the internal shellcode using the arguments. It then runs a normal program &#8220;certreq.exe&#8221; to inject the decrypted shellcode. The injected shellcode ultimately decrypts Gwisin to run it in the memory (besides &#8220;certreq.exe&#8221;, various normal Window processes are used to run the ransomware).</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2022/07/image-104.png?resize=330%2C115&amp;ssl=1" alt=""></figure>
<p>&nbsp;</p>
<p>After encrypting files, the ransomware changes the extension name to the name of the targeted company.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2022/07/image-105.png?resize=271%2C150&amp;ssl=1" alt=""></figure>
<p>The folder chosen to be encrypted contains a ransom note. The name of the note also contains the extension string such as &#8220;!!!_HOW_TO_UNLOCK_******_FILES_!!!.TXT&#8221;. The note file shows a list of stolen information and contacts.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2022/07/image-106.png?resize=827%2C330&amp;ssl=1" alt=""></figure>
<p>&nbsp;</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2022/07/image-107.png?resize=1024%2C444&amp;ssl=1" alt=""></figure>
<p>&nbsp;</p>
<p>The Gwisin cases show that the anti-malware products are neutralized before the infection process begins. As V3 products block Gwisin ransomware using such a method in the injection process through behavior-based detection, it is necessary to enable the &#8216;Behavior-based Detection&#8217; option.</p>
<p>Because the ransomware is installed and executed in various systems after dominating the internal system, companies must analyze how the infection happened in the first place. If the cause of the infection cannot be analyzed after a breach had occurred, another ransomware may infect the system in the future and cause a similar incident.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2022/07/image-102.png?resize=398%2C265&amp;ssl=1" alt=""></figure>
<p>&nbsp;</p>
<p><strong>[File Detection]</strong><br />– Ransomware/Win.Gwisin.C5214965 (2022.07.27.03)</p>
<p><strong>[Behavior Detection]</strong><br />– Injection/MDP.Event.M4387 (2022.07.28.00)</p>
<p>&nbsp;</p>
