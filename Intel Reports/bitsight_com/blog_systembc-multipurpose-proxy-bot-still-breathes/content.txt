<div  class="block-content block-content--type-text block-content--21976 block-content--view-mode-full container text-wysiwyg">

  
  

          <div class="text-wysiwyg__title">
      

    </div>
  
      <div class="text-wysiwyg__body">
      
            <div class="clearfix text-formatted field field--name-body field--type-text-with-summary field--label-hidden field__item">
                          <p>SystemBC is a <a data-entity-substitution="canonical" data-entity-type="node" data-entity-uuid="92be95e6-24bc-4dab-a986-eda833d0df71" href="/blog/what-is-malware">malware</a> written in C that turns infected computers into SOCKS5 proxies. The bot communicates with the command and control server using a custom binary protocol over TCP and uses RC4 encryption. This malware has evolved its capabilities since it was <a href="https://www.proofpoint.com/us/threat-insight/post/systembc-christmas-july-socks5-malware-and-exploit-kits">documented by Proofpoint</a> [1] back in 2019, and now it can also download and run additional files. Moreover, this malware can target both Windows and Linux platforms.</p>

<p>SystemBC is sold on underground marketplaces, and after completing the purchase, the buyer receives an archive containing the bot executable, the command and control (C2) server executable, and a PHP admin panel.</p>

<center>
<article class="media media--type-image media--view-mode-original">
  
      
            <div class="field field--name-field-image field--type-image field--label-hidden field__item">
                            <img loading="lazy" src="/sites/default/files/2022/09/19/SystemBC-figure-1-min.png" width="752" height="252" alt="SystemBC-figure-1" typeof="foaf:Image">


              </div>
      
  </article>

Figure 1. Archive contents</center>

<p>The main capability of SystemBC is to turn the infected computers into SOCKS5 proxies. Since most bots are not reachable from the internet, this malware uses a backconnect architecture that allows clients to use the proxies (infected computers) through the backconnect (C2) server without ever needing to interact directly with them.</p>

<p>In practice, what happens is that for each infected computer that connects to the backconnect (C2) server, the server opens a new TCP port that will accept the SOCKS5 traffic from clients. That traffic is wrapped inside SystemBC's communications protocol and forwarded to the infected computers that will unwrap the traffic, send it to the destination, and send the response back to the backconnect (C2) server that will finally forward that response back to the client that initialized the communication.</p>

<center>
<article class="media media--type-image media--view-mode-original">
  
      
            <div class="field field--name-field-image field--type-image field--label-hidden field__item">
                            <img loading="lazy" src="/sites/default/files/2022/09/19/SystemBC-figure-2-min.png" width="1994" height="940" alt="SystemBC-figure-2" typeof="foaf:Image">


              </div>
      
  </article>

Figure 2. SystemBC backconnect architecture</center>

<p>There are a couple of variants of SystemBC with some differences in the bot capabilities. All variants turn the infected computer into a SOCKS5 proxy and can perform self-updates, but the more complete variant of SystemBC also presents the following capabilities:</p>

<ul>
	<li>Support for C2 communications over TORCheck for the Emsisoft Anti-Malware process (a2guard.exe)</li>
	<li>Download and execute executable files (.exe and .dll)</li>
	<li>Download and execute shellcode</li>
	<li>Download and execute batch scripts (.cmd and .bat)</li>
	<li>Download and execute PowerShell scripts (.ps1)</li>
	<li>Download and execute Visual Basic scripts (.vbs)</li>
</ul>

<p>Several threat actors leveraged SystemBC to maintain a foothold within a company's network and launch additional post-exploitation tools. In some panels, we could find tasks showing that SystemBC was used to push <a href="https://www.cobaltstrike.com/">CobaltStrike</a> [2] and <a href="https://github.com/nettitude/PoshC2/">PoshC2</a> [3].</p>

<center>
<article class="media media--type-image media--view-mode-original">
  
      
            <div class="field field--name-field-image field--type-image field--label-hidden field__item">
                            <img loading="lazy" src="/sites/default/files/2022/09/19/SystemBC-figure-3-min.png" width="1982" height="570" alt="SystemBC-figure-3" typeof="foaf:Image">


              </div>
      
  </article>

Figure 3. Tasks issued to an infected machine</center>

<center>
<article class="media media--type-image media--view-mode-original">
  
      
            <div class="field field--name-field-image field--type-image field--label-hidden field__item">
                            <img loading="lazy" src="/sites/default/files/2022/09/19/SystemBC-figure-4-min.png" width="1976" height="548" alt="SystemBC-figure-4" typeof="foaf:Image">


              </div>
      
  </article>

Figure 4. Tasks issued to an infected machine</center>

<p>When the downloaded file is a DLL, SystemBC maps it into its own memory space and creates a new thread to execute the entry point meaning that the DLL file does not touch the disk. Similarly, to download and run files containing shellcode, SystemBC allocates memory inside its process memory and creates a new thread to start the execution.</p>

<p>All other types of files are downloaded and saved to disk under the <font face="courier">C:\Windows\Temp directory</font> with a random name and executed using scheduled tasks. The tasks registered to run the PowerShell files will launch <font face="courier">powershell.exe</font> with the following command line:<br>
<font face="courier">-WindowStyle Hidden -ep bypass -file “&lt;DOWNLOADED POWERSHELL FILE&gt;”.</font></p>

<h2>Infections telemetry</h2>

<p>While the usage of this malware has declined among threat actors, our telemetry via sinkholes and active command and control (C2) servers still reveals a significant number of infected systems. Since early August, Bitsight has observed over 56,000 unique IP addresses infected with SystemBC.</p>

<center>
<article class="media media--type-image media--view-mode-original">
  
      
            <div class="field field--name-field-image field--type-image field--label-hidden field__item">
                            <img loading="lazy" src="/sites/default/files/2022/09/19/SystemBC-figure-5-min.png" width="1664" height="1178" alt="SystemBC-figure-5" typeof="foaf:Image">


              </div>
      
  </article>

Figure 5. Daily infections observed by Bitsight</center>

<p>The geographic distribution reveals the wide impact of SystemBC around the world. Countries like India, Indonesia, Pakistan, Thailand, Brazil, United States, Egypt, Philippines, Algeria and Turkey are in the top 10 of the most affected countries, representing around 58% of the total number of infections observed.</p>

<center>
<article class="media media--type-image media--view-mode-original">
  
      
            <div class="field field--name-field-image field--type-image field--label-hidden field__item">
                            <img loading="lazy" src="/sites/default/files/2022/09/19/SystemBC-figure-6-min.png" width="1999" height="1130" alt="SystemBC-figure-6" typeof="foaf:Image">


              </div>
      
  </article>

Figure 6. Geographic distribution of infected systems</center>

<h2>Conclusion</h2>

<p>SystemBC is an interesting piece of malware that can be leveraged differently depending on the threat actor's goals. Sometimes infected computers are only used to send traffic, but they can also be instructed to download and run post-exploitation tools such as CobaltStrike. While the usage of this malware has decreased compared to the previous years, we continue to see significant numbers of victims all over the world.</p>
