          <h1>A Closer Look at the RobbinHood Ransomware</h1>
          <div class="cz-news-story-title-section">
            <div class="cz-news-title-left-area">
                            By <h6><a rel="author" href="https://www.bleepingcomputer.com/author/lawrence-abrams/" class="author"><span itemprop="author" itemscope itemtype="https://schema.org/Person"><span itemprop="name">Lawrence Abrams</span></span></a></h6>
            </div>
	    <meta itemprop="articleSection" content="Security"><div class="cz-news-title-right-area">
              <ul><li class="cz-news-date">April 26, 2019</li>
                <li class="cz-news-time">01:45 PM</li>
                
                <li class="cz-news-comment"><a href="#comments">5</a></li>
              </ul></div>
          </div>
	  <div class="articleBody">
	   
<p style="text-align:center"><img alt="RobbinHood" height="450" src="https://www.bleepstatic.com/content/hl-images/2019/04/26/robbinhood-header.jpg" width="1280"></p>

<p>The RobbinHood&nbsp;Ransomware is the latest player in the ransomware scene that is targeting companies and the computers on their network. This ransomware is not being distributed through spam but rather through other methods, which could include hacked remote desktop services or other Trojans that provide access to the attackers.</p>

<p>Since it first came out, samples of the RobbinHood ransomware have not been easy to come by. Yesterday, though, <a href="https://twitter.com/malwrhunterteam" rel="nofollow" target="_blank">MalwareHunterTeam</a> was able to find a sample so that it could be reverse engineered and tested to learn more about it.</p>

<h2>Taking a look at RobbinHood</h2>

<p>As we previously stated, it has not been confirmed how the ransomware gains access to a network and the computer's on it.&nbsp;</p>

<p>Security researcher&nbsp;<a href="https://twitter.com/VK_Intel" rel="nofollow" target="_blank">Vitali Kremez</a>, who <a href="https://twitter.com/VK_Intel/status/1121440931759128576" rel="nofollow" target="_blank">reverse engineered the sample</a>,&nbsp;told BleepingComputer that on execution,&nbsp;RobbinHood disconnects all network shares from the computer using the following command:</p>

<pre>
<code>cmd.exe /c net use * /DELETE /Y
</code></pre>

<p>This means that each computer is targeted individually and that other computers are not encrypted via connected shares. Kremez&nbsp;told us that this could indicate&nbsp;that the payload is being pushed to each individual machine via a domain controller or through a framework like Empire PowerShell and PSExec.</p>

<p class="cat_quote">"One of the most notable ones is "cmd.exe /c net use * /DELETE /Y" since the malware does not encrypt or crawl any shares and actually disconnects from network, which indicates each variant is likely pushed into each machine via the domain controller or some other automated means (maybe via psexec)"</p>

<p>Before continuing, the ransomware will now attempt to read a public RSA encryption key from C:\Windows\Temp\pub.key. If this key is not present, it will display the following message and the ransomware will exit.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="Can't find pub.key error" height="330" src="https://www.bleepstatic.com/images/news/ransomware/r/robbinhood/closer-look/needs-key-message.jpg" width="677"><figcaption><strong>Can't find pub.key error</strong></figcaption></figure></div>

<p>If a key is present, it will&nbsp;continue preparing the victim's computer for encryption. To test the ransomware, BleepingComputer generated a test public key and saved it to C:\Windows\Temp.</p>

<p>Next&nbsp;it will stop 181 Windows services associated with antivirus, database, mail server, and other software that could keep files open and prevent their encryption. It does this by issuing the "sc.exe stop" command as shown below.</p>

<pre>
<code>cmd.exe /c sc.exe stop AVP /y
</code></pre>

<p>A full list of services stopped by RobbinHood are found at the end of the article.</p>

<p>During this preparation stage, RobbinHood will also clear Shadow Volume Copies, clear event logs, and disable the Windows automatic repair by executing the following commands:</p>

<pre>
<code>vssadmin.exe delete shadows /all /quiet
WMIC shadowcopy delete
wevtutil.exe cl Application
wevtutil.exe cl Security
wevtutil.exe cl System
Bcdedit.exe /set {default} recoveryenabled no
Bcdedit.exe /set {default} bootstatuspolicy ignoreallfailures
</code></pre>

<p>Now that the computer is prepped, it begins to encrypt the victim's targeted files.</p>

<p>Kremez&nbsp;told BleepingComputer that when encrypting files an AES key is created for each file. The ransomware will then encrypt the&nbsp;AES key and the original filename with the public RSA encryption key and append it to the encrypted file.&nbsp;</p>

<p>Each encrypted file will then be renamed using the format <strong>Encrypted_[randomstring].enc_robbinhood</strong>&nbsp;as shown below.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="Encrypted RobbinHood Files" height="601" src="https://www.bleepstatic.com/images/news/ransomware/r/robbinhood/closer-look/encrypted-files.jpg" width="981"><figcaption><strong>Encrypted RobbinHood Files</strong></figcaption></figure></div>

<p>When encrypting files, RobbinHood will skip any files found in or under the following directories:</p>

<pre>
<code>ProgramData
Windows
bootmgr
Boot
$WINDOWS.~BT
Windows.old
Temp
tmp
Program Files
Program Files (x86)
AppData
$Recycle.bin
System Volume Information</code></pre>

<p>While running, RobbinHood has the ability to send debug output to the console. This feature is currently disabled in distributed versions of the ransomware and does not have a runtime value to enable it.</p>

<p>The ransomware will, though, create numerous log files under the C:\Windows\Temp folder. These files are called <strong>rf_,</strong> <strong>ro_l</strong>, and <strong>ro_s</strong>.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="Log Files" height="410" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" width="868" data-src="https://www.bleepstatic.com/images/news/ransomware/r/robbinhood/closer-look/log-files.jpg" class="b-lazy"><figcaption><strong>Log Files</strong></figcaption></figure></div>

<p>It is not currently known what each log file is for other than the rf_s file, which is used to log the creation of ransom notes in each folder.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="Example logfile for RobbinHood ransom note creation" height="537" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" width="844" data-src="https://www.bleepstatic.com/images/news/ransomware/r/robbinhood/closer-look/ransom-note-logs.jpg" class="b-lazy"><figcaption><strong>Example logfile for RobbinHood ransom note creation</strong></figcaption></figure></div>

<p>After encryption has been completed, these log files will be deleted.&nbsp; Below is an example of some of the debug messages that would be displayed during this cleanup stage if console output was enabled.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="Cleaning up Logs" height="374" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" width="1442" data-src="https://www.bleepstatic.com/images/news/ransomware/r/robbinhood/closer-look/clean-up.png" class="b-lazy"><figcaption><strong>Cleaning up Logs</strong></figcaption></figure></div>

<p>Furthermore, if console output is enabled in the ransomware, when done encrypting a computer it will display a final message stating "Enjoy buddy :)))" as shown below.&nbsp;</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="Final message when RobbinHood is done encrypting" height="196" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" width="658" data-src="https://www.bleepstatic.com/images/news/ransomware/r/robbinhood/closer-look/last-message.jpg" class="b-lazy"><figcaption><strong>Final message when RobbinHood is done encrypting</strong></figcaption></figure></div>

<p>&nbsp;</p>

<p>While&nbsp;encrypting the computer it will also create four different ransom note named&nbsp;<strong>_Decrypt_Files.html</strong>, <strong>_Decryption_ReadMe.html</strong>, <strong>_Help_Help_Help.html</strong>, and <strong>_Help_Important.html.</strong></p>

<p>These ransom notes contains information as to what has happened to the victims files and a bitcoin address that they can use to make a ransom payment. The ransom payments are currently set at 3 bitcoins per affected system or 13&nbsp;bitcoins for the entire network.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="RobbinHood Ransom Note" height="829" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" width="1275" data-src="https://www.bleepstatic.com/images/news/ransomware/r/robbinhood/closer-look/ransom-note.jpg" class="b-lazy"><figcaption><strong>RobbinHood Ransom Note</strong></figcaption></figure></div>

<p>Unfortunately, at this time no weakness has been found in the ransomware and there is no way to decrypt files for free.</p>

<h2>Protecting yourself from the RobbinHood Ransomware</h2>

<p>As ransomware is only damaging if you have no way of recovering your data, the most important thing is to always have a reliable backup of your files. These backups should be stored offline and not made accessible to ransomware, which have been <a href="https://www.bleepingcomputer.com/news/security/zenis-ransomware-encrypts-your-data-and-deletes-your-backups/" target="_blank" rel="nofollow noopener">known to target backups</a> in the past.</p>

<p>While this ransomware is not being spread via spam, it is possible that it is being installed by Trojans that are. Therefore, it is important that all users be trained on how to properly identify malicious spam and to not open any attachments without first confirming who and why they were sent.</p>

<p>Finally, it also important to make sure that your network does not make Remote Desktop Services publicly accessible via the Internet. Instead, you should put it behind a firewall and make it only accessible through a VPN.</p>

function cz_strip_tags(input, allowed) {
  allowed = (((allowed || '') + '')
    .toLowerCase()
    .match(/<[a-z][a-z0-9]*>/g) || [])
    .join(''); // making sure the allowed arg is a string containing only tags in lowercase (<a><b><c>)
  var tags = /<\/?([a-z][a-z0-9]*)\b[^>]*>/gi,
    commentsAndPhpTags = /<!--[\s\S]*?-->|<\?(?:php)?[\s\S]*?\?>/gi;
  return input.replace(commentsAndPhpTags, '')
    .replace(tags, function($0, $1) {
      return allowed.indexOf('<' + $1.toLowerCase() + '>') > -1 ? $0 : '';
    });
}
function cz_br2nl(str) {
	var regex = /<br\s*[\/]?>/gi;
    //var pure_str = str.replace(regex,"\n");
    var pure_str = str.replace(regex,"");
    return cz_strip_tags(pure_str,'');
}

			$('.bc_goto_top').fadeOut("slow");			
		}
	});

var prevScrollTop = 0;
$(window).scroll(function(event){
    scrollTop = $(this).scrollTop();

    if ( scrollTop < 0 ) {
        scrollTop = 0;
    }
    if ( scrollTop > $('body').height() - $(window).height() ) {
        scrollTop = $('body').height() - $(window).height();
    }

    if (scrollTop >= prevScrollTop && scrollTop) {
        $('header').addClass("nav-up");
    } else {
	if (scrollTop >=100)
	{
	 $('header').removeClass("nav-up"); 
	 main_nav_hide_timer = setTimeout("call_main_nav_hide()",5000);
	}
	else
	{
	 $('header').removeClass("nav-up");
	 clearInterval(main_nav_hide_timer);
	}
    }

    prevScrollTop = scrollTop;
});
