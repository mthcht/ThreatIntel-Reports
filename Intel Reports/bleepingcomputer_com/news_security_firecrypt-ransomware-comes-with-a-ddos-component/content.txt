<article><div class="article_section">
          <h1>FireCrypt Ransomware Comes With a DDoS Component</h1>
          <div class="cz-news-story-title-section">
            <div class="cz-news-title-left-area">
                            By <h6><a rel="author" href="https://www.bleepingcomputer.com/author/catalin-cimpanu/" class="author"><span itemprop="author" itemscope itemtype="https://schema.org/Person"><span itemprop="name">Catalin Cimpanu</span></span></a></h6>
            </div>
	    <meta itemprop="articleSection" content="Security"><div class="cz-news-title-right-area">
              <ul><li class="cz-news-date">January 4, 2017</li>
                <li class="cz-news-time">03:45 PM</li>
                
                <li class="cz-news-comment"><a href="#comments">3</a></li>
              </ul></div>
          </div>
	  <div class="articleBody">
	   
<p>A ransomware family named FireCrypt will encrypt the user's files, but also attempt to launch a very feeble DDoS attack on a URL hardcoded in its source code.</p>


<p>This threat was discovered today by <a href="https://twitter.com/malwrhunterteam" rel="nofollow" target="_blank">MalwareHunterTeam</a>. Below is an analysis of the ransomware's mode of operation, provided by MalwareHunterTeam and Bleeping Computer's Lawrence Abrams.</p>


<h2>FireCrypt comes as a ransomware building kit</h2>

<p>Malware is usually generated by compiling it from source code, or by using automated software that takes certain input parameters and outputs a customized malware payload on a per-campaign basis.</p>


<p>The latter are known in the industry as malware builders and usually come as command-line applications or GUI-based tools.</p>

<div align="center" style="width:98%; margin:0 auto; text-align:center; padding:4px; background:#f0f0f0; border:1px solid #ccc; border-radius:6px;">
 <a href="https://www.wiz.io/lp/ai-security-board-report-template?utm_source=bleepingcomputer&amp;utm_medium=display&amp;utm_campaign=FY26Q4_INB_FORM_AI-Security-Board-Report-Template&amp;sfcid=701Vh00000Wn7E1IAJ&amp;utm_term=FY27-bleepingcomputer-article-970x250&amp;utm_content=AI-Board-Report" rel="nofollow noopener" target="_blank"><img src="https://www.bleepstatic.com/c/w/ai-security-board-report-template.jpg" alt="Wiz" style="margin-top: 0px;"></a>
</div>


<p>The author of the FireCrypt ransomware uses a command-line application that automates the process of putting FireCrypt samples together, allowing him to modify basic settings without having to tinker with bulky IDEs that compile its source code.</p>


<p>FireCrypt's builder is named BleedGreen (seen below), and allows the FireCrypt author to generate a unique ransomware executable, give it a custom name, and use a personalized file icon. Compared to other ransomware builders, this is a very low-end application. Similar builders usually allow crooks to customize a wider set of options, such as the Bitcoin address where to receive payments, the ransom demand value, contact email address, and more.</p>


<p style="text-align:center"><img alt="BleedGreen builder for FireCrypt ransomware" height="342" src="https://www.bleepstatic.com/images/news/u/986406/Ransomware/FireCrypt/ransomware-builder-page-1.png" width="677"></p>


<p style="text-align:center"><img alt="BleedGreen builder for FireCrypt ransomware" height="342" src="https://www.bleepstatic.com/images/news/u/986406/Ransomware/FireCrypt/ransomware-builder-page-2.png" width="677"></p>


<p style="text-align:center"><img alt="BleedGreen builder for FireCrypt ransomware" height="342" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" width="677" data-src="https://www.bleepstatic.com/images/news/u/986406/Ransomware/FireCrypt/ransomware-builder-page-3.png" class="b-lazy"></p>


<p>The builder's role, besides disguising an EXE file under a PDF or DOC icon, is also to slightly alter the ransomware's binary, in order to generate a file with a different hash at every new compilation.</p>


<p>The technique is often used by malware developers to create so-called "polymorphic malware" that's harder to detect by standard antivirus software. According to MalwareHunterTeam, "the builder is very basic, so this shouldn't help anything against real AVs."</p>


<p>Nevertheless, this also tells us that FireCrypt author has at least some sort of experience in developing malware, and isn't your regular script kiddie that downloaded open-source ransomware from GitHub.</p>


<h2>FireCrypt infection process</h2>

<p>The FireCrypt infection process hinges on the ransomware's distributor's ability to trick users in launching the EXE file they just generated.</p>


<p>Once this happens, FireCrypt will kill the computer's Task Manager (taskmgr.exe) and begin to encrypt a list of 20 file types. FireCrypt encrypts files with the AES-256 encryption algorithm.</p>


<p>All encrypted files will have their original file name and extension appended with ".firecrypt". For example, a file named photo.png will be renamed into photo.png.firecrypt.</p>


<p style="text-align:center"><img alt="Files encrypted by FireCrypt" height="517" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" width="879" data-src="https://www.bleepstatic.com/images/news/u/986406/Ransomware/FireCrypt/encrypted-files.png" class="b-lazy"></p>


<p>Once the file encryption process ends, FireCrypt drops its ransom note on the user's Desktop.</p>


<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="FireCrypt ransom note" height="697" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" width="889" data-src="https://www.bleepstatic.com/images/news/u/986406/Ransomware/FireCrypt/ransom-note.png" class="b-lazy"><figcaption>FireCrypt ransom note</figcaption></figure></div>

<p>The ransom note is a nearly identical copy of the ransom note used by the Deadly for a Good Purpose Ransomware, <a href="https://www.bleepingcomputer.com/news/security/the-week-in-ransomware-october-14-2016-exotic-lockydump-comrade-and-more/" target="_blank" rel="nofollow noopener">discovered on October 14</a> by the same MalwareHunterTeam.</p>


<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="Deadly for a Good Purpose ransomware ransom note" height="615" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" width="1200" data-src="https://www.bleepstatic.com/images/news/columns/week-in-ransomware/10-15-16/CubA8lRWAAAXHcY%5B1%5D.jpg" class="b-lazy"><figcaption>Deadly for a Good Purpose Ransomware ransom note</figcaption></figure></div>

<p>At the time it was discovered in October 2016, the Deadly for a Good Purpose Ransomware appeared to be under development, as its source code would begin the file encryption process only if the victim's computer date were for a day in 2017 and later.</p>


<p>Compared to FireCrypt, the only difference is that the Deadly for a Good Purpose Ransomware also featured a logo at the top of the ransom note, now missing in FireCrypt. But, at a close inspection of Deadly's source code, MalwareHunterTeam was able to discover that both ransomware versions used the same email and Bitcoin addresses, showing a clear connection between the two, with FireCrypt being a rebranded version of the original Deadly for a Good Purpose Ransomware.</p>


<p style="text-align:center"><img alt="Comaprison between Deadly and FireCrypt ransomware source code" height="355" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" width="1600" data-src="https://www.bleepstatic.com/images/news/u/986406/Ransomware/FireCrypt/comparison.jpg" class="b-lazy"></p>


<h3>The DDoS function that fills your hard drive with junk files</h3>

<p>After dropping the ransom note, FireCrypt doesn't stop its malicious behavior. Its source code contains a function that continuously connects to a URL, downloads its content and saves it to disk in a file in the <em>%Temp%</em> folder, named <em>[random_chars]-[connect_number].html</em>.</p>


<p>If users aren't aware of this function, FireCrypt will quickly fill the <em>%Temp%</em> folder up with junk files.</p>


<p>Current versions of the FireCrypt ransomware will download the content of <a href="http://www.pta.gov.pk/index.php" rel="nofollow" target="_blank">http://www.pta.gov.pk/index.php</a>, which is the official portal of Pakistan's Telecommunication Authority. This URL cannot be modified using the ransomware's builder.</p>


<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="FireCrypt DDoS function" height="351" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" width="1216" data-src="https://www.bleepstatic.com/images/news/u/986406/Ransomware/FireCrypt/ddos.png" class="b-lazy"><figcaption>FireCrypt DDoS function</figcaption></figure></div>

<p>The FireCrypt author calls this feature as a "DDoSer," but this would be a stretch. The crook would have to infect thousands of victims before launching a DDoS attack large enough to cause any problems to the Authority's website.</p>


<p>Furthermore, all victims should be infected at the same time, and have their computers connected to the Internet in order to participate in the DDoS attack.</p>


<p>At the time of writing, there's no known method of recovering files encrypted with FireCrypt. Victims infected with this threat that are unable or unwilling to pay the $500 ransom demand should keep a copy of their encrypted files around, as a decrypter might be possibly released in the future.</p>


<h3>Targeted file extensions:</h3>

<pre>
<code>.txt, .jpg, .png, .doc, .docx, .csv, .sql, .mdb, .sln, .php, .asp, .aspx, .html, .htm, .csx, .psd, .aep, .mp3, .pdf, .torrent</code></pre>

<h3>Files associated with FireCrypt ransomware:</h3>

<pre>
<code>%AppData%\Microsoft\Windows\Start Menu\Programs\Startup\[random_chars].exe - Startup Executable
%Desktop%\[random_chars]-READ_ME.html - Ransom Note
%AppData%\SysWin32\files.txt - List of Encrypted Files
%Desktop%\random_chars]-filesencrypted.html - List of Encrypted Files
%Temp%\random_chars]-[connect_number].html - Files downloaded during the DDoS attack</code></pre>

<h3>Hashes associated with the FireCrypt ransomware:</h3>

function cz_strip_tags(input, allowed) {
  allowed = (((allowed || '') + '')
    .toLowerCase()
    .match(/<[a-z][a-z0-9]*>/g) || [])
    .join(''); // making sure the allowed arg is a string containing only tags in lowercase (<a><b><c>)
  var tags = /<\/?([a-z][a-z0-9]*)\b[^>]*>/gi,
    commentsAndPhpTags = /<!--[\s\S]*?-->|<\?(?:php)?[\s\S]*?\?>/gi;
  return input.replace(commentsAndPhpTags, '')
    .replace(tags, function($0, $1) {
      return allowed.indexOf('<' + $1.toLowerCase() + '>') > -1 ? $0 : '';
    });
}
function cz_br2nl(str) {
	var regex = /<br\s*[\/]?>/gi;
    //var pure_str = str.replace(regex,"\n");
    var pure_str = str.replace(regex,"");
    return cz_strip_tags(pure_str,'');
}
