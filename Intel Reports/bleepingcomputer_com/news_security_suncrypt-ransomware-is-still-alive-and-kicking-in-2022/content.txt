          <h1>SunCrypt ransomware is still alive and kicking in 2022</h1>
          <div class="cz-news-story-title-section">
            <div class="cz-news-title-left-area">
                            By <h6><a rel="author" href="https://www.bleepingcomputer.com/author/bill-toulas/" class="author"><span itemprop="author" itemscope itemtype="https://schema.org/Person"><span itemprop="name">Bill Toulas</span></span></a></h6>
            </div>
	    <meta itemprop="articleSection" content="Security"><div class="cz-news-title-right-area">
              <ul><li class="cz-news-date">March 28, 2022</li>
                <li class="cz-news-time">02:35 PM</li>
                
                <li class="cz-news-comment"><a href="#comment_form">0</a></li>
              </ul></div>
          </div>
	  <div class="articleBody">
	   
<p style="text-align:center"><img alt="sun" height="900" src="https://www.bleepstatic.com/content/hl-images/2022/03/28/sun.jpg?rand=1582336396" width="1600"></p>

<p>SunCrypt, a ransomware as service (RaaS) operation that reached prominence in mid-2020, is reportedly still active, even if barely, as its operators continue to work on giving its strain new capabilities.</p>

<p>SunCrypt was one of the early pioneers of triple extortion, including file encryption, threat to publish stolen data, and DDoS (distributed denial of service) attacks on non-paying victims.</p>

<p>Despite this and the <a href="https://www.bleepingcomputer.com/news/security/suncrypt-ransomware-shuts-down-north-carolina-school-district/" target="_blank" rel="nofollow noopener">lack of ethic-minded targeting restrictions</a> within the affiliate program, SunCrypt has failed to grow larger than a small private RaaS of a closed circle of affiliates.</p>

<p>According to a report by <a href="https://blog.minerva-labs.com/suncrypt-ransomware-gains-new-abilities-in-2022" target="_blank" rel="nofollow noopener">Minerva Labs</a>, this stagnation hasn't stopped the malware authors from working on a new and better version of their strain, which the analysts analyzed to determine what changed.</p>

<h2>New SunCrypt features</h2>

<p>The new capabilities of the 2022 SunCrypt variant include process termination, stopping services, and wiping the machine clean for ransomware execution.</p>

<p>These features have long existed on other ransomware strains, but for SunCrypt, they are very recent additions. As Minerva comments, this makes it seem like it's still in an early development phase.</p>

<p>The process termination includes resource-heavy processes that can block the encryption of open data files, such as WordPad (documents), SQLWriter (databases), and Outlook (emails).</p>

<p>The cleaning feature is activated at the end of the encryption routine, using two API calls to wipe all logs. Although one would be enough, the author probably used two for redundancy. After all the logs are erased, the ransomware deletes itself from the disk using cmd.exe.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="API calls that clear the event log" height="175" src="https://www.bleepstatic.com/images/news/u/1220909/ransomware/event-log-cleaning.jpg" width="417"><figcaption><strong>API calls that clear the event log</strong> <em>(Minerva)</em></figcaption></figure></div>

<p>One of the important&nbsp;<a href="https://www.bleepingcomputer.com/news/security/suncrypt-ransomware-sheds-light-on-the-maze-ransomware-cartel/" target="_blank" rel="nofollow noopener">old features</a>&nbsp;retained in the newest version is the use of I/O completion ports for faster encryption through process threading.</p>

<p>Also, SunCrypt continues to encrypt both local volumes and network shares, and still maintains an allowlist for the Windows directory, boot.ini, dll files, the recycle bin, and other items that render a computer inoperable if they're encrypted.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="Latest ransom note used by SunCrypt" height="600" src="https://www.bleepstatic.com/images/news/u/1220909/ransomware/ransom-note(1).jpg" width="736"><figcaption><strong>Latest ransom note used by SunCrypt</strong>&nbsp;<em>(Minerva)</em></figcaption></figure></div>

<h2>Activity and outlook</h2>

<p>According to stats from submissions to ID Ransomware, which provides a good idea of ransomware strain activity, SunCrypt is still encrypting victims but appears to have limited activity.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="SunCrypt submissions on ID Ransomware" height="502" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" width="1600" data-src="https://www.bleepstatic.com/images/news/u/1220909/Diagrams/activity.png" class="b-lazy"><figcaption><strong>SunCrypt submissions on ID Ransomware</strong></figcaption></figure></div>

<p>The group may be targeting high-value entities and keeping the ransom payment negotiations private, not drawing law enforcement attention and media coverage.</p>

<p>Minerva mentions Migros as one of SunCrypt's recent victims, which Switzerland's largest supermarket chain employing over 100,000 people.</p>

<p>In summary, SunCrypt is undoubtedly a real threat that hasn't been cracked yet, but whether or not the RaaS will grow into something more significant remains to be seen.</p>

function cz_strip_tags(input, allowed) {
  allowed = (((allowed || '') + '')
    .toLowerCase()
    .match(/<[a-z][a-z0-9]*>/g) || [])
    .join(''); // making sure the allowed arg is a string containing only tags in lowercase (<a><b><c>)
  var tags = /<\/?([a-z][a-z0-9]*)\b[^>]*>/gi,
    commentsAndPhpTags = /<!--[\s\S]*?-->|<\?(?:php)?[\s\S]*?\?>/gi;
  return input.replace(commentsAndPhpTags, '')
    .replace(tags, function($0, $1) {
      return allowed.indexOf('<' + $1.toLowerCase() + '>') > -1 ? $0 : '';
    });
}
function cz_br2nl(str) {
	var regex = /<br\s*[\/]?>/gi;
    //var pure_str = str.replace(regex,"\n");
    var pure_str = str.replace(regex,"");
    return cz_strip_tags(pure_str,'');
}
