<p align="center"><img alt="REMCOS Banner" src="/content/dam/fortinet-blog/new-images/uploads/remcos-a-new-rat-in-the-wild-2-578.png" data-style="width: 732px; height: 232px;"/></p>

<p>Remcos is <a href="https://blog.fortinet.com/2016/08/29/german-speakers-targeted-by-spam-leading-to-ozone-rat">another RAT</a> (Remote Administration Tool) that was first discovered being sold in hacking forums in the second half of 2016. Since then, it has been updated with more features, and just recently, we&rsquo;ve seen its payload being distributed in the wild for the first time.</p>

<p>This article demonstrates how this commercialized RAT is being used in an attack, and what its latest version (v1.7.3) is capable of doing. Remcos is currently being sold from $58 to $389, depending on the license period and the maximum number of masters or clients needed.</p>

<h2>Macro Executes Malware with High System Privilege</h2>

<p>We discovered that the Remcos RAT is being distributed through malicious Microsoft Office documents going by the filenames of <em>Quotation.xls </em>or<em> Quotation.doc</em>, which are most probably attached to SPAM emails. The structure and behavior of these documents are very similar to the ones that we documented in our previous <a href="https://blog.fortinet.com/2016/12/16/malicious-macro-bypasses-uac-to-elevate-privilege-for-fareit-malware">article</a>, which details a malicious document macro designed to bypass Microsoft Windows&rsquo; UAC security and execute malware with high privilege.</p>

<p><img alt="Malicious MS Office Documents" src="/content/dam/fortinet-blog/new-images/uploads/remcos-a-new-rat-in-the-wild-2-579.png" data-style="width: 1380px; height: 498px;"/></p>

<p align="center"><em><u>Figure 1: Malicious </u></em><em><u>MS Office documents</u></em></p>

<p>The affected documents contain an obfuscated macro that executes a shell command that downloads and runs the malware. Its obfuscation is simply achieved by adding garbage characters to the actual string.</p>

<p>To execute the downloaded malware with high system privilege, it utilizes an already known UAC-bypass technique. It attempts to execute it under Microsoft&rsquo;s Event Viewer (eventvwr.exe) by hijacking a registry (<em>HKCU\Software\Classes\mscfile\shell\open\command</em> ) that it queries to find the path of the Microsoft Management Console (mmc.exe).&nbsp; The Event Viewer simply executes whatever is in that path. Since the macro&rsquo;s shell command replaces the value from that registry entry to the malware&rsquo;s location, the malware is executed instead of the legitimate <em>mmc.exe.</em></p>

<p><img alt="hex dumps of the packed and unpacked server component" src="/content/dam/fortinet-blog/new-images/uploads/remcos-a-new-rat-in-the-wild-2-580.png"/></p>

<p align="center"><em><u>Figure 2: Execution of the malware from macro</u></em></p>

<p>In figure 2 we can see that when the command shell executed the downloaded malware, the integrity level was unexpectedly only set to &ldquo;Medium.&rdquo; At this point, the UAC bypass should have worked and the malware should have been executed with &ldquo;High&rdquo; integrity. So we took a closer look at the shell command and found erroneous slashes (&ldquo;\&rdquo;) in the registry path that caused the unsuccessful replacement of the registry value data. It was first thought that the technique worked, since the malware was executed with a &ldquo;High&rdquo; integrity level in the end. However, it was not executed under the Event Viewer. Since that attempt did not work, and yet the malware was still executed with &ldquo;High&rdquo; integrity level, we suspected that the malware binary itself has its own UAC-bypass technique, which was proven to be the case, as we demonstrate in the later part of this article.</p>

<h2>Multi-packed Payload Binary</h2>

<p>Remcos only includes UPX and MPRESS1 packers to compress and obfuscate its server component. In this sample, however, the attacker went further by adding another layer of custom packer on top of MPRESS1.</p>

<p align="center"><img alt="Hex dumps of the packed and unpacked server component" src="/content/dam/fortinet-blog/new-images/uploads/remcos-a-new-rat-in-the-wild-2-581.png"/></p>

<p align="center"><em><u>Figure 3: Hex dumps of the packed and unpacked server component</u></em></p>

<p>Obfuscation of the malware practically ended after the two packers. As seen in the screenshots below, the strings from the unpacked binary reveals that it&rsquo;s the server component built from the latest Remcos v1.7.3 Pro. According to their website,<em> Breaking-Security[.]Net, </em>this version was just released in Jan. 23, 2017.</p>

<p align="right"><img alt="Un-obfuscated strings identifying the remcos server component" src="/content/dam/fortinet-blog/new-images/uploads/remcos-a-new-rat-in-the-wild-2-582.png" data-style="width: 824px; height: 512px;"/></p>

<p align="center"><em><u>Figure 4: Un-obfuscated strings identifying the Remcos server component</u></em></p>

<p>Numerous commands that the server can carry out can also be seen in plain text.</p>

<p align="center"><img alt="Snippet of some commands" src="/content/dam/fortinet-blog/new-images/uploads/remcos-a-new-rat-in-the-wild-2-583.png" data-style="width: 583px; height: 558px;"/></p>

<p align="center"><em>Figure 5: Snippet of some commands</em></p>

<p>Figuring out all the commands through code analysis is tedious work. Fortunately, their website allows anyone to download a stripped down version of the Remcos client for free.</p>

<h2>Remcos v.1.7.3 and its Capabilities</h2>

<p>The Remcos Client has five main tabs with different specific functions. Although most of the parameters are disabled in the free version, we were able to simulate its client-server connection.</p>

<p>The <em>Connections</em> Tab is where all the active connections can be monitored. Each entry contains some basic information about the installed server component and the infected system. This is also the main tab for sending commands to the infected system. The image below shows the list of commands that can be executed in the infected system. It illustrates how much control the attacker can gain over an infected system. Most of them are fairly common with RAT applications, and as usual some of the commands may lean more towards intrusive spying than consented monitoring.</p>

<p><img alt="REMCOS command list" src="/content/dam/fortinet-blog/new-images/uploads/remcos-a-new-rat-in-the-wild-2-584.png"/></p>

<p align="center"><em><u>Figure 6: REMCOS command list</u></em></p>

<p><em>Automatic Tasks </em>is probably the most interesting feature of Remcos, as we haven&rsquo;t seen anything like it on other RATs. This feature configures the server component to automatically execute functions without any manual action from the client once a connection has been established. This makes it easy and convenient to create an infiltrate-exfiltrate-exit scheme without any trigger from the attacker, which is just how a common spyware or malware downloader behaves.</p>

<p align="center"><img alt="Automatic tasks tab" src="/content/dam/fortinet-blog/new-images/uploads/remcos-a-new-rat-in-the-wild-2-585.png"/></p>

<p align="center"><em><u>Figure 7: Automatic Tasks tab</u></em></p>

<p>The <em>Local Settings </em>tab consists of settings for the client side. Ports where the client machine waits for a connection from its servers are set here, together with the passwords to be used. Since Remcos uses the password for encryption, the listening port and the connecting server should have the same passwords for a successful connection. So basically, the password is used for both authentication and network traffic encryption.</p>

<p><img alt="local settings tab" src="/content/dam/fortinet-blog/new-images/uploads/remcos-a-new-rat-in-the-wild-2-586.png"/></p>

<p align="center"><em><u>Figure 8: Local Settings tab</u></em></p>

<p>Remcos uses a simple RC4 algorithm, using the password as the key to encrypt and decrypt network traffic between its client and server.</p>

<p><img alt="uses rc4 algorithm to encrypt network traffic" src="/content/dam/fortinet-blog/new-images/uploads/remcos-a-new-rat-in-the-wild-2-587.png"/></p>

<p align="center"><em><u>Figure 9: Uses RC4 algorithm to encrypt network traffic</u></em></p>

<p>The <em>Builder</em> tab is where the parameters of the created server binary can be customized. It can be divided into several sub-sections, as shown in the image below.</p>

<p><img alt="builder tab sub-sections" src="/content/dam/fortinet-blog/new-images/uploads/remcos-a-new-rat-in-the-wild-2-588.png"/></p>

<p align="center" data-style="margin-left:.5in;"><em><u>Figure 10: Builder tab sub-sections</u></em></p>

<p data-style="margin-left:.5in;"><em>Connection</em> &ndash; sets the client IP addresses and ports where the server connects to upon installation. It also allows a password to be set for authentication and encryption.</p>

<p data-style="margin-left:.5in;"><em>Installation</em> &ndash; configures the installation path, autorun registries, and a watchdog module that prevents termination of the process and deletion of its files and registries. Also included in this section is the setting for having its own UAC bypass, which we suspected to exist earlier in our article.</p>

<p data-style="margin-left:.5in;">So, it is possible that the attacker only used the document macro as a template to download and execute the binary, and never intended to use the script&rsquo;s UAC bypass since the server binary itself already has the same function. In fact, it uses the same UAC bypass technique, but this time with an added routine to revert the modified registry after gaining privilege. This is logical, because not restoring the registry can produce system errors that can cause suspicion from the user every time a .msc file needs to be opened.</p>

<p data-style="margin-left:.5in;"><em>Stealth</em> &ndash; this section dictates whether the server should appear on the system&rsquo;s tray icon. It also includes the settings for some basic anti-analysis/anti-sandbox routines and an option to hide the process through injection.</p>

<p data-style="margin-left:.5in;"><em>Keylogger</em> &ndash; this includes the usual parameters for a basic keylogger function. Interestingly enough, though, it can also provide the server component with a function to remove browser cookies and stored passwords. The hope is that that the user will have to re-type their passwords when logging in to websites and they can be captured using the keylogger.</p>

<p data-style="margin-left:.5in;"><em>Surveillance</em> &ndash; gives the server an option to take periodic screenshots of the system or when specific windows are active. It also features audio capture, which can be saved locally for later retrieval.</p>

<p><em>Build</em> &ndash; gives the option to pack the server binary using UPX and MPRESS.</p>

<p>The <em>Event Log </em>displays connection logs with the server, along with some information regarding the client&rsquo;s status (updates, ports, etc.).</p>

<p><img alt="event logs tab" src="/content/dam/fortinet-blog/new-images/uploads/remcos-a-new-rat-in-the-wild-2-589.png"/></p>

<p align="center"><em><u>Figure 11: Event Logs tab</u></em></p>

<p>The <em>About </em>tab contains acknowledgements and some promotions on other products that have been developed by an author named <em>Viotto</em>.</p>

<p><img alt="about tab" src="/content/dam/fortinet-blog/new-images/uploads/remcos-a-new-rat-in-the-wild-2-590.png"/></p>

<p align="center"><em><u>Figure 12: About tab</u></em></p>

<h2>Conclusion</h2>

<p>This article proves once again that one does not have to be an expert to launch fairly sophisticated malware attacks. More and more applications like Remcos are being released publicly, luring new perpetrators with their easy usage. And all it takes to be infected by one are a few clicks.</p>

<p>As for many RAT authors, the developer discourages malicious usage of the tool through a license ban if reported. This in most cases is nothing but a false shield to guard them liability when the thin veil of its being an administration tool is removed and it is exposed as a full-blown malware builder.</p>
