    <h1 class="text-center">Watchbog and the Importance of Patching</h1>

    <div class="text-center m-1">
        <div class="post-author">

                <a href="https://blog.talosintelligence.com/author/william-largent/">William Largent</a>

    <section class="post-content-wrapper mt-5">
        <div class="post-content">
            <p><br></p><h2 id="what-happened-cisco-incident-response-csirs-recently-responded-to-an-incident-involving-the-watchbog-cryptomining-botnet-the-attackers-were-able-to-exploit-cve-2018-1000861-to-gain-a-foothold-and-install-the-watchbog-malware-on-the-affected-systems-">What Happened? <br> Cisco Incident Response (CSIRS) recently responded to an incident involving the Watchbog <a href="https://blog.talosintelligence.com/malicious-xmr-mining/">cryptomining</a> botnet. The attackers were able to exploit <a href="https://jenkins.io/security/advisory/2018-12-05/">CVE-2018-1000861</a> to gain a foothold and install the <a href="https://www.bleepingcomputer.com/news/security/bluekeep-scanner-discovered-in-watchbog-cryptomining-malware/">Watchbog</a> malware on the affected systems.</h2><p>This Linux-based malware relied heavily on Pastebin for command and control (C2) and operated openly. CSIRS gained an accurate understanding of the attacker's intentions and abilities on a customer's network by analyzing the various Pastebins. As the investigation progressed, CSIRS identified and de-obfuscated multiple pastes using artifacts left on compromised hosts.</p><p>There were some attempts at obfuscation, such as base64 encoding URLs and Pastebins, but the attack was still relatively simple to uncover - this attacker did not practice particularly strong operational security.</p><p>The attackers behind Watchbog claimed to be providing a service by identifying security vulnerabilities and aiding the organization by exploiting said weaknesses before any "real" hackers could do so. During the investigation, Cisco IR found signs of hosts becoming a part of a separate botnet around the time of the Watchbog activity. This raises serious doubts about the "positive" intentions of this adversary. Below is a message left on a compromised system by the adversary:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-ujTm3bnl6Y4/XXkIAdu_6hI/AAAAAAAACks/aoCA1ZLep9wEg5GhKjKVvWZ1K1xeMaqeQCLcBGAsYHQ/s640/image5.png" class="kg-image" alt loading="lazy"></figure><h2 id="what-does-watchbog-do-the-watchbog-botnet-mines-monero-cryptocurrency-for-its-owners-while-researching-our-variant-we-came-across-a-post-by-alibaba-cloud-security-that-provides-some-insights-into-watchbog-this-post-coincided-with-our-findings-as-we-found-an-installation-script-that-performs-the-following-activities-">What does Watchbog do?  <br> The Watchbog botnet mines Monero cryptocurrency for its owners. While researching our variant we came across a <a href="https://www.alibabacloud.com/blog/return-of-watchbog-exploiting-jenkins-cve-2018-1000861_594798">post</a> by Alibaba Cloud Security that provides some insights into Watchbog. This post coincided with our findings as we found an installation script that performs the following activities.</h2><p>First the installation script checks for running processes matching other cryptocurrency miners. If the system was previously configured to mine cryptocurrency, the installation script would terminate their execution using the kill command:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-A77Z058CSas/XXkIGxjwNUI/AAAAAAAACkw/LU4PzjqWeuUFTzi0ie-5HiUI9aqAvbX0gCLcBGAsYHQ/s640/image19.png" class="kg-image" alt loading="lazy"></figure><p>The script then uses the touch command to determine its capability to write to various directories on the filesystem.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-hZreiq7fNuc/XXkIML9U97I/AAAAAAAACk0/3sYIcbzDfqwKO7yulMTA8cCO0AztQ9y2QCLcBGAsYHQ/s640/image10.png" class="kg-image" alt loading="lazy"></figure><p>It also checks the architecture of the system to determine if it is executing on a 32-bit or 64-bit operating system and then makes three attempts to download and install a '<a href="https://www.securityweek.com/jenkins-vulnerability-exploited-deliver-kerberods-malware">kerberods</a>' dropper using wget or curl.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-HPsroOq2qhA/XXkIQfB4zkI/AAAAAAAACk4/rsX2J_AT4QMKum9HYVpGSZllcOgdnJiDgCLcBGAsYHQ/s640/image6.png" class="kg-image" alt loading="lazy"></figure><p>Depending on permissions, the kerberods dropper is saved to one of the following directories:</p><ul><li>The current working directory</li><li>/usr/bin</li><li>/usr/libexec</li><li>/usr/local/bin</li><li>/tmp</li><li>/usr/sbin</li></ul><p>The script also retrieves the contents of a Pastebin URL containing a Monero wallet ID and mining information. CSIRS verified this as the same wallet ID as the one used by the attacker referenced in the Alibaba cloud post referenced earlier.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-h-5OOiEvD6E/XXkIVOCPzDI/AAAAAAAACk8/w64pGCY_Lu8_uETE9TIwJPdcz0U1BeqBACLcBGAsYHQ/s640/image15.png" class="kg-image" alt loading="lazy"></figure><p>Though the Pastebin URL in the previous screenshot is no longer accessible, the next step in the infection process is to download the cryptocurrency miner. We identified a script that 'kerberods' likely runs to reach out to GitHub to install the <a href="https://github.com/fireice-uk/xmr-stak">XMR-Stak</a> Monero miner.</p><p>The main part of the script checks to see if a process called 'watchbog' is running.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-VwVkn4qRI_o/XXkIaqorPsI/AAAAAAAAClA/k_fDNgqtyPcLigl06zAGz_G7r8BqDgW2wCLcBGAsYHQ/s640/image14.png" class="kg-image" alt loading="lazy"></figure><p>If the 'watchbog' process is not detected, the 'testa' or 'download' functions are called to install the version of the miner that's compatible with the host operating system and architecture and execute it to begin the mining process.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-by3LDE47GAs/XXkIfunS68I/AAAAAAAAClI/VPfDareDVrU8sqDw3zCfPrePM6I4GqaRwCLcBGAsYHQ/s640/image16.png" class="kg-image" alt loading="lazy"></figure><h3 id="-testa-function-as-previously-mentioned-the-testa-function-may-be-called-to-facilitate-the-infection-process-below-is-the-code-associated-with-this-function-this-code-is-responsible-for-writing-the-various-configuration-data-used-by-the-mining-software-the-function-declares-three-variables-and-assigns-base64-encoded-data-to-each-of-them-">'Testa' function <br> As previously mentioned, the 'testa' function may be called to facilitate the infection process. Below is the code associated with this function. This code is responsible for writing the various configuration data used by the mining software. The function declares three variables and assigns base64 encoded data to each of them.</h3><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-TiGg3jydpBE/XXkIl9UjzpI/AAAAAAAAClU/UlvIXZKQwNscT8a8rOTwe9jtw54KI046wCLcBGAsYHQ/s640/image18.png" class="kg-image" alt loading="lazy"></figure><p>The base64 encoded data is then decoded and written to various files.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-0T8gl-_prkE/XXkIqsy8W3I/AAAAAAAAClc/jxingEGh9NMTPJCXXZ5gQfhlQ9zOUZzbACLcBGAsYHQ/s640/image17.png" class="kg-image" alt loading="lazy"></figure><p>The base64 encoded values correspond to the following:</p><ul><li>St_64: This variable contains the URL of the Github repository that hosts the XMR-Stak mining client.</li><li>hXXps://github[.]com/fireice-uk/xmr-stak/releases/download/2.10.3/xmr-stak-linux-2.10.3-cpu.tar.xz</li><li>con_url: This variable contains the Pastebin URL that is used to host the configuration file for the mining client.</li><li>hXXps://pastebin[.]com/raw/YJH8sWr</li><li>Cpu_url: This variable contains an additional Pastebin URL. During our investigation the Pastebin URL was no longer accessible, but likely contains an additional configuration file to be used by the mining client.</li><li>hXXps://pastebin[.]com/raw/irzk5mSh</li><li>poo_url:This variable contains an additional Pastebin URL. During our investigation the Pastebin URL was no longer accessible, but likely contains an additional configuration file to be used by the mining client.</li><li>hXXps://pastebin[.]com/raw/aJkbTx6Y</li></ul><p>The script then starts the Watchbog process and deletes the text file after downloading the encoded Pastebins as a text file and giving it execution permissions. The following screenshot shows the configuration file that is referenced by the con_url variable in the 'testa' function.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-xSeuN5oL4aY/XXkIxPOeUEI/AAAAAAAAClk/N635X8ceQ0IdzDY7GJTclbF4rBLEwDYvQCLcBGAsYHQ/s640/image12.png" class="kg-image" alt loading="lazy"></figure><h3 id="-download-function-the-following-code-is-associated-with-the-download-function-referenced-by-the-installation-script-previously-described-similar-to-what-was-described-in-the-testa-function-it-contains-three-declared-variables-with-base64-encoded-assignments-">'download' function <br> The following code is associated with the 'download' function referenced by the installation script previously described. Similar to what was described in the 'testa' function, it contains three declared variables with base64 encoded assignments.</h3><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-aqSmprNOZXk/XXkI2aAmTJI/AAAAAAAAClo/vtEn5t-UvDUoeNnUn4OPSW9fh8iooJYPwCLcBGAsYHQ/s640/image13.png" class="kg-image" alt loading="lazy"></figure><p>These base64 encoded strings correspond to the following:</p><ul><li>mi_64: This variable contains the Github URL that hosts the XMrig monero mining client.</li><li>hXXps://github[.]com/xmrig/xmrig/releases/download/v2.14.1/xmrig-2.14.1-xenial-x64.tar.gz</li><li>mi_32: This variable contains a Pixeldrain URL. During our investigation the URL was no longer accessible.</li><li>hXXps://pixeldrain[.]com/api/file/ZuVWceWG</li><li>der_ke: This variable contains a Pastebin URL. The URL was used to host a file containing the attacker(s) Monero Wallet ID for the miner to use. This Wallet ID is used to facilitate payment to the attacker. All Monero successfully mined by clients under the attacker's control will transfer the Monero to the Wallet ID specified in this file. The same wallet is included in the Alibaba Cloud post mentioned earlier.</li><li>hXXps://pastebin[.]com/raw/hURdMBLd</li></ul><p>The download function then writes the contents retrieved from the specified URLs to various file locations. It then determines the architecture of the system and installs the appropriate mining client and executes it to initiate the mining process.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-3AAGL4Enxgc/XXkI8suvJoI/AAAAAAAAClw/wOHkv6rtfYMf9hYBUf_qJJDNon6L4ixNgCLcBGAsYHQ/s640/image2.png" class="kg-image" alt loading="lazy"></figure><p>The following screenshot contains the contents of the Monero wallet configuration associated with the der_ke variable in the 'download' function described earlier. It specifies the configuration parameters that will be used by the mining client, including the Wallet ID, mining pool URL, and other parameters that can be used to control CPU usage, logging, etc.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-f3DJaaFacSo/XXkJB_E7xYI/AAAAAAAACl0/DUEfC02BOTcFYExE4hGsDqyBd8Ek6ZuIgCLcBGAsYHQ/s640/image8.png" class="kg-image" alt loading="lazy"></figure><h2 id="lateral-movement-via-ssh-csirs-identified-that-the-adversary-was-using-ssh-to-spread-laterally-although-local-logs-were-unavailable-we-were-able-to-use-network-logs-to-gain-an-understanding-of-how-the-malware-was-spreading-as-we-viewed-the-logs-it-was-easy-to-determine-watchbog-s-lateral-movement-mechanism-because-they-were-generating-a-large-amount-of-ssh-traffic-this-could-have-been-easily-detected-using-internal-traffic-flow-monitoring-such-as-with-stealthwatch-cloud-or-other-netflow-monitoring-capability-">Lateral movement via SSH <br> CSIRS identified that the adversary was using SSH to spread laterally. Although local logs were unavailable, we were able to use network logs to gain an understanding of how the malware was spreading. As we viewed the logs, it was easy to determine Watchbog's lateral movement mechanism because they were generating a large amount of SSH traffic. This could have been easily detected using internal traffic flow monitoring, such as with StealthWatch Cloud or other netflow-monitoring capability.</h2><p>The following Bash script was used to facilitate the lateral movement process. It retrieves the contents of the known_hosts file on the infected system and then attempts to SSH into those systems. It also checks for the existence of SSH keys and leverages them to authenticate to the systems in the known_hosts file. If successful, it will retrieve the contents of the Pastebin URL previously described and initiate the infection process.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-5_xBPPopbZI/XXkJHnWUv6I/AAAAAAAACl8/s2tHmsrtZzc4gFcIFC2qNU6x8WBtCV5EACLcBGAsYHQ/s640/image3.png" class="kg-image" alt loading="lazy"></figure><h2 id="lateral-movement-via-jenkins-and-redis-servers-in-addition-to-leveraging-ssh-for-lateral-movement-the-watchbog-adversary-also-attempted-to-leverage-a-python-script-that-scans-for-open-jenkins-and-redis-ports-on-the-host-s-subnet-if-the-script-finds-any-vulnerable-servers-it-attempts-to-use-the-curl-or-wget-commands-to-retrieve-a-payload-from-pastebin-and-execute-it-on-the-target-">Lateral movement via Jenkins and Redis servers <br> In addition to leveraging SSH for lateral movement, the Watchbog adversary also attempted to leverage a Python script that scans for open Jenkins and Redis ports on the host's subnet. If the script finds any vulnerable servers, it attempts to use the curl or wget commands to retrieve a payload from Pastebin and execute it on the target.</h2><p>Based on the following string on line 71, the script targets CVE-2018-1000861, a vulnerability in the Staple web framework for versions up to Jenkins 2.138.1 or 2.145 which handles HTTP requests. It can provide attackers with RCE through particularly crafted URLs. A <a href="https://devco.re/blog/2019/01/16/hacking-Jenkins-part1-play-with-dynamic-routing-en/">post</a> by Orange Tsai shows how to exploit this vulnerability by using cross reference objects to bypass ACL policy.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-3TxIe0-nZsM/XXkJM986stI/AAAAAAAACmA/FgUyo8tO6YIEaZio5VFrJdNgEZtHrpA7gCLcBGAsYHQ/s640/image7.png" class="kg-image" alt loading="lazy"></figure><p>Though the pastes accessed in the script were no longer available, we believe the payload was the installation script for the XMR-Stak miner previously described. The following Python script is also downloaded and executed from the XMR-Stak miner script described above in a function called 'party.'</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-yYGvHmNwIzs/XXkJRnRvBjI/AAAAAAAACmI/E_GktVykecAKBOY6JBO9E0arUg6-rkkPwCLcBGAsYHQ/s640/image4.png" class="kg-image" alt loading="lazy"></figure><p>As can be seen above, the payload variable contains a base64 encoded blob which is then decoded and written to the /tmp directory and executes it. This base64 encoded blob contains a Pastebin URL (hXXps://pastebin[.]com/raw/DzgYb9mu) which was used to host the following Python script. The Python script is used to facilitate the exploitation of the aforementioned vulnerability and initiate the infection process. The following screenshots are associated with this Python script.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-ffsJuwLdgf4/XXkJXUd3YgI/AAAAAAAACmQ/H4hum_c5VLwooskH2NtFwGmRRUGVB8qjwCLcBGAsYHQ/s640/image21.png" class="kg-image" alt loading="lazy"></figure><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-e3bawStVKtA/XXkJcurJEqI/AAAAAAAACmY/kwJwp-GBGAA1fwJDVQLoIGSl0cxhp_pOQCLcBGAsYHQ/s640/image1.png" class="kg-image" alt loading="lazy"></figure><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-sMDwBEFIxqI/XXkJiHMLZlI/AAAAAAAACmg/FpktKT_qR9geHhpanXh4ElHHfWgwHu6agCLcBGAsYHQ/s640/image20.png" class="kg-image" alt loading="lazy"></figure><h2 id="persistence-watchbog-s-main-persistence-mechanism-appears-to-have-been-using-cron-jobs-below-is-the-system-function-from-the-kerberods-installation-script-which-ensures-the-dropper-will-call-out-to-pastebins-every-hour-for-new-information-the-below-screenshot-shows-the-way-that-watchbog-configures-the-cron-jobs-responsible-for-achieving-persistence-on-infected-systems-">Persistence <br> Watchbog's main persistence mechanism appears to have been using cron jobs. Below is the 'system' function from the 'kerberods' installation script which ensures the dropper will call out to Pastebins every hour for new information. The below screenshot shows the way that Watchbog configures the cron jobs responsible for achieving persistence on infected systems.</h2><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-0tfsLV-3M3U/XXkJoO47SsI/AAAAAAAACmk/Zomf6H49VL8zFtlb6NEuLWVMc2ZLlhfaQCLcBGAsYHQ/s640/image11.png" class="kg-image" alt loading="lazy"></figure><p>In a post by Renato Marinho from Morphus Labs, he mentions a very interesting way 'kerberods' achieves persistence as well. If it has root privileges, it will download and load a library into the operating system which hooks parts of Glibc to modify Glibc's behavior. The post also specifies that the hooks allow the miner to run as anyone (including root) and also obfuscates the network connection to the mining pool as well as the Redis/Jenkins server scans.</p><h2 id="covering-their-tracks-evidence-deletion-has-been-identified-in-previous-watchbog-variants-the-watchbog-variant-in-our-incident-continued-this-trend-evidence-deletion-was-performed-in-a-clear-manner-with-files-and-logs-being-deleted-or-overwritten-the-evidence-deletion-was-typically-added-to-the-end-of-a-handful-of-the-pastebin-scripts-with-the-xmr-stak-download-and-the-ssh-lateral-movement-scripts-being-prime-examples-the-loss-of-those-key-pieces-of-evidence-made-analysis-difficult-but-not-impossible-we-were-able-to-rely-upon-our-clients-centralized-logging-to-fill-in-those-holes-and-the-hosts-themselves-still-had-evidence-the-most-obvious-being-the-malware-variants-themselves-">Covering their tracks <br> Evidence deletion has been identified in previous Watchbog variants. The Watchbog variant in our incident continued this trend. Evidence deletion was performed in a clear manner with files and logs being deleted or overwritten. The evidence deletion was typically added to the end of a handful of the Pastebin scripts, with the Xmr-stak download and the SSH Lateral Movement scripts being prime examples. The loss of those key pieces of evidence made analysis difficult, but not impossible. We were able to rely upon our clients centralized logging to fill in those holes, and the hosts themselves still had evidence. The most obvious being the malware variants themselves.</h2><h2 id="conclusion-unpatched-web-applications-vulnerable-to-known-cves-are-a-major-target-for-attackers-adversaries-can-leverage-the-vulnerability-to-gain-a-foothold-into-the-web-server-and-network-environment-in-which-the-web-server-is-deployed-once-that-foothold-has-been-established-the-attacker-can-then-connect-to-their-c2-achieve-persistent-long-term-access-to-the-environment-and-spread-laterally-which-is-exactly-what-happened-in-this-case-the-best-way-to-prevent-such-activity-would-be-to-ensure-that-all-enterprise-web-applications-are-up-to-date-patching-can-cause-some-operational-gaps-and-delays-so-it-s-also-important-to-have-a-maintenance-window-and-a-test-environment-to-ensure-that-the-new-patches-do-not-cause-any-issues-identifying-cryptomining-activity-can-be-done-effectively-by-following-security-fundamentals-establish-a-baseline-for-internal-network-traffic-and-if-any-significant-deviations-occur-identify-and-investigate-them-even-if-there-is-an-existing-theory-for-the-activity-in-this-case-watchbog-generated-a-noticeable-spike-in-the-organization-s-ssh-traffic-">Conclusion  <br> Unpatched web applications vulnerable to known CVEs are a major target for attackers. Adversaries can leverage the vulnerability to gain a foothold into the web server and network environment in which the web server is deployed. Once that foothold has been established, the attacker can then connect to their C2, achieve persistent long-term access to the environment and spread laterally — which is exactly what happened in this case. The best way to prevent such activity would be to ensure that all enterprise web applications are up to date. Patching can cause some operational gaps and delays, so it’s also important to have a maintenance window and a test environment to ensure that the new patches do not cause any issues. Identifying cryptomining activity can be done effectively by following security fundamentals. Establish a baseline for internal network traffic and if any significant deviations occur, identify and investigate them. Even if there is an existing theory for the activity. In this case, Watchbog generated a noticeable spike in the organization’s SSH traffic.</h2><h2 id="coverage-intrusion-prevention-systems-such-as-snort-provide-an-effective-tool-to-detect-china-chopper-activity-due-to-specific-signatures-present-at-the-end-of-each-command-in-addition-to-intrusion-prevention-systems-it-is-advisable-to-employ-endpoint-detection-and-response-tools-edr-such-as-cisco-amp-for-endpoints-which-gives-users-the-ability-to-track-process-invocation-and-inspect-processes-try-amp-for-free-here-">Coverage Intrusion prevention systems such as <a href="https://snort.org/">SNORT®</a> provide an effective tool to detect China Chopper activity due to specific signatures present at the end of each command. In addition to intrusion prevention systems, it is advisable to employ endpoint detection and response tools (EDR) such as <a href="https://www.cisco.com/c/en/us/products/security/amp-for-endpoints/index.html">Cisco AMP for Endpoints</a>, which gives users the ability to track process invocation and inspect processes. Try AMP for free <a href="http://cisco.com/go/tryamp">here</a>.</h2><p>Additional ways our customers can detect and block these threats are listed below.</p><figure class="kg-card kg-image-card"><img loading="lazy"></figure><p>Cisco Cloud Web Security (<a href="https://www.cisco.com/c/en/us/products/security/cloud-web-security/index.html">CWS</a>) or<a
