    <h1 class="text-center">Agentic AI security: Why you need to know about autonomous agents now</h1>

    <div class="text-center m-1">
        <div class="post-author">
            <span>By </span>
                <a href="https://blog.talosintelligence.com/author/nick-biasini/">Nick Biasini</a>, 
                <a href="https://blog.talosintelligence.com/author/vitor-ventura/">Vitor Ventura</a>
        </div>

        <br/>
        <time class="post-datetime" datetime="March 11, 2026 06:00">
            Wednesday, March 11, 2026 06:00
        </time>

        <div class="m-3">
                        <a href="https://blog.talosintelligence.com/category/the-need-to-know/" class="category primary-category">
                            The Need to Know
                        </a>
                                            <a href="https://blog.talosintelligence.com/category/ai/" class="category primary-category">
                            AI
                        </a>
                            </div>
    </div>

    <section class="post-content-wrapper mt-5">
        <div class="post-content" id="post-content">
            <p>Agentic AI is making headlines&nbsp;worldwide&nbsp;for&nbsp;its&nbsp;potential force-multiplying capabilities,&nbsp;and organizations are understandably&nbsp;intrigued&nbsp;by&nbsp;how it can improve throughput and capabilities.&nbsp;However, as&nbsp;with any technological revolution,&nbsp;unforeseen issues&nbsp;are inevitable, and agentic AI is no exception.&nbsp;In organizations, these&nbsp;issues&nbsp;often arise from deploying&nbsp;personal assistants like&nbsp;OpenClaw&nbsp;or&nbsp;AI&nbsp;agents&nbsp;designed&nbsp;to&nbsp;optimize&nbsp;business and&nbsp;IT processes.&nbsp;Additionally, when personal&nbsp;assistants&nbsp;interact&nbsp;with “social networks”&nbsp;such as&nbsp;Moltbook, they introduce&nbsp;many&nbsp;hidden threats&nbsp;for&nbsp;organizations. These specific risks&nbsp;fall&nbsp;beyond the scope of this&nbsp;article, and&nbsp;will&nbsp;be&nbsp;addressed in a future blog.</p><p>This&nbsp;article will&nbsp;concentrate&nbsp;on&nbsp;agentic AI’s&nbsp;use&nbsp;within&nbsp;organizations&nbsp;and&nbsp;explore&nbsp;how&nbsp;these&nbsp;systems&nbsp;could potentially&nbsp;be used against&nbsp;them.&nbsp;There are two&nbsp;perspectives&nbsp;that must&nbsp;be&nbsp;taken&nbsp;into&nbsp;consideration when thinking about&nbsp;agentic AI:&nbsp;</p><ul><li>The perspective of&nbsp;organizations deploying&nbsp;agentic AI&nbsp;technologies to streamline their&nbsp;business&nbsp;and&nbsp;organizational processes&nbsp;</li><li>The perspective&nbsp;focused on potential&nbsp;impacts&nbsp;of&nbsp;malicious&nbsp;agentic AI&nbsp;in the future</li></ul><p>Both&nbsp;perspectives&nbsp;will be addressed, but&nbsp;let’s&nbsp;start&nbsp;with the first, which&nbsp;encompasses&nbsp;cybersecurity defense&nbsp;processes&nbsp;already&nbsp;in place,&nbsp;as well as the ways agentic AI can enhance those defenses.</p><h2 id="what-is-agentic-ai-how-can-it-benefit-organizations-and-what-are-the-dangers">What is&nbsp;agentic&nbsp;AI, how can it&nbsp;benefit&nbsp;organizations,&nbsp;and&nbsp;what are the dangers?</h2><p>At its core,&nbsp;agentic AI is an autonomous&nbsp;system&nbsp;tasked with&nbsp;an objective,&nbsp;equipped with specific&nbsp;tools&nbsp;and resources.&nbsp;This&nbsp;system&nbsp;is&nbsp;typically powered&nbsp;by&nbsp;large&nbsp;language&nbsp;models&nbsp;(LLMs)&nbsp;with&nbsp;advanced&nbsp;reasoning capabilities.&nbsp;These capabilities&nbsp;allow&nbsp;the agent&nbsp;to&nbsp;plan&nbsp;how&nbsp;to&nbsp;achieve its&nbsp;objective, implement that plan, and,&nbsp;most importantly,&nbsp;verify results and try different approaches&nbsp;if errors&nbsp;occur.&nbsp;</p><p>There are four questions an organization&nbsp;must&nbsp;ask&nbsp;when&nbsp;delegating&nbsp;a task to an AI&nbsp;agent:</p><ul><li>Traceability:&nbsp;Can I track&nbsp;all agent actions, regardless of whether&nbsp;the outcomes&nbsp;are&nbsp;global or intermediate?&nbsp;</li><li>Auditability:&nbsp;Is the task&nbsp;subject to&nbsp;regulatory&nbsp;oversight? Who is accountable&nbsp;for&nbsp;the&nbsp;outcomes&nbsp;produced&nbsp;by the agent?&nbsp;</li><li>Business&nbsp;risk management:&nbsp;Have&nbsp;I&nbsp;conducted&nbsp;a business risk assessment on the AI&nbsp;agent’s&nbsp;possible actions?&nbsp;</li><li>Cybersecurity&nbsp;threat management:&nbsp;Does the agent have&nbsp;guardrails&nbsp;to prevent&nbsp;malicious or disruptive actions&nbsp;during execution,&nbsp;regardless of its&nbsp;intent?&nbsp;</li></ul><p>AI agents can be incredibly powerful&nbsp;and&nbsp;task-oriented,&nbsp;so their actions&nbsp;must be&nbsp;scrutinized&nbsp;independently&nbsp;of intent.&nbsp;An agent may&nbsp;inadvertently&nbsp;destroy or expose data, while&nbsp;still successfully&nbsp;completing&nbsp;its task.&nbsp;&nbsp;</p><p>An AI agent needs to&nbsp;adhere to basic&nbsp;cybersecurity and risk management&nbsp;principles.&nbsp;Just&nbsp;as&nbsp;you&nbsp;wouldn’t&nbsp;hand a new&nbsp;employee&nbsp;keys to all the data in your enterprise,&nbsp;AI agent access should be&nbsp;tailored&nbsp;for&nbsp;its specific role. Following good practices like threat modeling and risk management&nbsp;provides&nbsp;a&nbsp;solid foundation&nbsp;for successfully&nbsp;deploying&nbsp;AI agents.&nbsp;The&nbsp;optimal&nbsp;approach is&nbsp;to&nbsp;apply&nbsp;existing organizational&nbsp;roles&nbsp;to AI agents&nbsp;and&nbsp;adjust&nbsp;the data access accordingly.&nbsp;&nbsp;The goal should be to ensure that the&nbsp;exposure from a&nbsp;compromised AI&nbsp;agent&nbsp;is no greater than from&nbsp;a compromised user;&nbsp;this is achievable&nbsp;only&nbsp;through strong access control.&nbsp;</p><p>AI agents are not immune to external interference or direct attacks.&nbsp;Agents can&nbsp;search the&nbsp;internet&nbsp;to&nbsp;determine&nbsp;the best actions to achieve their goals. These actions&nbsp;could&nbsp;be&nbsp;manipulated,&nbsp;leading&nbsp;the agent to&nbsp;run a tool with&nbsp;an undesired consequence.&nbsp;At the same time,&nbsp;the act&nbsp;of&nbsp;making queries to the internet&nbsp;can&nbsp;result in information leaks.</p><p>When&nbsp;addressing these kinds of issues,&nbsp;it’s&nbsp;important to recognize&nbsp;that LLMs are not deterministic in nature, meaning that the execution of an agent to solve a task may&nbsp;vary each&nbsp;time,&nbsp;even&nbsp;if&nbsp;the task is&nbsp;consistently completed. This means that the&nbsp;traditional allow/deny approach may not be enough to provide the necessary&nbsp;safety and security boundaries. It is crucial to evaluate&nbsp;the potential outcomes of an action before&nbsp;execution&nbsp;—&nbsp;not from the perspective of the&nbsp;task at hand,&nbsp;but from a safety and security&nbsp;standpoint, free from goal-related bias.&nbsp;</p><p>This&nbsp;oversight&nbsp;can be&nbsp;performed&nbsp;by a human&nbsp;operator,&nbsp;who&nbsp;authorizes&nbsp;critical steps&nbsp;in&nbsp;task resolution.&nbsp;It can&nbsp;also&nbsp;be provided by a&nbsp;separate&nbsp;model/agent&nbsp;tasked with&nbsp;evaluating&nbsp;the&nbsp;consequences&nbsp;of actions&nbsp;without&nbsp;regard to&nbsp;the overall&nbsp;objective.&nbsp;These&nbsp;evaluations&nbsp;can even be scored, triggering&nbsp;human review if a&nbsp;certain&nbsp;threshold is&nbsp;met.&nbsp;There may also be compliance requirements to track and log the&nbsp;actions&nbsp;agent actions,&nbsp;similar to those required&nbsp;for a user.&nbsp;</p><p>Just&nbsp;as&nbsp;no system&nbsp;is&nbsp;100% secure,&nbsp;no agent&nbsp;is&nbsp;100% safe,&nbsp;especially&nbsp;given&nbsp;their&nbsp;non-deterministic and&nbsp;try-error&nbsp;reasoning&nbsp;features. However, this&nbsp;is not a new challenge.&nbsp;This is a threat modeling and risk management problem,&nbsp;which organizations have been&nbsp;facing&nbsp;for several years now.&nbsp;&nbsp;</p><p>Organizations with mature cybersecurity&nbsp;practices&nbsp;model threat scenarios and&nbsp;prepare for incident response. They&nbsp;conduct&nbsp;business,&nbsp;information security,&nbsp;and cybersecurity risk evaluations&nbsp;for&nbsp;these scenarios and&nbsp;determine&nbsp;how each risk is managed.&nbsp;Using agentic&nbsp;AI should follow the same process:&nbsp;First, model threats based on&nbsp;agent&nbsp;privileges and capabilities,&nbsp;then&nbsp;evaluate&nbsp;the risks,&nbsp;and&nbsp;finally&nbsp;determine&nbsp;how to mitigate them.</p><p>Ultimately, we&nbsp;need to apply what we already know to&nbsp;this new&nbsp;context,&nbsp;drawing the&nbsp;appropriate&nbsp;parallels.</p><h2 id="near-and-not-so-far-impacts-of-malicious-agentic-ai">Near and not-so-far&nbsp;impacts&nbsp;of malicious&nbsp;agentic&nbsp;AI&nbsp;</h2><p>Agentic AI is already being used by malicious&nbsp;actors,&nbsp;as&nbsp;seen in cases&nbsp;like&nbsp;<a href="https://blog.talosintelligence.com/voidlink/" rel="noreferrer noopener"><u>VoidLink</u></a>. Nevertheless, this is just the&nbsp;tip of the iceberg, and&nbsp;defenders should&nbsp;be prepared for much more.</p><p>Agentic&nbsp;AI integration with attack frameworks&nbsp;is&nbsp;inevitable,&nbsp;and&nbsp;likely&nbsp;already&nbsp;underway;&nbsp;we just&nbsp;haven’t&nbsp;seen it yet. It may&nbsp;provide&nbsp;malicious&nbsp;operators&nbsp;with&nbsp;capabilities&nbsp;that could&nbsp;outpace defenders&nbsp;unless defenders&nbsp;also&nbsp;leverage&nbsp;agentic&nbsp;AI.&nbsp;</p><p>Our tracking of attack frameworks and&nbsp;their&nbsp;evolution&nbsp;provides clues on what the next steps may look like.</p><p>The next stage&nbsp;for these&nbsp;attack frameworks&nbsp;could&nbsp;easily be&nbsp;an agent&nbsp;that&nbsp;runs on the backend, awaiting&nbsp;operator requests. These requests&nbsp;might include&nbsp;searching for,&nbsp;compiling,&nbsp;and locally&nbsp;testing&nbsp;exploits&nbsp;for software the operator found on the&nbsp;target&nbsp;system.</p><p>But this is just the&nbsp;beginning.&nbsp;The list below illustrates&nbsp;other&nbsp;developments&nbsp;likely to&nbsp;be adopted by malicious operators:</p><ul><li>To&nbsp;accelerate operations,&nbsp;an&nbsp;agent may&nbsp;analyze&nbsp;the operator’s console and suggest actions&nbsp;based&nbsp;on console&nbsp;inputs.&nbsp;This would both&nbsp;allow the&nbsp;agent to infer the&nbsp;operator’s&nbsp;preferences and&nbsp;retain&nbsp;memories&nbsp;of&nbsp;the target environment&nbsp;— details&nbsp;the operator could&nbsp;otherwise&nbsp;miss.</li><li>More&nbsp;efficient use of an agent would&nbsp;involve the&nbsp;delegation of&nbsp;routine&nbsp;tasks, like environment&nbsp;exploration,&nbsp;system role recognition,&nbsp;and&nbsp;data exfiltration.</li><li>Eventually,&nbsp;an agent could&nbsp;be deployed&nbsp;directly&nbsp;in the&nbsp;victim&nbsp;environment&nbsp;to handle&nbsp;specific tasks,&nbsp;contacting the backend for inference.&nbsp;In this scenario,&nbsp;the operator&nbsp;simply assigns&nbsp;the agent a task and&nbsp;waits for a result, with the&nbsp;agent using&nbsp;covert channels,&nbsp;that&nbsp;don’t need to&nbsp;be&nbsp;synchronous.</li><li>The ultimate&nbsp;threat is&nbsp;a&nbsp;fully&nbsp;autonomous agent&nbsp;deployed and assigned&nbsp;a specific&nbsp;objective, using&nbsp;local inference&nbsp;and&nbsp;only contacting the backend&nbsp;upon&nbsp;task&nbsp;completion.&nbsp;Local inference&nbsp;reduces&nbsp;the risk of&nbsp;detection,&nbsp;as backend&nbsp;communications are&nbsp;kept&nbsp;to a minimum.&nbsp;Additionally,&nbsp;in long-term&nbsp;operations,&nbsp;the agent can perform tasks slowly, adapt&nbsp;its&nbsp;tactics&nbsp;from system to system, and even be&nbsp;instructed to use only&nbsp;living-off-the-land binaries (LOLBins).</li></ul><p>These scenarios&nbsp;can be adapted by defenders to&nbsp;automate threat hunting&nbsp;and&nbsp;response, but&nbsp;all&nbsp;strategies must account for&nbsp;the risks and guardrails discussed&nbsp;earlier.</p>

  function coverageView() {
    const coverageTable = document.getElementById('coverage-block');
    const sidebar = document.getElementById('side-bar');
    const postContent = document.getElementById('post-content');
    const currentWidth = window.innerWidth;  
  
    if (coverageTable && sidebar && postContent) {
      if (currentWidth < 1000) {
        // For viewport less than 1000px, append to post content
        postContent.appendChild(coverageTable);
      } else {
        // For viewport 1000px and wider, check for subscribe button
        const subscribeButton = sidebar.querySelector('.subscribe-button');
        
        if (subscribeButton) {
          // Insert coverage block after the subscribe button
          subscribeButton.insertAdjacentElement('afterend', coverageTable);
        } else {
          // No subscribe button found, prepend to top of sidebar
          sidebar.prepend(coverageTable);
        }
      }
    }
  }
  
  window.addEventListener("resize", coverageView);
  window.addEventListener("load", coverageView);
</script>
