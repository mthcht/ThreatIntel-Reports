    <h1 class="text-center">UAT-9244 targets South American telecommunication providers with three new malware implants</h1>

    <div class="text-center m-1">
        <div class="post-author">
            <span>By </span>
                <a href="https://blog.talosintelligence.com/author/asheer-malhotra/">Asheer Malhotra</a>, 
                <a href="https://blog.talosintelligence.com/author/brandon/">Brandon White</a>
        </div>

        <br/>
        <time class="post-datetime" datetime="March 5, 2026 06:00">
            Thursday, March 5, 2026 06:00
        </time>

        <div class="m-3">
                            <a href="/category/apt/" class="category primary-category">
                                APT
                            </a>
                                                    <a href="/category/malware/" class="category primary-category">
                                malware
                            </a>
                        
        </div>
    </div>

    <section class="post-content-wrapper mt-5">
        <div class="post-content" id="post-content">
            <ul><li>Cisco Talos is disclosing UAT-9244, who we assess with high confidence is a China-nexus advanced persistent threat (APT) actor closely associated with <a href="https://www.eset.com/us/about/newsroom/research/cyberespionage-attacks-by-the-china-aligned-famoussparrow-group-in-the-united-states-eset-research-discovers/">Famous Sparrow</a>.</li><li>Since 2024, UAT-9244 has targeted critical telecommunications infrastructure, including Windows and Linux-based endpoints and edge devices in South America, proliferating access via three malware implants.</li><li>The first backdoor, “TernDoor,” is a new variation of the previously disclosed, Windows-based, <a href="https://www.virusbulletin.com/uploads/pdf/conference/vb2023/slides/Slides-Unveiling-Activities-of-Tropic-Trooper.pdf">CrowDoor</a> malware.</li><li>Talos also discovered that UAT-9244 uses “PeerTime,” an ELF-based backdoor that uses the BitTorrent protocol to conduct malicious operations on an infected system.</li><li>UAT-9244’s third implant is a brute force scanner, which Talos tracks as “BruteEntry.” BruteEntry is typically installed on network edge devices, essentially converting them into mass-scanning proxy nodes, also known as <a href="https://cloud.google.com/blog/topics/threat-intelligence/china-nexus-espionage-orb-networks">Operational Relay Boxes</a> (ORBs) that attempt to brute force into SSH, Postgres, and Tomcat servers.</li></ul><hr><h2 id="introducing-terndoor-a-variant-of-crowdoor">Introducing TernDoor: A variant of CrowDoor</h2><p>UAT-9244 used dynamic-link library (DLL) side-loading to activate multiple stages of their infection chain. The actor executed “wsprint[.]exe”, a benign executable that loaded the malicious DLL-based loader “BugSplatRc64[.]dll”. The DLL reads a data file named “WSPrint[.]dll” from disk, decrypts its contents, and executes them in memory to activate TernDoor, the final payload.</p><p>TernDoor is a variant of <a href="https://www.virusbulletin.com/uploads/pdf/conference/vb2023/slides/Slides-Unveiling-Activities-of-Tropic-Trooper.pdf">CrowDoor</a>, a backdoor deployed in recent intrusions linked to China-nexus APTs such as <a href="https://www.welivesecurity.com/en/eset-research/you-will-always-remember-this-as-the-day-you-finally-caught-famoussparrow/">FamousSparrow</a> and <a href="https://www.trendmicro.com/en_us/research/24/k/breaking-down-earth-estries-persistent-ttps-in-prolonged-cyber-o.html">Earth Estries</a>. CrowDoor is a variant of SparrowDoor, another backdoor attributed to <a href="https://www.welivesecurity.com/en/eset-research/you-will-always-remember-this-as-the-day-you-finally-caught-famoussparrow/">FamousSparrow</a>. CrowDoor has also been observed in <a href="https://www.virusbulletin.com/conference/vb2023/abstracts/unveiling-activities-tropic-trooper-2023-deep-analysis-xiangoop-loader-and-entryshell-payload">previous Tropic Trooper intrusions</a>, &nbsp;indicating a close operational relationship with FamousSparrow. Based on the overlap in tooling; tactics, techniques, and procedures (TTPs); and victimology, we assess with high confidence that UAT-9244 closely overlaps with FamousSparrow and Tropic Trooper.</p><p>Although UAT-9244 and Salt Typhoon both target telecommunications service providers, Talos has not been able to verify or establish a solid connection between the two clusters.</p><h3 id="the-dll-based-loader">The DLL-based loader</h3><p>The DLL-based loader, “BugSplatRc64.dll”, will load the “WSPrint.dll” file from the current directory, which will be decoded using the key “qwiozpVngruhg123”.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2026/03/data-src-image-3f4029e0-5226-46cd-be3f-24331b99e7d4.png" class="kg-image" alt="" loading="lazy" width="619" height="843" srcset="https://blog.talosintelligence.com/content/images/size/w600/2026/03/data-src-image-3f4029e0-5226-46cd-be3f-24331b99e7d4.png 600w, https://blog.talosintelligence.com/content/images/2026/03/data-src-image-3f4029e0-5226-46cd-be3f-24331b99e7d4.png 619w"><figcaption><span style="white-space: pre-wrap;">Figure 1. DLL-based loader reading the encoded payload.</span></figcaption></figure><p>The decoded shellcode is position-independent and decodes and decompresses the final payload. The final payload is the TernDoor implant.</p><h2 id="terndoor">TernDoor</h2><p>The final shellcode consists of the TernDoor backdoor. TernDoor is a variant of CrowDoor, actively developed and used by UAT-9244 since at least November 2024. TernDoor deviates from CrowDoor in the following aspects:</p><ul><li>TernDoor consists of command codes that are different from previously disclosed variants of CrowDoor.</li><li>The TernDoor shellcode also consists of an embedded Windows driver (SYS file). The driver is encrypted using AES in the shellcode. The driver is used to suspend, resume, and terminate processes.</li></ul><h3 id="persistence">Persistence</h3><p>The TernDoor infection chain is persisted on the system using either a scheduled task or the Registry Run key.</p><p>The scheduled task is named “WSPrint” and created using the command:</p>
<!--kg-card-begin: html-->
<pre>
schtasks /create /tn WSPrint /tr "C:\ProgramData\WSPrint\WSPrint.exe" /ru "SYSTEM" /sc onstart /F
</pre>
<!--kg-card-end: html-->
<p>Furthermore, TernDoor modifies the following task-related registry keys to hide the task:</p><ul><li>Deletes HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Schedule\TaskCache\Tree\WSPrint | SD</li><li>Modifies HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Schedule\TaskCache\Tree\WSPrint | Index = from 1 to 0</li></ul><p>A Registry Run key may also be set to run the executable on user login:</p>
<!--kg-card-begin: html-->
<pre>
HKCU\Software\Microsoft\Windows\CurrentVersion\Run | Default = C:\ProgramData\WSPrint\WSPrint.exe
</pre>
<!--kg-card-end: html-->
<h3 id="command-line-switch">Command line switch</h3><p>Unlike CrowDoor, TernDoor only supports one command line switch: “-u”, passed to WSPrint.exe. This is the switch for uninstalling the malware from the system and it deletes all malware files from the operating directory, as well as terminates malicious processes.</p><h3 id="decoding-the-configuration">Decoding the configuration</h3><p>Like previous variants of CrowDoor, TernDoor also checks to ensure it has been injected into “msiexec[.]exe”. The implant decodes its configuration that can specify the following information:</p><ul><li>Command and control (C2) IP address</li><li>Number of tries to connect to the C2</li><li>C2 port number</li><li>User-Agent to use while connecting to C2 (if applicable)</li></ul><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2026/03/data-src-image-dc95599c-64b3-472e-a67c-fbbc38a6b1eb.png" class="kg-image" alt="" loading="lazy" width="936" height="246" srcset="https://blog.talosintelligence.com/content/images/size/w600/2026/03/data-src-image-dc95599c-64b3-472e-a67c-fbbc38a6b1eb.png 600w, https://blog.talosintelligence.com/content/images/2026/03/data-src-image-dc95599c-64b3-472e-a67c-fbbc38a6b1eb.png 936w" sizes="(min-width: 720px) 720px"><figcaption><span style="white-space: pre-wrap;">Figure 2. TernDoor configuration blob.</span></figcaption></figure><h3 id="terndoor-functionality">TernDoor functionality</h3><p>TernDoor’s capabilities resemble those of previously disclosed CrowDoor samples:</p><ul><li>Communicates with the C2 IP address</li><li>Creates processes and runs arbitrary commands via remote shell and independently</li><li>Reads and writes files</li><li>Collects system information such as computer and user name, IP address information, and OS bitness</li><li>Uninstalls itself from the infected system</li><li>Deploys the accompanying driver to hide malicious components and perform process management</li></ul><p>The accompanying Windows driver, WSPrint.sys, is dropped to disk and then activated using a windows service:</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2026/03/data-src-image-eb022910-8923-451d-887d-ace19d6bcf83.png" class="kg-image" alt="" loading="lazy" width="936" height="87" srcset="https://blog.talosintelligence.com/content/images/size/w600/2026/03/data-src-image-eb022910-8923-451d-887d-ace19d6bcf83.png 600w, https://blog.talosintelligence.com/content/images/2026/03/data-src-image-eb022910-8923-451d-887d-ace19d6bcf83.png 936w" sizes="(min-width: 720px) 720px"><figcaption><span style="white-space: pre-wrap;">Figure 3. Malicious driver service on the infected endpoint.</span></figcaption></figure><p>&nbsp;The driver creates a device named “\\Device\\VMTool” and symbolically links it to “\\DosDevices\\VMTool”. It can terminate, suspend, or resume processes specified by TernDoor — likely a means of evasion.</p><h3 id="terndoor-infrastructure">TernDoor infrastructure</h3><p>All the C2 IP addresses discovered by Talos were associated with the following SSL certificate on port 443:</p>

<pre>

<p>Pivoting off this certificate, Talos found an additional 18 IPs likely being used by UAT-9244. This list is provided in the indicators of compromise (IOCs) section.</p><p>One of the DLL-based loaders was also hosted on the IP “212.11.64[.]105”. On this server, we discovered a set of shell scripts and an accompanying malware family we track as “PeerTime.”</p><h2 id="peertime-uat-9244s-peer-to-peer-p2p-backdoor">PeerTime: UAT-9244's peer-to-peer (P2P) backdoor</h2><p>PeerTime is an ELF based backdoor that is compiled for a variety of architectures such as ARM, AARCH, PPC, MIPS etc., indicating that UAT-9244 can use it to infect a variety of embedded systems.</p><p>PeerTime is deployed through a shellscript that downloads the PeerTime loader ELF binary and an instrumentor binary.</p><p>The instrumentor ELF binary will check for the presence of docker on the compromised host using the commands docker and docker –q.</p><p>If docker is found, then the PeerTime loader is executed using:</p>
<!--kg-card-begin: html-->
<pre>
docker &lt;path_of_PeerTime_loader_ELF&gt;
</pre>
<!--kg-card-end: html-->
<p>The instrumentor consists of debug strings in Simplified Chinese, indicating that it is a custom binary created and deployed by Chinese-speaking threat actors:</p>
<!--kg-card-begin: html-->
<pre>
获取当前程序路径错误:     //Error retrieving current program path:
删除当前程序错误:                // Error deleting current program:
</pre>
<!--kg-card-end: html-->
<figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2026/03/UAT-9244-graphics_PeerTime_Installation.jpg" class="kg-image" alt="" loading="lazy" width="2000" height="1291" srcset="https://blog.talosintelligence.com/content/images/size/w600/2026/03/UAT-9244-graphics_PeerTime_Installation.jpg 600w, https://blog.talosintelligence.com/content/images/size/w1000/2026/03/UAT-9244-graphics_PeerTime_Installation.jpg 1000w, https://blog.talosintelligence.com/content/images/size/w1600/2026/03/UAT-9244-graphics_PeerTime_Installation.jpg 1600w, https://blog.talosintelligence.com/content/images/size/w2400/2026/03/UAT-9244-graphics_PeerTime_Installation.jpg 2400w" sizes="(min-width: 720px) 720px"><figcaption><span style="white-space: pre-wrap;">Figure 4. PeerTime installation/infection chain.</span></figcaption></figure><p>PeerTime consists of a loader that will decrypt and decompress the final PeerTime ELF payload and run it in memory. The PeerTime loader has the ability to rename its process to a benign process to evade detection.</p><p>PeerTime uses the BitTorrent protocol to obtain C2 information, download files from its peers, and execute them on the infected host. The payloads are written to disk and copied to the specified locations using BusyBox. As of now, PeerTime consists of two versions: one written in C/C++ and a newer version written in Rust.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2026/03/data-src-image-8b7ac579-0a44-403a-a61b-bda6db4561de.png" class="kg-image" alt="" loading="lazy" width="936" height="549" srcset="https://blog.talosintelligence.com/content/images/size/w600/2026/03/data-src-image-8b7ac579-0a44-403a-a61b-bda6db4561de.png 600w, https://blog.talosintelligence.com/content/images/2026/03/data-src-image-8b7ac579-0a44-403a-a61b-bda6db4561de.png 936w" sizes="(min-width: 720px) 720px"><figcaption><span style="white-space: pre-wrap;">Figure 5. PeerTime uses busybox to copy payloads.</span></figcaption></figure><p>PeerTime is also known as “angrypeer” and can be tracked in VirusTotal using the “<a href="https://www.virustotal.com/gui/search/malware_config%253Aangrypeer?type=files">malware_config:angrypeer</a>” query. Malware configurations in VirusTotal are identified using <a href="https://cloud.google.com/blog/topics/threat-intelligence/backscatter-automated-configuration-extraction">Mandiant’s/GTIG’s Backscatter tool</a>.</p><h2 id="setting-up-orbs-via-bruteentry">Setting up ORBs via BruteEntry</h2><p>Infrastructure used by UAT-9244 also hosts another set of shell scripts and payloads designed to establish compromised Linux based systems including edge devices as operational relay boxes (ORBs) that scan and brute force Tomcat, Postgres, and SSH servers.</p><p>The shell script will download two components:</p><ul><li>An instrumentor and daemon process that activates the actual brute forcer</li><li>The actual brute forcer (named BruteEntry) that obtains target IPs from the C2 server and scans the IPs</li></ul><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2026/03/UAT-9244-graphics_BruteEntry.jpg" class="kg-image" alt="" loading="lazy" width="2000" height="1291" srcset="https://blog.talosintelligence.com/content/images/size/w600/2026/03/UAT-9244-graphics_BruteEntry.jpg 600w, https://blog.talosintelligence.com/content/images/size/w1000/2026/03/UAT-9244-graphics_BruteEntry.jpg 1000w, https://blog.talosintelligence.com/content/images/size/w1600/2026/03/UAT-9244-graphics_BruteEntry.jpg 1600w, https://blog.talosintelligence.com/content/images/size/w2400/2026/03/UAT-9244-graphics_BruteEntry.jpg 2400w" sizes="(min-width: 720px) 720px"><figcaption><span style="white-space: pre-wrap;">Figure 6. BruteEntry infection chain.</span></figcaption></figure><h2 id="the-instrumentor-binary">The instrumentor binary</h2><p>The instrumentor binary is an ELF file written in GoLang. It checks if the BruteEntry is already running on the system using “pgrep”:</p>
<!--kg-card-begin: html-->
<pre>
pgrep &lt;path_to_BruteEntry&gt;
</pre>
<!--kg-card-end: html-->
<p>And then starts the brute forcer agent:</p>
<!--kg-card-begin: html-->
<pre>
./&lt;path_to_BruteEntry&gt;
</pre>
<!--kg-card-end: html-->
<h2 id="bruteentry">BruteEntry</h2><p>BruteEntry is also written in GoLang and begins by registering with the C2 server by providing it with the infected system’s IP address and computer name:</p>
<!--kg-card-begin: html-->
<pre>
{“ip”:“value”, “hostname”:“value”}
</pre>
<!--kg-card-end: html-->
<p>&nbsp;The C2 responds with a JSON that assigns an agent_id to the infected host:</p>
<!--kg-card-begin: html-->
<pre>
{“agent_id”:“value”, “server”:“value”}
</pre>
<!--kg-card-end: html-->
<p>where “server” = version string of BruteEntry such as “brute-force-server-v1.0”</p><p>&nbsp;BruteEntry will then ask the C2 for tasks to perform by sending a GET request to the C2 at the URI, where limit=1000 is the maximum number of vulnerable IPs to scan:</p>
<!--kg-card-begin: html-->
<pre>
/tasks/&lt;agent_id&gt;?limit=1000
</pre>
<!--kg-card-end: html-->
<p>The C2 responds with a JSON that consists of “tasks” containing the list of IPs to brute force:</p>
<!--kg-card-begin: html-->
<pre>
{"tasks":[
{"id":,"target":":","type":""},
{"id":,"target":":","type":""},
. . . . .
] }
</pre>
<!--kg-card-end: html-->
<p>&nbsp;The “type” field in the json defines the type of scan to conduct — either “tomcat”,“postgres”, or “ssh”.</p><p>The agent will then use a set of embedded credentials to attempt to brute force into either a Tomcat server application at the URL “https[://]&lt;IP&gt;:&lt;Port&gt;/manager/html”, or will brute force into a Postgres instance, either defined in the JSON (&lt;IP&gt;&lt;Port&gt;) from the C2 or using the port 5432 if no port is specified.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2026/03/data-src-image-6c6500bc-3344-418c-b909-9b240f4cd14c.png" class="kg-image" alt="" loading="lazy" width="936" height="930" srcset="https://blog.talosintelligence.com/content/images/size/w600/2026/03/data-src-image-6c6500bc-3344-418c-b909-9b240f4cd14c.png 600w, https://blog.talosintelligence.com/content/images/2026/03/data-src-image-6c6500bc-3344-418c-b909-9b240f4cd14c.png 936w" sizes="(min-width: 720px) 720px"><figcaption><span style="white-space: pre-wrap;">Figure 7. BruteEntry selecting the type of service to brute force into.</span></figcaption></figure><p>Any successful logins are then POSTED back to the C2:</p>
<!--kg-card-begin: html-->
<pre>
{"batch":[
{"task_id":&lt;task_id&gt;,"success":&lt;true/false&gt;,"note":" &lt;notes on the task&gt;"},
{"task_id":&lt;task_id&gt;,"success":&lt;true/false&gt;,"note":" &lt;notes on the task&gt;"},
......
]}
</pre>
<!--kg-card-end: html-->
<p>&nbsp;In this instance, “success” indicates if the brute force was successful (true or false), and “notes” provides specific information on whether the brute force was successful. If the login failed, the note reads “All credentials tried.” If it succeeded, the note reads “Cracked by agent &lt;agent_id&gt; | Version &lt;agent_version&gt;”.</p><h2 id="coverage">Coverage</h2><p>The following ClamAV signatures detect and block this threat:</p><ul><li>Win.Loader.PeerTime</li><li>Win.Malware.TernDoor</li><li>Unix.Malware.BruteEntry</li><li>Txt.Malware.PeerTime</li><li>Unix.Malware.PeerTime</li></ul><p>The following SNORT® rules (SIDs) detect and block this threat: <strong>65551</strong></p><h2 id="iocs">IOCs</h2><h3 id="terndoor-loader-dll">TernDoor Loader DLL</h3>

  function coverageView() {
    const coverageTable = document.getElementById('coverage-block');
    const sidebar = document.getElementById('side-bar');
    const postContent = document.getElementById('post-content');
    const currentWidth = window.innerWidth;  
  
    if (coverageTable && sidebar && postContent) {
      if (currentWidth < 1000) {
        // For viewport less than 1000px, append to post content
        postContent.appendChild(coverageTable);
      } else {
        // For viewport 1000px and wider, check for subscribe button
        const subscribeButton = sidebar.querySelector('.subscribe-button');
        
        if (subscribeButton) {
          // Insert coverage block after the subscribe button
          subscribeButton.insertAdjacentElement('afterend', coverageTable);
        } else {
          // No subscribe button found, prepend to top of sidebar
          sidebar.prepend(coverageTable);
        }
      }
    }
  }
  
  window.addEventListener("resize", coverageView);
  window.addEventListener("load", coverageView);
</script>
