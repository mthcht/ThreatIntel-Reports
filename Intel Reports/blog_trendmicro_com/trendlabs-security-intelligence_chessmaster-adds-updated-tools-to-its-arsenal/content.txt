		<div>
			<p>Trend Micro discovered the <a href="https://blog.trendmicro.com/trendlabs-security-intelligence/chessmaster-cyber-espionage-campaign/">ChessMaster</a> campaign back in July 2017 as part of our monitoring efforts to protect our customers. At the time, we found ChessMaster targeting different sectors from the academe to media and government agencies in Japan. The threat group used a variety of attack tools and techniques to spy on their target organizations.<br />
</p>
<p>Back then, we noted that ChessMaster's sophisticated nature implied that the campaign could evolve, before finding <a href="https://blog.trendmicro.com/trendlabs-security-intelligence/chessmasters-new-strategy-evolving-tools-tactics/">changes in the tools and tactics used in the campaign</a> a few months later.  While the original campaign was comprehensive and used remote access Trojans (RATs) such as ChChes and RedLeaves, this new campaign used a new backdoor (Detected by Trend Micro as BKDR_ANEL.ZKEI) that leverages the <a href="https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2017-8759">CVE-2017-8759</a> vulnerability for its cyberespionage activities.</p>
<p>In this blog post, we analyze ChessMaster's current status, including the updated tools in its arsenal — with a particular focus on the evolution of ANEL and how it is used in the campaign.</p>

<p><b><span class="body-subhead-title">Technical Analysis</span></b></p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/03/ChessMaster0.jpg"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/03/ChessMaster0.jpg" alt=" Figure 1. Infection Chain for the current ChessMaster campaign"/></a></p>
<p style="text-align: center;">Figure 1. Infection Chain for the current ChessMaster campaign</p>
<p>ChessMaster’s current iteration starts off with the familiar phishing attacks seen in the earlier campaigns that involved the use of an email with an attached malicious document using the doc, docx, rtf, csv and msg formats. The email title and attached file name were written in Japanese and contain general business, political, and economy-themed phrases such as</p>
<ul>
<li><span class="rte-red-bullet">世界経済(World economy)</span></li>
<li><span class="rte-red-bullet">経済政策(economic policy)</span></li>
<li><span class="rte-red-bullet">予算概算要求(budget estimation request)</span></li>
<li><span class="rte-red-bullet">日米対話(Japan-US dialogue)</span></li>
<li><span class="rte-red-bullet">安倍再任(re-appointment of Prime Minister Abe)</span></li>
<li><span class="rte-red-bullet">連絡網(contact network)</span></li>
<li><span class="rte-red-bullet">職員採用案(staff recruitment plan)</span></li>
<li><span class="rte-red-bullet">会議(meeting)</span></li>
</ul>
<p>However, there is a change in the exploit document. When we tracked ChessMaster back in November, we noted that it exploited the SOAP WSDL parser vulnerability <a href="https://blog.trendmicro.com/trendlabs-security-intelligence/microsoft-office-zero-day-vulnerability-addressed-september-patch-tuesday/">CVE-2017-8759</a> (patched in September 2017) within the Microsoft .NET framework to download additional malware. While ChessMaster still uses the previous exploit, it also added more methods to its arsenal: one exploits another vulnerability, <a href="https://www.trendmicro.com/vinfo/us/security/news/vulnerabilities-and-exploits/17-year-old-ms-office-flaw-cve-2017-11882-actively-exploited-in-the-wild">CVE-2017-11882</a> (patched in November 2017), which was also exploited <a href="https://blog.trendmicro.com/trendlabs-security-intelligence/cve-2017-11882-exploited-deliver-cracked-version-loki-infostealer/">to deliver illegal versions of the Loki infostealer</a>.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/03/ChessMaster1.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/03/ChessMaster1.png" alt=" Figure 2. Exploitation of CVE-2017-11882"/></a></p>

<p>ChessMaster can utilize any of these methods to download the next malware in the chain, the open source post-exploitation tool known as “Koadic,” which the previous campaign also used. This tool is responsible for stealing information — specifically the environment information — within the target system. Koadic executes the following command:</p>
<ul>
<li>%comspec% /q /c &lt;cmd&gt; 1&gt; &lt;Output&gt; 2&gt;&amp;1</li>
</ul>

<p style="text-align: center;">Figure 8. Command added when the Koadic RAT is downloaded (use of {Variable}.shell.exec command)</p>
<p>If Koadic finds that the system is conducive to the attacker’s interests, it downloads a base64-encrypted version of the ANEL malware from the Command-and-Control (C&amp;C) server and executes it.  Encrypted ANEL is decrypted using the “certutil -docode” command. When ANEL executes, a decrypted DLL file with the filename “lena_http_dll.dll” is expanded in memory. This file contains one export function — either “crt_main” or “lena_main”</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/03/ChessMaster8.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/03/ChessMaster8.png"/></a></p>
<p style="text-align: center;">Figure 9. Base64 encoded ANEL downloaded by Koadic</p>
<p>ANEL will send the infected environment’s information to the C&amp;C server. When sending the information, ANEL encrypts the data using blowfish, XOR, and Base64-based encryption methods. The format ANEL uses to send data is similar to ChChes, but ANEL's encryption method is easier to use.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/03/ChessMaster9.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/03/ChessMaster9.png"/></a></p>
<p style="text-align: center;">Figure 10. Encryption key using blowfish</p>
<p><b></b>We initially discovered the malware known as ANEL back in September 2017. At that time, ChessMaster was using ANEL as a backdoor into the target system then injects code into svchost.exe, which then decrypts and activates the embedded backdoor. This initial version of ANEL had a hardcoded version labeled “5.0.0 beta1” that contained incomplete code. We noted that this might signify the release of a future variant. Instead of just one new variant, we discovered four different versions of ANEL:</p>
<ul>
<li><span class="rte-red-bullet">5.0.0 beta1</span></li>
<li><span class="rte-red-bullet">5.1.1 rc</span></li>
<li><span class="rte-red-bullet">5.1.2 rc1</span></li>
<li><span class="rte-red-bullet">5.2.0 rev1</span></li>
</ul>
<p>The different versions contain changes in the ANEL loader and the main ANEL DLL. The figure below shows a summary of the changes between each version:</p>
<p style="text-align: center;"><a href="https://blog.trendmicro.com/trendlabs-security-intelligence/files/2018/03/ChessMaster17-1.jpg"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/03/ChessMaster10.png" alt="Figure 10. <p style="text-align: center;">Figure 11. Summary of the changes between each version of ANEL</p>

<p>The differences shown in the table above are subtle but present. For example, the initial ANEL version, “5.0.0 beta1,” uses a different C&amp;C server compared to the other versions. Once ANEL evolved to “5.1.1 rc,” it changed its file type to an executable, while also changing the C&amp;C server. The third version we found (5.1.2 rc1) reverts to a DLL file type but retains the C&amp;C server. The fourth version of ANEL (5.2.0 rev1) changes both the export function in the expanded main ANEL DLL and uses a different C&amp;C server. Overall, we can see subtle changes, which indicate that the threat actors behind ANEL are making incremental improvements to the malware to refine it.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/03/ChessMaster11.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/03/ChessMaster11.png"/></a></p>
<p style="text-align: center;">Figure 12. Backdoor function differences between ANEL 5.0.0 beta1/5.1.1 rc/5.1.2 rc1 (left) and ANEL 5.2.0 rev1 (right)</p>
<p>Once ANEL enters the user’s system, it will download various tools that could be used for malicious purposes, including password retrieval tools as well as malicious mail services and accessibility tools that will allow it to gather information about the system. These include Getpass.exe and Mail.exe, which are password and information stealers. It also downloads the following:</p>
<ul>
<li><span class="rte-red-bullet">Accevent.exe &lt;-&gt; Microsoft Accessible Event Watcher 7.2.0.0</span></li>
<li><span class="rte-red-bullet">event.dll &lt;-&gt; the loader of ssssss.ddd, (Detected as TROJ_ANELLDR)</span></li>
<li><span class="rte-red-bullet">ssssss.ddd (lena_http.bin) &lt;-&gt; encrypted BKDR_ANEL (Detected as BKDR_ANELENC)</span></li>
</ul>
<p>These three files work together using a common technique call DLL Side-Loading or DLL Hijacking. In this scenario, accevent.exe is the primary executable, which is usually legitimate.</p>
<p>After the execution of <i>accevent.exe,</i> it loads <i>event.dll</i>, which will be placed in the same folder (so it takes loading priority), after which <i>event.dll</i> decrypts and loads the encrypted backdoor <i>ssssss.ddd</i>, which is BKDR_ANEL. When we analyzed ANEL 5.1.1 rc, encrypted ANEL 5.1.2 rc1 was downloaded and executed.</p>
<p><b><span class="body-subhead-title">Short-term mitigation</span></b></p>
<p>When the user opens the document DDEAUTO or Link Auto Update, Office will display a message. If the user clicks on the “No” button, malicious activity will not initiate.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/03/ChessMaster12.jpg"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/03/ChessMaster12.jpg"/></a></p>

<p><b><span class="body-subhead-title">Medium- to long-term mitigation</span></b></p>
<p>At first glance, it seems ChessMaster’s evolution over the past few months involves subtle changes. However, the constant addition and changing of features and attack vectors indicate that the attackers behind the campaign are unlikely to stop and are constantly looking to evolve their tools and tactics.<br />
</p>
<p>Organizations can implement various techniques and best practices to defend against targeted attacks, such as regular patching to prevent vulnerability exploitation and using tools that provide protection across different network levels. Solutions that feature behavior monitoring, <a href="/en_us/business/products/user-protection/sps/endpoint.html">application control</a>, email gateway monitoring, and intrusion/detection systems can help with this.</p>
<p>Given how cybercriminal tools, tactics and procedures are evolving, organizations will have to go beyond their typical day-to-day security requirements and find a way to preempt attacks. Thus, there is a pressing need to detect and address threats via a proactive incident response strategy. Essentially, this involves creating a remediation plan for effectively combating the threat and using round-the-clock intrusion detection and threat analysis to prevent attacks from entering the system. A proactive strategy can be much more effective for targeted attacks, as these kinds of attacks are often designed to be elusive and difficult to detect, thus the need to scope them out. A comprehensive security strategy that involves proactive incident response will need the input of both decision makers and tech-savvy personnel, as they will need to be on the same page for it to be effective.</p>
<p>In addition to implementing both mitigation techniques and proactive strategies, organizations can also strengthen their security by employing solutions such <a href="/en_us/business/products/hybrid-cloud/deep-security.html">Trend Micro™ Deep Security</a>™ and TippingPoint, which protects endpoints from threats that abuse vulnerabilities.</p>
<p>In addition, comprehensive security solutions can be used to protect organizations from attacks. These include Trend Micro endpoint solutions such as <a href="/en_us/business/products/user-protection/sps.html">Trend Micro™ Smart Protection Suites</a> and <a href="/en_us/small-business/worry-free.html">Worry-Free™ Business Security</a>, which can protect users and businesses from these threats by detecting malicious files, well as blocking all related malicious URLs. <a href="/en_us/business/products/network/advanced-threat-protection/deep-discovery-threat-intelligence-network-analytics.html">Trend Micro Deep Discovery™</a> can protect enterprises by detecting malicious attachment and URLs.</p>
<p>Trend Micro OfficeScan™ with XGen™ endpoint security infuses high-fidelity machine learning with other detection technologies and global threat intelligence for comprehensive protection against all kinds of threats. A more detailed analysis of the Command-and-Control communication flow of ANEL can be found in this <a href="https://documents.trendmicro.com/assets/technical-brief-chessmaster-adds-updated-tools-to-its-arsenal.pdf">technical brief</a>.</p>
