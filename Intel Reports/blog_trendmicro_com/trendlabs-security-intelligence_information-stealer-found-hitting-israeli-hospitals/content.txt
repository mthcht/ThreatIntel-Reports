		<div>
			<p><a href="http://blog.trendmicro.com/trendlabs-security-intelligence/rising-trend-attackers-using-lnk-files-download-malware/">The abuse of shortcut (LNK) files is steadily gaining traction</a> among cybercriminals. We’ve seen a plethora of threats that leverage malicious LNK files: from <a href="https://www.trendmicro.com/vinfo/us/threat-encyclopedia/spam/3606/zip-within-a-zip-contains-locky-ransomware">well-known ransomware</a> families, <a href="http://blog.trendmicro.com/trendlabs-security-intelligence/sham-g20-summit-email-carries-split-backdoor/">backdoors</a> typically <a href="http://blog.trendmicro.com/trendlabs-security-intelligence/blackgear-espionage-campaign-evolves-adds-japan-target-list/">deployed in targeted attacks</a>, and <a href="http://blog.trendmicro.com/trendlabs-security-intelligence/zeuszbot-and-sality-jump-on-the-lnk-exploit-bandwagon/">banking Trojans</a> to <a href="http://blog.trendmicro.com/trendlabs-security-intelligence/certificated-invoices-exploiting-lnk-extension/">spam emails</a>, even an <a href="http://blog.trendmicro.com/trendlabs-security-intelligence/worm-posing-as-a-font-file-trigger-exploits/">exploit to a LNK vulnerability</a> itself. These threats are usually exacerbated by the further <a href="https://www.trendmicro.com/vinfo/us/security/news/security-technology/security-101-the-rise-of-fileless-threats-that-abuse-powershell">abuse of legitimate tools such as PowerShell</a>, or <a href="http://blog.trendmicro.com/trendlabs-security-intelligence/autoit-used-to-spread-malware-and-toolsets/">script automation utility AutoIt</a>. It’s thus not surprising that we discovered an information stealer employing LNK files, which our sensors detected in Israeli hospitals.</p>
<p><a href="https://www.trendmicro.com/vinfo/us/security/news/cyber-attacks/healthcare-under-attack-stolen-medical-records">Healthcare is considered a cybercriminal cash cow</a>, as it can be a lucrative source of <a href="http://blog.trendmicro.com/trendlabs-security-intelligence/healthcare-underground-electronic-health-records-sale/">personally identifiable information that can be monetized</a> in underground marketplaces. Initial findings revealed that any browser-based information, e.g., login credentials, can be stolen, making the use of browser-based management systems and applications important.</p>
<p>We have observed its attempts to gain footholds in the systems and the local networks’ shared folders. Another notable aspect we’re seeing so far is the combination of worm propagation and stealth capabilities.</p>
<p>Our monitoring and analyses are still ongoing and we will update this post as we find more details about the threat. Here’s what we know so far:</p>
<p><b><i>Propagation via worm.</i></b> Initial analysis of the malware indicates it propagates via a worm. It creates copies of itself, including shortcut files, a non-malicious AutoIt executable, and a malicious AutoIt script into the affected system’s root directory, i.e., <i>C:\WinddowsUpdated\&lt;file copy&gt;.</i></p>
<p><b><i>Masquerades as a Windows updater.</i></b> The shortcut files pose as browser and Windows updaters, a web 3D creation tool, and links to the system’s Downloads and Games folder.</p>
<p><b><i>Execution via AutoIt.</i></b> AutoIt is a legitimate scripting language software/executable designed to automate tasks (i.e., macros) for several programs in Windows. However, it’s known to be abused for wrapping various remote access trojans (RAT). In this case, a legitimate AutoIt executable is used to run a secondary file that contains the malicious commands. We’ve actually seen a similar threat in the form of the IPPEDO worm (<a href="https://www.trendmicro.com/vinfo/us/threat-encyclopedia/malware/worm_ippedo.b">WORM_IPPEDO.B</a>) back in 2014.</p>
<p><b><i>It gathers system information.</i></b> The malware executes a command to retrieve system information via <i>C:\WINDOWS\system32\cmd.exe /c SystemInfo.</i></p>
<p><b><i>The LNK files are spawned on the affected machines. </i></b>The LNK files are embedded with these malicious commands:</p>
<p style="padding-left: 30.0px;"><i>cmd.exe /c start ..\WinddowsUpdateCheck\WinddowsUpdater.exe &quot;..\WinddowsUpdateCheck\WinddowsUpdater.zip&quot; &amp; exit</i></p>
<p><b><i>The threat appears to be a highly obfuscated information stealer. </i></b>The samples we are currently analyzing were highly obfuscated, with payloads hidden under layers of encryption, for instance. The packages we saw each contain malicious 4 LNK files. These LNK files will issue commands leading to AutoIt’s execution of .TNT and .EXE files. Based on the behavior we’ve observed so far, it appears it conducts browser-based information theft and records keystrokes. This actually makes sense given the sensitive nature of the information that goes through healthcare organizations.</p>
<p>As <a href="https://www.trendmicro.com/vinfo/us/security/research-and-analysis/predictions/2017">the threat landscape continues to mature and diversify</a>, the IT/system administrators and information security professionals that secure organizations should do the same. Among these countermeasures: patch and keep the system updated, enforce the principle of least privilege, secure the gateways to reduce attack surface, and implement defense in depth by arraying multilayered security mechanisms—from <a href="/en_us/business/products/user-protection/sps.html" target="_self">endpoints</a>, <a href="/en_us/business/products/network/advanced-threat-protection.html" target="_self">networks</a>, and <a href="/en_us/business/products/hybrid-cloud.html" target="_self">servers</a>.</p>
<p><b><span class="body-subhead-title">Indicators of Compromise (IoCs)</span></b></p>
<ul>
<li><span class="rte-red-bullet">01e03241c42b12381e5c3ceb11e53f6c5c6bf0fa — <a href="https://www.trendmicro.com/vinfo/us/threat-encyclopedia/malware/WORM_RETADUP.A">WORM_RETADUP.A</a></span></li>
<li><span class="rte-red-bullet">1186e8d32677f6ac86a35704c9435ccd9ffa8484 — WORM_RETADUP.A</span></li>
<li><span class="rte-red-bullet">479dcd0767653e59f2653b8d3fcddb662a728df4 — <a href="https://www.trendmicro.com/vinfo/us/threat-encyclopedia/malware/LNK_RETADUP.A">LNK_RETADUP.A</a></span></li>
<li><span class="rte-red-bullet">580ff21d0c9d8aeda2b7192b4caaccee8aba6be4 — LNK_RETADUP.A</span></li>
<li><span class="rte-red-bullet">5f32f648610202c3e994509ca0fb714370d6761d — LNK_RETADUP.A</span></li>
<li><span class="rte-red-bullet">63ac13c121e523faa7a4b871b9c2f63bea05bbff — LNK_RETADUP.A</span></li>

<li><span class="rte-red-bullet">ce1b01eccf1b71d50e0f5dd6392bf1a4e6963a99 — LNK_RETADUP.A  </span></li>
</ul>
<p><b><span class="body-subhead-title">Update as of June 29, 2017, 2:00 PM (PDT)</span></b></p>
<p>Further analysis of the threat reveals that the malware is delivered as an executable file that is bundled with an identically-named file masquerading as another file type. For example, the file named <i>WinddowsUpdater.exe</i> comes with a file named <i>WinddowsUpdater.zip</i>. While the .EXE file is a legitimate AutoIt file, the alleged .ZIP file is actually an encrypted data file that contains the actual payload. This is the reason why the above mentioned LNK files use a command line to run the executable with only one argument, which is the same the name of the payload file. Looking into its code, the malware contains the following strings, which may indicate that it attempts to gather system information of affected machines:</p>
<ul>
<li><span class="rte-red-bullet">@ComputerName</span></li>
<li><span class="rte-red-bullet">@UserName</span></li>
<li><span class="rte-red-bullet">@LogonDomain</span></li>
<li><span class="rte-red-bullet">DriveGetSerial(&quot;C:&quot;)</span></li>
<li><span class="rte-red-bullet">@IPAddress1</span></li>
<li><span class="rte-red-bullet">EnvGet(&quot;OS&quot;)  and other os related strings</span></li>
<li><span class="rte-red-bullet">@OSLang</span></li>
<li><span class="rte-red-bullet">@OSVersion</span></li>
<li><span class="rte-red-bullet">@OSBuild</span></li>
</ul>
<p>It also connects to the following domain via HTTP:</p>
<ul>
<li><span class="rte-red-bullet">hxxp://palestineop[.]com/myblog/user</span></li>
</ul>
<p>By digging further, we found out that this domain appears to have been registered in November 2016, and that there is evidence that a phishing page has been hosted in the root folder of the domain just a few days after it was registered. The said phishing page, which entices users to click on a link that will supposedly lead them to Yahoo! Mail service, will instead point to the following URL:</p>
<ul>
<li><span class="rte-red-bullet">newsofpalestine[.]com/newss/gsan</span></li>
</ul>
<p>This second URL was already down when we tried to access the page. However, we managed to find evidence that that the domain used to host some news content in the past while also storing some malware. Thus, we strongly believe that the page hosted at <i>newsofpalestine[.]com/newss/gsan</i> must have also been a phishing page designed to retrieve email credentials.</p>
<p>We will continue to update this post as we uncover more details about this threat.  </p>
<p><b><span class="body-subhead-title">Update as of June 30, 2017, 4:05 AM (PDT)</span></b></p>
<p>Additional analyses indicate the main malware to be a backdoor (<a href="https://www.trendmicro.com/vinfo/us/threat-encyclopedia/malware/WORM_RETADUP.A">WORM_RETADUP.A</a>) in the form of a worm. It’s quite unique in that most remote access Trojans/backdoors deployed in these kinds of attacks often require the help of other malicious components in order to propagate.</p>
<p>RETADUP’s backdoor routines include:</p>
<ul>
<li><span class="rte-red-bullet">Downloading files</span></li>
<li><span class="rte-red-bullet">Connecting to URLs/command and control (C&amp;C) servers</span></li>
<li><span class="rte-red-bullet">Opening command-line (cmd) to execute commands</span></li>
<li><span class="rte-red-bullet">Installing a keylogger</span></li>
<li><span class="rte-red-bullet">Taking screenshots</span></li>
<li><span class="rte-red-bullet">Extracting passwords from web browsers Mozilla Firefox, Opera, and Google Chrome</span></li>
<li><span class="rte-red-bullet">Starting, terminating, and restarting processes</span></li>
<li><span class="rte-red-bullet">Issuing <i>sleep</i> command within a specified time</span></li>
<li><span class="rte-red-bullet">Shutting down, restarting, and logging off the machine</span></li>
<li><span class="rte-red-bullet">Displaying a message in a dialogue box</span></li>
<li><span class="rte-red-bullet">Updating a copy of itself from a specific Uniform Resource Identifier (URI) location via C&amp;C communications</span></li>
<li><span class="rte-red-bullet">Re-executing a copy of itself</span></li>
</ul>
<p>RETADUP is also notable for its stealth. It has a checklist of antivirus (AV) products, script file names, analysis, forensics, and debugging tools as well as sandboxes and virtual machines. It self-destructs if any of these are detected by the malware. Its propagation routine entails dropping copies of itself in all drives, including all existing folders in removable media.</p>
<p>Interestingly, it also checks for the presence of certain LNK files related to online payment and money remittance, indicating the malware may also be stealing information from those sites:</p>
<ul>
<li><span class="rte-red-bullet">C:\WinddowsUpdateCheck\ebay.lnk</span></li>
<li><span class="rte-red-bullet">C:\WinddowsUpdateCheck\hamazon.lnk</span></li>
<li><span class="rte-red-bullet">C:\WinddowsUpdateCheck\hebay.lnk</span></li>
<li><span class="rte-red-bullet">C:\WinddowsUpdateCheck\hmoneygram.lnk</span></li>
<li><span class="rte-red-bullet">C:\WinddowsUpdateCheck\hpaypal.lnk</span></li>
<li><span class="rte-red-bullet">C:\WinddowsUpdateCheck\hpayza.lnk</span></li>
<li><span class="rte-red-bullet">C:\WinddowsUpdateCheck\hskrill.lnk</span></li>
<li><span class="rte-red-bullet">C:\WinddowsUpdateCheck\hukash.lnk</span></li>
<li><span class="rte-red-bullet">C:\WinddowsUpdateCheck\hwestern union.lnk</span></li>
<li><span class="rte-red-bullet">C:\WinddowsUpdateCheck\moneygram.lnk</span></li>
<li><span class="rte-red-bullet">C:\WinddowsUpdateCheck\paypal.lnk</span></li>
<li><span class="rte-red-bullet">C:\WinddowsUpdateCheck\skrill.lnk</span></li>
<li><span class="rte-red-bullet">C:\WinddowsUpdateCheck\ukash.lnk</span></li>
<li><span class="rte-red-bullet">C:\WinddowsUpdateCheck\western union.lnk</span></li>
</ul>
<p><b><span class="body-subhead-title">Update as of July 4, 2017, 9:50 PM PDT</span></b></p>
<p>RETADUP’s original codes resemble another malware, ROWMANTI (<a href="https://www.trendmicro.com/vinfo/us/threat-encyclopedia/malware/worm_rowmanti.b">WORM_ROWMANTI.B</a>), which emerged in 2015 sporting similar capabilities as RETADUP’s. ROWMANTI, in turn, appears to be derived from the IPPEDO worm that surfaced a year earlier.</p>
<p>Underground conversations and code exchanges also know it as “rad worm”, released as a “final pack” in 2014 and was using a <a href="http://blog.trendmicro.com/trendlabs-security-intelligence/vbs-malware-spreading-in-latin-america/">Visual Basic Script-based RAT</a> (<a href="https://www.trendmicro.com/vinfo/us/threat-encyclopedia/malware/dunihi">DUNIHI</a>) controller that was modified to support “rad-worm” (IPPEDO). We’ve seen later versions but found that they were simply re-uploads of this “final pack” with predefined C&amp;C servers rather than the default “Your Domain Here” string.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/07/retadup.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/07/retadup.png"/></a><i>Snapshot of the controller; it only works with IPPEDO because the initial communication protocol has been changed for ROWMANTI and RETADUP.</i></p>
<p>The network protocols of the three malware look alike. ROWMANTI and RETADUP’s protocols are similar, for instance, but IPPEDO—their predecessor—uses a different separator and keyword at the start of the protocol. Another notable difference is that IPPEDO’s communication is in plain text, while ROWMANTI and RETADUP’s are encoded in Base64. They also differ in their use of separators and starting string for their initial phone-home communications.</p>
<p>The C&amp;C servers used by ROWMANTI in 2015 also contain the string “rad”, which is the original name of the malware used by their developers. We also saw an underground forum post from 2015 showing a code snippet of the Domain Generating Algorithm (DGA) section of the worm (by a different malware author), but this DGA code is present only in RETADUP. This indicates that while the codes of RETADUP originated from the “rad-worm”, it also integrated codes from other malware authors.</p>
