	<h1 class="article-details__title">KillDisk Variant Hits Latin American Financial Groups</h1>
	<p class="article-details__description">Trend Micro Research came across a new variant of the disk-wiping KillDisk targeting financial organizations in Latin America. Initial analysis reveals that it may be a component of another payload, or part of a bigger attack.</p>
	<p class="article-details__author-by">By: Gilbert Sison, Rheniel Ramos, Jay Yaneza, Alfredo Oliveira
		
			<time class="article-details__date">January 15, 2018</time>

		<div>
			<p><i>Updated as of January 15, 11:58 PM PDT to clarify that the new variant of KillDisk we found does not have a ransom note.</i></p>
<p>We came across a new variant of the disk-wiping KillDisk targeting financial organizations in Latin America. Trend Micro detects it as <a href="https://www.trendmicro.com/vinfo/tmr/?/us/threat-encyclopedia/malware/TROJ_KILLDISK.IUB">TROJ_KILLDISK.IUB</a>. Trend Micro™ <a href="/en_us/business/products/network/advanced-threat-protection/deep-discovery-threat-intelligence-network-analytics.html">Deep Discovery</a>™ proactively blocks any intrusions or attacks associated with this threat. Initial analysis (which is still ongoing) reveals that it may be a component of another payload, or part of a bigger attack. We are still analyzing this new KillDisk variant and we will update this post as we uncover more details about this threat.</p>
<p>KillDisk, along with the multipurpose, cyberespionage-related <a href="https://web.archive.org/web/20160212083029/http://blog.trendmicro.com/trendlabs-security-intelligence/killdisk-and-blackenergy-are-not-just-energy-sector-threats/" target="_blank">BlackEnergy</a>, was used in cyberattacks in late December 2015 against Ukraine’s <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/cyber-attacks/first-malware-driven-power-outage-reported-in-ukraine">energy sector</a> as well as its <a href="https://www.theregister.co.uk/2016/12/15/ukraine_banks_apt/">banking</a>, rail, and <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/cyber-attacks/cyber-threats-to-the-mining-industry">mining</a> industries. The malware has since metamorphosed into a threat used for <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/research-and-analysis/predictions/2018">digital extortion</a>, affecting <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/cybercrime-and-digital-threats/ransomware-recap-dec-19-dec-31-2016">Windows</a> and <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/cybercrime-and-digital-threats/ransomware-recap-january-1-13-2017">Linux</a> platforms. The note accompanying the ransomware versions, like in the case of <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/cybercrime-and-digital-threats/frequently-asked-questions-the-petya-ransomware-outbreak">Petya</a>, was a ruse: Because KillDisk also overwrites and deletes files (and don’t store the encryption keys on disk or online), recovering the scrambled files was out of the question. The new variant we found, however, does not include a ransom note.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-1.jpg"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-1.jpg"/></a><i>Figure 1. KillDisk’s infection chain</i></p>
<p><b><span class="body-subhead-title">How is it dropped in the system?</span></b></p>
<p>This KillDisk variant looks like it is intentionally dropped by another process/attacker. Its file path is hardcoded in the malware (<i style="">c:\windows\dimens.exe</i>), which means that it is tightly coupled with its installer or is a part of a bigger package.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-2.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-2.png"/></a></p>
<p style="text-align: center;"><i>Figure 2. The new KillDisk variant’s parameter to shut down the affected machine</i></p>
<p>KillDisk also has a self-destruct process, although it isn’t really deleting itself. It renames its file to <i>c:\windows\0123456789</i> while running. This string is hardcoded in the sample we analyzed. It expects its file path to <i>be c:\windows\dimens.exe</i> (also hardcoded). Accordingly, if disk forensics is performed and <i>dimens.exe</i> is searched, the file that will be retrieved will be the newly created file with 0x00 byte content.</p>
<p><b><span class="body-subhead-title">How does it delete files?</span></b></p>
<p>This new KillDisk variant goes through all logical drives (fixed and removable) starting from drive <i>b:</i>. If the logical drive contains the system directory, the files and folders in the following directories and subdirectories are exempted from deletion:</p>
<ul>
<li><span class="rte-red-bullet">WINNT</span></li>
<li><span class="rte-red-bullet">Users</span></li>
<li><span class="rte-red-bullet">Windows</span></li>
<li><span class="rte-red-bullet">Program Files</span></li>
<li><span class="rte-red-bullet">Program Files (x86)</span></li>
<li><span class="rte-red-bullet">ProgramData</span></li>
<li><span class="rte-red-bullet">Recovery (case-sensitive check)</span></li>
<li><span class="rte-red-bullet">$Recycle.Bin</span></li>
<li><span class="rte-red-bullet">System Volume Information</span></li>
<li><span class="rte-red-bullet">old</span></li>
<li><span class="rte-red-bullet">PerfLogs</span></li>
<li><i> </i></li>
</ul>
<p>Before a file is deleted, it is first randomly renamed. KillDisk will overwrite the first 0x2800 bytes of the file and another block that’s 0x2800-bytes big with 0x00.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-3-1.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-3-1.png"/></a><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-3-2.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-3-2.png"/></a></p>
<p style="text-align: center;"><i>Figure 3. Code snippets showing how KillDisk overwrites then deletes files</i></p>
<p><b><span class="body-subhead-title">How does it wipe the disk?</span></b></p>
<p>The malware attempts to wipe \\.\PhysicalDrive0 to \\.\PhysicalDrive4. It reads the Master Boot Record (MBR) of every device it successfully opens and proceeds to overwrite the first 0x20 sectors of the device with “0x00”. It uses the information from the MBR to do further damage to the partitions it lists. If the partition it finds is not an extended one, it overwrites the first 0x10 and last sectors of the actual volume. If it finds an extended partition, it will overwrite the Extended Boot Record (EBR) along with the two extra partitions it points to.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-4-1.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-4-1.png"/></a><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-4-2.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-4-2.png"/></a><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-4-3.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-4-3.png"/></a><i>Figure 4. Code snippets showing how KillDisk reads/scans the MBR (top, center), and overwrites the EBR (bottom)</i></p>
<p><b><span class="body-subhead-title">What happens after the MBR, files, and folders are overwritten and/or deleted?</span></b></p>
<p>KillDisk has a numeric parameter that denotes the number of minutes (15 being the default) it will wait before it shuts down the affected machine. To try to reboot the machine, it will try to terminate these processes:</p>
<ul>
<li><span class="rte-red-bullet">Client/server run-time subsystem (csrss.exe)</span></li>
<li><span class="rte-red-bullet">Windows Start-Up Application (wininit.exe)</span></li>
<li><span class="rte-red-bullet">Windows Logon Application (winlogon.exe)</span></li>
<li><span class="rte-red-bullet">Local Security Authority Subsystem Service (lsass.exe)</span></li>
<li> </li>
</ul>
<p>This is done most likely to force a reboot or dupe the user into restarting the machine. Terminating csrss.exe and wininit.exe, for instance, will cause a blue screen of death (BSOD). Terminating winlogon.exe will prompt the user to log in again, while terminating lsass.exe will cause a reboot. KillDisk also uses the <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/aa376868(v=vs.85).aspx"><i>ExitWindowsEx</i></a> function to forcefully restart the machine.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-5.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/01/killdisk-5.png"/></a><i>Figure 5. Code showing KillDisk forcefully rebooting the system</i></p>
<p><b><span class="body-subhead-title">What can organizations do?</span></b></p>
<p>KillDisk’s destructive capabilities, and how it could be just a part of a bigger attack, highlight the significance of defense in depth: securing the perimeters — from gateways, endpoints, and networks to servers — to further reduce the attack surface. Here are some best practices for organizations.</p>
<ul>
<li><span class="rte-red-bullet">Keep the system and its applications updated/patched to deter attackers from exploiting security gaps; consider virtual patching for legacy systems.</span></li>
<li><span class="rte-red-bullet">Regularly <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/virtualization-and-cloud/best-practices-backing-up-data">back up data</a> and ensure its integrity.</span></li>
<li><span class="rte-red-bullet">Enforce the principle of least privilege. <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/cyber-attacks/protecting-data-through-network-segmentation">Network segmentation</a> and <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/cyber-attacks/keeping-digital-assets-safe-need-for-data-classification">data categorization</a> help prevent lateral movement and further exposure.</span></li>
<li><span class="rte-red-bullet">Deploy security mechanisms such as <a href="/en_us/business/products/user-protection/sps/endpoint.html" target="_self">application control</a>/whitelisting and behavior monitoring, which can block suspicious programs from running and thwart anomalous system modifications.</span></li>
<li><span class="rte-red-bullet">Proactively monitor the system and network; enable and employ <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/security-technology/best-practices-deploying-an-effective-firewall">firewalls</a> as well as intrusion prevention and detection systems.</span></li>
<li><span class="rte-red-bullet">Implement a managed <a href="http://blog.trendmicro.com/trendlabs-security-intelligence/four-steps-to-an-effective-targeted-attack-response/">incident response</a> policy that will drive proactive remediation strategies; further strengthen the organization’s security posture by cultivating a cybersecurity-aware workplace.</span></li>
<li> </li>
</ul>
