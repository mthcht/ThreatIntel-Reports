		<div>
			<p>A one-time password (OTP) system involves the use of a generated password that can only be used once to log in and access specific online services. Often managed by a third-party provider, this rolling password system aims to reduce unauthorized intrusions to systems via compromised accounts.</p>
<p>We found an application sample in April called TinkaOTP that seemed like a normal OTP authentication tool. However, further investigation showed the application bearing a striking resemblance to Dacls remote access trojan (RAT), a Windows and Linux backdoor that 360 Netlab <a href="https://blog.netlab.360.com/dacls-the-dual-platform-rat-en/">discovered</a> in December 2019. The strings that the trojan’s command and control (C&amp;C) server uses to communicate with the storage samples links it to previous deployments by the cybercriminal group Lazarus.</p>
<p><b><span class="body-subhead-title">The Lazarus link</span></b></p>
<p>After running the TinkaOTP application, the sample appeared as a Disk iMaGe (DMG) application bundle meant to be mounted and run. Launching the app initially didn’t reveal any suspicious behavior, but it dropped a hidden file <i>~/Library/.mina</i> in the Library folder, along with a LaunchDaemon <i>/Library/LaunchDaemons/com.aex-loop.agent.plist</i> set to run the hidden file on startup.<a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure1.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure1.png"/></a></p>
<p style="text-align: center;">Figure 1. TinkaOTP dropping a hidden file</p>
<p>Another comparison showed that the main mach-o executable of the bundle —<i>/Contents/MacOS/TinkaOTP</i> — was a repackaged version of another OTP tool called <a href="https://github.com/MinaOTP/MinaOTP-MAC">MinaOTP</a>, an open-source authentication tool available at GitHub.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Fig1.jpg"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Fig1.jpg"/></a></p>
<p style="text-align: center;">Figure 2. TinkaOTP (left) and MinaOTP (right, image taken from GitHub)</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure3.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure3.png"/></a></p>
<p style="text-align: center;">Figure 3. Disassembly of TinkaOTP’s main executable showed direct references to MinaOTP modules, as well as the shell command for copying the malicious hidden file</p>
<p>Tracing the origin of the hidden .mina file showed that it is a copy of an included resource, renamed SubMenu.nib, from the application bundle and where the main backdoor functions were contained. It also has the same links to Lazarus’ Windows and Linux predecessors: the presence of the hardcoded strings <i>c_2910.cls</i> and <i>k_3872.cls</i>. Both strings were previously used during C&amp;C communication to the domain thevagabondsatchel[.]com as the sample storage of the cybercriminal group, as reported by 360 Netlab researchers.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure4.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure4.png"/></a></p>
<p style="text-align: center;">Figure 4. Strings used to communicate with the C&amp;C</p>
<p><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure5.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure5.png"/></a></p>
<p style="text-align: center;">Figure 5. Same HTTP Post message format as Windows and Linux backdoor discovered</p>
<p>We also found another variant with an executable that drops the main backdoor payload. The SubMenu.nib is nowhere to be found, but it downloads the payload from a hardcoded address and extracts it into the hidden payload to be run. A look into the address, loneeaglerecords[.]com, revealed that the domain has been registered and existing for 11 years, but the issued HTTPS certificate is fairly recent. The domain is assigned to 50[.]87[.]144[.]227, an IP investigated to have hosted different kinds of Windows malware such as ADWIND and URSNIF. Despite the differences, the hidden payload downloaded is the same in both variants.</p>
<p><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure6.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure6.png"/></a></p>
<p style="text-align: center;">Figure 6. Payload address of another variant</p>
<p>The group appeared to have made this version as a quick follow-up to the Windows/Linux Dacls RAT variants. This was confirmed by matching the latest IP assignment and HTTP certificate from the download address of the new variant.</p>
<p><b><span class="body-subhead-title">Backdoor and persistence routines</span></b></p>
<p>The backdoor installation sequence shows that it’s meant for persistence via <i>/LaunchAgents/com.aex-loop.agent.plist </i>and <i>/Library/LaunchDaemons/com.aex-loop.agent.plist</i>. It initiates the configuration file <i>/Library/Caches/com.applestore.db</i> to set the C&amp;C server IP and for remote session information. Loading the bot plugins, this enables connection to the server to open and wait for commands, update the configuration file based on the commands received, and encrypt the file via AES CBC. If the configuration file already exists, it will decrypt once a new session starts.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure7.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure7.png"/></a></p>
<p style="text-align: center;">Figure 7. Related code disassembly for persistence</p>
<p>Once installed, it checks if an existing configuration file exists. If not, it creates a config file and writes data related to the setup such as C&amp;C server addresses and other C&amp;C session information. The initial C&amp;C server IPs — 67[.]43[.]239[.]146 and 185[.]62[.]58[.]207 — the bot connects to are hardcoded in the backdoor file and written to the config file. The config file will then be dropped as <i>/Library/Caches/com.applestore.db</i> and encrypted. Once completed, the backdoor proceeds to load the plugins responsible for specific functions.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure8.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure8.png"/></a></p>
<p style="text-align: center;">Figure 8. Initial server IPs written on the config file. The file may have the IPs rewritten to an updated IP address of a new server.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure9.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure9.png"/></a></p>
<p style="text-align: center;">Figure 9. Dropping the config file and encrypting with AES CBC</p>
<p>A look into the server IPs also showed that the domain hosts legitimate traffic; blocking the said domains may have side effects for other sites.</p>
<p><b><span class="body-subhead-title">Attack routines upon infection</span></b></p>
<p>During the initial analysis of the disassembled codes, the strings used to load the IPs and plugins appear to be missing characters. However, a closer inspection of the first characters showed that these strings were intentionally separated — likely an effort to evade detection.</p>
<p><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure10.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure10.png"/></a></p>
<p style="text-align: center;">Figure 10. Evading detection</p>
<p>After receiving a response from the server, it will check the headers to determine the plugins to run, with each plugin templated to load their respective functions on initialization. Comparing the plugins used for MacOS and its Windows/Linux variants showed some differences and additional capabilities.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure11.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure11.png"/></a></p>
<p style="text-align: center;">Figure 11. Scanning headers</p>

<p>The <i>bash/cmd</i> plugin is used for executing shell commands in the form of appending them as a bash script and then running it on the terminal. Based on the arguments set in the packet received, it also has the capability to run a reverse shell.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure12.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure12.png"/></a></p>
<p style="text-align: center;">Figure 12. The bash/cmd plugin</p>
<p>The <i>file</i> plugin runs the same functions as the Windows and Linux variants: it can read, write, delete, and download files, as well as scan a directory for a specific file.</p>
<p><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure13.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure13.png"/></a></p>
<p style="text-align: center;">Figure 13. The file plugin</p>
<p>Looking into the arguments shows that the <i>process</i> plugin comes from the received packet to execute functions such as collecting process information, running a new process, and terminating a running one. The process information collected includes the username, user ID, group ID, and process parent ID of the target process.</p>
<p>Whereas the other plugins may directly call and execute for the arguments’ function passed by the server, the process plugin differs in that the server indirectly calls the function from the plugin itself; the location of the process plugin functions’ addresses are first called prior to the execution.</p>
<p><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure14.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure14.png"/></a></p>
<p style="text-align: center;">Figure 14. The process plugin</p>
<p><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure15.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure15.png"/></a></p>
<p style="text-align: center;">Figure 15. Process plugin formatting the collected information before sending</p>
<p>The <i>test</i> plugin attempts to connect to a provided address to check access to the network. Meanwhile, the <i>reverse P2P</i> plugin creates a proxy server to bridge the C&amp;C and the client. This creates another connection to another C&amp;C specified in the commands to act as a proxy, redirecting traffic from the infected machine to the real C&amp;C server.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure16.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure16.png"/></a></p>
<p style="text-align: center;">Figure 16. The test plugin</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure17.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure17.png"/></a></p>
<p style="text-align: center;">Figure 17. The reverse P2P plugin</p>
<p>The <i>logsend</i> plugin collects system information by scanning the system using function <i>start_scan_worm</i> and send data to the log server specified. Meanwhile, the <i>socks</i> plugin creates a connection via socks4 for Secure Sockets Layer-related (SSL) transactions. This plugin was not observed in the Windows/Linux variants.</p>
<p><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure18.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure18.png"/></a></p>
<p style="text-align: center;">Figure 18. The logsend plugin</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure_19a.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure_19a.png"/></a></p>
<p><a href="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure_19b.png"><img src="/content/dam/trendmicro/global/en/migrated/trendlabs-security-intelligence-07/files/2020/05/Lazarus-macos-dacls-rat-backdoor_Figure_19b.png"/></a></p>
<p style="text-align: center;">Figure 19. The socks plugin</p>
<p><b><span class="body-subhead-title">Conclusion</span></b></p>
<p>The discovery of the Windows/Linux variants and analysis of this routine has shown Lazarus’ range of expertise. As mentioned in a previous Lazarus <a href="https://blog.trendmicro.com/trendlabs-security-intelligence/mac-backdoor-linked-to-lazarus-targets-korean-users/">discovery</a> involving MacOS, this shift in focus towards attacking multiple operating systems indicate an expansion of targets. It also shows that they’re experimenting for future-related cases, highlighted by the additional plugin that has not been observed in similar routines.</p>
<p>To note, shortly after our first discovery of Lazarus’ interest in MacOS in 2019 via poisoned spreadsheets, the group was able to follow up with fileless <a href="https://securelist.com/operation-applejeus-sequel/95596/">AppleJeus</a>, showing a rapid development in research. Related to this routine, the dropper from the first variant’s only implemented evasion method was the use of copying the payload into a hidden file; the second variant improved on this by downloading the payload instead. While considerably not as sophisticated yet, given how the AppleJeus malware quickly followed with a fileless version, we could expect the group to deploy a similar execution of this routine soon.</p>
<p>We also suspect that the group may be targeting specific users for bot distribution, taking advantage of users’ needs for layered security. OTP authentication tools have also been used to manage cryptocurrency wallets and exchanges, another common target for fraud by the cybercriminal group. The group might also be expanding to mobile platforms. Considering that the original MinaOTP source contains a separate project called <a href="https://github.com/MinaOTP/MinaOTP-iOS">MinaOTP-iOS</a>, the group might be planning to rebuild a repurposed version of this for mobile. We will continue monitoring Lazarus deployments and activities.</p>
<p>To protect systems from this type of threat, users should only download apps from official and legitimate marketplaces. Users can also consider multilayered security solutions such as <a href="/en_us/forHome/products/antivirus-for-mac.html">Trend Micro Antivirus for Mac</a>, which provides comprehensive security and multi-device protection against cyberthreats.</p>
<p>Enterprises can take advantage of Trend Micro’s <a href="/en_us/business/products/user-protection/sps.html">Smart Protection Suites</a> with XGen™ security, which infuses high-fidelity <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/definition/machine-learning">machine learning</a> into a blend of threat protection techniques to eliminate security gaps across any user activity or endpoint.  </p>
