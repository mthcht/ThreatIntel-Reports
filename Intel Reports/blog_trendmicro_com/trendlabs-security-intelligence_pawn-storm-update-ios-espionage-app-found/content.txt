	<h1 class="article-details__title">Pawn Storm Update: iOS Espionage App Found</h1>
	<p class="article-details__description">In our continued research on Operation Pawn Storm, we found one interesting poisoned pawn—spyware specifically designed for espionage on iOS devices.</p>
	<p class="article-details__author-by">By: Feike Hacquebord, Fernando Merces
		
			<time class="article-details__date">February 04, 2015</time>
		
		
		<span>Read time:&nbsp;</span><span class="eta"></span> (<span class="words"></span> words)

            <div class="richText">
	
    


	
		<div>
			<p><i>Updated February 6, 2015, 10:30 AM PST</i> <a href="/en_us/forHome/products/mobile-security.html">Trend Micro™ Mobile Security</a> protects users' iOS devices and stops threats before they reach them. Trend Micro Mobile Security offers protection and detects these malware using the cloud-based <a href="/en_us/business/capabilities/solutions-for/cloud.html">Smart Protection Network™</a> and Mobile App Reputation technology.</p>
<p><i>Updated February 11, 2015, 7:52 PM PST</i> In a previous version of this blog posting, we stated that the iOS device doesn’t have to be jailbroken per se for the malware to be installed. We revisited this finding and found that the iOS device indeed needs to be jailbroken. The exact way how the actors install the espionage malware on iOS devices is currently unknown to us. It is very likely that social engineering is an important part.</p>
<p>In our continued research on Operation Pawn Storm, we found one interesting poisoned pawn—spyware specifically designed for espionage on iOS devices. While spyware targeting Apple users is highly notable by itself, this particular spyware is also involved in a targeted attack.</p>
<h4><b><span class="body-subhead-title">Background of Operation Pawn Storm</span></b></h4>
<p><a href="http://www.trendmicro.com/vinfo/tmr/?/us/security/news/cyber-attacks/pawn-storm-espionage-attacks-use-decoys-deliver-sednit">Operation Pawn Storm</a> is an active economic and political cyber-espionage operation that targets a wide range of entities, like the military, governments, defense industries, and the media. The actors of Pawn Storm tend to first move a lot of pawns in the hopes they come close to their actual, high profile targets. When they finally successfully infect a high profile target, they might decide to move their next pawn forward: advanced espionage malware.</p>
<p>The iOS malware we found is among those advanced malware. We believe the iOS malware gets installed on already compromised systems, and it is very similar to next stage SEDNIT malware we have found for Microsoft Windows’ systems. We found two malicious iOS applications in Operation Pawn Storm. One is called <i>XAgent</i> (detected as IOS_XAGENT.A) and the other one uses the name of a legitimate iOS game, <i>MadCap </i>(detected as IOS_ XAGENT.B). After analysis, we concluded that both are applications related to SEDNIT. The obvious goal of the SEDNIT-related spyware is to steal personal data, record audio, make screenshots, and send them to a remote command-and-control (C&amp;C) server. As of this publishing, the C&amp;C server contacted by the iOS malware is live.</p>
<p><b><span class="body-subhead-title">Analysis of XAgent</span></b></p>
<p>The XAgent app is fully functional malware. After being installed on iOS 7, the app’s icon is hidden and it runs in the background immediately. When we try to terminate it by killing the process, it will restart almost immediately. Installing the malware into an iOS 8 device yields different results. The icon is not hidden and it also cannot restart automatically. This suggests that the malware was designed prior to the release of iOS 8 last September 2014.</p>
<p><b><span class="body-subhead-title">Data Theft Capabilities</span></b></p>
<p>The app is designed to collect all kind of information on an iOS device. It is able to perform the following routines:</p>
<ul>
<li><span class="rte-red-bullet">Collect text messages</span></li>
<li><span class="rte-red-bullet">Get contact lists</span></li>
<li><span class="rte-red-bullet">Get pictures</span></li>
<li><span class="rte-red-bullet">Collect geo-location data</span></li>
<li><span class="rte-red-bullet">Start voice recording</span></li>
<li><span class="rte-red-bullet">Get a list of installed apps</span></li>
<li><span class="rte-red-bullet">Get a list of processes</span></li>
<li><span class="rte-red-bullet">Get the Wi-Fi status</span></li>
</ul>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2015/01/pwnstrm13.png"/></p>
<p style="text-align: center;"><i>Figure 1. XAgent code structure</i></p>
<p><b><span class="body-subhead-title">C&amp;C Communication</span></b></p>
<p>Besides collecting information from the iOS device, the app sends the information out via HTTP. It uses POST request to send messages, and GET request to receive commands.</p>
<p><b>Formatted Log Messages</b></p>
<p>The malware’s log messages are written in HTML and color coded, making it easier for human operators to read. Error messages tend to be in red, while others are in green as shown in the figure below.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2015/01/pwnstrm1.png"/></p>
<p style="text-align: center;"><i>Figure 2. Color-coded HTML log messages</i></p>
<p><b>A Well-Designed Code Structure</b></p>
<p>We can see that the code structure of the malware is very organized. The malware looks carefully maintained and consistently updated.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2015/01/pwnstrm2.png"/></p>
<p style="text-align: center;"><i>Figure 3. XAgent code structure</i></p>
<p>The app uses the commands <i>watch</i>, <i>search</i>, <i>find</i>, <i>results,</i> <i>open, </i>and<i> close.</i></p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2015/01/pwnstrm3.png"/></p>
<p style="text-align: center;"><i>Figure 4. List of base URIs</i></p>
<p><b>Randomly Generated URI</b></p>
<p>The full uniform resource identifier (URI) for C&amp;C HTTP requests is randomly generated, according to a template agreed upon with the C&amp;C server. The base URI can be seen in Figure 4, and parameters are chosen from the list below and appended to the base URI.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2015/01/pwnstrm4.png"/></p>
<p style="text-align: center;"><i>Figure 5. List of parameters used with URIs</i></p>
<p>Here are corresponding implementations we got during our reversing:</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2015/01/pwnstrm5.png"/><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2015/01/pwnstrm6.png"/></p>
<p style="text-align: center;"><i>Figures 6 and 7. Code for URI generation</i></p>
<p><b>Token Format and Encoding</b></p>
<p>The malware uses a token to identify which module is communicating. The token is Base64 encoded data, but padded with a 5-byte random prefix so that it looks like valid Base64 data. See the first line “ai=” part in the figure below.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2015/01/pwnstrm7-1.png"/></p>
<p style="text-align: center;"><i>Figure 8. Client (XAgent) request</i></p>
<p>Reverse engineering also revealed additional communication functions.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2015/01/pwnstrm11.png"/></p>
<p style="text-align: center;"><i>Figure 9. HTTP communication functions</i></p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2015/01/pwnstrm11.png"/></p>
<p style="text-align: center;"><i>Figure 10. C2 server</i></p>
<p><b><span class="body-subhead-title">FTP Communication</span></b></p>
<p>The app is also able to upload files via FTP protocol.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2015/01/pwnstrm8.png"/></p>
<p style="text-align: center;"><i>Figure 11. FTP communication functions</i></p>
<p><span class="body-subhead-title"><b>Analysis of &quot;MadCap&quot;</b> &quot;</span></p>
<p>Madcap&quot; is similar to the XAgent malware, but the former is focused on recording audio. &quot;Madcap&quot; can only be installed on jailbroken devices.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2015/01/pwnstrm9.png"/></p>
<p style="text-align: center;"><i>Figure 12. Code structure of Madcap</i></p>
<p><b><span class="body-subhead-title">Possible Infection Methods</span></b></p>
<p>The exact methods of installing these malware is unknown. As far as we can tell the iOS device has to be jailbroken to install the Xagent malware. However we have seen one instance wherein a lure involving XAgent simply says “Tap Here to Install the Application.&quot;</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2015/01/pwnstrm10.png"/></p>
<p style="text-align: center;"><i>Figure 13. Site used in downloading XAgent</i></p>
<div><div id="ColorBandedcontent"><div id="imcontent"><div>It is good to note that it is still possible to install the malicious app into non-jailbroken devices if the app is signed  using Apple's enterprise certificate. Another possible scenario is infecting an iPhone after connecting it to a compromised or infected Windows laptop via a USB cable.</div>
</div>
