		<div>
			<p style="text-align: left;">Trickbot, which used to be a simple banking trojan, has come a long way. Over time, we’ve seen how cybercriminals continue to add more features to this malware. Last March, Trickbot <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/cybercrime-and-digital-threats/evolving-trickbot-adds-detection-evasion-and-screen-locking-features">added a new module</a> that gave it increased detection evasion and a screen-locking feature. This month, we saw that Trickbot (detected by Trend Micro as <a href="https://www.trendmicro.com/vinfo/tmr/?/us/threat-encyclopedia/malware/TSPY_TRICKBOT.THOIBEAI">TSPY_TRICKBOT.THOIBEAI</a>) now has a password grabber module (pwgrab32) that steals access from several applications and browsers, such as Microsoft Outlook, Filezilla, WinSCP, Google Chrome, Mozilla Firefox, Internet Explorer, and Microsoft Edge. Based on our telemetry, we saw that this Trickbot variant has affected users mainly in the United States, Canada, and the Philippines.</p>
<p style="text-align: left;"><b><span class="body-subhead-title">Analyzing Trickbot’s modules</span></b></p>
<p style="text-align: left;">Malware authors continue to cash in on Trickbot’s modular structure — its ability to continually update itself by downloading new modules from a C&amp;C server and change its configuration make for a malware that’s ripe for updating. To gain a better understanding of this threat, we analyzed Trickbot’s different modules, starting with the new <i>pwgrab32</i> module that we saw this month.<br />
<i><u><br />
 </u><b>pwgrab32 module</b></i></p>
<p style="text-align: left;">Trickbot’s new module, called pwgrab32 or PasswordGrabber, steals credentials from applications such as Filezilla, Microsoft Outlook, and WinSCP.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/10/fig1.jpg"/></p>
<p style="text-align: center;">Figure 1. A screen capture of Trickbot’s new module, pwgrab32, in an affected system</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/10/fig2.jpg"/></p>
<p style="text-align: center;">Figure 2. A screen capture of the new module’s code that steals FTP passwords from FileZilla</p>
<p><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/10/fig3.jpg"/></p>
<p style="text-align: center;">Figure 3. A screen capture of the new module’s code that steals Microsoft Outlook credentials</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/10/fig4.jpg"/></p>
<p style="text-align: center;">Figure 4. A screen capture of Trickbot harvesting passwords from open-source FTP WinSCP</p>
<p>Aside from stealing credentials from applications, it also steals the following information from several popular web browsers such as Google Chrome, Mozilla Firefox, Internet Explorer, and Microsoft Edge:</p>
<ul>
<li><span class="rte-red-bullet">Usernames and Passwords</span></li>
<li><span class="rte-red-bullet">Internet Cookies</span></li>
<li><span class="rte-red-bullet">Browsing History</span></li>
<li><span class="rte-red-bullet">Autofills</span></li>
<li><span class="rte-red-bullet">HTTP Posts</span></li>
</ul>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/10/fig5.jpg"/></p>
<p style="text-align: center;">Figure 5. A screen capture of Trickbot’s code that is structured to steal passwords from popular web browsers</p>
<p>It should be noted that this Trickbot variant is not capable of stealing passwords from third-party password manager applications. We are studying this malware further to see if it is able to steal passwords from password managers that have browser plugins.<br />
<i><u><br />
 </u><b>shareDll32 module</b></i><br />
<b></b><br />
Trickbot uses the shareDll32 module to help propagate itself throughout the network. It connects to a C&amp;C server http[:]//185[.]251[.]39[.]251/radiance[.]png to download a copy of itself and save it as setuplog.tmp.<br />
<img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/10/fig6.jpg"/></p>
<p style="text-align: center;">Figure 6. Trickbot’s shareDll32 module allows it to connect to a C&amp;C server to download a copy of itself</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/10/fig7.jpg"/></p>
<p style="text-align: center;">Figure 7. The downloaded file is saved as <i>setuplog.tmp</i></p>
<p style="text-align: left;">The shareDll32 module then enumerates and identifies systems connected on the same domain using <i>WNetEnumResource </i>and <i>GetComputerNameW</i>.<img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/10/fig8.jpg"/></p>
<p style="text-align: center;">Figure 8. Screen capture of code that enumerates and identifies connected systems using WNetEnumResourceW and GetComputerNameW</p>
<p style="text-align: left;">The file <i>setuplog.tmp</i> is then copied in the administrative shares of the discovered machines or systems.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/10/fig9.jpg"/></p>
<p style="text-align: center;">Figure 9. A screenshot of setuplog.tmp copied in the administrative shares</p>
<p>To make the malware more persistent, it has an auto-start service that allows Trickbot to run whenever the machine boots. This service can have the following display names:</p>
<ul>
<li><span class="rte-red-bullet">Service Techno</span></li>
<li><span class="rte-red-bullet">Service_Techno2</span></li>
<li><span class="rte-red-bullet">Technics-service2</span></li>
<li><span class="rte-red-bullet">Technoservices</span></li>
<li><span class="rte-red-bullet">Advanced-Technic-Service</span></li>
<li><span class="rte-red-bullet">ServiceTechno5</span><br />
<br />
<i style="font-weight: 300;"><u><br />
 </u><b>wormDll module</b></i><br />
</li>
</ul>
<p><br />
The wormDll32 module attempts to identify servers and domain controllers in the network using NetServerEnum and LDAP queries. Trickbot’s worm-like propagation capability was first <a href="https://www.flashpoint-intel.com/blog/new-version-trickbot-adds-worm-propagation-module/" target="_blank">observed</a> by security researchers from Flashpoint in 2017.<img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/10/fig10.jpg"/></p>
<p style="text-align: center;">Figure 10. Screen capture of code that identifies workstations and servers in a domain using NetServerEnum</p>
<p><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/10/fig11-1024x639.jpg"/></p>
<p style="text-align: center;">Figure 11. Screen capture of code that identifies domain controllers in a network using LDAP queries</p>
<p><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/10/fig12-1024x807.jpg"/></p>
<p style="text-align: center;">Figure 12. Screen capture of code that identifies machines which are not domain controllers in a network using LDAP queries</p>
<p style="text-align: left;">We also discovered that there is a possible SMB protocol implementation using “pysmb,” utilizing the NT LM 0.12 query for older Windows operating systems and IPC shares. It should be noted that this function seems to still be in development.<img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/10/fig13.jpg"/></p>
<p style="text-align: center;">Figure13. Screen capture of code showing possible SMB communication</p>
<p><i><b>networkDll32<br />
 </b><u></u></i><br />
Trickbot uses this encrypted module to scan the network and steal relevant network information. It executes the following commands to gather information on the infected system:</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/10/fig14.jpg"/></p>
<p style="text-align: center;">Figure 14. Screen capture of the commands executed by the networkDll32 module to gather network information</p>
<p><i><b>Wormdll32 module</b></i></p>
<p>Wormdll32 is an encrypted module that Trickbot uses to propagate itself via SMB and LDAP queries. It is used together with the module “wormDll” to propagate across the network.</p>
<p><i><b>importDll32 module</b></i></p>
<p>This module is responsible for stealing browser data such as browsing history, cookies, and plug-ins, among others.</p>
<p><b><i>systeminfo32 module</i> <br />
 </b><br />
Once successfully installed in a system, Trickbot will gather system information such as OS, CPU, and memory information, user accounts, lists of installed programs and services.</p>
<p><b><i>mailsearcher32 module</i> <br />
 </b><br />
This module searches the infected system’s files to gather email addresses for information-stealing purposes. Collecting email addresses for spam campaign-related needs is usual malware behavior, however, Kryptos Research recently <a href="https://blog.kryptoslogic.com/malware/2018/10/31/emotet-email-theft.html" target="_blank">reported</a> that the Emotet banking trojan doesn't just steal email addresses; it also harvests emails sent and received via Microsoft Outlook on an Emotet-infected device. Emotet, according to <a href="https://isc.sans.edu/diary/One+Emotet+infection+leads+to+three+follow-up+malware+infections/24140" target="_blank">previous research</a> by Brad Duncan, is also responsible for delivering this password-grabbing Trickbot variant, as well as <a href="https://www.trendmicro.com/vinfo/tmr/?/us/threat-encyclopedia/spam/3661/spam-campaign-leads-to-azorult-trojan-spyware">Azorult</a>, to users.</p>
<p><i><b>injectDll32 module</b></i></p>
<p>This encrypted module monitors websites that banking applications might use. It's also used to inject code into its target processes using the <a href="https://github.com/stephenfewer/ReflectiveDLLInjection" target="_blank"><i>Reflective DLL Injection</i></a> technique.</p>
<p>The injectDll32 monitors banking-related websites for two different credential-stealing methods:</p>
<p>First, when a user logs in to any of the monitored banking websites on its list such as Chase, Citi, Bank of America, Sparda-Bank, Santander, HSBC, Canadian Imperial Bank of Commerce (CIBC), and Metrobank, Trickbot will then send a POST response to the C&amp;C server to extract the user’s login credentials.</p>
<p>Second, Trickbot monitors if a user accesses certain banking-related websites on its list, such as C. Hoare &amp; Co bank, St. James’s Place Bank, and Royal Bank of Scotland, and will redirect users to fake phishing websites.</p>
<p>The banking URLs Trickbot monitors include websites from the United States, Canada, UK, Germany, Australia, Austria, Ireland, London, Switzerland, and Scotland.</p>
<p><b><span class="body-subhead-title">Trickbot’s other notable tricks</span></b></p>
<p>Trickbot is usually sent via <a href="https://blog.trendmicro.com/trendlabs-security-intelligence/look-locky-ransomwares-recent-spam-activities/?_ga=2.67425105.153681460.1540774662-609786015.1505725418">malicious spam campaigns</a>. The malware disables Microsoft’s built-in antivirus Windows Defender by executing certain commands and modifying registry entries. Additionally, it also terminates Windows Defender-related processes such as MSASCuil.exe, MSASCui.exe, and antispyware utility <i>Msmpeng.exe</i>. It also has an autostart mechanism (Msntcs) that is triggered at system startup and every ten minutes after it is first executed.</p>
<p>It disables the following anti-malware services:</p>
<ul>
<li><span class="rte-red-bullet">MBamService (Malwarebytes-related Process)</span></li>
<li><span class="rte-red-bullet">SAVService (Sophos AV-related process)</span><br />
 </li>
</ul>
<p>Its anti-analysis capability checks the system and terminates itself when it finds certain modules, such as pstorec.dll, vmcheck.dll, wpespy.dll, and dbghelp.dll.</p>
<p><b><span class="body-subhead-title">Defending against Trickbot’s tricks: Trend Micro solutions</span></b></p>
<p>Malware authors continue to update banking trojans like Trickbot and Emotet with new modules that make it more difficult to detect and combat. Users and enterprises can benefit from protection that use a multi-layered approach to mitigate the risks brought by threats like banking trojans.</p>
<p><a href="/en_us/business/products/user-protection/sps.html">Trend Micro Smart Protection Suites</a> provide a cross-generational blend of threat defense techniques to protect systems from all types of threats, including banking trojans, ransomware, and cryptocurrency-mining malware. With capabilities like web/URL filtering, behavioral analysis, and custom sandboxing, XGen security protects against today’s threats that bypass traditional controls; exploit known, unknown, or undisclosed vulnerabilities; either steal or encrypt personally identifiable data; or conduct malicious cryptocurrency mining. Smart, optimized, and connected, XGen security powers Trend Micro’s suite.<br />

 <span class="body-subhead-title">Indicators of Compromise</span></b></p>
<p><u>Trickbot C&amp;C servers</u></p>
<ul>
<li><span class="rte-red-bullet">103[.]10[.]145[.]197:449</span></li>
