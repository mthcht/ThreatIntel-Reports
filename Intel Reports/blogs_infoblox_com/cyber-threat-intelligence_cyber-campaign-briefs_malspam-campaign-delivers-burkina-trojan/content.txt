<h3><strong>TLP: WHITE </strong></h3>
<p>&nbsp;</p>
<h3>Overview</h3>
<p>From 21 to 23 March, we observed a malspam campaign distributing the Burkina trojan. First seen in October 2017, Burkina is a trojan distributed through executable (EXE) files sent via email.</p>
<h3>Customer Impact</h3>
<p>Burkina infects a victim’s computer and attempts to harvest credentials, interrupt standard processes, conceal network connections, and other malicious actions. The malware then reaches out to a command and control (C&amp;C) server to receive additional instructions.</p>
<p>The threat actor can use the stolen credentials to carry out additional malicious acts, including dropping a ransomware package or distributing additional payloads such as Trickbot.<sup>1</sup></p>
<h3>Campaign Analysis</h3>
<p>The campaign we observed delivered Burkina via spam emails. Both the subject line &#8211; <em>WG:Re:AG:Re:New order.</em> &#8211; and body &#8211; <em>See attached PDF</em> &#8211; of each message were identical. The emails carried a malicious EXE attachment masquerading as a PDF with the filename <em>SPL6677.pdf.exe</em>.</p>
<h3>Attack Chain</h3>
<p>Once the user opens the attached file, the executable will present the message “SimpleGameLib has stopped working” to the user.</p>
<p>Burkina drops another executable file named <em>WerFault.exe</em> and updates the AutoRun Registry to include itself. It then attempts to harvest credentials and saved passwords once the user opens a browser. Finally, the malware attempts to connect to a C&#038;C server to receive further instructions as well as to exfiltrate the stolen data.</p>
<h3>Vulnerabilities &#038; Mitigation</h3>
<p>Infoblox recommends the following precautions to reduce the possibility of infection by Burkina:</p>
<ul>
<li>Be cautious of emails from unfamiliar senders and inspect   unexpected attachments before opening them.</li>
<li>Always be suspicious of vague emails, especially if there     is a prompt to open an attachment or click on a URL or  clickable text.</li>
<li>Filter attachments to reduce the likelihood of malicious  content reaching a user’s workstation.</li>
<li>Be aware of any attachment’s file type and never openfiles that could be a script (.vbs, .cmd, .bat) or another  executable (.<em>exe</em>).</li>
<li>Ensure you read the file type correctly as many threat  actors will include a trusted file type description in the  name (.pdf, .docx, .xls, etc.)</li>
</ul>
