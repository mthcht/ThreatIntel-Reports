<p>An info stealer is malicious software (malware) that seeks to steal private data from a compromised device, including passwords, cookies, autofill information from browsers, and cryptocurrency wallet information.</p>
<p>Since the beginning of 2019, the Raccoon malware has been offered as malware-as-a-service on various cybercrime forums. The Raccoon Stealer group, however, was disbanded in March 2022 as a result of the death of one of its senior developers in the Ukraine-Russia war.</p>
<p>In June 2022, a new version of the Raccoon stealer was identified in the wild by the researchers at <a href="https://blog.sekoia.io/raccoon-stealer-v2-part-1-the-return-of-the-dead/">Sekoia</a>. Initially, the malware was named â<strong>Recordbreaker</strong>â but was later identified as a revived version of Raccoon stealer. The developer of the Raccoon stealer (MaaS) is very active on underground forums, regularly updating the malware, and posting about the new feature builds on the forum.</p>
<figure id="attachment_20567" aria-describedby="caption-attachment-20567" style="width: 1104px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-20567" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e23d254f0137_word-image-20566-1.png" alt="Post describing the technical details of recent samples and modifications made in the Raccoon Stealer" width="1104" height="534" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e23d254f0137_word-image-20566-1.png 1104w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-1-300x145.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-1-1024x495.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-1-768x371.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-1-150x73.png 150w" sizes="(max-width: 1104px) 100vw, 1104px"><figcaption id="caption-attachment-20567" class="wp-caption-text">Post describing the technical details of recent samples and modifications made in the Raccoon Stealer</figcaption></figure>
<p>&nbsp;</p>
<h2><a id="post-20566-_hjzugv4j4wzg"></a>The Malware</h2>
<p>Raccoon samples have been spotted in the wild on numerous occasions. While some of these were protected by commercial code protectors like VmProtect and Themida, others were seen packed in popular community packers like Armadillo.</p>
<p><a href="https://cloudsek.com/">CloudSEK</a>âs telemetry was able to pick up a very interesting Raccoon sample that employed very effective anti-analysis and anti-debugging techniques to foil analysis attempts. The sample covered in this report is unique in terms of the deployment of the malware.</p>
<h2><a id="post-20566-_ygeo39pgpr39"></a>The Malware Deployment</h2>
<p>The packer used to obfuscate the stealer is specifically designed to perform the two main tasks:</p>
<ul>
<li>Identify sandbox and debugging</li>
<li>Perform hooking in order to control transfer to the stealer</li>
</ul>
<h3><a id="post-20566-_gznu0jm3ui16"></a>The Process of Anti Analysis &amp; Anti Debugging</h3>
<ul>
<li>For detecting sandboxed environments, especially virtual environments, the packer makes use of <strong>Read Time Stamp Counter (RDTSC)</strong>, a very well known CPU instruction used to detect VM by calculating the time difference (delta) between two calls to RDTSC. RDTSC has also been observed, querying system information like the firmware information table to identify VMs.</li>
<li>To prevent anti-debugging, the malware includes process-level debug checks and sets the main thread hidden from the debugger.</li>
</ul>
<h3><a id="post-20566-_gvtohkbfg7fe"></a>Malicious Hooks</h3>
<p>The malwareâs API trace provided a greater understanding of the internals of the packer, without having to spend much time in a debugger. A very interesting behavior found in the trace log is shown below.</p>
<ul>
<li>The threads in the current process are enumerated by using the following APIs:
<ul>
<li><strong><em>kernel32!CreateToolhelp32Snapshot</em></strong></li>
<li><strong><em>kernel32!Thread32Next</em></strong></li>
</ul>
</li>
<li>The threads are then opened and suspended.</li>
<li>Once the threads are suspended, some memory is allocated and data is added to it.</li>
<li>Finally, the memory protections are changed from RWX to RX.</li>
</ul>
<figure id="attachment_20568" aria-describedby="caption-attachment-20568" style="width: 1030px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-20568" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e262e84f0126_word-image-20566-2.png" alt="API trace present in the malware" width="1030" height="473" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e262e84f0126_word-image-20566-2.png 1030w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-2-300x138.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-2-1024x470.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-2-768x353.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-2-150x69.png 150w" sizes="(max-width: 1030px) 100vw, 1030px"><figcaption id="caption-attachment-20568" class="wp-caption-text">API trace present in the malware</figcaption></figure>
<p>&nbsp;</p>
<p>The above sequence of operations is performed twice, and then the packer resumes the suspended threads.</p>
<figure id="attachment_20569" aria-describedby="caption-attachment-20569" style="width: 1188px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-20569" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead112ba7e216cf4f0141_word-image-20566-3.png" alt="Image of the packer resuming the suspended threads" width="1188" height="177" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead112ba7e216cf4f0141_word-image-20566-3.png 1188w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-3-300x45.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-3-1024x153.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-3-768x114.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-3-150x22.png 150w" sizes="(max-width: 1188px) 100vw, 1188px"><figcaption id="caption-attachment-20569" class="wp-caption-text">Image of the packer resuming the suspended threads</figcaption></figure>
<p>&nbsp;</p>
<p>The data written by the malware was retrieved by CloudSEKâs researchers with the help of instrumentation.</p>
<ul>
<li>As shown in the image below, a call was made to <strong><em>kernel32!WriteProcessMemory</em></strong> was intercepted to see the passed data. It is interesting to note that the<strong> <em>lpAddress</em></strong> parameter in both calls points to <strong><em>ntdll.dll</em> </strong>in the memory of the malware. A total of <strong>five bytes</strong> of data was written in the memory region of the loaded ntdll.</li>
</ul>
<p><img decoding="async" loading="lazy" width="759" height="179" class="wp-image-20570" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e238e94f0133_word-image-20566-4.png" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e238e94f0133_word-image-20566-4.png 759w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-4-300x71.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-4-150x35.png 150w" sizes="(max-width: 759px) 100vw, 759px"></p>
<figure id="attachment_20571" aria-describedby="caption-attachment-20571" style="width: 751px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-20571" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e271f54f0121_word-image-20566-5.png" alt="Hooking the NT API Calls" width="751" height="155" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e271f54f0121_word-image-20566-5.png 751w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-5-300x62.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-5-150x31.png 150w" sizes="(max-width: 751px) 100vw, 751px"><figcaption id="caption-attachment-20571" class="wp-caption-text">Hooking the NT API Calls</figcaption></figure>
<p>&nbsp;</p>
<ul>
<li>The written data is a <strong>JMP </strong>(jump) instruction, followed by a specific address that points to one of the segments in the packer.</li>
</ul>
<p><img decoding="async" loading="lazy" width="671" height="74" class="wp-image-20572" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead112ba7e258ca4f013e_word-image-20566-6.png" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead112ba7e258ca4f013e_word-image-20566-6.png 671w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-6-300x33.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-6-150x17.png 150w" sizes="(max-width: 671px) 100vw, 671px"></p>
<figure id="attachment_20573" aria-describedby="caption-attachment-20573" style="width: 656px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-20573" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e213714f012c_word-image-20566-7.png" alt="Updated function entry after hooking" width="656" height="51" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e213714f012c_word-image-20566-7.png 656w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-7-300x23.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-7-640x51.png 640w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-7-150x12.png 150w" sizes="(max-width: 656px) 100vw, 656px"><figcaption id="caption-attachment-20573" class="wp-caption-text">Updated function entry after hooking</figcaption></figure>
<p>&nbsp;</p>
<p>Hooking plays a major role in the stealer loading phase and the packer is hooking the following two APIs:</p>
<ul>
<li><strong><em>ntdll!DbgUiRemoteBreakin</em></strong> &#8211; The hooked <em>DbgUiRemoteBreakin</em> will take the control flow to exit. This is another anti-debugging technique in which, the targeted API is used mainly by Windows debuggers to do a software break. Hence, the packer redirects the flow, which leads to the termination of the malware.</li>
<li><strong><em>ntdll!ZwProtectVirtualMemory &#8211; </em></strong>If the above doesnât happen, the packer makes a call to <em>ntdll!ZwProtectVirtualMemory</em> and deploys the Raccoon Stealer v2 on the target system.</li>
</ul>
<p>Experimenting with the return values of the <strong><em>kernel32!WriteProcessMemory</em></strong> call during analysis helped to confirm the hooking of <em>ntdll!ZwProtectVirtualMemory</em>, which is a crucial step in the infection process. Failure to hook <em>ntdll!ZwProtectVirtualMemory</em> causes the malware to terminate and the following warning to appear.</p>
<figure id="attachment_20574" aria-describedby="caption-attachment-20574" style="width: 338px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-20574" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e21d084f0128_word-image-20566-8.png" alt="Warning popup triggered upon failure of hooking" width="338" height="172" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e21d084f0128_word-image-20566-8.png 338w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-8-300x153.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-8-150x76.png 150w" sizes="(max-width: 338px) 100vw, 338px"><figcaption id="caption-attachment-20574" class="wp-caption-text">Warning popup triggered upon failure of hooking</figcaption></figure>
<p>&nbsp;</p>
<p>This behavior is not observed when the malware fails to hook <em>ntdll!DbgUiRemoteBreakin</em>, as the program doesn&#8217;t get terminated.</p>
<h2><a id="post-20566-_113xic7sp51"></a>The Malware Execution</h2>
<h3><a id="post-20566-_up9me939sg7a"></a>Dynamic API Loading</h3>
<p>Once Raccoon Stealer is executed, APIs are dynamically loaded into the memory. These APIs are later used by the malware to perform malicious activities on the compromised machine.</p>
<figure id="attachment_20575" aria-describedby="caption-attachment-20575" style="width: 960px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-20575" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e279804f0132_word-image-20566-9.png" alt="Code responsible for runtime dynamic linking of DLLs" width="960" height="777" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e279804f0132_word-image-20566-9.png 960w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-9-300x243.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-9-768x622.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-9-150x121.png 150w" sizes="(max-width: 960px) 100vw, 960px"><figcaption id="caption-attachment-20575" class="wp-caption-text">Code responsible for runtime dynamic linking of DLLs</figcaption></figure>
<p>&nbsp;</p>
<h3><a id="post-20566-_893y4c8iz2aq"></a>String Decoding</h3>
<p>After successfully loading the libraries, the stealer decodes all the strings in memory. The previous versions of the stealer used RC4 decryption to encrypt the strings.</p>
<figure id="attachment_20576" aria-describedby="caption-attachment-20576" style="width: 371px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-20576" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e2271f4f0125_word-image-20566-10.png" alt="RC4 decryption routine used in the old malware samples" width="371" height="111" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e2271f4f0125_word-image-20566-10.png 371w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-10-300x90.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-10-150x45.png 150w" sizes="(max-width: 371px) 100vw, 371px"><figcaption id="caption-attachment-20576" class="wp-caption-text">RC4 decryption routine used in the old malware samples</figcaption></figure>
<p>&nbsp;</p>
<p>However, the recent version uses a custom XOR-based encoding to encrypt the strings.</p>
<figure id="attachment_20577" aria-describedby="caption-attachment-20577" style="width: 444px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-20577" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e269554f0122_word-image-20566-11.png" alt="Custom XOR encoding used in new malware samples" width="444" height="188" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e269554f0122_word-image-20566-11.png 444w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-11-300x127.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-11-150x64.png 150w" sizes="(max-width: 444px) 100vw, 444px"><figcaption id="caption-attachment-20577" class="wp-caption-text">Custom XOR encoding used in new malware samples</figcaption></figure>
<p>&nbsp;</p>
<h3><a id="post-20566-_gjg44n805xww"></a>Russian Language Detection</h3>

<h3><a id="post-20566-_3ztlier27a3a"></a>Mutex</h3>

<h2><a id="post-20566-_19dsd0a6pioh"></a>Admin Check</h2>
<p>Once the Mutex is created, Raccoon checks the privileges of the user process by following the steps below:</p>
<ul>

<li><strong><em>Advapi32.GetTokenInformation</em></strong> is called on the acquired process token handle by passing <em>TOKEN_USER</em> as the value for <em>TokenInformationClass</em> parameter, which returns a <strong>user SID </strong>structure.</li>
<li>The SID structure is converted to a string by calling <strong><em>Advapi32!ConvertSidToStringSidW</em></strong>.</li>
<li>The SID string is compared with the value <strong>âS-1-5-18â</strong>, the SID value for <strong>Local/SYSTEM</strong> or members in the <strong>Local Admin</strong> group.</li>
<li>If the user process is elevated, the value 0 is returned.</li>
</ul>

<h2><a id="post-20566-_wq7abflo1fwb"></a>Additional Payload Execution</h2>
<p>The Raccoon stealer, like any other malware in its class, has the ability to execute user-provided additional malware (such as RATs) on the compromised system. <p>An info stealer is malicious software (malware) that seeks to steal private data from a compromised device, including passwords, cookies, autofill information from browsers, and cryptocurrency wallet information.</p>
<p>Since the beginning of 2019, the Raccoon malware has been offered as malware-as-a-service on various cybercrime forums. The Raccoon Stealer group, however, was disbanded in March 2022 as a result of the death of one of its senior developers in the Ukraine-Russia war.</p>
<p>In June 2022, a new version of the Raccoon stealer was identified in the wild by the researchers at <a href="https://blog.sekoia.io/raccoon-stealer-v2-part-1-the-return-of-the-dead/">Sekoia</a>. Initially, the malware was named â<strong>Recordbreaker</strong>â but was later identified as a revived version of Raccoon stealer. The developer of the Raccoon stealer (MaaS) is very active on underground forums, regularly updating the malware, and posting about the new feature builds on the forum.</p>

<p>&nbsp;</p>
<h2><a id="post-20566-_hjzugv4j4wzg"></a>The Malware</h2>
<p>Raccoon samples have been spotted in the wild on numerous occasions. While some of these were protected by commercial code protectors like VmProtect and Themida, others were seen packed in popular community packers like Armadillo.</p>
<p><a href="https://cloudsek.com/">CloudSEK</a>âs telemetry was able to pick up a very interesting Raccoon sample that employed very effective anti-analysis and anti-debugging techniques to foil analysis attempts. The sample covered in this report is unique in terms of the deployment of the malware.</p>
<h2><a id="post-20566-_ygeo39pgpr39"></a>The Malware Deployment</h2>
<p>The packer used to obfuscate the stealer is specifically designed to perform the two main tasks:</p>
<ul>
<li>Identify sandbox and debugging</li>
<li>Perform hooking in order to control transfer to the stealer</li>
</ul>
<h3><a id="post-20566-_gznu0jm3ui16"></a>The Process of Anti Analysis &amp; Anti Debugging</h3>
<ul>
<li>For detecting sandboxed environments, especially virtual environments, the packer makes use of <strong>Read Time Stamp Counter (RDTSC)</strong>, a very well known CPU instruction used to detect VM by calculating the time difference (delta) between two calls to RDTSC. RDTSC has also been observed, querying system information like the firmware information table to identify VMs.</li>
<li>To prevent anti-debugging, the malware includes process-level debug checks and sets the main thread hidden from the debugger.</li>
</ul>
<h3><a id="post-20566-_gvtohkbfg7fe"></a>Malicious Hooks</h3>
<p>The malwareâs API trace provided a greater understanding of the internals of the packer, without having to spend much time in a debugger. A very interesting behavior found in the trace log is shown below.</p>
<ul>
<li>The threads in the current process are enumerated by using the following APIs:
<ul>
<li><strong><em>kernel32!CreateToolhelp32Snapshot</em></strong></li>
<li><strong><em>kernel32!Thread32Next</em></strong></li>
</ul>
</li>
<li>The threads are then opened and suspended.</li>
<li>Once the threads are suspended, some memory is allocated and data is added to it.</li>
<li>Finally, the memory protections are changed from RWX to RX.</li>
</ul>
<figure id="attachment_20568" aria-describedby="caption-attachment-20568" style="width: 1030px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-20568" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e262e84f0126_word-image-20566-2.png" alt="API trace present in the malware" width="1030" height="473" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e262e84f0126_word-image-20566-2.png 1030w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-2-300x138.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-2-1024x470.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-2-768x353.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-2-150x69.png 150w" sizes="(max-width: 1030px) 100vw, 1030px"><figcaption id="caption-attachment-20568" class="wp-caption-text">API trace present in the malware</figcaption></figure>
<p>&nbsp;</p>
<p>The above sequence of operations is performed twice, and then the packer resumes the suspended threads.</p>
<figure id="attachment_20569" aria-describedby="caption-attachment-20569" style="width: 1188px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-20569" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead112ba7e216cf4f0141_word-image-20566-3.png" alt="Image of the packer resuming the suspended threads" width="1188" height="177" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead112ba7e216cf4f0141_word-image-20566-3.png 1188w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-3-300x45.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-3-1024x153.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-3-768x114.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-3-150x22.png 150w" sizes="(max-width: 1188px) 100vw, 1188px"><figcaption id="caption-attachment-20569" class="wp-caption-text">Image of the packer resuming the suspended threads</figcaption></figure>
<p>&nbsp;</p>
<p>The data written by the malware was retrieved by CloudSEKâs researchers with the help of instrumentation.</p>
<ul>
<li>As shown in the image below, a call was made to <strong><em>kernel32!WriteProcessMemory</em></strong> was intercepted to see the passed data. It is interesting to note that the<strong> <em>lpAddress</em></strong> parameter in both calls points to <strong><em>ntdll.dll</em> </strong>in the memory of the malware. A total of <strong>five bytes</strong> of data was written in the memory region of the loaded ntdll.</li>
</ul>

<li>The written data is a <strong>JMP </strong>(jump) instruction, followed by a specific address that points to one of the segments in the packer.</li>

<p>&nbsp;</p>
<p>Hooking plays a major role in the stealer loading phase and the packer is hooking the following two APIs:</p>
<ul>
<li><strong><em>ntdll!DbgUiRemoteBreakin</em></strong> &#8211; The hooked <em>DbgUiRemoteBreakin</em> will take the control flow to exit. This is another anti-debugging technique in which, the targeted API is used mainly by Windows debuggers to do a software break. Hence, the packer redirects the flow, which leads to the termination of the malware.</li>
<li><strong><em>ntdll!ZwProtectVirtualMemory &#8211; </em></strong>If the above doesnât happen, the packer makes a call to <em>ntdll!ZwProtectVirtualMemory</em> and deploys the Raccoon Stealer v2 on the target system.</li>
</ul>
<p>Experimenting with the return values of the <strong><em>kernel32!WriteProcessMemory</em></strong> call during analysis helped to confirm the hooking of <em>ntdll!ZwProtectVirtualMemory</em>, which is a crucial step in the infection process. Failure to hook <em>ntdll!ZwProtectVirtualMemory</em> causes the malware to terminate and the following warning to appear.</p>
<figure id="attachment_20574" aria-describedby="caption-attachment-20574" style="width: 338px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-20574" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e21d084f0128_word-image-20566-8.png" alt="Warning popup triggered upon failure of hooking" width="338" height="172" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e21d084f0128_word-image-20566-8.png 338w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-8-300x153.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-8-150x76.png 150w" sizes="(max-width: 338px) 100vw, 338px"><figcaption id="caption-attachment-20574" class="wp-caption-text">Warning popup triggered upon failure of hooking</figcaption></figure>
<p>&nbsp;</p>
<p>This behavior is not observed when the malware fails to hook <em>ntdll!DbgUiRemoteBreakin</em>, as the program doesn&#8217;t get terminated.</p>
<h2><a id="post-20566-_113xic7sp51"></a>The Malware Execution</h2>
<h3><a id="post-20566-_up9me939sg7a"></a>Dynamic API Loading</h3>
<p>Once Raccoon Stealer is executed, APIs are dynamically loaded into the memory. These APIs are later used by the malware to perform malicious activities on the compromised machine.</p>
<figure id="attachment_20575" aria-describedby="caption-attachment-20575" style="width: 960px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-20575" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e279804f0132_word-image-20566-9.png" alt="Code responsible for runtime dynamic linking of DLLs" width="960" height="777" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e279804f0132_word-image-20566-9.png 960w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-9-300x243.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-9-768x622.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-9-150x121.png 150w" sizes="(max-width: 960px) 100vw, 960px"><figcaption id="caption-attachment-20575" class="wp-caption-text">Code responsible for runtime dynamic linking of DLLs</figcaption></figure>
<p>&nbsp;</p>
<h3><a id="post-20566-_893y4c8iz2aq"></a>String Decoding</h3>
<p>After successfully loading the libraries, the stealer decodes all the strings in memory. The previous versions of the stealer used RC4 decryption to encrypt the strings.</p>
<figure id="attachment_20576" aria-describedby="caption-attachment-20576" style="width: 371px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-20576" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e2271f4f0125_word-image-20566-10.png" alt="RC4 decryption routine used in the old malware samples" width="371" height="111" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e2271f4f0125_word-image-20566-10.png 371w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-10-300x90.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-10-150x45.png 150w" sizes="(max-width: 371px) 100vw, 371px"><figcaption id="caption-attachment-20576" class="wp-caption-text">RC4 decryption routine used in the old malware samples</figcaption></figure>
<p>&nbsp;</p>
<p>However, the recent version uses a custom XOR-based encoding to encrypt the strings.</p>
<figure id="attachment_20577" aria-describedby="caption-attachment-20577" style="width: 444px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-20577" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e269554f0122_word-image-20566-11.png" alt="Custom XOR encoding used in new malware samples" width="444" height="188" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead102ba7e269554f0122_word-image-20566-11.png 444w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-11-300x127.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-11-150x64.png 150w" sizes="(max-width: 444px) 100vw, 444px"><figcaption id="caption-attachment-20577" class="wp-caption-text">Custom XOR encoding used in new malware samples</figcaption></figure>
<p>&nbsp;</p>
<h3><a id="post-20566-_gjg44n805xww"></a>Russian Language Detection</h3>
<p>The stealer calls the <strong><em>kernel32!GetDefaulLocaleName</em></strong> to retrieve the system language (locale name), and then checks it against the string <strong><em>âRUâ</em></strong>. In case of a positive match, no logic is implemented for execution, which shows that the malware is still under development. In the future, we can expect the stealer to terminate itself after a match is found.</p>
<h3><a id="post-20566-_3ztlier27a3a"></a>Mutex</h3>
<p>After the locale name check, the stealer looks for any active malware samples, by calling <strong><em>kernel32.OpenMutexW</em></strong>. If an active malware process is found, the current malware execution is terminated, else a new mutex is created on the system.</p>

<h2><a id="post-20566-_19dsd0a6pioh"></a>Admin Check</h2>
<p>Once the Mutex is created, Raccoon checks the privileges of the user process by following the steps below:</p>
<ul>
<li><strong><em>Advapi32.OpenProcess</em></strong><em> is called to obtain a</em> handle to the process token.</li>
<li><strong><em>Advapi32.GetTokenInformation</em></strong> is called on the acquired process token handle by passing <em>TOKEN_USER</em> as the value for <em>TokenInformationClass</em> parameter, which returns a <strong>user SID </strong>structure.</li>
<li>The SID structure is converted to a string by calling <strong><em>Advapi32!ConvertSidToStringSidW</em></strong>.</li>
<li>The SID string is compared with the value <strong>âS-1-5-18â</strong>, the SID value for <strong>Local/SYSTEM</strong> or members in the <strong>Local Admin</strong> group.</li>
<li>If the user process is elevated, the value 0 is returned.</li>
</ul>
<figure id="attachment_20579" aria-describedby="caption-attachment-20579" style="width: 798px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-20579" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead112ba7e272374f0140_word-image-20566-13.png" alt="Administrator check performed by the stealer" width="798" height="443" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead112ba7e272374f0140_word-image-20566-13.png 798w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-13-300x167.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-13-768x426.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20566-13-150x83.png 150w" sizes="(max-width: 798px) 100vw, 798px"><figcaption id="caption-attachment-20579" class="wp-caption-text">Administrator check performed by the stealer</figcaption></figure>

<h3><a id="post-20566-_k7ggbctdfnso"></a>Process Enumeration</h3>
<p>If the process is elevated, the processes running on the system are enumerated as shown below:</p>
<ul>
<li><strong><em>Kernel32!CreateToolhelp32Snapshot</em></strong> is called by passing the flag TH32CS_SNAPPROCESS  to include all processes running on the system in the snapshot.</li>
<li><em>The </em><strong><em>Kernel32!Process32First</em></strong> and <strong><em>Kernel32!Process32Next</em></strong> APIs are used to walk through the snapshot which contains the information of processes running on the system.</li>
</ul>

<h2><a id="post-20566-_2j8913xt0igm"></a>C2 Network</h2>
<p>Attackers employ a set of tools and procedures known as command and control infrastructure, usually abbreviated as C2 or C&amp;C, to keep in touch with compromised devices after the initial access has been gained. The Raccoon stealer calls home for the first time by sending a unique string to the C2. The string, for the communication, is crafted with the following information:</p>
<ul>
<li><strong>Machine GUID</strong> retrieved from the following location in the registry:</li>
</ul>
<p>Computer\HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Cryptography</p>
<ul>
<li>The <strong>username,</strong> fetched via the <em>Advapi32!GetUserNameW</em> API.</li>
<li>The <strong>configuration ID</strong>, which is decoded using the RC4 key in some samples and a unique alphanumeric string in others.</li>
</ul>
