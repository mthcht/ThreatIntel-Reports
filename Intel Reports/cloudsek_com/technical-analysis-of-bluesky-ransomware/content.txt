<h2>What is BlueSky Ransomware?</h2>
<p>BlueSky Ransomware is a modern malware using advanced techniques to evade security defences. It predominantly targets Windows hosts and utilizes the Windows multithreading model for fast encryption. It first emerged in late June 2022 and has been observed to spread via phishing emails, phishing websites, and trojanized downloads.</p>
<p>This deep-dive analysis of BlueSky Ransomware covers the following technical aspects:</p>
<ul>
<li>Procedure for privilege escalation</li>
<li>Persistence</li>
<li>Encryption mechanism</li>
<li>Evasion techniques</li>
</ul>

<li>After loading the required libraries, the ransomware proceeds to perform the following tasks:
<ul>
<li>Checks that the running process is 32 bit via <strong>kernel32.IsWow64Process</strong></li>
<li>Decrypts strings</li>
<li>Adjust the privilege of the process to <strong>SE_DEBUG</strong> via <strong>ntdll.RtlAdjustPrivilege</strong></li>
<li>Retrieves the following:
<ul>
<li><strong>MachineGUID</strong> from SOFTWARE\\Microsoft\\Cryptography</li>
<li><strong>DigitalProductID</strong> and <strong>InstallDate</strong> from SOFTWARE\\Microsoft\\Windows NT\\CurrentVersion</li>
</ul>
</li>
<li>Hides the main thread from debugger by calling<strong> ntdll.ZwSetInformationThread</strong> by passing ThreadHideFromDebugger (0x11) as ThreadInformationClass</li>
</ul>
</li>
<li>The ransomware updates the status as âCompletedâ after the initial phase and the user data is locked.</li>

<h2><a id="post-21147-_z8wovf836sg8"></a>Pre-Encryption</h2>
<h3><a id="post-21147-_o15z9rfpiqzb"></a>Cryptographic Algorithm</h3>
<p>Cryptographic context is a type of additional authenticated data consisting of non-secret arbitrary name-value pairs. During the initialization phase, the ransomware acquires cryptographic context from <strong>advapi32.CryptAcquireContext</strong> API. The cryptographic provider used by the malware is âMicrosoft Enhanced Cryptographic Provider v1.0â and the encryption scheme selected is RSA.</p>

<h3><a id="post-21147-_in2how17muhm"></a>Ransom Note</h3>
<p>If writing the decryption data fails, the ransomware will not execute the routine responsible for the encryption of user data. After a successful registry operation, the ransomware generates a ransom note as the initial task in the function that performs the locking.</p>
<figure id="attachment_21153" aria-describedby="caption-attachment-21153" style="width: 1242px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-21153" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e38d5c07d05_word-image-21147-6.png" alt="Ransom note generation" width="1242" height="247" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e38d5c07d05_word-image-21147-6.png 1242w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-6-300x60.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-6-1024x204.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-6-768x153.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-6-150x30.png 150w" sizes="(max-width: 1242px) 100vw, 1242px"><figcaption id="caption-attachment-21153" class="wp-caption-text">Ransom note generation</figcaption></figure>
<p>&nbsp;</p>
<p>The following steps are performed:</p>
<ul>
<li>A random and unique recovery ID for the victim is generated and stored in the heap buffer.</li>
<li>The Bluesky ransomware creates ransom note in â.txtâ and â.htmlâ formats.</li>
<li>Two blocks of 1000 (4096) bytes of heap memory are allocated to hold the final ransom notes.</li>
<li>Two temporary buffers (txt_ransom_note_buffer and html_ransom_note_buffer) are allocated to hold encoded notes retrieved from the binary.</li>
<li>A place format string specifier is used as a placeholder for the recovery ID generated in the initial step.</li>
<li>The function â<strong>sub_2866E0</strong>â is responsible for formatting the note by replacing the â%sâ with the recovery ID value which is 242 characters long.</li>
<li>The result is then stored in memory, to be later used by the function responsible for writing the note to the filesystem.</li>
</ul>
<figure id="attachment_21154" aria-describedby="caption-attachment-21154" style="width: 523px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21154" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e0a02c07cf2_word-image-21147-7.png" alt="Decoded note in the buffer" width="523" height="148" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e0a02c07cf2_word-image-21147-7.png 523w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-7-300x85.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-7-150x42.png 150w" sizes="(max-width: 523px) 100vw, 523px"><figcaption id="caption-attachment-21154" class="wp-caption-text">Decoded note in the buffer</figcaption></figure>
<h3><a id="post-21147-_ezk2mxt443wh"></a>Process Termination</h3>
<p>After creating the ransom note, the ransomware enumerates the processes running on the compromised system. The <strong>ntdll.ZwQuerySystemInformation</strong> API is called by passing the SystemInformation class (0x5) to get the process list from the system. The list is used by the ransomware to selectively kill the processes.</p>
<figure id="attachment_21155" aria-describedby="caption-attachment-21155" style="width: 1225px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21155" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead19c9d13ef6b7c07d0c_word-image-21147-8.png" alt="Enumeration of processes running on the compromised system" width="1225" height="82" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead19c9d13ef6b7c07d0c_word-image-21147-8.png 1225w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-8-300x20.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-8-1024x69.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-8-768x51.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-8-150x10.png 150w" sizes="(max-width: 1225px) 100vw, 1225px"><figcaption id="caption-attachment-21155" class="wp-caption-text">Enumeration of processes running on the compromised system</figcaption></figure>

<p>&nbsp;</p>
<p>The following steps are performed to terminate the running processes:</p>
<ul>
<li>The ransomware starts to analyze the process structure to retrieve the image name and uses <strong>shlwapi.PathRemoveExtensionW</strong> API to remove the extension (.exe) from the name.</li>
<li>Once the name of the process without extension is retrieved, the ransomware calls <strong>sub_2869B0</strong> to calculate the size of the process name.</li>
<li>Next a call is made to <strong>sub_2868C0</strong> to convert the characters to lowercase for uniformity.</li>
<li>Finally, a custom byte encoding is used to convert the string to a hex value.</li>
<li>The generated hex value is checked against an array of encoded values of processes to be terminated.</li>
</ul>
<figure id="attachment_21157" aria-describedby="caption-attachment-21157" style="width: 651px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21157" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead19c9d13e1ca1c07d0f_word-image-21147-10.png" alt="Process names the threat actor wants to terminate" width="651" height="503" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead19c9d13e1ca1c07d0f_word-image-21147-10.png 651w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-10-300x232.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-10-150x116.png 150w" sizes="(max-width: 651px) 100vw, 651px"><figcaption id="caption-attachment-21157" class="wp-caption-text">Process names the threat actor wants to terminate</figcaption></figure>
<p>&nbsp;</p>
<ul>
<li>At the initial phase the handle to âShell_Traywndâ, which is obtained using <strong>user32.FindWindowA,</strong> is passed to the <strong>GetWindowThreadProcessId </strong>API in order to get the process ID of explorer.exe. (explorer.exe is responsible for creating âShell_Traywndâ). The process ID is stored in the memory.</li>
<li>If there is a match, the target process ID, obtained at the initial phase, is passed to <strong>sub_2910F0.</strong></li>
<li>The malware checks if the process ID is of its own process or of explorer.exe. After the check, a handle to process is retrieved via <strong>kernel32.OpenProcess</strong> API.</li>
<li>Only ânon-criticalâ processes are terminated to prevent bug check (Blue Screen of Death). If the passed process handle is not critical, it is terminated via <strong>kernel32.TerminateProcess</strong>.</li>
</ul>
<figure id="attachment_21158" aria-describedby="caption-attachment-21158" style="width: 1004px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21158" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e59eec07cfb_word-image-21147-11.png" alt="The function sub_2910F0" width="1004" height="265" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e59eec07cfb_word-image-21147-11.png 1004w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-11-300x79.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-11-768x203.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-11-150x40.png 150w" sizes="(max-width: 1004px) 100vw, 1004px"><figcaption id="caption-attachment-21158" class="wp-caption-text">The function sub_2910F0</figcaption></figure>
<p>&nbsp;</p>
<ul>
<li>The ransomware calls <strong>ntdll.NtQueryInformationProcess</strong> by passing ProcessBreakOnTermination (0x1d) as the InformationClass to identify critical processes.</li>
</ul>
<figure id="attachment_21159" aria-describedby="caption-attachment-21159" style="width: 1047px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21159" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e7120c07cf4_word-image-21147-12.png" alt="Call to NtQueryInformationProcess Class" width="1047" height="97" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e7120c07cf4_word-image-21147-12.png 1047w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-12-300x28.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-12-1024x95.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-12-768x71.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-12-150x14.png 150w" sizes="(max-width: 1047px) 100vw, 1047px"><figcaption id="caption-attachment-21159" class="wp-caption-text">Call to NtQueryInformationProcess Class</figcaption></figure>

<h2><a id="post-21147-_miejrhoczr9p"></a>Encryption</h2>
<h3><a id="post-21147-_1cy9dtkqhkr5"></a>Threading Model: Windows IO Completion Ports in Nutshell</h3>
<ul>
<li>The Bluesky ransomware performs the encryption by utilizing IO completion ports. I/O completion ports provide an efficient threading model for processing multiple asynchronous input-output (I/O) requests on a multiprocessor system.</li>
</ul>
<figure id="attachment_21161" aria-describedby="caption-attachment-21161" style="width: 2048px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-21161" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead19c9d13efd5fc07d18_word-image-21147-14.png" alt="Threading model using the IO ports" width="2048" height="1187" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead19c9d13efd5fc07d18_word-image-21147-14.png 2048w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-14-300x174.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-14-1024x594.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-14-768x445.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-14-1536x890.png 1536w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-14-150x87.png 150w" sizes="(max-width: 2048px) 100vw, 2048px"><figcaption id="caption-attachment-21161" class="wp-caption-text">Threading model using the IO ports</figcaption></figure>
<p>&nbsp;</p>
<ul>
<li>The main thread creates the IO completion port via <strong>CreateIOCompletionPort</strong>. The created port can be associated with many file handles. When the asynchronous IO operation on one of the file handles is completed, an IO completion packet is queued in FIFO order to the associated port.</li>
<li>The worker thread performs a call to <strong>PostQueuedCompletionStatus </strong>to enqueue the associated data. In the case of ransomware, the data will be the absolute path of the user files waiting in the queue to get encrypted.</li>
<li>Another worker thread performs <strong>GetQueuedCompletionStatus </strong>to dequeue the contents from the main queue. Usually, in ransomware, this thread is responsible for performing encryption and ransom note generation.</li>
<li>The following section contains an depth description of each of the above-mentioned functions.</li>
</ul>
<h3><a id="post-21147-_qjgn4z75mbcj"></a>CreateIOCompletionPort</h3>
<p>The call to CreateIOCompletionPort involves the following steps:</p>
<ul>
<li>The main thread retrieves the processor count from the PEB (Process Environment Block) structure.</li>
<li>A call to <strong>CreateIoCompletionPort </strong>is made by passing processor count as NumberOfConcurrentThreads parameter value.</li>
<li>Multiple worker threads are created by calling <strong>kernel32.CreateThread</strong>.</li>
<li>For each thread, an affinity mask (a bit mask indicating what processor a thread should run on) is set by calling <strong>kernel32.SetThreadAffinityMask</strong>.</li>
<li>The main thread performs basic drive enumeration and calls <strong>PostQueuedCompletionStatus</strong>.</li>
</ul>
<figure id="attachment_21162" aria-describedby="caption-attachment-21162" style="width: 963px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-21162" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13ec028c07cf0_word-image-21147-15.png" alt="Calling CreatIoCompletionPort" width="963" height="294" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13ec028c07cf0_word-image-21147-15.png 963w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-15-300x92.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-15-768x234.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-15-150x46.png 150w" sizes="(max-width: 963px) 100vw, 963px"><figcaption id="caption-attachment-21162" class="wp-caption-text">Calling CreatIoCompletionPort</figcaption></figure>
<p>&nbsp;</p>
<figure id="attachment_21163" aria-describedby="caption-attachment-21163" style="width: 1148px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-21163" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e8bb6c07cf8_word-image-21147-16.png" alt="Retrieving processor count from PEB" width="1148" height="90" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e8bb6c07cf8_word-image-21147-16.png 1148w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-16-300x24.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-16-1024x80.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-16-768x60.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-16-150x12.png 150w" sizes="(max-width: 1148px) 100vw, 1148px"><figcaption id="caption-attachment-21163" class="wp-caption-text">Retrieving processor count from PEB</figcaption></figure>
<p>&nbsp;</p>
<figure id="attachment_21164" aria-describedby="caption-attachment-21164" style="width: 944px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21164" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e2995c07cf5_word-image-21147-17.png" alt="PostQueuedCompletionStatus Function" width="944" height="54" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e2995c07cf5_word-image-21147-17.png 944w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-17-300x17.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-17-768x44.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-17-150x9.png 150w" sizes="(max-width: 944px) 100vw, 944px"><figcaption id="caption-attachment-21164" class="wp-caption-text">PostQueuedCompletionStatus Function</figcaption></figure>
<p>&nbsp;</p>
<p>Following APIs are used for drive enumeration on the system:</p>
<ul>
<li>kernel32.GetLogicalDriveStringsW</li>
<li>kernel32.GetDriveTypeW</li>
</ul>
<p>Further enumeration of files is performed by creating worker thread for <strong>PostQueuedCompletionStatus.</strong></p>
<figure id="attachment_21165" aria-describedby="caption-attachment-21165" style="width: 1023px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21165" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e3eb7c07cf6_word-image-21147-18.png" alt="Creation of worker thread for PostQueuedCompletionStatus" width="1023" height="399" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e3eb7c07cf6_word-image-21147-18.png 1023w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-18-300x117.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-18-768x300.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-18-150x59.png 150w" sizes="(max-width: 1023px) 100vw, 1023px"><figcaption id="caption-attachment-21165" class="wp-caption-text">Creation of worker thread for PostQueuedCompletionStatus</figcaption></figure>
<p>&nbsp;</p>
<p>The main thread calls <strong>mpr.WNetOpenEnumW</strong> for enumerating network resources and creates a worker thread same as above that performs the PostQueuedCompletionStatus call.</p>
<figure id="attachment_21166" aria-describedby="caption-attachment-21166" style="width: 923px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21166" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e1909c07d03_word-image-21147-19.png" alt="Calling mpr.WNetOpenEnumW function" width="923" height="205" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e1909c07d03_word-image-21147-19.png 923w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-19-300x67.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-19-768x171.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-19-150x33.png 150w" sizes="(max-width: 923px) 100vw, 923px"><figcaption id="caption-attachment-21166" class="wp-caption-text">Calling mpr.WNetOpenEnumW function</figcaption></figure>
<p>&nbsp;</p>
<h3><a id="post-21147-_vmh2kcdq9otw"></a>Worker Thread: PostQueuedCompletionStatus</h3>
<figure id="attachment_21167" aria-describedby="caption-attachment-21167" style="width: 1225px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21167" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e4c07c07d00_word-image-21147-20.png" alt="The worker thread that performs the PostQueuedCompletionStatus" width="1225" height="630" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e4c07c07d00_word-image-21147-20.png 1225w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-20-300x154.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-20-1024x527.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-20-768x395.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-20-150x77.png 150w" sizes="(max-width: 1225px) 100vw, 1225px"><figcaption id="caption-attachment-21167" class="wp-caption-text">The worker thread that performs the PostQueuedCompletionStatus</figcaption></figure>
<p>&nbsp;</p>
<p>The newly created thread for PostQueuedCompletionStatus leads to the following:</p>
<ul>
<li>The files are enumerated via <strong>kernel32.FindFirstFileExW</strong> and<strong> kernel32.FindNextFileW</strong>.</li>
<li>If it is a directory, the thread function is recursively called to perform the file enumeration.</li>
<li>If it is a user file, then the absolute path is enqueued to the completion queue via <strong>PostQueuedCompletionStatus </strong>call.</li>
<li>This worker thread is responsible for gathering the files for encryption.</li>
</ul>
<h3><a id="post-21147-_98543zr3qugp"></a>Worker Thread: GetQueuedCompletionStatus</h3>
<p>This worker thread is responsible for doing the actual locking of the user files. The ransomware hides this thread from the debugger via <strong>ntdll.ZwSetInformationThread</strong> by passing ThreadHideFromDebugger as the ThreadInformationClass.</p>
<figure id="attachment_21168" aria-describedby="caption-attachment-21168" style="width: 1037px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-21168" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e8442c07cf3_word-image-21147-21.png" alt="Calling ntdll.ZwSetInformationThread function" width="1037" height="426" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e8442c07cf3_word-image-21147-21.png 1037w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-21-300x123.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-21-1024x421.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-21-768x315.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-21-150x62.png 150w" sizes="(max-width: 1037px) 100vw, 1037px"><figcaption id="caption-attachment-21168" class="wp-caption-text">Calling ntdll.ZwSetInformationThread function</figcaption></figure>
<p>&nbsp;</p>
<p>The thread decodes the file extension â.blueskyâ and proceeds to perform the encryption. The <strong>kernel32.GetQueuedCompletionStatus</strong> is called in an infinite loop to retrieve the absolute path of the user data.</p>
<figure id="attachment_21169" aria-describedby="caption-attachment-21169" style="width: 1221px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21169" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13edb6bc07d07_word-image-21147-22.png" alt="Decoding file extension â.blueskyâ" width="1221" height="530" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13edb6bc07d07_word-image-21147-22.png 1221w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-22-300x130.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-22-1024x444.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-22-768x333.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-22-150x65.png 150w" sizes="(max-width: 1221px) 100vw, 1221px"><figcaption id="caption-attachment-21169" class="wp-caption-text">Decoding file extension â.blueskyâ</figcaption></figure>
<p>&nbsp;</p>
<p>The <strong>sub_288780 </strong>function is responsible for encrypting the data. The thread checks if the dequeued item is a directory or a file.</p>
<ul>
<li>If it is a file then it proceeds to encrypt the data by using the following APIs:
<ul>
<li>kernel32.CreateFileW</li>
<li>kernel32.SetFilePointer</li>
<li>kernel32.ReadFile</li>
<li>kernel32.WriteFile</li>
</ul>
</li>
<li>If the item is a directory then <strong>sub_28EDA0</strong> is executed to dump the ransom note. The file name strings are decoded dynamically.</li>
</ul>
<figure id="attachment_21170" aria-describedby="caption-attachment-21170" style="width: 665px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21170" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13eccfac07cf9_word-image-21147-23.png" alt="File name strings being decoded" width="665" height="138" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13eccfac07cf9_word-image-21147-23.png 665w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-23-300x62.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-23-150x31.png 150w" sizes="(max-width: 665px) 100vw, 665px"><figcaption id="caption-attachment-21170" class="wp-caption-text">File name strings being decoded</figcaption></figure>
<p>&nbsp;</p>
<figure id="attachment_21171" aria-describedby="caption-attachment-21171" style="width: 702px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21171" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e0799c07cf1_word-image-21147-24.png" alt="Execution of sub_28EDA0" width="702" height="99" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e0799c07cf1_word-image-21147-24.png 702w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-24-300x42.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-24-150x21.png 150w" sizes="(max-width: 702px) 100vw, 702px"><figcaption id="caption-attachment-21171" class="wp-caption-text">Execution of sub_28EDA0</figcaption></figure>
<p>&nbsp;</p>
<p>The note content generated by the ransomware is written on the disk by calling:</p>
<ul>
<li>kernel32.CreateFileW</li>
<li>kernel32.WriteFile</li>
</ul>
<figure id="attachment_21172" aria-describedby="caption-attachment-21172" style="width: 939px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21172" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13eff2ac07d02_word-image-21147-25.png" alt="Ransom note being written on the disk" width="939" height="236" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13eff2ac07d02_word-image-21147-25.png 939w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-25-300x75.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-25-768x193.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-25-150x38.png 150w" sizes="(max-width: 939px) 100vw, 939px"><figcaption id="caption-attachment-21172" class="wp-caption-text">Ransom note being written on the disk</figcaption></figure>
<h2><a id="post-21147-_v8a6xq1s0lgw"></a>Post Encryption</h2>
<p>Once the user data is successfully locked, the ransomware performs the following operations:</p>
<ul>
<li>Releases the mutex created at the initial phase</li>
<li>Sets the thread state to ES_Continous</li>
<li>Destroys the allocated heap</li>
<li>Exits the process via kernel32.ExitProcess</li>
</ul>
<figure id="attachment_21173" aria-describedby="caption-attachment-21173" style="width: 1226px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-21173" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13ec59ec07cff_word-image-21147-26.png" alt="Post encryption functions" width="1226" height="153" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13ec59ec07cff_word-image-21147-26.png 1226w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-26-300x37.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-26-1024x128.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-26-768x96.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-26-150x19.png 150w" sizes="(max-width: 1226px) 100vw, 1226px"><figcaption id="caption-attachment-21173" class="wp-caption-text">Post encryption functions</figcaption></figure>

<h2>What is BlueSky Ransomware?</h2>
<p>BlueSky Ransomware is a modern malware using advanced techniques to evade security defences. It predominantly targets Windows hosts and utilizes the Windows multithreading model for fast encryption. It first emerged in late June 2022 and has been observed to spread via phishing emails, phishing websites, and trojanized downloads.</p>
<p>This deep-dive analysis of BlueSky Ransomware covers the following technical aspects:</p>
<ul>
<li>Procedure for privilege escalation</li>
<li>Persistence</li>
<li>Encryption mechanism</li>
<li>Evasion techniques</li>
</ul>

<li>After loading the required libraries, the ransomware proceeds to perform the following tasks:
<ul>
<li>Checks that the running process is 32 bit via <strong>kernel32.IsWow64Process</strong></li>
<li>Decrypts strings</li>
<li>Adjust the privilege of the process to <strong>SE_DEBUG</strong> via <strong>ntdll.RtlAdjustPrivilege</strong></li>
<li>Retrieves the following:
<ul>
<li><strong>MachineGUID</strong> from SOFTWARE\\Microsoft\\Cryptography</li>
<li><strong>DigitalProductID</strong> and <strong>InstallDate</strong> from SOFTWARE\\Microsoft\\Windows NT\\CurrentVersion</li>
</ul>
</li>
<li>Hides the main thread from debugger by calling<strong> ntdll.ZwSetInformationThread</strong> by passing ThreadHideFromDebugger (0x11) as ThreadInformationClass</li>
</ul>
</li>
<li>The ransomware updates the status as âCompletedâ after the initial phase and the user data is locked.</li>
</ul>

<h2><a id="post-21147-_z8wovf836sg8"></a>Pre-Encryption</h2>
<h3><a id="post-21147-_o15z9rfpiqzb"></a>Cryptographic Algorithm</h3>
<p>Cryptographic context is a type of additional authenticated data consisting of non-secret arbitrary name-value pairs. During the initialization phase, the ransomware acquires cryptographic context from <strong>advapi32.CryptAcquireContext</strong> API. The cryptographic provider used by the malware is âMicrosoft Enhanced Cryptographic Provider v1.0â and the encryption scheme selected is RSA.</p>

<p>&nbsp;</p>
<p>The following steps are performed:</p>
<ul>
<li>A random and unique recovery ID for the victim is generated and stored in the heap buffer.</li>
<li>The Bluesky ransomware creates ransom note in â.txtâ and â.htmlâ formats.</li>
<li>Two blocks of 1000 (4096) bytes of heap memory are allocated to hold the final ransom notes.</li>
<li>Two temporary buffers (txt_ransom_note_buffer and html_ransom_note_buffer) are allocated to hold encoded notes retrieved from the binary.</li>
<li>A place format string specifier is used as a placeholder for the recovery ID generated in the initial step.</li>
<li>The function â<strong>sub_2866E0</strong>â is responsible for formatting the note by replacing the â%sâ with the recovery ID value which is 242 characters long.</li>
<li>The result is then stored in memory, to be later used by the function responsible for writing the note to the filesystem.</li>
</ul>

<ul>
<li>The ransomware starts to analyze the process structure to retrieve the image name and uses <strong>shlwapi.PathRemoveExtensionW</strong> API to remove the extension (.exe) from the name.</li>

<h2><a id="post-21147-_miejrhoczr9p"></a>Encryption</h2>
<h3><a id="post-21147-_1cy9dtkqhkr5"></a>Threading Model: Windows IO Completion Ports in Nutshell</h3>
<ul>
<li>The Bluesky ransomware performs the encryption by utilizing IO completion ports. I/O completion ports provide an efficient threading model for processing multiple asynchronous input-output (I/O) requests on a multiprocessor system.</li>
</ul>
<figure id="attachment_21161" aria-describedby="caption-attachment-21161" style="width: 2048px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-21161" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead19c9d13efd5fc07d18_word-image-21147-14.png" alt="Threading model using the IO ports" width="2048" height="1187" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead19c9d13efd5fc07d18_word-image-21147-14.png 2048w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-14-300x174.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-14-1024x594.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-14-768x445.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-14-1536x890.png 1536w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-14-150x87.png 150w" sizes="(max-width: 2048px) 100vw, 2048px"><figcaption id="caption-attachment-21161" class="wp-caption-text">Threading model using the IO ports</figcaption></figure>
<p>&nbsp;</p>
<ul>
<li>The main thread creates the IO completion port via <strong>CreateIOCompletionPort</strong>. The created port can be associated with many file handles. When the asynchronous IO operation on one of the file handles is completed, an IO completion packet is queued in FIFO order to the associated port.</li>
<li>The worker thread performs a call to <strong>PostQueuedCompletionStatus </strong>to enqueue the associated data. In the case of ransomware, the data will be the absolute path of the user files waiting in the queue to get encrypted.</li>
<li>Another worker thread performs <strong>GetQueuedCompletionStatus </strong>to dequeue the contents from the main queue. Usually, in ransomware, this thread is responsible for performing encryption and ransom note generation.</li>
<li>The following section contains an depth description of each of the above-mentioned functions.</li>
</ul>
<h3><a id="post-21147-_qjgn4z75mbcj"></a>CreateIOCompletionPort</h3>
<p>The call to CreateIOCompletionPort involves the following steps:</p>
<ul>
<li>The main thread retrieves the processor count from the PEB (Process Environment Block) structure.</li>
<li>A call to <strong>CreateIoCompletionPort </strong>is made by passing processor count as NumberOfConcurrentThreads parameter value.</li>
<li>Multiple worker threads are created by calling <strong>kernel32.CreateThread</strong>.</li>
<li>For each thread, an affinity mask (a bit mask indicating what processor a thread should run on) is set by calling <strong>kernel32.SetThreadAffinityMask</strong>.</li>
<li>The main thread performs basic drive enumeration and calls <strong>PostQueuedCompletionStatus</strong>.</li>
</ul>
<figure id="attachment_21162" aria-describedby="caption-attachment-21162" style="width: 963px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-21162" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13ec028c07cf0_word-image-21147-15.png" alt="Calling CreatIoCompletionPort" width="963" height="294" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13ec028c07cf0_word-image-21147-15.png 963w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-15-300x92.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-15-768x234.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-15-150x46.png 150w" sizes="(max-width: 963px) 100vw, 963px"><figcaption id="caption-attachment-21162" class="wp-caption-text">Calling CreatIoCompletionPort</figcaption></figure>
<p>&nbsp;</p>
<figure id="attachment_21163" aria-describedby="caption-attachment-21163" style="width: 1148px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-21163" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e8bb6c07cf8_word-image-21147-16.png" alt="Retrieving processor count from PEB" width="1148" height="90" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e8bb6c07cf8_word-image-21147-16.png 1148w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-16-300x24.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-16-1024x80.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-16-768x60.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-16-150x12.png 150w" sizes="(max-width: 1148px) 100vw, 1148px"><figcaption id="caption-attachment-21163" class="wp-caption-text">Retrieving processor count from PEB</figcaption></figure>

<figure id="attachment_21164" aria-describedby="caption-attachment-21164" style="width: 944px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21164" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e2995c07cf5_word-image-21147-17.png" alt="PostQueuedCompletionStatus Function" width="944" height="54" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e2995c07cf5_word-image-21147-17.png 944w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-17-300x17.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-17-768x44.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-17-150x9.png 150w" sizes="(max-width: 944px) 100vw, 944px"><figcaption id="caption-attachment-21164" class="wp-caption-text">PostQueuedCompletionStatus Function</figcaption></figure>
<p>&nbsp;</p>
<p>Following APIs are used for drive enumeration on the system:</p>
<ul>
<li>kernel32.GetLogicalDriveStringsW</li>
<li>kernel32.GetDriveTypeW</li>
</ul>
<p>Further enumeration of files is performed by creating worker thread for <strong>PostQueuedCompletionStatus.</strong></p>
<figure id="attachment_21165" aria-describedby="caption-attachment-21165" style="width: 1023px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21165" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e3eb7c07cf6_word-image-21147-18.png" alt="Creation of worker thread for PostQueuedCompletionStatus" width="1023" height="399" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e3eb7c07cf6_word-image-21147-18.png 1023w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-18-300x117.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-18-768x300.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-18-150x59.png 150w" sizes="(max-width: 1023px) 100vw, 1023px"><figcaption id="caption-attachment-21165" class="wp-caption-text">Creation of worker thread for PostQueuedCompletionStatus</figcaption></figure>
<p>&nbsp;</p>
<p>The main thread calls <strong>mpr.WNetOpenEnumW</strong> for enumerating network resources and creates a worker thread same as above that performs the PostQueuedCompletionStatus call.</p>
<figure id="attachment_21166" aria-describedby="caption-attachment-21166" style="width: 923px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21166" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e1909c07d03_word-image-21147-19.png" alt="Calling mpr.WNetOpenEnumW function" width="923" height="205" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e1909c07d03_word-image-21147-19.png 923w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-19-300x67.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-19-768x171.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-19-150x33.png 150w" sizes="(max-width: 923px) 100vw, 923px"><figcaption id="caption-attachment-21166" class="wp-caption-text">Calling mpr.WNetOpenEnumW function</figcaption></figure>
<p>&nbsp;</p>
<h3><a id="post-21147-_vmh2kcdq9otw"></a>Worker Thread: PostQueuedCompletionStatus</h3>
<figure id="attachment_21167" aria-describedby="caption-attachment-21167" style="width: 1225px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-21167" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e4c07c07d00_word-image-21147-20.png" alt="The worker thread that performs the PostQueuedCompletionStatus" width="1225" height="630" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e4c07c07d00_word-image-21147-20.png 1225w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-20-300x154.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-20-1024x527.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-20-768x395.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-20-150x77.png 150w" sizes="(max-width: 1225px) 100vw, 1225px"><figcaption id="caption-attachment-21167" class="wp-caption-text">The worker thread that performs the PostQueuedCompletionStatus</figcaption></figure>
<p>&nbsp;</p>
<p>The newly created thread for PostQueuedCompletionStatus leads to the following:</p>
<ul>
<li>The files are enumerated via <strong>kernel32.FindFirstFileExW</strong> and<strong> kernel32.FindNextFileW</strong>.</li>
<li>If it is a directory, the thread function is recursively called to perform the file enumeration.</li>
<li>If it is a user file, then the absolute path is enqueued to the completion queue via <strong>PostQueuedCompletionStatus </strong>call.</li>
<li>This worker thread is responsible for gathering the files for encryption.</li>
</ul>
<h3><a id="post-21147-_98543zr3qugp"></a>Worker Thread: GetQueuedCompletionStatus</h3>
<p>This worker thread is responsible for doing the actual locking of the user files. The ransomware hides this thread from the debugger via <strong>ntdll.ZwSetInformationThread</strong> by passing ThreadHideFromDebugger as the ThreadInformationClass.</p>
<figure id="attachment_21168" aria-describedby="caption-attachment-21168" style="width: 1037px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-21168" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e8442c07cf3_word-image-21147-21.png" alt="Calling ntdll.ZwSetInformationThread function" width="1037" height="426" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead18c9d13e8442c07cf3_word-image-21147-21.png 1037w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-21-300x123.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-21-1024x421.png 1024w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-21-768x315.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/10/word-image-21147-21-150x62.png 150w" sizes="(max-width: 1037px) 100vw, 1037px"><figcaption id="caption-attachment-21168" class="wp-caption-text">Calling ntdll.ZwSetInformationThread function</figcaption></figure>

<p>The thread decodes the file extension â.blueskyâ and proceeds to perform the encryption. The <strong>kernel32.GetQueuedCompletionStatus</strong> is called in an infinite loop to retrieve the absolute path of the user data.</p>

<p>&nbsp;</p>
<p>The <strong>sub_288780 </strong>function is responsible for encrypting the data. The thread checks if the dequeued item is a directory or a file.</p>
<ul>
<li>If it is a file then it proceeds to encrypt the data by using the following APIs:
<ul>
<li>kernel32.CreateFileW</li>
<li>kernel32.SetFilePointer</li>
<li>kernel32.ReadFile</li>
<li>kernel32.WriteFile</li>
</ul>
</li>
<li>If the item is a directory then <strong>sub_28EDA0</strong> is executed to dump the ransom note. The file name strings are decoded dynamically.</li>
</ul>

<p>The note content generated by the ransomware is written on the disk by calling:</p>
<ul>
<li>kernel32.CreateFileW</li>
<li>kernel32.WriteFile</li>
</ul>

<h2><a id="post-21147-_v8a6xq1s0lgw"></a>Post Encryption</h2>
<p>Once the user data is successfully locked, the ransomware performs the following operations:</p>
<ul>
<li>Releases the mutex created at the initial phase</li>
<li>Sets the thread state to ES_Continous</li>
<li>Destroys the allocated heap</li>
<li>Exits the process via kernel32.ExitProcess</li>
</ul>

const elements = document.querySelectorAll('[ms-code-truncate]');

elements.forEach((element) => {
  const charLimit = parseInt(element.getAttribute('ms-code-truncate'));

  // Create a helper function that will recursively traverse the DOM tree
  const traverseNodes = (node, count) => {
    for (let child of node.childNodes) {
      // If the node is a text node, truncate if necessary
      if (child.nodeType === Node.TEXT_NODE) {
        if (count + child.textContent.length > charLimit) {
          child.textContent = child.textContent.slice(0, charLimit - count) + '...';
          return count + child.textContent.length;
        }
        count += child.textContent.length;
      }
      // If the node is an element, recurse through its children
      else if (child.nodeType === Node.ELEMENT_NODE) {
        count = traverseNodes(child, count);
      }
    }
    return count;
  }

  // Create a deep clone of the element to work on. This is so that we don't modify the original element
  // until we have completely finished processing.
  const clone = element.cloneNode(true);

  // Traverse and truncate the cloned node
  traverseNodes(clone, 0);

  // Replace the original element with our modified clone
  element.parentNode.replaceChild(clone, element);
});
