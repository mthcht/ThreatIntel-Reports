<p>Research indicates that a Ransomware attack occurs every 11 seconds roughly translating to an approximate 3 million attacks throughout the year. Ransomware attacks are no longer reserved events. Companies are at a constant threat to their revenue, data, brand, image, and subsequent shutdown of the business.</p>
<p><strong>Redeemer</strong> ransomware was initially identified in June 2021, and since then, four public versions (1.0, 1.5, 1.7, and 2.0) have been released. This article contains the technical analysis of the Redeemer ransomware and its various features.</p>
<h2><a id="post-20485-_hesdktt664n"></a><strong>Evolution of the Redeemer Ransomware 2.0</strong></h2>
<p>The threat actor, <strong>Cerebrate </strong>operating on a cybercrime forum named <strong>Dread</strong> has been actively promoting the Redeemer ransomware. They have recently started operating on the Breached forum and have released its latest version (version 2.0) on the same.</p>
<p>Redeemer has gone through four version changes since September 2021. The latest version includes improved graphical features such as a GUI builder interface, an icon change for encrypted files, a detailed instructions list, etc. The threat actor also claims to have added support for Windows 11 along with few cryptographic changes to the latest version. The image below describes the features added with each version release of the Redeemer ransomware.</p>

<h2><a id="post-20485-_lpvdtxbhvv6r"></a><strong>Details of the Ransomware</strong></h2>
<ul>
<li>This Ransomware is written in C++ and comes with a builder and decrypter executable.</li>
<li>It uses the following encryption algorithms:
<ul>
<li>AES256 is used to encrypt the files on the victimâs computer</li>
<li>RSA is used to encrypt the key</li>
</ul>
</li>
<li>The ransomware clones itself with the name of a system executable file (eg. <em>conhost.exe</em>), and creates a hidden folder for itself in the Windows directory.</li>
<li>It terminates all the running processes and executables which may pose a threat to the encryption routine.</li>
<li>It deletes all shadow copies of files and clears all event as well as application logs using <em>wevtutil, vssadmin</em>, and <em>wbadmin.</em></li>
<li>It uses multithreading in order to enumerate the filesystem and encrypt files. It creates 35 different threads that point to the same encryption routine.</li>
<li>It also modifies the Winlogon registry value and sets it to display the ransom note. Thus, when a user logs into the machine, the ransom note is displayed.</li>
</ul>
<h2><a id="post-20485-_l6rsfkxq0boe"></a><strong>Technical Analysis</strong></h2>
<h3><a id="post-20485-_rluz15nunnba"></a>Ransomware Signature</h3>
<p>The signature of this executable shows us that it is written in C++. When conducting the string analysis, multiple Base64 encoded strings were observed, some of which get decoded to the public key used for encryption, and powershell commands. Upon decoding one of these strings, the following translation was obtained: âRedeemer Ransomware &#8211; Your Data Is Encryptedâ.</p>

<h3><a id="post-20485-_e3u4ki4u9l1s"></a>Stage I &#8211; Pre-Encryption Operations</h3>
<h4><a id="post-20485-_u478y0cuhpii"></a>Mutex Creation</h4>
<p>Upon execution, Redeemer first hides its console window by using a call to the<strong> <em>ShowWindow </em></strong>Windows API. It then creates a Mutex, called the <strong><em>RedeemerMutex</em></strong><em>, </em>in order to make sure that multiple instances of the ransomware are not running on the same system.</p>
<figure id="attachment_20493" aria-describedby="caption-attachment-20493" style="width: 961px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="wp-image-20493" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead10e7c3202b9d3f3917_word-image-20485-8.png" alt="Code for hiding the process window and creation of the Mutex" width="961" height="106" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead10e7c3202b9d3f3917_word-image-20485-8.png 961w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20485-8-300x33.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20485-8-768x85.png 768w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20485-8-150x17.png 150w" sizes="(max-width: 961px) 100vw, 961px"><figcaption id="caption-attachment-20493" class="wp-caption-text">Code for hiding the process window and creation of the Mutex</figcaption></figure>

<h4><a id="post-20485-_y3jclmvpu5ws"></a>String Encoding</h4>
<p>An RSA public key, ransom amount, and contact email ID are then loaded as Base64 values into memory and decoded for further usage. This Ransomware heavily uses Base64 for string encoding purposes.</p>

<h3><a id="post-20485-_4fhf518j9l47"></a>Stage II &#8211; Preparing for Encryption</h3>
<p>The second stage of the ransomware is dictated by the transfer of control to a specific logic section that is controlled by the argument count value. This is done by moving itself under a different name to a <strong>world writable directory</strong> as shown in the image below.</p>

<h3><a id="post-20485-_2iyp0vl8zc91"></a>Encryption</h3>
<p>Redeemer is capable of enumerating and encrypting both local files and network-attached drives.</p>
<figure id="attachment_20502" aria-describedby="caption-attachment-20502" style="width: 703px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="wp-image-20502" src="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead10e7c320faa63f391f_word-image-20485-17.png" alt="The encryption loop" width="703" height="142" srcset="https://cdn.prod.website-files.com/635e632477408d12d1811a64/63bead10e7c320faa63f391f_word-image-20485-17.png 703w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20485-17-300x61.png 300w, https://cdn.cloudsek.com/wp-content/uploads/2022/09/word-image-20485-17-150x30.png 150w" sizes="(max-width: 703px) 100vw, 703px"><figcaption id="caption-attachment-20502" class="wp-caption-text">The ransomware encryption loop</figcaption></figure>

<ul>
<li>For the local files, it uses <strong><em><code>SHGetFolderPath</code> </em></strong></li>
<li>For network assets, it uses <strong><em><code>WNetEnumResource</code></em></strong>.</li>
</ul>
<p>It executes these operations using a loop with <strong><em><code>FindFirstFile</code></em></strong> and <strong><em><code>FindNextFile</code></em></strong>.</p>

<p>Research indicates that a Ransomware attack occurs every 11 seconds roughly translating to an approximate 3 million attacks throughout the year. Ransomware attacks are no longer reserved events. Companies are at a constant threat to their revenue, data, brand, image, and subsequent shutdown of the business.</p>
<p><strong>Redeemer</strong> ransomware was initially identified in June 2021, and since then, four public versions (1.0, 1.5, 1.7, and 2.0) have been released. This article contains the technical analysis of the Redeemer ransomware and its various features.</p>
<h2><a id="post-20485-_hesdktt664n"></a><strong>Evolution of the Redeemer Ransomware 2.0</strong></h2>
<p>The threat actor, <strong>Cerebrate </strong>operating on a cybercrime forum named <strong>Dread</strong> has been actively promoting the Redeemer ransomware. They have recently started operating on the Breached forum and have released its latest version (version 2.0) on the same.</p>
<p>Redeemer has gone through four version changes since September 2021. The latest version includes improved graphical features such as a GUI builder interface, an icon change for encrypted files, a detailed instructions list, etc. The threat actor also claims to have added support for Windows 11 along with few cryptographic changes to the latest version. The image below describes the features added with each version release of the Redeemer ransomware.</p>
