<div class="cmp-text" id="text-04b2c092b1" data-cmp-name="cmp-text-" data-cmp-is="text" data-target-location="false">
    
       <span><span style="	font-weight: 400;
">In this blog, we describe a campaign of recent activity where CrowdStrike observed an actor likely related to </span><a href="https://adversary.crowdstrike.com/en-US/adversary/carbon-spider.html"><span style="	font-weight: 400;
">CARBON SPIDER</span></a><span style="	font-weight: 400;

<h3><span style="	font-weight: 400;
">Pivoting to the Web Server to Confirm Initial Access Vector</span></h3>
<span style="	font-weight: 400;
">After reviewing the initial OverWatch alert, Falcon Complete pivoted to reviewing the IIS logs from the customer’s web server at the time of the incident to confirm the initial access vector as SQL Injection.<p> </p> Through analysis of the logs, Falcon Complete confirmed that the actor used</span><span style="	font-weight: 400;
"> the SQL </span><span style="	font-weight: 400;
"><code>xp_cmdshell</code></span><span style="	font-weight: 400;
"> command to execute multiple PowerShell (PS) payloads.</span>

<span style="	font-weight: 400;
">In an attempt to hide their intent and to possibly bypass security technologies, such as web-application firewalls, the actor made use of multiple encoding techniques within the SQL-injection statements. An example injection can be found here:</span>
<ul><li>
<pre><code>‘;dEClarE &#64;czyd VaRcHar(8000);SEt &#64;czyd&#61;0x6563686f2031;iNsERt inTo SqLMaPoUtPUT(data) eXec master..xp_cmdshell &#64;czyd--</code></pre>
</li></ul>
This injection finishes the legitimate SQL statement using a single quote (‘), proceeds to create the variable named <code>czyd</code>, sets its contents to the hexadecimal-encoded value of <code>0x6563686f2031</code> (which decodes to <code>echo 1</code>), and executes the variable using <code>eXec master..xp_cmdshell &#64;czyd</code>. The use of <code>iNsERt inTo SqLMaPoUtPUT(data)</code> indicates the actor likely leveraged the <em>sqlmap</em> tool to perform automated SQL injection against the target web application.

<span style="	font-weight: 400;
">CARBON SPIDER’s PowerShell payloads were executed using the encoding patterns described above within additional SQL-injection statements.</span>

<span style="	font-weight: 400;
">An interesting artifact recovered from the web server IIS logs showed the actor’s failed attempts to continue exploitation after the SQL server was network contained. After their remote download was blocked, the actor attempted to echo the </span><em><span style="	font-weight: 400;
">Demux</span></em><span style="	font-weight: 400;
"> PowerShell content into a file to execute, however the command failed due to the SQL server being contained. By reviewing the IIS logs, Falcon Complete was then able to provide the customer with specific information on the SQL Injection Vulnerabilities, and provide them with key information for fixing the vulnerable web page.</span>

<span style="	font-weight: 400;
">Falcon Complete began response within 6 minutes, contained the threat in under 30 minutes, and fully remediated the host within 1 hour and 45 minutes, with active support from Falcon OverWatch and CrowdStrike Intel. By stopping the breach before the actor could achieve their goals and cause any significant impact, Falcon Complete again demonstrated the critical importance of rapid response.</span>

<h2><span style="	font-weight: 400;
">Conclusion</span></h2>
<span style="	font-weight: 400;
">This incident described in this blog is attributed with low confidence to CARBON SPIDER, based on use of the </span><em><span style="	font-weight: 400;
">Demux</span></em><span style="	font-weight: 400;
"> Loader and stager DLL combination. CARBON SPIDER’s likely use of <a href="/cybersecurity-101/sql-injection.html">SQL injection</a> as an initial access method represents a new development, marking a departure from the typical indiscriminate spam campaigns the actor continues to conduct. Gaining code execution on an MSSQL server within an environment could provide an avenue for the actor to move laterally into the corporate network if compromised servers are not located in a Demilitarized Zone (DMZ).</span>

<span style="	font-weight: 400;
">This campaign demonstrates CARBON SPIDER’s willingness and ability to introduce different TTPs in order to compromise victims. It is likely the actor will continue to leverage both spam and server exploitation to achieve initial access in the near-term.</span>
