        <h1>Shlayer Malvertising Campaigns Still Using Flash Update Disguise</h1>
        
        <div class="publish_info">
            <p>July 19, 2021</p>
            <span>|</span>
            <a href="/en-us/blog/author.aspen-lindblom---joseph-goodwin---chris-sheldon/" title="Posts by Aspen Lindblom - Joseph Goodwin - Chris Sheldon" rel="author">
                Aspen Lindblom - Joseph Goodwin - Chris Sheldon</a>
            <span>|</span>
                
            <a href="/en-us/blog/category.engineering-and-technology/" title="Engineering &amp; Tech">Engineering &amp; Tech</a>
        </div>
        <div class="post_image">
            <img width="1060" height="698" src="https://assets.crowdstrike.com/is/image/crowdstrikeinc/Blog_1060x698-43-us?wid=1060&hei=698&fmt=png-alpha&qlt=95,0&resMode=sharp2&op_usm=3.0,0.3,2,0" class="attachment-post-thumbnail size-post-thumbnail wp-post-image" alt="" decoding="async" fetchpriority="high" srcset="/content/dam/crowdstrike/www/en-us/wp/2024/12/Blog_1060x698-43-us.jpeg 1060w" sizes="(max-width: 1060px) 100vw, 1060px"/>

<div class="text text--blog-content aem-GridColumn aem-GridColumn--default--12">



<div class="cmp-text" id="text-feaee63f36" data-cmp-name="cmp-text-" data-cmp-is="text" data-target-location="false">
    
       <span><span style="	font-weight: 400;
">Malvertising campaigns delivering Shlayer malware for macOS are still ongoing, despite the patching of a critical zero-day vulnerability (CVE-2021-30657) abused for months to compromise victims by dodging built-in OS protections such as Gatekeeper and also bypassing File Quarantine and Application Notarization. Recent Shlayer malvertising campaigns have gone back to using fake Flash updates and </span><a href="/en-us/cybersecurity-101/social-engineering-attacks/"><span style="	font-weight: 400;
">social engineering tactics</span></a><span style="	font-weight: 400;
"> to trick victims into manually installing the macOS malware and compromising their systems.</span>

<span style="	font-weight: 400;
">Although Flash Player reached end of life for macOS as of Dec. 31, 2020, this has not stopped Shlayer operators from continuing to abuse it. Shlayer operators may not be using a zero-day vulnerability anymore, but they’re still resourceful.</span>
<h2><span style="	font-weight: 400;
">Malvertising and How It Works</span></h2>
<span style="	font-weight: 400;
">As the internet has grown, so have the avenues it can be used to abuse end users. Websites now exist whose sole purpose is to redirect the end user to advertisements. Attackers have taken advantage of this by aggressively redirecting users to malicious content. Although these domains are often taken down very quickly, some attackers have found ways to stay under the radar by serving both legitimate and malicious advertisements, also known as </span><a href="/en-us/cybersecurity-101/malware/malvertising/"><span style="	font-weight: 400;
">malvertisements</span></a><span style="	font-weight: 400;
">.</span>

<span style="	font-weight: 400;
">In these cases, the attacker can decide whether you are redirected to malicious or non-malicious sites depending on a few factors such as your user-agents, IP address and whether this is your first visit to the site. Your user-agent is a way of identifying the browser, browser version and operating system. This allows the web server to render the site differently based on the browser and operating system you are using.</span>

<span style="	font-weight: 400;
">For example, if you’re on macOS using Chrome you could be sent to non-malicious websites associated with the attacker’s ad network, further generating ad revenue by falsely clicking on ad links through redirects. However, if you visit the same initial domain on a different browser (e.g., Safari) you will be redirected to the malicious website. In most cases, after the initial visit to the malicious domain, any additional visits will redirect to a parked domain. Domain parking allows for monetization to occur while the domain is “under construction,” giving the domain owner the ability to display links from ad affiliates.</span>

<span style="	font-weight: 400;
">These schemes range from attempting to trick you into calling “Technical Support” to remove a virus to tricking you into installing “Adobe Flash Player” after it “detects” that your machine is running an out-of-date version.</span>
<h2><span style="	font-weight: 400;
">Meet the macOS Shlayer<p> </p></span></h2>
<span style="	font-weight: 400;
">Shlayer, discovered in 2018, is constantly maintained and also evolving. The graph below is representative of Shlayer continually being a go-to piece of malware that attackers use to compromise the victim&#39;s machine. We observed an uptick in Shlayer detections occurring before the release of CVE-2021-30657 (the Gatekeeper bypass) that was being exploited by Shlayer. This vulnerability was subsequently patched on April 26, 2021.<p> </p></span>

<span style="	font-weight: 400;
">However, even without exploiting a </span><a href="/en-us/cybersecurity-101/cyberattacks/zero-day-exploit/"><span style="	font-weight: 400;
">zero-day</span></a><span style="	font-weight: 400;
">, the data suggests that Shlayer is still a popular tool used to infect a machine. Although our telemetry registered a drop in detections after the vulnerability was addressed, Shlayer operators resumed their campaigns days after, driving home the fact that it continues to evolve and it is constantly maintained.</span>

<div class="wp-caption" id="attachment_31188"><img class="size-full wp-image-31188" src="/content/dam/crowdstrike/www/en-us/wp/2021/07/image3-1.png" alt="" width="1823" height="591"/> Figure 1. Shlayer detections, April-June 2021</div>

<span style="	font-weight: 400;
">The largest number of Shlayer detections started in the second half of April 2021, with peaks on April 13, April 15 and April 21, coinciding with Shlayer using the CVE-2021-30657 vulnerability. The release of a patch for CVE-2021-30657 led to an immediate drop in Shlayer detections. However, in less than one month, Shlayer operators quickly adapted to overcome the patch addressing the exploited vulnerability and have gone back to old habits (fake Flash updates and social engineering tactics).</span>

<span style="	font-weight: 400;
">The most common method of Shlayer’s distribution is through malvertisements that redirect Safari users to sites displaying an alert about an out-of-date Adobe Flash Player. An example of a recent and ongoing malvertising campaign involves </span><code><span style="	font-weight: 400;
">approvedfornext&lt;.&gt;com</span></code><span style="	font-weight: 400;
">, which redirects Safari users to a site that displays this out-of-date Adobe Flash Player alert (see below).</span>

<div class="wp-caption" id="attachment_31189"><img class="size-full wp-image-31189" src="/content/dam/crowdstrike/www/en-us/wp/2021/07/image2-4.png" alt="" width="1015" height="674"/> Figure 2. Fake Adobe Flash Player update popup</div>

<span style="	font-weight: 400;
">Once the user clicks on this fake software update popup, Shlayer is then downloaded and mounted on the victim&#39;s machine. However, it still needs to be installed, which is where the attacker relies on the user&#39;s inability to spot the threat by leveraging social engineering tactics. To help increase the chances that the user will install this on the machine, the .dmg file displays easy-to-follow, step-by-step instructions on how they can do this, as shown in the image below.</span>

<div class="wp-caption" id="attachment_31190"><img class="size-full wp-image-31190" src="/content/dam/crowdstrike/www/en-us/wp/2021/07/image1.jpg" alt="" width="650" height="353"/> Figure 3. Social engineering the user into installing Shlayer</div>

<span style="	font-weight: 400;
">This is actually an image file that is hidden in the .dmg, and the Install icon is an alias. The image attempts to hide that from the unsuspecting user, but taking a peek under the hood will reveal this information.</span>

<div class="wp-caption" id="attachment_31191"><img class="size-full wp-image-31191" src="/content/dam/crowdstrike/www/en-us/wp/2021/07/Screen-Shot-2021-07-19-at-9.15.03-AM.png" alt="" width="1866" height="558"/> Figure 4. Using “ls -la” command to display hidden files and folders in the .dmg</div>

<span style="	font-weight: 400;
">The Install alias is pointing to </span><code><span style="	font-weight: 400;
">Install.command</span></code><span style="	font-weight: 400;
">, a script located in a hidden folder conveniently named .hidden. </span><code><span style="	font-weight: 400;
">Install.command</span></code><span style="	font-weight: 400;
"> will execute as the user follows the instructions. To understand what occurs during the installation, the contents of the .hidden folder and </span><code><span style="	font-weight: 400;
">Install.command</span></code><span style="	font-weight: 400;
"> files will need to be reviewed. The contents of this .hidden folder can be seen below.</span>

<div class="wp-caption" id="attachment_31192"><img class="size-full wp-image-31192" src="/content/dam/crowdstrike/www/en-us/wp/2021/07/Screen-Shot-2021-07-19-at-9.16.32-AM.png" alt="" width="1868" height="472"/> Figure 5. Using “ls -la” command to display files in the .hidden folder</div>

<span style="	font-weight: 400;
">There is another file in the .hidden folder. Looking at the script provides clues as to what will happen next.</span>

<div class="wp-caption" id="attachment_31193"><img class="size-full wp-image-31193" src="/content/dam/crowdstrike/www/en-us/wp/2021/07/Screen-Shot-2021-07-19-at-9.17.25-AM.png" alt="" width="1884" height="1274"/> Figure 6. Contents of <code>Install.command</code> script</div>

<span style="	font-weight: 400;
">The script contains a simple substitution cipher with Base64 encoding and AES encryption. Breaking down the script will make it easier to understand what actions will be performed by the script:</span>
<ul><li style="	font-weight: 400;
"><span style="	font-weight: 400;
">The first line of the script initializes and assigns letters to 10 variables that will be used by the substitution cipher to decode part of the command seen in the </span><code><span style="	font-weight: 400;
">decryptedFommand</span></code><span style="	font-weight: 400;
"> variable and nohup command.</span></li><li style="	font-weight: 400;
"><span style="	font-weight: 400;
">The </span><code><span style="	font-weight: 400;
">appDir</span></code><span style="	font-weight: 400;
"> variable will be set to /Volumes/Install/.hidden.</span></li><li style="	font-weight: 400;
"><span style="	font-weight: 400;
">The </span><code><span style="	font-weight: 400;
">tmpDir</span></code><span style="	font-weight: 400;
"> variable will be set to the temporary directory in /tmp created by mktemp -d.</span></li><li style="	font-weight: 400;
"><span style="	font-weight: 400;
">The </span><code><span style="	font-weight: 400;
">binFile</span></code><span style="	font-weight: 400;
"> variable will be set to the name of the only other file located in the .hidden directory, </span><code><span style="	font-weight: 400;
">uaQf9bkKsOGo</span></code><span style="	font-weight: 400;
">, but will be reversed to </span><code><span style="	font-weight: 400;
">oGOsKkb9fQau</span></code><span style="	font-weight: 400;
">.</span></li><li style="	font-weight: 400;
"><span style="	font-weight: 400;
">The archive variable will be set to </span><code><span style="	font-weight: 400;
">uaQf9bkKsOGo</span></code><span style="	font-weight: 400;
">.</span></li><li style="	font-weight: 400;
"><span style="	font-weight: 400;
">The </span><code><span style="	font-weight: 400;
">commandArgs</span></code><span style="	font-weight: 400;
"> variable contains a Base64-encoded and AES-encrypted command.</span></li><li style="	font-weight: 400;
"><code>decryptedFommand</code> will echo <code>commandArgs</code> and pipe it to openssl in order to decode and decrypt the command. The decrypted command will be:</li></ul>
<div class="wp-caption" id="attachment_31194"><img class="size-full wp-image-31194" src="/content/dam/crowdstrike/www/en-us/wp/2021/07/Screen-Shot-2021-07-19-at-9.21.04-AM.png" alt="" width="1882" height="242"/> Figure 7. <code>decryptedFommand</code> decrypted</div>

<span style="	font-weight: 400;
">The decrypted command is then passed to </span><code><span style="	font-weight: 400;
">nohup</span></code><span style="	font-weight: 400;
">, a utility that will invoke a command immune to hangup signals, with a substitution cipher that decodes to the following:</span>

<div class="wp-caption" id="attachment_31195"><img class="size-full wp-image-31195" src="/content/dam/crowdstrike/www/en-us/wp/2021/07/Screen-Shot-2021-07-19-at-9.22.35-AM.png" alt="" width="1874" height="312"/> Figure 8. <code>Nohup</code> decrypted</div>

<span style="	font-weight: 400;
">This will allow the decrypted command to continue to run until it is completed, even if the user logs off. Breaking down the command, it will decode and decrypt </span><code><span style="	font-weight: 400;
">uaQf9bkKsOGo</span></code><span style="	font-weight: 400;
">, saving it the temporary directory created in </span><code><span style="	font-weight: 400;
">/tmp</span></code><span style="	font-weight: 400;
"> as </span><code><span style="	font-weight: 400;
">oGOsKkb9fQau</span></code><span style="	font-weight: 400;
">. The &#34;</span><code><span style="	font-weight: 400;
">xattr -c</span></code><span style="	font-weight: 400;
">” command will clear all extended attribute flags from the temporary directory, including the </span><code><span style="	font-weight: 400;
">com.apple.quarantine</span></code><span style="	font-weight: 400;
"> flag that is added to all downloaded files. Clearing the quarantine flag allows the file to avoid notarization and Gatekeeper. The </span><code><span style="	font-weight: 400;
">chmod</span></code><span style="	font-weight: 400;
"> command will grant read, write and execute permissions to the file, followed by the file getting executed and the temporary folder getting deleted. Lastly, the </span><code><span style="	font-weight: 400;
">Install.command</span></code><span style="	font-weight: 400;
"> script will terminate all running Terminal processes.</span>

<span style="	font-weight: 400;
">The file dropped from Shlayer’s </span><code><span style="	font-weight: 400;
">Install.command</span></code><span style="	font-weight: 400;
"> script, </span><code><span style="	font-weight: 400;
">oGOsKkb9fQau</span></code><span style="	font-weight: 400;
">, is a Mach-O executable file known as Bundlore — adware that, amongst other things, will drop more adware families on the infected machine affecting your device’s performance and security.</span>
