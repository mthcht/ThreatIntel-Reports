								<h1>Darkside RaaS in Linux version</h1>
								<div class="date">June 22, 2021 &nbsp;|&nbsp;
									<a href="/blogs/author/ofer-caspi">Ofer Caspi</a></div>
							</div>
							<div class="blog-body">
								<h2>Executive summary</h2>

<p>LevelBlue Labs recently analyzed the Linux version of the Darkside ransomware, one of the most active ransomware in the last quarter. Shortly after hitting Colonial Pipeline, Darkside developers announced they would be closing operations.</p>

<h4>Key Points:</h4>

<ul>
	<li>Unlike common Linux ransomwares which mostly zip files with a password, Darkside encrypts files using crypto libraries. This likely makes recovery impossible without the encryption key, if properly implemented.</li>
</ul>

<h2>Background</h2>

<p>Linux and UNIX servers have always been a preferred option for servers and data centers, likely due to the small attack surface of the servers, tight configurations, and lack of user interaction. However, they are often set up and then forgotten, left without detection or protection mechanisms. This makes them very attractive to attackers. By infecting unprotected virtualization servers, attackers can perform devastating attacks on companies, taking down all the services of a company with a single infection.</p>

<p>First discovered in August 2020, Darkside is a group that operates ransomware as a service (RaaS), and attacks by their network of affiliates have infected many companies worldwide. The most high profile infection happened when a <a href="http://www.bbc.com/news/business-57050690" target="_blank">Darkside affiliate hit the network of one of the major oil pipeline companies</a>&nbsp;in the US, Colonial Pipeline, causing the company to shut down operations for days. After this attack &mdash; likely due to the fact that it drew unwanted levels of attention to the group globally &mdash; the malware authors announced publicly the closure of their service. Nevertheless, there is evidence that the group has completed a Linux version of its malware that is targeting ESXi, servers hosting VMware virtual machines. To this point, the authors announced the Darkside 2.0 version with Linux capabilities on March 9, 2021 in the XSS Forum:</p>

<p style="text-align:center"><img alt="Xss Forum" src="https://cdn.levelblue.com/blog-content/xss_forum.jpg" /></p>

<p style="text-align:center">Figure 1. Example of Linux version features and information announced by malware authors on XSS Forum. &nbsp;</p>

<h2>Analysis</h2>

<p>Unlike the Windows version of the malware that targets any Windows endpoint, Darkside Linux version is mostly targeting ESXi servers. Its default configuration includes the root path of ESX server machines. &nbsp;Targeted extensions are &#39;vmdk&#39;, &#39;log&#39;, &#39;vmem&#39;, &#39;vmsn&#39; that are used in ESX servers for saving virtual machines information, data, and logs.</p>

<p>The malware is quite informative and prints to the screen most of the actions it performs, which is an uncommon behavior for malware. This could imply that the malware is being deployed manually. The malware is written in C++ and it&rsquo;s also using several open-source libraries that were imported and compiled with the malware code into one binary. Some of these libraries are: <a href="https://www.cryptopp.com/" target="_blank">crypto++</a>&nbsp;used for encryption, <a href="https://www.boost.org/doc/libs/1_63_0/libs/log/doc/html/index.html" target="_blank">boost</a>&nbsp;used for logging purposes, and <a href="https://github.com/curl/curl" target="_blank">curl</a>&nbsp;used for HTTP requests. The use of these libraries enables the final binary to be a 2.7MB file size. For example, to communicate with an infected machine with its Command and Control (C&amp;C), the malware uses libcurl functions that were compiled with the rest of the code.</p>

<p>In addition, the malware supports command line parameters during execution to replace almost, if not all, of the default configuration variables.</p>

<p><img alt="malware parameters" src="https://cdn.levelblue.com/blog-content/parameters.jpg" /></p>

<p style="text-align:center">Figure 2. Parameters supported by the malware.</p>

<p>The malware also supports shutting down virtual machines by executing the esxcli commands, a special console on ESX servers that allows it to interact with virtual machines from the command line.</p>

<p><img alt="abusing malware" src="https://cdn.levelblue.com/blog-content/abusing_1.png" /></p>

<p><img alt="abusing malware" src="https://cdn.levelblue.com/blog-content/abusing_2.jpg" /></p>

<p style="text-align:center">Figure 3. Abusing esxcli command.</p>

<p>When executed, the malware prints its configuration to the terminal. This includes the root path to encrypt, RSA key information, targeted file extensions to encrypt, C2 addresses, and more, as seen in figure 4.</p>

<p><img alt="Darkside configuration" src="https://cdn.levelblue.com/blog-content/darkside_configuration.jpg" /></p>

<p style="text-align:center">Figure 4. Darkside configuration.</p>

<p>The C&amp;C addresses are encrypted using a rotated XOR key, which will be decrypted when the malware is executed. The decrypted addresses can be found in Appendix A:</p>

<p><img alt="decryption loop" src="https://cdn.levelblue.com/blog-content/decryption_loop.png" /></p>

<p style="text-align:center">Figure 5. Decryption loop to extract C&amp;C.</p>

<p><img alt="malware XOR key" src="https://cdn.levelblue.com/blog-content/malware_XOR_key.jpg" /></p>

<p style="text-align:center">Figure 6. Malware XOR key.</p>

<p>The malware will then count the files to be encrypted, and it collects information from the infected machine, sending it to the C&amp;C server after encryption. The exfiltrated information includes: user name, OS version, hostname, build, and more:</p>

<p><img alt="collecting system information" src="https://cdn.levelblue.com/blog-content/collecting_system_information.jpg" /></p>

<p style="text-align:center">Figure 7. Collecting system information.</p>

<p>Figure 8 shows an example of the information collected from an infected machine.</p>

<p><img alt="infected machine" src="https://cdn.levelblue.com/blog-content/infected_machine.jpg" /></p>

<p style="text-align:center">Figure 8. Example of collected system information.</p>

<p>The malware will loop through the files to be encrypted and then encrypt them using ChaCha20 with the RSA 4096 key taken from the configuration. After encryption, the malware will add a tail at the end of the encrypted file that includes a constant and the cipher, as seen in figure 9.</p>

<p><img alt="adding header" src="https://cdn.levelblue.com/blog-content/adding_header.jpg" /></p>

<p style="text-align:center">Figure 9. Adding header after successful encryption</p>

<p>After encryption, the malware creates a ransom note in each folder where files were encrypted.</p>

<p><img alt="ransom note" src="https://cdn.levelblue.com/blog-content/ransom_note.jpg" /></p>

<p style="text-align:center">Figure 10. Ransom note</p>

<h2>Recommended actions</h2>

<ol>
	<li>Keep software up to date with security updates.</li>
	<li>Carefully monitor and manage suspicious emails.</li>
	<li>Use a backup system to backup server files.</li>
	<li>Install Antivirus and/or endpoint detection and response (EDR) in all endpoints.</li>
	<li>Make sure two-factor authentication is enabled in all services.</li>
</ol>

<h2>Conclusion</h2>

<p>Ransomwares remains one of the biggest threats to companies globally, especially when it comes to virtual machine servers that may contain multiple machines that are primary targets for Darkside malware.</p>

<p>Darkside will search files on the main folder of the infected server and encrypt any file that matches its configuration limits. It will also collect information from the system and send it to its C&amp;C.</p>

<p>The following associated detection methods are in use by LevelBlue Labs. They can be used by readers to tune or deploy detections in their own environments or for aiding additional research.</p>
