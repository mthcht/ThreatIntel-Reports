<h2>Summary</h2>

<p>LevelBlue Labs&trade; has recently observed the presence of a new remote access trojan (RAT) malware in its threat analysis systems. The malware, known as FatalRAT (Firstly&nbsp;<a href="https://urldefense.com/v3/__https://twitter.com/c3rb3ru5d3d53c/status/1396821380256178179__;!!BhdT!0VBQ4oPQtWljVtTRSQfB5C_O6U46lbDFbQBZM068w7b7wckv_yV2zHNmEGV5XQ$" style="color:#1155cc; font-family:Arial, Helvetica, sans-serif; font-size:small; background-color:#ffffff" target="_blank" rel="noopener">named</a>&nbsp;by @c3rb3ru5d3d53c), appears to be distributed via forums and Telegram&nbsp;channels, hidden in download links that attempt to lure the user via software or media articles.</p>

<h2>Key takeaways:</h2>

<ul>
	<li>LevelBlue Labs performed a malware analysis of the FatalRAT threat.</li>
	<li>We have observed a new spreading mechanism via Telegram channels.</li>
	<li>Analyzed samples are capable of performing defense evasion techniques, obtaining system persistence, logging user keystrokes, collecting system information, exfiltrating over encrypted command and control (C&amp;C) channel.</li>
</ul>

<h2>Analysis</h2>

<p>FatalRAT is a remote access trojan with a wide set of capabilities that can be executed remotely by an attacker. The malware runs several tests before fully infecting a system, checking the existence of multiple virtual machine products, disk space, number of physical processors, and more (<a href="https://attack.mitre.org/techniques/T1497/001/" target="_blank" rel="noopener">T1497.001</a>).</p>

<p style="text-align:center"><img alt="Fatalrat analysis" src="https://cdn.levelblue.com/blog-content/fatalrat_analysis.jpg" /></p>

<p style="text-align:center">Figure 1. AntiVM techniques from a system report.</p>

<p>One of the tests run by FatalRAT involves checking for existence of virtual machine services, as shown in figure 2.</p>

<p style="text-align:center"><img alt="antivm techniques" src="https://cdn.levelblue.com/blog-content/antiVM_techniques.jpg" />Figure 2. AntiVM techniques includes searching for services.</p>

<p>Another includes querying the registry, as shown in figure 3.</p>

<p style="text-align:center"><img alt="FatalRAT registry" src="https://cdn.levelblue.com/blog-content/fatalrat_registry.jpg" /></p>

<p style="text-align:center">Figure 3. AntiVM techniques - registry check.</p>

<p>If the machine passes the malware AntiVM tests, FatalRAT will then starts its malicious activity.</p>

<p>First, it decrypts each of the configuration strings separately (<a href="https://attack.mitre.org/techniques/T1027/">T1027</a>). These configuration strings include the Command and Control (C&amp;C) address, new malware file name, service name, and other settings.</p>

<p style="text-align:center"><img alt="FatalRAT decryption" src="https://cdn.levelblue.com/blog-content/FatalRAT_decryption.jpg" /></p>

<p style="text-align:center">Figure 4. Decrypting configuration strings.</p>

<p>Figure 5 shows the routine used by FatalRAT for decrypting strings.&nbsp;</p>

<p style="text-align:center"><img alt="FatalRAT decryption strings" src="https://cdn.levelblue.com/blog-content/FatalRAT_decryption_strings.jpg" /></p>

<p style="text-align:center">Figure 5. Strings decryption routine.</p>

<p>The malware will disable the ability to lock the computer by using CTRL+ALT+DELETE. For this purpose, the registry key DisableLockWorkstation is set to &ldquo;1&rdquo; as shown in figure 6 (<a href="https://attack.mitre.org/techniques/T1112/" target="_blank" rel="noopener">T1112</a>).</p>

<p style="text-align:center"><img alt="FatalRAT disable lockdown" src="https://cdn.levelblue.com/blog-content/FatalRAT_disable_lockdown.jpg" /></p>

<p style="text-align:center">Figure 6. Disable computer lockdown.</p>

<p>After the computer lockdown is disabled, the malware activates a keylogger (<a href="https://attack.mitre.org/techniques/T1056/001/" target="_blank" rel="noopener">T1056.001</a>). See figure 7.</p>

<p style="text-align:center"><img alt="FatalRAT keylogger" src="https://cdn.levelblue.com/blog-content/FatalRAT_keylogger.jpg" /></p>

<p style="text-align:center">Figure 7. FatalRat keylogger.</p>

<p>FatalRat can persist either by modifying the registry (<a href="https://attack.mitre.org/techniques/T1547/001/" target="_blank" rel="noopener">T1547.001</a>) or by creating a new service (<a href="https://attack.mitre.org/techniques/T1569/002/" target="_blank" rel="noopener">T1569.002</a>). If persistence is done by modifying the registry, it will create the value &#39;Software\Microsoft\Windows\CurrentVersion\Run\SVP7&#39; to execute the malware at boot time. When using setting service for persistence, FatalRat will retrieve the description from its configuration as seen above in figure 4.</p>

<p style="text-align:center"><img alt="FatalRAT registry persistence" src="https://cdn.levelblue.com/blog-content/FatalRAT_registry_persistence.jpg" /></p>

<p style="text-align:center">Figure 8. FatalRat registry persistence.</p>

<p>The malware collects information from an infected machine and sends it to the C&amp;C (<a href="https://attack.mitre.org/techniques/T1020/" target="_blank" rel="noopener">T1020</a>), as shown in figure 9. This includes external IP address, username, and other information about the victim.</p>

<p>Additionally, as a defense evasion technique, FatalRAT identifies all security products running on the machine by iterating through all running processes and searching for the existence of a predefined list of security products (<a href="https://attack.mitre.org/techniques/T1562/001/" target="_blank" rel="noopener">T1562.001</a>), as shown in figure 10.</p>

<p style="text-align:center"><img alt="FatalRAT collecting information" src="https://cdn.levelblue.com/blog-content/Fatalrat_collecting_information.jpg" /></p>

<p style="text-align:center">Figure 9. Collecting information.</p>

<p style="text-align:center"><img alt="security product search" src="https://cdn.levelblue.com/blog-content/security_product_search.jpg" /></p>

<p style="text-align:center">Figure 10. Security product process search.</p>

<p>Furthermore, to make it easier for the attacker to detect which security products are installed, it will convert the process name above to a product name before sending the list to the C&amp;C. See figure 11.</p>

<p style="text-align:center"><img alt="security product name" src="https://cdn.levelblue.com/blog-content/security_product_name.jpg" /></p>

<p style="text-align:center">Figure 11. Security product process to name.</p>

<p>To communicate back to the C&amp;C, the malware uses an arithmetic routine to encrypt the data sent between the victim and the attacker. This encryption includes a one-byte XOR key and the addition of a constant to the obtained value (121 in this case) (<a href="https://attack.mitre.org/techniques/T1573/001/" target="_blank" rel="noopener">T1573.001</a>). See figure 12.</p>

<p style="text-align:center"><img alt="FatalRat c&amp;c server" src="https://cdn.levelblue.com/blog-content/fatalrat_cc.jpg" /></p>

<p style="text-align:center">Figure 12. C&amp;C communication decrypting routine.</p>

<p>The encrypted message is then sent to the C&amp;C as seen in the config from figure 4 through port 8081.</p>

<p style="text-align:center"><img alt="FatalRAT communication with c&amp;c" src="https://cdn.levelblue.com/blog-content/fatalrat_cc_communication.jpg" /></p>

<p style="text-align:center">Figure 13. Encrypted communication between infected host and C&amp;C.</p>

<p>The malware then waits for the attacker&rsquo;s commands, of which it supports a wide range. For example, it has several routines to handle different browsers. Some of these routines include deleting user info for specific browsers (Edge, 360Secure Browser, QQBrowser, SogouBrowser, and Firefox). For Chrome, it will query for user info and then delete the content. Deleting saved information will force the user to input, for example, user and password which the malware can capture with its keylogger. (<a href="https://attack.mitre.org/techniques/T1119/" target="_blank" rel="noopener">T1119</a>)</p>

<p style="text-align:center"><img alt="parsing FatalRAT c&amp;c" src="https://cdn.levelblue.com/blog-content/parsing_cc.jpg" /></p>

<p style="text-align:center">Figure 14. Parsing C&amp;C command.</p>

<p>Some of the commands FatalRat supports are shown below.</p>

<p>Spreading itself on the victim network by brute-forcing weak passwords through IPC$. If successful, the malware copies itself to the dedicated folder as %Folder%\hackshen.exe and will execute the copied file remotely. See figures 15-17.</p>

<p style="text-align:center"><img alt="FatalRAT attacking weak passwords" src="https://cdn.levelblue.com/blog-content/fatalrat_attacking_weak_passwords.jpg" /></p>

<p style="text-align:center">Figure 15. Weak user and password for brute force.</p>

<p style="text-align:center"><img alt="FatalRAT brute force loop" src="https://cdn.levelblue.com/blog-content/fatalrat_brute_force_loop.jpg" /></p>

<p style="text-align:center">Figure 16. Brute force loop.</p>

<p style="text-align:center"><img alt="FatalRAT after brute force attack" src="https://cdn.levelblue.com/blog-content/fatalrat_after_brute_force_attack.jpg" /></p>

<p style="text-align:center">Figure 17. Copy and execute remote computer on successful brute-force attack.</p>

<p>Either steal stored data (for chrome for example) or delete saved information with a handler for different browsers. See figure 18.</p>

<p style="text-align:center"><img alt="malware handlers" src="https://cdn.levelblue.com/blog-content/malware_handlers.jpg" /></p>

<p style="text-align:center">Figure 18. Malware handlers for different browser.</p>

<p>FatalRat also includes additional commands, for the following:</p>

<ul>
	<li>Keylogger</li>
	<li>Change resolution</li>
	<li>Uninstall UltraViewer</li>
	<li>Download and install AnyDesk</li>
	<li>Execute shell commands</li>
	<li>Modify registry keys</li>
	<li>Download and execute a file</li>
</ul>

<h2>Recommended actions</h2>

<ol>
	<li>Do not click links or install software from unknown sources.</li>
	<li>Install an antivirus and keep your system updated.</li>
	<li>Always use an endpoint detection and response (EDR) solution or enable system log monitoring to allow SIEM correlation.</li>
	<li>Monitor network traffic to command and control (C&amp;C) patterns.</li>
</ol>

<h2>Conclusion</h2>

<p>The newly identified FatalRat malware has been using techniques like obfuscation, anti-sandbox and antivirus evasion, encrypted configurations, logging user keystrokes, system persistence, login brute force, collection of system data, and encrypted communications with command and control server. LevelBlue Labs has discovered multiple samples in the past few months, with a slight dip in July. However, we expect to continue to see the presence of FatalRat and its variants in our samples in the near future. LevelBlue&nbsp;Labs will continue to monitor this threat and update intelligence as activity emerges.</p>
