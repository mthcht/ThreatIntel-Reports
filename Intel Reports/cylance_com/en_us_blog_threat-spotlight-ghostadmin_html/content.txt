    <div class='cylance-blog-container'><h2>Introduction<b></b></h2> <p>Data breaches are constantly in the news these days, targeting everyone from local restaurants to Fortune 500 companies. User accounts, names, credit card numbers, even your social security number are stored (you hope securely) in databases across the globe by numerous different companies. Your data is stored with your understanding that it will be protected and kept private from those who might abuse it.&nbsp;Unfortunately, time and time again, we discover that’s not always the case.</p> <p>With all these breaches in the media, it’s easy to forget that big companies aren’t the only target. Malware is just as capable of stealing a database on an enterprise server as it is stealing a single user’s credentials and data from their personal laptop. You just don’t typically hear about that kind of breach in the news. GhostAdmin 2.0 is a botnet <a href="https://www.bleepingcomputer.com/news/security/new-ghostadmin-malware-used-for-data-theft-and-exfiltration/" target="_blank" adhocenable="false">found recently by MalwareHunterTeam</a>&nbsp;(also known as Ghost iBot). It is capable of anything from stealing files to gaining full remote access, and all this is done using standard Windows libraries. Since its discovery in mid-January of 2017, new variants have been showing up on an almost weekly basis, and all with low conviction ratings by traditional antivirus (AV) products.</p> <h2>Code: Keylogging and Screen Capturing</h2> <p>Upon gaining entry, GhostAdmin doesn’t waste any time, immediately starting a new thread for its keystroke logging (<i>Figure 1</i>). In order to store these logs, GhostAdmin attempts to hide them, pretending they are logs for the AV product Symantec Protection. It stores them in a folder created under “C:\Users\[current user]\AppData\Roaming\Symantec\”.</p> <p style="text-align: center;"><img width="100%" src="/content/dam/blogs-blackberry-com/images/blogs/2017/03/Fig1_GhostAdmin.png"><i style="font-size: 15px; font-weight: 300;"><br> Figure 1: GhostAdmin EntryPoint Getting Down to Business</i></p> <p>The log is stored in an easily readable HTML file, going as far as recording keystrokes specific to the active window (<i>Figure 2</i>):</p> <p style="text-align: center;"><img width="100%" src="/content/dam/blogs-blackberry-com/images/blogs/2017/03/Fig2_GhostAdmin.png"><i style="font-size: 15px; font-weight: 300;">Figure 2: GhostAdmin's Usage of HTML for Keystroke Logs</i></p> <p>Each time GhostAdmin runs, it downloads an update for its settings file. This settings file contains all the necessary command and control (C2) information for the client to connect to the newest server. This makes it easy for the server to be migrated regularly, making it easier to avoid detection-based AV on IP or URL blacklists. This settings file is base64-encoded and easily decodes into a standard format. The malware uses standard coding practices to read these settings from the file (Figure 3).</p> <p>Here is an example of the settings information:</p> <p><a></a><a></a><a></a><a></a><a>&lt;quotebox&gt;</a><br> FTP Server: accounts-security-settings(dot)com<br> FTP User: renard(at)accounts-security-settings.com<br> FTP Password: gho$t(at)dmin<br> IRC Server: irc.blafasel(dot)de<br> IRC Port: 6667<br> IRC Channel: #bobby7<br> Mail Sender: bucketshovel76(at)gmail.com<br> Mail Recipient: bucketshovel76(at)gmail.com<br> Mail Password: bigbobby12<br> FTP Folder: renard<br> Screen Image Timer: 300000000<br> Log File Timer: 36000000<br> GhostAdmin2.0<br> &lt;/quotebox&gt;</p> <p style="text-align: center;"><img width="100%" src="/content/dam/blogs-blackberry-com/images/blogs/2017/03/Fig3_GhostAdmin.png"><br> <i style="font-size: 15px; font-weight: 300;">Figure 3: Straightforward Processing of the Downloaded Settings File</i></p> <p>GhostAdmin then sets a timer for one hour. This triggers an upload of the most recent logs to the FTP server. This transfer is straightforward, with a web client being created and used to upload the files (<i>Figure 4</i>). In this case, there is even an error handling step, throwing any errors into the C2 channel:</p> <p style="text-align: center;"><img width="100%" src="/content/dam/blogs-blackberry-com/images/blogs/2017/03/Fig4_GhostAdmin.png"><br> <i style="font-size: 15px; font-weight: 300;">Figure 4: Simple WebClient Code to Upload Keystroke Logs</i></p> <p>Creating yet another directory, GhostAdmin uses the 'CopyFromScreen' function from Drawing.dll to take complete captures of the user’s screen. Each capture is indexed by user name and&nbsp; a timestamp and saved as a .jpg. Immediately after the screenshot is taken, the file is uploaded to the FTP server and deleted from the local host. These captures are taken on regular intervals as determined by the ScreenImageTimer variable in the settings.ini file once it is downloaded. Again, the author outputs any errors to the C2.</p> <h2>Code: IRC</h2> <p>The final thread created is for GhostAdmin’s C2 channel and is used to establish an IRC channel. This portion of GhostAdmin accounts for, by far, the largest portion of the code and functionality. The exact IRC channel and server are derived from the settings.ini file downloaded by the second thread. Then a TcpClient instance connects and sends its logon messages to the C2 and even emails the author with Machine Name, User, IP, and client version (<i>Figure 5</i>).</p> <p style="text-align: center;"><img width="100%" src="/content/dam/blogs-blackberry-com/images/blogs/2017/03/Fig5_GhostAdmin.png"><br> <i style="font-size: 15px; font-weight: 300;">Figure 5: Email Function to Notify Author That a GhostAdmin Bot is Online</i></p> <p>After a quick ‘PING’ – ‘PONG’ to check for life on the IRC server, the client connects to the appropriate channel and starts listening for the bot master to send text starting with ‘PRIVMSG,’ indicating the master is giving a command. This IRC client is kept alive, reconnecting every one minute if it fails.<br> </p> <p>The list of commands is quite large in GhostAdmin. A full list is at the end of this blog, but for now, here is a high-level overview of the functions and their capabilities.</p> <h2>Host Information</h2> <p><i>GetIP, version, platform, checkfie, checkfolder, drives, tasklist, ipconfig, os, user, idletime.</i><br> These are functions designed to gather basic information about the host the client is running on, from the IP to currently running processes.</p> <h2>Data &amp; Audio Exfiltration</h2> <p><i>Logfile, readfile, getfiles, uploadfile, screenshot, audio<br>  </i>These functions upload data to either the C2 or the FTP server.</p> <p><b>The function AUDIO is worth noting, as it uses winmm.dll to access the machine’s default microphone and start recording for the specified amount of time</b> (<i>Figure 6</i>). This could be troublesome for an enterprise if the machine is kept in a location where sensitive information is discussed, for instance, near a corporate boardroom or in an executive team member’s office. Typical of GhostAdmin, the audio file is immediately uploaded to the FTP server and then deleted from the local host.<br> </p> <p style="text-align: center;"><img width="100%" src="/content/dam/blogs-blackberry-com/images/blogs/2017/03/Fig6_GhostAdmin.png"><br> <i style="font-size: 15px; font-weight: 300;">Figure 6: AUDIO Command from GhostAdmin That Records From Default Microphone</i></p> <h2>Interaction</h2> <p><i>Turn(on/off)monitor, visit, download, delete, run, taskkill, kill, copy, enableremoteDesktop, shutdownWindows, restartWindows, (enable/disable)inputDevices, deleteLogs, deleteBrowserData, SQL*, Update<br>  </i>Functions capable of interacting with the infected host, these give the malicious actor several options for access. This includes browsing or downloading from URLs, killing the client or any process (taskkill), and restarting or powering off the host.&nbsp;EnableremoteDesktop modifies the registry value at 'HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server' to allow remote desktop connections to the infected host <i style="font-size: 15px; font-weight: 300;">(Figure 7</i>).</p> <p style="text-align: center;"><img width="100%" src="/content/dam/blogs-blackberry-com/images/blogs/2017/03/Fig7_GhostAdmin.png"><br> <i style="font-size: 15px; font-weight: 300;">Figure 7: GhostAdmin Enables Remote Desktop via Registry Modification</i></p> <p><i>DisableInputDevices</i><br> This uses the user32.dll function BlockInput to disable both mouse and keyboard interaction on the local host, essentially removing all control from the User. It is worth noting that in this situation hitting <i>Ctrl+Alt+Delete</i> will override this block. This is a security feature built into Windows, and throws a hard system error.</p> <p><i>SQL*</i><br> This group of functions allows the malicious actor to manipulate any SQL database that the infected user has access to. It first searches for any databases through System.data.sqlClient. If found it attempts login with credentials supplied through the IRC command. Once logged in, the ability to run select, insert, delete, and update are available.</p> <p><i>DeleteBrowserData</i><br> This iterates through the 5 most common browsers, and deletes all session and user data (<i>Figure 8</i>). This is likely to force the user to log back into all their accounts, allowing GhostAdmin to capture as many of their credentials as possible.</p> <p style="text-align: center;"><img width="100%" src="/content/dam/blogs-blackberry-com/images/blogs/2017/03/Fig8_GhostAdmin.png"><br> <i style="font-size: 15px; font-weight: 300;">Figure 8: GhostAdmin Can Delete Browser Data From Five Major Browsers</i></p> <h2>GhostAdmin: Incomplete</h2> <p>As it turns out, this appears to not be the final version of GhostAdmin, as there are a few functions that are marked as “coming soon” (<i>Figure 9</i>).<br> </p> <p style="text-align: center;"><img width="100%" src="/content/dam/blogs-blackberry-com/images/blogs/2017/03/Fig9_GhostAdmin.png"><br> <i style="font-size: 15px; font-weight: 300;">Figure 9: Apparently, GhostAdmin Even Has a Roadmap For Features!</i></p> <h2>Conclusion<b></b></h2> <p><b>GhostAdmin contains all the necessary functionality to gain complete control over a victim’s computer.</b> It is just as capable of capturing credentials from a single web browser form as it is modifying or downloading a large database.</p> <p>With new variants being released with some frequency, and an incomplete codebase, GhostAdmin shows that it doesn’t take complex malware to be effective. While we have been unable to find a live C2 to analyze, MalwareHunterTeam indicates that even with its small presence, <a href="https://twitter.com/malwrhunterteam/status/822525690108968961" target="_blank" adhocenable="false">hundreds of GBs</a> of data have been stolen from both personal and corporate machines. We can assume that as this malware matures, variants will become both harder to detect and sent in even bigger deployments.</p> <p><i>If you use our endpoint protection product,&nbsp;</i><a href="https://www.cylance.com/en_us/products/our-products/protect.html" target="_blank" adhocenable="false"><i>CylancePROTECT®</i></a><i>, you were&nbsp;<a href="https://www.cylance.com/en_us/blog/cylance-vs-ghostadmin.html" target="_blank" adhocenable="false">already protected</a> from this attack.
