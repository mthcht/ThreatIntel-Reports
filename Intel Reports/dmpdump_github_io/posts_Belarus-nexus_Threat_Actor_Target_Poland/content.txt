data-toc-skip>Likely Belarus-Nexus Threat Actor Delivers Downloader to Poland</h1><div class="post-meta text-muted"> <span> Posted <time data-ts="1752192000" data-df="ll" data-bs-toggle="tooltip" data-bs-placement="bottom" > Jul 11, 2025 </time> </span> <span> Updated <time data-ts="1752354372" data-df="ll" data-bs-toggle="tooltip" data-bs-placement="bottom" > Jul 12, 2025 </time> </span><div class="d-flex justify-content-between"> <span> By <em> <a href="https://github.com/dmpdump">dmpdump</a> </em> </span><div> <span class="readtime" data-bs-toggle="tooltip" data-bs-placement="bottom" title="1295 words" > <em>7 min</em> read</span></div></div></div></header><div class="content"><p>On June 30, 2025, a file named <code class="language-plaintext highlighter-rouge">deklaracja.chm</code> (“declaration.chm”) was uploaded to VirusTotal from Poland.</p><p><a href="/assets/images/Belarus_Poland/vtsubmission.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/vtsubmission.png" alt="sshot" loading="lazy"></a></p><p>The file is a Microsoft Compiled HTML Help file, a proprietary online help format which consists of a binary that contains a collection of compressed files, including HTML files and other objects. Decompressing <code class="language-plaintext highlighter-rouge">deklaracja.chm</code>, we can see the following files:</p><p><a href="/assets/images/Belarus_Poland/content.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/content.png" alt="sshot" loading="lazy"></a></p><p>The files with the <code class="language-plaintext highlighter-rouge">#</code> prefix are standard CHM files. The rest of the files are malicious and lead to an infection chain that drops, extracts, and executes a C++ downloader.</p><ul><li><code class="language-plaintext highlighter-rouge">index.htm</code> is an HTML file with an obfuscated JS script that starts the infection chain<li><code class="language-plaintext highlighter-rouge">desktop.mp3</code> is cabinet file which contains a downloader DLL<li><code class="language-plaintext highlighter-rouge">deklaracja.png</code> is a decoy image of a bank transfer receipt from PKO Bank, a Polish bank</ul><p>CHM files are executed by <code class="language-plaintext highlighter-rouge">hh.exe</code> by default. When <code class="language-plaintext highlighter-rouge">deklaracja.chm</code> is executed, we see a decoy image displayed to the victim. We can also observe a series of processes that lead to the extraction and execution of a DLL named <code class="language-plaintext highlighter-rouge">unt32.dll</code>.</p><p><a href="/assets/images/Belarus_Poland/decoy.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/decoy.png" alt="sshot" loading="lazy"></a></p><p><a href="/assets/images/Belarus_Poland/infectionchain.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/infectionchain.png" alt="sshot" loading="lazy"></a></p><h1 id="infection-chain">Infection chain</h1><p>Reviewing the content of <code class="language-plaintext highlighter-rouge">index.htm</code>, we can immediately see a common pattern in code obfuscated with <a href="https://obfuscator.io/">obfuscator.io</a>, including the presence of <code class="language-plaintext highlighter-rouge">_0x</code> in and runtime string retrieval from array indices.</p><p><a href="/assets/images/Belarus_Poland/html1.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/html1.png" alt="sshot" loading="lazy"></a></p><p>After deobfuscating that script, we get a simple hex decoder for a large hex string (truncated for readability). The hexadecimal string is decoded with function <code class="language-plaintext highlighter-rouge">a0_0x22e3bf</code> and subsequently processed with <code class="language-plaintext highlighter-rouge">document.write</code>.</p><p><a href="/assets/images/Belarus_Poland/html2.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/html2.png" alt="sshot" loading="lazy"></a></p><p>Hex decoding the string, we can see the HTML content that is executed when the CHM is run by the victim:</p><p><a href="/assets/images/Belarus_Poland/html3.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/html3.png" alt="sshot" loading="lazy"></a></p><p>This is a simple but clever HTML file. It does the following:</p><ul><li>Creates an iframe to load the decoy image<li>Attempts to load <code class="language-plaintext highlighter-rouge">desktop.mp3</code> via the deprecated <code class="language-plaintext highlighter-rouge">&lt;bgsound&gt;</code> tag, primarily supported by Internet Explorer. I suspect this is used to have the file downloaded as a .tmp file for subsequent extraction. The .mp3 file is a CAB file which contains <code class="language-plaintext highlighter-rouge">uNT32.dll</code>, the payload.</ul><p><a href="/assets/images/Belarus_Poland/cab.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/cab.png" alt="sshot" loading="lazy"></a></p><ul><li>Creates an ActiveX object with CLSID <code class="language-plaintext highlighter-rouge">adb880a6-d8ff-11cf-9377-00aa003b7a11</code>, which is associated with the HTML Help ActiveX Control (<code class="language-plaintext highlighter-rouge">hhctrl.ocx</code>), documented <a href="https://learn.microsoft.com/en-us/previous-versions/windows/desktop/htmlhelp/html-help-activex-control-overview">here</a>. We can confirm this in the registry:</ul><p><a href="/assets/images/Belarus_Poland/registry.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/registry.png" alt="sshot" loading="lazy"></a></p><ul><li><p>A button is created with the value <code class="language-plaintext highlighter-rouge">Bitmap::Shortcut</code> and the target command is listed in the <code class="language-plaintext highlighter-rouge">Item1</code> parameter. Finally, the button is clicked via <code class="language-plaintext highlighter-rouge">Click()</code>, leading to the command execution. The command executed by the CHM:</p><li>Starts a minimized cmd prompt<li>Navigates to the <code class="language-plaintext highlighter-rouge">%temp%</code> folder<li>Uses the <code class="language-plaintext highlighter-rouge">forfiles.exe</code> native executable. The use of this LOLbin is common in the indirect execution of commands, potentially breaking the detection of suspicious parent/child process relationships. The parameter <code class="language-plaintext highlighter-rouge">/M</code> is used to search for .tmp files<li>A command is then executed to check if the size of the identified .tmp files is 180738 bytes, which is the size of <code class="language-plaintext highlighter-rouge">desktop.mp3</code><li>If the .tmp file is exactly of 180738 bytes, the DLL within the CAB file is extracted using <code class="language-plaintext highlighter-rouge">expand</code><li>Finally, the extracted DLL (<code class="language-plaintext highlighter-rouge">uNT32.dll</code>) is loaded via rundll32.exe, invoking an exported function with ordinal #1</ul><h1 id="downloader">Downloader</h1><p><code class="language-plaintext highlighter-rouge">uNT32.dll</code> is a C++ downloader with XOR-encrypted strings. The core download functionality of this DLL is implemented at offset <code class="language-plaintext highlighter-rouge">0x403720</code>, followed by a PE header validation function which, if unsuccessful, leads to program termination.</p><p><a href="/assets/images/Belarus_Poland/dll1.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/dll1.png" alt="sshot" loading="lazy"></a></p><p><a href="/assets/images/Belarus_Poland/dll2.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/dll2.png" alt="sshot" loading="lazy"></a></p><p>The string decryption is implemented using a 128 byte buffer as a rotating key. Strings are XOR decrypted in chunks of varying lenghts, depending on the string length. For example, the User-Agent string used for the payload download is decrypted in 5 byte chunks.</p><p><a href="/assets/images/Belarus_Poland/dll3.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/dll3.png" alt="sshot" loading="lazy"></a></p><p>The downloader uses the WinHTTP APIs to download and read payload delivered in the following file from the threat actor infrastructure: <code class="language-plaintext highlighter-rouge">hxxps://rustyquill[.]top/shw/the-magnus-protoco1.jpg</code>. The domain and file name refer to the Magnus Protocol podcast from <a href="https://rustyquill.com/show/the-magnus-protocol/">Rusty Quill</a>.</p><p><a href="/assets/images/Belarus_Poland/dll4.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/dll4.png" alt="sshot" loading="lazy"></a></p><p>The downloaded payload is read into a buffer which is size checked. The size check validates if the payload is bigger than 289109 bytes. If it is, it strips the first 289109 bytes and XOR decrypts the remainder using the same 128 byte key buffer used for the string decryption. The decrypted payload is saved to disk for execution.</p><p><a href="/assets/images/Belarus_Poland/dll5.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/dll5.png" alt="sshot" loading="lazy"></a></p><p>Unfortunately, I could not retrieve a copy of <code class="language-plaintext highlighter-rouge">the-magnus-protoco1.jpg</code> with payload. I found an image with the same name and delivered from the same infrastructure, however, the size of that image is 289109 bytes, with no anomalies after the .jpg end marker (<code class="language-plaintext highlighter-rouge">FF D9</code>). Given the explicit check for data that exceeds 289109 bytes, and further decryption of the buffer minus the initial 289109 bytes, I suspect that at some point the threat actor may have hosted benign images, either temporarily or for unintended downloaders (e.g. if geolocation was implemented). A malicious image delivering payload is very likely the same image that I found with content added after the <code class="language-plaintext highlighter-rouge">FF D9</code> end marker. This is likely because the downloader strips the exact same number of bytes that the image I found has.</p><p><a href="/assets/images/Belarus_Poland/goodimage1.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/goodimage1.png" alt="sshot" loading="lazy"></a></p><p>The benign image (d3f0f747e56431c6d7c0259bc2afa2769898810140e8382af55d8297142a8529) is related to Rusty Quill:</p><p><a href="/assets/images/Belarus_Poland/goodimage.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/goodimage.png" alt="sshot" loading="lazy"></a></p><p>I attempted to find the same image but with payload using a YARA rule with the following attributes:</p><ul><li>A JPEG header<li>Sequential bytes from the image I found<li>Size greater than 289109 bytes</ul><div class="language-plaintext highlighter-rouge"><div class="code-header"> <span data-label-text="Plaintext"><i class="fas fa-code fa-fw small"></i></span> <button aria-label="copy" data-title-succeed="Copied!"><i class="far fa-clipboard"></i></button></div><div class="highlight"><code><table class="rouge-table"><tbody><tr><td class="rouge-gutter gl"><pre class="lineno">1

There is always a possibility that the threat actor delivered other images which do not share the byte sequence. However, given the exact number of stripped bytes, I suspect the image I found is the same image that had encrypted payload appended to it.</p><p>The downloaded and decrypted payload is a DLL saved to a newly created folder under: <code class="language-plaintext highlighter-rouge">C:\Users\%user%\AppData\Local\TaskSync\net32.dll</code>. The payload is executed with rundll32.exe, calling an exported function with ordinal #1. A Scheduled Task is created for the DLL execution using the TaskScheduler API via COM.</p><p><a href="/assets/images/Belarus_Poland/dll6.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/dll6.png" alt="sshot" loading="lazy"></a></p><p><a href="/assets/images/Belarus_Poland/schtask.png" class="popup img-link shimmer"><img src="/assets/images/Belarus_Poland/schtask.png" alt="sshot" loading="lazy"></a></p><h1 id="previous-samples-and-attribution">Previous Samples and attribution</h1><p>The <code class="language-plaintext highlighter-rouge">rustyquill[.]top</code> domain is also associated with a previous CHM file shared by <a href="https://x.com/malwrhunterteam/status/1909235735850279070">@MalwareHunterTeam</a> on April 7, 2025.</p><ul><li>Zip file name: <code class="language-plaintext highlighter-rouge">dowód_wpłaty.zip</code> (“proot of payment”)<li>Zip SHA2: 4d09fad2630ec33ab6bb45b85455c6a6ac7b52f8dae9b90736db4a5f00f72ea9<li>File name: <code class="language-plaintext highlighter-rouge">dowod.chm</code> (“evidence”)<li>File SHA2: 0631696f484633f4aeb8f817af2a668806ab4dca70f006dc56fc9cd9dcda4dbe</ul><p>These CHM files are very likely associated with a threat actor tracked as <code class="language-plaintext highlighter-rouge">FrostyNeighbor</code> and <code class="language-plaintext highlighter-rouge">UNC1151</code>, historically attributed to Belarus. Previous targeting documented in reports such as the <a href="https://cloud.google.com/blog/topics/threat-intelligence/unc1151-linked-to-belarus-government/">following</a> suggest an interest in Ukraine, Lithuania, Latvia, Poland, and Germany, which is consistent with the upload geography for these malicious CHM files.</p><h1 id="iocs">IOCs</h1><ul><li>deklaracja.chm:
