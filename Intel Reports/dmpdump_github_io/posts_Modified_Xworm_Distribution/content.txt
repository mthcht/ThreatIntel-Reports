data-toc-skip>Modified XWORM Distribution by Chinese-Speaking Threat Actor</h1><div class="post-meta text-muted"> <span> Posted <time data-ts="1750032000" data-df="ll" data-bs-toggle="tooltip" data-bs-placement="bottom" > Jun 16, 2025 </time> </span><div class="d-flex justify-content-between"> <span> By <em> <a href="https://github.com/dmpdump">dmpdump</a> </em> </span><div> <span class="readtime" data-bs-toggle="tooltip" data-bs-placement="bottom" title="838 words" > <em>4 min</em> read</span></div></div></div></header><div class="content"><p>While hunting for MSI installers that typically distribute Gh0stRAT and RATs that share some of the Gh0stRAT code, such as WinOS/ValleyRAT, I came across an infection chain leading to a slightly modified <code class="language-plaintext highlighter-rouge">XWORM RAT</code>:<br /> <code class="language-plaintext highlighter-rouge">whats-install.msi</code> (37e42839ea6f1c97c7256eeec99e420e46e4d920bf629cb84aa260e78ee7f60f)</p><p>This MSI was created with Advanced Installer and its comments are in Chinese: <code class="language-plaintext highlighter-rouge">此安装数据库包含了安装 whats-install 所需的逻辑和数据。</code> (“This installation database contains the logic and data required to install whats-install.”). It masquerades as a legitimate WhatsApp installer.</p><p><a href="/assets/images/modified_xworm/msi_comments.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/msi_comments.png" alt="sshot" loading="lazy"></a></p><p>Several fields in the installer are also in Chinese, suggesting that it was likely created by a Chinese-speaking threat actor.</p><p><a href="/assets/images/modified_xworm/excerptmsi.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/excerptmsi.png" alt="sshot" loading="lazy"></a></p><p>The MSI contains files that lead to an infection chain that loads a modified <code class="language-plaintext highlighter-rouge">XWORM RAT</code> in memory.</p><p><a href="/assets/images/modified_xworm/msi_contents.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/msi_contents.png" alt="sshot" loading="lazy"></a></p><p>When the MSI is executed, one execution chain runs <code class="language-plaintext highlighter-rouge">xmplay.exe</code>, leading to the <code class="language-plaintext highlighter-rouge">XWORM</code> infection, while the other (through files in <code class="language-plaintext highlighter-rouge">installer.exe_1</code>) leads to a Chromium-based desktop application that is supposed to be a custom WhatsApp application. This WhatsApp application is not reviewed here.</p><p><a href="/assets/images/modified_xworm/execchain.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/execchain.png" alt="sshot" loading="lazy"></a></p><p>When the legitimate <code class="language-plaintext highlighter-rouge">xmplay.exe</code> is executed, it side-loads a malicious DLL named <code class="language-plaintext highlighter-rouge">xmpcd.dll</code>. The DLL first executes a function called <code class="language-plaintext highlighter-rouge">AccurateRemote()</code> which runs an embedded PowerShell script to create a Scheduled Task with a random name for persistence. The Scheduled Task runs <code class="language-plaintext highlighter-rouge">xmplay.exe</code> during logon from <code class="language-plaintext highlighter-rouge">C:\Program Files (x86)\whats-install\whats-install\6000\xmplay.exe</code>, the folder in which the MSI drops content.</p><p><a href="/assets/images/modified_xworm/schtask.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/schtask.png" alt="sshot" loading="lazy"></a></p><p>After creating the Scheduled Task, the <code class="language-plaintext highlighter-rouge">installer.exe</code> executable is run from the same folder.</p><p><a href="/assets/images/modified_xworm/runinstaller.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/runinstaller.png" alt="sshot" loading="lazy"></a></p><p><code class="language-plaintext highlighter-rouge">installer.exe</code> is a simple shellcode loader. This loader:</p><ul><li>Reads the content after the <code class="language-plaintext highlighter-rouge">IEND</code> delimeter from <code class="language-plaintext highlighter-rouge">resource_data.png</code> (IEND marks the end of the image data stream in a PNG image). The content read is an encrypted shellcode<li>It allocates memory with PAGE_READWRITE permissions<li>It copies the encrypted shellcode to the newly allocated memory<li>It modifies the memory protection to PAGE_EXECUTE_READ<li>It executes the shellcode and creates a delay with a function called <code class="language-plaintext highlighter-rouge">verify_system_compatibility()</code></ul><p>Shellcode loader function:</p><p><a href="/assets/images/modified_xworm/installer.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/installer.png" alt="sshot" loading="lazy"></a></p><p>Encrypted shellcode in <code class="language-plaintext highlighter-rouge">resource_data.png</code>:</p><p><a href="/assets/images/modified_xworm/png.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/png.png" alt="sshot" loading="lazy"></a></p><p>Function copying the shellcode after the <code class="language-plaintext highlighter-rouge">IEND</code> delimeter:</p><p><a href="/assets/images/modified_xworm/read_sc.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/read_sc.png" alt="sshot" loading="lazy"></a></p><p>Delay via the creation and subsequent deletion of a .tmp file in <code class="language-plaintext highlighter-rouge">verify_system_compatibility()</code>:</p><p><a href="/assets/images/modified_xworm/delay.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/delay.png" alt="sshot" loading="lazy"></a></p><p>The shellcode loaded by <code class="language-plaintext highlighter-rouge">installer.exe</code> is encrypted with the <a href="https://github.com/TheWover/donut">Donut</a> loader. After decrypting the loaded payload, we can see that it’s an easily recognizable <code class="language-plaintext highlighter-rouge">XWORM RAT</code> client configured with the following C2: <code class="language-plaintext highlighter-rouge">27.124.2[.]138:6000</code>. This C2 is unusually embedded in cleartext, and not encrypted in the Settings section, as it is usually the case in XWORM.</p><p><a href="/assets/images/modified_xworm/xworm.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/xworm.png" alt="sshot" loading="lazy"></a></p><p>The following configuration settings are decrypted from the Settings (with the standard AES in ECB mode decryption from XWORM)</p><ul><li>Key: <code class="language-plaintext highlighter-rouge">&lt;123456789&gt;</code><li>SPL (delimeter): <code class="language-plaintext highlighter-rouge">&lt;Underbytemm&gt;</code><li>Groub (version): <code class="language-plaintext highlighter-rouge">Underbyte V5.6</code><li>USBNM: <code class="language-plaintext highlighter-rouge">USB.exe</code></ul><p>The configuration is decrypted using key <code class="language-plaintext highlighter-rouge">0xBC44ABB3B3EE67C0480855A9681079BC44ABB3B3EE67C0480855A96810791E00</code>. When we look at the Settings section, we can decrypt a different, unused C2: <code class="language-plaintext highlighter-rouge">45.125.216[.]54:7000</code>. The mutex configured in the sample is <code class="language-plaintext highlighter-rouge">sKGCo7sB9Ni6uaEY</code>. The configuration suggests that the threat actor replaced the common <code class="language-plaintext highlighter-rouge">XWORM v5.6</code> version with their own <code class="language-plaintext highlighter-rouge">Underbyte</code> version, also used as the delimeter (SPL).</p><p>In addition to the custom hard-coded C2 in cleartext, this XWORM version includes a custom Telegram identification and reporting function, suggesting that the threat actor is interested in checking if the victim has Telegram installed, potentially for further targeting.</p><p>The <code class="language-plaintext highlighter-rouge">Info()</code> function calls the standard <code class="language-plaintext highlighter-rouge">Spread()</code> function.</p><p><a href="/assets/images/modified_xworm/info.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/info.png" alt="sshot" loading="lazy"></a></p><p>However, unlike the typical XWORM <code class="language-plaintext highlighter-rouge">Spread()</code> function, which verifies if the current file name matches the ‘spread’ file name (typically <code class="language-plaintext highlighter-rouge">USB.exe</code>), this <code class="language-plaintext highlighter-rouge">Spread()</code> function validates if Telegram is installed on the host, returning <code class="language-plaintext highlighter-rouge">是的</code> (‘Yes’) if it is, and <code class="language-plaintext highlighter-rouge">不</code> (‘No’) if it is not.</p><p><a href="/assets/images/modified_xworm/spread.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/spread.png" alt="sshot" loading="lazy"></a></p><p><code class="language-plaintext highlighter-rouge">Spread()</code> uses the <code class="language-plaintext highlighter-rouge">SearchForAnyTelegram()</code> function to locate Telegram on disk.</p><p><a href="/assets/images/modified_xworm/searchforanytelegram.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/searchforanytelegram.png" alt="sshot" loading="lazy"></a></p><p>For comparison, this is the <code class="language-plaintext highlighter-rouge">Spread()</code> function that I observed in other XWORM samples:</p><p><a href="/assets/images/modified_xworm/spreadorig.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/spreadorig.png" alt="sshot" loading="lazy"></a></p><p>Pivoting on the <code class="language-plaintext highlighter-rouge">Underbyte</code> version used by this threat actor, we can see a group of XWORM RATs submitted to VirusTotal, including other trojanized installers. At least some of the other distributions include similar Donut-encrypted shellcodes embedded into PNG files. Some of the XWORM samples below have minor customizations, but they don’t target Telegram. Some examples are:</p><p><code class="language-plaintext highlighter-rouge">ChromeSetup.zip</code> (2d32a0d7709b7ebf8494647a76be24dcca8e1fa31d10fc08f9dcacbaa27182c1), uploaded on June 11, 2025.</p><p><a href="/assets/images/modified_xworm/chromesetup.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/chromesetup.png" alt="sshot" loading="lazy"></a></p><p><code class="language-plaintext highlighter-rouge">Installer.zip</code> (e39609faa71c659305a9bd198127a2739f7525e7f735850703e52b2237af5906), uploaded on June 10, 2025.</p><p><a href="/assets/images/modified_xworm/installerzip.png" class="popup img-link shimmer"><img src="/assets/images/modified_xworm/installerzip.png" alt="sshot" loading="lazy"></a></p><p><code
