Atlas) with high confidence.</p><p>Cloud Atlas-related operations previously used scripting frameworks for a variety of purposes as part of operations, as documented by researchers from <a href="https://unit42.paloaltonetworks.com/unit42-inception-attackers-target-europe-year-old-office-vulnerability/">Palo Alto</a>. In previous instances, Cloud Atlas-related scripts performed functions such as the following:</p><ul role="list"><li>Initial victim system reconnaissance and system survey.</li><li>System data and file collection.</li><li>Data exfiltration.</li><li>Anti-analysis and anti-forensics operations. </li></ul><p>The script object retrieved in this case seems more limited, focusing primarily on creating persistence mechanisms and evading analysis while attempting to download an additional payload. Based on prior analysis of Cloud Atlas-related activity, this iterative nature is not unexpected although the precise persistence mechanisms appear new.</p><p>For example, the following establishes persistence via a scheduled task:</p><figure class="w-richtext-figure-type-image"><div><img loading="lazy" alt="The following establishes persistence via a scheduled task." src="https://cdn.prod.website-files.com/6914bc46af7a937a5c07eea3/69401a38d5ea55841fca227d_the-continuous-conundrum-of-cloud-atlas-blog-image-3.jpeg"/></div></figure><p>The portion of the script provided below checks for previous retrieval of a follow-on payload and for the presence of a scheduled task (“Display renovation”), while also modifying system parameters via the Windows Registry to “hide” the taskeng.exe window through a hard-coded placement value off-screen.</p><figure class="w-richtext-figure-type-image"><div><img loading="lazy" alt="The portion of the script provided below checks for previous retrieval of a follow-on payload and for the presence of a scheduled task (“Display renovation”), while also modifying system parameters via the Windows Registry to “hide” the taskeng.exe window through a hard-coded placement value off-screen." src="https://cdn.prod.website-files.com/6914bc46af7a937a5c07eea3/69401a38d5ea55841fca2270_the-continuous-conundrum-of-cloud-atlas-blog-image-4.jpeg"/></div></figure><p>Since taskeng.exe will launch a window (even if momentarily), the above will “hide” this aspect of execution. The technique is superficially similar to one <a target="_blank" href="https://cantoriscomputing.wordpress.com/2016/07/22/powershell-malware/">previously documented by researchers</a>, and <a target="_blank" href="https://unit42.paloaltonetworks.com/unit42-inception-attackers-target-europe-year-old-office-vulnerability/">deployed in Cloud Atlas-related activity</a>.</p><p>Finally, the script contains a function to eliminate Temporary Internet Files artifacts associated with script execution:</p><figure class="w-richtext-figure-type-image"><div><img loading="lazy" alt="The script contains a function to eliminate Temporary Internet Files artifacts associated with script execution." src="https://cdn.prod.website-files.com/6914bc46af7a937a5c07eea3/69401a38d5ea55841fca2295_the-continuous-conundrum-of-cloud-atlas-blog-image-5.jpeg"/></div></figure><p>Based on descriptions in previous work from Kaspersky and Palo Alto, the above appears more limited in functionality than earlier Cloud Atlas-linked script objects. However, without actual possession of such scripts for comparison, degree of similarity (or difference) is not possible to determine with the information currently available.</p><p>Overall, the retrieved script features many overlaps with behaviors documented by other researchers and linked to Cloud Atlas in 2018 (Palo Alto) and 2019 (Kaspersky). While previous analysis indicates Cloud Atlas-related activity will frequently re-use or maintain capabilities for many years, the appearance of this script two years after public documentation, combined with inexact replication of previously-documented capabilities and the network communication items described previously, would argue for some caution in definitively linking this file to the Cloud Atlas cluster of behaviors. While the overlaps certainly exist, and associated network infrastructure ties in to documented, recent Cloud Atlas tendencies, DomainTools associates the above with this behavioral cluster with medium confidence at this time for the reasons noted above.</p><h2>Adversary Themes and Possible Motivations</h2><p>Moving away from the scripting object which is likely—although not definitively—linked to Cloud Atlas behaviors, the overall themes as well as probable geographic targeting of the observed malicious documents largely align with previously documented activity from this entity.</p><p>Examining items discovered from December 2020 through February 2021, DomainTools identified the following “themes” or lures:</p><ul role="list"><li>A document purportedly from the <a target="_blank" href="https://www.iss.europa.eu/">European Union Institute for Security Studies</a> (EUISS) on common defense questions for the European Union, likely appearing in France.</li><li>A document from the “Ministry of Labor and Social Policy” of the unrecognized Ukrainian breakaway region known as the Luhansk People’s Republic, submitted from Ukraine.</li><li>An agenda for a training course on customs regulation from the Belarusian “Trade and Industrial Chamber,” identified in Belarus.</li><li>A news bulletin concerning Belarusian adoption of an International Atomic Energy Association (IAEA) action plan for the development of nuclear energy, first identified in Russia.</li><li>A document concerning the creation of a common natural gas market within the <a target="_blank" href="http://www.eaeunion.org/?lang=en">Eurasian Economic Union</a> (EAEU), submitted from Uzbekistan.</li><li>A listing of personnel allegedly belonging to the <a target="_blank" href="https://en.wikipedia.org/wiki/Almaz_(Special_unit)">SPBT Almaz</a> special anti-terrorist unit of the Belarussian security forces, first seen in Belarus.</li></ul><p> </p><figure class="w-richtext-figure-type-image"><div><img loading="lazy" alt="Screenshot 1" src="https://cdn.prod.website-files.com/6914bc46af7a937a5c07eea3/69401a38d5ea55841fca2274_the-continuous-conundrum-of-cloud-atlas-blog-image-6.jpeg"/></div></figure><p>The items continue targeting trends and lure themes <a href="https://www.domaintools.com/resources/blog/current-events-to-widespread-campaigns-pivoting-from-samples-to-identify">observed in late 2020</a>:</p><ul role="list"><li>Primary focus on countries formerly part of the Soviet Union with an emphasis on energy and political themes.</li><li>Particular focus on the unrecognized breakaway regions of Ukraine such as Luhansk as well as Donetsk.</li><li>Additional targeting of Western European and NATO-related defense interests.</li></ul><p>Based on the observed activities, lures, and likely geographic targeting, DomainTools assesses with high confidence that the campaigns in question form part of unspecified espionage operations. While further speculation on particular attribution is possible, insufficient technical evidence exists that would allow DomainTools to attribute this activity to any distinct entity or country.</p><h3>Outlier Samples</h3><p>Adding pause to the question of attribution are two similarly-structured but outlier samples both in technical behavior and targeting. Whereas the majority of the malicious documents using the same template string spawn communication via HTTPS for follow-on payload retrieval, two items (one of which has several variants with identical functionality and document content) instead utilize communication to resources via SMB, such as the following:</p><p><code>\185.70.184[.]32soarnegroidmeanalkydapresowntipslushing[.]png</code></p><p><code>\139.60.161[.]74appalcanedentrecentlyconvergenting[.]png</code></p><p>In addition to the difference in protocol, the naming convention (one long string of text without dividing numbers) and use of a file extension (PNG) are also different from other samples. DomainTools researchers were unsuccessful in attempting to retrieve the PNG objects referenced, making further analysis not possible at this time. While domain links are not possible on these items, referenced IP addresses do at least conform to hosting practices used by recent Cloud Atlas-linked activity: favoring specific providers largely located and operating in Europe.</p><p>Technical observations aside, the “themes” of the documents were also different, reflecting the following topics:</p><ul role="list"><li>The <a target="_blank" href="https://www.nytimes.com/2020/05/18/us/politics/justice-department-al-qaeda-florida-naval-base-shooting.html">Pensacola shooting</a> that was later linked by US authorities to Al Qaeda operations.</li><li>A travel form linked to COVID-19 precautions for travelers to the United Kingdom.</li></ul><p> </p><figure class="w-richtext-figure-type-image"><div><img loading="lazy" alt="Screenshot 2" src="https://cdn.prod.website-files.com/6914bc46af7a937a5c07eea3/69401a38d5ea55841fca226c_the-continuous-conundrum-of-cloud-atlas-blog-image-7.jpeg"/></div></figure><p>While the COVID-19 form at least appears to originate in Europe, the Pensacola shooting document first appears in the Middle East. While researchers previously identified Cloud Atlas-linked activity in Central Asia, publicly available information contains no references to operations in the Gulf region, where this item appears to have originated.</p><p>Overall, these documents retain the template string unifying all observed items since 2019, but otherwise appear to differ in behaviors, themes, and possible targeting. At this time, insufficient evidence exists to determine if these items represent a closely-linked, but operationally independent, group to Cloud Atlas with access to similar tools, or merely variations on common delivery vectors ultimately leading to the same payloads.</p><h2>Conclusion</h2><p>DomainTools researchers continue to track activity of interest through sustained monitoring of known malicious infrastructure creation tendencies. Through this work, DomainTools researchers identified persistent activity linked to previous analysis of initial access activity associated with an entity referred to as Cloud Atlas. While some parts of this entity’s operations have shifted in the past six months of tracking them, overall this group continues to exhibit common tendencies in both infrastructure registration and malicious document design. </p><p>By identifying these fundamental behaviors linked to a known threat actor, network defenders and threat intelligence analysts can keep pace with adversaries over time. While DomainTools anticipates eventual alterations in this group’s activity due to public scrutiny, the likelihood that all aspects of this group’s operations (network infrastructure, malicious document format, and possibly scripting behaviors) will change simultaneously is rather low. Through incorporation of appropriate monitoring and tracking strategies linked with this threat’s fundamental behaviors, defenders can ensure continuous coverage against this actor moving forward.</p></div></div></div></article><div data-wf--section-space--variant="large" class="g-section-space w-variant-da9deacd-6bd2-1cf6-af5c-3a3c0552044d"></div></section><div function initSearchExpand() {

  const searchWrap = document.querySelector('.search-enable');
  const searchInput = document.querySelector('.filter_search-input.is-nav');
  const searchForm = searchWrap.querySelector('.search-input_wrap');

  if (!searchBtn || !searchWrap || !searchInput) return;

  // Reset state on load
  searchWrap.classList.remove('is-expanded');
  searchInput.value = '';

  let isExpanded = false;

  searchBtn.addEventListener('click', function (e) {
    e.preventDefault();

    if (!isExpanded) {
      // First click - expand
      searchWrap.classList.add('is-expanded');
      searchInput.focus();
      isExpanded = true;
    } else {
      // Second click - submit if has value
      if (searchInput.value.trim()) {

      }
    }
  });

      searchWrap.classList.remove('is-expanded');
      isExpanded = false;
    }
  });

  // Submit on Enter key
  searchInput.addEventListener('keypress', function (e) {
    if (e.key === 'Enter' && this.value.trim()) {
      searchForm.submit();
    }
  });
}

  initSearchExpand();

    if (!clearLabel) return;
    
    const clearInput = clearLabel.querySelector('input');
    const filterInputs = dropdown.querySelectorAll('.filter_tags_cms_item input');
    const isRadio = filterInputs[0]?.type === 'radio';
    
    filterInputs.forEach(input => {
      input.addEventListener('change', () => {
        if (isRadio) {
          // Radio: any selection deactivates "All"
          if (input.checked) {
            clearLabel.classList.remove('is-active');
            if (clearInput) clearInput.checked = false;
          }
        } else {
          // Checkbox: check if any filter is active
          const anyChecked = [...filterInputs].some(i => i.checked);
          if (anyChecked) {
            clearLabel.classList.remove('is-active');
            if (clearInput) clearInput.checked = false;
          } else {
            // All unchecked - restore "All" state
            clearLabel.classList.add('is-active');
            if (clearInput) clearInput.checked = true;
          }
        }
      });
    });
    
    // When "All" is clicked
    if (clearInput) {
      clearInput.addEventListener('change', () => {
        if (clearInput.checked) {
          clearLabel.classList.add('is-active');
        }
      });
    }
  });
}

document.addEventListener('DOMContentLoaded', initDropdownClearState);
</script>
<script>
function initSecureBlankLinks() {
  const links = document.querySelectorAll('a[target="_blank"]');
  
  links.forEach(link => {
    const existingRel = link.getAttribute('rel') || '';
    const relValues = new Set(existingRel.split(' ').filter(Boolean));
    
    // Always add noopener for security
    relValues.add('noopener');
    
    // Only hide referrer for external links
    if (link.hostname !== window.location.hostname) {
      relValues.add('noreferrer');
    }
    
    link.setAttribute('rel', [...relValues].join(' '));
  });
}
document.addEventListener('DOMContentLoaded', initSecureBlankLinks);
</script>

function initTableWrappers() {
  const tables = document.querySelectorAll('.data-table, .wp-block-table');
  
  tables.forEach(table => {
    // Skip if already wrapped
    if (table.parentNode.classList.contains('table-wrapper')) return;
    
    const wrapper = document.createElement('div');
    wrapper.className = 'table-wrapper';
    table.parentNode.insertBefore(wrapper, table);
    wrapper.appendChild(table);
  });
}
