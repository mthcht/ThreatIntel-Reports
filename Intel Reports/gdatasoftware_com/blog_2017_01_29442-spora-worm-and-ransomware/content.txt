href="/blog/malware" class="nm-article-image-category"> Malware </a> </div> <p class="nm-article-teaser">Spora spreads via USB drives like Gamarue and Dinihou aka Jenxcus whilst also encrypting files. The sophistication of this threat could easily make it the new Locky. We discuss its infection and encryption procedure and show how it uses statistical values about encrypted files to calculate the ransom amount.</p> <hr> <div class="nm-article-content" id="blogpage-content-container"> <div id="c160283" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> HTA email Attachment as common infection vector </h2></header><p>Spora's ransom note was first spotted by the&nbsp;<a href="https://id-ransomware.malwarehunterteam.com/" target="_blank" class="external-link" rel="noreferrer nofollow">ID Ransomware</a>&nbsp;maintainers and announced via Twitter by&nbsp;<a href="https://twitter.com/malwrhunterteam/status/818787821980893184" target="_blank" class="external-link" rel="noreferrer nofollow">MalwareHunterTeam</a>. Several malware researchers and Twitter users were amazed by the good-looking, professional ransomware website and ransom note. Experience showed that most of these websites are in a bad shape. The first sample was provided by a member of Bleepingcomputer and discussed in their&nbsp;<a href="https://www.bleepingcomputer.com/forums/t/636975/spora-ransomware-support-and-help-topic/?p=4157580" target="_blank" class="external-link" rel="noreferrer nofollow">Spora support topic</a>. </p><p>This sample is an HTA application with obfuscated VBScript code. According to Bleepingcomputer it&nbsp;<a href="https://www.bleepingcomputer.com/news/security/spora-ransomware-works-offline-has-the-most-sophisticated-payment-site-as-of-yet/" target="_blank" class="external-link" rel="noreferrer nofollow">arrived in a ZIP archive via email attachment</a>. Submissions on VirusTotal show the filename <span class="codetext">Скан-копия _ 10 января 2017г. Составлено и подписано главным бухгалтером. Экспорт из 1С.a01e743_рdf.hta</span>. </p><p>The HTA file writes a JScript file to <span class="codetext">%TEMP%\close.js</span> and executes it. The JScript file in turn is a dropper for a Word document that is written to <span class="codetext">%TEMP%\doc_6d518e.docx</span> and a PE file that is saved to <span class="codetext">%TEMP%\81063163ded.exe</span>. Both files are opened by <span class="codetext">close.js</span>, the Word document with a parameter to show and focus the window, and the PE file with a parameter to hide it. As a result the document will be opened by the set default application for .docx files, e.g., Word, but an error message is shown because it is corrupt.&nbsp;The PE file&nbsp;<span class="codetext">81063163ded.exe</span> has a seemingly random name, but it is actually hardcoded by the dropper. The PE file is UPX packed and contains the actual payload.</p><div></div></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><a href="/fileadmin/_processed_/0/4/G_DATA_spora_word_document_24f99bb911.png" class="fancybox"><img class="image-embed-item " title="Error message, appears after opening the corrupt document" alt="Screenshot of the corrupt Word document used in Spora infection" src="/fileadmin/web/general/images/blog/2017/01_2017/G_DATA_spora_word_document.png" width="1439" height="588" loading="lazy" /></a><figcaption class="image-caption"> Error message, appears after opening the corrupt document </figcaption></figure></div></div></div></div> <div id="c160284" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> Worm-like behavior similar to Dinihou and Gamarue </h2></header><p>While ZCryptor had already been&nbsp;deemed a combination of ransomware and worm&nbsp;due to its usage of autorun.inf, Spora goes some steps further using the same techniques as Gamarue and Dinihou. The functionality of autorun.inf had been removed in Windows 7 and was patched on Windows XP and Windows Vista more than seven years ago, thus making it an ineffective technique for worms to spread via removable drives. The trick is: <a href="/blog/2015/03/24274-the-andromeda-gamarue-botnet-is-on-the-rise-again">Gamarue</a>, Dinihou and now also Spora use Windows shortcuts (.LNK files) instead.</p><p>Spora adds the&nbsp;hidden&nbsp;attribute to files and folders on the desktop, in the root of removable drives and the system drive. These hidden files and folders are, with the standard folder options, not visible anymore. Spora then puts Windows shortcuts with the same name and icon as the hidden files and folders as a visible replacement. Those .LNK files open the original file to avoid raising any suspicion and simultaneously execute the malware. An example: the folder C:\Windows will be hidden and a file named C:\Windows.lnk will be created; it looks exactly like the original folder if the standard folder options on Windows are set.</p><p>The .LNK files use the following command to execute the worm and open the original file. If the original file is a folder it will open Windows Explorer to show its contents:</p><p>/c explorer.exe "&lt;originalfile&gt;" &amp; type "&lt;worm&gt;" &gt; "%%tmp%%\&lt;worm&gt;" &amp; start "&lt;originalfile&gt;" "%%tmp%%\&lt;worm&gt;"</p><p>The worm copies itself as hidden file alongside the .LNK files, its filename is generated by calculating the CRC32 checksum for the VolumeSerialNumber. The result is put into the pattern %08x-%04x-%04x-%02x%02x-%02x%02x%02x%02 (see address 0x405492). This means, the name for the malware file can be, e.g., a277a133-ecde-c0f5-1591-ab36e22428bb.exe.</p></div><div class="ce-gallery row small-up-1 medium-up-2 large-up-2 "><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2017/01_2017/G_DATA_spora_volume_serial.png" class="fancybox"><img class="image-embed-item " title="This function calculates the CRC32 based on the VolumeSerialNumber of the disk" alt="Screenshot of IDA, showing a function calculating CRC32" src="/fileadmin/web/general/images/blog/2017/01_2017/G_DATA_spora_volume_serial.png" width="515" height="555" loading="lazy" /></a><figcaption class="image-caption"> This function calculates the CRC32 based on the VolumeSerialNumber of the disk </figcaption></figure></div><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2017/01_2017/G_DATA_spora_shortcuts_system_drive.png" class="fancybox"><img class="image-embed-item " title=".LNK files and a copy of the malware have been created in the root of the system drive." alt="Screenshot of Windows Explorer, showing shortcuts in system drive" src="/fileadmin/web/general/images/blog/2017/01_2017/G_DATA_spora_shortcuts_system_drive.png" width="754" height="582" loading="lazy" /></a><figcaption class="image-caption"> .LNK files and a copy of the malware have been created in the root of the system drive. </figcaption></figure></div></div></div></div> <div id="c160285" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p>The worm deletes the registry value&nbsp;<span class="codetext">HKCR\lnkfile\isShortcut</span>&nbsp;with the effect that the shortcut icons don't show the characteristic bent arrow in the lower left corner, which would be telltale sign to the user that something is wrong. </p><p>Simply navigating through the folders on your system and desktop using double-click will execute the worm. Using this strategy, it will not only spread to removable drives like USB thumb drives, it will also encrypt newly created files on the system. This renders the system unusable, for storing or working on any pictures or documents, until it is disinfected.&nbsp;</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><img class="image-embed-item " title="Function that deletes the isShortcut value in the registry" alt="Screenshot taken with IDA, isShortcut registry value deletion" src="/fileadmin/web/general/images/blog/2017/01_2017/G_DATA_spora_del_isshortcut.png" width="466" height="695" loading="lazy" /><figcaption class="image-caption"> Function that deletes the isShortcut value in the registry </figcaption></figure></div></div></div></div> <div id="c160286" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> Encryption </h2></header><p>Spora actually does not rename encrypted files and targets a comparably small set of extensions.&nbsp;The encryption procedure is shown in the diagram below. </p><p><span class="codetext">.backup, .7z, .rar, .zip, .tiff, .jpeg, .jpg, .accdb, .sqlite, .dbf, .1cd, .mdb, .cd, .cdr, .dwg, .psd, .pdf, .odt, .rtf, .docx, .xlsx, .doc, .xls</span></p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><img class="image-embed-item " title="The Spora encryption shown in an info graphic" alt="G DATA info graphic of encryption steps Spora takes" src="/fileadmin/web/general/images/blog/2017/01_2017/G_DATA_spora_encryption_infographic_web.jpg" width="894" height="615" loading="lazy" /><figcaption class="image-caption"> The Spora encryption shown in an info graphic </figcaption></figure></div></div></div></div> <div id="c160287" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p>Spora generates a pair of RSA keys, C1 and C2 (1024 bit). This newly generated public RSA key C2 is used to encrypt the per-file AES keys which are also generated by Spora. The generated private RSA key C1 on the other hand is stored in the .KEY file. That file is encrypted using a newly generated AES Key B (256 bit).&nbsp;The attacker's public RSA key A2 is used to encrypt AES key B. The encrypted key B is appended to the .KEY file. The figure below shows the code that writes the .KEY file's content including the encrypted AES key B to disk. </p><p>A second important file is the .LST file which contains a list of all encrypted files. Its encryption works analogous to the .KEY file encryption. The user has to upload the .KEY file to the payment site.&nbsp; </p><p>The .KEY file is only decryptable by the ransomware authors. Using their private RSA key A1 they could decrypt the AES Key B that was appended to the .KEY file.&nbsp; They could decrypt the remaining .KEY file contents including the user's private RSA key C1 using AES key B. Then they may put the private RSA key C1 into a decrypter that they send to the user after they have received the payment. This handling ensures that the attackers' private RSA key A1 is not exposed and that the decrypter only works for one user.However, this also means that there is only one private RSA key A1 for several infections. If that key is leaked or obtained by law enforcement, it can be used to decrypt all files that were encrypted by this variant of Spora and as such we can consider it a master key.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><img class="image-embed-item " title="The Spora payment site includes a chat system and provides several decryption packages with varying prices" alt="Screenshot of the Spora payment site" src="/fileadmin/web/general/images/blog/2017/01_2017/G_DATA_spora_website.png" width="1259" height="738" loading="lazy" /><figcaption class="image-caption"> The Spora payment site includes a chat system and provides several decryption packages with varying prices </figcaption></figure></div></div></div></div> <div id="c160290" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-above"><div class="ce-bodytext"><header class=" "><h2 class=" "> Statistics about encrypted files </h2></header><p>Spora counts the number of encrypted files for six different extension categories.
