class="nm-article-image-category"> CyberCrime </a> </div> <p class="nm-article-teaser">Babax not only changes its name but also adds a Ring 3 rootkit and lateral spreading capabilities. Furthermore it has a ransomware component called OsnoLocker. Is this combination as dangerous as it sounds?</p> <hr> <div class="nm-article-content" id="blogpage-content-container"> <div data-magellan-target="c210734" class="small-12 column"> <div class="row"> <div id="c210734" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> Emergence of Babax and Osno </h2></header><p>Babax stealer is at least around since June 2019. At that time a user named <strong>ForlayPy</strong> gave away the source code for free after being dissatisfied with a customer they sold the source to.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><a href="/fileadmin/_processed_/7/9/babax_giveaway_41b712b4d1.png" class="fancybox"><img class="image-embed-item " src="/fileadmin/web/general/images/blog/2020/05_2020/babax_giveaway.png" width="1676" height="330" loading="lazy" alt="" /></a></figure></div></div></div></div> <div id="c210726" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p>A colleague of mine discovered the first Osno stealer sample<sup>[1]</sup> on 5th October 2020. The sample<sup>[1] </sup>is a packed .NET assembly with the module name <strong>FallGuysStats</strong>. The module name indicates that it is using a statistics generator for the Steam game Fall Guys as a lure. The config shows version <strong>Osno 2.1.5</strong> and has placeholders for some of the functions, including FTP and Telegram settings.</p><p>By the end of October researcher <a href="https://twitter.com/backsla3h" target="_blank" rel="noreferrer">@backsla3h</a> noted that the stealer is <a href="https://twitter.com/backsla3h/status/1321246484617175042" target="_blank" rel="noreferrer">sold on forums</a>. The advertisment comparison of Babax and Osno shows not only an increased price but also four more features or "Benefits" for Osno: r77 and network spreading, Anti-AV and evasion of WindowsDefender via allowlist, AnarchyGrabber and microphone records. Additionally there is a ransomware module which is not advertised (yet). Most of these features are described in the following sections.</p></div><div class="ce-gallery row small-up-1 medium-up-2 large-up-2 "><div class="column column-block"><figure class="image"><img class="image-embed-item " src="/fileadmin/web/general/images/blog/2020/05_2020/babax_advertisment.png" width="1400" height="5692" loading="lazy" alt="" /></figure></div><div class="column column-block"><figure class="image"><img class="image-embed-item " src="/fileadmin/web/general/images/blog/2020/05_2020/osno_advertisment.png" width="1400" height="6022" loading="lazy" alt="" /></figure></div></div></div></div> </div> </div> <div data-magellan-target="c210746" class="small-12 column"> <div class="row"> <div id="c210746" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> Rootkit r77 </h2></header><p>Although the advertisment calls this an exploit, it is actually an <a href="https://github.com/bytecode77/r77-rootkit" target="_blank" rel="noreferrer">open source rootkit</a> by bytecode77. The Github repo provides DLL's for this rootkit as well as an installer. Osno does the installation itself. Just like the rootkit installer, Osno registers the rootkit DLL to <strong>AppInit_DLLs </strong>and enables <strong>LoadAppInit_DLLs</strong> so it is loaded with every process. Because the rookit DLLs are not signed, it sets <strong>RequireSignedAppInit_DLLs</strong> to 0.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><img class="image-embed-item " src="/fileadmin/web/general/images/blog/2020/05_2020/r77_installationcode.png" width="1254" height="380" loading="lazy" alt="" /></figure></div></div></div></div> <div id="c210727" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p>The rootkit uses MinHook to redirect WinAPI calls, so that it hides processes and file names, e.g., from explorer and taskmanager. The proof-of-concept binaries for the rootkit hide files and processes that start with "$77". Since Osno uses the proof-of-concept binaries, it needs to add the prefix "$77" to its own files to make it work.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><img class="image-embed-item thumbnail" src="/fileadmin/web/general/images/blog/2020/05_2020/r77.gif" width="1125" height="593" loading="lazy" alt="" /><figcaption class="image-caption"> File hiding demonstration by r77 rootkit. Image from https://github.com/bytecode77/r77-rootkit </figcaption></figure></div></div></div></div> <div id="c210747" class="fce column large-12 small-12 frame-layout-0 " ><p>The Github README.md for r77 states that the rootkit is still work in progress. Because of that hiding files for x86 is currently unstable and disabled in the proof-of-concept files.</p></div> </div> </div> <div data-magellan-target="c210735" class="small-12 column"> <div class="row"> <div id="c210735" class="fce column large-12 small-12 frame-layout-0 " ><header class=" "><h2 class=" "> Lateral movement via SharpExec </h2></header><p>Osno collects all accessible IP adresses in the local network, then downloads <a href="https://github.com/anthemtotheego/SharpExec" target="_blank" rel="noreferrer">SharpExec</a> binaries from Github. SharpExec is a tool with various commands for lateral movement. Osno executes the following command for every collected IP and domain:</p><p>&nbsp;</p><pre><code>&lt;sharpexec&gt; -m=psexec -i=&lt;collected-ip&gt; -d=&lt;collected-domain&gt; -f=&lt;path-in:%TEMP%/gpustats.bx&gt; -e=%TEMP%/&lt;randomname&gt;</code></pre><p>&nbsp;</p><p>The file gpustats.bx contains the path to the Osno executable. This command attempts to upload Osno to the given IP into the %TEMP% folder and executes it. That way Osno is able to spread to all accessible computers within the network.</p></div> </div> </div> <div data-magellan-target="c210753" class="small-12 column"> <div class="row"> <div id="c210753" class="fce column large-12 small-12 frame-layout-0 " ><header class=" "><h2 class=" "> Anti-AV </h2></header><p>The following Anti-AV features are those of the unpacked Osnoe sample<sup>[2]</sup>. The packer stub itself has also Anti-AV which is beyond the scope of this article.</p><h3>Windows Defender</h3><p>Osno adds its own path and the root of drive C: as exclusion folder for Windows Defender using the following PowerShell commands:</p><p>&nbsp;</p><pre><code>Add-MpPreference -ExclusionPath C:\
Add-MpPreference -ExclusionPath &lt;path-to-malware&gt;</code></pre><p>&nbsp;</p><p>This can only be successful if the malware has already gotten foothold on the system and obtained administrator privileges.</p><h3>Other AVs</h3><p>Osno searches for Window titles and process names to kill the processes of Antivirus software. It does this for the following window titles and process names:</p><ul class="list-black-square"><li>Window title "Malwarebytes&nbsp;Anti-Malware" and process name either mbamgui or mbam</li><li>Process names: avgidsagent, avgfws, avgtray, avgemcx, avgwdsvc, avgnsx, avgcsrvx, avgrsx, Toolbar Updater</li></ul><p>That means only Malwarebytes Anti-Malware and AVG are affected.</p></div> </div> </div> <div data-magellan-target="c210728" class="small-12 column"> <div class="row"> <div id="c210728" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> Osno ransomware is a wiper </h2></header><p>Osno ransomware, or OsnoLocker as it is called in the code, has an implementation for <a href="https://en.wikipedia.org/wiki/XXTEA" target="_blank" rel="noreferrer">XXTEA</a>. However, it is not used in the current sample. Instead it overwrites the original content of the files with a marker. The marker is the string "OsnoRansom" appended by a randomly created string of length 50-200 characters consisting of uppercase letters A-Z and digits 0-9. Described as a regex this would be <strong>^OsnoRansom([A-Z0-9]){50,200}$</strong>. The randomly created string will be different for every affected file.</p><p>OsnoLocker adds the <strong>.osnoed</strong> extension to these files.</p><p>OsnoLocker puts a ransom message into a file called <strong>RecInstruct.osnoned</strong> (sic!). It proceeds to write an executable to disk named <strong>Osno Decryptor.exe</strong><sup>[3]</sup> whose purpose is to lock the screen, display the ransom message that was placed in <strong>RecInstruct.osnoned</strong> and ask for a decryption code. This <strong>Osno Decryptor.exe</strong> has the module name <strong>FakeRansomware</strong>.</p><p>Osno is a wiper in its current form and payment will not help to get any files back. Recovery of files via shadow volumes copies can work, though. Future versions of Osno might use the already implemented XXTEA to encrypt files.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2020/05_2020/osno_ransom.png" class="fancybox"><img class="image-embed-item " src="/fileadmin/web/general/images/blog/2020/05_2020/osno_ransom.png" width="790" height="838" loading="lazy" alt="" /></a></figure></div></div></div></div> </div> </div> <div data-magellan-target="c210748" class="small-12 column"> <div class="row"> <div id="c210748" class="fce column large-12 small-12 frame-layout-0 " ><header class=" "><h2 class=" "> AnarchyGrabber and other copied tools </h2></header><p>Osno stealer implements the code of AnarchyGrabber 3. <a href="https://www.bleepingcomputer.com/news/security/discord-client-turned-into-a-password-stealer-by-updated-malware/" target="_blank" rel="noreferrer">An article by Bleepingcomputer</a> describes the additional features of the latest AnarchyGrabber version. Just like that version, Osno will force Discord to load JScript files <strong>inject.js</strong> and <strong>discordmod.js</strong>. To do that it puts the JScript files into the folder %AppData%\Discord\&lt;version&gt;\modules\discord_desktop_core\<strong>osno</strong>. Discord will then act as a stealer.</p><p>Another tool that this Osno sample uses is <strong>Da pure C++ Clipper</strong><sup>[4]</sup>. A native binary for clipbanking.</p><p>Furthermore, <a href="https://twitter.com/backsla3h/status/1321498443974471680" target="_blank" rel="noreferrer">@backsla3h pointed out</a> that the RunPE method and VM/Debugger/Sandbox detection code are taken from <strong>CSharp-RunPE</strong> and <strong>Anti-Analysis</strong> by <strong>NYAN-x-CAT</strong>.</p><p>So we have already identified six different copied sources and tools that are deployed by Osno: SharpExec, AnarchyGrabber, Da Pure C++ Clipper, CSharp-RunPE, Anti-Analysis, r77 rootkit. It is likely that there are more copied sources in those functions I didn't look at as they are beyond the scope of this article.</p></div> </div> </div> <div data-magellan-target="c210758" class="small-12 column"> <div class="row"> <div id="c210758" class="fce column large-12 small-12 frame-layout-0 " ><header class=" "><h2 class=" "> Conclusion </h2></header><p>Osno is not just a stealer anymore. Although that is still the main focus, the added capabilities pose a more serious threat, especially RDP access, lateral movement and file destruction.</p><p>However, none of that seems particularly scary.</p><p>Firstly, most of the serious sounding features are only possible<strong> after </strong>the malware sucessfully accessed the system and gained administrator privileges. That includes the rootkit and the anti-AV. The lateral movement portion depends on an external tool that needs to be downloaded first. It is only successful if network adminstrators disregard security measures alltogether, thus, unlikely to cause serious outbreaks.</p><p>Secondly, many of the stealer's features have been taken from public respositories and are known to defenders, making detection of the malware easier. Osno seems to have been worked around some of those tools. E.g., it uses the r77 rootkit binaries as is, although they are unfinished and only work with drawbacks. Osno renames its files to make them work for the rootkit binaries instead of implementing a rootkit that works for the Osno files.</p><p>The ransomware, which may have been self-implemented, seems not finished yet, which is confirmed by existence of non-implemented XXTEA code and the fact that this feature is not advertised. Later versions will likely use encryption instead of destroying files.</p><p>Due to the mishmash of open-source code and tools from other malware Osno is best described as a patchwork Frankenstein's monster .</p></div> </div> </div> <div data-magellan-target="c210560" class="small-12 column"> <div class="row"> <div id="c210560" class="fce column large-12 small-12 frame-layout-0 " ><header class="
