class="nm-page-section-content-left"> <div class="nm-article-blog"> <p class="nm-article-teaser">A new ransomware uses an unusual symmetric encryption method named "Fernet". It is Python based and appends .CYRAT to encrypted files.</p> <div class="nm-article-blog-image-wrapper"> <img src="/fileadmin/web/general/images/blog/2020/09_2020/G_DATA_Blog_Cyrat_Header.jpg" alt="DLL Fixer leads to Cyrat Ransomware" class="nm-article-image"> <a href="/blog/cybercrime" class="nm-article-image-category"> CyberCrime </a> </div> <div class="nm-article-content" id="blogpage-content-container"> <div data-magellan-target="c208060" class="small-12 column"> <div class="row"> <div id="c208060" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-intext-right"><header class=" " role="generic"><h3 class=" "> Discovery & Initial Analysis </h3></header><div class="image-gallery-wrap"><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><img class="image-embed-item " src="/fileadmin/user_upload/Presse/Deutschland/2020/09/cryrat_die.png" width="622" height="406" loading="lazy" alt="" /></figure></div></div></div><p>While hunting for new malware we often use Yara rules to find suspicious samples. One of my generic ransomware hunt rules found this new ransomware sample. At the time it had only 2 detections on Virustotal. The first submission date is 25. August 2020.</p><p><a href="http://ntinfo.biz/index.html" target="_blank" rel="noreferrer">DiE</a> and <a href="https://github.com/katjahahn/PortEx/tree/master/progs" target="_blank" rel="noreferrer">PortexAnalyzer</a> note that there is a zlib archive in the overlay of the file. Further inspection shows references to the python37.dll and the archive's name PYZ-00.pyz which is typical for <a href="https://www.pyinstaller.org/" target="_blank" rel="noreferrer">PyInstaller</a> executables.</p><p>It means Cyrat ransomware was compiled with Python 3.7 and converted to a Windows PE file using PyInstaller.</p><p>Unpacking and decompiling this file requires <a href="https://github.com/extremecoders-re/pyinstxtractor/blob/master/pyinstxtractor.py" target="_blank" rel="noreferrer">PyInstxtractor</a>, <a href="https://www.python.org/downloads/release/python-370/" target="_blank" rel="noreferrer">Python 3.7</a> and, e.g., <a href="https://pypi.org/project/uncompyle6/" target="_blank" rel="noreferrer">uncompyle6</a>. The Python version needs to be the same as the malware executable while extracting the .pyc files. Otherwise PyInstxtractor has issues properly constructing the files.</p></div></div> </div> </div> <div data-magellan-target="c208062" class="small-12 column"> <div class="row"> <div id="c208062" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" " role="generic"><h3 class=" "> Disguise as DLL fixer </h3></header><p>When I first tried to run the malware it crashed because of a missing fonts for <a href="http://www.figlet.org/" target="_blank" rel="noreferrer">pyfiglet</a>. To see it in action, I had to install pyfiglet and replicate the code that displays the ransomware's output.</p><p>The malware disguises as DLL fixer 2.5 (see image below). Upon execution it will display a randomly created number of corrupted DLLs it pretends to have found on the system. After the system has been encrypted, a success message for fixing the DLLs is shown.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><img class="image-embed-item " src="/fileadmin/web/general/images/blog/2020/09_2020/cryrat_screen.png" width="605" height="534" loading="lazy" alt="" /></figure></div></div></div></div> </div> </div> <div data-magellan-target="c208091" class="small-12 column"> <div class="row"> <div id="c208091" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" " role="generic"><h3 class=" "> Encryption </h3></header><p>Cyrat ransomware uses Fernet to encrypt files. This is a symmetric encryption method meant for small data files that fit into RAM. While Fernet is not unusual itself, it is not common for ransomware and in this case even problematic. This ransomware encrypts whole files regardless how big they are, whereas Fernet is unsuitable for big files.</p><p>A public RSA key is used to encrypt the Fernet key. This public key is downloaded from Mediafire instead of shipping it with the ransomware. This adds another dependency. The encrypted Fernet key is saved in <strong>Desktop\EMAIL_US.txt</strong>. A user with an infected system is required to send this file to the criminals.</p><p>Cyrat appends <strong>.CYRAT</strong> to encrypted files. It has a list of folders that it checks for target files. Those folders are 'Desktop', 'Downloads', 'Pictures', 'Music', 'Videos', and 'Documents'.</p><p>It targets files with the following extensions: 'doc', 'docx', 'xls', 'xlsx', 'ppt', 'pptx', 'boop', 'pst', 'ost', 'msg', 'eml', 'vsd', 'vsdx', 'txt', 'csv', 'rtf', '123', 'wks',&nbsp; 'wk1', 'pdf', 'dwg', 'onetoc2', 'snt', 'jpeg', 'jpg', 'docb', 'docm', 'dot', 'dotm', 'dotx', 'xlsm', 'xlsb', 'xlw', 'xlt', 'xlm', 'xlc', 'xltx', 'xltm', 'pptm', 'pot', 'pps', 'ppsm', 'ppsx', 'ppam', 'potx', 'potm', 'edb', 'hwp', '602', 'sxi', 'sti', 'sldx', 'sldm', 'sldm', 'vdi', 'vmdk', 'vmx', 'gpg', 'aes', 'PAQ', 'bz2', 'tbk', 'bak', 'tar', 'tgz', 'gz', '7z', 'rar', 'zip', 'backup', 'iso', 'vcd', 'bmp', 'png', 'gif', 'raw', 'tif', 'tiff', 'nef', 'psd', 'ai', 'svg', 'djvu', 'm4u', 'm3u', 'mid', 'wma', 'flv', '3g2', 'asf', 'mpeg', 'vob', 'mpg', 'swf', 'wav', 'mp3', 'sh', 'class', 'jar', 'java', 'rb', 'asp', 'php', 'jsp', 'brd', 'dch', 'dip', 'pl', 'vb', 'vbs', 'ps1', 'bat', 'cmd', 'asm', 'h', 'pas', 'c', 'cs', 'suo', 'sln', 'ldf', 'mdf', 'ibd', 'myi', 'myd', 'frm', 'odb', 'dbf', 'db', 'mdb', 'accdb', 'sql', 'sqlitedb', 'sqlite3', 'lay6', 'lay', 'mml', 'sxm', 'otg', 'odg', 'uop', 'std', 'sxd', 'otp', 'odp', 'wb2', 'slk', 'dif', 'stc', 'sxc', 'ots', 'ods', '3dm', 'max', '3ds', 'uot', 'stw', 'sxw', 'ott', 'odt', 'p12', 'csr', 'key', 'pfx', 'der', 'deb', 'mpeg', 'WEBM', 'MPG', 'MP2', 'MPEG', 'MPE', 'MPV', 'OGG', '3gp', 'mp3', 'json', 'css', 'html', 'py', 'exe', 'MP2', 'MPEG', 'MPE', 'MPV', 'OGG', '3gp', 'mp3'</p><p>The ransomware lists a few more extensions with a dot in them which is a bug: '.ARC', '.cpp', '.cgm', '.js', '.fla', '.asc', '.crt', '.sch'. These extensions will never be found by Cyrat because the file path is stripped from dots before it is compared with the target extension.</p><p>A ransom note named <strong>RANSOME_NOTE.txt</strong> is placed in every target folder. Furthermore a ransomware stock photo is downloaded from <strong>images.idgesg.net</strong> to <strong>Documents\background_img.png</strong> and set as wallpaper. The wallpaper does not contain any ransom message. In this state the stock photo's only purpose is to draw attention to the user.</p></div><div class="ce-gallery row small-up-1 medium-up-2 large-up-2 "><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2020/09_2020/ransomware_attack_worried_businessman_by_andrey_popov_gettyimages-1199291222_cso_2400x1600-100840844-large.jpg" class="fancybox"><img class="image-embed-item thumbnail" src="/fileadmin/web/general/images/blog/2020/09_2020/ransomware_attack_worried_businessman_by_andrey_popov_gettyimages-1199291222_cso_2400x1600-100840844-large.jpg" width="1200" height="800" loading="lazy" alt="" /></a><figcaption class="image-caption"> Cyrat sets this stock photo as wallpaper </figcaption></figure></div><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2020/09_2020/cyrat_ransomnote.png" class="fancybox"><img class="image-embed-item thumbnail" src="/fileadmin/web/general/images/blog/2020/09_2020/cyrat_ransomnote.png" width="1270" height="340" loading="lazy" alt="" /></a><figcaption class="image-caption"> Cyrat&#039;s ransom note (click to enlarge) </figcaption></figure></div></div></div></div> </div> </div> <div data-magellan-target="c208205" class="small-12 column"> <div class="row"> <div id="c208205" class="fce column large-12 small-12 frame-layout-0 " ><header class=" " role="generic"><h3 class=" "> Registry Changes and Persistence </h3></header><p>The ransomware deletes shadow volume copies and disables CMD, taskmanager, regstry tools via policy settings in the registry. It also removes the RUN command from the start menu and task manager. It uses bcedit to set <strong>recoveryenabled</strong> to <strong>No</strong> and <strong>bootstatuspolicy</strong> to <strong>ignoreallfailures</strong></p><p>For persistence Cyrat copies itself to the autostart folder <strong>\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup</strong></p><p>A reboot of the system, and thus re-run of the rasomware, will most likely encrypt the ransom notes. I did not find any safeguard in the code to prevent this from happening.</p></div> </div> </div> <div data-magellan-target="c208173" class="small-12 column"> <div class="row"> <div id="c208173" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-intext-right"><header class=" " role="generic"><h3 class=" "> Target Platforms </h3></header><div class="image-gallery-wrap"><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><img class="image-embed-item thumbnail" src="/fileadmin/user_upload/Presse/Deutschland/2020/09/cryrat_linux.png" width="292" height="189" loading="lazy" alt="" /></figure></div></div></div><p>The target platform is undoubtedly Windows because the PyInstaller executable only works there. The trojanized DLL fixer will only lure Windows users. Registry commands and persistence mechanisms are also Windows based.</p><p>Oddly, there are also checks for Linux and Darwin (core of many Apple operating systems) in the main body without any changes in the following code which is still Windows targeted.</p></div></div> </div> </div> <div data-magellan-target="c208437" class="small-12 column"> <div class="row"> <div id="c208437" class="fce column large-12 small-12 frame-layout-0 " ><header class=" " role="generic"><h3 class=" "> Contacting the Threat Actor(s) </h3></header><p>After<a href="https://twitter.com/struppigel/status/1298592057087361030" target="_blank" rel="noreferrer"> I posted the Cyrat sample</a> on Twitter, another Twitter user, who wants to be referred to as <strong>alex27</strong>, contacted the ransomware crook(s). He asked them for decryption and shared their email exchange with me. This shows that the threat actor(s) are expecting help requests from affected users, and thus, actively distributing the malware.</p><blockquote><p>We got your message, calm down your files are safe you will only get them back if you obey and follow the instruction. You know about bitcoin now? Did The page we put there to help you get bitcoin help? If no you can use google and find how to get bitcoin in your country and get $1,000 worth of bitcoin sent to this bitcoin address: &lt;redacted&gt; Your two days count start now, we have full access to your computer and we know you are reading this now. $500 will be added each day once the two days elapse and you didn't make payment for the decryptor. Don't think we won't delete those files if we don't hear from you because we will.</p></blockquote><p>I redacted the bitcoin address because it is likely individually created for every case. It has currently no transactions.&nbsp;<strong>alex27</strong> pointed out that their responses as well as the ransom note contain many grammar mistakes, making it unlikely that the author is a native English speaker.</p></div> </div> </div> <div data-magellan-target="c208149" class="small-12 column"> <div class="row"> <div id="c208149" class="fce column large-12 small-12 frame-layout-0 " ><header class=" " role="generic"><h3 class=" "> Conclusion </h3></header><p>As it is often the case with brand new malware discoveries, this sample is buggy and not yet ready to infect any system because it crashes in it's current state. However, the threat actor's reply shows they are active and might have already published versions that work. It's usually just a matter of time until those flaws are fixed. The problematic choice of the Fernet encryption method may take its toll on systems while they try to encrypt gigabyte sized files in RAM all at once.</p><p>Some parts of the code show an intention of also infecting Darwin and Linux systems, which may be added later on.</p><p>Unfortunately, there is currently no known way to decrypt files without the key.</p></div> </div> </div> <div
