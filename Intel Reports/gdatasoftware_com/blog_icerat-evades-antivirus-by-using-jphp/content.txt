class="nm-article-image-category"> CyberCrime </a> </div> <p class="nm-article-teaser">IceRat keeps low detections rates for weeks by using an unusual language implementation: JPHP. But there are more reasons than the choice of the compiler. This article explores IceRat and explains a way to analyze JPHP malware.</p> <hr> <div class="nm-article-content" id="blogpage-content-container"> <div data-magellan-target="c211246" class="small-12 column"> <div class="row"> <div id="c211246" class="fce column large-12 small-12 frame-layout-0 " ><header class=" "><h2 class=" "> Discovery of IceRat </h2></header><p>User <a href="https://malwaretips.com/members/mcmcbrad.89360/" target="_blank" rel="noreferrer">McMcbrad</a> of the <a href="https://malwaretips.com/" target="_blank" rel="noreferrer">Malwaretips.com</a> forums discovered the first IceRat samples<sup>[5][7]</sup>. The malware caught his interest due to the low detection rates on VirusTotal for most related samples. At the time of discovery only 2 to 3 engines showed a detection despite the samples being a month old.</p><p>Static analysis reveals that most components of IceRat are written in <strong>JPHP</strong>. This is a PHP implementation that runs on the Java VM. This implementation uses .phb files instead of Java .class files -- a file type that, as I suspect, is not commonly supported by antivirus products. So far I haven't heard or found any other malware that uses JPHP which partially explains the low detection rates on VirusTotal.</p><p>The name <strong>IceRat</strong> is based on the module name of an older sample<sup>[11]</sup> that McMcbrad found.</p></div> </div> </div> <div data-magellan-target="c211424" class="small-12 column"> <div class="row"> <div id="c211424" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> Decompiling JPHP </h2></header><p>There don't seem to be any tools to decompile JPHP code yet. But JPHP has to produce Java Bytecode in order to run in the Java VM. So decompilation to Java code is possible.</p><p>Unpacking the executable<sup>[5]</sup> with 7zip reveals the following structure.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><img class="image-embed-item thumbnail" src="/fileadmin/user_upload/Presse/Deutschland/2020/12/jphp_unpacked_files.png" width="611" height="464" loading="lazy" alt="" /></figure></div></div></div></div> <div id="c211430" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p>As I noticed after looking at several JPHP samples, the entrypoint for the main JPHP code is under <strong>.system\application.conf</strong> (see picture below). So for our klient.exe sample<sup>[5]</sup> the main code resides in <strong>app\forms\rqfdeqwf.phb</strong>.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><img class="image-embed-item " src="/fileadmin/user_upload/Presse/Deutschland/2020/12/jphp_application_conf.png" width="413" height="183" loading="lazy" alt="" /></figure></div></div></div></div> <div id="c211431" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p>The <strong>.phb</strong> files contain the <strong>0xCAFEBABE</strong> magic bytes for Java .class files somewhere down below. Removing the first part of the file excluding the magic bytes makes it possible to decompile these files into Java code with, e.g., Fernflower. The right side of the picture below shows how the file should look like after modification.</p></div><div class="ce-gallery row small-up-1 medium-up-2 large-up-2 "><div class="column column-block"><figure class="image"><a href="/fileadmin/user_upload/Presse/Deutschland/2020/12/jphp_cafebabe.png" class="fancybox"><img class="image-embed-item " src="/fileadmin/user_upload/Presse/Deutschland/2020/12/jphp_cafebabe.png" width="701" height="433" loading="lazy" alt="" /></a></figure></div><div class="column column-block"><figure class="image"><a href="/fileadmin/user_upload/Presse/Deutschland/2020/12/jphp_cafebabe2.png" class="fancybox"><img class="image-embed-item " src="/fileadmin/user_upload/Presse/Deutschland/2020/12/jphp_cafebabe2.png" width="704" height="431" loading="lazy" alt="" /></a></figure></div></div></div></div> <div id="c211433" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p>The decompiled code is still hard to read. class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> Infection chain and components </h2></header><p>IceRat consists of several small components instead of putting all functionality into one file. As a result most of these files may not attract any attention if their context is missing. E.g., a downloader is only malicious if the downloaded file is malware. If information about the downloaded file is missing and cannot be inferred, there is no reason to detect the downloader as malware.</p><p>The chain of infection and related files is in the graphic below. White boxes show non-malicious files. At least four of these files are JPHP EXE files, namely <strong>cheats.exe</strong><sup>[4]</sup>, <strong>1.exe</strong><sup>[12]</sup>, <strong>klient.exe</strong><sup>[5]</sup> and <strong>klip.exe</strong><sup>[7]</sup>. The main component of IceRat is <strong>klient.exe</strong><sup>[5]</sup>.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><a href="/fileadmin/user_upload/Presse/Deutschland/2020/12/IceRat_infection_chain.png" class="fancybox"><img class="image-embed-item " src="/fileadmin/user_upload/Presse/Deutschland/2020/12/IceRat_infection_chain.png" width="1024" height="768" loading="lazy" alt="" /></a><figcaption class="image-caption"> Infection chain and components of IceRat </figcaption></figure></div></div></div></div> <div id="c211436" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p>According to McMcbrad the first IceRat sample came from a malicious document for which he didn't keep a hash or file. The first part of the chain that I could find is <strong>Browes.exe</strong><sup>[1]</sup> which may have been distributed as trojanized software download for CryptoTab. <strong>Browes.exe</strong> is a selfextracting WinRAR archive that drops and executes the Windows Cabinet file <strong>1.exe</strong><sup>[2]</sup>.</p><p>The Windows Cabinet file is also a dropper for two more files, namely a non-malicious setup<sup>[3]</sup> for CryptoTab software, and a malware downloader named <strong>cheats.exe</strong><sup>[4]</sup>. CryptoTab is a browser with mining features, but its installation is not silent. The affected user will see the browser setup window (see image below) which is why I assume CryptoTab is provided as a lure. To summarize: The infection chain starts with a downloader in a trojanized dropper in a dropper.</p><ul class="list-black-square"></ul></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><img class="image-embed-item thumbnail" src="/fileadmin/user_upload/Presse/Deutschland/2020/12/cryptotab_window.png" width="652" height="302" loading="lazy" alt="" /></figure></div></div></div></div> <div id="c211439" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-above"><div class="ce-bodytext"><p>The JPHP file <strong>cheats.exe</strong><sup>[4]</sup> has the project name <strong>droper</strong> (sic). It accesses IceRat's main server to download the backdoor <strong>klient.exe</strong><sup>[5]</sup>. It chooses randomly one of the following names from a list:</p><ul class="list-black-square"><li>System</li><li>Jawas</li><li>WindowsShell</li><li>exploler</li><li>antiDrw</li><li>antiSsl</li><li>ADB</li><li>Microsoft</li><li>system</li></ul><p>Then it will write the file into the following locations:</p><ul class="list-black-square"><li>%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\.&lt;name&gt;.exe</li><li>c:\Windows\Temp\.&lt;name&gt;.exe</li><li>d:\Windows\Temp\&lt;name&gt;.exe</li></ul><p>This file, klient.exe, is the main component that will be controlled by the server.</p></div></div></div> </div> </div> <div data-magellan-target="c211441" class="small-12 column"> <div class="row"> <div id="c211441" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> Command and Control </h2></header><p>Although the name IceRat indicates a remote access trojan, the current malware is better described as a backdoor. Features for actual remote control, e.g., moving the mouse or typing the keyboard, are missing.</p><p>The command and control happens by periodically checking the contents of certain files on the malware server. E.g. <strong>klient.exe</strong><sup>[5]</sup> will check the content of the file hxxp://malina1306.zzz.com.ua/<strong>dow_stil.txt</strong>. If that file contains a line that matches the string <strong>&lt;MAC&gt;:&lt;OS&gt;:&lt;RAM&gt;:&lt;processor&gt;:&lt;username&gt;</strong> for the infected system (see image below), <strong>klient.exe</strong> will download the stealer<sup>[6]</sup> from hxxp://malina1306.zzz.com.ua/<strong>stel.exe </strong>and save it to <strong>c:\Windows\Temp\.Browser.exe.</strong></p><p>Similarly, a coinminer downloader<sup>[7] </sup>will be obtained if hxxp://malina1306.zzz.com.ua/<strong>dow_klip.txt</strong> has a corresponding line for the infected system. It will be downloaded from hxxp://malina1306.zzz.com.ua/<strong>klip.exe </strong>to <strong>c:\Windows\Temp\.Chrome.exe.</strong></p><p>The file 1.exe<sup>[12]</sup> is downloaded from hxxp://malina1306.zzz.com.ua/<strong>1.exe</strong> or hxxp://bests.zzz.com.ua/<strong>1.exe</strong> and saved under a randomly generated name by creating a random number between 10000 and 1000000. The resulting file location is <strong>c:\Windows\Temp\.&lt;10000-1000000&gt;.exe</strong>. This component communicates via Telegram to the malware operator.</p><p>Two more files are referenced in klient.exe but don't exist anymore<strong>: </strong>hxxp://malina1306.zzz.com.ua/<strong>min.exe</strong> would be downloaded to <strong>c:\Windows\Temp\.Jawaw Se binar.exe</strong>.&nbsp;hxxp://malina1306.zzz.com.ua/<strong>klog.exe</strong> would be downloaded to <strong>c:\Windows\Temp\.Windows Push.exe.</strong> Based on the filenames one would assume that <strong>min.exe</strong> should be the coinminer whereas <strong>klip.exe</strong> rather sounds like a clipbanker. But that was not provided by the server. <strong>klog.exe</strong> might have been a keylogger.</p><p>&nbsp;</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><a href="/fileadmin/user_upload/Presse/Deutschland/2020/12/jphp_logfilelisting.png" class="fancybox"><img class="image-embed-item " src="/fileadmin/user_upload/Presse/Deutschland/2020/12/jphp_logfilelisting.png" width="777" height="549" loading="lazy" alt="" /></a><figcaption class="image-caption"> Listing of infected clients, format: &lt;MAC&gt;:&lt;OS&gt;:&lt;RAM&gt;:&lt;processor&gt;:&lt;username&gt;. The MAC address is obfuscated by us. </figcaption></figure></div></div></div></div> </div> </div> <div data-magellan-target="c211445" class="small-12 column"> <div class="row"> <div id="c211445" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> Stealer and coinminer </h2></header><p>Unlike other IceRat components the stealer<sup>[6]</sup> is written in Python 3 and was compiled with PyInstaller to an EXE file. It steals credentials from the following browsers:</p><ul class="list-black-square"><li>Firefox</li><li>Yandex</li><li>Filezilla</li><li>Chrome</li><li>Amigo</li><li>kometa</li><li>Orbitum</li><li>Chromium</li><li>K-Melon</li></ul><p>The coinminer downloader obtains the configuration file <strong>MMMMMM.MMMM</strong><sup>[8]</sup>, the driver <strong>WinRing0x64.sys</strong><sup>[10]</sup> by OpenLibSys.org, as well as the coinminer <strong>Winlogin.exe</strong><sup>[9]</sup> from&nbsp;hxxp://malina1306.zzz.com.ua/p/. The configuration shows the user <strong>dimargo2003@gmail.com</strong>.</p><p>At the time of writing this article the stealer and the coinminer are well-detected with more than 40 detections on Virustotal. This is a remarkable contrast to the low detection rates of the JPHP components.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 "><div class="column column-block"><figure class="image"><img class="image-embed-item thumbnail" src="/fileadmin/user_upload/Presse/Deutschland/2020/12/icerat_detectionrate_jphp.png" width="1300" height="234" loading="lazy" alt="" /></figure></div><div class="column column-block"><figure class="image"><img class="image-embed-item thumbnail" src="/fileadmin/user_upload/Presse/Deutschland/2020/12/icerat_detectionrate_miner.png" width="1304" height="232" loading="lazy" alt="" /><figcaption class="image-caption"> Detection rate of Telegram communicator[12] (top) is much lower than the coinminer[9] (bottom), although the coinminer was scanned 1.5 weeks ago </figcaption></figure></div></div></div></div> </div> </div> <div data-magellan-target="c211448" class="small-12 column"> <div class="row"> <div id="c211448" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> Hosting domain </h2></header><p>The malware host and CnC server hxxp://malina1306.zzz(.)com.ua also provides a Russian website with two buttons and a text field. The field seems to require a username because the text is translated to "Enter User". The buttons say "Download miner (v1)" and "Download miner (v2)".</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1 nm-count-1 "><div class="column column-block"><figure class="image"><a href="/fileadmin/user_upload/Presse/Deutschland/2020/12/manila_panel.png" class="fancybox"><img class="image-embed-item " src="/fileadmin/user_upload/Presse/Deutschland/2020/12/manila_panel.png" width="874" height="633" loading="lazy" alt="" /></a></figure></div></div></div></div> </div> </div> <div data-magellan-target="c211447" class="small-12 column"> <div class="row"> <div id="c211447" class="fce column large-12 small-12 frame-layout-0 " ><header class=" "><h2 class=" "> Severity and targeted regions </h2></header><p>IceRat has gone unnoticed for longer than usual. I attribute this mainly to the choice of using JPHP as well as the fragmentation of the malware's features into many small files. "Small" does not mean the size of the files here. These are comparably large because they carry the JPHP runtime with them. "Small" rather refers to the amount of features they have or capability of the code. If one file does only little on its own, it won't show malicious behaviour to an automated analysis system. That way it stays undetected.</p><p>The log files that are used to communicate with the server contain more than 200 entries with different systems. Many usernames of the infected systems are kyrillic which indicates that mostly East European and Russian regions are affected.</p><p>Antivirus engines may have to upgrade their engines to support
