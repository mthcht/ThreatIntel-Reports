    <h1 id="entryTitle" class="title mb-5">Released: Decryptor for Cl0p ransomware&#8217;s Linux variant</h1>
    <aside id="65395128_t1" data-65395="t1" class="65395121 6539512_t1" style="position:relative;display:block;box-sizing:border-box;margin:auto;padding:0;text-align:center;"></aside>
    <div class="mb-5 line-height-1 content" id="entryContent">
        <p>Flawed encryption logic used in Cl0p (Clop) ransomware&#8217;s Linux (ELF) variant has allowed SentinelOne researchers to create and release a free <a href="https://github.com/SentineLabs/Cl0p-ELF-Decryptor">decryptor</a>.</p>
<p><img decoding="async" src="https://img2.helpnetsecurity.com/posts2019/keys2.jpg" class="aligncenter" alt="Cl0p decryptor Linux" title="Keys"></p>
<p>&#8220;The [Cl0p] Windows variant encrypts the generated RC4 key responsible for the file encryption using the asymmetric algorithm RSA and a public key. In the Linux variant, the generated RC4 key is encrypted with a RC4 [hardcoded] &#8216;master-key&#8217;,&#8221; the researchers explained. </p>
<h3>The differences between Windows and Linux variants</h3>
<p>The Linux Cl0p variant is relatively new, and was first spotted by the researchers in late December 2022.</p>
<p>&#8220;It appears to be in its initial development phases as some functionalities present in the Windows versions do not currently exist in this new Linux version,&#8221; they <a href="https://www.sentinelone.com/labs/cl0p-ransomware-targets-linux-systems-with-flawed-encryption-decryptor-available/">noted</a>. </p>
<p>&#8220;A reason for this could be that the threat actor has not needed to dedicate time and resources to improve obfuscation or evasiveness due to the fact that it is currently undetected by all 64 security engines on <a href="https://www.helpnetsecurity.com/tag/virustotal/">VirusTotal</a>.&#8221;</p>
<p>The differences between the Windows and Linux variant are many. For example, the former avoids encrypting specific folders, files and files with specific extenstions, and the latter does not. The former can be executed with different parameters to guide which drives will be targeted for encryption, while the latter is focused on encrypting just the specified hardcoded folders. The former carries an encrypted ransom note that gets encrypted, but the former stores the note as plain text.</p>
<p>But the most consequential difference &#8211; from the victims&#8217; perspective, that is &#8211; is the flaw that made possible the creation of the decryptor.</p>
<p>&#8220;Over the last twelve months or so we have continued to observe the increased targeting of multiple platforms by individual ransomware operators or variants,&#8221; the researchers noted. </p>
<p>&#8220;While the Linux-flavored variation of Cl0p is, at this time, in its infancy, its development and the almost ubiquitous use of Linux in servers and cloud workloads suggests that defenders should expect to see more Linux-targeted ransomware campaigns going forward.&#8221;</p>
<p>It is to be expected that Cl0p ransomware developers will fix the vulnerability soon. In the meantime, victims can use the decryption tool and look into better protecting their systems against ransomware attacks in general. </p>
