				<h1>New Apple Mac Trojan Called OSX/Crisis Discovered</h1>

        <p class="about-post">
          Posted on
          <time itemProp="datePublished" datetime="2012-07-24">July 24th, 2012</time> by

          <span itemprop="author">
            <a href="https://www.intego.com/mac-security-blog/author/lysam/" title="Posts by Lysa Myers" class="author url fn" rel="author">Lysa Myers</a>          </span>

          <img alt='' src='https://secure.gravatar.com/avatar/783af524dca7753ceb3cd9a576398a0e?s=18&#038;d=mm&#038;r=g' srcset='https://secure.gravatar.com/avatar/783af524dca7753ceb3cd9a576398a0e?s=36&#038;d=mm&#038;r=g 2x' class='avatar avatar-18 photo' height='18' width='18' loading='lazy'/>        </p>
			</header>

			<p>Intego has discovered a new Trojan called <strong>OSX/Crisis</strong>. This threat is a dropper which creates a backdoor when it&#8217;s run. It installs silently, without requiring a password, and works only in OSX versions 10.6 and 10.7 – Snow Leopard and Lion. <strong>Update:</strong> This threat may run on Leopard 10.5, but it has a tendency to crash. It does not run on the new Mountain Lion 10.8.</p>
<p>The Trojan preserves itself against reboots (i.e. it establishes persistence), so it will continue to run until it’s removed. Depending on whether or not the dropper runs on a user account with Admin permissions, it will install different components. We have not yet seen if or how this threat is installed on a user&#8217;s system; it may be that an installer component will try to establish Admin permissions.</p>
<p>If the dropper runs on a system with Admin permissions, it will drop a rootkit to hide itself. In either case, it creates a number of files and folders to complete its tasks. It creates 17 files when it’s run with Admin permissions, 14 files when it’s run without. Many of these are randomly named, but there are some that are consistent.</p>
<p>With or without Admin permissions, this folder is created in the infected user&#8217;s home directory:</p>
<ul>
<li>~/Library/ScriptingAdditions/appleHID/</li>
</ul>
<p>Only with Admin permissions, this folder is created:</p>
<ul>
<li>/System/Library/Frameworks/Foundation.framework/XPCServices/</li>
</ul>
<p>The backdoor component calls home to the IP address 176.58.100[.]37 every 5 minutes, awaiting instructions. The file is created in a way that is intended to make reverse engineering tools more difficult to use when analyzing the file. This sort of anti-analysis technique is common in Windows malware, but is relatively uncommon for OS X malware.</p>
<p>It uses low-level system calls to hide its activities, as shown in the following images:</p>
<p><a href="https://www.intego.com/mac-security-blog/new-apple-mac-trojan-called-osxcrisis-discovered-by-intego-virus-team/crisis3/" rel="attachment wp-att-5225"><img loading="lazy" class="alignnone size-full wp-image-5225 aligncenter" src="https://www.intego.com/mac-security-blog/wp-content/uploads/2012/07/Crisis3.png" alt="" width="620" height="731" srcset="https://www.intego.com/mac-security-blog/wp-content/uploads/2012/07/Crisis3.png 620w, https://www.intego.com/mac-security-blog/wp-content/uploads/2012/07/Crisis3-127x150.png 127w, https://www.intego.com/mac-security-blog/wp-content/uploads/2012/07/Crisis3-254x300.png 254w, https://www.intego.com/mac-security-blog/wp-content/uploads/2012/07/Crisis3-100x117.png 100w" sizes="(max-width: 620px) 100vw, 620px" /></a></p>
<p><a href="https://www.intego.com/mac-security-blog/new-apple-mac-trojan-called-osxcrisis-discovered-by-intego-virus-team/crisis4/" rel="attachment wp-att-5226"><img loading="lazy" class="alignnone size-full wp-image-5226 aligncenter" src="https://www.intego.com/mac-security-blog/wp-content/uploads/2012/07/Crisis4.png" alt="" width="620" height="530" srcset="https://www.intego.com/mac-security-blog/wp-content/uploads/2012/07/Crisis4.png 620w, https://www.intego.com/mac-security-blog/wp-content/uploads/2012/07/Crisis4-150x128.png 150w, https://www.intego.com/mac-security-blog/wp-content/uploads/2012/07/Crisis4-300x256.png 300w, https://www.intego.com/mac-security-blog/wp-content/uploads/2012/07/Crisis4-100x85.png 100w" sizes="(max-width: 620px) 100vw, 620px" /></a></p>
<p>Intego found samples of this malware on the <a href="https://www.virustotal.com/">VirusTotal website</a>, a site used by security companies to share malware samples. This threat has <s>not yet</s> been found in the wild<s>, and so far there is no indication that this Trojan has infected users so right now the threat is considered to be a low risk</s> (note: see updates below). Nonetheless, Intego VirusBarrier X6 detects and removes this malware using today’s definitions. It detects the dropper component as <strong>OSX/Crisis</strong>, and the backdoor component as <strong>Backdoor:OSX/Crisis</strong>. It will also block connections with the IP address the backdoor component seeks to connect with.</p>
<p><a href="https://www.intego.com/virusbarrier">Intego VirusBarrier X6</a> users should update as soon as possible to get protection from this threat.</p>
<p>We are still analyzing the threat at this time. We will post a more in-depth analysis as we have more details. <strong>Update:</strong> We have posted <a href="https://www.intego.com/mac-security-blog/more-on-osxcrisis-advanced-spy-tool/">a deeper dive into OSX/Crisis</a>, and details about <a href="https://www.intego.com/mac-security-blog/osxcrisis-has-been-used-as-part-of-a-targeted-attack/">how this OSX/Crisis variant was used in a targeted attack</a>. We have also written several <a href="https://www.intego.com/mac-security-blog/topic/osxcrisis/">write-ups about later OSX/Crisis variants</a>. You may also be interested in <a href="https://www.intego.com/mac-security-blog/an-analysis-of-the-cross-platform-backdoor-netweirdrc/">our write-up of OSX/NetWeirdRC</a> (aka NetWire), another commercial macOS remote access tool (RAT). See also <a href="https://www.intego.com/mac-security-blog/category/malware/">our latest malware write-ups</a>.<a name="learnmore"></a></p>
