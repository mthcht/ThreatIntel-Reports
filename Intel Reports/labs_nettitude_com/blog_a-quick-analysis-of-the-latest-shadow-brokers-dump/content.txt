function getCookie(name) {
    const cookieArray = document.cookie.split("; ");
    for (const cookie of cookieArray) {
        const [key, value] = cookie.split("=");
        if (key === name) {
            return decodeURIComponent(value);
        }
    }
    return null;
}

}

<div class="post-content">
<p>Just in time for Easter, the Shadow Brokers released the latest installment of an NSA data dump, which contained an almost overwhelming amount of content &ndash; including, amongst other things, a number of Windows exploits. We thought we&rsquo;d run some quick analysis on various elements of said content.</p>
<h2>Before we get started</h2>
<p>We&rsquo;re going to largely avoid the obvious elements of the dump because there&rsquo;s already been a lot of very helpful analysis of those elements. However, before we get to that, here&rsquo;s what you need to know:</p>
<ul>
<li>Patch!&nbsp; The majority of the high impact Microsoft vulnerabilities have recently been addressed in the MS17-010 patch.</li>
<li>Disable SMBv1.</li>
<li>Remove all Windows XP and 2003 machines from your network.&nbsp; These contain vulnerabilities that will not be patched.</li>
</ul>
<p>The following table (raw data available at<span>&nbsp;</span><a href="https://pastebin.com/5gkb6HLJ" target="_blank" rel="noopener noreferrer">https://pastebin.com/5gkb6HLJ</a>&nbsp;and courtesy of<span>&nbsp;</span><a href="https://twitter.com/etlow" target="_blank" rel="noopener noreferrer">@etlow</a>) contains some of the more pertinent information.</p>
<div id="attachment_12180" class="wp-caption aligncenter"><img class="size-full wp-image-12180" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/exploits.jpg?epslanguage=en" alt="Shadow Brokers Exploit Table" width="1200" height="412" aria-describedby="caption-attachment-12180" />
<p id="caption-attachment-12180" class="wp-caption-text">Shadow Brokers Exploit Table</p>
</div>
<p>We can also recommend the following script by Luke Jennings, which is designed to<span>&nbsp;</span><span class="col-11 text-gray-dark mr-2">sweep a network to find Windows systems compromised with the dumps DOUBLEPULSAR implant:<span>&nbsp;</span><a href="https://github.com/countercept/doublepulsar-detection-script" target="_blank" rel="noopener noreferrer">https://github.com/countercept/doublepulsar-detection-script</a></span><br />With that out of the way&hellip;</p>
<h2>Metadata, or a lack of</h2>
<p>Throughout the Equation Group leak via the Shadow Brokers, there are a number of different languages being used. One interesting element is how it appears that there was originally a preference for Perl, that was then replaced with Python &ndash; we think that this mirrors how the offensive security industry has evolved, too.<br />As the age of the dump is pinned at some point in 2013, we would have expected to see a little bit of PowerShell; this was really starting to come into favor around that time. Now, this post isn&rsquo;t about dropping a new l33t PowerShell technique gained from the dump, but rather looking at what the capability was at the point in time. Staying with the timing of the dump for a minute, we are reminded of the following series of Tweets from Edward Snowden back in August last year, when the ShadowBrokers [6] first dropped.<br /><img class="wp-image-12159 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image.png?epslanguage=en" width="577" height="380" /><br />We know we run the risk of taking these out of context, and it is entirely possible that his mind has been changed since, however we find the following piece of information interesting. According to the time line from the Guardian [5], the first release of the material he took was on the 5<sup>th</sup><span>&nbsp;</span>June 2013. It&rsquo;s probable that other dumps have since has contradicted this and the view of when the hacker/s were kicked off has been able to be narrowed, but we am unaware of this (so please if you know different answers on a postcard).<br />Examining of the tools<span>&nbsp;</span><em>makedmgd.exe</em>, part of a toolkit DAMAGEDGOODS that is used within in a PowerShell delivery framework ZIPO we see the following. One of the first things that we noticed is that yeah hmmm the build date is baked into the exe. Also some different implants not within the dump are there<span>&nbsp;</span><em>&ldquo;distantuncle&rdquo;</em><span>&nbsp;</span>and<span>&nbsp;</span><em>&ldquo;finkdiffernt&rdquo;;</em><span>&nbsp;</span>some of the coders definitely have a certain sense of humor.<br /><img class="wp-image-12160 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-1.png?epslanguage=en" width="697" height="297" /><br />Using Sysinternals excellent sigcheck.exe [7] we could view the publisher, version and build date in order to correlate. Yes, it is one of the many ways to list a binarys metadata, but some of its other superb features are that, as the name implies, it will verify the signature if the binary has been signed using Authenticode and it is also able to send the binary straight to VirusTotal and look at all files within a directory tree recursively. Running sigcheck, unsurprisingly we get the following information or, some would say, a lack of.<br /><img class="wp-image-12161 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-2.png?epslanguage=en" width="711" height="295" /><br />Any trace of publisher or company which, to be fair, will be set in Visual Studio (or your toolchain of choice have either been stripped or not set). The Link date is there, which correlates to the build date, which is also five weeks after Snowden&rsquo;s material was first dropped. It is entirely possible to mess with and edit these dates, of course, before releasing the dump. We do find it strange to go the level of stripping all other information but hard coding a build date, particularly in a tool that will be released to a workstation. The directory structure that this is in implies it may have been copied in rather than part of a release, as it was new and may not have been sanitised properly (although there is a real danger of reading too much into it).</p>
<h2>First steps into PowerShell</h2>
<p>As stated above, we would have expected to see a reasonable amount of PowerShell considering the year, but actually there is very little. The only real example that we have found is a tool called ZiPo which can be found within the dump at /Resources/Ops/Tools/ZiPo. It contains the following tools</p>
<ul>
<li>decryptor_downloader.base</li>
<li>makedmgd.exe</li>
<li>ZIPO.py</li>
<li>ps_base.txt</li>
<li>powershellify.py</li>
</ul>
<p>In order to run this tool we call ZIPO.py, which first asks you to select a &ldquo;project&rdquo; directory then presents a menu asking if we want to:</p>
<ol>
<li>Upload / Create Execute an Egg</li>
<li>Upload/ Create PowerShell script</li>
<li>Create Compressed script to be run manually</li>
</ol>
<p>Now Egg is a term that is used quite heavily throughout the dump and we&rsquo;re not entirely sure what it means at this point in time. Pretty sure it is an Equation Group term.<br /><img class="wp-image-12162 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-3.png?epslanguage=en" width="684" height="580" /><br />Choosing PowerShell script we are then asking for the location of it, what the IP address and port of the &ldquo;redirector&rdquo; which we assume is a proxy and then the local IP address and proxy. This is so that the script can spin up a HTTPd listener to serve up the files that have been created.<br />In order to test, we created a very simple PowerShell script containing:</p>
<pre class="lang:ps decode:true ">[System.Reflection.Assembly]::LoadWithPartialName("System.Windows.Forms")
[System.Windows.Forms.MessageBox]::Show("Hey mate, do you wanna run some powershell?", "you know you want too", 'Ok')</pre>
<p><img class="wp-image-12163 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-4.png?epslanguage=en" width="1199" height="451" /><br />It has generated a public/private key pair, created an index.html &amp; index.htm, provided us with a script to run on the target and also started up a HTTPd so that we could download the payloads on the target. That&rsquo;s not too bad for a couple of commands.<br /><img class="wp-image-12164 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-5.png?epslanguage=en" width="820" height="371" /><br />Looking at the command to run its pretty standard PowerShell from the time, in fact we find it really interesting there is absolutely no attempt at obfuscating anything here. They are encrypting the payload and building a chain to download/decrypt etc, but no effort is made at hiding what the command is doing or where it is obtaining the script from (of course we would be very interested to see what they are doing now).<br />So what is contained within the two index files? Well, index.html is base64 PowerShell script, which is why it was executed as an encodedCommand; decoding you get the output below. It encrypts a known &ldquo;questionable&rdquo; password value using RSA, another WebClient is created which has the encrypted value set as a cookie. The index.html is then downloaded and decrypted using the key, which is a SHA1 hash of the &ldquo;questionable value&rdquo;. The payload is then executed and on the server the two files are then deleted. This is a lot of effort to hide the final payload and once again absolutely no effort to obfuscate any of the script.<br /><img class="wp-image-12165 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-6.png?epslanguage=en" width="1200" height="620" data-recalc-dims="1" /><br />This is how it looks when it is run:<br /><img class="wp-image-12166 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-7.png?epslanguage=en" width="802" height="741" /></p>
<h2>DAMAGEDGOODS</h2>
<p>The next thing that we did was to just create a meterpreter payload; nothing special and wasn&rsquo;t going to get to connect back, but we felt that AV should still be able to pick it up.<br /><img class="wp-image-12167" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-8.png?epslanguage=en" width="1200" height="162" /><br />Running Zipo again, we selected the third option. It asks you for a payload DLL and also the ordinal [8] that you want to fire. This is where DAMAGEDGOODS comes into play; makedmged.exe is the exe that appears to do some kind of shellcode encoding. In this case it takes the encoded binary with a script called ps_base.txt, then compresses/base64 encodes and then builds a decompression payload around it.<br /><img class="wp-image-12168 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-9.png?epslanguage=en" width="797" height="444" /><br />The script that is output at the end of this using the name you supplied is the decode/decompression/execute mentioned above and is shown below.<br /><img class="wp-image-12169 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-10.png?epslanguage=en" width="1131" height="628" /><br />Decoding it you get the following, which is quite interesting; it&rsquo;s a PowerShell script that allocates memory, writes the shellcode into it, creates a thread and then executes the shellcode, all in memory. The shellcode in this case is going to be the meterpreter DLL that we originally used. Running it multiple times over the same DLL you get a different version. There appears to be some kind of prologue in the shellcode that doesn&rsquo;t change, but it is pretty short, running the script multiple times and then diffing with Scooter Software&rsquo;s excellent Beyond Compare you find that the only section that has changed is the shellcode except for:<br /><img class="wp-image-12170 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-11.png?epslanguage=en" width="1200" height="842" data-recalc-dims="1" /><br />This series of bytes which appears to be some kind of prologue probably a decoder for the rest of the shellcode. What does it do, how does it work? Well that, we&rsquo;re afraid, is for part 2 as we&rsquo;ve spent too much time away from the family already this easter ;o)</p>

<h2>This kinda looks familiar&hellip;.</h2>
<p>Now the great irony in a dump like this is finding code that appears to have come from GitHub but doesn&rsquo;t appear to have the same licence or any at all for that matter [1]. This script is built from file called<span>&nbsp;</span><em>ps_base.txt.<span>&nbsp;</span></em>This is primarily used to dynamically build a type that will eventually hold a function pointer to a native function. This is then used to store the fp&rsquo;s for native functions Win32 functions such as VirtualAlloc[2], GetProcessAddress[3] &amp; GetModuleHandle[4] that can be used to perform some actions such as allocating memory and looking up the addresses of exports within DLL&rsquo;s. Further are shown in this screen shot:<br /><img class="wp-image-12172 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-13.png?epslanguage=en" width="1200" height="222" /><br />Now the method to create the delegate&rsquo;s used in the above code is:<br /><img class="wp-image-12173 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-14.png?epslanguage=en" width="1200" height="669" data-recalc-dims="1" /><br />Programmers (ourselves included) can be utter sticklers for formatting, so it is conspicuous that there is such a big difference in formatting between the code in ps_base.txt vs decryptor_downloader.base. It&rsquo;s almost as if ps_base.txt has come from somewhere else.<br /><img class="wp-image-12174 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-15.png?epslanguage=en" width="698" height="517" /><br />Well funnily enough it bears more than just a striking resemblance to some code from Powersploit[1]; screenshots from GitHub are below. Surprisingly not too much effort has been made to change the method names.<br /><img class="wp-image-12175 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-16.png?epslanguage=en" width="1200" height="757" /><br />And also&hellip;<br /><img class="wp-image-12176 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-17.png?epslanguage=en" width="1200" height="757" /><br />The commit date for this code is&hellip;<br /><img class="wp-image-12177 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-18.png?epslanguage=en" width="1200" height="210" /><br />And as stated above we have a built date of July 2013; does this mean we will find StackOverflow answer code within the dump at some point?<br />But anyway back to makedmg.exe running it we get this list of other implants that are not in this dump; obviously still a lot out there.<br /><img class="wp-image-12178 aligncenter" src="/globalassets/cyber-labs/a-quick-analysis-of-the-latest-shadow-brokers-dump/word-image-19.png?epslanguage=en" width="514" height="754" /></p>
<h2>DOUBLEPULSAR</h2>
<p>From analysis we did on some implant configuration files, Darkpulsar appears to create a service called &lsquo;dapu&rsquo; It also seems that when it upgrades itself it drops the new file using the following path: &lsquo;c:\windows\system32\sipauth32.tsp&rsquo;.<br />We also had a look at tdip.sys driver.<br />(sha256: A5EC4D102D802ADA7C5083AF53FD9D3C9B5AA83BE9DE58DBB4FAC7876FAF6D29)<br />We found same magic DWORDs as those mentioned by Kaspersky Labs in the following link:<span>&nbsp;</span><a href="https://securelist.com/blog/incidents/75812/the-equation-giveaway/" target="_blank" rel="noopener noreferrer">https://securelist.com/blog/incidents/75812/the-equation-giveaway/</a><span>&nbsp;</span>which contains information from a previous &lsquo;ShadowBrokers&rsquo; dump.<br />The following code snippet is taken from tdip.sys:</p>
