<div class=" bs-column col-sm-12  col-lg-8 col-md-8   bs-column-1ca2941759c923e2c1aada87854c0e268646c7c4 bs-column---default bs-column--blog-inner-tags     "><div class="wp-block-post-author contributor"><div class="wp-block-post-author__avatar"><img decoding="async" src="https://vipre.com/wp-content/uploads/2024/05/vipre-author-logo.png" alt="VIPRE Labs"></div><div class="wp-block-post-author__content"><p class="wp-block-post-author__name"><a href="https://vipre.com/contributor/vipre-labs/" style="color:#1B2149;">VIPRE Labs</a></p><div class="wp-block-post-author__bio"></div></div></div>



<p>The COVID-19 pandemic has resulted in people ramping up online activities working from home, online shopping and relying more on online services. Recently, we came across a spam email lurking in the wild. This spam email is disguised as a known logistics company and has an unusual attachment. Malicious attackers trick the victim into believing that the email is legitimate by using a legitimate domain in the sender&#8217;s email address. The content is also properly constructed and also uses a known logo making it difficult to spot that it is a malicious email.</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-emlfile_censored-768x559-1.jpg" alt="Figure 1.0 Spam email with .jnlp attachment"/></figure>



<p><em><strong>Figure 1.0 Spam email with .jnlp attachment</strong></em></p>



<p>As threats become more prominent, we should always be cautious. These are some indicators that will show that this email is suspicious and not legitimate:</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-emlheader_censored.jpg" alt="Figure 2.0 The email header"/></figure>



<p><em><strong>Figure 2.0 The email header</strong></em></p>



<ul>
<li>Checking the email header, we can see that the &#8220;received from&#8221; which is in the green box in Figure 2.0, didn&#8217;t match with the &#8220;from&#8221; field (the visible sender of the email). The &#8220;received from&#8221; data is the most reliable and it is where we can see the real sender of the email. Upon researching, the domain in the &#8220;received from&#8221; header is not related to DHL. With this, the email header is forged.</li>



<li>An Additional checker is the Received-SPF: softfail. It says that the &#8220;domain of DHL.COM does not designate 45.88.105.192 as permitted sender&#8221;. Upon checking, the IP address 45.88.105.192 in the &#8220;received from&#8221; is not also related to DHL.</li>



<li>The attachment of the email is a .jnlp file is a Java Network Launch Protocol which is an unusual attachment for an email.</li>
</ul>



<p><strong>Analyzing the attachment</strong></p>



<p>We will now proceed on the analysis of the jnlp file attachment that has a filename &#8220;invoice.jnlp&#8221;. We said earlier that .jnlp stands for Java Network Launch Protocol, that&#8217;s used for launching java applications from a hosted web server on a remote desktop client. Checking the jnlp file, we can see that the file will download invoice.jar from a web server hxxp://invoicesecure[.]net/documents when executed.</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-jnlp.png" alt="Figure 3.0 The jnlp file"/></figure>



<p><em><strong>Figure 3.0 The jnlp file</strong></em></p>



<p>The downloaded file is an invoice.jar file which is a Java Archive file. When we tried to launch the file it will show this output:</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-outputjar-300x155-1.png" alt="Figure 4.0 The error message upon launching invoice.jar"/></figure>



<p><em><strong>Figure 4.0 The error message upon launching invoice.jar</strong></em></p>



<p>With this message, the victim will think that it was an error and will ignore the file. But upon analyzing the invoice.jar, we found out that this message is just a decoy. The attackers just made this technique to trick their victims and make the malware run without suspicion. Based on its code after showing an error message, it will start to read the data from &#8220;hxxp://invoicesecure[.]net/img/footer[.]jpg&#8221; and saved it as &#8220;C:\ProgramData\drvr32.exe&#8221;. Then use Desktop.getDesktop().open() to open drvr32.exe.</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-join.png" alt="Figure 5.0 The decompiled invoice.jar"/></figure>



<p><em><strong>Figure 5.0 The decompiled invoice.jar</strong></em></p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-wiresharkgetrequest-676x78-1.png" alt="Figure 6.0 The HTTP GET Request once invoice.jar was executed"/></figure>



<p><em><strong>Figure 6.0 The HTTP GET Request once invoice.jar was executed</strong></em></p>



<p><strong>The Buer Loader</strong></p>



<p>The malicious downloaded file was named &#8220;drvr32.exe&#8221; and disguised as a legitimate xls viewer application:</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-disguise_censored-768x278-1.jpg" alt="Figure 7.0 Disguising as a legitimate file"/></figure>



<p><em><strong>Figure 7.0 Disguising as a legitimate file</strong></em></p>



<p>This file was identified as a type of a malware loader known as&nbsp;Buer Loader. This loader was first seen in 2019 and commonly distributed through malicious spam email campaigns.</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-drvr32-399x114-1.png" alt="Figure 8.0 The buer loader"/></figure>



<p><em><strong>Figure 8.0 The buer loader</strong></em></p>



<p>When executed, it will first create its installation folder &#8220;zsadsadsad&#8221; at the Startup folder and create a copy of itself in %AppData%. The created folder &#8220;zsadsadsad&#8221; contains LNK shortcut file. We decoded the LNK file to analyze all the available information it contains and we found out that it will link to the created copy.</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-persistence.png" alt="Figure 9.0 The installation folder zsadsadsad and the lnk shortcut file"/></figure>



<p><em><strong>Figure 9.0 The&nbsp;installation folder&nbsp;&#8220;zsadsadsad&#8221; and the lnk shortcut file</strong></em></p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-decodedlnk.png" alt="Figure 10.0 The decoded information of LNK file linking to the created copy"/></figure>



<p><em><strong>Figure 10.0 The decoded information of LNK file linking to the created copy</strong></em></p>



<p>Throughout our analysis, we found out that this loader has an anti-analysis. It will check if the following DLLs are existing in the place where the malware is running:</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-checkdllrunning.png" alt="Figure 11.0 The DLLs to check"/></figure>



<p><em><strong>Figure 11.0 The DLLs to check</strong></em></p>



<p>As per checking, some of the checked DLLs above are related to anti-virus and debuggers.</p>



<p>Then, it will call functions like GetCurrentHwProfileA, GetComputerNameW, and GetVolumeInformation to collect the information of the infected machine. The collected information will be combined in an allocated memory and will be formatted using wsprintfw function.</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-formatstrings.png" alt="Figure 12.0 Routine for formatting the collected information"/></figure>



<p><em><strong>Figure 12.0 Routine for formatting the collected information</strong></em></p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/combinecollecteddataFormatted-768x336-1.png" alt="Figure 13.0 The formatted string of victim's machine information"/></figure>



<p><em><strong>Figure 13.0 The formatted string of victim&#8217;s machine information</strong></em></p>



<p>After this, it will call other functions to retrieve more information of the infected machine and to use these information for the malware&#8217;s next actions:</p>



<ul>
<li>RtlGetVersion</li>



<li>GetNativeSystemInfo</li>



<li>GetComputerNameW</li>



<li>GetDriveTypeA</li>



<li>GetDiskFreeSpaceExA</li>



<li>GetUserNameW</li>



<li>NetWkstaGetInfo</li>
</ul>



<p>All of the other retrieved information will be combined to the formatted string above and the output is this:</p>



<p><strong>cb9f1daacc0545c0e5bf0c67ead614d24884570889cf7435572fa55e134b1012|bc31re1bs5a8d1fc4ddb3cc4b75594c31b8c00de3fdfa31fgg1ad15e876dbfhs|Windows 7|x32|1|User|WIN-SPF5F5SH244|14/59|WORKGROUP|test|0&nbsp;</strong></p>



<p>Then, this output will be formatted again using wsprintfw function and the result is this:</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-formatstrings33a.png" alt="Figure 14.0 The 2nd round formatting of string of the victim's machine information"/></figure>



<p><em><strong>Figure 14.0 The 2nd round formatting of string of the victim&#8217;s machine information</strong></em></p>



<p>After retrieving and formatting the needed information of the victim&#8217;s machine,&nbsp;Buer Loader will make it to a base64 string:</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-combinebased64-1024x429-1.png" alt="Figure 15.0 Converting to base64 string"/></figure>



<p><em><strong>Figure 15.0 Converting to base64 string</strong></em></p>



<p>Digging deeper into our analysis, we encountered InternetOpenA function to initialize a use of the WinINet functions. Then, it will try to open an http session to &#8220;verstudiosan[.]com&#8221; using InternetConnectW function.</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-internetconnect.png" alt="Figure 16.0 Opens an HTTP session"/></figure>



<p><em><strong>Figure 16.0 Opens an HTTP session</strong></em></p>



<p>It has GET method to download additional malware and POST method to send the collected victim&#8217;s machine information to the server:</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-httpopenrequest.png" alt="Figure 17.0 HTTP POST Request method"/></figure>



<p><em><strong>Figure 17.0 HTTP POST Request method</strong></em></p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-httpsendrequest-768x69-1.png" alt="Figure 18.0 Sending the specified request"/></figure>



<p><em><strong>Figure 18.0 Sending the specified request</strong></em></p>



<p>We searched the domain and found out that this domain was just recently created. We also learned that this domain is no longer reachable and possibly just used for malicious activity.</p>



<figure class="wp-block-image"><img decoding="async" src="/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-whois.png" alt="Figure 19.0 The recently created domain"/></figure>



<p><em><strong>Figure 19.0 The recently created domain</strong></em></p>



<figure class="wp-block-image size-full"><img fetchpriority="high" decoding="async" width="1024" height="346" src="https://vipre.com/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-unreachableserver_censored-1024x346-1.jpg" alt="Figure 20.0 Unreachable server" class="wp-image-24610" srcset="https://vipre.com/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-unreachableserver_censored-1024x346-1.jpg 1024w, https://vipre.com/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-unreachableserver_censored-1024x346-1-300x101.jpg 300w, https://vipre.com/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-unreachableserver_censored-1024x346-1-768x260.jpg 768w, https://vipre.com/wp-content/uploads/2024/04/buer-loader-found-in-an-unusual-email-attachment-unreachableserver_censored-1024x346-1-18x6.jpg 18w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure>
