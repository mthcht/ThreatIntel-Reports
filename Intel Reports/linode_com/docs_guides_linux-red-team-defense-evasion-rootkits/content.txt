Rootkits are programs that hide the existence of malware by intercepting/hooking and modifying operating system API calls that supply system information.</p><p>Rootkits are typically utilized as a part of the defense evasion tactic and are set up after an initial foothold has been obtained.</p><p>The type of rootkit you utilize will depend on the target configuration and your requirements.</p><p>The primary objective of a rootkit is to maintain some form of clandestine access for the attacker.</p><p>In this case, we will be setting up a rootkit in the form of an Apache module that will be used to provide us with backdoor access to the target system.</p><p>As mentioned previously, when it comes to Rootkits and backdoors, simplicity and efficiency is key. Furthermore, having an understanding of the various pieces of software installed on the target provides you with an opportunity to utilize one of these services/programs to maintain access.</p><p>Additionally, utilizing software that is installed on the target provides you with anonymity as system administrators or security analysts will find it difficult to detect any anomalies or rogue processes.</p><h2 id=setting-up-apache-rootkit>Setting Up apache-rootkit<a href=#setting-up-apache-rootkit aria-label="Linux Red Team Defense Evasion - Rootkits: Setting Up apache-rootkit" class=group><svg class="ml-2 inline-block w-5 h-5 text-brand group-hover:text-blue-400"><use href="#icon--hashtag"/></svg></a></h2><p>In this case, the Linux target that I have compromised is used by the organization to host a website.
