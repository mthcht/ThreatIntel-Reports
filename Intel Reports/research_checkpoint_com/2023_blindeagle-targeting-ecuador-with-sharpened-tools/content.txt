<h1 class="wp-block-heading">HIGHLIGHTS:</h1>



<ul class="wp-block-list">
<li><em>APT-C-36, also known as Blind Eagle, is a financially motivated threat group that has been launching indiscriminate attacks against citizens of various countries in South America</em> since at least 2018.</li>



<li><em>In a recent campaign targeting Ecuador based organizations, CPR detected a new infection chain that involves a more advanced toolset.</em></li>



<li><em>The backdoor chosen for this campaign is typically used by espionage campaigns, which is unusual for this group</em></li>
</ul>



<h1 class="wp-block-heading">ACTIVE CAMPAIGNS AGAINST COLOMBIAN TARGETS</h1>



<p class="wp-block-paragraph">For the last few months, we have been observing the ongoing campaigns orchestrated by Blind Eagle, which have mostly adhered to the TTPs described above — phishing emails pretending to be from the Colombian government. One typical example is an email purportedly from the Ministry of Foreign Affairs, threatening the recipient with issues when leaving the country unless they settle a bureaucratic matter.</p>



<p class="wp-block-paragraph">Such emails usually feature either a malicious document or a malicious link, but in this case, the attackers said “why not both?” and included both a link and a terse attached PDF directing the unfortunate victim to the exact same link.</p>


<div class="wp-block-image">
<figure class="aligncenter size-full"><img fetchpriority="high" decoding="async" width="593" height="389" src="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_0.png" alt="" class="wp-image-27147" srcset="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_0.png 593w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_0-300x197.png 300w" sizes="(max-width: 593px) 100vw, 593px" /></figure>
</div>


<p class="wp-block-paragraph">In both cases, the link in question consists of a legitimate link-shortening service URL that geolocates victims and makes them communicate with a different “server” depending on the original country (<code>https://gtly[.]to/QvlFV_zgh</code>). If the incoming HTTP request originates from outside Colombia, the server aborts the infection chain, acts innocent and redirects the client to the official website of the migration department of the Colombian Ministry of Foreign Affairs.</p>


<div class="wp-block-image">
<figure class="aligncenter size-full"><img decoding="async" width="656" height="86" src="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_1.png" alt="" class="wp-image-27137" srcset="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_1.png 656w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_1-300x39.png 300w" sizes="(max-width: 656px) 100vw, 656px" /></figure>
</div>


<p class="wp-block-paragraph">If the incoming request seems to arrive from Colombia, the infection chain proceeds as scheduled. The server responds to the client with a file for download. This is a malware executable hosted on the file-sharing service MediaFire. The file is compressed, similar to a ZIP file, using the <a href="https://en.wikipedia.org/wiki/LHA_(file_format)">LHA algorithm</a>. It is password-protected, making it impervious against naive static analysis and even naive sandbox emulation. The password is found both in the email and in the attached PDF.</p>


<div class="wp-block-image">
<figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="96" src="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_2-1024x96.png" alt="" class="wp-image-27138" srcset="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_2-1024x96.png 1024w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_2-300x28.png 300w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_2-768x72.png 768w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_2.png 1090w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /></figure>
</div>


<p class="wp-block-paragraph">The malicious executable inside the LHA is written in .Net and packed. When unpacked, a modified sample of QuasarRAT is revealed.</p>



<p class="wp-block-paragraph">QuasarRAT is an open source trojan available in multiple sources like Github. The (probably Spanish-speaking) actors behind this APT group have added some extra capabilities over the last few years, which are easy to spot due to the names of functions and variables in Spanish. This process, by which threat actors abuse access to malware sources and each create their own special versions of that malware, is sadly not without precedent in the security landscape and always <a href="https://research.checkpoint.com/2020/gozi-the-malware-with-a-thousand-faces/">makes us heave a sad sigh when we encounter it</a>.</p>



<p class="wp-block-paragraph">Although QuasarRAT is not a dedicated banking Trojan, it can be observed from the sample’s embedded strings that the group’s main goal in the campaign was to intercept victim access to their bank account.</p>


<div class="wp-block-image">
<figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="627" height="388" src="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_3.png" alt="" class="wp-image-27139" srcset="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_3.png 627w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_3-300x186.png 300w" sizes="auto, (max-width: 627px) 100vw, 627px" /></figure>
</div>


<p class="wp-block-paragraph">This is a complete list of targeted entities:</p>



<figure id="36066d0a-c9e6-44aa-91fc-b11e8bf22db6" class="wp-block-table"><table><tbody><tr><td>Bancolombia Sucursal Virtual Personas</td></tr><tr><td>Sucursal_Virtual_Empresas_</td></tr><tr><td>Portal Empresarial Davivienda</td></tr><tr><td>BBVA Net Cash</td></tr><tr><td>Colpatria &#8211; Banca Empresas</td></tr><tr><td>bancaempresas.bancocajasocial.com</td></tr><tr><td>Empresarial Banco de Bogota</td></tr><tr><td>conexionenlinea.bancodebogota.com</td></tr><tr><td>AV Villas &#8211; Banca Empresarial</td></tr><tr><td>Bancoomeva Banca Empresarial</td></tr><tr><td>TRANSUNION</td></tr><tr><td>Banco Popular</td></tr><tr><td>portalpymes</td></tr><tr><td>Blockchain</td></tr><tr><td>DashboardDavivienda</td></tr></tbody></table></figure>



<p class="wp-block-paragraph">Some extra features added to Quasar by this group are a function named “ActivarRDP” (activate RDP) and two more to activate and deactivate the system Proxy:</p>


<div class="wp-block-image">
<figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="98" src="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_4-1024x98.png" alt="" class="wp-image-27140" srcset="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_4-1024x98.png 1024w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_4-300x29.png 300w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_4-768x73.png 768w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_4.png 1268w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /></figure>
</div>


<p class="wp-block-paragraph">Along with a few more commands that incur technical debt by impudently disregarding Quasar’s convention for function name and parameter order:</p>


<div class="wp-block-image">
<figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="950" height="686" src="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_5.png" alt="" class="wp-image-27141" srcset="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_5.png 950w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_5-300x217.png 300w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_5-768x555.png 768w" sizes="auto, (max-width: 950px) 100vw, 950px" /></figure>

<h1 class="wp-block-heading">A BETTER CAMPAIGN FEATURING NEWER TOOLS</h1>



<p class="wp-block-paragraph">One specific sample caught our attention as it was related to a government institution from Ecuador and not from Colombia. While Blind Eagle attacking Ecuador is not unprecedented, it is still unusual. Similarly to the campaign described above, the geo-filter server in this campaign redirects requests outside of Ecuador and Colombia to the website of the Ecuadorian Internal Revenue Service:</p>


<div class="wp-block-image">
<figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="678" height="345" src="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_6.png" alt="" class="wp-image-27142" srcset="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_6.png 678w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_6-300x153.png 300w" sizes="auto, (max-width: 678px) 100vw, 678px" /></figure>
</div>

<div class="wp-block-image">
<figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="523" height="49" src="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_7.png" alt="" class="wp-image-27143" srcset="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_7.png 523w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_7-300x28.png 300w" sizes="auto, (max-width: 523px) 100vw, 523px" /></figure>

<p class="wp-block-paragraph">If contacted from Colombia or Ecuador, the downloaded file from Mediafire will be a RAR archive with a password. But instead of a single executable consisting of some packed RAT, the infection chain, in this case, is much more elaborate:</p>


<div class="wp-block-image">
<figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="959" height="317" src="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_8.png" alt="" class="wp-image-27144" srcset="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_8.png 959w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_8-300x99.png 300w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_8-768x254.png 768w" sizes="auto, (max-width: 959px) 100vw, 959px" /></figure>

<p class="wp-block-paragraph">Inside the RAR archive, there is an executable built with PyInstaller with a rather simplistic Python 3.10 code. This code just adds a new stage in the infection chain:</p>



<pre class="EnlighterJSRAW" data-enlighter-language="generic" data-enlighter-theme="" data-enlighter-highlight="" data-enlighter-linenumbers="" data-enlighter-lineoffset="" data-enlighter-title="" data-enlighter-group="">import os

os.system(wsx)
</pre>



<p class="wp-block-paragraph"><code>mshta</code> is a utility that executes Microsoft HTML Applications, and the attackers abuse it here to download and execute the next stage, which contains VBS code embedded in an HTML.</p>



<pre class="EnlighterJSRAW" data-enlighter-language="generic" data-enlighter-theme="" data-enlighter-highlight="" data-enlighter-linenumbers="" data-enlighter-lineoffset="" data-enlighter-title="" data-enlighter-group="">&lt;script language="VBScript">

</pre>



<p class="wp-block-paragraph">Usually campaigns by Blind Eagle abuse legitimate file sharing services such as Mediafire or free dynamic domains like &#8220;*.linkpc.net&#8221;; this case is different, and the next stage is hosted at the malicious domain upxsystems[.]com.</p>



<p class="wp-block-paragraph">This next-stage downloads and executes yet another next-stage, a script written in Powershell:</p>



<pre class="EnlighterJSRAW" data-enlighter-language="generic" data-enlighter-theme="" data-enlighter-highlight="" data-enlighter-linenumbers="" data-enlighter-lineoffset="" data-enlighter-title="" data-enlighter-group="">function StartA{

<p class="wp-block-paragraph">The above Powershell checks the system version and downloads the appropriate additional Powershell. This additional OS-specific Powershell checks for installed AV tools and behaves differently based on its findings.</p>



<p class="wp-block-paragraph">The main difference between each next stage consists in different pieces of code that will try to disable the security solution (for example Windows Defender), but in all cases, regardless of the type of security solution installed on the computer, the next stagewill download a version of python suitable for the target OS and install it:</p>

        Set-ItemProperty -path 'hkcu:\\Software\\Classes\\Local Settings\\Software\\Microsoft\\Windows\\Shell\\MuiCache\\' -name "$env:PUBLIC\\py\\python.exe.ApplicationCompany" -value "Python Software Foundation"
        Set-ItemProperty -path 'hkcu:\\Software\\Classes\\Local Settings\\Software\\Microsoft\\Windows\\Shell\\MuiCache\\' -name "$env:PUBLIC\\py\\python.exe.FriendlyAppName" -value "Python"

       }
....
  
</pre>



<p class="wp-block-paragraph">It will then download two scripts named <code>mp.py</code> and <code>ByAV2.py</code> which will be stored in the user <code>%Public%</code> folder and for which it will create a scheduled task that will run every 10 minutes. For Windows 7 the task will be created by downloading an XML from the C2 &#8220;upxsystems[.]com&#8221;, while for Windows 8, 8.1, and 10 the malware will create the task using the cmdlet &#8220;New-ScheduledTask*&#8221;.</p>



<p class="wp-block-paragraph">In the case of Windows 7, the task is preconfigured to be executed as System and contains the following description</p>



<blockquote class="wp-block-quote is-layout-flow wp-block-quote-is-layout-flow">
<p class="wp-block-paragraph"><em>&lt;Description&gt; Mantiene actualizado tu software de Google. Si esta tarea se desactiva o se detiene, tu software de Google no se mantendrá actualizado, lo que implica que las vulnerabilidades de seguridad que puedan aparecer no podrán arreglarse y es posible que algunas funciones no anden. Esta tarea se desinstala automáticamente si ningún software de Google la utiliza. &lt;/Description&gt;</em></p>
</blockquote>



<p class="wp-block-paragraph">It’s written using the kind of Spanish that is commonly spoken in the target countries, which can be noticed for example with the use of “<em>es posible que algunas funciones <strong>no anden</strong></em>” instead of “<strong>no se ejecuten</strong>” or any other variation more common in different geographic regions.</p>



<p class="wp-block-paragraph">The full description can be translated to:</p>



<p class="wp-block-paragraph">“<em>Keeps your Google software up to date. If this task is disabled or stopped, your Google software will not be kept up to date, which means that security vulnerabilities that may appear cannot be fixed and some features may not work. This task is automatically uninstalled if no Google software uses it.</em>”</p>



<p class="wp-block-paragraph">After downloading the Python scripts and adding persistence, the malware will try to kill all processes related to the infection.</p>



<p class="wp-block-paragraph">Regarding the two downloaded scripts, both are obfuscated using homebrew encoding that consists of base64 repeated 5 times (we will never, ever, tire of responding to such design choices with “<em>known to be 5 times as secure as vanilla base64</em>”):</p>


<div class="wp-block-image">
<figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="544" height="163" src="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_9.png" alt="" class="wp-image-27145" srcset="https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_9.png 544w, https://research.checkpoint.com/wp-content/uploads/2023/01/BlindEagle_9-300x90.png 300w" sizes="auto, (max-width: 544px) 100vw, 544px" /></figure>
</div>


<p class="wp-block-paragraph">After deciphering these strings for each script we obtain two different types of Meterpreter samples.</p>



<h2 class="wp-block-heading"><a href="http://ByAV2.py">ByAV2.py</a></h2>



<p class="wp-block-paragraph">This code consists of an in-memory loader developed in Python, which will load and run a normal Meterpreter sample in DLL format that uses “tcp://systemwin.linkpc[.]net:443” as a C2 server.</p>



<p class="wp-block-paragraph">Python has a built-in PRNG, and in principle no one is stopping you from constructing a stream cipher based on it, which is what the malware authors do here. The embedded DLL is decrypted using this makeshift “randint stream cipher” with an embedded key (in this construction the key is used as the seed to prime the <code>random</code> library). In the grand tradition of cryptography used inside of malware purely to obfuscate buffers using a hardcoded key, the question of how secure this makeshift cipher is has exactly zero consequences.</p>


<div class="wp-block-image">
<figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="441" src="https://research.checkpoint.com/wp-content/uploads/2023/01/image-9-1024x441.png" alt="" class="wp-image-27202" srcset="https://research.checkpoint.com/wp-content/uploads/2023/01/image-9-1024x441.png 1024w, https://research.checkpoint.com/wp-content/uploads/2023/01/image-9-300x129.png 300w, https://research.checkpoint.com/wp-content/uploads/2023/01/image-9-768x331.png 768w, https://research.checkpoint.com/wp-content/uploads/2023/01/image-9.png 1448w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /></figure>
</div>


<h2 class="wp-block-heading"><a href="http://mp.py">mp.py</a></h2>



<p class="wp-block-paragraph">The second script basically consists of another sample of Meterpreter — this time a version <a href="https://github.com/rapid7/metasploit-payloads/blob/master/python/meterpreter/meterpreter.py">developed entirely in Python</a> and using the same C2 server. We can only speculate on why the server was configured to drop the same payload with the same C2 server but written in a different language; possibly one of the samples acts as a plan B in case of the other sample gets detected by some antivirus solution and removed.</p>

<h1 class="wp-block-heading">CONCLUSION</h1>



<p class="wp-block-paragraph">Blind Eagle is a strange bird among APT groups. Judging by its toolset and usual operations, it is clearly more interested in cybercrime and monetary gain than in espionage; however, unlike most such groups that just attack the entire world indiscriminately, Blind Eagle has a very narrow geographical focus, most of the time limited to a single country. This latest campaign targeting Ecuador highlights how, over the last few years, Blind Eagle has matured as a threat — refining their tools, adding features to leaked code bases, and experimenting with elaborate infection chains and “Living off the Land” as seen with the clever abuse of <code>mshta</code>. If what we’ve seen is any indication, this group is worth keeping an eye on so that victims aren’t blindsided by whatever clever thing they try next.</p>



<p class="wp-block-paragraph">Check Point’s anti-phishing <a href="https://www.checkpoint.com/harmony/anti-phishing/">solutions</a> for office 365 &amp; G suite analyzes all historical emails in order to determine prior trust relations between the sender and receiver, increasing the likelihood of identifying user impersonation or fraudulent messages. Artificial Intelligence (AI) and Indicators of Compromise (IoCs) used in the past train the <a href="https://www.checkpoint.com/harmony/email-security/email-office/">Harmony Email &amp; Office</a> platform for what to look for in complex zero-day phishing attacks.</p>

function calcScreenW(){
	var screenC = false;
	curentW= jQuery(window).width();	
	if ( curentW > 1450 ) { 
		if (screenW != 'desktop'){
			screenW = 'desktop';

		}
	} else if (curentW > 973){
		if (screenW != 'laptop'){
			screenW = 'laptop';	
			screenC = true;
		}		
	} else if (curentW >767){
		if (screenW != 'tablet'){

			screenC = true;
		}		
	} else {
		if (screenW != 'mobile'){
			screenW = 'mobile';

		}		
	}
	if (screenC) {
		jQuery('.bg-change-done').removeClass('bg-change-done');
		screenC = false;
		animate_elems();

function animate_elems() {
	wintop = jQuery(window).scrollTop(); // calculate distance from top of window
	// loop through each item to check when it animates
	$elems.each(function(){
		$elm = jQuery(this);
		if($elm.hasClass('lazyload')|| $elm.hasClass('bg-change-done')) { return true; } // if already animated skip to the next item

		topcoords = $elm.offset().top; // element's distance from top of page in pixels

		if(wintop > (topcoords - (winheight))) {			
			$elm.addClass('bg-change-done');		
			loadImg($elm[0]);
		} 
	});
