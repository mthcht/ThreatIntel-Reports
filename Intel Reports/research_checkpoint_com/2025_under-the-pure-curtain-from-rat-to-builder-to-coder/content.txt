<h2 class="wp-block-heading" id="key-points">Key Points</h2>



<ul class="wp-block-list">
<li>Check Point Research conducted a forensic analysis of a&nbsp;<strong>ClickFix</strong>&nbsp;campaign that lured victims with fake job offers that resulted in an&nbsp;<strong>eight-day intrusion.</strong>&nbsp;The threat actor deployed multiple tools, including a&nbsp;<strong>Rust Loader</strong>,&nbsp;<strong>PureHVNC RAT</strong>, and the&nbsp;<strong>Sliver</strong>&nbsp;command-and-control framework.</li>



<li>In this publication, we analyzed the associated files, providing one of the most comprehensive analyses of&nbsp;<strong>PureHVNC RAT</strong>, including its complete set of commands and plugins.</li>



<li>During communication with the command and control server, the bot received three&nbsp;<strong>GitHub</strong>&nbsp;URLs containing supporting files for specific PureHVNC functionalities. Analysis confirmed that both the URLs and the associated GitHub accounts were directly linked to the&nbsp;<strong>developer</strong>&nbsp;of the Pure malware family.</li>



<li>Where previously little to no information was known about&nbsp;<em><strong>PureCoder</strong></em>, this publication sheds light on the developer’s&nbsp;<strong>timezone of operation (UTC+0300)</strong>&nbsp;and potential countries of residence. This lead may enable further intelligence gathering by relevant agencies.</li>



<li>Further investigation led to the discovery of a&nbsp;<strong>PureRAT builder</strong>, revealing insights into the RAT’s capabilities and highlighting features linked to&nbsp;<strong>PureCrypter</strong>, another tool developed by&nbsp;<em><strong>PureCoder</strong></em>, the author behind the Pure malware suite.</li>
</ul>



<p><br></p>



<h2 class="wp-block-heading" id="introduction">Introduction</h2>



<p>The&nbsp;<strong>Pure malware family</strong>&nbsp;is a suite of malicious tools developed and sold by the author known as&nbsp;<em><strong>PureCoder</strong></em>. This suite includes&nbsp;<strong>PureHVNC RAT</strong>&nbsp;(a remote administration tool and predecessor to&nbsp;<strong>PureRAT</strong>),&nbsp;<strong>PureCrypter</strong>&nbsp;(a malware obfuscator),&nbsp;<strong>PureLogs</strong>&nbsp;(a stealer/logger), and several other tools. The malicious software is advertised and distributed through underground forums, Telegram channels, and dedicated websites. These products are often combined to maximize their effectiveness across a wide range of malicious operations.</p>



<p>While&nbsp;<strong>PureCoder</strong>&nbsp;is responsible for building and maintaining the malware ecosystem, cybercriminal customers primarily use these tools to conduct campaigns. In 2025, there has been a noticeable increase in the use of Pure malware products, with threat actors distributing them through various methods, including malspam, phishing websites, and the ClickFix technique.</p>



<p>During a Check Point Incident Response (IR) engagement, our team investigated and contained an eight-day intrusion. The threat actor distributed a Rust loader, which deployed&nbsp;<strong>PureHVNC RAT</strong>&nbsp;with campaign IDs&nbsp;<code>2a</code>&nbsp;and&nbsp;<code>amazon3</code>. The attacker lured the victim through fake job advertisements, allowing the attacker to execute malicious PowerShell code through the ClickFix phishing technique.</p>



<p>The RAT’s Command and Control Server (C&amp;C) has been observed to deliver three GitHub URLs to infected victims and download the related files. These downloaded files support various commands used by&nbsp;<strong>PureHVNC RAT</strong>&nbsp;and were determined to be part of the Pure development and operation infrastructure rather than the threat actor itself. As a result, the GitHub accounts involved were attributed to the developer of Pure malware families,&nbsp;<strong>PureCoder</strong>.</p>



<p>While limited information is currently available about the author of these products, analysis of the associated GitHub accounts revealed a timezone of operation set to&nbsp;<code>UTC+0300</code>, which corresponds to several countries, including Russia.</p>



<p><br></p>



<h2 class="wp-block-heading" id="clickfix-campaign-forensics-artifacts">ClickFix Campaign &amp; Forensics Artifacts</h2>



<p><strong>ClickFix</strong>&nbsp;is a social engineering phishing technique in which victims are presented with deceptive instructions designed to trick them into running a malicious command. In this campaign, the victim was lured to the ClickFix phishing page through fake job offers. Upon visiting the page, a&nbsp;<strong>PowerShell</strong>&nbsp;command was automatically copied to their clipboard, delivering a malicious JavaScript file.</p>



<p>During the eight-day intrusion, the attacker used&nbsp;<strong>malicious JavaScript files</strong>, deployed&nbsp;<strong>two instances of PureHVNC RAT</strong>, established persistence on the victim’s system, and finally executed the&nbsp;<strong>Sliver</strong>&nbsp;Command and Control (C2) framework.</p>

<p>If any of those anti-analysis techniques detects that the malware is being analyzed and monitored under an emulated or sandbox environment, it will sleep for a random time between 10 to 30 minutes, by executing the command&nbsp;<code>cmd /c timeout /t {random_time} &gt;null</code>, once awake, if still running in the monitored environment, it reruns the anti-analysis process, and if it is again detected, it falls into a random sleep again.</p>



<p>The Rust Loader requires that it be executed with a specific command line parameter that, if it is not present, terminates its execution. The malware retrieves the parameters passed to the process and tries to find the parameter&nbsp;<code>/i:--type=renderer</code>.</p>



<p>As a final anti-analysis and evasion technique, immediately following the successful decryption of its payload, the malware actively implements an AMSI bypass by injecting a hook into the native&nbsp;<code>LdrLoadDll</code>&nbsp;function within&nbsp;<code>ntdll.dll</code>. This hook intercepts attempts to load the&nbsp;<code>amsi.dll</code>&nbsp;library—responsible for the Windows Anti-malware Scan Interface—and effectively prevents it from being loaded into the process. By doing so, the malware disables AMSI’s runtime scanning capabilities, thereby evading detection and analysis by security products that rely on AMSI for real-time malware inspection.</p>

<h3 class="wp-block-heading" id="persistence">Persistence</h3>



<p>The malware checks if the process is not running with administrative privileges, then executes the PowerShell command below to grant the malware higher privileges.</p>



<pre class="EnlighterJSRAW" data-enlighter-language="generic" data-enlighter-theme="" data-enlighter-highlight="" data-enlighter-linenumbers="" data-enlighter-lineoffset="" data-enlighter-title="" data-enlighter-group="">"powershell" -Command "
while ($true) {
    try {
        Start-Process -FilePath 'cmd.exe' `
            -Verb runas `
            -ArgumentList '/c start "" /B "regsvr32.exe" {MALWARE} /i:--type=renderer';
        exit
    } catch {}
}
"</pre>



<p>This PowerShell will try to execute the malicious DLL with administrative privileges in an infinite loop until the User accepts the UAC (User Account Control) prompt. Once the prompt is accepted, the process terminates itself, and the infection continues with the one that obtained higher privileges.</p>



<p>The loader, to avoid being executed twice, creates a Mutex&nbsp;<code>MistyRoseNavy</code>&nbsp;and then executes a PowerShell command, which will maintain persistence to the system via scheduled tasks.</p>



<p>The first PowerShell command will try to see if any Scheduled Task is already registered in the system.</p>



<pre class="EnlighterJSRAW" data-enlighter-language="generic" data-enlighter-theme="" data-enlighter-highlight="" data-enlighter-linenumbers="" data-enlighter-lineoffset="" data-enlighter-title="" data-enlighter-group="">"powershell" -Command "
if (
    Get-ScheduledTask | Where-Object {
        $_.Actions.Execute -eq 'regsvr32' -and
        $_.Actions.Arguments -eq '/s /i:--type=renderer \"%APPDATA%\Microsoft\SystemCertificates\{MALWARE}\"'
    }
) {
    exit 0
} else {
    exit 1
}
"</pre>



<p>If not, it will proceed to create persistence on the infected machine. The Task name and path were made to mimic a Google Updater:</p>



<pre class="EnlighterJSRAW" data-enlighter-language="generic" data-enlighter-theme="" data-enlighter-highlight="" data-enlighter-linenumbers="" data-enlighter-lineoffset="" data-enlighter-title="" data-enlighter-group="">Register-ScheduledTask `
  -Action (
    New-ScheduledTaskAction `
      -Execute "regsvr32" `
      -Argument "/s /i:--type=renderer "%APPDATA%\Microsoft\SystemCertificates\{MALWARE}""
  ) `
  -Trigger (
    New-ScheduledTaskTrigger `
      -Once -At (Get-Date).AddMinutes(1) `
      -RepetitionInterval (New-TimeSpan -Minutes 1)
  ) `
  -TaskName 'GoogleUpdaterTaskSystem196.6.2928.90.{FD10B0DF-9A2C-41C2-B9E7-3C3C6F193A83}' `
  -TaskPath '\GoogleSystem\GoogleUpdater' `
  -Description 'GoogleUpdater Task System 196.6.2928.90' `
  -Settings (
    New-ScheduledTaskSettingsSet `
      -AllowStartIfOnBatteries `
      -DontStopIfGoingOnBatteries `
      -ExecutionTimeLimit 0 `
      -DontStopOnIdleEnd
  ) `
  -RunLevel Highest</pre>



<p>The execution of the above script will be performed by running it as a&nbsp;<strong>PowerShell over stdin</strong>, executing first the below command and writing into a Pipe&nbsp;<code>WriteFileEx</code>&nbsp;the persistence PowerShell script.</p>



<pre class="EnlighterJSRAW" data-enlighter-language="generic" data-enlighter-theme="" data-enlighter-highlight="" data-enlighter-linenumbers="" data-enlighter-lineoffset="" data-enlighter-title="" data-enlighter-group="">PowerShell.exe -NoProfile -NonInteractive -Command -</pre>



<h3 class="wp-block-heading" id="payload-decryption-execution">Payload Decryption &amp; Execution</h3>



<p>The payload is embedded inside the Rust binary and is decrypted using a simple XOR with a 32-byte key.</p>

<p>Later, the malware will validate the decrypted value, which will determine the next step. The validation occurs by XORing the decrypted bytes and generating a hash value.</p>

<p>This hash value is further processed and compared with the value 0x6. During this function, we also observe the hexadecimal value&nbsp;<code>0xDEADBEEF</code>&nbsp;even though it does not affect the output, which appears to be used solely as a marker.</p>

<h2 class="wp-block-heading" id="purehvnc-rat---technical-analysis">PureHVNC RAT &#8211; Technical Analysis</h2>



<p><strong>PureHVNC</strong>&nbsp;is a product of the Pure family of malicious software developed by&nbsp;<strong>PureCoder</strong>. The malware provides HVNC capabilities (Hidden Virtual Network Computing), which allows an attacker to control an infected machine without the session being visible to the infected user.</p>



<p>The analyzed malware contains obfuscated strings and function calls that are dynamically decrypted. By using&nbsp;<a href="https://github.com/SychicBoy/NETReactorSlayer">NETReactorSlayer</a>, we can obtain a cleaner version of this malware.</p>

<p>The malware initially tries to check if the endpoint is currently alive, first trying to&nbsp;<code>Connect</code>&nbsp;via socket and then sends four bytes to the C&amp;C&nbsp;<code>04 00 00 00</code>.</p>

<figure class="aligncenter size-full is-resized"><img loading="lazy" decoding="async" width="767" height="369" src="https://research.checkpoint.com/wp-content/uploads/2025/09/Screenshot_2025-07-04_162711.png" alt="" class="wp-image-31870" style="aspect-ratio:2.0786383712162015;width:515px;height:auto" srcset="https://research.checkpoint.com/wp-content/uploads/2025/09/Screenshot_2025-07-04_162711.png 767w, https://research.checkpoint.com/wp-content/uploads/2025/09/Screenshot_2025-07-04_162711-300x144.png 300w" sizes="auto, (max-width: 767px) 100vw, 767px" /><figcaption class="wp-element-caption">Figure 13 — Initial communication.</figcaption></figure></div>


<p>The malware creates an SSLStream and performs an SSL handshake, verifying the server’s certificate against a certificate embedded in its configuration.</p>



<p>During this stage,&nbsp;<strong>PureHVNC</strong>&nbsp;collects Bot information that will be sent to the attacker’s server. This information will be serialized and then compressed using&nbsp;<strong>Gzip</strong>. If the compressed data exceeds approximately 1 MB (1,000,000 bytes), it will be sent in 16 KB chunks. </p>

<h3 class="wp-block-heading" id="collected-data">Collected Data</h3>



<p><strong>PureHVNC</strong>&nbsp;collects and sends data from the infected system to the C&amp;C server. This includes installed antivirus products, bot identifier, user domain and privileges, OS version, executable filename, idle time, and installed applications of interest. The information also contains metadata such as the malware version (<code>4.1.9</code>) and campaign ID (<code>amazon3</code>) obtained from the malware configuration.</p>


<div class="wp-block-image">
<figure class="aligncenter size-large is-resized"><img loading="lazy" decoding="async" width="1024" height="438" src="https://research.checkpoint.com/wp-content/uploads/2025/09/Screenshot_2025-07-04_163753-1024x438.png" alt="" class="wp-image-31873" style="aspect-ratio:2.3380012177089675;width:692px;height:auto" srcset="https://research.checkpoint.com/wp-content/uploads/2025/09/Screenshot_2025-07-04_163753-1024x438.png 1024w, https://research.checkpoint.com/wp-content/uploads/2025/09/Screenshot_2025-07-04_163753-300x128.png 300w, https://research.checkpoint.com/wp-content/uploads/2025/09/Screenshot_2025-07-04_163753-768x328.png 768w, https://research.checkpoint.com/wp-content/uploads/2025/09/Screenshot_2025-07-04_163753.png 1275w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /><figcaption class="wp-element-caption">Figure 16 — Data sent to C&amp;C.</figcaption></figure></div>


<p>To retrieve the Antivirus products queries Windows Management Instrumentation (WMI) for installed antivirus products on the system,&nbsp;<code>"SELECT * FROM AntiVirusProduct”</code>.</p>



<p>In addition to collecting information about installed antivirus products, the malware also checks for the presence of specific applications of interest on the host system. It scans predefined directories, files, and registry keys associated with known crypto wallet software, browser extensions, and messaging or email clients to find them (see&nbsp;<strong>Appendix</strong>).</p>



<p>Among the data sent to the C&amp;C, there are three pieces of information that can also serve as anti-sandbox indicators, helping the attacker determine whether the bot resides on a real user system or within a sandboxed or virtualized analysis environment.</p>



<p>The first technique returns a Boolean value indicating whether the infected machine has any devices recognized as cameras or imaging devices. This is determined by executing the WMI query,&nbsp;<code>SELECT * FROM Win32_PnPEntity WHERE (PNPClass = 'Image' OR PNPClass = 'Camera')</code>. Sandboxes and virtual machines often lack physical devices like webcams or imaging devices, and the absence of such devices can indicate a non-physical environment.</p>



<p>The second collected data retrieves the tick count of the last input event (keyboard/mouse) and the last user activity on the machine. The third one is the executable name and path, that during this campaign the malware was installed in a fixed folder. This three information combined could be used by the attacker to determine whether the infected machine is a real user or not.</p>



<h3 class="wp-block-heading" id="plugins-commands">Plugins &amp; Commands</h3>



<p><strong>PureHVNC</strong>&nbsp;stores plugins inside the registry key. The registry key uses the BotID, while the name and value are received by the C&amp;C. The data of the plugin is compressed and reversed, and each time the plugin is loaded, it reverses the bytes and decompresses the buffer. The registry path is located at&nbsp;<code>HKEY_CURRENT_USER\Software\{BOT_ID}</code>&nbsp;and the Bot ID is generated with the following Python logic:</p>



<pre class="EnlighterJSRAW" data-enlighter-language="generic" data-enlighter-theme="" data-enlighter-highlight="" data-enlighter-linenumbers="" data-enlighter-lineoffset="" data-enlighter-title="" data-enlighter-group=""># PureHVNC BotID generation logic
import hashlib

class PureBotID:
    def __init__(self, user_name: str, domain_name: str, processor_id: str, drive_sn: str, memory_sn: str):
        # Environment.UserName
        self.user_name = user_name
        # Environment.UserDomainName
        self.domain_name = domain_name
        # "Win32_Processor", "ProcessorId"
        self.processor_id = processor_id
        # "Win32_DiskDrive", "SerialNumber"
        self.drive_sn = drive_sn
        # "Win32_PhysicalMemory", "SerialNumber"
        self.memory_sn = memory_sn
        # generating Bot ID PureHVNC logic
        self.bot_id = self.generate_bot_id()

    def generate_bot_id(self) -> str:
        user_domain = f"{self.user_name}[{self.domain_name}]" if self.domain_name else self.user_name
        result = f"{self.processor_id}{self.drive_sn}{self.memory_sn}{self.domain_name}{user_domain}"
        return hashlib.md5(result.encode()).hexdigest().upper()</pre>

<h2 class="wp-block-heading" id="conclusion">Conclusion</h2>



<p>The forensic investigation of the ClickFix campaign provides a comprehensive view of the&nbsp;<strong>Pure malware ecosystem</strong>&nbsp;and its operational mechanics. The eight-day intrusion highlighted the coordinated use of multiple tools, including&nbsp;<strong>Rust Loader</strong>,&nbsp;<strong>PureHVNC RAT</strong>, and the&nbsp;<strong>Sliver C2 framework</strong>, demonstrating the threat actor’s sophistication. Check Point Research’s in-depth analysis of&nbsp;<strong>PureHVNC RAT,</strong>&nbsp;including all its commands, plugins, and associated supporting files, sheds light on previously opaque aspects of this malware family.</p>



<p>Significantly, the investigation linked several GitHub repositories directly to the developer,&nbsp;<em><strong>PureCoder</strong></em>, offering rare insights into their operational practices, including a&nbsp;<strong>UTC+0300 timezone</strong>&nbsp;and potential geographic locations. The discovery of the&nbsp;<strong>PureRAT builder</strong>&nbsp;and additional information on&nbsp;<strong>PureCrypter</strong>&nbsp;further illustrate the modular and evolving nature of this malware suite, emphasizing the ongoing threat posed by Pure malware to organizations globally.</p>



<p>Overall, this research not only enhances understanding of the Pure malware family but also provides actionable intelligence that can assist cybersecurity professionals and law enforcement agencies in tracking and mitigating future campaigns conducted by both&nbsp;<strong>PureCoder</strong>&nbsp;and the cybercriminals leveraging their tools.</p>

function loadImg(element){
    var bg = '';
	switch (screenW) {
		case  'mobile':
			bg = element.getAttribute('data-bg-mobile');
			if(bg)
				break; 
		case 'tablet':
			bg = element.getAttribute('data-bg-tablet');
			if(bg)
				break; 
		case 'laptop':
			bg = element.getAttribute('data-bg-laptop');
			if(bg)
				break; 
		default: 
			bg = element.getAttribute('data-bg');
	}
	if(bg)
		element.style.backgroundImage = 'url(' + bg + ')';	
}

function calcScreenW(){
	var screenC = false;
	curentW= jQuery(window).width();	
	if ( curentW > 1450 ) { 
		if (screenW != 'desktop'){
			screenW = 'desktop';	
			screenC = true;
		}
	} else if (curentW > 973){
		if (screenW != 'laptop'){
			screenW = 'laptop';	
			screenC = true;
		}		
	} else if (curentW >767){
		if (screenW != 'tablet'){
			screenW = 'tablet';	
			screenC = true;
		}		
	} else {
		if (screenW != 'mobile'){
			screenW = 'mobile';	
			screenC = true;
		}		
	}
	if (screenC) {
		jQuery('.bg-change-done').removeClass('bg-change-done');
		screenC = false;
		animate_elems();
	}
}

var $elems = jQuery('.responsive-bg');

	animate_elems();

function animate_elems() {
	wintop = jQuery(window).scrollTop(); // calculate distance from top of window
	// loop through each item to check when it animates
	$elems.each(function(){
		$elm = jQuery(this);
		if($elm.hasClass('lazyload')|| $elm.hasClass('bg-change-done')) { return true; } // if already animated skip to the next item

		topcoords = $elm.offset().top; // element's distance from top of page in pixels

		if(wintop > (topcoords - (winheight))) {			
			$elm.addClass('bg-change-done');		
			loadImg($elm[0]);
		} 
	});	
}
