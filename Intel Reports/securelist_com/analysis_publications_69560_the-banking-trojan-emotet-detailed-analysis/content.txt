													<h2 id="introduction">Introduction</h2>
<p>In the summer of 2014, the company Trend Micro <a href="https://blog.trendmicro.com/trendlabs-security-intelligence/new-banking-malware-uses-network-sniffing-for-data-theft/">announced</a> the detection of a new threat &#8211; the  banking Trojan Emotet.  The description  indicated that the malware could  steal bank account details by intercepting traffic.  We call this modification version 1. </p>
<p>In the autumn of that year a new version  of Emotet was found.  It caught our attention for the following reasons: </p>
<ul>
<li>The developers of this Trojan had begun  to use technology that stole money automatically from victims&#8217; bank accounts &#8211; so called &#8220;Automatic  Transfer System (ATS)&#8221;. </li>
<li>The Trojan had a modular  structure: it contained its own installation module, a banking module, a spam  bot module, a module for stealing address books from MS Outlook and a module  for organizing DDoS attacks (Nitol DDoS bot).</li>
<li>The creators made a significant  effort to remain unnoticed: they didn&#8217;t attack users in the RU zone but targeted the clients of a small number of German and Austrian  banks (other well-known banking Trojans are less discerning in their choice of  target),and the domain name of the ATS server  changed frequently (once or several times a day).</li>
</ul>
<p>We are going to refer to this  modification as Emotet version 2. The bot contains and transfers the numbers one and seven to the command and control center  (C&amp;C), which suggests that the Trojan&#8217;s authors considers this variant to be version  1.7. </p>
<p>Both versions of the Trojan attacked  clients of German and Austrian banks.</p>
<blockquote class=""><p>#Trojan #Emotet targeted the clients of a small number of German, Austrian and Swiss banks</p><a href="https://twitter.com/share?url=https%3A%2F%2Fsecurelist.com%2Fthe-banking-trojan-emotet-detailed-analysis%2F69560%2F&text=%23Trojan+%23Emotet+targeted+the+clients+of+a+small+number+of+German%2C+Austrian+and+Swiss+banks" class="btn btn-twhite" data-lang="en" data-count="0">Tweet</a></blockquote>
<p>We closely monitored Emotet version 2.  In December 2014 it ceased activity and the command servers stopped responding  to infected computers.  We recorded the  last command sent from the command centers on 10/12/2014, at 11:33:43 Moscow time.</p>
<p>However, the thoroughness with which the authors had approached the development  of this Trojan and the high level of automation in its operation, left little doubt that this was not the end of the  story.  And so it turned out &#8211; after a  short break in January 2015, Emotet <a href="https://www.securityweek.com/new-emotet-variant-targets-banking-credentials-german-speakers">reappeared</a>!   We are calling this modification version 3 (the bot contains and transfers the  numbers one and 16 to the C&amp;C, which we assume  means that the authors consider this variant to be  version 1.16).</p>
<p>In essence, Emotet version 3 is not that different to version 2 &#8211; the main differences are designed to  make the Trojan less visible. Of the changes we noted, we would like to highlight the  following:</p>
<ul>
<li>The Trojan has a new built-in  public RSA key and, although the communication protocols with the command center  are identical for Emotet versions 2 and 3, if the old key is used the bot does  not receive the correct answer from the command center.</li>
<li>The ATS scripts are partially  cleaned of debugging information and comments.</li>
<li><strong>New  targets! </strong>Emotet is  now also targeting clients of Swiss banks. </li>
<li>There has been a slight change in  the technology used to inject code into the address space of  explorer.exe.  Version 2 used a classic model for code injection:  OpenProcess+WriteProcessMemeory+CreateRemoteThread. Version 3 uses only two stages of the previous  model:OpenProcess+WriteProcessMemory;  and the injected code is initiated with the help of modified code of the  ZwClose function in the address space of the explorer.exe process, which is  also achieved using WriteProcessMemory.</li>
<li>Emotet version 3 resists investigation: if the Trojan  detects that it has been started in a virtual machine it functions as usual but  uses a different address list for the command centers.  However, all these addresses are false and are  used only to mislead investigators. </li>
<li>The Trojan contains very few lines of text:   all lines that could warn investigators are encrypted using RC4 and are  decrypted in allocated memory directly before use and deleted after use.</li>
</ul>
<p>On the whole, we formed the impression that the  main techniques used in version 3  of the banking Trojan were developed  &#8220;in the field&#8221; using version 2 as a basis, and with the addition of improved stealth techniques.</p>
<p>Kaspersky Lab products detect all versions of this Trojan as Trojan-Banker.Win32.Emotet.  We also detect the following  modulesof Emotet:</p>
<ul>
<li>Module for modifying HTTP(S)  traffic &#8211; Trojan-Banker.Win32.Emotet. </li>
<li>Spam module &#8211;  Trojan.Win32.Emospam. </li>
<li>Module for the collection of email  addresses &#8211; Trojan.Win32.Emograbber. </li>
<li>Module for stealing email account  data &#8211; Trojan-PSW.Win32.Emostealer.</li>
<li>Module designed for organising  DDoS attacks — Trojan.Win32.ServStart. </li>
</ul>
<p>We have seen the last module used with  other malware and assume that it was added to  Emotet by a cryptor.  It is quite  possible that Emotet&#8217;s authors are totally unaware of the presence of this  module in their malware.  Whatever the  case may be, the command centers of this  module do not respond and the module has not been updated (its compilation date is 19 October 2014).</p>
<h2 id="infection">Infection</h2>
<p>We currently know of  only one method of distribution for the Emotet banking Trojan: distribution of spam mailings that  include malicious attachments or links.</p>
<p>The attached files are usually ZIP archives containing the Emotet  loader.  The files in the archives have  long names, e.g.  rechnung_november_2014_11_0029302375471_03_44_0039938289.exe.  This is done on purpose: a user opening the  archive in a standard Windows panel might not see the extension .exe, as the  end of the file name might not be displayed.   Sometimes there is no attachment and the text in the main body of the email contains a link to a malicious  executable file or archive.</p>

<p>When the email contains a link to malware, it downloads it from the addresses of  compromised legitimate sites:</p>
<p>hxxp://*******/82nBRaLiv (for version 2)<br />
  or from the addresses<br />
  hxxp://*******/dhl_paket_de_DE and hxxp://*******/dhl_paket_de_DE (for  version 3).</p>
<p>In Emotet version 3, when addresses are contacted with the form hxxp://*/dhl_paket_de_DE, the user receives a ZIP archive of the  following form hxxp://*/dhl_paket_de_DE_26401756290104624513.zip.</p>
<p>The archive contains an exe-file with a  long name (to hide the extension) and a PDF document icon.</p>
<h2 id="loading-the-trojan">Loading  the Trojan</h2>
<p>The Trojan file is packed by a cryptor,  the main purpose of which is to avoid detection by anti-virus  programs.  After being started and processed by the cryptor, control is passed to the  main Emotet module &#8211; the loader.  This has to embed itself in the system, link  with the command server, download additional modules and then run them. </p>
<p>Consolidation in the system is fairly standard — Emotet version 2 saves itself in %APPDATA%\Identities  with a random name of eight characters (for example — wlyqvago.exe); adds itself to the autoloader  (HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run) and  then deletes its source file with the help of a launched bat-file that is  created in %APPDATA% with the name &#8220;ms[7_random_numbers].bat.</p>
<p>Emotet version 3 saves itself in %APPDATA%\Microsoft\  with a name in the format msdb%x.exe&#8221; (for example – C:\Documents and  Settings\Administrator\Application Data\Microsoft\msdbfe1b033.exe); adds itself to the autoloader  (HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run) and then deletes itself with the help of a  launched bat-file (which is created in %APPDATA%\del%x.bat).</p>
<p>After consolidating itself in the system, Emotet obtains a list of the names of  all processes running and calculates a hash from the name of every function,  comparing the resulting value with the hardcoded  0xB316A779 (this hash corresponds to the process  explorer.exe).   In this way, Emotet locates the process into which to inject itself.  Further, the Trojan unpacks its main code and  injects it into the process explorer.exe.</p>
<h2 id="communication-with-the-command-center">Communication  with the command center</h2>
<p>The main module of the Trojan, the  loader, communicates with the C&amp;C using RC4 encryption.</p>
<p>The port used by the loader is hardcoded  into it &#8211; 8080. </p>

This is probably done to confuse any  investigators and give them the impression that the Trojan command centers are  dead.  A similar approach was used previously in the high-profile banking Trojan, Citadel.</p>

<p>The detection of a virtual machine is  organized quite simply — by the names of processes that are usual for various  virtual machines.  The following  algorithm is used to calculate a hash value from the name of every process in the system:</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062424/banker_emotet_pic05.png" class="magnificImage"><img loading="lazy" decoding="async" id="69523" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062424/banker_emotet_pic05.png" alt="banker_emotet_pic05" width="668" height="220" class="aligncenter size-full wp-image-69523" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062424/banker_emotet_pic05.png 668w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062424/banker_emotet_pic05-300x99.png 300w" sizes="auto, (max-width: 668px) 100vw, 668px" /></a></p>
<p style="text-align:center;font-weight:bold"><em>Algorithm  for calculation of a hash value from a process name</em></p>
<p>The resulting hash value is then  compared with a list of values hardcoded into the Trojan:</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062421/banker_emotet_pic06.png" class="magnificImage"><img loading="lazy" decoding="async" id="69524" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062421/banker_emotet_pic06.png" alt="banker_emotet_pic06" width="187" height="362" class="aligncenter size-full wp-image-69524" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062421/banker_emotet_pic06.png 187w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062421/banker_emotet_pic06-155x300.png 155w" sizes="auto, (max-width: 187px) 100vw, 187px" /></a></p>
<p style="text-align:center;font-weight:bold"><em>Hashes  from the names of processes used for the detection of virtual machines</em></p>
<p>We derived the names of the processes  for several hashes. For example, hash 0xBCF398B5 corresponds to the  process vboxservice.exe, hash 0x2C967737 to the process vmacthlp.exe, hash  0xE3EBFE44 to the process vmtoolsd.exe, and 0x61F15513 to the process vboxtray.exe.</p>

<p>A request to the command center appears  in the traffic as follows (the example given is from  version 2, but a version 3 request looks the same):</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062416/banker_emotet_pic07.png" class="magnificImage"><img loading="lazy" decoding="async" id="69525" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062416/banker_emotet_pic07.png" alt="banker_emotet_pic07" width="766" height="589" class="aligncenter size-full wp-image-69525" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062416/banker_emotet_pic07.png 766w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062416/banker_emotet_pic07-300x231.png 300w" sizes="auto, (max-width: 766px) 100vw, 766px" /></a></p>
<p style="text-align:center;font-weight:bold"><em>Dialogue  between the Emotet bot and its command center</em></p>
<p>The URL-path that the bot communicates  with appears as follows: /722ffc5e/355c7a0a/, where 722ffc5e is a number  calculated on the basis of information from the access marker of the user, and   0x355c7a0a = 0x722ffc5e xor 0x47738654 (the value 0x47738654 is  hardcoded into the bot).</p>
<p>The data sent by the bot and the command  center are encrypted using RC4 and the answers received from the command center  are signed with a digital signature.   Probably this is done to make it difficult to seize control over the  botnet: in order for the bot to accept a packet it must be signed and for that  it is necessary to know the secret key.</p>
<p>There is a public RSA key in the body of  the bot. In PEM format for version 2 it  appears as follows:</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062412/banker_emotet_pic08.png" class="magnificImage"><img loading="lazy" decoding="async" id="69526" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062412/banker_emotet_pic08.png" alt="banker_emotet_pic08" width="608" height="152" class="aligncenter size-full wp-image-69526" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062412/banker_emotet_pic08.png 608w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062412/banker_emotet_pic08-300x75.png 300w" sizes="auto, (max-width: 608px) 100vw, 608px" /></a></p>
<p style="text-align:center;font-weight:bold"><em>PEM  representation of the open RSA key coded into the bot in version 2</em></p>
<p>As noted above, in version 3 the key changed.  In PEM format it looks like this:</p>

<p style="text-align:center;font-weight:bold"><em>PEM  representation of the open RSA key coded into the bot in version 3</em></p>
<p>A packet sent to the server is made up  as follows: </p>
<ul>
<li>A request is generated containing  the identifier of the infected computer, a value presumably indicating the  version of the bot; information about the system (OS  version, service pack version, product type); a hardcoded dword (value in the  investigated sample — seven); control sums for the banker module; and information about the  web-injects.  Information about the  web-injects contains: a page address (with jokers), into which the injection is needed; data  coming before the injected data; data coming after injected data; and injected data.</li>
<li>An SHA1 hash is calculated from  the generated request.</li>
<li>The request is encrypted with a  randomly generated 128 bit RC4 key. </li>
<li>The generated RC4 key is encrypted  using the public RSA key.</li>
<li>The total packet is the  concatenation of the results obtained at steps 4, 2 and 3.</li>
</ul>
<p>The request packet can be represented by  the following diagram:</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062404/banker_emotet_pic10.png" class="magnificImage"><img loading="lazy" decoding="async" id="69528" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062404/banker_emotet_pic10.png" alt="banker_emotet_pic10" width="678" height="72" class="aligncenter size-full wp-image-69528" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062404/banker_emotet_pic10.png 678w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062404/banker_emotet_pic10-300x32.png 300w" sizes="auto, (max-width: 678px) 100vw, 678px" /></a></p>

<h2 id="getting-round-two-factor-authentication">Getting round two-factor authentication</h2>
<p>The main  purpose of the scripts looked at above is to carry out the illicit transfer of  money from the user&#8217;s account.  However  the bot cannot independently get round the system of two-factor authentication (Chip TAN or SMS  TAN), it needs the user&#8217;s help.  To  mislead the potential victim, social engineering techniques are  used: the message injected into the webpage using the script informs the user  that the site is introducing a new security system and normal operations cannot  be continued until the user has tested it in the demo-regime.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062313/banker_emotet_pic21.png" class="magnificImage"><img loading="lazy" decoding="async" id="69539" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062313/banker_emotet_pic21.png" alt="banker_emotet_pic21" width="1093" height="809" class="aligncenter size-full wp-image-69539" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062313/banker_emotet_pic21.png 1093w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062313/banker_emotet_pic21-300x222.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062313/banker_emotet_pic21-1024x758.png 1024w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062313/banker_emotet_pic21-768x568.png 768w" sizes="auto, (max-width: 1093px) 100vw, 1093px" /></a></p>

<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062309/banker_emotet_pic22.png" class="magnificImage"><img loading="lazy" decoding="async" id="69540" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062309/banker_emotet_pic22.png" alt="banker_emotet_pic22" width="1156" height="155" class="aligncenter size-full wp-image-69540" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062309/banker_emotet_pic22.png 1156w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062309/banker_emotet_pic22-300x40.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062309/banker_emotet_pic22-1024x137.png 1024w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062309/banker_emotet_pic22-768x103.png 768w" sizes="auto, (max-width: 1156px) 100vw, 1156px" /></a></p>
<p>In fact, instead of a test transfer the malicious script  carries out a real transfer of money from the victim&#8217;s account to the account  of a nominated person — the so-called &#8220;drop&#8221;, and the user themselves confirms this transfer using the Chip  TAN or SMS TAN.</p>
<p>Details of  the accounts for the transfer of the stolen money are not initially indicated  in the script, but are received from the command  server of the criminals using a special request.  In reply the command  server returns a line with information about the &#8220;drop&#8221; for each specific  transaction.  In the comments in one  script we found the following line: </p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062304/banker_emotet_pic23.png" class="magnificImage"><img loading="lazy" decoding="async" id="69541" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062304/banker_emotet_pic23.png" alt="banker_emotet_pic23" width="775" height="35" class="aligncenter size-full wp-image-69541" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062304/banker_emotet_pic23.png 775w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062304/banker_emotet_pic23-300x14.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062304/banker_emotet_pic23-768x35.png 768w" sizes="auto, (max-width: 775px) 100vw, 775px" /></a></p>
<p>Clearly  the criminals tested this script with a transfer of 1500.9 EUR to a test account.</p>
<p>In addition, this script contained the following  information about the drop:</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062300/banker_emotet_pic24.png" class="magnificImage"><img loading="lazy" decoding="async" id="69542" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062300/banker_emotet_pic24.png" alt="banker_emotet_pic24" width="806" height="199" class="aligncenter size-full wp-image-69542" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062300/banker_emotet_pic24.png 806w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062300/banker_emotet_pic24-300x74.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062300/banker_emotet_pic24-768x190.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062300/banker_emotet_pic24-800x199.png 800w" sizes="auto, (max-width: 806px) 100vw, 806px" /></a></p>
<p>In the  corresponding script in Emotet version 3, designed to attack the same bank, we also found information on the drop,  but this time another one:</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062257/banker_emotet_pic25.png" class="magnificImage"><img loading="lazy" decoding="async" id="69543" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062257/banker_emotet_pic25.png" alt="banker_emotet_pic25" width="437" height="187" class="aligncenter size-full wp-image-69543" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062257/banker_emotet_pic25.png 437w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062257/banker_emotet_pic25-300x128.png 300w" sizes="auto, (max-width: 437px) 100vw, 437px" /></a></p>
<p>Let&#8217;s  compare the fields JSON __DropParam and the fields in the legitimate form from  a demo-access to the online system of the attacked bank.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062252/banker_emotet_pic26.png" class="magnificImage"><img loading="lazy" decoding="async" id="69544" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062252/banker_emotet_pic26.png" alt="banker_emotet_pic26" width="602" height="569" class="aligncenter size-full wp-image-69544" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062252/banker_emotet_pic26.png 602w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/04/08062252/banker_emotet_pic26-300x284.png 300w" sizes="auto, (max-width: 602px) 100vw, 602px" /></a></p>

<p>The JSON  __DropParam fields correspond to the fields in the form.</p>
<p>In this  way the bot receives all the necessary information about the  drop from its server and draws up a transfer to it, and the misled user confirms the  transfer using the Chip TAN or SMS TAN and waves goodbye to their money.  </p>
<h2 id="conclusion">Conclusion</h2>
<p>The Emotet  Trojan is a highly automated and developing, territorially-targeted bank threat. Its small size, the dispersal methods used  and the modular architecture, all make Emotet a very effective  weapon for the cyber-criminal.</p>
<blockquote class=""><p>The #Emotet #Trojan is a highly automated and developing, territorially-targeted bank threat</p><a href="https://twitter.com/share?url=https%3A%2F%2Fsecurelist.com%2Fthe-banking-trojan-emotet-detailed-analysis%2F69560%2F&text=The+%23Emotet+%23Trojan+is+a+highly++automated+and+developing%2C+territorially-targeted+bank+threat" class="btn btn-twhite" data-lang="en" data-count="0">Tweet</a></blockquote>
<p>However  this banking Trojan doesn&#8217;t incorporate conceptually new technology and so the  use of a modern anti-virus program can provide an effective defense against the threat. </p>
<p>Furthermore,  the Trojan cannot function effectively without the participation of the user —  the Emotet creators have actively used social engineering techniques to achieve  their criminal ends.</p>
<p>And so the  alertness and technical awareness of the user, together with the use of a  modern anti-virus program can provide reliable protection against not only  Emotet but other` new banking threats working in a similar  way. </p>
