												<div class="c-wysiwyg">
													<p>For the Lazarus threat actor, financial gain is one of the prime motivations, with a particular emphasis on the cryptocurrency business. As the price of cryptocurrency surges, and the popularity of non-fungible token (NFT) and decentralized finance (DeFi) businesses continues to swell, the Lazarus group&#8217;s targeting of the financial industry keeps evolving.</p>
<p>We recently discovered a Trojanized DeFi application that was compiled in November 2021. This application contains a legitimate program called DeFi Wallet that saves and manages a cryptocurrency wallet, but also implants a malicious file when executed. This malware is a full-featured backdoor containing sufficient capabilities to control the compromised victim. After looking into the functionalities of this backdoor, we discovered numerous overlaps with other tools used by the Lazarus group.</p>
<p>The malware operator exclusively used compromised web servers located in South Korea for this attack. To take over the servers, we worked closely with the <a href="https://www.krcert.or.kr/main.do">KrCERT</a> and, as a result of this effort, we had an opportunity to investigate a Lazarus group C2 server. The threat actor configured this infrastructure with servers set up as multiple stages. The first stage is the source for the backdoor while the goal of the second stage servers is to communicate with the implants. This is a common scheme used in Lazarus infrastructure.</p>
<h2 id="background">Background</h2>
<p>In the middle of December 2021, we noticed a suspicious file uploaded to VirusTotal. At first glance, it looked like a legitimate application related to decentralized finance (DeFi); however, looking closer we found it initiating an infection scheme. When executed, the app drops both a malicious file and an installer for a legitimate application, launching the malware with the created Trojanized installer path. Then, the spawned malware overwrites the legitimate application with the Trojanized application. Through this process, the Trojanized application gets removed from the disk, allowing it to cover its tracks.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/28150403/Lazarus_Trojanized_DeFi_application_to_deliver_malware_01.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-large wp-image-106201" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/28150403/Lazarus_Trojanized_DeFi_application_to_deliver_malware_01-1024x329.png" alt="Infection timeline" width="1024" height="329" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/28150403/Lazarus_Trojanized_DeFi_application_to_deliver_malware_01-1024x329.png 1024w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/28150403/Lazarus_Trojanized_DeFi_application_to_deliver_malware_01-300x96.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/28150403/Lazarus_Trojanized_DeFi_application_to_deliver_malware_01-768x246.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/28150403/Lazarus_Trojanized_DeFi_application_to_deliver_malware_01-545x175.png 545w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/28150403/Lazarus_Trojanized_DeFi_application_to_deliver_malware_01-370x119.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/28150403/Lazarus_Trojanized_DeFi_application_to_deliver_malware_01-872x280.png 872w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/28150403/Lazarus_Trojanized_DeFi_application_to_deliver_malware_01-800x257.png 800w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/28150403/Lazarus_Trojanized_DeFi_application_to_deliver_malware_01.png 1508w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /></a></p>
<p style="text-align: center"><strong><em>Infection timeline</em></strong></p>
<h2 id="initial-infection">Initial infection</h2>
<p>While it&#8217;s still unclear how the threat actor tricked the victim into executing the Trojanized application (<a href="https://opentip.kaspersky.com/0b9f4612cdfe763b3d8c8a956157474a/?utm_source=SL&amp;utm_medium=SL&amp;utm_campaign=SL" rel="noopener" target="_blank">0b9f4612cdfe763b3d8c8a956157474a</a>), we suspect they sent a spear-phishing email or contacted the victim through social media. The hitherto unknown infection procedure starts with the Trojanized application. This installation package is disguised as a DeFi Wallet program containing a legitimate binary repackaged with the installer.</p>
<p>Upon execution, it acquires the next stage malware path (C:\ProgramData\Microsoft\GoogleChrome.exe) and decrypts it with a one-byte XOR (Key: 0x5D). In the process of creating this next malware stage, the installer writes the first eight bytes including the &#8216;MZ&#8217; header to the file GoogleChrome.exe and pushes the remaining 71,164 bytes from the data section of the Trojanized application. Next, the malware loads the resource CITRIX_MEETINGS from its body and saves it to the path C:\ProgramData\Microsoft\CM202025.exe. The resulting file is a legitimate DeFi Wallet application. Eventually, it executes the previously created malware with its file name as a parameter:</p>
<p>C:\ProgramData\Microsoft\GoogleChrome.exe &#8220;[current file name]&#8221;</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121529/Lazarus_Trojanized_DeFi_application_to_deliver_malware_02.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-large wp-image-106203" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121529/Lazarus_Trojanized_DeFi_application_to_deliver_malware_02-1024x456.png" alt="Malware creation diagram" width="1024" height="456" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121529/Lazarus_Trojanized_DeFi_application_to_deliver_malware_02-1024x456.png 1024w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121529/Lazarus_Trojanized_DeFi_application_to_deliver_malware_02-300x134.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121529/Lazarus_Trojanized_DeFi_application_to_deliver_malware_02-768x342.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121529/Lazarus_Trojanized_DeFi_application_to_deliver_malware_02-393x175.png 393w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121529/Lazarus_Trojanized_DeFi_application_to_deliver_malware_02-370x165.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121529/Lazarus_Trojanized_DeFi_application_to_deliver_malware_02-629x280.png 629w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121529/Lazarus_Trojanized_DeFi_application_to_deliver_malware_02-800x356.png 800w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121529/Lazarus_Trojanized_DeFi_application_to_deliver_malware_02.png 1468w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /></a></p>
<p style="text-align: center"><strong><em>Malware creation diagram</em></strong></p>
<h2 id="backdoor-creation">Backdoor creation</h2>
<p>The malware (<a href="https://opentip.kaspersky.com/d65509f10b432f9bbeacfc39a3506e23/?utm_source=SL&amp;utm_medium=SL&amp;utm_campaign=SL" rel="noopener" target="_blank">d65509f10b432f9bbeacfc39a3506e23</a>) generated by the above Trojanized application is disguised as a benign instance of the Google Chrome browser. Upon launch, the malware checks if it was provided with one argument before attempting to copy the legitimate application &#8220;C:\ProgramData\Microsoft\CM202025.exe&#8221; to the path given as the command line parameter, which means overwriting the original Trojanized installer, almost certainly in an attempt to conceal its prior existence. Next, the malware executes the legitimate file to deceive the victim by showing its benign installation process. When the user executes the newly installed program, it shows the DeFi Wallet software built with the public source code<a href="#_ftn1" name="_ftnref1"><sup>[1]</sup></a>.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121604/Lazarus_Trojanized_DeFi_application_to_deliver_malware_03.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-large wp-image-106204" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121604/Lazarus_Trojanized_DeFi_application_to_deliver_malware_03-1024x646.png" alt="Screenshot of the manipulated application" width="1024" height="646" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121604/Lazarus_Trojanized_DeFi_application_to_deliver_malware_03-1024x646.png 1024w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121604/Lazarus_Trojanized_DeFi_application_to_deliver_malware_03-300x189.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121604/Lazarus_Trojanized_DeFi_application_to_deliver_malware_03-768x484.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121604/Lazarus_Trojanized_DeFi_application_to_deliver_malware_03-277x175.png 277w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121604/Lazarus_Trojanized_DeFi_application_to_deliver_malware_03-370x233.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121604/Lazarus_Trojanized_DeFi_application_to_deliver_malware_03-444x280.png 444w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121604/Lazarus_Trojanized_DeFi_application_to_deliver_malware_03-800x505.png 800w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/03/29121604/Lazarus_Trojanized_DeFi_application_to_deliver_malware_03.png 1316w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /></a></p>

<p>The malware randomly chooses a C2 server address and sends a beacon signal to it. This signal is a hard-coded &#8216;0x60D49D94&#8217; DWORD without encryption; the response data returned from the C2 carries the same value. If the expected value from the C2 server is received, the malware starts its backdoor operation.</p>
<p>Following further communication with the C2, the malware encrypts data by a predefined method. The encryption is done via RC4 and the hard-coded key 0xD5A3 before additionally being encoded with base64.</p>
<p>The malware generates POST parameters with hard-coded names. The request type (msgID), victim identification value, and a randomly generated value are merged into the &#8216;jsessid&#8217; parameter. It also uses the &#8216;cookie&#8217; parameter to store four randomly generated four-byte values. These values are again encrypted with RC4 and additionally base64 encoded. Based on our investigation of the C2 script, we observed this malware not only uses a parameter named &#8216;jsessid&#8217;, but also &#8216;jcookie&#8217; as well.</p>

<h2 id="attribution">Attribution</h2>
<p>We believe with high confidence that the Lazarus group is linked to this malware as we identified similar malware in the CookieTime cluster. The CookieTime cluster, called <a href="https://blogs.jpcert.or.jp/en/2021/01/Lazarus_malware2.html">LCPDot</a> by JPCERT, was a malware cluster that was heavily used by the Lazarus group until recently. We&#8217;ve seen Lazarus group target the defence industry using the CookieTime cluster with a job opportunity decoy. We have already published several reports about this cluster to our Threat Intelligence Service customers, and we identified a Trojanized Citrix application (<a href="https://opentip.kaspersky.com/5b831eaed711d5c4bc19d7e75fcaf46e/?utm_source=SL&amp;utm_medium=SL&amp;utm_campaign=SL" rel="noopener" target="_blank">5b831eaed711d5c4bc19d7e75fcaf46e</a>) with the same code signature as the CookieTime malware. The backdoor discovered in the latest investigation, and the previously discovered Trojanized application, are almost identical. They share, among other things, the same C2 communication method, backdoor functionalities, random number generation routine and the same method to encrypt communication data. Also, this malware was mentioned in an <a href="https://cn.ahnlab.com/global/upload/download/asecreport/ASEC%20REPORT_vol.102_ENG%20(4).pdf">article</a> by Ahnlab discussing connections with the CookieTime (aka LCPDot) malware.</p>

<h2 id="conclusions">Conclusions</h2>
<p>In a previous investigation we discovered that the <a href="https://securelist.com/the-bluenoroff-cryptocurrency-hunt-is-still-on/105488/">BlueNoroff</a> group, which is also linked to Lazarus, compromised another DeFi wallet program called MetaMask. As we can see in the latest case, the Lazarus and BlueNoroff groups attempt to deliver their malware without drawing attention to it and have evolved sophisticated methods to lure their victims. The cryptocurrency and blockchain-based industry continues to grow and attract high levels of investment. For this reason, we strongly believe Lazarus&#8217;s interest in this industry as a major source of financial gain will not diminish any time soon.</p>
