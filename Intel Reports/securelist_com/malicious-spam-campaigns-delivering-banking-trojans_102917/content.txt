													<p>In mid-March 2021, we observed two new spam campaigns. The messages in both cases were written in English and contained ZIP attachments or links to ZIP files. Further research revealed that both campaigns ultimately aimed to distribute banking Trojans. The payload in most cases was IcedID (Trojan-Banker.Win32.IcedID), but we have also seen a few QBot (Backdoor.Win32.Qbot, also known as QakBot) samples. During campaign spikes we observed increased activity of these Trojans: more than a hundred detections a day.</p>
<p>IcedID is a banking Trojan capable of web injects, VM detection and other malicious actions. It consists of two parts – the downloader and the main body that performs all the malicious activity. The main body is hidden in a PNG image, which is downloaded and decrypted by the downloader.</p>
<p>QBot is also a banking Trojan. It&#8217;s a single executable with an embedded DLL (main body) capable of downloading and running additional modules that perform malicious activity: web injects, email collection, password grabbing, etc.</p>
<p>Neither of these malware families are new – we&#8217;ve seen them being distributed before via spam campaigns and different downloaders, like the recently taken-down <a href="https://securelist.com/the-chronicles-of-emotet/99660/" target="_blank" rel="noopener">Emotet</a>. However, in the recent campaign we observed several changes to the IcedID Trojan.</p>
<h2 id="technical-details">Technical details</h2>
<h3 id="initial-infection">Initial infection</h3>
<p><strong>DotDat</strong></p>
<p>The first campaign we called &#8216;DotDat&#8217;. It distributed ZIP attachments that claimed to be some sort of cancelled operation or compensation claims with the names in the following format [document type (optional)]-[some digits]-[date in MMDDYYYY format]. We assume the dates correspond with the campaign spikes. The ZIP archives contained a malicious MS Excel file with the same name.</p>
<p>The Excel file downloads a malicious payload via a macro (see details below) from a URL with the following format [host]/[digits].[digits].dat and executes it. The URL is generated during execution using the Excel function <em>NOW()</em>. The payload is either the IcedID downloader (Trojan.Win32.Ligooc) or QBot packed with a polymorph packer.</p>
<p><strong>Excel macro details (<a href="https://opentip.kaspersky.com/3e12880c20c41085ea5e249f8eb85ded/?utm_source=SL&amp;utm_medium=SL&amp;utm_campaign=SL" target="_blank" rel="noopener">3e12880c20c41085ea5e249f8eb85ded</a>)</strong></p>
<p>The Excel file contains obfuscated Excel 4.0 macro formulas to download and execute the payload (IcedID or QBot). The macro generates a payload URL and calls the WinAPI function <em>URLDownloadToFile </em>to download the payload.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141009/IcedID_and_Qbot_spam_01.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-102920" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141009/IcedID_and_Qbot_spam_01.png" alt="" width="624" height="252" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141009/IcedID_and_Qbot_spam_01.png 624w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141009/IcedID_and_Qbot_spam_01-300x121.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141009/IcedID_and_Qbot_spam_01-433x175.png 433w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141009/IcedID_and_Qbot_spam_01-370x149.png 370w" sizes="auto, (max-width: 624px) 100vw, 624px" /></a></p>
<p style="text-align: center"><strong><em>Macro downloads IcedID downloader</em></strong></p>
<p>After a successful download, the payload is launched using the EXEC function and Windows Rundll32 executable.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141134/IcedID_and_Qbot_spam_02.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-102921" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141134/IcedID_and_Qbot_spam_02.png" alt="" width="411" height="151" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141134/IcedID_and_Qbot_spam_02.png 411w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141134/IcedID_and_Qbot_spam_02-300x110.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141134/IcedID_and_Qbot_spam_02-370x136.png 370w" sizes="auto, (max-width: 411px) 100vw, 411px" /></a></p>
<p style="text-align: center"><strong><em>Macro starts payload</em></strong></p>
<p><strong>Summer.gif</strong></p>
<p>The spam emails of the second campaign contained links to hacked websites with malicious archives named &#8220;documents.zip&#8221;, &#8220;document-XX.zip&#8221;, &#8220;doc-XX.zip&#8221; where XX stands for two random digits. Like in the first campaign, the archives contained an Excel file with a macro that downloaded the IcedID downloader. According to our data, this spam campaign peaked on 17/03/2021. By April, the malicious activity had faded away.</p>
<p><strong>Excel macro details (<a href="https://opentip.kaspersky.com/c11bad6137c9205d8656714d362cc8e4/?utm_source=SL&amp;utm_medium=SL&amp;utm_campaign=SL" target="_blank" rel="noopener">c11bad6137c9205d8656714d362cc8e4</a>)</strong></p>
<p>Like in the other case, Excel 4.0 macro formulas and the <em>URLDownloadToFile</em> function are used in this campaign. The main difference in the download component is that the URL is stored in a cell inside the malicious file.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141207/IcedID_and_Qbot_spam_03.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-102922" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141207/IcedID_and_Qbot_spam_03.png" alt="" width="557" height="389" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141207/IcedID_and_Qbot_spam_03.png 557w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141207/IcedID_and_Qbot_spam_03-300x210.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141207/IcedID_and_Qbot_spam_03-251x175.png 251w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141207/IcedID_and_Qbot_spam_03-370x258.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141207/IcedID_and_Qbot_spam_03-401x280.png 401w" sizes="auto, (max-width: 557px) 100vw, 557px" /></a></p>
<p style="text-align: center"><strong><em>Payload download</em></strong></p>
<p>Though the URL seems to refer to a file named &#8220;summer.gif&#8221;, the payload is an executable, not a GIF image. To execute the payload, the macro uses WMI and regsvr32 tools.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141425/IcedID_and_Qbot_spam_04.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-102923" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141425/IcedID_and_Qbot_spam_04.png" alt="" width="409" height="156" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141425/IcedID_and_Qbot_spam_04.png 409w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141425/IcedID_and_Qbot_spam_04-300x114.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141425/IcedID_and_Qbot_spam_04-370x141.png 370w" sizes="auto, (max-width: 409px) 100vw, 409px" /></a></p>
<p style="text-align: center"><strong><em>Macro starts payload</em></strong></p>
<h3 id="icedid">IcedID</h3>
<p>As we mention above, IcedID consists of two parts – downloader and main body. The downloader sends some user information (username, MAC address, Windows version, etc.) to the C&amp;C and receives the main body. In the past, the main body was distributed as a shellcode hidden in a PNG image. The downloader gets the image, decrypts the main body in the memory and executes it. The main body maps itself into the memory and starts to perform its malicious actions such as web injects, data exfiltration to the C&amp;C, download and execution of additional payloads, exfiltration of system information and more.</p>
<p><strong>IcedID new downloader</strong></p>
<p>Besides the increase in infection attempts, the IcedID authors also changed the downloader a bit. In previous versions it was compiled as an x86 executable and the malware configuration after decryption contained fake C&amp;C addresses. We assume this was done to complicate analysis of the samples. In the new version, the threat actors moved from x86 to an x86-64 version and removed the fake C&amp;Cs from the configuration.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141552/IcedID_and_Qbot_spam_05.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-102924" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141552/IcedID_and_Qbot_spam_05.png" alt="" width="473" height="187" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141552/IcedID_and_Qbot_spam_05.png 473w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141552/IcedID_and_Qbot_spam_05-300x119.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141552/IcedID_and_Qbot_spam_05-443x175.png 443w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141552/IcedID_and_Qbot_spam_05-370x146.png 370w" sizes="auto, (max-width: 473px) 100vw, 473px" /></a></p>
<p style="text-align: center"><strong><em>Configuration of the old version of IcedID downloader</em></strong></p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141612/IcedID_and_Qbot_spam_06.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-102925" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141612/IcedID_and_Qbot_spam_06.png" alt="" width="473" height="72" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141612/IcedID_and_Qbot_spam_06.png 473w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141612/IcedID_and_Qbot_spam_06-300x46.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21141612/IcedID_and_Qbot_spam_06-370x56.png 370w" sizes="auto, (max-width: 473px) 100vw, 473px" /></a></p>
<p style="text-align: center"><strong><em>New version configuration</em></strong></p>
<p>We also observed a minor change in the malware&#8217;s main body. While it&#8217;s still distributed as a PNG image, and the decryption and C&amp;C communication methods remain the same, the authors decided not to use shellcode. Instead, IcedID&#8217;s main body is distributed as a standard PE file with some loader-related data in the beginning.</p>
<p><strong>Geography of IcedID attacks</strong></p>
<div style="width: 1010px" class="wp-caption aligncenter"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21144104/01-en-icedid-and-qbot-maps.png" target="_blank" rel="noopener" class="magnificImage"><img decoding="async" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21144104/01-en-icedid-and-qbot-maps.png" target="_blank" rel="noopener" alt="" class="aligncenter size-full"/></a><p class="wp-caption-text"><em>Geography of IcedID downloader detections, March 2021  (<a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21144104/01-en-icedid-and-qbot-maps.png" target="_blank" rel="noopener">download</a>)</em></p></div>
In March 2021, the greatest number of users attacked by Ligooc (IcedID downloader) were observed in China (15.88%), India (11.59%), Italy (10.73%), the United States (10.73%) and Germany (8.58%).</p>
<h3 id="qbot">Qbot</h3>
<p>Unlike IcedID, QBot is a single executable with an embedded DLL (main body) stored into the resource PE section. In order to perform traffic interception, steal passwords, perform web injects and take remote control of the infected system, it downloads additional modules: web inject module, hVNC (remote control module), email collector, password grabber and others. All the details on Qbot, as well as IoCs, MITRE ATT&amp;CK framework data, YARA rules and hashes relating to this threat are available to users of our <a href="https://www.kaspersky.com/enterprise-security/threat-intelligence" target="_blank" rel="noopener">Financial Threat Intelligence services</a>.</p>
<p><strong>Geography of Qbot attacks</strong></p>
<div style="width: 1010px" class="wp-caption aligncenter"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21144130/02-en-icedid-and-qbot-maps.png" target="_blank" rel="noopener" class="magnificImage"><img decoding="async" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21144130/02-en-icedid-and-qbot-maps.png" target="_blank" rel="noopener" alt="" class="aligncenter size-full"/></a><p class="wp-caption-text"><em>Geography of QBot attacks, March 2021  (<a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/21144130/02-en-icedid-and-qbot-maps.png" target="_blank" rel="noopener">download</a>)</em></p></div>
In March 2021, QBot was also most active in China (10.78%), India (10.78%) and the United States (4.66%), but we also observed it in Russia (7.60%) and France (7.60%).</p>
