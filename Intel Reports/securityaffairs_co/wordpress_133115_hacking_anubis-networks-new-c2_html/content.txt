							<h2>Anubis Networks is back with new C2 server</h2>
						</div>
						<div class="post-time mb-3">
							<span><i><img src="https://securityaffairs.com/wp-content/themes/security_affairs/images/user-icon.svg" alt="" /></i> <a href="https://securityaffairs.com/author/paganinip">Pierluigi Paganini</a></span>
							<span><i><img src="https://securityaffairs.com/wp-content/themes/security_affairs/images/clock-icon.svg" alt="" /></i> July 11, 2022</span>
						</div>
						<img class="img-fluid mb-4" src="https://i0.wp.com/securityaffairs.com/wp-content/uploads/2022/07/anubis.jpg?fit=690%2C461&ssl=1" alt="" />

<h2 class="wp-block-heading">A large-scale phishing campaign leveraging the Anubis Network is targeting Brazil and Portugal since March 2022.</h2>



<p>A large-scale phishing campaign is targeting Internet-end users in Brazil and Portugal since March 2022. Anubis Network is a C2 portal developed to control fake portals and aims to steal credentials to fully access the real systems.</p>



<p>This C2 server is controlled by a group of operators that come from the previous analysis in 2022, the various brands being divided among the operators of the group (in a call center&nbsp;<em>modus operandi</em>).</p>



<p>This campaign is <strong><a href="https://seguranca-informatica.pt/the-evolution-of-phishing-infrastructure-a-large-scale-campaign-called-anubis-that-targets-brazilian-and-portuguese-internet-users/#.Ysq_gHbMIQ9">highlighted by Segurança Informática in 2020</a></strong>, and the high-level diagram of this new campaign can be observed below.<gwmw style="display:none;"></p>



<figure class="wp-block-image"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/anubis-flow-diagram-.png?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/anubis-flow-diagram-.png?ssl=1" alt="" class="wp-image-10515"/></a></figure>



<p><em><strong>Figure 1:</strong> High-level diagram of the ANUBIS phishing network and its components (2020).</em><gwmw style="display:none;"></p>



<p>In detail, this fresh campaign is composed of three crucial operating components:</p>



<ul class="wp-block-list"><li><strong>the delivery vehicle to propagate the landing page in the wild; usually carried out through smishing (SMS) and phishing (email)</strong></li><li><strong>a malicious landing page hosted on a cloud server, composed of a user interface and layout very similar to the real system</strong></li><li><strong>an operation back-end that allows criminals to manage the details of users who have fallen into the trap.</strong></li></ul>



<p>Figure 2 presents an example of an SMS sent to Internet end-users during the ANUBIS social engineering wave. The image is related to an ongoing campaign in Portugal impersonating a specific organization to steal banking credentials.</p>


<div class="wp-block-image">
<figure class="aligncenter"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/phone.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/phone.jpg?ssl=1" alt="" class="wp-image-10516"/></a><figcaption><gwmw style="display:none;"></figcaption></figure>
</div>


<p><em><strong>Figure 2:</strong> Example of SMS sent during the social engineering wave.</em><gwmw style="display:none;"></p>



<p>SMSs are sent based on a list created by the C2 owner, namely:&nbsp;<strong>1kk-rusha-01.txt</strong>.</p>



<p><strong>Fake domains hosted automatically on Cloudflare CDN</strong><gwmw style="display:none;"></p>



<p>The ANUBIS network phishing campaigns are masked through the Cloudflare CDN. Operators can easily make this configuration through an interface that uses the CloudFlare API for configuring new DNS zones.</p>



<figure class="wp-block-image"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/cdn.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/cdn.jpg?ssl=1" alt="" class="wp-image-10518"/></a></figure>



<p><em><strong>Figure 3:</strong>&nbsp;Feature of adding new domains and configuring them behind the Cloudflare CDN via the ANUBIS back office portal.</em></p>



<p><strong>The Phishing template</strong><gwmw style="display:none;"></p>



<p>One of the last campaigns disseminated by criminals is impersonating a popular service in Portugal with the goal of stealing credentials of home banking portals.</p>



<p>After clicking on the link distributed via smishing, the victims are redirected to a specific landing page that collects the mobile phone number and the associated code (PIN). As observed, criminals are using the Let’s Encrypt CA to create valid HTTPs certificates.</p>


<div class="wp-block-image">
<figure class="alignleft"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/temlate.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/temlate.jpg?ssl=1" alt="" class="wp-image-10520"/></a></figure>
</div>


<p><em><strong><br>Figure 4:</strong>&nbsp;Phishing template of ANUBIS Network campaign.</em></p>



<p>After clicking on “<strong>CONTINUAR</strong>“, a new page is presented. Additional data from the victim are requested by the server-side and added to session cookies.</p>



<figure class="wp-block-image"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/cookies.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/cookies.jpg?ssl=1" alt="" class="wp-image-10521"/></a></figure>



<p><em><strong>Figure 5:</strong>&nbsp;Additional details about the victims are stored on the session cookies.</em></p>



<p>As observed, 12 target banks operating in Portugal are listed in this specific campaign.</p>


<div class="wp-block-image">
<figure class="aligncenter"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/home-banking.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/home-banking.jpg?ssl=1" alt="" class="wp-image-10522"/></a></figure>
</div>


<p><em><strong>Figure 6:</strong>&nbsp;Target banks present on the Anubis Network campaign in Portugal.</em></p>



<p>In the next step, credentials to access the target portals are requested.</p>



<figure class="wp-block-image"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/portal.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/portal.jpg?ssl=1" alt="" class="wp-image-10523"/></a></figure>



<p><em><strong>Figure 7:</strong>&nbsp;Credentials to access the real systems are requested.</em></p>



<p>Additional details related to credit cards are also requested by criminals. A specific loading page is then presented, and ANUBIS operators can request other details via the C2 portal in a call center&nbsp;<em>modus operandi</em>.</p>



<figure class="wp-block-image"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/portal2.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/portal2.jpg?ssl=1" alt="" class="wp-image-10524"/></a></figure>



<p><em><strong>Figure 8:</strong>&nbsp;Additional information requested by criminals.</em></p>



<p><strong>Anubis Network C2 Panel</strong><gwmw style="display:none;"></p>



<p>By analyzing the landing page source code, the URL of the C2 server can be obtained.</p>



<figure class="wp-block-image"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/c2.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/c2.jpg?ssl=1" alt="" class="wp-image-10525"/></a></figure>



<p><em><strong>Figure 9:</strong>&nbsp;Endpoint of the Anubis Network C2 server present on the source code.</em></p>



<p>As observed, the C2 login page is linked to a legitimate system in order to confuse threat analysts.</p>



<figure class="wp-block-image"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/login.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/login.jpg?ssl=1" alt="" class="wp-image-10529"/></a></figure>



<p><em><strong>Figure 10:</strong>&nbsp;Login page of Anubis Network C2 server.</em></p>



<p>The features observed inside the C2 server are very similar to the analysis performed in 2020. Operators can control all the infection flow by requesting additional details and accessing the real system in the background.</p>



<figure class="wp-block-image"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/painel2.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/painel2.jpg?ssl=1" alt="" class="wp-image-10530"/></a></figure>



<p><em><strong>Figure 10:</strong>&nbsp;Internal pages where Anubis Network operators can control all the malicious flow.</em></p>



<p>In detail, global administrators are capable of adding users to specific target organizations as observed below.</p>



<figure class="wp-block-image"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/users1.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/users1.jpg?ssl=1" alt="" class="wp-image-10533"/></a></figure>



<figure class="wp-block-image"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/user.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/user.jpg?ssl=1" alt="" class="wp-image-10532"/></a></figure>



<p><em><strong>Figure 11:</strong>&nbsp;Anubis Network operators and permissions page with the target organizations.</em></p>



<p>According to the MySQL database that supports the system, there are 77 operators in the system – which represents the business and operational volume of this malicious scheme.</p>

<p>An interesting feature also implemented in this new version of the C2 portal is the&nbsp;<strong>email temp</strong>. By using this feature, criminals can create new domains and use internal emails to manage all the processes.</p>



<figure class="wp-block-image"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/email.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/email.jpg?ssl=1" alt="" class="wp-image-10535"/></a></figure>



<p><em><strong>Figure 12:</strong>&nbsp;Anubis Network email temp feature.</em></p>



<p>The landing pages presented to the victims and specific data can be configured on the Anubis Network administrative portal. The path of the folder and the target brand can be observed on this specific page.</p>



<figure class="wp-block-image"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/fake.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/fake.jpg?ssl=1" alt="" class="wp-image-10539"/></a></figure>



<figure class="wp-block-image"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/brands1.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/brands1.jpg?ssl=1" alt="" class="wp-image-10538"/></a></figure>



<p><em><strong>Figure 13:</strong>&nbsp;Target organizations of Anubis Network C2 server – Jully 2022.</em></p>



<p>Since the malicious network is made up of many people, a channel on Telegram was created in order to provide technical support to operators in the performance of their duties.</p>


<div class="wp-block-image">
<figure class="aligncenter"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/tele.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/tele.jpg?ssl=1" alt="" class="wp-image-10534"/></a></figure>
</div>


<p><em><strong>Figure 14:&nbsp;</strong>Telegram channel created as a technical support channel.</em></p>



<p><strong>The MySQL database</strong><gwmw style="display:none;"></p>



<p>The heart of the ANUBIS network is a MySQL database. This database is used for data synchronization between all components of the malicious ecosystem and maintains everything up-to-date each second.</p>


<div class="wp-block-image">
<figure class="aligncenter"><a href="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/db.jpg?ssl=1"><img data-recalc-dims="1" decoding="async" src="https://i0.wp.com/seguranca-informatica.pt/wp-content/uploads/2022/07/db.jpg?ssl=1" alt="" class="wp-image-10541"/></a></figure>
</div>


<p><em><strong>Figure 15:</strong> Database schema of the ANUBIS phishing network.</em></p>

function emojiSetsRenderIdentically( context, set1, set2 ) {
	// Cleanup from previous test.
	context.clearRect( 0, 0, context.canvas.width, context.canvas.height );
	context.fillText( set1, 0, 0 );
	const rendered1 = new Uint32Array(
		context.getImageData(

			0,

		context.getImageData(

			0,
			context.canvas.width,
			context.canvas.height

	);

			/*
			 * Test for Sark flag compatibility. This is the least supported of the letter locale flags,
			 * so gives us an easy test for full support.
			 *
			 * To test for support, we try to render it, and compare the rendering to how it would look if
			 * the browser doesn't render it correctly ([C] + [Q]).
			 */
			isIdentical = emojiSetsRenderIdentically(
				context,
				'\uD83C\uDDE8\uD83C\uDDF6', // as the sequence of two code points
				'\uD83C\uDDE8\u200B\uD83C\uDDF6' // as the two code points separated by a zero-width space
			);

			if ( isIdentical ) {
				return false;
			}

			/*
			 * Test for English flag compatibility. England is a country in the United Kingdom, it
			 * does not have a two letter locale code but rather a five letter sub-division code.
			 *
			 * To test for support, we try to render it, and compare the rendering to how it would look if
			 * the browser doesn't render it correctly (black flag emoji + [G] + [B] + [E] + [N] + [G]).
			 */
			isIdentical = emojiSetsRenderIdentically(
				context,
				// as the flag sequence
				'\uD83C\uDFF4\uDB40\uDC67\uDB40\uDC62\uDB40\uDC65\uDB40\uDC6E\uDB40\uDC67\uDB40\uDC7F',
				// with each code point separated by a zero-width space
				'\uD83C\uDFF4\u200B\uDB40\uDC67\u200B\uDB40\uDC62\u200B\uDB40\uDC65\u200B\uDB40\uDC6E\u200B\uDB40\uDC67\u200B\uDB40\uDC7F'
			);

			return ! isIdentical;
		case 'emoji':
			/*
			 * Is there a large, hairy, humanoid mythical creature living in the browser?
			 *
			 * To test for Emoji 17.0 support, try to render a new emoji: Hairy Creature.
			 *
			 * The hairy creature emoji is a single code point emoji. Testing for browser
			 * support required testing the center point of the emoji to see if it is empty.
			 *
			 * 0xD83E 0x1FAC8 (\uD83E\u1FAC8) == 🫈 Hairy creature.
			 *
			 * When updating this test, please ensure that the emoji is either a single code point
			 * or switch to using the emojiSetsRenderIdentically function and testing with a zero-width
			 * joiner vs a zero-width space.
			 */
			const notSupported = emojiRendersEmptyCenterPoint( context, '\uD83E\u1FAC8' );
			return ! notSupported;
	}

	return false;
}

/**
 * Checks emoji support tests.
 *
 * This function may be serialized to run in a Worker. Therefore, it cannot refer to variables from the containing
 * scope. Everything must be passed by parameters.
 *
 * @since 6.3.0
 *
 * @private
 *
 * @param {string[]} tests Tests.
 * @param {Function} browserSupportsEmoji Reference to browserSupportsEmoji function, needed due to minification.
 * @param {Function} emojiSetsRenderIdentically Reference to emojiSetsRenderIdentically function, needed due to minification.
 * @param {Function} emojiRendersEmptyCenterPoint Reference to emojiRendersEmptyCenterPoint function, needed due to minification.
 *
 * @return {SupportTests} Support tests.
 */
function testEmojiSupports( tests, browserSupportsEmoji, emojiSetsRenderIdentically, emojiRendersEmptyCenterPoint ) {
	let canvas;
	if (
		typeof WorkerGlobalScope !== 'undefined' &&
		self instanceof WorkerGlobalScope
	) {
		canvas = new OffscreenCanvas( 300, 150 ); // Dimensions are default for HTMLCanvasElement.
	} else {
		canvas = document.createElement( 'canvas' );
	}

	const context = canvas.getContext( '2d', { willReadFrequently: true } );

	/*
	 * Chrome on OS X added native emoji rendering in M41. Unfortunately,
	 * it doesn't work when the font is bolder than 500 weight. So, we
	 * check for bold rendering support to avoid invisible emoji in Chrome.
	 */
	context.textBaseline = 'top';
	context.font = '600 32px Arial';

	const supports = {};
	tests.forEach( ( test ) => {
		supports[ test ] = browserSupportsEmoji( context, test, emojiSetsRenderIdentically, emojiRendersEmptyCenterPoint );
	} );
	return supports;
}
