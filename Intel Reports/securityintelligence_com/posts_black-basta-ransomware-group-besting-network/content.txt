&lt;h2&gt;X-Force assessment&lt;/h2&gt;
&lt;p&gt;Black Basta’s sudden appearance and subsequent high volume of successful compromises highlights the speed with which ostensibly “new” ransomware groups are able to become central figures in the threat landscape. While this group has yet to be linked by X-Force to existing or past ransomware operations, the impact that this group has had in such a short amount of time points to a practiced and effective set of tactics, techniques, and procedures (TTPs) that pose a significant threat to enterprise networks. Based on current intelligence, it does not appear that Black Basta targets specific industries or verticals. However, entities that collect large amounts of data that is attractive for extortion operations such as personally identifiable information (PII), financial information, or other sensitive data, are likely to stand out as lucrative targets for attackers.&lt;/p&gt;
&lt;p&gt;We encourage security practitioners to review the IBM X-Force &lt;a href=&quot;https://www.ibm.com/downloads/cas/EV6NAQR4&quot; target=&quot;_blank&quot; rel=&quot;noopener&quot;&gt;Definitive Guide to Ransomware&lt;/a&gt; and the mitigations highlighted below, action available IOCs, and ensure that detections are in place to alert on observed malicious behaviors.&lt;/p&gt;
&lt;p&gt;Additional Risk Mitigations:&lt;/p&gt;
&lt;ul&gt;
&lt;li&gt;Establish and maintain backup routines, including offline backups. Ensure you have backup redundancy stored separately from network zones attackers could access with read-only access. The availability of effective backups is a significant differentiator for organizations and can support recovery from a ransomware attack.&lt;/li&gt;
&lt;li&gt;Implement a strategy to prevent unauthorized data theft, especially as it applies to uploading large amounts of data to legitimate cloud storage platforms that attackers can abuse.&lt;/li&gt;
&lt;li&gt;Employ user behavior analytics to identify potential security incidents. When triggered, assume a breach has taken place. Audit, monitor and quickly act on suspected abuse related to privileged accounts and groups.&lt;/li&gt;
&lt;li&gt;Employ multifactor authentication on all remote access points into an enterprise network — with particular care given to secure or disable remote desktop protocol (RDP) access. Multiple ransomware attacks have been known to exploit weak RDP access to gain initial entry into a network.&lt;/li&gt;
&lt;/ul&gt;

&lt;h2&gt;Ransomware analysis details&lt;/h2&gt;
&lt;h3&gt;Behavioral analysis&lt;/h3&gt;
&lt;p&gt;Behavioral analysis describes the malware behavior observed on a system during execution. Behavioral analysis typically includes actions performed on the system such as files dropped, persistence, details surrounding process execution and any C2 communications. It should be noted that behavioral analysis may not capture all notable malware behavior as certain functions may only be performed by the malware under specific conditions.&lt;/p&gt;
&lt;p&gt;&lt;strong&gt;Loader&lt;/strong&gt;&lt;/p&gt;

The decoded payload is then RC4 decrypted using the key &quot;&lt;em&gt;1xJr13puJChWqiMeOnFDDSOmoSmws5n&lt;/em&gt;&quot;. Black Basta is then injected into a process instance of the loader and executed in memory. The injected Black Basta sample has a SHA256 hash: ef1382770f820e4b2e65981bb7b3a62d5f93e3b87763f83012ef7f7cb1bc9469 and compile time: &lt;strong&gt;Thursday, 28.04.2022 15:15:43 UTC&lt;/strong&gt;.&lt;/p&gt;

&lt;h3&gt;Static analysis&lt;/h3&gt;
&lt;p&gt;Static analysis is a deeper dive into the technical analysis of the malware. Static analysis typically includes further details about the functionality, obfuscation or packing in the sample, encryption used by the malware, configuration information or other notable technical detail.&lt;/p&gt;
&lt;p&gt;&lt;strong&gt;Loader&lt;/strong&gt;&lt;/p&gt;
&lt;p&gt;The injection process for the loader is as follows:&lt;/p&gt;
&lt;ol&gt;
&lt;li&gt;The loader creates a suspended process instance of itself with &lt;em&gt;CreateProcessA()&lt;/em&gt;.&lt;/li&gt;
&lt;li&gt;Then it calls &lt;em&gt;GetThreadContext()&lt;/em&gt; to get the thread context of the suspended process.&lt;/li&gt;
&lt;li&gt;Next, it calls &lt;em&gt;ReadProcessMemory()&lt;/em&gt; to obtain the suspended process&#039; image base from the Process Environment Block retrieved via the thread context.&lt;/li&gt;
&lt;li&gt;The loader proceeds to allocate memory in the suspended process using &lt;em&gt;VirtualAllocEx()&lt;/em&gt; and writes the payload to the buffer using &lt;em&gt;WriteProcessMemory()&lt;/em&gt;.&lt;/li&gt;
&lt;li&gt;Finally, it sets the entry point of the injected payload using &lt;em&gt;SetThreadContext()&lt;/em&gt; and executes the injected payload using &lt;em&gt;ResumeThread()&lt;/em&gt;.&lt;/li&gt;
&lt;/ol&gt;
&lt;p&gt;&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz1mN2Y3ZTg0MmRkMTQxMWVjYjU0ZDYyNWRmZjM5NWNhMw==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiZjdmN2U4NDJkZDE0MTFlY2I1NGQ2MjVkZmYzOTVjYTMiXSwiZXhwIjoxNjUzNTkyMDA3fQ.8H_zn8g8UMF0Ohc9e2SHjc4l8AZOGMzax9j_f_46c5c&quot; alt=&quot;Basta Fig 4.png&quot; width=&quot;499.71428571428567&quot; height=&quot;720&quot; /&gt;&lt;/p&gt;

&lt;h2&gt;Black Basta&lt;/h2&gt;
&lt;h3&gt;Encryption&lt;/h3&gt;
&lt;p&gt;To start encrypting files, Black Basta calls the &lt;em&gt;FindFirstVolumeW()&lt;/em&gt; and &lt;em&gt;FindNextVolumeW()&lt;/em&gt; functions to enumerate volumes on the victim system. For each volume, the ransomware calls &lt;em&gt;GetVolumePathNamesForVolumeNameW() &lt;/em&gt;to obtain a list of drive letters and mounted folder paths for the specified volume. Analysis indicates that Black Basta uses &lt;em&gt;ChaCha20&lt;/em&gt; encryption during the encryption process. Notably, it is observed that some files are not fully encrypted, possibly in an effort to hasten the encryption process. Black Basta encrypts some files in 64-byte blocks every 128 bytes as shown in the example encrypted notepad.exe file below.&lt;/p&gt;
&lt;p&gt;&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz0wMmRiZDU2NmRkMTUxMWVjYTVhMzlhZmQ0N2ZiYTJkOA==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiMDJkYmQ1NjZkZDE1MTFlY2E1YTM5YWZkNDdmYmEyZDgiXSwiZXhwIjoxNjUzNTkyMDA3fQ.Y9nUL7ujcxNGUpiw8n3mdF3wuXCXbkbh2F_jUf83Vsc&quot; alt=&quot;Basta Fig 5.png&quot; width=&quot;600&quot; height=&quot;638&quot; /&gt;&lt;/p&gt;

<html><body><p><em>This post was written with contributions from Chris Caridi and Kat Weinberger.</em></p>
<p>IBM Security X-Force has been tracking the activity of Black Basta, a new ransomware group that <a href="https://cyberintelmag.com/malware-viruses/newly-discovered-black-basta-ransomware-could-be-associated-with-conti-gang/" target="_blank" rel="noopener nofollow"  target="_blank" rel="noopener nofollow" >first appeared in April 2022</a>. To date, this group has claimed attribution of 29 different victims across multiple industries using a double extortion strategy where the attackers not only execute ransomware but also steal data and threaten to release it publicly if the ransom demands are not met. The data disclosure element of these attacks takes place on a data leak site available on the <a href="https://www.sciencedirect.com/topics/computer-science/onion-router" target="_blank" rel="noopener nofollow"  target="_blank" rel="noopener nofollow" >Tor network</a>. As a mechanism to apply pressure to coerce the victim to pay the ransom, the operators of Black Basta will gradually release stolen data on the leak site.</p>
<p>The Black Basta group is still in the early stages of their organization and X-Force has not observed any attempts to advertise the malware or hire affiliates on underground forums. Due to operational similarities and the absence of affiliation attempts, it has been reported that Black Basta may potentially be a <a href="https://www.securityweek.com/new-black-basta-ransomware-possibly-linked-conti-group" target="_blank" rel="noopener nofollow"  target="_blank" rel="noopener nofollow" >rebranded version of the Conti gang</a>, a well-known ransomware group that already has affiliates. However, the Conti group announced on May 12 that they had no affiliations with the Black Basta ransomware group. X-Force&rsquo;s assessment of the possible relationship between these groups is ongoing.</p>
<p>The Black Basta ransomware variant acts at such a high speed that it rarely causes symptoms that would tip off defenders to the compromise before the ransomware has been deployed.</p>
<p>This blog post will detail X-Force&rsquo;s insights into the Black Basta ransomware, technical analysis of the sample, and indicators of compromise (IoC) to help organizations protect themselves from this malware.</p>
<h2>X-Force assessment</h2>
<p>Black Basta&rsquo;s sudden appearance and subsequent high volume of successful compromises highlights the speed with which ostensibly &ldquo;new&rdquo; ransomware groups are able to become central figures in the threat landscape. While this group has yet to be linked by X-Force to existing or past ransomware operations, the impact that this group has had in such a short amount of time points to a practiced and effective set of tactics, techniques, and procedures (TTPs) that pose a significant threat to enterprise networks. Based on current intelligence, it does not appear that Black Basta targets specific industries or verticals. However, entities that collect large amounts of data that is attractive for extortion operations such as personally identifiable information (PII), financial information, or other sensitive data, are likely to stand out as lucrative targets for attackers.</p>
<p>We encourage security practitioners to review the IBM X-Force <a href="https://www.ibm.com/downloads/cas/EV6NAQR4"  target="_blank" rel="noopener nofollow" >Definitive Guide to Ransomware</a> and the mitigations highlighted below, action available IOCs, and ensure that detections are in place to alert on observed malicious behaviors.</p>
<p>Additional Risk Mitigations:</p>
<ul>
<li>Establish and maintain backup routines, including offline backups. Ensure you have backup redundancy stored separately from network zones attackers could access with read-only access. The availability of effective backups is a significant differentiator for organizations and can support recovery from a ransomware attack.</li>
<li>Implement a strategy to prevent unauthorized data theft, especially as it applies to uploading large amounts of data to legitimate cloud storage platforms that attackers can abuse.</li>
<li>Employ user behavior analytics to identify potential security incidents. When triggered, assume a breach has taken place. Audit, monitor and quickly act on suspected abuse related to privileged accounts and groups.</li>
<li>Employ multifactor authentication on all remote access points into an enterprise network &mdash; with particular care given to secure or disable remote desktop protocol (RDP) access. Multiple ransomware attacks have been known to exploit weak RDP access to gain initial entry into a network.</li>
</ul>
<a class="btn orange1 large" href="https://www.ibm.com/downloads/cas/EV6NAQR4" >Download the Definitive Guide to Ransomware</a>
<h2>Ransomware analysis details</h2>
<h3>Behavioral analysis</h3>
<p>Behavioral analysis describes the malware behavior observed on a system during execution. Behavioral analysis typically includes actions performed on the system such as files dropped, persistence, details surrounding process execution and any C2 communications. It should be noted that behavioral analysis may not capture all notable malware behavior as certain functions may only be performed by the malware under specific conditions.</p>
<p><strong>Loader</strong></p>
<p>Upon execution, the loader base64 decodes a Black Basta payload using the <em>CryptStringToBinaryA() </em>API function. The decoded payload is then RC4 decrypted using the key &ldquo;<em>1xJr13puJChWqiMeOnFDDSOmoSmws5n</em>&ldquo;. Black Basta is then injected into a process instance of the loader and executed in memory. The injected Black Basta sample has a SHA256 hash: ef1382770f820e4b2e65981bb7b3a62d5f93e3b87763f83012ef7f7cb1bc9469 and compile time: <strong>Thursday, 28.04.2022 15:15:43 UTC</strong>.</p>

<ul>

<h3>Static analysis</h3>
<p>Static analysis is a deeper dive into the technical analysis of the malware. Static analysis typically includes further details about the functionality, obfuscation or packing in the sample, encryption used by the malware, configuration information or other notable technical detail.</p>
<p><strong>Loader</strong></p>
<p>The injection process for the loader is as follows:</p>
<ol>
<li>The loader creates a suspended process instance of itself with <em>CreateProcessA()</em>.</li>
<li>Then it calls <em>GetThreadContext()</em> to get the thread context of the suspended process.</li>
<li>Next, it calls <em>ReadProcessMemory()</em> to obtain the suspended process&rsquo; image base from the Process Environment Block retrieved via the thread context.</li>
<li>The loader proceeds to allocate memory in the suspended process using <em>VirtualAllocEx()</em> and writes the payload to the buffer using <em>WriteProcessMemory()</em>.</li>
<li>Finally, it sets the entry point of the injected payload using <em>SetThreadContext()</em> and executes the injected payload using <em>ResumeThread()</em>.</li>
</ol>

<h2>Black Basta</h2>
<h3>Encryption</h3>
<p>To start encrypting files, Black Basta calls the <em>FindFirstVolumeW()</em> and <em>FindNextVolumeW()</em> functions to enumerate volumes on the victim system. For each volume, the ransomware calls <em>GetVolumePathNamesForVolumeNameW() </em>to obtain a list of drive letters and mounted folder paths for the specified volume. Analysis indicates that Black Basta uses <em>ChaCha20</em> encryption during the encryption process. Notably, it is observed that some files are not fully encrypted, possibly in an effort to hasten the encryption process. Black Basta encrypts some files in 64-byte blocks every 128 bytes as shown in the example encrypted notepad.exe file below.</p>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz0wMmRiZDU2NmRkMTUxMWVjYTVhMzlhZmQ0N2ZiYTJkOA==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiMDJkYmQ1NjZkZDE1MTFlY2E1YTM5YWZkNDdmYmEyZDgiXSwiZXhwIjoxNjUzNTkyMDA3fQ.Y9nUL7ujcxNGUpiw8n3mdF3wuXCXbkbh2F_jUf83Vsc" layout="intrinsic" class="" alt="Basta Fig 5.png" width="600" height="638" lightbox="lightbox"></amp-img></p>
