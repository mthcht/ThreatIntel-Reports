					<h1 dir="ltr"><strong>Executive summary</strong></h1>
<p dir="ltr">The BlackCat/ALPHV ransomware is a complex threat written in Rust that appeared in November 2021. In this post, we describe a real engagement that we recently handled by giving details about the tools, techniques, and procedures (TTPs) used by this threat actor. Firstly, the attacker targeted an unpatched Microsoft Exchange server and successfully dropped webshells on the machine. After getting initial access, the ransomware installed the MobaXterm software and then started to dump the credentials using Mimikatz or by creating an LSASS dump file with Process Hacker. The SoftPerfect Network scanner was used to perform network discovery. Before running the ransomware executable, the TA compressed the targeted files using WinRAR or 7zip and then exfiltrated them using rclone and MEGAsync.</p>
<p dir="ltr">Our Digital Forensics and Incident Response (DFIR) team was engaged in investigating a ransomware infection. We were able to determine that the ransomware involved is a new version of the BlackCat ransomware, based on the fact that the <a href="https://securityscorecard.com/blog/what-is-malware/">malware</a> added new command line parameters that were not documented before.</p>
<p dir="ltr">As shown in Figure 1, the ransomware added a parameter called “–safeboot” that is used to reboot in Safe Mode. Whether the malware is running with the “–sleep-restart” parameter, the process sleeps for a specified number of seconds and then restarts the machine.</p>
<p dir="ltr"><img fetchpriority="high" decoding="async" class="aligncenter wp-image-2813 size-large" src="https://securityscorecard.com/wp-content/uploads/2024/01/Figure-1-2-818x1024.png" alt="" width="640" height="801" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/Figure-1-2-818x1024.png 818w, https://securityscorecard.com/wp-content/uploads/2024/01/Figure-1-2-240x300.png 240w, https://securityscorecard.com/wp-content/uploads/2024/01/Figure-1-2-768x962.png 768w, https://securityscorecard.com/wp-content/uploads/2024/01/Figure-1-2.png 892w" sizes="(max-width: 640px) 100vw, 640px" /></p>
<h6><em>Figure 1</em></h6>
<p dir="ltr">A complete analysis of the BlackCat ransomware can be found <a href="https://securityscorecard.com/research/deep-dive-into-alphv-blackcat-ransomware">here</a>.</p>
<p dir="ltr">By accessing the onion link specified in the ransom note called “RECOVER-&lt;extension&gt;-FILES.txt”, the victim is presented with multiple tabs that contain information such as the ransom amount in Bitcoin and Monero, the threat actor’s wallets addresses, a live chat, and a trial decrypt that can be used to decrypt a few files for free (see Figure 2).</p>
<p dir="ltr"><img decoding="async" class="aligncenter size-large wp-image-2814" src="https://securityscorecard.com/wp-content/uploads/2024/01/Figure-2-1-1024x754.png" alt="" width="640" height="471" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/Figure-2-1-1024x754.png 1024w, https://securityscorecard.com/wp-content/uploads/2024/01/Figure-2-1-300x221.png 300w, https://securityscorecard.com/wp-content/uploads/2024/01/Figure-2-1-768x565.png 768w, https://securityscorecard.com/wp-content/uploads/2024/01/Figure-2-1.png 1375w" sizes="(max-width: 640px) 100vw, 640px" /></p>
<h6><em>Figure 2</em></h6>
<h1 dir="ltr"><strong>Initial access</strong></h1>
<p dir="ltr">According to our analysis, the entry point in the organization was an Exchange server that was vulnerable to Microsoft Exchange vulnerabilities. Multiple webshells have been identified on the impacted server.</p>
<h1 dir="ltr"><strong>Remote access tools</strong></h1>
<p dir="ltr">After gaining access to the internal network, the ransomware installed the legitimate tools MobaXterm and mottynew.exe (MobaXterm terminal).</p>
<h1 dir="ltr"><strong>Lateral Movement</strong></h1>
<p dir="ltr">As we already know from the malware analysis of the ransomware, BlackCat steals credentials from the victim’s environment and incorporates them into its configuration (“credentials” field). Mimikatz was utilized to dump the credentials, and then the malware pivots from one machine to another via Remote Desktop Protocol (RDP).</p>
<p dir="ltr">An old version of the SoftPerfect Network scanner was used to perform network scanning in order to discover additional targets in the local network. Figure 3 reveals the interface of the network scanner:</p>
<p dir="ltr"><img decoding="async" class="aligncenter size-full wp-image-2815" src="https://securityscorecard.com/wp-content/uploads/2024/01/Figure-3-2.png" alt="" width="811" height="616" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/Figure-3-2.png 811w, https://securityscorecard.com/wp-content/uploads/2024/01/Figure-3-2-300x228.png 300w, https://securityscorecard.com/wp-content/uploads/2024/01/Figure-3-2-768x583.png 768w" sizes="(max-width: 811px) 100vw, 811px" /></p>
<h6><em>Figure 3</em></h6>
<h1 dir="ltr"><strong>Data exfiltration</strong></h1>
<p dir="ltr">Once the threat actor decided which files to exfiltrate, the malware compressed them using WinRAR or 7zip. The ransomware installed a tool called rclone that is utilized to upload data to cloud storage providers. A second tool called MEGAsync is also installed by the process, which can upload data to the MEGA Cloud Storage.</p>
<p dir="ltr">We’ve also observed the ransomware installing tools such as FileZilla and WinSCP that could be used to perform <a href="https://securityscorecard.com/blog/what-is-data-exfiltration-how-to-prevent-it">data exfiltration</a>.</p>
<h1 dir="ltr"><strong>Other tools installed</strong></h1>
<p dir="ltr">We’ve investigated and found out that the ransomware installed the cURL tool to download additional files. Process Hacker was also installed by the malware and could be used to dump the memory of the LSASS process. In the same directory with Process Hacker, the BlackCat ransomware dropped a copy of the PEView tool, which is a viewer for Portable Executable (PE) files.</p>
<h1 dir="ltr"><strong>Conclusion</strong></h1>
<p dir="ltr">BlackCat ransomware remains a serious threat because it targets Windows hosts, Linux hosts, and VMWare ESXi. The access token that the malware is running with makes the automated analysis impossible and increases the difficulty of the dynamic malware analysis. The usage of legitimate tools to perform malicious activities increases the chance of not being detected by endpoint detection and response (EDR) or antivirus software.</p>
