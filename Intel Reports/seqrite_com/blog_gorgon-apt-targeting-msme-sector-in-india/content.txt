			<p>From the past few months, we have been monitoring cyber-threats on <strong>MSME [Micro, Small and Medium Enterprises] sector</strong> within India. MSME sector is considered to be the backbone of the Indian economy. MSME employs around 40% of the country&#8217;s workforce, contributing nearly 45% to manufacturing output and 40% of exports. Staring at a major financial resource crunch, MSME&#8217;s are worst affected due to the ongoing COVID-19 pandemic.</p>
<p>We observed one similar wave on MSME in late April 2020 — it was a <a href="https://www.seqrite.com/blog/protecting-your-enterprise-from-phishing-attacks/" data-wpel-link="internal" target="_self" rel="follow">phishing</a> campaign luring victims with COVID-19 themed maldocs. From this campaign, one prominent file was:</p>

<h2>Technical Details</h2>
<p>Victims received an email with attached zip “face mask order.zip” which contained the aforementioned maldoc responsible to drop malware into the victim machine. The attack begins after opening “face mask order.doc”.  This RTF is weaponized with exploit which triggers CVE-2017-11882 vulnerability to execute arbitrary code.</p>
<figure id="attachment_4434" aria-describedby="caption-attachment-4434" style="width: 1258px" class="wp-caption alignnone"><img decoding="async" class="size-full wp-image-4434" src="https://www.seqrite.com/blog/wp-content/uploads/2020/08/2.png" alt="Figure 2: Process Infection Chain" width="1258" height="556" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/08/2.png 1258w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/2-300x133.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/2-768x339.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/2-1024x453.png 1024w" sizes="(max-width: 1258px) 100vw, 1258px" /><figcaption id="caption-attachment-4434" class="wp-caption-text">Figure 2: Process Infection Chain</figcaption></figure>
<h2>CVE-2017-11882 Analysis:</h2>
<p>Malicious rtf document contains two malicious ole objects.</p>
<figure id="attachment_4435" aria-describedby="caption-attachment-4435" style="width: 732px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4435" src="https://www.seqrite.com/blog/wp-content/uploads/2020/08/3.png" alt="Figure 3: RTF Objects" width="732" height="587" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/08/3.png 732w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/3-300x241.png 300w" sizes="(max-width: 732px) 100vw, 732px" /><figcaption id="caption-attachment-4435" class="wp-caption-text">Figure 3: RTF Objects</figcaption></figure>
<p>The tools extracts the ole objects from RTF file. OLE object (#0) is a VBScript file (i.e. ServerCrypted.vbs script) and object (#2) contains Equation Editor Exploit and command to execute file “<em>CmD.exe /C cscript %tmp%\ServerCrypted.vbs</em>”.</p>
<p>The OLE object (#0) is an executable file (i.e. ServerCrypted.vbs script) as shown in figure 4.</p>
<figure id="attachment_4436" aria-describedby="caption-attachment-4436" style="width: 826px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4436" src="https://www.seqrite.com/blog/wp-content/uploads/2020/08/4.png" alt="Figure 4: Embedded VBScript" width="826" height="692" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/08/4.png 826w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/4-300x251.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/4-768x643.png 768w" sizes="(max-width: 826px) 100vw, 826px" /><figcaption id="caption-attachment-4436" class="wp-caption-text">Figure 4: Embedded VBScript</figcaption></figure>
<p>The class name for this object (#2) is Equation.3 the exploit (CVE-2017-11882) as shown in the below image.</p>
<figure id="attachment_4437" aria-describedby="caption-attachment-4437" style="width: 614px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4437" src="https://www.seqrite.com/blog/wp-content/uploads/2020/08/5.png" alt="Figure 5: RTF contents" width="614" height="312" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/08/5.png 614w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/5-300x152.png 300w" sizes="(max-width: 614px) 100vw, 614px" /><figcaption id="caption-attachment-4437" class="wp-caption-text">Figure 5: RTF contents</figcaption></figure>
<p>It also contains the command to run ServerCrypted.vbs script as shown in below figure 6.</p>
<figure id="attachment_4438" aria-describedby="caption-attachment-4438" style="width: 617px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4438" src="https://www.seqrite.com/blog/wp-content/uploads/2020/08/6.png" alt="Figure 6: RTF contents" width="617" height="313" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/08/6.png 617w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/6-300x152.png 300w" sizes="(max-width: 617px) 100vw, 617px" /><figcaption id="caption-attachment-4438" class="wp-caption-text">Figure 6: RTF contents</figcaption></figure>
<p>CVE-2017-11882 is present in the Microsoft Office Equation Editor (EQNEDT32.EXE) component. The attacker can successfully exploit a stack buffer overflow vulnerability in the equation editor component of MS Office and execute arbitrary code. The root cause of this vulnerability is copy unbounded string of FONT name defined within a FONT record structure of Equation Editor OLE object data.</p>
<p>The below figure shows the stack buffer overflow scenario while copying the font name into a locally created buffer.</p>
<figure id="attachment_4439" aria-describedby="caption-attachment-4439" style="width: 1013px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4439" src="https://www.seqrite.com/blog/wp-content/uploads/2020/08/7.png" alt="Figure 7: Stack buffer overflow scenario" width="1013" height="628" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/08/7.png 1013w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/7-300x186.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/7-768x476.png 768w" sizes="(max-width: 1013px) 100vw, 1013px" /><figcaption id="caption-attachment-4439" class="wp-caption-text">Figure 7: Stack buffer overflow scenario</figcaption></figure>
<p>In this case, the function will return “back” to 0x430c12, which is the address of WinExec, and the argument is the “font name” and command which the attacker wants to execute.</p>
<figure id="attachment_4440" aria-describedby="caption-attachment-4440" style="width: 1011px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4440" src="https://www.seqrite.com/blog/wp-content/uploads/2020/08/8.png" alt="Figure 8: Return address, overwritten with WinExec" width="1011" height="630" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/08/8.png 1011w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/8-300x187.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/8-768x479.png 768w" sizes="(max-width: 1011px) 100vw, 1011px" /><figcaption id="caption-attachment-4440" class="wp-caption-text">Figure 8: Return address, overwritten with WinExec</figcaption></figure>
<h2>VBScript Analysis:</h2>
<p>After successful exploitation, cmd.exe is executed with commands:</p>
<p><em>“/c cscript %tmp%\ServerCrypted.vbs &gt;&gt; AC”</em></p>
<p>This command creates a cscript.exe process to execute code in ServerCrypted.vbs. VBScript file was already dropped in <em>%temp%</em> folder by WinWord process. Below Figure shows the VBScript code. As shown in figure 9 and figure 10, actors used some AV-vendors names in function names, variable names, and strings.  This VBScript is responsible to execute two processes, cmd.exe and powershell.exe. Figure 10 shows the obfuscated PowerShell script.</p>
<figure id="attachment_4441" aria-describedby="caption-attachment-4441" style="width: 837px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4441" src="https://www.seqrite.com/blog/wp-content/uploads/2020/08/9.png" alt="Figure 9: Contents of ServerCrypted.vbs" width="837" height="567" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/08/9.png 837w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/9-300x203.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/9-768x520.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/9-370x250.png 370w" sizes="(max-width: 837px) 100vw, 837px" /><figcaption id="caption-attachment-4441" class="wp-caption-text">Figure 9: Contents of ServerCrypted.vbs</figcaption></figure>

<h2>PowerShell Analysis:</h2>
<p>Figure 11 shows the de-obfuscated <a href="https://www.seqrite.com/blog/this-is-how-hackers-can-invade-your-system-without-installing-malware/" data-wpel-link="internal" target="_self" rel="follow">PowerShell</a> script.</p>
<p>As shown below, the script downloads two files with a .jpg extension. First file “15882060891.jpg” is a PowerShell script which contains encoded injector DLL written in C#, as shown in figure 12. This DLL is loaded in-memory by PowerShell. In this script, actors used some interesting names of class and function of injector DLL like FlorianRoth and Cyb3rOps.  Florian Roth is a well-known security researcher and  CTO of <a href="https://www.nextron-systems.com/" data-wpel-link="external" target="_blank" rel="nofollow">Nextron Systems</a>. Cyb3rOps is his twitter handle name. The second file which is downloaded by PowerShell is “15882060892.jpg”, is an encoded agent tesla payload. This payload is injected in Windows native binary RegAsm.exe.</p>
<figure id="attachment_4443" aria-describedby="caption-attachment-4443" style="width: 1172px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4443" src="https://www.seqrite.com/blog/wp-content/uploads/2020/08/11.png" alt="Figure 11: Decoded PowerShell script" width="1172" height="149" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/08/11.png 1172w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/11-300x38.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/11-768x98.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/11-1024x130.png 1024w" sizes="(max-width: 1172px) 100vw, 1172px" /><figcaption id="caption-attachment-4443" class="wp-caption-text">Figure 11: Decoded PowerShell script</figcaption></figure>

<h2>Final Payload &#8211; Agent Tesla:</h2>
<p>Below figure shows injected Agent Tesla payload in RegAsm.exe.</p>
<figure id="attachment_4447" aria-describedby="caption-attachment-4447" style="width: 802px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4447" src="https://www.seqrite.com/blog/wp-content/uploads/2020/08/15.png" alt="Figure 15: Injected payload" width="802" height="499" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/08/15.png 802w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/15-300x187.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/08/15-768x478.png 768w" sizes="(max-width: 802px) 100vw, 802px" /><figcaption id="caption-attachment-4447" class="wp-caption-text">Figure 15: Injected payload</figcaption></figure>
<p>Agent Tesla is a well-known keylogger and infostealer written in DotNet. This malware steals information from a variety of applications like Web Browsers, Email Clients, FTP Clients, Messenger applications, VPN clients, etc. and can also take screenshots of the system. All stolen data is exfiltrated over SMTP.</p>
<p>We have already explored and analysed Agent Tesla in our last couple of blogs:</p>
<p><a href="https://www.seqrite.com/blog/advance-campaign-targeting-manufacturing-and-export-sectors-in-india/" data-wpel-link="internal" target="_self" rel="follow">Advance Campaign Targeting Manufacturing and Export Sectors in India</a></p>
<p><a href="https://www.seqrite.com/blog/coronavirus-themed-campaign/" data-wpel-link="internal" target="_self" rel="follow">Coronavirus-themed Campaign delivers Agent Tesla Malware</a></p>
<h2>Conclusion</h2>
<p>Most TTPs shared above, have been seen on several occasions in the last few years. Looking at malware, C2 and technique execution, Quick Heal correlates this campaign on MSME sector to Gorgon group [a.k.a. Subaat]. All members of the Gorgon cyber-criminal group purport to have Pakistan-based interests/connections. Recently, <a href="https://www.seqrite.com/blog/advance-campaign-targeting-manufacturing-and-export-sectors-in-india/" data-wpel-link="internal" target="_self" rel="follow">another Gorgon campaign</a> was uncovered a few months back which used the same commodity malware RATs to accomplish their objective.</p>
<p>Given the global impact of COVID-19, threat actors will likely continue to use COVID-19-themed emails to deliver malware broadly in support of their objectives. Considering this trend, we encourage Micro, Small and Medium Enterprises to apply extra scrutiny to COVID-19-related emails containing attachments. Though large organizations, critical government infrastructures, and others have somewhat built resilience to such cyber threats; but MSME still needs to cover-up and remain extra vigilant with a robust strategy to mitigate risks.</p>
<h2>Threat Protection</h2>
<p>Our Seqrite and Quick Heal line of products protect against top cyber threats including Microsoft Office Memory Corruption Vulnerability (CVE-2017-11882) and variants of Agent Tesla RAT. Our advanced signature-less behaviour-based detection successfully blocks Agent Tesla variants.</p>
<p>Quick Heal advises users to exercise ample caution and avoid opening attachments &amp; clicking on web links in unsolicited emails. Users should also keep their Operating System updated and have a full-fledged security solution installed on all devices.</p>
<p>While organizations with appropriate spam filtering, proper system administration, and up-to-date Windows hosts have a much lower risk of infection, we further encourage organizations to validate the installation of the <a href="https://portal.msrc.microsoft.com/en-US/security-guidance/advisory/CVE-2017-11882" data-wpel-link="external" target="_blank" rel="nofollow">Microsoft patch for CVE 2017-11882</a>.</p>
<p>Quick Heal’s research team is proactively monitoring all campaigns targeting MSME’s and working relentlessly to ensure the safety of our customers</p>
