			<h2><strong>Summary:</strong></h2>
<p>We have been dealing with a new spyware for the past two months, named MassLogger. This advanced keylogger and spyware are distributed via MalSpam attachments and has more features than other present keylogger tools. It has been observed that this campaign is using several different file types as malicious attachments as an initial infection vector. Also, the dynamic behaviour of this camping is not constant across multiple samples. It comes with several functionalities like keylogger, Windows Defender exclusion, taking Screenshots, spreading via USB, clipboard stealing, VM detection, etc.</p>
<p><strong> </strong><strong>Technical Details:</strong></p>
<p>Here are different file types used as spam attachments in this campaign:</p>
<ul>
<li>zip</li>
<li>rar</li>
<li>gz</li>
<li>7z</li>
<li>img</li>
<li>iso</li>
<li>doc</li>
<li>arj</li>
<li>xz</li>
<li>ace</li>
<li>docm</li>
<li>z</li>
<li>xlsm</li>
<li>cab</li>
</ul>
<p><strong> </strong>After looking at the above list, we can see two major categories of attachment— first is archive file and second is a document file. In the case of archive files, there is .NET masslogger payload after extraction, while in the case of document file it contains VBA macro and exploit which downloads masslogger payload from a remote server.</p>

<p>“1C00” is the header of Equation Editor, in the right side, the shellcode is present containing cmd.exe initially renames the VB script and passes it to Wscript to execute that VB Script. After overflow occurs, this whole data is passed to WinExec function which does the further activity. For more info related to CVE-2017-1182 exploit, please refer our <a href="https://blogs.quickheal.com/malspam-campaigns-exploiting-recent-ms-office-vulnerability-cve-2017-11882/" data-wpel-link="internal" target="_self" rel="follow">blog</a> post.</p>
<p>To increase to chances of payload delivery, the attacker uses both exploit and VBA macros. When exploit fails on a patched system, another component, VBA macros are also present in the document file. The similar VBS code is present in VBA macros and macro code has the responsibility of dropping the VBS file in “C:\programdata\” folder and execute it as VBS Job which does further similar activity as that of the Equation Native exploit.</p>
<h2><strong>Payload Analysis:</strong></h2>
<p>The <a href="https://www.seqrite.com/blog/formbook-phishing-campaign-analysis/" data-wpel-link="internal" target="_self" rel="follow">payload</a> is downloaded from different initial attack vectors as discussed above when it executes and goes in sleep for a few seconds. There is a lot of sleep code present in this binary. There are a total of 4 components present with 2 layers of the packed file.</p>
<h2><strong>Stage 1 layer:</strong></h2>
<p>In the 1<sup>st</sup> layer, when it gets executed it has a simple code hidden in a Form() component. This code is responsible to extract a dll file from resource directory in present in reverse data in Base64 format which further gets resolved and dumps a dll with name AndroidStudio.dll.</p>
<figure id="attachment_4402" aria-describedby="caption-attachment-4402" style="width: 1366px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4402" src="https://www.seqrite.com/blog/wp-content/uploads/2020/07/10-2.png" alt="Fig 10: Fetch data from resources" width="1366" height="590" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/07/10-2.png 1366w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/10-2-300x130.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/10-2-768x332.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/10-2-1024x442.png 1024w" sizes="(max-width: 1366px) 100vw, 1366px" /><figcaption id="caption-attachment-4402" class="wp-caption-text"><strong>Fig 10: Fetch data from resources</strong></figcaption></figure>
<p>AndroidStudio.dll have a responsibility to decompress and decrypt a buffer which passes to it.</p>
<figure id="attachment_4403" aria-describedby="caption-attachment-4403" style="width: 1366px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="size-full wp-image-4403" src="https://www.seqrite.com/blog/wp-content/uploads/2020/07/11-1.png" alt="Fig 11: Android Studio code" width="1366" height="609" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/07/11-1.png 1366w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/11-1-300x134.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/11-1-768x342.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/11-1-1024x457.png 1024w" sizes="(max-width: 1366px) 100vw, 1366px" /><figcaption id="caption-attachment-4403" class="wp-caption-text"><strong>Fig 11: Android Studio code</strong></figcaption></figure>
<p>GZip decompression method is used to decompress the buffer passed from the resource directory. This dll is used to dump another PE file which is responsible for further activity.</p>
<h2><strong>Stage 2 Layer: Lazarus.exe</strong></h2>
<p>The Lazarus.exe gets dumped which is highly obfuscated .NET file which is now unpacked from the parent file. We have decoded this file using de4dot tool successfully. In execution, it goes in sleep for a few seconds, it checks if it’s own copy is present at “%appdata%” location. If not, it drops a self-copy at “%appdata%” location. After that, to stay persistent in the system, it creates an entry in task scheduler. For this, it creates and drops a.XML config file at “%temp%” location which is the input for creating task scheduler. The metadata for XML file is hardcoded and stored in PE resource. All data gets replaced at runtime.</p>
<figure id="attachment_4404" aria-describedby="caption-attachment-4404" style="width: 1364px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="size-full wp-image-4404" src="https://www.seqrite.com/blog/wp-content/uploads/2020/07/12-1.png" alt="Fig 12: Task Scheduler XML" width="1364" height="584" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/07/12-1.png 1364w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/12-1-300x128.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/12-1-768x329.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/12-1-1024x438.png 1024w" sizes="(max-width: 1364px) 100vw, 1364px" /><figcaption id="caption-attachment-4404" class="wp-caption-text"><strong>Fig 12: Task Scheduler XML</strong></figcaption></figure>
<p>The name of starts with string “Update\” followed by file name dropped at %appdata% location.</p>
<p>Following command gets executed to add an entry in task scheduler.</p>
<p><em>&#8220;C:\Windows\System32\schtasks.exe&#8221; /Create /TN &#8220;Updates\&lt;filename&gt;&#8221; /XML &#8220;C:\Users\&lt;username&gt;\AppData\Local\Temp\tmp&lt;USERID&gt;.tmp&#8221;</em></p>
<p>Now time to move to the final payload which is MassloggerBin.exe. Using Process Hollowing technique, it injects code into its own process. Following image shows the use of the self-hollowing technique to do its further activity.</p>
<figure id="attachment_4405" aria-describedby="caption-attachment-4405" style="width: 1366px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="size-full wp-image-4405" src="https://www.seqrite.com/blog/wp-content/uploads/2020/07/13-1.png" alt="Fig 13: Process Hollowing" width="1366" height="718" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/07/13-1.png 1366w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/13-1-300x158.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/13-1-768x404.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/13-1-1024x538.png 1024w" sizes="(max-width: 1366px) 100vw, 1366px" /><figcaption id="caption-attachment-4405" class="wp-caption-text"><strong>Fig 13: Process Hollowing</strong></figcaption></figure>
<p>When it successfully writes and creates a new process, the parent process gets terminated and code injected process runs as an orphan. The code of this process is also highly obfuscated. All function and class names are modified to random/obfuscated string.</p>
<h2><strong>Stage 3 layer: MassLoggerBin.exe </strong></h2>
<p>With the start, it extracts a dll file having name “Ionic.Zip.Reduced.dll” from its resources. The Ionic.Zip.Reduced.dll is a DotNetZip free fast class library used for manipulating zip files. The code used by the attacker in Masslogger is available on this <a href="https://archive.codeplex.com/?p=DotNetZip" data-wpel-link="external" target="_blank" rel="nofollow">site</a>. The main motive of using this dll is to create a zip file containing a compressed package of files like snapshots, keyloggers, user info etc.</p>
<p>The internal config-based functionality is used by MassLogger to fetch the required accordingly which is then assigned to a specific variable.</p>
<p>Following are the variables that fetch data stored in its internal config fig — by going to particular offset is the first parameter and the config array from where data gets fetched is the second parameter</p>
<figure id="attachment_4406" aria-describedby="caption-attachment-4406" style="width: 805px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4406" src="https://www.seqrite.com/blog/wp-content/uploads/2020/07/14.png" alt="Fig 14: Retrieves Config data" width="805" height="306" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/07/14.png 805w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/14-300x114.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/14-768x292.png 768w" sizes="(max-width: 805px) 100vw, 805px" /><figcaption id="caption-attachment-4406" class="wp-caption-text"><strong>Fig 14: Retrieves Config data</strong></figcaption></figure>
<figure id="attachment_4407" aria-describedby="caption-attachment-4407" style="width: 1289px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="size-full wp-image-4407" src="https://www.seqrite.com/blog/wp-content/uploads/2020/07/15.png" alt="Fig 15: Config Data" width="1289" height="247" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/07/15.png 1289w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/15-300x57.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/15-768x147.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/15-1024x196.png 1024w" sizes="(max-width: 1289px) 100vw, 1289px" /><figcaption id="caption-attachment-4407" class="wp-caption-text"><strong>Fig 15: Config Data</strong></figcaption></figure>
<p>It starts collecting system information like name of the system, Windows version, CPU, GPU, AV installed, Public IP which it gets from URL: “hxxp[:]//api[.]ipify[.]org”, also gets running process information.</p>
<figure id="attachment_4408" aria-describedby="caption-attachment-4408" style="width: 1366px" class="wp-caption alignnone"><img decoding="async" loading="lazy" class="size-full wp-image-4408" src="https://www.seqrite.com/blog/wp-content/uploads/2020/07/16.png" alt="Fig 16: Running Processes" width="1366" height="726" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/07/16.png 1366w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/16-300x159.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/16-768x408.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/16-1024x544.png 1024w" sizes="(max-width: 1366px) 100vw, 1366px" /><figcaption id="caption-attachment-4408" class="wp-caption-text"><strong>Fig 16: Running Processes</strong></figcaption></figure>
<p>MassLogger also stores a running process windows name in its log file.</p>
<h2><strong> </strong><strong>MassLogger Functionality:</strong></h2>
<p><strong>1. Application Data Stealer:</strong></p>
<p>Following are some list of applications where it tries to steal user data and which further sends to its C2 server.</p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-4409" src="https://www.seqrite.com/blog/wp-content/uploads/2020/07/17.png" alt="" width="914" height="42" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/07/17.png 914w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/17-300x14.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/17-768x35.png 768w" sizes="(max-width: 914px) 100vw, 914px" /></p>
<p>By checking data from hardcoded path stored in this binary, it checks for particular data and installation of these applications, if it does not find any details, it creates an entry in the following format,</p>
<p><em>&lt;|| Application-name ||&gt;</em></p>
<p><em>Not Installed</em></p>
<p>The following modules are present in MassLogger binary. Following is the list of that:</p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-4410" src="https://www.seqrite.com/blog/wp-content/uploads/2020/07/18.png" alt="" width="838" height="39" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/07/18.png 838w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/18-300x14.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/18-768x36.png 768w" sizes="(max-width: 838px) 100vw, 838px" />2. <strong>Windows Defender Exclusion</strong></p>
<p>It has a module named as “WD Exclusion” which is a Windows Defender Exclusion. Using command “<em>Add-MpPreference –ExclusionPath &lt;path&gt;</em>&#8220;, it exclude it-self from Windows Defender Anti-Virus.</p>
<p><strong>3. USB Spread</strong></p>
<p>Another module, USB Spread, it uses an open-source code of <a href="https://github.com/NYAN-x-CAT/LimeUSB-Csharp/blob/master/LimeUSB-Csharp/Resources/Source.cs" data-wpel-link="external" target="_blank" rel="nofollow">LimeUSB</a> available on GitHub. It is used to infect files stored on the USB drive. When files on USB gets executed, it executes its own code as well as infected code.</p>
<figure id="attachment_4411" aria-describedby="caption-attachment-4411" style="width: 614px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4411" src="https://www.seqrite.com/blog/wp-content/uploads/2020/07/19.png" alt="Fig 17: USB Spread Module" width="614" height="308" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/07/19.png 614w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/19-300x150.png 300w" sizes="(max-width: 614px) 100vw, 614px" /><figcaption id="caption-attachment-4411" class="wp-caption-text"><strong>Fig 17: USB Spread Module</strong></figcaption></figure>
<p><strong>4. Keylogger and Clipboard</strong></p>
<p>It has a key log capture module, using “SetWindowHookEx” api it captures all keyboard keys and logs it.</p>
<figure id="attachment_4412" aria-describedby="caption-attachment-4412" style="width: 1366px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4412" src="https://www.seqrite.com/blog/wp-content/uploads/2020/07/20.png" alt="Fig 18: Keyboard Hooking" width="1366" height="595" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/07/20.png 1366w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/20-300x131.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/20-768x335.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/20-1024x446.png 1024w" sizes="(max-width: 1366px) 100vw, 1366px" /><figcaption id="caption-attachment-4412" class="wp-caption-text"><strong>Fig 18: Keyboard Hooking</strong></figcaption></figure>
<p><strong>5. Anti VM</strong></p>
<p>It also has Anti-VM techniques by checking for Video_Controller adapter using WMI “<em>Select * from Win32_VideoController</em>” which retrieves which information related to the graphics card. If the process is executing on Virtual Box then it returns “Virtual Box Graphics Adapter”.</p>
<figure id="attachment_4413" aria-describedby="caption-attachment-4413" style="width: 1362px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4413" src="https://www.seqrite.com/blog/wp-content/uploads/2020/07/21.png" alt="Fig 19: Video Adapter" width="1362" height="587" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/07/21.png 1362w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/21-300x129.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/21-768x331.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/21-1024x441.png 1024w" sizes="(max-width: 1362px) 100vw, 1362px" /><figcaption id="caption-attachment-4413" class="wp-caption-text"><strong>Fig 19: Video Adapter</strong></figcaption></figure>
<p><strong>6. Search And Upload</strong></p>
<p>As per config file, it searches for some file which it wants to send to the C2 server that stores in “SearchAndUpload.zip” archive.</p>
<p>All data is stored and retrieved from its config file. Following is the view of MassLogger config file.</p>
<figure id="attachment_4414" aria-describedby="caption-attachment-4414" style="width: 1252px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4414" src="https://www.seqrite.com/blog/wp-content/uploads/2020/07/22.png" alt="Fig 20: Config File" width="1252" height="326" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/07/22.png 1252w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/22-300x78.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/22-768x200.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/22-1024x267.png 1024w" sizes="(max-width: 1252px) 100vw, 1252px" /><figcaption id="caption-attachment-4414" class="wp-caption-text"><strong>Fig 20: Config File</strong></figcaption></figure>
<p>Once all data collection is done, it creates a log file containing all data like when Masslogger Process is started and ended and other collected details. After that, it compresses using ZIP and gets stored at the location “C:\Users\&lt;USERNAME&gt;\AppData\Local”.</p>
<p>Following is an image showing MassLogger log file.</p>
<figure id="attachment_4415" aria-describedby="caption-attachment-4415" style="width: 971px" class="wp-caption aligncenter"><img decoding="async" loading="lazy" class="size-full wp-image-4415" src="https://www.seqrite.com/blog/wp-content/uploads/2020/07/23.png" alt="Fig 21: MassLogger log file" width="971" height="348" srcset="https://www.seqrite.com/blog/wp-content/uploads/2020/07/23.png 971w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/23-300x108.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2020/07/23-768x275.png 768w" sizes="(max-width: 971px) 100vw, 971px" /><figcaption id="caption-attachment-4415" class="wp-caption-text">Fig 21: MassLogger log file</figcaption></figure>
<p><strong>Conclusion:</strong></p>
<p>Masslogger is a highly configurable and modular keylogger and spyware. The author behind Masslogger tried to make it more sophisticated in features than other keyloggers, these features make it hard to detect this advanced malware.</p>
