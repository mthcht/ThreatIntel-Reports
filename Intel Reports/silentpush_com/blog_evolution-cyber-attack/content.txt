					<h1 class="entry-title">Investigating Crimeware Name Servers</h1>                    <div class='ep-post-terms term-tag'><a class='ep-item-term ep-item-term-threat' href='https://www.silentpush.com/tag/threat/' >threat</a></div>					<div class="single-post-meta">
												<div class="single-post-date">
							May 28, 2019						</div>
					</div>

												<div class="single-post-thumbnail "><img width="1920" height="1080" src="https://www.silentpush.com/wp-content/uploads/Crimeware.png" class="wp-image-4202 avia-img-lazy-loading-4202 attachment-post-thumbnail size-post-thumbnail wp-post-image" alt="Crimeware Cyber Attack" decoding="async" loading="lazy" /></div>
						
				</div>
				
<p class="has-medium-font-size">Very few security products and services give enough consideration to the reputation and quality of the name server associated with the domains they are looking at.</p>



<h2 class="wp-block-heading has-large-font-size" id="h-example">Example</h2>



<p class="has-medium-font-size">In this example, we pick a High Value Suspicious Domain and check what else is on the same name server to examine what is discovered through the process.</p>



<p class="has-medium-font-size">Our seeding domain will be service-update[.]link &nbsp;<br>Looking at the name server info from our API</p>



<p class="has-medium-font-size">&#8220;domain&#8221;: &#8220;service-update[.]link&#8221;,</p>



<p class="has-medium-font-size">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &#8220;ns_avg_ttl&#8221;: 4149,</p>



<p class="has-medium-font-size">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &#8220;ns_domain&#8221;: &#8220;dendrite.network&#8221;,</p>



<p class="has-medium-font-size">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &#8220;num_domains&#8221;: 203,</p>



<p class="has-medium-font-size">This is a low density name server where we have only seen 203 domains on it. It has a very low average TTL across all the domains on there which may suggest a lot of changes compared to other name servers.</p>



<p class="has-medium-font-size">A quick lookup using our explore API gives us a number of suspicious domains with matching characteristics. These appear to be in two groups.</p>



<h2 class="wp-block-heading has-medium-font-size" id="h-group-1">Group 1:</h2>



<p class="has-medium-font-size">Commonalities- Tech related theme. Nameserver domain- dendrite.network. Have an A record-AS holder ColoCrossing- Registrar Namesilo- &#8220;address&#8221;: &#8220;Tavernier St., Wall House&#8221;,</p>



<p class="has-medium-font-size">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &#8220;city&#8221;: &#8220;Loubiere&#8221;,</p>



<p class="has-medium-font-size">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &#8220;country&#8221;: &#8220;DM&#8221;,</p>



<figure class="wp-block-table"><table><tbody><tr><td>Domain</td><td>PTR Record</td><td>Potential Target</td><td>Registrar</td></tr><tr><td>update-support[.]network</td><td>*colocrossing.com</td><td>3 Mobile UK</td><td>Registrar</td></tr><tr><td>service-update[.]link</td><td>*colocrossing.com</td><td>Live soon</td><td>Registrar</td></tr><tr><td>hs-securealerts[.]com</td><td>*colocrossing.com</td><td>HSBC</td><td>Registrar</td></tr><tr><td>ref0948a[.]com</td><td>*colocrossing.com</td><td>unknown</td><td>Registrar</td></tr><tr><td>ref0948a[.]com</td><td>*colocrossing.com</td><td>HMRC</td><td>Registrar</td></tr><tr><td>aem-new[.]com</td><td>*colocrossing.com</td><td>3 Mobile UK</td><td>Registrar</td></tr><tr><td>aempath[.]com</td><td>*colocrossing.com</td><td>3 Mobile UK</td><td>Registrar</td></tr><tr><td>com-gb[.]mobi</td><td>*colocrossing.com</td><td>Mobile UK</td><td>Registrar</td></tr></tbody></table></figure>



<p class="has-medium-font-size">It appears that Group 1 have either been used maliciously already or are waiting to be used maliciously. Monitoring the “waiting” group for changes is key as this may lead to being able to block their activity once the domain moves into an actively malicious mode. There are a number of ways to do this, such as monitoring for new changes in DNS or associated records. Moving of infrastructure can often mark an activation of a malicious domain. A good example of this is when the domain avsvmcloud[.]com was activated for the Solarwinds breach, it switched to its own name servers for the active part of the campaign.</p>



<h2 class="wp-block-heading has-medium-font-size" id="h-group-2">Group 2:</h2>



<p class="has-medium-font-size">This group seems to lead to more and more indicators so I’ll post them over time.</p>



<p class="has-medium-font-size">Commonalities- Tech related theme. Nameserver domain- dendrite.network. Domains have been aged. Have an A record-AS holder Nice IT Services Group Inc.</p>



<p class="has-medium-font-size">&#8211; Registrar Namesilo- &#8220;address&#8221;: &#8220;Tavernier St., Wall House&#8221;,</p>



<p class="has-medium-font-size">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &#8220;city&#8221;: &#8220;Loubiere&#8221;&#8221;country&#8221;: &#8220;DM</p>



<figure class="wp-block-table"><table><tbody><tr><td>Domain</td><td>AS Name of IP</td><td>Potential Target</td><td>Registrar</td></tr><tr><td>Paypalservice[.]support</td><td>Nice IT Services</td><td>Paypal</td><td>Namesilo</td></tr><tr><td>small-url[.]cc</td><td>Nice IT Services</td><td>adobe and others</td><td>Namesilo</td></tr><tr><td>election[.]finance</td><td>Nice IT Services</td><td>malware download</td><td>Namesilo</td></tr><tr><td>Ulsterbankonlineltd[.]com</td><td>Nice IT Services</td><td>Ulster Bank(RBS)</td><td>Namesilo</td></tr><tr><td>Choicebank[.]online</td><td>Nice IT Services</td><td>Choice Bank/First Choice</td><td>Namesilo</td></tr><tr><td>Documentcloud[.]pw</td><td>Nice IT Services</td><td>adobe and others</td><td>Namesilo</td></tr><tr><td>rbscotland-online[.]com</td><td>Nice IT Services</td><td>Royal Bank of Scotland</td><td>Namesilo</td></tr><tr><td>Btctools[.]net</td><td>Nice IT Services</td><td>Bitcoin Wallet stealer</td><td>Namesilo</td></tr><tr><td>gb-kpmg[.]com</td><td>Nice IT Services</td><td>KPMG</td><td>Namesilo</td></tr><tr><td>secure-id[.]cloud</td><td>Nice IT Services</td><td>Secure ID</td><td>Namesilo</td></tr><tr><td>service-ca-verification[.]com</td><td>Nice IT Services</td><td>Flagged as spammer</td><td>Namesilo</td></tr><tr><td>Teamtnt[.]red</td><td>Nice IT Services</td><td>Malware Downloads</td><td>Namesilo</td></tr></tbody></table></figure>



<p class="has-medium-font-size">So just touching on these two groups based on similarities, these of course could be one group, this name server is very heavily used by malicious actors and definitely one threat actor group called Team TNT.</p>



<p class="has-medium-font-size">How can I use this information?</p>



<p class="has-medium-font-size">All the information above was collected by our API and can be leveraged for threat hunting or detections. The information is pre-collected and cached so new lookups don’t have to be done each time you have a new indicator to check. We’ve already collected all this information and run some analysis on it to give things like reputation scores for the name server, the AS number reputation, the subnet reputation etc.</p>



<p class="has-medium-font-size">A security team can use Yara rules over this information to try and find “High Value Malicious Domains “ in their logs or associated IP addresses.</p>



<p class="has-medium-font-size">We also have threat hunting API endpoints that gather behavioral clusters for you so you can quickly create your own new intelligence, or we can do it for you.</p>
