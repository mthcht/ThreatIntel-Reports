					<h1 class="entry-title">IcedID Command and Control Infrastructure</h1>                    <div class='ep-post-terms term-tag'><a class='ep-item-term ep-item-term-threat' href='https://www.silentpush.com/tag/threat/' >threat</a></div>					<div class="single-post-meta">
												<div class="single-post-date">
							April 25, 2021						</div>
					</div>

												<div class="single-post-thumbnail "><img width="1920" height="1080" src="https://www.silentpush.com/wp-content/uploads/5-4.png" class="wp-image-4209 avia-img-lazy-loading-4209 attachment-post-thumbnail size-post-thumbnail wp-post-image" alt="A frozen lake with the ice breaking apart" decoding="async" loading="lazy" /></div>
						
				</div>
				
<p class="has-medium-font-size">Earlier this week, the DFIR Report published an <a href="https://thedfirreport.com/2021/03/29/sodinokibi-aka-revil-ransomware/">interesting analysis</a> of an intrusion with the notorious SodinokibiREvil ransomware. The intrusion used IcedID as the initial access broker: many ransomware actors use another malware campaign to gain access to an internal network and IcedID has become a very popular choice for that.</p>



<p class="has-medium-font-size">This blog post demonstrates how the IOCs shared by the DFIR Report can uncover more command and control infrastructure linked to IcedID, some of which has not been published before.</p>



<p class="has-medium-font-size">IcedID, also known as Bokbot, was <a href="https://securityintelligence.com/new-banking-trojan-icedid-discovered-by-ibm-x-force-research/">discovered</a> by IBM X-Force in November 2017. Initially operating as a banking trojan, it has since made the same move that Emotet had made previously and is now used to serve a foothold within a network. This is then later used by a ransomware operation.</p>



<p class="has-medium-font-size">The DFIR Report’s analysis lists cikawemoret34[.]space and nomovee[.]website as IcedID command and control servers used during the intrusion. These domains were hosted on the IP addresses 206.189.10[.]247 and 161.35.109[.]168 respectively.</p>



<p class="has-medium-font-size">It is always a good idea to see what other domains were hosted on these IP addresses. Using Silent Push passive DNS data, on 206.189.10[.]247, Martijn also found the following domains:</p>



<pre class="wp-block-code has-medium-font-size"><code>33nachoscocso&#91;.]website
berxion9&#91;.]online
chinavillage&#91;.]uno
emanielepolikutuo1&#91;.]website
gommadrilla&#91;.]space
oskolko&#91;.]uno
prolomstenn&#91;.]fun</code></pre>



<p class="has-medium-font-size">While on 161.35.109[.]168, Martijn found:</p>



<pre class="wp-block-code has-medium-font-size"><code>aspergerr&#91;.]top
kneelklil&#91;.]uno
newstationcosmo8&#91;.]space</code></pre>



<p class="has-medium-font-size">Unsurprisingly, most of these domains have been publicly linked to IcedID.</p>



<p class="has-medium-font-size">All the domains were registered through Porkbun in February or March and parked there initially before switching to Cloudflare’s name servers and pointing to the aforementioned IP addresses. This switching happened at different times for different domains, suggesting that the switch was made just before a domain was used in a campaign.</p>



<p class="has-medium-font-size">One domain stands out: </p>



<p class="has-medium-font-size">emanielepolikutuo1[.]</p>



<p class="has-medium-font-size">This website first switched to using name servers belonging to Russia’s Server Space and pointing to the IP address 143.198.25[.]214, before switching to Cloudflare and 206.189.10[.]247 a little over a week later.</p>



<p class="has-medium-font-size">So, looking at 143.198.25[.]214, the following domains hosted there can be found:</p>



<pre class="wp-block-code has-medium-font-size"><code>apouvtios2&#91;.]uno
awefoplou5&#91;.]site
chajkovsky&#91;.]space
daserwewlollipop&#91;.]club
dastemodaste&#91;.]fun
emanielepolikutuo1&#91;.]website
ohbluebennihill&#91;.]website
seconwowa&#91;.]cyou
violonchelistto&#91;.]space
zomonedu3&#91;.]website</code></pre>



<p class="has-medium-font-size">All but one of these domains were registered at Porkbun, the exception is the slightly older seconwowa[.]cyou, which was registered through NameSilo. &nbsp;</p>



<p class="has-medium-font-size">Just like the previous set of domains, all these domains switched to using Cloudflare’s nameservers at some point and switched IP addresses at the same time. However, some first pointed to 83.97.20[.]176 before pointing to 143.198.25[.]214. On the former IP addresses, four more domains were found:</p>



<pre class="wp-block-code has-medium-font-size"><code>ameripermanentno&#91;.]website
mazzappa&#91;.]fun
odichaly&#91;.]space
vaccnavalcod&#91;.]website</code></pre>



<p class="has-medium-font-size">Again, these used same pattern of registering at Porkbun before switching to Cloudflare’s name servers and the above IP address.</p>



<p class="has-medium-font-size">Of the latter two lists of domains, only some have been publicly linked to IcedID activity. However, the similarities noted above, as well as the choice of TLDs, suggest these domains belong to the same infrastructure and either have been or will be used in IcedID campaigns.</p>



<p class="has-medium-font-size">There is a pattern there: a domain gets registered, usually at Porkbun, and parked there for a while before its name servers switch to those of Cloudflare when the domain points to a new IP address. This IP address hosts multiple of these domains. There is also a preference for slightly unusual top-level domains.</p>



<p class="has-medium-font-size">Using this pattern, one can dig into the Silent Push data trove to look for other domains that satisfied this pattern. After sifting through the results to filter out false positives, the analyst ends up with a list of domain names and corresponding IP addresses of which he considered very likely to belong to IcedID’s infrastructure.</p>



<p class="has-medium-font-size">Many of these indicators have been published previously, for example on Maltrail’s <a href="https://github.com/stamparm/maltrail/blob/master/trails/static/malware/icedid.txt">GitHub</a>, but many others have not been publicly linked to IcedID before.</p>



<p class="has-medium-font-size">You can find the full list of 58 IP addresses and 323 domain names (and 402 combinations: some domain names have pointed to multiple IP addresses) on our <a href="https://github.com/Silent-Push/IOCs/blob/main/icedid_c2.csv">GitHub page</a>. &nbsp;</p>



<h2 class="wp-block-heading has-large-font-size">Conclusion</h2>



<p class="has-medium-font-size">Malware like IcedID plays a crucial role in many large cybercrime campaigns, including ransomware, which can be very costly for the victim organization. Early knowledge of indicators is thus important, even if these indicators haven’t all been publicly linked to the malware. This blog post demonstrated how to find hundreds of such indicators by spotting some patterns in the domain behaviour. </p>



<p class="has-medium-font-size"><em>Thank you to John Jensen and Ken Bagnall for their contributions.</em></p>
