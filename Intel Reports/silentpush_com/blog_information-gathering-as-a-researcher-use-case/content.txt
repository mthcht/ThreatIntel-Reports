					<h1 class="entry-title">Information Gathering as a Researcher: a use case</h1>                    <div class='ep-post-terms term-tag'><a class='ep-item-term ep-item-term-platform' href='https://www.silentpush.com/tag/platform/' >platform</a></div>					<div class="single-post-meta">
												<div class="single-post-date">
							April 29, 2021						</div>
					</div>

												<div class="single-post-thumbnail "><img width="1200" height="816" src="https://www.silentpush.com/wp-content/uploads/Silent-Push-Information-Gathering-As-A-Researcher-Blog.jpg" class="wp-image-1564 avia-img-lazy-loading-1564 attachment-post-thumbnail size-post-thumbnail wp-post-image" alt="Military professionals working in a control room" decoding="async" loading="lazy" /></div>
						
				</div>
				
<p class="has-medium-font-size">A penetration tester was asked to try the Silent Push service to see how it could help him and his team to get their work done quicker. This is written in the tester&#8217;s own words and only uses the Silent Push &#8216;DNS Explore&#8217; feature.</p>



<h2 class="wp-block-heading has-large-font-size">Reconnaissance</h2>



<p class="has-medium-font-size">Reconnaissance is performed to gain as much information on the target before beginning the penetration testing. ‘Recon’ is an essential element of any penetration testing. Recon on a target can be done in two ways: passive and active reconnaissance.</p>



<p class="has-medium-font-size">During the recon process researchers try to collect information about the subdomains associated with the target and their respective IP address. Most applications today are protected using WAFs and CDNs and it is often challenging to identify the real IP address associated with an application. That is where the subdomains associated with the application help researchers get more information about the main application and expand the attack surface.</p>



<p class="has-medium-font-size">The Silent Push application can be used for passive reconnaissance quickly.</p>



<h2 class="wp-block-heading has-large-font-size"><strong>Case Study:</strong></h2>



<p class="has-medium-font-size">Domain : magicbricks[.]com</p>



<p class="has-medium-font-size">For the past few years, the tester has been testing web applications and spent around 2 to 5 days on collecting the information about each target. The information that he collected included all the domains that are associated with the company, their respective subdomains and IP addresses, as well as the information about the OS.</p>



<p class="has-medium-font-size">There are different search engines available for collections of the above information but there is no single place where more information can be found at any one time.</p>



<p class="has-medium-font-size">The tester worked on the application mentioned above a couple of months back and was not able to collect more information. Then, he gave it a try with Silent Push and the information was gathered in just a few minutes. Normally, it would take the tester several days to get that information from different sources.</p>



<p class="has-medium-font-size"><strong>1. The tester started by using their Explore DNS feature which accepts wildcards:</strong></p>



<figure class="wp-block-image size-full" id="yui_3_17_2_1_1701969469232_96"><img loading="lazy" decoding="async" width="1736" height="696" src="https://www.silentpush.com/wp-content/uploads/Screenshot2021-04-29at14.08.01.png" alt="" class="wp-image-2217"/><figcaption class="wp-element-caption"><em>Explore DNS history using wildcards proved very powerful</em></figcaption></figure>



<p class="has-medium-font-size"><strong>2. The tester then searched for any records associated with the test domain:</strong></p>



<figure class="wp-block-image size-full" id="yui_3_17_2_1_1701969469232_115"><img loading="lazy" decoding="async" width="1804" height="776" src="https://www.silentpush.com/wp-content/uploads/Screenshot2021-04-29at14.10.48.png" alt="" class="wp-image-2218"/></figure>



<figure class="wp-block-image size-full" id="yui_3_17_2_1_1701969469232_124"><img loading="lazy" decoding="async" width="2264" height="1334" src="https://www.silentpush.com/wp-content/uploads/Screenshot2021-04-29at14.13.34.png" alt="" class="wp-image-2219"/><figcaption class="wp-element-caption"><em>Gathering all the DNS information in one place using the explore feature</em></figcaption></figure>



<p class="has-medium-font-size"><strong>3.</strong> <strong>The tester then realized he could use a wildcard and gather subdomains and see what CNAME records were gathered and what IPs subdomains were using.</strong></p>



<figure class="wp-block-image size-full" id="yui_3_17_2_1_1701969469232_133"><img loading="lazy" decoding="async" width="2238" height="1324" src="https://www.silentpush.com/wp-content/uploads/Screenshot2021-04-29at14.17.32.png" alt="" class="wp-image-2220"/><figcaption class="wp-element-caption"><em>Gathering all subdomain info using a wildcard</em></figcaption></figure>



<p class="has-medium-font-size"><strong>4. This allowed the tester to pivot off this information and see what else was pointed to the same infrastructure</strong>:</p>



<figure class="wp-block-image size-full" id="yui_3_17_2_1_1701969469232_142"><img loading="lazy" decoding="async" width="2152" height="1246" src="https://www.silentpush.com/wp-content/uploads/pivot_in_passiveDNS.png" alt="" class="wp-image-2222"/></figure>



<figure class="wp-block-image size-full" id="yui_3_17_2_1_1701969469232_151"><img loading="lazy" decoding="async" width="2194" height="1284" src="https://www.silentpush.com/wp-content/uploads/whatdomainspointatmyipaddress.png" alt="" class="wp-image-2224"/><figcaption class="wp-element-caption"><em>Mahesh could see all A records pointing to the same IP straight away</em></figcaption></figure>



<p class="has-medium-font-size"><strong>5. The tester could enrich that information to find out more about ‘the neighbours’ and see what sort of reputation was associated with them.</strong></p>



<figure class="wp-block-image size-full" id="yui_3_17_2_1_1701969469232_160"><img loading="lazy" decoding="async" width="2262" height="1558" src="https://www.silentpush.com/wp-content/uploads/Screenshot2021-04-29at14.26.42.png" alt="" class="wp-image-2225"/><figcaption class="wp-element-caption"><em>This looked like a clean domain</em></figcaption></figure>



<h2 class="wp-block-heading has-large-font-size">Conclusion</h2>



<p class="has-medium-font-size">Even though the tester was not on a threat intelligence team, the simple data gathering capabilities of this part of the Silent Push application saved him enormous amounts of time. Quite literally, this saves the tester <em>days</em> per job. The use cases across entire security teams is tremendous.</p>
