<p>Continuing on some past research on <a href="https://www.sneakymonkey.net/2019/05/22/trickbot-analysis/">TRICKBOT's arsenal of modules</a>, I knew that there was a PowerShell <strong>EMPIRE</strong> module <code>NewBCtestDll64 </code> but never saw it ITW (in-the-wild) myself.</p>
<blockquote class="twitter-tweet"><p lang="en" dir="ltr">2018-10-08 - Quick post: <a href="https://twitter.com/hashtag/Trickbot?src=hash&amp;ref_src=twsrc%5Etfw">#Trickbot</a> gtag sat75 infection with <a href="https://twitter.com/hashtag/PowershellEmpire?src=hash&amp;ref_src=twsrc%5Etfw">#PowershellEmpire</a> traffic - <a href="https://t.co/sU86nZJnh2">https://t.co/sU86nZJnh2</a> - part of ongoing US-based Paypal-themed Trickbot <a href="https://twitter.com/hashtag/malspam?src=hash&amp;ref_src=twsrc%5Etfw">#malspam</a> campaign. Powershell Empire traffic seems tied to the NewBCtestDll64 module (and probably the 32 bit version <a href="https://t.co/A5m64aIRd7">pic.twitter.com/A5m64aIRd7</a></p>&mdash; Brad (@malware_traffic) <a href="https://twitter.com/malware_traffic/status/1049898741229064193?ref_src=twsrc%5Etfw">October 10, 2018</a></blockquote> <script async src="https://platform.twitter.com/widgets.js" charset="utf-8"></script>
<p>This soon escalated to <strong>COBALT-STRIKE</strong> connectivity and <strong>BLOODHOUND</strong> reconnaissance.</p>
<p><img src="https://sneakymonkey.net/content/images/2019/10/giphy.gif" alt="giphy" loading="lazy"></p>
<p>In this observed activity, EMPIRE was used for reconnaissance and privledge escalation and COBALT-STRIKE for delivering further recon via the means of BLOODHOUND - both tools attempted credential dumping with MIMIKATZ. The endgame here, is for the adversary to priv esc all the way to Domain Admin for full domain compromise to then deliver one of the many ransomware variants such as RYUK [2] üí£üí£üí£üí•üí≤üí≤üí≤.</p>
<h1 id="recentnotesonempire">RECENT NOTES ON EMPIRE</h1>
<p>For those that do not know, PowerShell EMPIRE is a post exploitation framework written in PowerShell. This project has recently retired due to the heighten uptake in PowerShell visibility over the last few years, the project has stated it has reached its goal and has ended support.</p>
<blockquote class="twitter-tweet"><p lang="en" dir="ltr">PSA for Empire development: The original objective of the Empire project was to demonstrate the post-exploitation capabilities of PowerShell and bring awareness to PowerShell attacks used by (at the time) more advanced adversaries.</p>&mdash; Chris (@xorrior) <a href="https://twitter.com/xorrior/status/1156626181107736576?ref_src=twsrc%5Etfw">July 31, 2019</a></blockquote> <script async src="https://platform.twitter.com/widgets.js" charset="utf-8"></script>
<p>Although EMPIRE is now in retirement it is still being used ITW (in-the-wild). It still works, just not supported. I'd expect an uptake on other C2 post exploitation frameworks many such are listed here <a href="https://github.com/infosecn1nja/Red-Teaming-Toolkit">Remote Access Tools</a></p>

<h1 id="hellopowershellempire">HELLO POWERSHELL EMPIRE</h1>
<p>To start off we identify the newly established EMPIRE connectivity.</p>
<blockquote>
<p>The initial &quot;stager&quot; is the way the victim talks back to the EMPIRE C2 that is listening for the connection to then download stage 2 which is the EMPIRE agent. The default launcher/stager is a PowerShell Base64 encoded/obsfucated command.</p>
</blockquote>
<p>By capturing the PowerShell activity on our box (PowerShell Logging, Command Line audit logging EID 4688), decoding and identifying the EMPIRE stager wasn't too difficult due to the fact most if not all the EMPIRE defaults were left the same. CyberChef EMPIRE stager reciped used is available <a href="https://gchq.github.io/CyberChef/#recipe=From_Base64('A-Za-z0-9%2B/%3D',true)Remove_null_bytes()Generic_Code_Beautify()Syntax_highlighter('powershell')">here</a>.</p>
<h2 id="empirestager">EMPIRE Stager</h2>
<p><img src="https://sneakymonkey.net/content/images/2019/10/empire_callout.png" alt="empire_callout" loading="lazy"></p>
<p>On decoding this Base64 blob of data, the key items to look for are the default settings for an EMPIRE stager as documented in the official EMPIRE Github repo. These defaults are;</p>
<p>User-agent:<br>
<code>Mozilla/5.0 (Windows NT 6.1; WOW64; Trident/7.0; rv:11.0) like Gecko</code><br>
Session Cookie field:<br>
<code>&quot;Cookie&quot;, &quot;session=XXXXXXXXXXXXXX&quot;</code><br>
On of the following URLs:<br>
<code>/login/process.php</code><br>
<code>/admin/get.php</code><br>
<code>/admin/news.php</code></p>
<p>You can see the decoded result and highlighted fields<br>
<img src="https://sneakymonkey.net/content/images/2019/10/empirestager-1.png" alt="empirestager-1" loading="lazy"></p>
<p>We can see that the values were left as defaults as per <a href="https://github.com/EmpireProject/Empire/blob/master/data/agent/agent.py#L43">Github EMPIRE Repo</a>. For reference, the default user agent string and URLs for EMPIRE.</p>
<p><img src="https://sneakymonkey.net/content/images/2019/10/code.jpg" alt="code" loading="lazy"></p>
<h1 id="stepinpowersploit">STEP IN POWERSPLOIT</h1>
<p>Once the EMPIRE connection is established we see plenty of follow up POWERSPLOIT activity.</p>
<blockquote>
<p>PowerSploit is a collection of Microsoft PowerShell modules that can be used to aid penetration testers during all phases of an assessment. PowerSploit is comprised of the following modules: CodeExecution, ScriptModification, Persistence, AntivirusBypass, Exfiltration, Mayhem, Privesc, Recon.</p>
</blockquote>
<p>Simply put, the threat actor starts profiling or gathering a siutational awareness of their new environment they have landed in.</p>
<pre><code class="language-powershell">Get-NetComputer  | Out-String | %{$_ + &quot;`n&quot;};&quot;`nGet-NetComputer completed!&quot;
</code></pre>
<pre><code class="language-powershell">Get-NetComputer  -OperatingSystem *server* | Out-String | %{$_ + &quot;`n&quot;};&quot;`nGet-NetComputer completed!&quot;
</code></pre>
<pre><code class="language-powershell">Get-NetDomainTrust  | Out-String | %{$_ + &quot;`n&quot;};&quot;`nGet-NetDomainTrust completed!&quot;
</code></pre>
<pre><code class="language-powershell">Get-NetDomainController  | Out-String | %{$_ + &quot;`n&quot;};&quot;`nGet-NetDomainController completed!&quot;
</code></pre>
<pre><code class="language-powershell">Invoke-MapDomainTrust | ConvertTo-Csv -NoTypeInformation | Out-String | %{$_ + &quot;`n&quot;};&quot;`nInvoke-MapDomainTrust completed!&quot;
</code></pre>
<p>From the manual - descriptions of each.</p>
<ul>
<li>Get-NetComputer -  gets a list of all current servers in the domain</li>
<li>Get-NetDomainTrust -  gets all trusts for the current user's domain</li>
<li>Get-NetForestTrust -  gets all trusts for the forest associated with the current user's domain</li>
<li>Invoke-MapDomainTrust -   try to build a relational mapping of all domain trusts</li>
<li>Get-NetDomainController -  gets the domain controllers for the current computer's domain</li>
</ul>
<p>Shortly after followed by an attempted credential dump using POWERSPLOIT's <code>Invoke-Mimikatz -DumpCreds</code> which actually failed in this environment.</p>
<blockquote>
<p>&quot;VirtualAlloc failed to allocate memory for PE. If PE is not ASLR compatible, try running the script in a new PowerShell process (the new PowerShell process will have a different memory layout, so the address the PE wants might be free).&quot;</p>
</blockquote>
<p>MIMIKATZ within EMPIRE v2.1.1 20171106 / POWERSPLOIT v2.0 alpha seems problematic with the newer updates of Windows 10. MIMIKATZ at the time of writing is at version v2.2.0-20190813. These frameworks are using an outdated version. (Cheers DP -  REDTEAM FRIEND üëçüëç)</p>
<h1 id="stepincobaltstrikeandbloodhound">STEP IN COBALTSTRIKE AND BLOODHOUND üê∂</h1>
<p>A quick background for those not in the know.</p>
<blockquote>
<p>Cobalt Strike is software for Adversary Simulations and Red Team Operations...Cobalt Strike gives you a post-exploitation agent and covert channels to emulate a quiet long-term embedded actor in your customer's network.<br>
<a href="https://www.cobaltstrike.com/">https://www.cobaltstrike.com/</a></p>
</blockquote>
<blockquote>
<p>BloodHound uses graph theory to reveal the hidden and often unintended relationships within an Active Directory environment. Attackers can use BloodHound to easily identify highly complex attack paths that would otherwise be impossible to quickly identify<br>
<a href="https://github.com/BloodHoundAD/BloodHound">https://github.com/BloodHoundAD/BloodHound</a></p>
</blockquote>
<p>I was originally suprised about the use of <code>Invoke-BloodHound</code> at first which is from the default <a href="https://github.com/BloodHoundAD/BloodHound/blob/master/Ingestors/SharpHound.ps1">BLOODHOUND ingester SharpHound</a>  - ingester = data gatherer. SharpHound.ps1 - Runs the BloodHound C# Ingestor using reflection. With this little Bloodhound 101 first up was the CS stager and connectivity.</p>
<p>The following code kicks off a COBALT-STRIKE 'beacon stager'. This hosted stager, uses @MrUn1k0d3r's &quot;DONT KILL MY CAT&quot; (<a href="https://github.com/Mr-Un1k0d3r/DKMC">DKMC</a>) üôÄ which obfuscates the shellcode to avoid detection when executed on the endpoint. This DKMC 'template' gives away the use of COBALT-STRIKE.</p>

<code>User-Agent: Mozilla/5.0 (compatible; MSIE 9.0; Windows NT 6.1; WOW64; Trident/5.0; FunWebProducts</code><br>
<code>185.147.14[.]242</code></p>
<p>Once an active connection was made to the infected victim, a follow up series of recon commands were carried out via <code>Get-NetLoggedon</code>, <code>Get-NetComputer  | Out-String | %{$_ + &quot;`n&quot;};&quot;`nGet-NetComputer completed!&quot;</code> and another attempt to dump credentials was made with <code>Invoke-Mimikatz</code>. Lastly we see the domain recon via BLOODHOUND</p>
<pre><code class="language-powershell">Set-Alias Get-BloodHoundData Invoke-BloodHound
Invoke-BloodHound -Threads 20 -CollectionMethod Default -Throttle 1000 -CSVFolder $(Get-Location) | Out-String | %{$_ + &quot;`n&quot;};&quot;`nInvoke-BloodHound completed!&quot;
</code></pre>

<h1 id="othertrickbotobservationsesentutledbs">OTHER TRICKBOT OBSERVATIONS - ESENTUTL? .EDBs?</h1>
<p>One other thing I noted with this TRICKBOT infection was the use of ESENT UTIL to gather IE and Explorer browser history and webcache. The below command was seen;</p>
<pre><code class="language-powershell">esentutl /p /o C:\Users\USER\AppData\Local\Temp\grabber_temp.edb
</code></pre>
<blockquote>
<p>&quot;a centralized meta-data store for the browser using the proven &quot;JET Blue&quot; Extensible Storage Engine (ESE) database format&quot;</p>
</blockquote>
<blockquote>
<p>&quot;Remember that even if a user never opens Internet Explorer, there may still be valuable records in their IE database including files opened on the local system, network shares, and removable devices&quot;<br>
<a href="https://digital-forensics.sans.org/blog/2015/06/03/ese-databases-are-dirty/">https://digital-forensics.sans.org/blog/2015/06/03/ese-databases-are-dirty/</a></p>
</blockquote>
<blockquote>
<p>&quot;browser history data and while Chrome and Firefox allow copying of the history files, the WebCacheV01.dat file that IE and Edge history are stored in is a locked file and cannot be copied using native copy&quot;<br>
<a href="https://dfironthemountain.wordpress.com/tag/ese-database/">https://dfironthemountain.wordpress.com/tag/ese-database/</a></p>
</blockquote>
<p>Artefacts on disk will be in the form of a .RAW capture</p>

<p>You can download, <a href="https://www.nirsoft.net/utils/ese_database_view.html">NIRSOFT ESE Viewer</a> to peak inside the .edb file to see what data has been staged. In summary, the threat actors are looking for files and locations of interest on the network (?). ESENTUTL and EDB files are one to be aware of and possibly a contender for DFIR professionals and üîµ teams to use also!</p>

<h1 id="wrapup">WRAP UP</h1>
<p>The combination of EMOTET's access-as-a-service model and TRICKBOT's offensive set of modular tooling, they pose a real and current threat to businesses large and small. As stated before, once the infected systems have reported back - the threat actors can be pivoting further in your environment within (in this instance) &lt;48hrs and start to elevate privledges to own the domain to ultimately deliver further badness such as ransomware. Using off the shelf offensive tooling such as EMPIRE, COBALTSTRIKE, BLOODHOUND, POWERSPLOIT and the infamous MIMIKATZ, detecting these tools are key in stopping the likes of TRICKBOT from moving further. Also to note, is the time taken to detect, investigate and remediate. Doing this in a timely manner is highly recommended. With the likes of offensive PowerShell becoming easiler to detect its only a matter of time before these TTPs change once more. üê±üêà`üê≠üêÅ</p>
<h2 id="recommendations">RECOMMENDATIONS</h2>
<p>To avoid the &quot;cyber-aids&quot;. I recommend;</p>
<p><a href="https://www.sneakymonkey.net/2018/06/25/blue-team-tips/">As always defense in depth.</a></p>
<ul>
<li>Powershell visibility is still key for detection. (+transcription logging) - ship these off the host ASAP.</li>
<li>CommandLine Logging - EventID 4688, Sysinternal SYSMON, EDR products.</li>
<li>Active Directory auditing and logging to capture BLOODHOUND recon on AD objects. This could be incredible noisey depending on the environment but check out &quot;<a href="https://blog.stealthbits.com/honey-token-threat-detection-with-stealthdefend/">honey tokens</a>&quot; for fake accounts that should never be queried.</li>
<li>Detect over the wire Bloodhound via IDS/NSM - large LDAP queries from unexpected hosts like clients. Know whats normal first.</li>
<li>Further segregation of your network where possible to hinder lateral movement.</li>
<li>üëæ Puple teaming excercises üëæ . Pre-running BLOODHOUND and proactively going after the same fuit of the attackers. Harden and repeat. #PurpleTeaming</li>
<li>Detect TRICKBOT recon <code>ipconfig /all</code>, <code>net config workstation</code>, <code>net view /all /domain</code>,<code>nltest /domain_trusts</code>, <code>nltest /domain_trusts</code> within a short time frame/chained together).</li>
</ul>
