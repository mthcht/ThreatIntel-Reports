      <h1 style="position:absolute;width:1px;height:1px;padding:0;margin:-1px;overflow:hidden;clip:rect(0,0,0,0);white-space:nowrap;border-width:0">AsyncRAT Crusade: Detections and Defense</h1>
      <div>
        <p><em>Splunk is committed to using inclusive and unbiased language. This blog post might contain terminology that we no longer use. For more information on our updated terminology and our stance on biased language, please visit <a href="/en_us/blog/leadership/biased-language-has-no-place-in-tech-a-follow-up.html">our blog post.</a> We appreciate your understanding as we work towards making our community more inclusive for everyone.</em></p>
        <p>In January 2019 AsyncRAT was released as an open source remote administration tool project on <a href="https://github.com/NYAN-x-CAT/AsyncRAT-C-Sharp">GitHub</a>. AsyncRAT is a popular malware commodity and tools <a href="https://malpedia.caad.fkie.fraunhofer.de/details/win.asyncrat">used by</a> attackers and APT groups. Threat actors and adversaries used several interesting script loaders and <a href="/en_us/blog/learn/spear-phishing.html">spear phishing</a> attachments to deliver AsyncRAT to targeted hosts or networks in different campaigns.</p>
        <p>One prevalent campaign in the wild with this remote access trojan is the use of a Microsoft OneNote spear phishing attachment to load a .HTA file that downloads and runs an obfuscated batch script to execute the actual AsyncRAT code.</p>
        <p>Of the many features of AsyncRAT, it encrypts C2 communication protocol and contains several features via plugin including:</p>

          <li>
            <p>Extract Browser Credentials</p>
          </li>
          <li>
            <p>View and Record Desktop Screen</p>
          </li>
          <li>
            <p>Run Miner</p>
          </li>
          <li>
            <p>Send Files</p>
          </li>

        <h3 id="t1566004---phishing-spear-phishing-attachment">T1566.004 - Phishing: Spear Phishing Attachment</h3>
        <h4 id="malicious-onenote-attachment"><strong>Malicious OneNote Attachment</strong></h4>
        <p>The <a href="/en_us/blog/author/secmrkt-research.html">Splunk Threat Research Team</a> (STRT) found several phishing email campaigns that contain malicious .one (OneNote) attachments. The malicious OneNote document will lure the targeted user to click through the warning to view the document as seen in Figure 2.</p>
        <p>As soon as the user clicks, it will automatically load a malicious .HTA file to download the second stage of this infection chain.</p>

  by Processes.process_name Processes.parent_process_name Processes.parent_process Processes.process Processes.process_guid Processes.dest Processes.user

        <h2 id="detections">Detections</h2>
        <p>The Splunk Threat Research Team has curated relevant detections and tagged them to the <strong>AsyncRAT Analytic Story</strong> to help security analysts detect adversaries leveraging the AsyncRAT malware. This analytic story introduces <a href="https://research.splunk.com/stories/asyncrat/">23 detections</a> across MITRE ATT&#x26;CK techniques.</p>
        <p>For this release, we used and considered the relevant data endpoint telemetry sources such as:</p>
        <ul>
          <li>Process Execution &#x26; Command Line Logging</li>
          <li>Windows Security Event Id 4688, Sysmon, or any Common Information Model compliant EDR technology</li>
          <li>Windows Security Event Log</li>
          <li>Windows System Event Log</li>
          <li>Windows PowerShell Script Block Logging</li>
        </ul>

The following community Splunk SOAR playbooks can also be used in conjunction with some of the previously described analytics:</p>

        <h2 id="why-should-you-care">Why Should You Care?</h2>
        <p>With this article, the Splunk Threat Research Team (STRT) enables security analysts, blue teamers and Splunk customers to identify <strong>AsyncRAT malware</strong>. This article helps the community discover AsyncRAT tactics, techniques and procedures that are being used by several threat actors and <a href="/en_us/blog/learn/apts-advanced-persistent-threats.html">adversaries (APT)</a>. By understanding its behaviors, we were able to generate telemetry and datasets to develop and test Splunk detections designed to defend and respond against this threat.</p>
