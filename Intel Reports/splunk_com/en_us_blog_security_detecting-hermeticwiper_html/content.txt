      <h1 style="position:absolute;width:1px;height:1px;padding:0;margin:-1px;overflow:hidden;clip:rect(0,0,0,0);white-space:nowrap;border-width:0">Detecting HermeticWiper</h1>
      <div>
        <p><em>Splunk is committed to using inclusive and unbiased language. This blog post might contain terminology that we no longer use. For more information on our updated terminology and our stance on biased language, please visit <a href="/en_us/blog/leadership/biased-language-has-no-place-in-tech-a-follow-up.html">our blog post</a>. We appreciate your understanding as we work towards making our community more inclusive for everyone.</em></p>
        <p>As stated in our previous threat advisory <a href="/en_us/blog/security/threat-advisory-strt-ta02-destructive-software.html">STRT-TA02</a> in regards to destructive software, past historical data suggests that for malicious actors to succeed in long-standing campaigns they must improve and add new ways of making their payloads stealthier, resistant, and damaging. HermeticWiper introduces some unique features, applying destructive actions on compromised hosts. In addition to other commonly known wiper destructive features, <a href="https://www.sentinelone.com/labs/hermetic-wiper-ukraine-under-attack/">HermeticWiper</a> also presents the following unique behaviors:</p>
        <ul>
          <li>Interacts with the system via signed driver</li>
          <li>Disables crash dump functionality (Anti-Forensic)</li>
          <li>Modifies “GlobalFolderOptions” registry at file permission level (NTFS)</li>
          <li>Checks for FAT (Windows XP) and NTFS (Windows OS newer than XP using NTFS)</li>
          <li>Corrupts (Destroys) MBR and NTFS file system</li>
          <li>Reported to have been deployed via Group Policy Object (Windows Active Directory Group Policy Object)</li>
        </ul>
        <p>This payload is another destructive tool in the ongoing campaign which has included DDoS attacks, web defacements, <a href="https://www.cisa.gov/mdm">MDM attacks</a>, <a href="https://www.cvedetails.com/cve/CVE-2021-1636/">Microsoft SQL attacks</a> and now <a href="https://www.cisa.gov/uscert/ncas/alerts/aa22-057a">two known as of yet destructive payloads.</a></p>
        <p>STRT has also released a new analytic story covering <a href="https://research.splunk.com/stories/hermetic_wiper/">HermeticWiper</a> itself. We have collected information about the observed vectors in relation to HermeticWiper according to several security vendors including <a href="https://symantec-enterprise-blogs.security.com/blogs/threat-intelligence/ukraine-wiper-malware-russia">Symantec</a>, <a href="https://www.welivesecurity.com/2022/03/01/isaacwiper-hermeticwizard-wiper-worm-targeting-ukraine/">ESET</a>, <a href="https://www.sentinelone.com/labs/hermetic-wiper-ukraine-under-attack/">Sentinel One</a>. The following diagram shows a visual flow of the observed attack vectors per tactic.</p>

        <h2 id="dropping-driver-component-base-on-windows-version-xp-or-above">Dropping Driver Component Base on Windows Version (XP or above)</h2>
        <p>This wiper will first adjust its token privileges with “SeShutdownPrivilege” and “SeBackupPrivilege” for later purposes like initiating shutdown or accessing files with high-security descriptor context.</p>
        <p>It contains 4 compressed drivers in its RSRCsection. It will drop one of those drivers depending on the Windows version or OS architecture of the compromised host by using VerifyVersionW API. Below is the summary table of the RSRC TYPE ID and the name of its rsrc entry for each driver.</p>
        <div class="table no-header">
          <div>
            <div><strong>RSRC TYPE ID</strong></div>
            <div><strong>RSRC NAME</strong></div>
            <div><strong>Description</strong></div>
          </div>
          <div>
            <div>RCDATA</div>
            <div>DRV_X64</div>
            <div>Driver for x64 bit architecture</div>
          </div>
          <div>
            <div>RCDATA</div>
            <div>DRV_X32</div>
            <div>Driver for x32 bit architecture</div>
          </div>
          <div>
            <div>RCDATA</div>
            <div>DRV_XP_X64</div>
            <div>Driver for lower version OS (e.g XP) x64 bit architecture</div>
          </div>
          <div>
            <div>RCDATA</div>
            <div>DRV_XP_X64</div>
            <div>Driver for lower version OS (e.g XP) x32 bit architecture</div>
          </div>
        </div>
        <p>Then it will generate random characters based on the current process ID of its running process. Once the wiper parses the needed rsrc entry, and has a filename, It will locate the C:\windows\system32\Drivers folder to drop its driver component.</p>

              <p>Registry.registry_value_data Registry.process_guid | `drop_dm_object_name(Registry)`</p>
              <p>|rename process_guid as proc_guid |join proc_guid, _time [| tstats `security_content_summariesonly`</p>
              <p>count FROM datamodel=Endpoint.Processes by _time span=1h Processes.process_id Processes.process_name</p>
              <p>Processes.process Processes.dest Processes.parent_process_name Processes.parent_process</p>
              <p>Processes.process_guid | `drop_dm_object_name(Processes)` |rename process_guid as</p>
              <p>proc_guid | fields _time dest user parent_process_name parent_process process_name</p>
              <p>process_path process proc_guid registry_path registry_value_name registry_value_data]</p>
              <p>| table _time dest user parent_process_name parent_process process_name process_path</p>
              <p>process proc_guid registry_path registry_value_name registry_value_data</p>
            </div>
