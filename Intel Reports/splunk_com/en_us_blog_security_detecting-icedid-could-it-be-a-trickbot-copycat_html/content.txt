            <div>IcedID is a banking trojan, it is designed to be stealthy and built to collect financial information. IcedID harvests user credentials and banking sessions to commit financial crimes, including carding, money laundering, and transferring of funds to foreign financial institutions. In recent research published by Splunk Threat Research Team (STRT) the inclusion of cryptocurrency exchange information was also included by Trickbot in the web inject code. IcedID shares many of the same payloads featured in Emotet or Trickbot and in some cases, IcedID has been observed downloading Emotet or Trickbot, as a way to provide operators a way to use diverse carriers as well.</div>
          </div>
        </div>
        <p>IcedID targets financial institutions across different countries including banks, payment card providers, and e-commerce sites. IcedID has also been observed deployed in conjunction with other malware payloads such as <a href="https://blogs.infoblox.com/cyber-threat-intelligence/valak-downloader-infostealer-delivers-icedid-banking/">Valak</a>, <a href="https://www.bleepingcomputer.com/news/security/qbot-malware-is-back-replacing-icedid-in-malspam-campaigns/">Qakbot</a>, <a href="https://thedfirreport.com/2021/05/12/conti-ransomware/">Conti Ransomware</a>. It is clear from studying past campaigns that the actors behind IcedID have expanded beyond banking information in order to extend similar features and coverage as other popular carriers such as Emotet or trickbot and by doing so current iterations of IcedID look more like a copycat or maybe even a successor.</p>
        <h2 id="spear-phishing-documents">Spear Phishing Documents</h2>
        <p>In a <a href="https://www.jdsupra.com/legalnews/docusign-alert-new-malicious-hacking-5278845/">recent campaign</a>, malicious actors were observed using a document builder to simulate legitimate DocuSign documents and embedding exploitation code for <a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2017-8570">CVE-2017-8570</a> to trigger the installation of IcedID. These documents were delivered via <a href="/en_us/blog/learn/spear-phishing.html">spear-phishing technique</a>.</p>

        <h2 id="icedid-initial-downloader-stage-1">IcedID Initial Downloader (Stage 1)</h2>
        <p>The initial IceID loader binary will decrypt another .dll file in memory to download the 2nd stage payload (png or .dat) files. This is done by initially connecting to aws.amazon.com to check the internet connection and to prepare its initial <a href="/en_us/blog/learn/c2-command-and-control.html">C2</a> communication.</p>
        <p>

        <h2 id="icedid-png-steganography-and-licensedat-payload">IcedID .PNG Steganography and “License.dat” Payload</h2>
        <p>The PNG payload uses steganography to hide the shellcode inside the PNG. The encrypted shellcode and the 8 bytes rc4 decryption keys are placed in the IDAT chunk type structure of the PNG header file. A python script was developed <a href="https://github.com/tccontre/KnowledgeBase/tree/main/malware_re_tools/iceid_stego_shell_decryptor">(IceIdPNGShellcodeExtractor.py</a>) to automatically extract the shellcode on the said payload.</p>
        <p>For the “license.dat” IcedID payload, it will decrypt it using its customized decryption algorithm using its last 16 bytes as the decryption key. In this case, the <a href="https://github.com/BinaryDefense/IcedDecrypt">IceIdDecrypt.py</a> tool can be used to decrypt license.dat and do a static analysis of the file.</p>
