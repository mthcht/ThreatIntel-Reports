      <h1 style="position:absolute;width:1px;height:1px;padding:0;margin:-1px;overflow:hidden;clip:rect(0,0,0,0);white-space:nowrap;border-width:0">Threat Update: CaddyWiper</h1>
      <div>
        <p><em>Splunk is committed to using inclusive and unbiased language. This blog post might contain terminology that we no longer use. For more information on our updated terminology and our stance on biased language, please visit <a href="/en_us/blog/leadership/biased-language-has-no-place-in-tech-a-follow-up.html">our blog post</a>. We appreciate your understanding as we work towards making our community more inclusive for everyone.</em></p>
        <p>As the conflict in Eastern Europe continues, the Splunk Threat Research Team (STRT) is constantly monitoring new developments, especially those related to destructive software. As we have showcased in previous releases in relation to <a href="/en_us/blog/security/threat-advisory-strt-ta02-destructive-software.html">destructive software</a> and <a href="/en_us/blog/security/detecting-hermeticwiper.html">HermeticWiper</a>, malicious actors modify their TTPs in order to become more effective and achieve their objectives. In the case of HermeticWiper, we witnessed the introduction of <a href="/en_us/blog/security/detecting-hermeticwiper.html">new features</a> since the increment of malicious cyber activity targeting Ukraine from last month.</p>
        <p>We now have a new payload recently discovered by <a href="https://twitter.com/ESETresearch/status/1503436420886712321">ESET</a> named CaddyWiper, indicating no code sharing with previous malicious payloads during this campaign. There is one thing however that has been seen during the deployment of payloads, and that is the use of Group Policy Objects (GPOs).</p>
        <p><a href="https://docs.microsoft.com/en_us/previous-versions/windows/desktop/policy/group-policy-objects">Group Policy Objects</a> are Microsoft Active Directory network policies that can be applied selectively to computers, organizational units, applications, and individual users. Splunk Security research has previously shown how to use <a href="https://www.rsaconference.com/Library/presentation/USA/2017/automated-prevention-of-ransomware-with-machine-learning-and-gpos">GPOs to defend against Ransomware</a>, as the selective and massive application of these settings helps streamline, enforce and harden security policies.</p>
        <p>However, as we have witnessed, GPOs can be used to harm if malicious actors can compromise domain administrators. This new malicious payload, incorporates the following features:</p>
        <ul>
          <li>Domain Controller killswitch. If payload detects installation on a Domain Controller it stops its functions.</li>
          <li>If not in a Domain Controller it destroys users data “C:\Users” and subsequent mapped drives (this may include network mapped drives).</li>
          <li>If not in a Domain Controller it destroys drive partitions including boot partitions (\\.\PhysicalDrive9 to \\.\PhysicalDrive0)</li>
        </ul>
        <p>The above new features indicate the intention of malicious actors to maintain access to Domain Controllers and deploy destructive software without the need to have to compromise and get access again if they were destroyed and had to be reinstalled. This approach is much more tactical and it also gives attackers the possibility to modify, re-apply, or enforce GPOs that can achieve the deployment of this destructive payload. Below is a breakdown of these features.</p>
        <h2 id="domain-controller-kill-switch">Domain Controller Kill Switch</h2>
        <p>This wiper will prepare the module name and API name string on the stack to dynamically parse it upon execution. Then it will execute DsRolePrimaryDomainInformation() API to retrieve the state data of the targeted host. If the state role of the computer is DsRole_RolePrimaryDomainController caddywiper will exit its process.</p>

        <h2 id="overwriting-files-with-zeroed-buffer">Overwriting Files with Zeroed Buffer</h2>
        <p>If the computer is not a Domain Controller it will start to do its payload. One of them is overwriting files in C:\users directory and from Drive D:\ until Drive Z:\.</p>

        <p>If it finds a file that is not a folder and has a hidden system attribute, it will adjust the Security identifier permission of its process as well as its TokenPrivileges to “SeTokenOwnershipPrivilege” to be able to access those files.</p>

        <p>After that checking, Caddywiper will initialize a zeroed buffer based on the file size of the file it found. If the file size is greater than 0xA00000, It will set the maximum zeroed buffer size to 0xA00000. That buffer will be used to overwrite the files and make them unrecoverable.</p>

        <h2 id="wiping-boot-partitions">Wiping Boot Partitions</h2>
        <p>This payload will enumerate all possible boot sectors partitions from \\.\PhysicalDrive9 to \\.\PhysicalDrive0 to overwrite it with a zeroed buffer with size of 1920 bytes. The wiping was executed using DeviceIoControl IOCTL_DISK_SET_DRIVE_LAYOUT_EX.</p>

        <h2 id="mitigate-via-gpo">Mitigate via GPO</h2>
        <p>As mentioned in this Threat Update <a href="https://docs.microsoft.com/en_us/windows/security/threat-protection/security-policy-settings/how-to-configure-security-policy-settings">GPOs</a> can also <a href="https://www.rsaconference.com/Library/presentation/USA/2017/automated-prevention-of-ransomware-with-machine-learning-and-gpos">be used defensively</a> and the Splunk Security Research has previously shown how to apply them in a defensive manner. Here are some examples of GPO that can be applied to protect against destructive software attacks:</p>
        <ul>
          <li>Force logoff</li>
          <li>Remove Computer from Domain</li>
          <li>Disable password changes</li>
          <li>Disable access to network shares</li>
          <li>Enforce account lockout</li>
          <li>Prevent further download of payloads from the internet</li>
          <li>Apply firewall rules</li>
          <li>Prevent reboot of computers</li>
        </ul>
        <p>The above GPO settings in combination with Splunk SOAR playbooks such as <a href="https://research.splunk.com/playbooks/ransomware_investigate_and_contain/">Ransomware Investigate and Contain</a> may improve defenses and containment of these types of attacks.</p>
