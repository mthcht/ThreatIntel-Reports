      <h1 style="position:absolute;width:1px;height:1px;padding:0;margin:-1px;overflow:hidden;clip:rect(0,0,0,0);white-space:nowrap;border-width:0">Threat Update: Industroyer2</h1>
      <div>
        <p><em>Splunk is committed to using inclusive and unbiased language. This blog post might contain terminology that we no longer use. For more information on our updated terminology and our stance on biased language, please visit <a href="/en_us/blog/leadership/biased-language-has-no-place-in-tech-a-follow-up.html">our blog post</a>. We appreciate your understanding as we work towards making our community more inclusive for everyone.</em></p>
        <p>The Splunk Threat Research Team (STRT) continues to monitor new relevant payloads to the ongoing conflict in Eastern Europe. One of these new payloads was found by the <a href="https://cert.gov.ua/article/39518">Ukranian CERT</a> named “Industroyer2.” The name of this new payload references the original "<a href="https://collaborate.mitre.org/attackics/index.php/Software/S0001">Industroyer</a>" malicious payload used against the country of Ukraine's power grid in 2016 and allegedly was able to affect <a href="https://en.wikipedia.org/wiki/Industroyer">a fifth of the power capacity of the city of</a> Kyiv.</p>
        <p>According to the recent <a href="https://cert.gov.ua/article/39518">Ukraine CERT</a> and <a href="https://www.welivesecurity.com/2022/04/12/industroyer2-industroyer-reloaded/">ESET</a> report, Industroyer2 resembles the former Industroyer in functionality and is also targeting the electric grid containing commands targeting high-voltage electrical substations. It was reported that Industroyer2 was also used along with <a href="/en_us/blog/security/threat-update-caddywiper.html">CaddyWiper,</a> another payload recently addressed by the Splunk Threat Research Team. This payload — in combination with previous featured destructive payloads — targets <a href="/en_us/blog/security/strt-ta03-cpe-destructive-software.html">CPEs</a>. These customer premise devices such as modems, cable modems, and internet gateways are devices that provide connectivity to the great majority of commercial and residential customers, and speak to the attacker’s intention of overwhelming or degrading the victim's infrastructure.</p>
        <p>The following is an analysis of relevant detection opportunities of this payload and observed <a href="/en_us/blog/learn/ttp-tactics-techniques-procedures.html">TTPs</a> during the deployment of this payload.</p>

        <h2 id="terminate-process-and-rename-process-file-path">Terminate Process and Rename Process File Path</h2>
        <p>This function enumerates all running processes in the targeted host and looks for the process named “PServiceControl.exe” and also the process name stated in its config data. It will also look for the file path of that process in a specific folder that is in the config file and rename it with “.MZ” file extension.</p>

          <div>
            <div>TTP</div>
            <div><a href="https://research.splunk.com/endpoint/winevent_scheduled_task_created_within_public_path/">WinEvent Scheduled Task Created Within Public Path</a> (Updated)</div>
            <div><a href="https://attack.mitre.org/techniques/T1053/005/">T1053.005</a></div>
            <div>Execution, Persistence, Privilege Escalation</div>
            <div>The following query utilizes Windows Security EventCode 4698. A scheduled task was created to identify suspicious tasks registered on Windows either via schtasks.exe OR TaskService with a command to be executed from a user-writable file path.</div>
          </div>
          <div>
            <div>Hunting</div>
            <div><a href="https://research.splunk.com/endpoint/winevent_windows_task_scheduler_event_action_started/">WinEvent Windows Task Scheduler Event Action Started</a></div>
            <div><a href="https://attack.mitre.org/techniques/T1053/005/">T1053.005</a></div>
            <div>Execution, Persistence, Privilege Escalation</div>
            <div>This hunting analytic assists with identifying suspicious tasks that have been registered and run in Windows using EventID 200 (action run) and 201 (action completed).</div>
          </div>
          <div>
            <div>TTP</div>
            <div><a href="https://research.splunk.com/endpoint/schtasks_run_task_on_demand/">Schtasks Run Task On Demand</a></div>
            <div><a href="https://attack.mitre.org/techniques/T1053/">T1053</a></div>
            <div>Execution, Persistence, Privilege Escalation</div>
            <div>This analytic identifies an on-demand run of a Windows Schedule Task through shell or command-line.</div>
          </div>
          <div>
            <div>TTP</div>
            <div><a href="https://research.splunk.com/endpoint/attempted_credential_dump_from_registry_via_reg_exe/">Attempted Credential Dump From Registry via Reg exe</a></div>
            <div><a href="https://attack.mitre.org/techniques/T1003/">T1003</a></div>
            <div>Credential Access</div>
            <div>This analytic identifies the use of reg.exe attempting to export Windows registry keys that contain hashed credentials. Adversaries will utilize this technique to capture and perform offline password cracking.</div>
          </div>
          <div>
            <div>TTP</div>
            <div><a href="https://research.splunk.com/endpoint/dump_lsass_via_comsvcs_dll/">Dump LSASS via comsvcs DLL</a></div>
            <div><a href="https://attack.mitre.org/techniques/T1003/001/">T1003.001</a></div>
            <div>Credential Access</div>
            <div>This analytic identifies the usage of comsvcs.dll for dumping the lsass process.</div>
          </div>
          <div>
            <div>TTP</div>
            <div><a href="https://research.splunk.com/endpoint/executable_file_written_in_administrative_smb_share/">Executable File Written in Administrative SMB Share</a></div>
            <div><a href="https://attack.mitre.org/techniques/T1021/002/">T1021.002</a></div>
            <div>Lateral Movement</div>
            <div>This analytic identifies executable files (.exe or .dll) being written to Windows administrative SMB shares (Admin$, IPC$, C$).</div>
          </div>
          <div>
            <div>TTP</div>
            <div><a href="https://research.splunk.com/endpoint/suspicious_process_file_path/">Suspicious Process File Path</a></div>
            <div><a href="https://attack.mitre.org/techniques/T1543/">T1543</a></div>
            <div>Persistence, Privilege Escalation</div>
            <div>This analytic identifies a suspicious process running in a file path where a process is not commonly seen and is most commonly used by malicious software.</div>
          </div>
          <div>
            <div>TTP</div>
            <div><a href="https://research.splunk.com/endpoint/executables_or_script_creation_in_suspicious_path/">Executables Or Script Creation In Suspicious Path</a></div>
            <div>T1036</div>
            <div>Defense Evasion</div>
            <div>This analytic identifies suspicious executables or scripts (known file extensions) in a list of suspicious file paths in Windows.</div>
          </div>
          <div>
            <div>TTP</div>
            <div><a href="https://research.splunk.com/endpoint/impacket_lateral_movement_commandline_parameters/">Impacket Lateral Movement Commandline Parameters</a></div>
            <div><a href="https://attack.mitre.org/techniques/T1021/">T1021</a><br><a href="https://attack.mitre.org/techniques/T1021/002/">T1021.002</a><br><a href="https://attack.mitre.org/techniques/T1021/003/">T1021.003</a><br><a href="https://attack.mitre.org/techniques/T1047/">T1047</a><br><a href="https://attack.mitre.org/techniques/T1543/003/">T1543.003</a></div>
            <div>Lateral Movement<br>Execution<br>Persistence, Privilege Escalation</div>
            <div>This analytic identifies the presence of suspicious command line parameters typically present when using Impacket tools.</div>
          </div>
          <div>
            <div>Anomaly</div>
            <div><a href="https://research.splunk.com/endpoint/linux_system_network_discovery/">Linux System Network Discovery</a></div>
            <div>T1016</div>
            <div>Discovery</div>
            <div>This analytic identifies possible enumeration of local <a href="/en_us/blog/learn/network-configuration.html">network configuration</a>. This technique is commonly used as part of recon of adversaries or threat actors to know some network information for its next or further attack.</div>
          </div>
          <div>
            <div>TTP</div>
            <div><a href="https://splunkresearch.com/endpoint/recon_using_wmi_class/">Recon Using WMI Class</a></div>
            <div><a href="https://attack.mitre.org/techniques/T1592/">T1592</a></div>
            <div>Reconnaissance</div>
            <div>This analytic identifies suspicious PowerShell via EventCode 4104, where WMI is performing an event query looking for running processes or running services.</div>
          </div>
          <div>
            <div>Hunting</div>
            <div><a href="https://research.splunk.com/endpoint/linux_adding_crontab_using_list_parameter/">Linux Adding Crontab Using List Parameter</a> (New)</div>
            <div><a href="https://attack.mitre.org/techniques/T1053/003/">T1053.003</a></div>
            <div>Execution, Persistence, Privilege Escalation</div>
            <div>This analytic identifies a suspicious cron jobs modification using crontab list parameters.</div>
          </div>
          <div>
            <div>TTP</div>
            <div><a href="https://research.splunk.com/endpoint/linux_deleting_critical_directory_using_rm_command/">Linux Deleting Critical Directory Using RM Command</a> (New)</div>
            <div><a href="https://attack.mitre.org/techniques/T1485/">T1485</a></div>
            <div>Impact</div>
            <div>This analytic identifies a suspicious deletion of a critical folder in a Linux machine using rm command.</div>
          </div>
          <div>
            <div>TTP</div>
            <div><a href="https://research.splunk.com/endpoint/linux_disable_services/">Linux Disable Services</a> (New)</div>
            <div><a href="https://attack.mitre.org/techniques/T1489/">T1489</a></div>
            <div>Impact</div>
            <div>This analytic identifies events that attempt to disable a service.</div>
          </div>
          <div>
            <div>TTP</div>
            <div><a href="https://research.splunk.com/endpoint/linux_shred_overwrite_command/">Linux Shred Overwrite Command</a> (New)</div>
            <div><a href="https://attack.mitre.org/techniques/T1485/">T1485</a></div>
            <div>Impact</div>
            <div>This analytic identifies a shred process to overwrite files in a Linux machine.</div>
          </div>
          <div>
            <div>TTP</div>
            <div><a href="https://research.splunk.com/endpoint/linux_stop_services/">Linux Stop Services</a> (New)</div>
            <div><a href="https://attack.mitre.org/techniques/T1489/">T1489</a></div>
            <div>Impact</div>
            <div>This analytic identifies events that attempt to stop or clear a service.</div>
          </div>
          <div>
            <div>Anomaly</div>
            <div><a href="https://research.splunk.com/endpoint/windows_processes_killed_by_industroyer2_malware/">Windows Processes Killed By Industroyer2 Malware</a></div>
            <div><a href="https://attack.mitre.org/techniques/T1489/">T1489</a></div>
            <div>Impact</div>
            <div>This analytic identifies known processes killed by Industroyer2 malware.</div>
          </div>
          <div>
            <div>TTP</div>
            <div><a href="https://research.splunk.com/endpoint/windows_hidden_schedule_task_settings/">Windows Hidden Schedule Task Settings</a> (New)</div>
            <div><a href="https://attack.mitre.org/techniques/T1053/">T1053</a></div>
            <div>Execution, Persistence, Privilege Escalation</div>
            <div>
              <p>This query utilizes Windows Security EventCode 4698.</p>
              <p>A scheduled task was created to identify suspicious tasks registered on</p>
              <p>Windows either via schtasks.exe OR TaskService with a hidden setting.</p>
            </div>
          </div>
          <div>
            <div>Anomaly</div>
            <div><a href="https://research.splunk.com/endpoint/windows_linked_policies_in_adsi_discovery/">Windows Linked Policies In ADSI Discovery</a></div>
            <div><a href="https://attack.mitre.org/techniques/T1087/002/">T1087.002</a></div>
            <div>Discovery</div>
            <div>This analytic utilizes PowerShell Script Block Logging (EventCode=4104) to identify the `[Adsisearcher]` type accelerator being used to query Active Directory for domain groups.</div>
          </div>
          <div>
            <div>Anomaly</div>
            <div><a href="https://research.splunk.com/endpoint/windows_root_domain_linked_policies_discovery/">Windows Root Domain linked policies Discovery</a></div>
            <div><a href="https://attack.mitre.org/techniques/T1087/002/">T1087.002</a></div>
            <div>Discovery</div>
            <div>This analytic utilizes PowerShell Script Block Logging (EventCode=4104) to identify the `[Adsisearcher]` type to enumerate root domain linked policies for situational awareness and Active Directory Discovery.</div>
          </div>
        </div>
