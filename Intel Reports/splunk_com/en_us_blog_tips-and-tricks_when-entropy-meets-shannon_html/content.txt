      <h1 style="position:absolute;width:1px;height:1px;padding:0;margin:-1px;overflow:hidden;clip:rect(0,0,0,0);white-space:nowrap;border-width:0">When entropy meets Shannon</h1>
      <div>
        <p>This is the third post on URL analysis, please have a look at the two other posts for more context about what can be done with Splunk to analyze URLs:</p>
        <ul>
          <li><a href="http://blogs.splunk.com/2016/03/22/splunking-1-million-urls/">Splunking 1 million URLs</a></li>
          <li><a href="http://blogs.splunk.com/2016/04/01/hunting-that-evil-typosquatter/">Hunting that evil typosquatter</a></li>
        </ul>
        <p>You will find in this article information on how one can detect DNS tunnels. While you can find <a href="https://splunkbase.splunk.com/apps/#/page/1/search/dns/order/relevance">lots of very useful apps on Splunkbase</a> to help you analyze DNS data, it is always good for curious individuals to discover some techniques being used underneath.</p>
        <p>A lot of captive portals are bypassed everyday by anyone able to run a DNS request, if someone can run on their machine the following command:</p>
        <pre><code>$ host splunk.com
splunk.com has address 54.69.58.243
...
</code></pre>
        <p>Without being authenticated on the captive portal, then they can use any service on the internet using a DNS tunnel. There are <a href="https://github.com/iagox86/dnscat2">a lot</a> <a href="http://wiki.kartbuilding.net/index.php/Dnstunnel">of</a> <a href="http://code.kryo.se/iodine/">tools</a> <a href="https://www2.deloitte.com/fr/fr/pages/risque-compliance-et-controle-interne/articles/cyber-academy.html/ressources/outils/dns2tcp/.html">out</a> <a href="https://github.com/FedericoCeratto/dnscapy">there</a> to create those tunnels. And for a great paper on the topic, I encourage you to read the <a href="https://www.sans.org/reading-room/whitepapers/dns/detecting-dns-tunneling-34152">Detecting DNS Tunneling</a> from SANS Institute.</p>
        <h3 id="claude-shannon-to-the-rescue">Claude Shannon to the rescue!</h3>
        <p>
          <picture>
            <source type="image/webp" srcset="./media_1447043fe48637bbecc501830236fd2c1e3224aea.avif?width=2000&#x26;format=webply&#x26;optimize=medium" media="(min-width: 600px)">

        <p>Claude Shannon<br>By Jacobs, Konrad - <a href="https://opc.mfo.de/detail?photo_id=3807">https://opc.mfo.de/detail?photo_id=3807</a>, <a href="https://creativecommons.org/licenses/by_sa/2.0/de/deed.en" title="Creative Commons Attribution-Share Alike 2.0 de">CC BY-SA 2.0 de</a>, <a href="https://commons.wikimedia.org/w/index.php?curid=45380422">Link</a></p>
        <p>Long time ago, the venerable <a href="https://en.wikipedia.org/wiki/Claude_Shannon">Claude E. Shannon</a> wrote the paper “<a href="http://math.harvard.edu/~ctm/home/text/others/shannon/entropy/entropy.pdf">A Mathematical Theory of Communication</a>“, which I strongly encourage to read for its clarity and amazing source of information.</p>
        <p>He invented a great algorithm known as the <a href="https://en.wikipedia.org/wiki/Entropy_(information_theory)">Shannon Entropy</a> which is useful to discover the statistical structure of a word or message.</p>
        <p>If you consider a word, being a discrete source of the finite number of characters type which can be considered, for each possible character there will be a set of probabilities which would produce various outputs. There will be an entropy for each character. This entropy on the chosen word is defined as the average of the output weighted on the probability of occurrence of the characters.</p>
        <p>The previous paragraph can easily be translated into the following Python code (taken from the excellent URL Toolbox on Splunkbase:</p>
        <pre><code>def shannon(word):
    entropy = 0.0
    length = len(word)
</code></pre>
        <pre><code>    occ = {}
    for c in word :
        if not c in occ:
            occ[ c ] = 0
        occ += 1

    for (k,v) in occ.iteritems():
        p = float( v ) / float(length)
        entropy -= p * math.log(p, 2) # Log base 2
</code></pre>
        <pre><code>    return entropy
</code></pre>
        <p>Which can be run directly from any word you can have in Splunk:</p>

        <h3 id="catching-dns-tunnels-from-subdomains-in-urls">Catching DNS tunnels from subdomains in URLs</h3>
        <pre><code>If we run the following query, interesting results are shown:
sourcetype="isc:bind:query" | eval list="mozilla" | `ut_parse(query, list)` | `ut_shannon(ut_subdomain)` | table ut_shannon, query | sort ut_shannon desc
