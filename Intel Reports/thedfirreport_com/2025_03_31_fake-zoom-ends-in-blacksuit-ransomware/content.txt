			<h1 class="entry-title">Fake Zoom Ends in BlackSuit Ransomware</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2025/03/31/fake-zoom-ends-in-blacksuit-ransomware/" rel="bookmark"><time class="entry-date published" datetime="2025-03-31T00:01:24+00:00">March 31, 2025</time></a></span>		</div><!-- .entry-meta -->

<h2 class="wp-block-heading">Key Takeaways</h2>



<ul class="wp-block-list">
<li>The threat actor gained initial access by a fake Zoom installer that used d3f@ckloader and IDAT loader to drop SectopRAT.</li>



<li>After nine days of dwell time, the SectopRAT malware dropped Cobalt Strike and Brute Ratel.</li>



<li>Lateral movement was achieved using various remote services and later RDP. To facilitate RDP lateral movement the threat actor employed a malware with proxy capabilities known as QDoor.</li>



<li>The threat actor used WinRAR to archive various files and then upload them to a cloud SaaS application named Bublup.</li>



<li>Finally, the threat actor deployed and executed BlackSuit ransomware across all Windows systems, using PsExec.</li>
</ul>

<h2 id="case-summary" class="wp-block-heading"><a href="#case-summary">Case Summary</a></h2>



<p>This case from May 2024 started with a malicious download from a website mimicking the teleconferencing application Zoom. When visiting the website and downloading a file that seems intended for installing Zoom, the user was, in fact, installing a malicious program created with Inno Setup.</p>



<p>The malicious program was a <a href="https://www.esentire.com/blog/d3f-ck-loader-the-new-maas-loader" target="_blank" rel="noreferrer noopener">d3f@ck loader</a> based on the Pascal scripting language and included several references to the subsequent stages in the execution chain, such as OneDrive, Telegram, and Steam. After executing a batch script to exclude the payload folder from Windows Defender and mark it as hidden, the program connected to a Steam Community page for the next stage IP address. </p>



<p>Two archive files were downloaded from this IP and extracted by another batch script. The script then proceeded to execute a payload from each archive. From one archive the script ran the legitimate Zoom installer so that the end user would get the program they thought they downloaded lowering the chances of the user reporting the event to IT or security staff.</p>



<p>The second payload executed was <a href="https://malpedia.caad.fkie.fraunhofer.de/details/win.hijackloader" target="_blank" rel="noreferrer noopener">IDAT loader</a> and an encrypted payload file, that resulted in the injection of SectopRAT into MSBuild.exe. The MSbuild.exe process called out to Pastebin to receive an IP address to use for it&#8217;s command and control endpoint. After this, command and control traffic was established and activity ceased for eight days.</p>



<p>On the ninth day of the intrusion, SectopRAT spawned a new command shell and executed a new payload. This was for a Brute Ratel payload, commonly referred to as &#8216;Badgers&#8217;. This Badger ran a series of Windows commands for discovery before executing a Cobalt Strike executable beacon. The Cobalt Strike beacon injected into a dllhost process and then was observed accessing LSASS memory.</p>



<p>After this, the threat actor began moving laterally using Cobalt Strike psexec_psh to launch a remote service for a PowerShell Cobalt beacon on a domain controller. Once on the domain controller they continued discovery actions using utilities like nltest, net, and systeminfo. They continued to run Cobalt Strike beacons across several more hosts throughout the environment, repeating the pattern observed on the domain controller.</p>



<p>On one domain controller and a backup server the threat actor dropped a binary svhost.exe. This file was a proxy tool know as QDoor. The threat actors executed this file using WMIC on the hosts and passed an IP address to provide a remote server for QDoor to proxy traffic to. Using this tunnel the threat actor proxied RDP traffic from their server to connect to hosts through the domain controller.</p>



<p>They first connected to a file share server where they used the Edge browser to download WinRAR. They then used WinRAR to create an archive of targeted file shares. They then exfiltrated these archives using the SaaS project management suite Bublup via the Edge browser.</p>



<p>Following this they then connected to the first domain controller via RDP. There they downloaded WinRAR again using the Edge browser, followed by a RAR archive from the temporary file sharing site <a href="http://temp.sh" target="_blank" rel="noreferrer noopener">temp.sh</a>. This RAR archive contained all the files needed to stage and deploy their ransomware.</p>



<p>After extracting the archive contents they began to proceed with ransomware deployment. They executed a batch file that copied the ransomware executable to remote hosts using a series of text files containing the remote targets using PsExec. This was followed with a second batch script that utilized the same text files and PsExec to run the ransomware on remote hosts.</p>



<p>The ransomware was BlackSuit, which after execution on the remote hosts it used vssadmin to delete shadow copies, encrypted the local files, and then drop a ransom note. After completing the remote ransomware deployment the threat actor used WMIC to execute the ransomware locally on the domain controller they used to stage the remote deployment. The Time to Ransomware (TTR) in this case was a little over 194 hours over nine calendar days.</p>

<h2 id="execution" class="wp-block-heading"><a href="#execution">Execution</a></h2>



<p>The Zoom installer was created using <a href="https://jrsoftware.org/isinfo.php" target="_blank" rel="noreferrer noopener">Inno Setup</a>, a free installer for Windows programs, and served as the delivery mechanism for a multi-stage malware deployment and execution chain.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/03/29354_007.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-42837" src="https://thedfirreport.com/wp-content/uploads/2025/03/29354_007.png" alt="" width="1341" height="509" srcset="https://thedfirreport.com/wp-content/uploads/2025/03/29354_007.png 1341w, https://thedfirreport.com/wp-content/uploads/2025/03/29354_007-300x114.png 300w, https://thedfirreport.com/wp-content/uploads/2025/03/29354_007-1024x389.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/03/29354_007-768x292.png 768w" sizes="auto, (max-width: 1341px) 100vw, 1341px" /></a></p>





<p>The trojanized installer was a downloader, more publicly known as “d3f@ckloader”, and is built upon the Pascal Scripting language. String analysis of the compiled code revealed references to various zip/cmd files, executables and URLs. The embedded URLs pointed to various platforms, including OneDrive, Telegram, and Steam.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/03/29354_008.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-42838" src="https://thedfirreport.com/wp-content/uploads/2025/03/29354_008.png" alt="" width="1399" height="654" srcset="https://thedfirreport.com/wp-content/uploads/2025/03/29354_008.png 1399w, https://thedfirreport.com/wp-content/uploads/2025/03/29354_008-300x140.png 300w, https://thedfirreport.com/wp-content/uploads/2025/03/29354_008-1024x479.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/03/29354_008-768x359.png 768w" sizes="auto, (max-width: 1399px) 100vw, 1399px" /></a></p>

<p><strong>Brute Ratel</strong></p>



<p>On the ninth day of the intrusion we observed the SectopRAT process spawn a new command shell and execute the DLL 2905.dll using regsvr32.exe .</p>

<h2 id="defense-evasion" class="wp-block-heading"><a href="#defense-evasion">Defense Evasion</a></h2>



<p>The Cobalt Strike beacon payload was successfully injected into the dllhost.exe process as indicated by Sysmon event id 10 in the screenshot below.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/03/29354_026.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-42856" src="https://thedfirreport.com/wp-content/uploads/2025/03/29354_026.png" alt="" width="916" height="261" srcset="https://thedfirreport.com/wp-content/uploads/2025/03/29354_026.png 916w, https://thedfirreport.com/wp-content/uploads/2025/03/29354_026-300x85.png 300w, https://thedfirreport.com/wp-content/uploads/2025/03/29354_026-768x219.png 768w" sizes="auto, (max-width: 916px) 100vw, 916px" /></a></p>





<p>During the initial loader execution the threat actor used the Windows <a href="https://learn.microsoft.com/en-us/windows-server/administration/windows-commands/attrib" target="_blank" rel="noreferrer noopener">attrib</a> utility to change the attributes using the following command:</p>



<pre class="wp-block-preformatted">attrib  +s +h /D "C:\Program Files\Windows NT\*.*"
</pre>



<p>The command flags provided perform the following actions:</p>



<ul class="wp-block-list">
<li>+s Sets the System file attribute. If a file uses this attribute set, you must clear the attribute before you can change any other attributes for the file.</li>



<li>+h Sets the Hidden file attribute. If a file uses this attribute set, you must clear the attribute before you can change any other attributes for the file.</li>



<li>/D Applies <strong>attrib</strong> and any command-line options to directories.</li>
</ul>



<p>This would have hidden the initial access loader files from being visible when browsing via Explorer, lowering the chance of being discovered.</p>



<p>During the intrusion the threat actor dropped SectopRAT to disk in %USERPROFILE%\AppData\Local\Temp.</p>
