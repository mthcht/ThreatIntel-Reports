			<h1 class="entry-title">Navigating Through The Fog</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2025/04/28/navigating-through-the-fog/" rel="bookmark"><time class="entry-date published" datetime="2025-04-28T00:03:02+00:00">April 28, 2025</time></a></span>		</div><!-- .entry-meta -->

<h2 class="wp-block-heading">Key Takeaways</h2>



<ul class="wp-block-list">
<li>An open directory associated with a ransomware affiliate, likely linked to the Fog ransomware group, was discovered in December 2024. It contained tools and scripts for reconnaissance, exploitation, lateral movement, and persistence.</li>



<li>Initial access was gained using compromised SonicWall VPN credentials, while other offensive tools facilitated credential theft, exploitation of Active Directory vulnerabilities, and lateral movement.</li>



<li>Persistence was maintained through AnyDesk, automated by a PowerShell script that preconfigured remote access credentials.</li>



<li>Sliver C2 executables were hosted on the server for command-and-control operations, alongside Proxychains tunneling.</li>



<li>The victims spanned multiple industries, including technology, education, and logistics, across Europe, North America, and South America, highlighting the affiliate’s broad targeting scope.</li>
</ul>

<h2 id="case-summary" class="wp-block-heading"><a href="#case-summary">Case Summary</a></h2>



<p>The DFIR Report’s Threat Intel Group identified an open directory in December 2024, hosted at 194.48.154.79:80. The directory was likely linked to a ransomware operator associated with the Fog group, first observed in mid-2024. Analysis of its contents revealed a comprehensive toolkit used for reconnaissance, exploitation, credential theft, and command-and-control activities.</p>



<p>Among the tools were SonicWall Scanner for exploiting VPN credentials, DonPAPI for extracting Windows DPAPI-protected credentials, Certipy for abusing Active Directory Certificate Services (AD CS), Zer0dump, and Pachine/noPac for exploiting Active Directory vulnerabilities like CVE-2020-1472.</p>



<p>The affiliate also leveraged AnyDesk for persistence through an automated PowerShell script and hosted Sliver C2 components on the server for managing implants. Proxychains and Powercat were used to facilitate stealthy lateral movement and reverse shells. Victim data found in the directory indicated targets across multiple industries, including technology, education, and logistics, with a geographic focus on Italy, Greece, Brazil, and the USA.</p>
<p>Further information about Fog ransomware can be found at the following blog posts: <a class="ql-link" href="https://arcticwolf.com/resources/blog/lost-in-the-fog-a-new-ransomware-threat/" target="_blank" rel="noopener noreferrer">Arctic Wolf</a>, <a class="ql-link" href="https://www.sentinelone.com/anthology/fog/" target="_blank" rel="noopener noreferrer">SentinelOne</a> and <a class="ql-link" href="https://any.run/malware-trends/fog/" target="_blank" rel="noopener noreferrer">any.run</a>.</p>

<h3 class="wp-block-heading">Initial Access</h3>



<p><strong>Valid Account</strong></p>



<p><strong>SonicWall Scanner:</strong> The server contained a ZIP file called sonic_scan.zip, in which a Python script and a file, data.txt, with potentially compromised VPN credentials were present, that had the following structure:</p>



<ul class="wp-block-list">
<li>Target IP Address</li>



<li>Username</li>



<li>Password</li>



<li>Domain Name</li>



<li>VPN Software</li>
</ul>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/04/34568_008.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-43437" src="https://thedfirreport.com/wp-content/uploads/2025/04/34568_008.png" alt="" width="802" height="203" srcset="https://thedfirreport.com/wp-content/uploads/2025/04/34568_008.png 802w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_008-300x76.png 300w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_008-768x194.png 768w" sizes="auto, (max-width: 802px) 100vw, 802px" /></a></p>





<p>The script sonic_scan/main.py allows the threat actor to authenticate to SonicWall VPN appliances and execute a port scan by performing the following actions:</p>



<ul class="wp-block-list">
<li>Reads and parses data.txt to get the IP address, username, password and domain name (if any) of the target appliance.</li>



<li>For every entry in data.txt, which represents a potential valid account, obtains the domain name associated with the specific IP address and then executes the NetExtender process to login into the target VPN. NetExtender is a SonicWall command-line utility which allows access a VPN server. Specifically, NetExtender is executed via this command-line: netextender &lt;target IP address&gt; &#8211;username &lt;target username&gt; &#8211;password &lt;username password&gt; &#8211;domain &lt;target domain name&gt; &#8211;always-trust .</li>



<li>After the execution of NetExtender, the script parses the output produced by the VPN and, if the connection was successful, a loop is initiated until the string &#8220;NetExtender connected successfully&#8221; is found. During this loop, IP addresses are extracted and appended to a list which is then provided to Nmap for port scanning.</li>



<li>Once the Nmap scan is completed, the NetExtender process is terminated.</li>
</ul>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/04/34568_009.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-43438" src="https://thedfirreport.com/wp-content/uploads/2025/04/34568_009.png" alt="" width="1263" height="1382" srcset="https://thedfirreport.com/wp-content/uploads/2025/04/34568_009.png 1263w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_009-274x300.png 274w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_009-936x1024.png 936w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_009-768x840.png 768w" sizes="auto, (max-width: 1263px) 100vw, 1263px" /></a></p>







<p><a href="https://arcticwolf.com/resources/blog/arctic-wolf-labs-observes-increased-fog-and-akira-ransomware-activity-linked-to-sonicwall-ssl-vpn/" target="_blank" rel="noopener">Arctic Wolf</a> also linked the usage of compromised SonicWall credentials to Fog Ransomware.</p>

<h3 class="wp-block-heading">Credential Access</h3>



<p><strong>Credentials from Password Stores</strong></p>



<p><strong>DonPAPI &amp; Impacket dpapi:</strong> DonPAPI provides functionalities to locate and retrieve Windows Data Protection API (DPAPI) protected credentials from compromised Windows systems, such as:</p>



<ul class="wp-block-list">
<li>Chromium browser credentials, cookies and tokens.</li>



<li>Firefox browser credentials and cookies.</li>



<li>Windows certificates.</li>



<li>Credential manager.</li>



<li>Vaults credentials.</li>
</ul>



<p>The .bash_history shows that the threat actor employed DonPAPI with the following command-line options:</p>



<ul class="wp-block-list">
<li><strong>-pvk &lt;pvk name&gt;.pvk:</strong> specifies the Private Key File (PVK) used to decrypt DPAPI-protected credentials.</li>



<li><strong>&#8211;no_vnc:</strong> disables the collection of VNC credentials.</li>



<li><strong>&#8211;no_remoteops:</strong> disables the collection of credentials from remote systems.</li>



<li><strong>&#8211;GetHashes:</strong> enables the collection of NTLM hashes.</li>



<li><strong>&#8211;no_recent:</strong> disables the collection of recent files and documents</li>



<li><strong>&#8211;no_sysadmins:</strong> excludes systems administrators from the credential collection process</li>



<li><strong>-o &lt;filename&gt;:</strong> saves the output into a specific folder.</li>
</ul>



<figure class="wp-block-image"></figure>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/04/34568_012.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-43441" src="https://thedfirreport.com/wp-content/uploads/2025/04/34568_012.png" alt="" width="2000" height="325" srcset="https://thedfirreport.com/wp-content/uploads/2025/04/34568_012.png 2000w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_012-300x49.png 300w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_012-1024x166.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_012-768x125.png 768w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_012-1536x250.png 1536w" sizes="auto, (max-width: 2000px) 100vw, 2000px" /></a></p>



<p>The threat actor also employs dpapi.py from the Impacket suite to retrieve domain backup keys.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/04/34568_013.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-43442" src="https://thedfirreport.com/wp-content/uploads/2025/04/34568_013.png" alt="" width="1538" height="225" srcset="https://thedfirreport.com/wp-content/uploads/2025/04/34568_013.png 1538w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_013-300x44.png 300w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_013-1024x150.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_013-768x112.png 768w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_013-1536x225.png 1536w" sizes="auto, (max-width: 1538px) 100vw, 1538px" /></a></p>





<figure class="wp-block-image"></figure>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/04/34568_014.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-43443" src="https://thedfirreport.com/wp-content/uploads/2025/04/34568_014.png" alt="" width="2000" height="595" srcset="https://thedfirreport.com/wp-content/uploads/2025/04/34568_014.png 2000w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_014-300x89.png 300w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_014-1024x305.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_014-768x228.png 768w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_014-1536x457.png 1536w" sizes="auto, (max-width: 2000px) 100vw, 2000px" /></a></p>



<p><strong>Steal or Forge Authentication Certificates</strong></p>



<p><strong>Certipy:</strong> Certipy is a Python-based tool created for interacting with Active Directory Certificate Services (AD CS), specifically to enumerate, identify and exploit vulnerable certificate templates with the final aim of impersonating high-privileged accounts.</p>



<p>The following image shows how the threat actor launched the find functionality to detect vulnerable AD CS templates.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/04/34568_015.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-43444" src="https://thedfirreport.com/wp-content/uploads/2025/04/34568_015.png" alt="" width="1562" height="321" srcset="https://thedfirreport.com/wp-content/uploads/2025/04/34568_015.png 1562w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_015-300x62.png 300w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_015-1024x210.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_015-768x158.png 768w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_015-1536x316.png 1536w" sizes="auto, (max-width: 1562px) 100vw, 1562px" /></a></p>





<p><strong>Steal or Forge Kerberos Tickets: Kerberoasting</strong></p>



<p><strong>Orpheus:</strong> Orpheus is an offensive Python tool designed by Trustedsec based on Impacket&#8217;s GetUserSPNs.py and kerberosv5.py. The tool is built with defense evasion mechanisms since it changes the encryption type from RC4 (eType 23), which is common for Kerberoasting attack, to AES-256 (eType 18) that is usual in legitimate traffic.</p>



<p>Based on the available .bash_history file, the threat actor only downloaded the tool.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/04/34568_016.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-43445" src="https://thedfirreport.com/wp-content/uploads/2025/04/34568_016.png" alt="" width="642" height="57" srcset="https://thedfirreport.com/wp-content/uploads/2025/04/34568_016.png 642w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_016-300x27.png 300w" sizes="auto, (max-width: 642px) 100vw, 642px" /></a></p>





<h3 class="wp-block-heading">Privilege Escalation</h3>



<p><strong>Exploitation for Privilege Escalation</strong></p>



<p><strong>Zer0dump:</strong> Zer0dump is a Proof-of-Concept (PoC) exploit for Zerologon vulnerability (CVE-2020-1472). The tool allows the compromise of unpatched Domain Controllers to obtain Domain Admins privileges in Active Directory environments.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/04/34568_017.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-43446" src="https://thedfirreport.com/wp-content/uploads/2025/04/34568_017.png" alt="" width="792" height="152" srcset="https://thedfirreport.com/wp-content/uploads/2025/04/34568_017.png 792w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_017-300x58.png 300w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_017-768x147.png 768w" sizes="auto, (max-width: 792px) 100vw, 792px" /></a></p>





<p>The Zerologon exploitation is made by the following steps:</p>



<ul class="wp-block-list">
<li>Leverages a cryptographic weakness in Netlogon&#8217;s AES-CFB8 implementation, which allows a threat actor to bypass authentication by sending Netlogon messages with a ClientCredential field filled with zeros.</li>



<li>Resets the Domain Controller&#8217;s machine account password to an empty string, effectively granting administrative access to the domain.</li>



<li>Integrates with Impacket&#8217;s secretsdump.py to extract credentials from the compromised Domain Controller.</li>



<li>After exploitation, zer0dump attempts to restore the original machine account password.</li>
</ul>



<p><strong>Pachine &amp; noPac:</strong> Pachine and noPac are offensive tools developed to exploit CVE-2021-42278 and CVE-2021-42287 vulnerabilities, allowing privilege escalation in Active Directory. They leverage the Kerberos Privilege Attribute Certificate (PAC) to potentially impersonate a domain administrator starting from a standard domain account.</p>



<p>The Pachine&#8217;s scan options enumerate the domain controller to determine if it is vulnerable to CVE-2021-42278.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/04/34568_018.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-43447" src="https://thedfirreport.com/wp-content/uploads/2025/04/34568_018.png" alt="" width="1546" height="121" srcset="https://thedfirreport.com/wp-content/uploads/2025/04/34568_018.png 1546w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_018-300x23.png 300w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_018-1024x80.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_018-768x60.png 768w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_018-1536x120.png 1536w" sizes="auto, (max-width: 1546px) 100vw, 1546px" /></a></p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/04/34568_019.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-43448" src="https://thedfirreport.com/wp-content/uploads/2025/04/34568_019.png" alt="" width="1454" height="253" srcset="https://thedfirreport.com/wp-content/uploads/2025/04/34568_019.png 1454w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_019-300x52.png 300w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_019-1024x178.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_019-768x134.png 768w" sizes="auto, (max-width: 1454px) 100vw, 1454px" /></a></p>







<p>To exploit the previously mentioned vulnerabilities, noPac requires valid domain account credentials which are abused to obtain a Kerberos ticket that can be leveraged to compromise a domain admin.</p>



<p>The following image shows the usage of noPac to impersonate the domain administrator administrator.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/04/34568_020.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-43449" src="https://thedfirreport.com/wp-content/uploads/2025/04/34568_020.png" alt="" width="2000" height="423" srcset="https://thedfirreport.com/wp-content/uploads/2025/04/34568_020.png 2000w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_020-300x63.png 300w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_020-1024x217.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_020-768x162.png 768w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_020-1536x325.png 1536w" sizes="auto, (max-width: 2000px) 100vw, 2000px" /></a></p>





<p>The main difference between the two tools is that:</p>



<ul class="wp-block-list">
<li>Pachine adds machine accounts and obtains TGS for specific SPNs.</li>



<li>NoPac also provides the capability to drop shells and dump hashes.</li>
</ul>



<h3 class="wp-block-heading">Command &amp; Control</h3>



<p><strong>Proxy</strong></p>



<p><strong>Proxychains:</strong> Proxychains is a tool used to route network traffic through proxy servers and it supports multiple types of proxies such as SOCKS4, SOCKS5 and HTTPS. Proxychains can be combined with post-exploitation frameworks (e.g., Sliver, Cobalt Strike, etc.) to route the traffic through the implant.</p>



<p>The following image shows how the threat actor leveraged proxychains to execute Python scripts like certipy or noPac.py from the command and control server into the target environment.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/04/34568_021.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-43450" src="https://thedfirreport.com/wp-content/uploads/2025/04/34568_021.png" alt="" width="2000" height="779" srcset="https://thedfirreport.com/wp-content/uploads/2025/04/34568_021.png 2000w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_021-300x117.png 300w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_021-1024x399.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_021-768x299.png 768w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_021-1536x598.png 1536w" sizes="auto, (max-width: 2000px) 100vw, 2000px" /></a></p>





<p>The following image shows how proxychains works. The tool is very powerful because a threat actor can execute tools from the command-and-control server leaving less footprint compared to the artifacts that could be left if those were run on a monitored system.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/04/34568_022.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-43451" src="https://thedfirreport.com/wp-content/uploads/2025/04/34568_022.png" alt="" width="1152" height="615" srcset="https://thedfirreport.com/wp-content/uploads/2025/04/34568_022.png 1152w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_022-300x160.png 300w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_022-1024x547.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_022-768x410.png 768w" sizes="auto, (max-width: 1152px) 100vw, 1152px" /></a></p>





<p><strong>Application Layer Protocol: Web Protocols</strong></p>



<p><strong>Sliver:</strong> There were two binaries associated with Sliver present on the open directory: slv.bin and sliver-client_linux. We also note in the infrastructure section that this IP address was once observed as a Sliver C2.</p>



<p>It was not possible to analyze slv.bin and obtain further information out of the shellcode since it was encrypted. Likely, it was used to connect back to the same Sliver server. The binary sliver-client_linux, is the executable used to connect to a Sliver server, however no further usage of Sliver was observed.</p>



<p><strong>Protocol Tunneling</strong></p>



<p><strong>Powercat:</strong> A copy of Powercat was present on the host as powercat.ps1. Powercat is a PowerShell-based networking tool designed to replicate the functionalities provided by Netcat. It can be used for:</p>



<ul class="wp-block-list">
<li>Networking communications since it supports both TCP and UDP connections.</li>



<li>Bind and reverse shell creation to listen for incoming connections or connect back to a command-and-control server.</li>



<li>Data transfer over TCP, UDP and also DNS (dnscat2) which can be useful for evading network detection mechanisms.</li>



<li>Payload generation and encoding.</li>



<li>Traffic tunnelling to connect to systems that otherwise would be inaccessible.</li>
</ul>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/04/34568_023.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-43452" src="https://thedfirreport.com/wp-content/uploads/2025/04/34568_023.png" alt="" width="1281" height="952" srcset="https://thedfirreport.com/wp-content/uploads/2025/04/34568_023.png 1281w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_023-300x223.png 300w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_023-1024x761.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/04/34568_023-768x571.png 768w" sizes="auto, (max-width: 1281px) 100vw, 1281px" /></a></p>
