                            <h1 class="title-post" itemprop="name">Simple Methods to Spot Disabled Antivirus (Windows Defender) on Windows Systems</h1>
                    <div class="date">
            <span itemprop="datePublished"><a href="https://threat.boutique/archive/2026-01" title="Show all posts made on this month">18 January 2026</a></span>
            - Posted in            <span itemprop="articleSection"><a href="https://threat.boutique/category/incident-response">Incident Response</a></span> by            <span itemprop="author"><a href="https://threat.boutique/author/b3vSclAuunBMsdt">Jok3r</a></span> -

                    <div class="featured-image">
                <a href="https://threat.boutique/2026/01/simple-methods-to-spot-disabled-antivirus-windows-defender-on-windows-systems"><img src="https://threat.boutique/content/images/20260118164734-exclusion.jpg" alt="Simple Methods to Spot Disabled Antivirus (Windows Defender) on Windows Systems"/></a>
            </div>
                                        <div class="post-body post-1768739572" itemprop="articleBody">
            <h4><strong>Background:</strong></h4>

<p>During incident response, it’s not always the case that advanced, highly sophisticated AV bypass techniques are used. Sometimes, attackers rely on simple, out-of-the-box methods to evade detection, such as excluding their tools using legitimate features or disabling your antivirus through readily available means. In this example, we will focus specifically on Microsoft Defender.</p>

<h4><strong>Methods of detection:</strong></h4>

<p>One method is to disable antivirus through policy settings, which results in Windows generating a trace in C:\Windows\System32\GroupPolicy\Machine\Registry.pol
<img src="https://threat.boutique/content/images/20260118165914-disabledefenderfromgrouppolicyfiletraces.png" alt="enter image description here" />
Monitoring these files is extremely important for forensic investigations.</p>

<p>Another method is to monitor changes that occur when interacting (Disabling defender) with Microsoft Defender via PowerShell. During this interaction, services.exe targets the registry key HKLM\SYSTEM\CurrentControlSet\Services\WdNisDrv\Start. The WdNisDrv record  is responsible for Windows Defender.</p>

<p><img src="https://threat.boutique/content/images/20260118170445-disabledefenderfrompowershelltraces.png" alt="enter image description here" /></p>

<p>Another method used by attackers is to add their malicious files or folders to the exclusion list using the Windows Defender user interface (UI). These interactions are easily detectable in the event logs, as any scenario involving exclusions will result in the system interacting with the HKLM\Software\Microsoft\Windows Defender\Exclusions registry key. Whether exclusions are added through the UI or manually via the registry, the process ultimately involves this registry path as the final destination.
<img src="https://threat.boutique/content/images/20260118171314-exclusionEventDefender.png" alt="enter image description here" />
<img src="https://threat.boutique/content/images/20260118171416-ExclusionOVerRegistryViaPolicies.png" alt="enter image description here" /></p>

<p>In some cases, Windows systems may temporarily exclude certain paths that are not visible in the Defender UI. These exclusions are often used to install Windows Defender updates.</p>

<p>The relevant registry path is HKLM\Software\Microsoft\Windows Defender\Exclusions\TemporaryPaths.</p>

<p><img src="https://threat.boutique/content/images/20260118171831-temppathhiddenplace.png" alt="enter image description here" /></p>

<h4><strong>Conclusion:</strong></h4>

<p>While attackers may sometimes employ advanced techniques to bypass antivirus protection, many rely on simple, easily detectable methods to evade Microsoft Defender. By monitoring group policy changes, PowerShell interactions, registry modifications, and exclusions—both visible and hidden—incident responders can identify signs of tampering or attempted evasion. Vigilant oversight of these straightforward indicators remains essential for effective detection and response.</p>
