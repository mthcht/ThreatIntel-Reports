<div  class='av-special-heading av-a36gy4-b2d5b2ae3f2ba5e7836218919c8c8431 av-special-heading-h2 blockquote modern-quote  avia-builder-el-1  el_after_av_heading  el_before_av_textblock '><h2 class='av-special-heading-tag'  itemprop="headline"  >Overview</h2><div class="special-heading-border"><div class="special-heading-inner-border"></div></div></div>
<section  class='av_textblock_section av-k66hx8rk-3172ac2b6ea5ab0195e1ea17bb9235fd'  itemscope="itemscope" itemtype="https://schema.org/BlogPosting" itemprop="blogPost" ><div class='avia_textblock'  itemprop="text" ><p>Unlike other state sponsored threat actors, SectorC08 appears to be only concerned with a single target: Ukraine. Artifacts of their likely activity have been found as far back as 2013 and up till today their modus operandi in their initial stages of operation has not changed much.</p>
<p>We analyzed over 50 of their executable malware files found very recently in order to look at similarities, differences, and outliers. We found that while a few samples still used SectorC08&#8217;s executable file structure which contained batch scripts which were split out into many files (e.g. Wariables.cmd) or batch scripts together with a decoder executable and an encoded executable, most of them followed the structure we will be detailing below.</p>
</div></section>
<div  class='av-special-heading av-8ixfpo-185b54fa86cd0382cbf274264cea8ddc av-special-heading-h2 blockquote modern-quote  avia-builder-el-3  el_after_av_textblock  el_before_av_textblock '><h2 class='av-special-heading-tag'  itemprop="headline"  >Example of a Typical First Stage Structure (a8f849d536481d7d8a0fa59a7bcc03dd3387ab4cc14c0342371ae295817f505c)</h2><div class="special-heading-border"><div class="special-heading-inner-border"></div></div></div>
<section  class='av_textblock_section av-k66hxm6w-269ada2514b5f54006daaa5bbd4451f2'  itemscope="itemscope" itemtype="https://schema.org/BlogPosting" itemprop="blogPost" ><div class='avia_textblock'  itemprop="text" ><p>All samples which we can confirm came in the months of May and June used the same structure in their malware which we will be describing below: a 7zSFX archive which opens a password protected WinRARSFX archive, which then attempts to use a version of wget to download its third stage malware which is another WinRARSFX archive such as UltraVNC.</p>
</div></section>
<section  class='av_textblock_section av-k66hxwy9-7015360c25c19da46e409defc20a8303'  itemscope="itemscope" itemtype="https://schema.org/BlogPosting" itemprop="blogPost" ><div class='avia_textblock'  itemprop="text" ><p> <br />
<strong>Fake Documents</strong><br />
 <br />
Some of the malware samples we found contained an embedded fake document in them pertaining to Ukrainian issues. We observed six such embedded fake documents which were sometimes reused against different targets. These documents are opened from the embedded batch file in the 7zSFX archive environment.</p>
</div></section>

</div></section>

</div></section>

<section  class='av_textblock_section av-k66i0vsi-e5f69888b72a5ecb4dfd640f2e724a6e'  itemscope="itemscope" itemtype="https://schema.org/BlogPosting" itemprop="blogPost" ><div class='avia_textblock'  itemprop="text" ><p>The fake documents are always in Ukrainian and pertain to Ukrainian issues such as legal, political, military or police issues.</p>
<p>By comparing the document content date to the malware internal versioning code (described later) and from our knowledge of the malware’s previous versioning codes and dates, we can conclude that when the malware internal versioning code corresponds to a date, it is at least a roughly accurate timestamp and we can create a partial timeline of events.</p>

<strong>Basic Anti-Analysis</strong><br />
 <br />
At the start of this batch script, the malware looks for Wireshark and Process Explorer using the TaskList command. If any of these exist, the script exits using an unspecified label “exit”. But due to an error in their programming logic, this does not actually do everything which the attacker thinks it does.</p>
</div></section>

While SectorC08 sometimes changes the WinRARSFX password (or simply uses another 7zSFX unprotected archive), we observed this particular password being used at least 11 times across their various malware samples.
