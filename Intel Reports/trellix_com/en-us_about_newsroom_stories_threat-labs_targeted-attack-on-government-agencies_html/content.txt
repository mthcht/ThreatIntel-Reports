        <h1 class="entry-title">Targeted Attack on Government Agencies</h1> 
        <p class="dateline">By <b><a href="/about/newsroom/stories/contributors/sushant-kumar-arya/" datalink-id="newco:about:newsroom:stories:contributors:sushant-kumar-arya:" datalink-type="internal">Sushant Kumar Arya</a>, <a href="/about/newsroom/stories/contributors/mohsin-dalla/" datalink-id="newco:about:newsroom:stories:contributors:mohsin-dalla:" datalink-type="internal">Mohsin Dalla</a></b> · July 13, 2022</p> 
        <h2 class="h4">Executive summary</h2> 
        <p>The Trellix Email Security Research Team has discovered a malicious campaign targeting government agencies of Afghanistan, India, Italy, Poland, and the United States since 2021. The attack starts with a spear phishing email with a geo-political theme. The spear phishing emails were themed around India Afghanistan relationship. Attacker used politics as a lure to trick users into clicking on a malicious link. The email used for this phishing attack contains an attachment or a weaponized URL that delivers an Excel sheet. Upon opening the Excel sheet, Excel executes an embedded malicious macro which then decrypts and installs a Remote Access Trojan (AysncRAT &amp; LimeRAT) and maintains persistence. Once the Remote Access Trojan is installed on the victim machine, it establishes communication with a Command-and-Control server used to exfiltrate victim data. The Remote Access Trojan is capable of taking screenshots, capturing keystrokes, recording credentials/confidential information, and adding infected systems to botnets. It can also perform network discovery and move laterally to other systems in the affected organization. The email used in this attack originated from the South Asia region which suggests the involvement of a South Asian threat actor. Trellix Email Security has detection coverage for this malicious campaign.</p> 
        <img src="/en-us/img/newsroom//stories/targeted-attack-on-government-agencies-img1.jpg" alt="Figure 1. Target countries">

        <h2 class="h4">Threat landscape</h2> 
        <p>The Trellix Email Security product can follow the entire attack chain and analyze the final payload. In this scenario, it followed the chain: EMAIL -&gt; URL -&gt; ZIP -&gt; XLS -&gt; Macro. Finally, our threat database was able to detect the malicious macro performing decryption, creating an executable object, performing process injection, and utilizing other malicious techniques. Trellix Email Security has detection for the malicious Excel sheet with name - <b>FE_APT_Dropper_Macro_DoubleHide_1.</b></p> 
        <img src="/en-us/img/newsroom//stories/targeted-attack-on-government-agencies-img2.jpg" alt="Figure 2. Attack timeline"> 
        <figcaption style="font-size:12px;margin-top:0px;color:#a0a0a0;">

        <br> 
        <h2 class="h4">Attack timeline</h2> 
        <p>As seen in Figure 2, the attack was active for over a year. The attacker sent emails for a short interval and then went back into hiding. This was followed by subsequent similar waves. The first wave of attack was noticed during March-April 2021, followed by another in July 2021, then again in December 2021, and most recently during end of March 2022.</p> 
        <h2 class="h4">Email details</h2> 
        <p>The attackers used the free mail service Gmail to send the spear phishing emails. Based on email header analysis, it was evident that the emails originated from Google servers and were sent from the South Asia region. The time zone of the email sender <b>(+0500 UTC)</b> further suggests the involvement of South Asian threat actors.</p> 
        <img src="/en-us/img/newsroom//stories/targeted-attack-on-government-agencies-img3.jpg" alt="Figure 3. Email headers"> 
        <figcaption style="font-size:12px;margin-top:0px;color:#a0a0a0;">
         <b>Figure 3. Email headers</b>
        </figcaption> 
        <br> 
        <p>The spear phishing email was themed around geopolitical news related to India like <b>"Indian Nationals ( who were hidden in Kabul ) Killing in Kabul Tonight"</b> and <b>“Indian workers missing from the dam project.”</b> More recently, the email used a <b>COVID</b> theme with the subject - <b>"31 Covid Deaths In 24 Hours: Information campaign by NDTV"</b>. The email had a Google drive link serving a malicious ZIP file. In some cases, the malicious ZIP was sent as an email attachment. The ZIP contains an Office document which is used to drop a RAT (Remote Access Trojan).</p>

        <h2 class="h4">Technical details</h2> 
        <p>The document file (DOC/XLS) acts as a dropper, which drops and executes a file named "msword.exe". The Excel sheet contains a VBA macro which is enabled when the document file is opened. The malicious executable code is stored in the document file itself (within a form text field) in the base64 encoded format. The VBA macro reads the base64 content, decodes it, and then decrypts the decoded content with a hardcoded XOR key. Multiple levels of base64 decoding and XOR decryption are used to obfuscate the malicious executable file.</p>

        <p>Upon execution, "msword.exe" drops the RAT files shown in the table above. These RAT executables are obfuscated using “Crypto Obfuscator For .Net”. “msword.exe” then starts the process “igfx.exe” which performs the following actions:</p> 
        <ul> 
         <li> Checks the .NET version in the registry; based on the installed version, renames the compatible RAT file to “excel.exe” </li> 
         <li>Checks the registry keys to determine the .NET version in the order listed below. If found, a version of the runtime file (AsyncRAT) is picked corresponding to the .NET version. If none of the registry keys are found, the file “3_5” (LimeRAT) is used. 
          <ul> 
           <li>HKLM\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4</li> 
           <li>HKLM\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4.5</li> 
           <li>HKLM\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4.5.1</li> 
           <li>HKLM\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4.5.2</li> 
           <li>HKLM\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4.6</li> 
           <li>HKLM\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4.6.1</li> 
           <li>HKLM\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4.7</li> 
           <li>HKLM\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4.7.2</li> 
          </ul> </li> 
         <li>Sets the file attributes of “excel.exe” to hidden and read-only.</li> 
         <li>Adds a “Run” registry entry for persistence.</li> 
         <li>Deletes the unused RAT executable files.</li> 
         <li>Starts the “excel.exe” process.</li> 
        </ul>
