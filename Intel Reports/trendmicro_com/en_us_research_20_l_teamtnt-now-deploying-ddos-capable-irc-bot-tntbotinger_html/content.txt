	<h1 class="article-details__title">TeamTNT Now Deploying DDoS-Capable IRC Bot TNTbotinger</h1>
	<p class="article-details__description">We discuss TeamTNT’s latest attack, which involves the use of the group’s own IRC (Internet Relay Chat) bot. The IRC bot is called TNTbotinger and is capable of distributed denial of service (DDoS).</p>

            <div class="richText">
	
    


	
		<div>
			<p>Earlier this year, we saw how  the cybercrime group TeamTNT attacked <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/virtualization-and-cloud/coinminer-ddos-bot-attack-docker-daemon-ports">exposed Docker APIs </a> using the XMRig cryptocurrency miner. Over time, we observed how TeamTNT expanded the functionality of its attacks, which has come to include the stealing of Amazon Web Services (AWS) secure shell (SSH) credentials and a self-replicating behavior for propagation.</p>
<p>Here we discuss TeamTNT’s latest attack, which involves the use of the group’s own IRC (Internet Relay Chat) bot. The IRC bot is called TNTbotinger and is capable of distributed denial of service (DDoS).</p>
<p>It’s important to note that the attackers must first be able to perform remote code execution (RCE) on the initial target machine in order to successfully wage this attack on a system. Malicious actors can perform RCE by exploiting misconfiguration issues, abusing unpatched <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/definition/vulnerability">vulnerabilities</a>, and taking advantage of security flaws such as weak or reused passwords and keys, or leaked credentials.</p>
<h2><span class="body-subhead-title">Technical analysis</span></h2>
<p>The initial spread starts with a malicious shell script that’s run on a victim machine. The shell script checks for the presence of the <i>/dev/shm/.alsp</i> file, which can also be an indicator of compromise. If the file is not found, the script starts doing its job.</p>

		</div>
	

</div>
            <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/20/l/teamtnt-now-deploying-ddos-capable-irc-bot-tntbotinger/Fig%201%20TeamTNT%20Now%20Deploying%20DDoS-Capable%20IRC%20Bot%20TNTbotinger.png" alt="The malicious script checks for the presence of the /dev/shm/.alsp file in a system."/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 1. The malicious script checks for the presence of the /dev/shm/.alsp file in a system</figcaption>
			
		</div>
	</figure>

</div>
            <div>




    
    
    <div class="richText">
	
    


	
		<div>
			<p>The script will then try to install <i>curl, wget, bash, make, gcc,</i> and <i>pnscan</i> packages. </p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/20/l/teamtnt-now-deploying-ddos-capable-irc-bot-tntbotinger/Fig%202%20TeamTNT%20Now%20Deploying%20DDoS-Capable%20IRC%20Bot%20TNTbotinger.png" alt="The malicious script attempts to install curl, wget, make, gcc, and pnscan packages"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 2. The malicious script attempts to install curl, wget, make, gcc, and pnscan packages</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>Because of the package managers used in the malicious script, specifically <i>apt-get</i> and <i>yum</i>, we assume that the authors implemented support for both Debian and Red Hat-based distributions of Linux.</p>
<p>The script will then attempt to download and execute multiple binaries, including <i>pnscan</i>, a tool used for port scanning. This tool is also downloaded manually in case it is not found in the expected directory.</p>
<p>The following are the executed binaries in this attack:</p>
<ul>
<li><span class="rte-red-bullet">/dev/shm/sbin</span></li>
<li><span class="rte-red-bullet">/usr/bin/tshd</span></li>
<li><span class="rte-red-bullet">/usr/bin/kube</span></li>
<li><span class="rte-red-bullet">﻿/usr/bin/bioset</span></li>
</ul>
<p>Afterward, the script steals several pieces of confidential information from the infected system, such as:<br />
</p>
<ul>
<li><span class="rte-red-bullet">﻿RSA (Rivest-Shamir-Adleman) keys used for SSH access (AWS path included)</span></li>
<li><span class="rte-red-bullet">Bash history</span></li>
<li><span class="rte-red-bullet">AWS and Docker configuration files</span></li>
<li><span class="rte-red-bullet">/etc group, /etc/passwd, /etc/shadow, /etc/gshadow</span></li>
</ul>
<p>The malicious actors will then upload this stolen information using a <a href="https://www.winzip.com/win/en/tgz-file.html#:~:text=A%20TGZ%20file%20is%20a,for%20easier%20distribution%20or%20backup.&amp;text=TGZ%20file." style="background-color: rgb(255,255,255);" target="_blank">TGZ</a> (tar.gz) file via an HTTP POST request to an attacker-provided URL. We suspect that the collected information will serve as a knowledge base for the improvement of subsequent attacks.<br />
</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/20/l/teamtnt-now-deploying-ddos-capable-irc-bot-tntbotinger/Fig%203%20TeamTNT%20Now%20Deploying%20DDoS-Capable%20IRC%20Bot%20TNTbotinger.png" alt="Stolen information from an infected machine is uploaded via a TGZ file to a malicious URL"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 3. Stolen information from an infected machine is uploaded via a TGZ file to a malicious URL</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>The script also tries to find accessible devices, based on the output of the <i>ip route</i> command, which would show routes to accessible networks. This information is then passed to the <i>pnscan</i> tool for a scan of the active SSH daemons on the network. The keys found on the system are used for authentication attempts on the newly discovered devices. If these attempts are successful, the same payload is deployed on the new devices and the attack propagates.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/20/l/teamtnt-now-deploying-ddos-capable-irc-bot-tntbotinger/Fig%204%20A%20TeamTNT%20Now%20Deploying%20DDoS-Capable%20IRC%20Bot%20TNTbotinger.png" alt="Image 1 of 2. The malicious script performs a scan of active SSH daemons on an infected network and attempts to use stolen keys to access network-connected devices"/>
		
		<div class="caption-image-container ">
			
			
		</div>
	</figure>

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/20/l/teamtnt-now-deploying-ddos-capable-irc-bot-tntbotinger/Fig%204%20A%20TeamTNT%20Now%20Deploying%20DDoS-Capable%20IRC%20Bot%20TNTbotinger.png" alt="Image 2 of 2. The malicious script performs a scan of active SSH daemons on an infected network and attempts to use stolen keys to access network-connected devices"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 4. The malicious script performs a scan of active SSH daemons on an infected network and attempts to use stolen keys to access network-connected devices</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<h2><span class="body-subhead-title">A closer look at relevant binaries</span></h2>
<p>The binary target platform is CPUs based on the x86-64 instruction set. The first layer of all these binaries is packed by the well-known UPX packer.</p>
<p><b>/dev/shm/sbin</b></p>
<p>The binary is compiled using the Go compiler and contains an ELF (Executable and Linkable Format) file that uses AES (Advanced Encryption Standard) encryption. We presume that the packer used is a Go version of LaufzeitCrypter<i>.</i>  </p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/20/l/teamtnt-now-deploying-ddos-capable-irc-bot-tntbotinger/Fig%205%20TeamTNT%20Now%20Deploying%20DDoS-Capable%20IRC%20Bot%20TNTbotinger.png" alt="A Go-compiled binary that contains an AES-encrypted ELF file"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 5. A Go-compiled binary that contains an AES-encrypted ELF file</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>After decrypting the file, we found the binary’s final payload: an XMRig cryptocurrency miner.</p>
<p><b>/usr/bin/tshd</b></p>
<p>This bind shell listens on TCP (Transmission Control Protocol) port 51982. The communication is encrypted with a hard-coded key.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/20/l/teamtnt-now-deploying-ddos-capable-irc-bot-tntbotinger/Fig%206%20TeamTNT%20Now%20Deploying%20DDoS-Capable%20IRC%20Bot%20TNTbotinger.png" alt="The bind shell that listens on TCP port 51982"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 6. The bind shell that listens on TCP port 51982</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p><b>/usr/bin/bioset</b></p>
<p>This is a bind shell that listens on TCP port 1982. The communication is encrypted using the <a href="https://www.schneier.com/academic/blowfish/" target="_blank">Blowfish encryption algorithm</a> with a hard-coded key. After analysis, we discovered that this implementation does not work correctly on some platforms. The binary also renames its process name to <i>systemd</i>.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/20/l/teamtnt-now-deploying-ddos-capable-irc-bot-tntbotinger/Fig%207%20TeamTNT%20Now%20Deploying%20DDoS-Capable%20IRC%20Bot%20TNTbotinger.png" alt="The bind shell that listens on TCP port 1982"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 7. The bind shell that listens on TCP port 1982</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div class="responsive-table-wrap">
			<p><b>/usr/bin/kube</b></p>
<p>This binary is Go-compiled and contains an AES-encrypted ELF file. This is dynamically loaded during execution, and the same packer with the Go version of LaufzeitCrypter is used. The AES key and initialization vector (IV) are hard-coded in the binary.</p>

<h2><span class="body-subhead-title">Conclusion and security recommendations</span></h2>
<p>The Linux threat landscape is constantly evolving. This latest attack from TeamTNT is a good example of how the whole network segment, including the cloud, could be compromised by malicious actors. We are seeing just how serious they are in ensuring the increased success rates and stability of their attacks, as evidenced here by TeamTNT’s use of the wget/curl binaries for payload deployment and use of bind shell redundancy.</p>
<p>In a successful TNTbotinger attack, attackers will be able to infiltrate an infected system. Once inside, they will be able to see vulnerable instances on accessible network segments, and they can perform RCE on devices that are supposed to be shielded from the outside world.</p>
<p>It’s important for enterprises to adopt stringent security practices such as the following to keep their systems secure:<br />
</p>
<ul>
<br />
<li><span class="rte-red-bullet">Implement policies that prioritize <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/virtualization-and-cloud/automating-security-continuous-monitoring-and-auditing-in-devops" target="_blank">continuous monitoring and auditing</a> of devices, especially those used to access the office network.</span></li>
<li><span class="rte-red-bullet">Adhere to the <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/virtualization-and-cloud/securing-the-4-cs-of-cloud-native-systems-cloud-cluster-container-and-code">principle of least privilege</a> when granting permissions.</span></li>
<li><span class="rte-red-bullet">Regularly <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/vulnerabilities-and-exploits/virtual-patching-patch-those-vulnerabilities-before-they-can-be-exploited">patch and update systems</a> to reduce exposure to vulnerabilities and other critical threats.</span></li>
<li><span class="rte-red-bullet">Ensure that <a href="https://blog.trendmicro.com/are-your-passwords-secure-enough/" target="_blank">passwords are strong</a>. <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/internet-of-things/the-first-steps-in-effective-iot-device-security">Change default passwords</a> and adjust security settings based on the enterprise’s needs.</span></li>
</ul>
