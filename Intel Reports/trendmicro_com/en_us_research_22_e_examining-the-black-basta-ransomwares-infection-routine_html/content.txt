            <div class="richText">
	
    


	
		<div>
			<p>Black Basta, a new <a href="https://www.trendmicro.com/vinfo/us/security/definition/ransomware">ransomware</a> gang, has swiftly risen to prominence in recent weeks after it caused massive breaches to organizations in a short span of time.</p>
<p>On April 20, 2022, a user named Black Basta posted on underground forums known as XSS.IS and EXPLOIT.IN to advertise that it intends to buy and monetize corporate network access credentials for a share of the profits. The advertisement also specified that it was looking for organizations based in the United States, Canada, United Kingdom, Australia, and New Zealand, which are all English-speaking countries. A <a href="https://blog.minerva-labs.com/new-black-basta-ransomware-hijacks-windows-fax-service" target="_blank">report</a> noted that malicious actors acquired stolen credentials from some darknet websites that peddle an enormous amount of exfiltrated data to the underground market.</p>
<p>On April 26, Twitter user PCrisk <a href="https://twitter.com/pcrisk/status/1518881420910936065" target="_blank">tweeted</a> about the new Black Basta ransomware that appends the extension .basta and changes the desktop wallpaper.</p>
<p>This blog entry takes a closer look at the Black Basta ransomware and analyzes this newcomer’s familiar infection techniques.</p>
<p><span class="body-subhead-title">The infection routine</span></p>
<p>Black Basta ransomware needs administrator rights to run. It otherwise displays a command prompt message as shown on Figure 1.</p>

		</div>
	

</div>
            <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/examining-the-black-basta-ransomwares-infection-routine/blackbasta01PII.png" alt="Figure 1. A command prompt is displayed if Black Basta ransomware is not run with administrator rights"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 1. A command prompt is displayed if Black Basta ransomware is not run with administrator rights.</figcaption>
			
		</div>
	</figure>

</div>
            <div>




    
    
    <div class="richText">
	
    


	
		<div>
			<p>After running the ransomware as administrator, it removes shadow copies, disables Windows recovery and repair, and boots the PC in safe mode.</p>
<ul>
<li><span class="rte-circle-bullet" style="font-weight: normal;">C:\Windows\SysNative\vssadmin.exe delete shadows /all /quiet</span></li>
<li><span class="rte-circle-bullet" style="font-weight: normal;">C:\Windows\SysNative\bcdedit.exe /deletevalue safeboot</span></li>
<li><span class="rte-circle-bullet" style="font-weight: normal;">C:\Windows\SysNative\bcdedit /set safeboot networkChanges</span></li>
</ul>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/examining-the-black-basta-ransomwares-infection-routine/blackbasta02PII.png" alt="Figure 2. Commands such as &#34;C:\Windows\SysNative\bcedit /set safeboot networkChanges&#34; are embedded in the binary and can be viewed easily"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 2. Commands such as &#34;C:\Windows\SysNative\bcedit /set safeboot networkChanges&#34; are embedded in the binary and can be viewed easily.</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>It also drops the following files, which will be used later when changing the desktop wallpaper and icons for encrypted files:</p>
<ul>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">%Temp%\fkdjsadasd.ico</span></span></li>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">%Temp%\dlaksjdoiwq.jpg</span></span></li>
</ul>
<p>Before booting the infected device into safe mode, it changes the desktop wallpaper by dropping the .jpg file into the %temp% folder and creating the following registry entry:</p>
<ul>
<li><span class="rte-circle-bullet" style="font-weight: normal;">Key: HKCU\Control Panel\Desktop;  Value: Wallpaper; Data:%Temp%\dlaksjdoiwq.jpg;</span></li>
</ul>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/examining-the-black-basta-ransomwares-infection-routine/blackbasta03.png" alt="Figure 3. The registry entry created after Black Basta ransomware changes the wallpaper on the infected machine"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 3. The registry entry created after Black Basta ransomware changes the wallpaper on the infected machine</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">

		<div class="caption-image-container ">
			<figcaption>Figure 4. The desktop wallpaper created by the ransomware from the .jpg file dropped in the %temp% folder</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>After changing the desktop wallpaper, it then adds the following registry keys to change the icon of the encrypted files with the .basta extension:</p>
<ul>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">HKLM\SOFTWARE\Classes\.basta</span></span></li>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">HKLM\SOFTWARE\Classes\.basta\DefaultIcon data: %TEMP%\fkdjsadasd.ico</span></span></li>
</ul>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/examining-the-black-basta-ransomwares-infection-routine/blackbasta05.png" alt="Figure 5. The registry keys added by the ransomware to change the icon of the files with the .basta extension"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 5. The registry keys added by the ransomware to change the icon of the files with the .basta extension</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>The ransomware proceeds to encrypt files while the device is in safe mode, appending all encrypted files with the .basta extension. The ransom note is found in all the folders the ransomware has affected.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/examining-the-black-basta-ransomwares-infection-routine/blackbasta06.png" alt="Figure 6. The infected files shown with the .basta extension"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 6. The infected files shown with the .basta extension</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>The ransom note indicates the malicious actor’s onion site and a company ID. Despite running the same ransomware (SHA256 hash: 5d2204f3a20e163120f52a2e3595db19890050b2faa96c6cba6b094b0a52b0aa) on different virtual machines, the company ID the gang provides is the same across all devices.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">

    <div class="richText">
	
    


	
		<div>
			<p>Using another binary (SHA256 hash: 7883f01096db9bcf090c2317749b6873036c27ba92451b212b8645770e1f0b8a), a different company ID is shown on the ransom note. The files are likewise appended with the .basta extension.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/examining-the-black-basta-ransomwares-infection-routine/blackbasta08PII.PNG" alt="Figure 8. A different company ID is given when another binary is used"/>
		
		<div class="caption-image-container ">

		</div>

    <div class="richText">
	
    


	
		<div>
			<p><span class="body-subhead-title">Analyzing the infection routine</span></p>
<p>Black Basta’s recent entry to the cybercrime world suggests that information about their operations is still limited. According to a <a href="https://www.bleepingcomputer.com/news/security/new-black-basta-ransomware-springs-into-action-with-a-dozen-breaches/" target="_blank">report</a>, the gang has neither started marketing its operations nor has it begun recruitment of affiliates in underground forums. Based on advertisements they posted before the attacks, the malicious actor likely uses stolen credentials — purchased in darknet websites or underground forums — to get into an organization’s system.</p>
<p>We probed further and found that the company ID written in the ransom note is hardcoded in the binary file.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/examining-the-black-basta-ransomwares-infection-routine/blackbasta09PII.PNG" alt="Figure 9. The company ID in the ransom note is hardcoded in the binary file"/>
		
		<div class="caption-image-container ">

		</div>

    <div class="richText">
	
    


	
		<div>
			<p>Black Basta attempts to delete shadow copies using vssadmin.exe and boots the device in safe mode using bcdexit.exe from different paths, specifically, %SysNative% and %System32%.</p>

		</div>
	

</div>


    
    
    <div class="image">

		<div>
			<p><span class="body-subhead-title">Extortion phase</span></p>
<p>For a newcomer in the field, Black Basta is quite prolific for having compromised at least a dozen organizations in just a few weeks. The group’s <a href="https://www.bleepingcomputer.com/news/security/new-black-basta-ransomware-springs-into-action-with-a-dozen-breaches/" target="_blank">first known attack</a> using the Black Basta ransomware occurred in the second week of April 2022. But an <a href="https://twitter.com/Arkbird_SOLG/status/1519452352763281408?s=20&amp;t=D9hGjGpXFgAVNxxok5S-lw" target="_blank">earlier sample</a> was also spotted back in February 2022 with the ransomware name “no_name_software,”  which appends the extension “encrypted” to encrypted files. According to some <a href="https://twitter.com/Arkbird_SOLG/status/1519452352763281408?s=20&amp;t=D9hGjGpXFgAVNxxok5S-lw" target="_blank">threat researchers</a>, it appears that Black Basta has been in development since early February 2022.</p>

<p>The gang carries out the extortion phase of its attacks on its Tor site, Basta News, which contains a list of all the victims who have not paid the ransom.</p>

		<div>
			<p><span class="body-subhead-title">Possible relation to an APT</span></p>
<p>Security researchers exchanged speculations on Twitter that Black Basta is possibly a rebranding of the <a href="https://www.trendmicro.com/vinfo/us/security/news/ransomware-spotlight/ransomware-spotlight-conti">Conti</a> ransomware operation. <a href="https://twitter.com/malwrhunterteam/status/1519301421958578177" target="_blank">MalwareHunterTeam</a> pointed out many similarities in its leak site, payment site, and negotiation style to those of Conti’s. Twitter user <a href="https://twitter.com/malwrhunterteam/status/1519301421958578177" target="_blank">Arkbird</a> echoed the same observation.<sup>  </sup>Lawrence Abrams of BleepingComputer also mentioned that the malicious actors behind Black Basta seem like they are exerting a lot of effort to avoid any resemblance to their previous identity.</p>
<p>We have also noticed some similarities between the Black Basta and Black Matter payment sites. Like Black Matter, Black Basta implements user verification on its Tor site. However, the leak site does not implement a session key.</p>

		<div>
			<p><span class="body-subhead-title">New findings: QAKBOT possibly related to Black Basta</span></p>
<p>Based on our analysis of another set of samples monitored within a 72-hour timeframe, we discovered a possible correlation between <a href="https://www.trendmicro.com/en_us/research/21/k/qakbot-loader-returns-with-new-techniques-and-tools.html">QAKBOT</a> and Black Basta ransomware. We observed the following:<br />
</p>
<ul>
<li><span class="rte-red-bullet">As with QAKBOT, the malware is downloaded and executed from a malicious Excel file. Similar to the typical routine of the QAKBOT binary, it then executes certain PowerShell commands as part of its staging phase.</span></li>
<li><span class="rte-red-bullet">From information gathered in our telemetry, we found the presence of the Black Basta ransomware within the 72-hour period in which it encrypted files on victims’ machine. Trend Micro detects this as Ransom.Win32.BASTACRYPT.YACEDT.</span></li>
<li><span class="rte-red-bullet">Other <a href="https://www.cynet.com/attack-techniques-hands-on/quakbot-strikes-with-quaknightmare-exploitation/">researchers</a> pointed out that QAKBOT deploys its version of the exploit PRINTNIGHTMARE (aka QUAKNIGHTMARE) during the same timeframe. We also observed the presence of the weaponized exploit. This, in turn, executes another binary with escalated privilege. Unfortunately, we were unable to find the presence of the file pointed by the hardcoded path. string to complete the attack chain. Trend Micro detects this as Trojan.Win64.QUAKNIGHTMARE.YACEJT.</span></li>
</ul>
<p>Malicious actors also use certain tools as seen through our sensors, but we were unable to obtain the complete kit. We have so far gathered paths related to the tools themselves that include the following:</p>
<ul>
<li><span class="rte-red-bullet">AdwareCleaner (C:\AdwCleaner\*)</span></li>
<li><span class="rte-red-bullet">Either PC-Clea</span>ner or Pervasive PSQL/SQL (C:\pvsw\*)</li>
</ul>
<p>The structure of the ransomware loader is also different from the external article. In this case, instead of dropping and executing the ransomware itself, the loader downloads to the device’s memory then uses reflective loading to launch the ransomware.</p>
<p>The information we have collected so far indicates that the malicious actor behind Black Basta possibly used QAKBOT as a new means to deliver the ransomware.<br />
</p>
<p><b><span class="body-subhead-title">Insights</span></b><br />
</p>
<p>The malicious actors could be using a unique binary for each organization that they target. This can be seen from the ransom note that they drop, which is hardcoded in the malware itself. A ransomware typically creates a unique ID for each victim despite being infected by the same executable. Their choice of target organizations also suggests this to be the case. They buy corporate network access credentials in underground markets, which could mean that they do not distribute their malware sporadically. Instead, they use a certain kind of binary or variant for a specific organization.<br />
</p>
<p><span class="body-subhead-title">Recommendations </span></p>
<p>Threat researchers suggest that the recent attacks by Black Basta can be seen as early manifestations of Conti’s rebranding efforts. True or not, organizations should keep a watchful eye against ransomware threats. An organization’s thorough assessment of its security posture and its implementation of solid cybersecurity defenses give it a better fighting chance against such threats. </p>
<p>To protect systems against similar attacks, organizations can establish security frameworks that allocate resources systematically for establishing a strong defense strategy against ransomware. Here are some best practices that organizations can consider: <br />
</p>
<p>Audit and inventory <br />
</p>
<ul>
<li><span class="rte-circle-bullet" style="font-weight: normal;">Take an inventory of assets and data</span></li>
<li><span class="rte-circle-bullet" style="font-weight: normal;">Identify authorized and unauthorized devices and software </span></li>
<li><span class="rte-circle-bullet" style="font-weight: normal;">Audit event and incident logs </span></li>
</ul>
<p>Configure and monitor </p>
<ul>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">Manage hardware and software configurations </span></span></li>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">Grant admin privileges and access only when necessary to an employee’s role </span></span></li>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">Monitor network ports, protocols, and services </span></span></li>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">Activate security configurations on network infrastructure devices such as firewalls and routers</span></span></li>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">Establish a software allowlist that only executes legitimate applications </span></span></li>
</ul>
<p>Patch and update </p>
<ul>
<li><span class="rte-circle-bullet" style="font-weight: normal;">Conduct regular vulnerability assessments </span></li>
<li><span class="rte-circle-bullet" style="font-weight: normal;">Perform patching or virtual patching for operating systems and applications</span></li>
<li><span class="rte-circle-bullet" style="font-weight: normal;">Update software and applications to their latest versions </span></li>
</ul>
<p>Protect and recover <br />
</p>
<ul>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">Implement data protection, backup, and recovery measures</span></span></li>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">Enable multifactor authentication (MFA) </span></span></li>
</ul>
<p>Secure and defend </p>
<ul>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">Employ sandbox analysis to block malicious emails </span></span></li>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">Deploy the latest versions of security solutions to all layers of the system, including email, endpoint, web, and network </span></span></li>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">Detect early signs of an attack such as the presence of suspicious tools in the system </span></span></li>
<li><span class="rte-circle-bullet"><span style="font-weight: normal;">Use advanced detection technologies such as those powered by AI and machine learning </span></span></li>
</ul>
<p>Train and test </p>
<ul>
<li><span class="rte-circle-bullet" style="font-weight: normal;">Regularly train and assess employees in security skills</span></li>
<li><span class="rte-circle-bullet" style="font-weight: normal;">Conduct red-team exercises and penetration tests </span></li>
</ul>
<p>A multilayered approach can help organizations guard possible entry points into their system (endpoint, email, web, and network). Security solutions can detect malicious components and suspicious behavior, which can help protect enterprises. </p>
