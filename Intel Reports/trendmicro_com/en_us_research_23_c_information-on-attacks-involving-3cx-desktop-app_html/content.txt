<p>In late March 2023, security researchers revealed that threat actors abused a popular business communication software from 3CX — in particular, <a href="https://www.sentinelone.com/blog/smoothoperator-ongoing-campaign-trojanizes-3cx-software-in-software-supply-chain-attack/">the</a> <a href="https://www.crowdstrike.com/blog/crowdstrike-detects-and-prevents-active-intrusion-campaign-targeting-3cxdesktopapp-customers/">reports</a> mention that a version of the 3CX VoIP (Voice over Internet Protocol) desktop client was being employed to target 3CX's customers as part of an attack.<br />
</p>
<p>On its forums, 3CX has <a href="https://www.3cx.com/community/threads/3cx-desktopapp-security-alert.119951/">posted an update</a> that recommends uninstalling the desktop app and using the <a href="https://www.3cx.com/blog/releases/web-client-pwa/">Progressive Web App</a> (PWA) client instead. The company also mentioned that they are working on an update to the desktop app.</p>
<p>For a more comprehensive scope of protection against possible attacks associated with the 3CX Desktop App, the Trend Micro XDR platform can help organizations mitigate the impact by collecting and analyzing extensive activity data from various sources. By applying XDR analytics to the data gathered from its native products, Trend Micro XDR generates correlated and actionable alerts.  </p>
<p>Trend Micro customers can also take advantage of Trend Micro Vision One™ to search for and monitor potential threats associated with the 3CX Desktop App, and to better understand observed attack vectors. For more information on how to utilize Trend Micro Vision One features, you may download the PDF guide <a href="https://documents.trendmicro.com/assets/guides/Vision-One-Guide-3CX-Desktop-App.pdf">here</a>.</p>
<p>Additional guidance for Trend Micro customers including help with protection and detection can be found on our <a href="https://success.trendmicro.com/dcx/s/solution/000292711?language=en_US">support page</a>.</p>
<h1><span class="body-subhead-title">What is the compromised application?</span></h1>
<p>The 3CX app is a private automatic branch exchange (PABX) software that provides several communication functions for its users, including video conferencing, live chat, and call management. The app is available on most major operating systems, including Windows, macOS, and Linux. Additionally, the client is available as a mobile application for both Android and iOS devices, while a Chrome extension and the PWA version of the client allow users to access the software through their browsers.</p>
<p>The issue was said to be limited to the Electron (non-web versions) of their Windows package (versions 18.12.407 and 18.12.416) and macOS clients (versions 18.11.1213, 18.12.402, 18.12.407 and 18.12.416).</p>
<p>According to the company’s website, more than 600,000 businesses and over 12 million daily users around the world use 3CX's VoIP IPBX software.</p>
<p><span class="body-subhead-title">How does the attack work?</span></p>
<p>The attack is reportedly a multi-stage chain in which the initial step involves a compromised version of the 3CX desktop app. Based on initial analysis, the MSI package (detected by Trend Micro as Trojan.Win64.DEEFFACE.A and Trojan.Win64.DEEFFACE.SMA) is the one that is compromised with possible trojanized DLLs, since the .exe file has the same name.</p>
<p>The infection chain begins with<i> 3CXDesktopApp.exe </i>loading <i>ffmpeg.dll </i>(detected as Trojan.Win64.DEEFFACE.A andTrojan.Win64.DEEFFACE.SMA). Next, <i>ffmpeg.dll </i>reads and decrypts the encrypted code from <i>d3dcompiler_47.dll</i> (detected as Trojan.Win64.DEEFFACE.A and Trojan.Wind64.DEEFACE.SMD3D).</p>
<p>The decrypted code seems to be the backdoor payload that tries to access the IconStorages GiHub page to access an ICO file (detected as Trojan.Win32.DEEFFACE.ICO) containing the encrypted C&amp;C server that the backdoor connects to in order to retrieve the possible final payload. In addition, <i>d3dcompiler_47.dll</i> also abuses <a href="https://nvd.nist.gov/vuln/detail/CVE-2013-3900" target="_blank">CVE-2013-3900</a> to make it appear that it is legitimately signed.</p>

<p><span class="body-subhead-title"><span class="rte-sub-menu-text">Execution flow</span></span></p>
<p>Upon execution, the MSI package installer will drop the following files that are related to malicious behavior. Trend Micro Smart Scan Pattern (cloud-based) TBL 21474.300.40 can detect these files as Trojan.Win64.DEEFFACE.A.</p>
<ul>
<li><span class="rte-red-bullet"><b><i>3CXDesktopApp.exe</i>: A normal file that is abused to load the trojanized DLL</b></span></li>
<li><span class="rte-red-bullet"><b><i>ffmpeg.dll</i>: A trojanized DLL used to read, load, and execute a malicious shellcode from <i>d3dcompiler_47.dll</i></b></span></li>
<li><span class="rte-red-bullet"><b><i>d3dcom</i></b></span><b><i>piler_47.dll</i>: A DLL appended with an encrypted shellcode after the <i>fe ed fa ce</i> hex string</b></li>
</ul>
<p>Some conditions are necessary for execution. For example, the sleep timestamp varies depending on the following conditions: First, it checks if the manifest file is present, as well as if it is using a specified date. If the file is not present or if it is using the specified date, the timestamp will generate a random number and use the formula <i>rand() % 1800000 + current date + 604800 </i>(604,800 is seven days).  After the date is computed, the malware will continue its routine.</p>
<p>Upon execution of <i>3CXDesktopApp.exe</i>, <i>ffmpeg.dll</i>, which seems to be a trojanized or patched DLL, will be loaded. It will still contain its normal functionalities, but it will have an added malicious function that reads <i>d3dcompiler_47.dll</i> to locate an encrypted shellcode after the <i>fe ed fa ce</i> hex strings.</p>

		<div>
			<p><span class="body-subhead-title">What is its potential impact?</span></p>
<p>Due to its widespread use and its importance in an organization’s communication system, threat actors can cause major damage (for example, by monitoring or rerouting both internal and external communication) to businesses that use this software.</p>
<p><span class="body-subhead-title">What can organizations do about it?</span></p>
<p>Organizations that are potentially affected should stop using the vulnerable version if possible and apply the patches or mitigation workarounds if these are available. IT and security teams should also scan for confirmed compromised binaries and builds and monitor for anomalous behavior in 3CX processes, with a particular focus on C&amp;C traffic. </p>
<p>Meanwhile, enabling behavioral monitoring in security products can help detect the presence of the attack within the system.</p>
