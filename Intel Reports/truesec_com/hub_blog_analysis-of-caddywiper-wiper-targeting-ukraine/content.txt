<h2>Execution Flow</h2>
<p>Upon start the wiper uses the API call DsRoleGetPrimaryDomainInformation to check if the computer is the primary domain controller by comparing to the hard coded value 0x5, that comes from the struct DSROLE_MACHINE_ROLE. If it is the primary domain controller it will exit. This is probably done because the threat actor is using the domain controller as the source of distribution of the wiper and not to ruin its own foothold.</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/DsRoleGetPrimaryDomainInformation.png?w=1385&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1647376069&#038;s=59b39b21e17d4c5fa364822e1487efcb" alt="" /></figure>
<p>The next part of the wiper is the file destruction part. It calls the function 0x4022A0 that iterates over the files, using the API calls that are resolved from the stack strings, and writes over the first 0xA00000 bytes with zeros.</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/file_destruction.png?w=1272&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1647376166&#038;s=f1f7edbc06710cc0d08cf3a66087ebb2" alt="" /></figure>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/file_destruction2.png?w=874&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1647376190&#038;s=24ae8dea37892b59bdd4dc583db331ec" alt="" /></figure>
<p>Then the wiper loops through the alphabet (0x18), starting with D all the way up to Z and then one additional iteration, and applies the data destruction from the function in 0x4022A0 to the files in every partition it finds.</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/partition_loop.png?w=1585&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1647376264&#038;s=c4883793564bc2da2490506812a70802" alt="" /></figure>
<p>This is the last iteration and has gone past Z to Z+1.</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/partition_loop_last.png?w=1666&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1647377236&#038;s=a59af1a666827d0ac9105876124e21fe" alt="" /></figure>
<p></p>
<p>Lastly the wiper loops through a list of open raw access to \\.\PHYSICALDRIVE9 &#8211; \\.\PHYSICALDRIVE0 and writing to it using IOCTL_DISK_SET_DRIVE_LAYOUT_EX (0x7c054) by using the API DeviceIoControl. By doing so it erases the Master Boot Record.</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/DeviceIoControl.png?w=768&#038;auto=compress%2Cformat&#038;fit=clip&#038;dm=1647377272&#038;s=e04ec1fdfaea94c368ee7b26a741d3d9" alt="" /></figure>
<p></p>
<h2>Detection</h2>
<p>Since the wiper is using stack strings for obfuscation of the part that interacts with the disk, that part can be used as Yara rule for detection.</p>
