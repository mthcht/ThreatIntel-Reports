These websites have the capability to change their background and logo depending on the user’s domain. The phishing site is stored in IPFS (InterPlanetary File System) and after reviewing the URLs used by the attacker, we noticed an increasing number of phishing emails containing IPFS URLs as their payload.&nbsp;</span><span>&nbsp;</span></p>
<!--more-->
<p><span>We have observed more than 3,000 emails containing phishing URLs that have utilized IPFS for the past 90 days and it is evident that IPFS is increasingly becoming a popular platform for phishing websites.</span><span>&nbsp;</span></p>
<h3>What’s with IPFS and why do attackers use it?</h3>
<p><span>IPFS was created in 2015 and is a distributed, peer-to-peer file-sharing system for storing and accessing files, websites, applications, and data. Contents are available through peers located worldwide, who might be transferring information, storing it, or doing both. IPFS can locate a file using its content address rather than its location. To be able to access a piece of content, users need a gateway hostname and the content identifier (CID) of the file.</span></p>
<p style="text-align: center;"><span><strong>https://&lt;Gateway&gt;/ipfs/&lt;CID Hash&gt;</strong></span><span>&nbsp;</span></p>
<p><span>Currently, most data is transferred across the internet using Hypertext Transfer Protocol (HTTP) which employs a centralized client-server approach. IPFS, on the other hand, is a project that aims to create a completely decentralized web that works through a P2P network.</span><span>&nbsp;</span></p>
<p><img loading="lazy" src="https://www.trustwave.com/hs-fs/hubfs/Web/Blogs/SpiderLab/18854_picture1r.webp?width=361&amp;height=143&amp;name=18854_picture1r.webp" alt="18854_picture1r" width="361" height="143" style="height: auto; max-width: 100%; width: 361px; margin-left: auto; margin-right: auto; display: block;" srcset="https://www.trustwave.com/hs-fs/hubfs/Web/Blogs/SpiderLab/18854_picture1r.webp?width=181&amp;height=72&amp;name=18854_picture1r.webp 181w, https://www.trustwave.com/hs-fs/hubfs/Web/Blogs/SpiderLab/18854_picture1r.webp?width=361&amp;height=143&amp;name=18854_picture1r.webp 361w, https://www.trustwave.com/hs-fs/hubfs/Web/Blogs/SpiderLab/18854_picture1r.webp?width=542&amp;height=215&amp;name=18854_picture1r.webp 542w, https://www.trustwave.com/hs-fs/hubfs/Web/Blogs/SpiderLab/18854_picture1r.webp?width=722&amp;height=286&amp;name=18854_picture1r.webp 722w, https://www.trustwave.com/hs-fs/hubfs/Web/Blogs/SpiderLab/18854_picture1r.webp?width=903&amp;height=358&amp;name=18854_picture1r.webp 903w, https://www.trustwave.com/hs-fs/hubfs/Web/Blogs/SpiderLab/18854_picture1r.webp?width=1083&amp;height=429&amp;name=18854_picture1r.webp 1083w" sizes="(max-width: 361px) 100vw, 361px"></p>
<p><span>With the IPFS configuration, shared files are distributed to other machines acting as nodes throughout the networked file system; hence it can be accessed whenever needed. The file is retrieved from any participating node on the network that has the requested content.</span></p>
<p><span>In a centralized network, data is not accessible if the server is down or if a link gets broken.&nbsp; Whereas with IPFS, data is persistent. Naturally, this extends to the malicious content stored in the network. Taking down phishing content stored on IPFS can be difficult because even if it is removed in one node, it may still be available on other nodes.</span></p>
<p><span>Another thing to consider is the difficulty of discovering malicious traffic in a legitimate P2P network. With data persistence, robust network, and little regulation, IPFS is perhaps an ideal platform for attackers to host and share malicious content.</span></p>
<h3>How do we identify IPFS URLs?</h3>
<p><span>As mentioned earlier, a CID is a label that is used to point to content in an IPFS network. Instead of location-based addressing, data is requested using the hash of that content.&nbsp;</span><span>IPFS uses sha-256 hashing algorithm by default.</span></p>
<p><span>The CID version 0 of IPFS was first designed to use base 58-encoded multihashes as the content identifiers. Version 0 starts with “Qm” and has a length of 46 characters.</span><span>&nbsp;</span></p>
<p><span>However, in the latest CID v1 it contains some leading identifiers that clarify exactly which representation is used, along with the content-hash itself. It includes a decoding algorithm links to existing software implementations for decoding CIDs.</span><span>&nbsp;</span></p>
<p><span>The subdomain gateways convert paths with custom bases like base16 to base32 or base36, in an effort to fit a CID in a DNS label:</span></p>
<p><span>Sample URL:&nbsp;</span><span>&nbsp;</span></p>
<p><span>dweb[.]link/ipfs/f01701220c3c4733ec8affd06cf9e9ff50ffc6bcd2ec85a6170004bb709669c31de9</span><span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;</span><span>4391a&nbsp;</span><span>&nbsp;</span><br><span>returns a HTTP 301 redirect:</span><span>&nbsp;<br></span><span>bafybeigdyrzt5sfp7udm7hu76uh7y26nf3efuylqabf3oclgtqy55fbzdi[.]ipfs[.]dweb[.]link</span><span>&nbsp;</span></p>
<p><span>IPFS links usually have a common format of:</span><span>&nbsp;</span></p>
<ul>
<li><span>https://ipfs[.]io/<strong>ipfs</strong>/<em>{46 random character string}</em>?(filename|key)=<em>{random character string}</em></span><span></span></li>
<li><span>https://ipfs[.]io/<strong>ipfs</strong>/<em>{46 random character string}</em>?filename=<em>{file name}</em>\.html</span><span>&nbsp;&amp;emailtoken=<em>{email address}</em></span><span></span></li>
<li><span>https://ipfs[.]io/<strong>ipfs</strong>/<em>{46 random character string}#{user email address}</em></span></li>
</ul>

<h3>Conclusion</h3>
<p><span>Phishing techniques have taken a leap by utilizing the concept of decentralized cloud services using IPFS.</span><span>&nbsp;</span></p>
<p><span>One of the main reasons why IPFS&nbsp;<u>has</u>&nbsp;become a new playground for phishing is that many web hosting, file storage or cloud services are now offering IPFS services. This means that there’s more flexibility for the phishers in creating new types of phishing URLs. In addition, the spammers can easily camouflage their activities by hosting their content in a legitimate web hosting services or use multiple URL redirection techniques to help thwart scanners using URL reputation or automated URL analysis.&nbsp;</span><span>&nbsp;</span></p>
<p><span>Keeping up to date with the latest technology and cyber threats is beneficial in preventing users from being victimized by web threats such as phishing. As always, we remind everyone to stay vigilant in this ever-changing digital landscape.</span><span>&nbsp;</span></p>
