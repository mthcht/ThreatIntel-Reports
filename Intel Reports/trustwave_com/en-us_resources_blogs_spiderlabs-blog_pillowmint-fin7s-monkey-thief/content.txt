Pillowmint is point-of-sale malware capable of capturing Track 1 and Track 2 credit card data. We came across Pillowmint a couple of times in the last year and there is not much information around on it. The malware has been attributed to the FIN7 group that has been actively attacking the hospitality and restaurant industry for the past three years. This is a notorious financially-motivated cybercriminal group also referred to as the <a href="https://www.trustwave.com/en-us/resources/blogs/spiderlabs-blog/operation-grand-mars-a-comprehensive-profile-of-carbanak-activity-in-201617/">Carbanak group</a>, after the Carbanak malware which it has used in the past.</p>
<!--more--><p>&nbsp;</p>
<h3 style="font-weight: bold;">Analysis: Installation</h3>
<p>Pillowmint is usually installed through a malicious shim database which allows the malware to persist in the system.</p>
<p>Shim databases are used by Windows Application Compatibility Framework -&nbsp; created by Microsoft so that legacy Windows applications will still work on newer Windows operating systems. This is done by overlaying code to a target process which is usually a legacy application. This will enable that application to run in a Windows operating system environment that is not designed for. A more elaborate explanation of shimming has been published in a Tech Community <a href="https://techcommunity.microsoft.com/t5/ask-the-performance-team/demystifying-shims-or-using-the-app-compat-toolkit-to-make-your/ba-p/374947" rel="noopener" target="_blank">article</a> from Microsoft.</p>
<p>By installing a malicious shim an attacker can leverage this Windows feature and use it to gain persistence in a compromised system.</p>

<p>To install a malicious shim database, the attacker invokes a Microsoft utility called sdbinst.exe through a PowerShell script. For example:</p>
<p>When the malicious shim database is registered, the SDB file is copied to the shim database path at %windir%\AppPatch\Custom\Custom64\{GUID.sdb}. The shim database is also added to the Window’s Application Compatibility Program Inventory and a registry key created HKLM\Software\Microsoft\Windows NT\CurrentVersion\AppCompatFlags\Custom\services.exe Value Name: {GUID}.sdb</p>

<p>The relative virtual address (RVA) in services.exe that the malware will be patching is at RVA 0x0000F93B. This is the RVA of <u>services.exe!ScRegisterTCPEnpoint</u> function. This will cause the malware shellcode to execute when the aforementioned services.exe function is called.</p>

<p>&nbsp;</p>
<p>As mentioned, the registry key (HKLM\SOFTWARE\Microsoft\DRM) is where the malicious payload is stored. In this case, this is the Pillowmint Trojan. Pillowmint is stored and compressed in the registry key. In order to decompress and launch it, small shellcode is executed to decompress and allocate it into the parent process memory space. Execution is then transferred to the payload.</p>

<h3 style="font-weight: bold;">Payload: Pillowmint</h3>
<p>Pillowmint injects its code to svchost.exe, but instead of using commonly monitored syscalls such as VirtualAlloc and WriteProcessMemory, It uses a mapping injection technique that utilizes CreateMapping, MapViewOfFIle, NtQueueApcThread, ResumeThread, CreateRemoteThread syscalls for stealthier process injection.</p>
<p><a style="display: inline;" href="https://npercoco.typepad.com/.a/6a0133f264aa62970b0264e2e1b6da200d-pi" class="asset-img-link"><img loading="lazy" class="asset  asset-image at-xid-6a0133f264aa62970b0264e2e1b6da200d image-full img-responsive" src="https://web.archive.org/web/20230329152204im_/https://npercoco.typepad.com/.a/6a0133f264aa62970b0264e2e1b6da200d-800wi" border="0" alt="Picture 6" title="Picture 6"></a></p>
<p>&nbsp;</p>
<p>This malware is capable of logging its own activity. This log is dropped in the path: "%WinDir%\System32\MUI" or “%WinDir%\System32\Sysvols” depending on the variant and it uses the file name <em>log.log</em>.&nbsp; It has 8 different levels of logging:</p>
<p><strong>Level 0&nbsp;</strong>- No logging<br><strong>Level 1 (CRITICAL)&nbsp;</strong>- it logs only critical activities such as exceptions, this is logged to the file named "critical.log" in the path "%WinDir%\&lt;system32 or sysnative&gt;\sysvols"</p>
<p>Example content of critical.log</p>

<h3 style="font-weight: bold;">Memory Scraper</h3>
<p>There are two versions that we have encountered so far. Depending on the Pillowmint version, the malware may contain three main threads. The older version has three threads:</p>
<ol>
<li>Memory Scraper</li>
<li>Process list updater</li>
<li>Process command</li>
</ol>
<p><a style="display: inline;" href="https://npercoco.typepad.com/.a/6a0133f264aa62970b0263e94fa042200b-pi" class="asset-img-link"><img loading="lazy" class="asset  asset-image at-xid-6a0133f264aa62970b0263e94fa042200b image-full img-responsive" src="https://web.archive.org/web/20230329152204im_/https://npercoco.typepad.com/.a/6a0133f264aa62970b0263e94fa042200b-800wi" border="0" alt="Picture 8" title="Picture 8"></a></p>
<p>&nbsp;</p>
<h3 style="font-weight: bold;">Thread 1: Credit Card Data - Memory Scraper</h3>
<p>This thread first sleeps for 10 seconds before a call back to the thread's main function.</p>
<p><a style="display: inline;" href="https://npercoco.typepad.com/.a/6a0133f264aa62970b0264e2e1b6e8200d-pi" class="asset-img-link"><img loading="lazy" class="asset  asset-image at-xid-6a0133f264aa62970b0264e2e1b6e8200d image-full img-responsive" src="https://web.archive.org/web/20230329152204im_/https://npercoco.typepad.com/.a/6a0133f264aa62970b0264e2e1b6e8200d-800wi" border="0" alt="Picture 9" title="Picture 9"></a></p>
<p>&nbsp;</p>
<p>Like any other typical PoS malware, Pillowmint iterates a list of processes and process them two at a time. it uses the API OpenProcess() using the&nbsp;PROCESS_VM_READ and PROCESS_QUERY_INFORMATION flags to obtain a handle then reads the memory’s content via ReadProcessMemory() API two chunks at a time. It then captures Track 1 and Track 2 credit card (CC) data. Depending on the Pillowmint version, it may encrypt the stolen CC data with AES encryption algorithm + Base64. Other versions may just encode the plain Credit Card Data it with Base64. This is then written to a file named "ldb_e.log" in Windows System directory.&nbsp;</p>
<p>The AES key used for encryption is also obscured and may be decoded with a simple algorithm:</p>
<p style="padding-left: 40px;"><code>decode_byte = encoded_byte –(index * 3) - 2</code></p>
<p>&nbsp;</p>
<p><a style="display: inline;" href="https://npercoco.typepad.com/.a/6a0133f264aa62970b0264e2e1b6ec200d-pi" class="asset-img-link"><img loading="lazy" class="asset  asset-image at-xid-6a0133f264aa62970b0264e2e1b6ec200d img-responsive" src="https://web.archive.org/web/20230329152204im_/https://npercoco.typepad.com/.a/6a0133f264aa62970b0264e2e1b6ec200d-800wi" border="0" alt="Picture 10" title="Picture 10"></a></p>

<h3 style="font-weight: bold;">Thread 3: Process Commands</h3>
<p>The older version we investigated came with remnants of command and control functionality. This third thread reads for attacker's command from either a registry key:</p>
<p style="padding-left: 40px;"><code>HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\Tcpip\Parameters\Interfaces</code><br><code>command&nbsp; = &lt;command&gt;</code></p>
<p>or from a file:</p>
<p style="padding-left: 40px;"><code>%WinDir%\&lt;system32 or sysnative&gt;\sysvols\commands.txt</code></p>
<p>There are only two commands:</p>
<h3 style="font-weight: bold;">&nbsp;</h3>
<h3 style="font-weight: bold;">Command 1&nbsp;</h3>
<p>s:&nbsp;or&nbsp;S:&nbsp;- terminate malware process</p>
<p style="padding-left: 40px;"><code>&nbsp;&nbsp;&nbsp; if ( pCommand[1] == ':' &amp;&amp; !((*pCommand - 0x53) &amp; 0xDF) )// if command is "s:" or "S:" then uninstall<br></code><code>&nbsp;&nbsp;&nbsp; {<br></code><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; output.capacity = 15i64;<br></code><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; output.len = 0i64;<br></code><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; output.buf[0] = 0;<br></code><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; F7_StrAssign(&amp;output, "exit requested...", 0x11ui64);</code></p>
<p>This command also uninstalls a service named "IntelDevMonServer" and deletes a file from the path %APPDATA%\Intel\devmonsrv.exe. This file however was not &nbsp;installed by the malware. It is &nbsp;probably a code remnant from the original malware source, &nbsp;left by the malware author.</p>
<h3 style="font-weight: bold;">&nbsp;</h3>
<h3 style="font-weight: bold;">Command 2</h3>
<p>crash&nbsp;- simulate crash</p>
<p>As the name implies, it simulates its own process to crash.</p>
<p>&nbsp;&nbsp;<code>&nbsp;&nbsp;&nbsp; pcrash = aCrash[crashLen++];<br></code><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; if ( pcrash != pCommand[crashLen - 1] )<br></code><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; break;<br></code><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; if ( crashLen == 6 )<br></code><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; {<br></code><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; logString.capacity = 15i64;<br></code><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; logString.len = 0i64;<br></code><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; logString.buf[0] = 0;<br></code><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; F7_StrAssign(&amp;logString, "simulating crash", 0x10ui64);<br></code><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; EnterCriticalSection(&amp;CriticalSection);</code></p>
<p>&nbsp;</p>
<h3 style="font-weight: bold;">Final remarks</h3>
<p>An analogy is that Pillowmint is like a monkey thief. It just grabs all the money from the victim and places the stolen goods to one side, to be later collected by the criminal. Pillowmint doesn’t exfiltrate its stolen credit card data log files. This makes sense because usually attackers already have complete control of the compromised system and the data is exfiltrated by other malicious applications installed by the attacker. However, the downside is that this leaves a footprint that an investigator can later easily discover.</p>
<p>Although the use of a shim database for persistence in a compromised system is not new, it is rare to find malware that uses this method. FIN7 actors have used <a href="https://www.trustwave.com/en-us/resources/blogs/spiderlabs-blog/would-you-exchange-your-security-for-a-gift-card/">creative techniques</a> and <a href="https://www.trustwave.com/en-us/resources/blogs/spiderlabs-blog/carbanak-continues-to-evolve-quietly-creeping-into-remote-hosts/">sophisticated attacks</a> in the past, and this demonstrates it again.</p>
