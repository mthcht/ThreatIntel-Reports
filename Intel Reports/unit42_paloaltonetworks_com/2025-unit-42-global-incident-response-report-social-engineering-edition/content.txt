Enterprises\u2019 size and complexity make it easier for malicious activity to blend in with routine requests, increasing the time to detection.\r\n\r\nHigh-touch operations are often financially motivated, driven by threat actors who invest time and research to breach identity defenses without triggering alerts. They impersonate employees, exploit trust and escalate quickly from user-level access to privileged control. We have investigated multiple high-impact cases where attackers bypassed MFA and convinced help desk staff to reset credentials. In one recent case, the attacker progressed from gaining initial access to obtaining domain administrator rights in minutes, without deploying malware at all.\r\n\r\nGroups such as Muddled Libra, a global, financially motivated cybercrime operation, exemplify this model. Instead of phishing broadly, these attackers identify key personnel, build a profile using public data and impersonate them convincingly. As a result, these groups gain deep access, broad system control and the ability to monetize attacks quickly .\r\n\r\nNot all high-touch operations are profit-driven. We have also tracked state-aligned actors using similar tactics for espionage and strategic infiltration. Campaigns attributed to state-aligned threat actors such as Iran-affiliated Agent Serpens and threat groups from North Korea have relied on spoofed institutional identities, custom-crafted lures and counterfeit documentation to compromise diplomatic and public sector targets.\r\nProfile #1: Muddled Libra\r\nAmong financially motivated actors, few have demonstrated the persistence, adaptability and technical fluency of Muddled Libra. Tracked by Unit 42 across multiple IR cases and threat research streams, this group is known for its ability to move from initial access to domain-level control in under 40 minutes, without relying on malware or exploits.\r\n\r\nMuddled Libra operates within a broader cluster of actors. These threat actors are financially motivated, bypassing technical controls by exploiting identity systems directly. Muddled Libra stands out, however, due to its persistence, speed and human-led tradecraft.\r\n\r\nThe group doesn\u2019t just phish for credentials. It impersonates employees in real time, often targeting help desk staff to reset MFA, take over identities and gain access to internal systems. Once inside, Muddled Libra leverages remote monitoring and management (RMM) tools to maintain persistent access and establish control.\r\n\r\nTheir infrastructure is lightweight and evasive. For command and control (C2), we have observed repeated misuse of tunneling services, allowing the group to operate from outside the network perimeter without raising immediate red flags.\r\n\r\nMFA resets, subscriber identity module (SIM) swapping and the misuse of public-facing trust signals are recurring patterns. In several cases, attackers gathered detailed personal data from sources like LinkedIn to build convincing personas, increasing their chances of bypassing identity verification steps.\r\n\r\nProfile #2: Nation-State Actors\r\nHigh-touch social engineering is not exclusive to cybercriminal groups. Nation-state actors have long relied on human-first tradecraft to achieve strategic access, whether for espionage, surveillance or geopolitical disruption. Recent campaigns show that state-aligned intrusions increasingly mirror the same identity-centric methods seen in financially motivated operations.\r\n\r\nIran-affiliated actors are one such group. We tracked Iran-affiliated Agent Serpens campaigns in which the Iran-affiliated group impersonated trusted institutions, often delivering malware via spoofed emails that mimicked legitimate document-sharing workflows.\r\n\r\nNorth Korean actors have developed a distinct variation of high-touch compromise known as synthetic insiders. These campaigns involve attackers posing as job applicants, using fabricated curriculum vitae (CVs) and professional personas to secure remote employment in target organizations.\r\n\r\nFrom Muddled Libra\u2019s financially driven intrusions to the strategic targeting seen in Agent Serpens and North Korean campaigns, these attackers\u2019 strategies remain consistent. Manipulate people, mimic trust and escalate access without triggering detection.\r\nThreat Actors and Their Tactics, Techniques and Procedures (TTPs) Tradecraft\r\n\r\nHigh-Touch Intrusions in the Field\r\nThe following case studies drawn from our IR caseload illustrate how attackers can bypass controls not through exploits, but by navigating human systems with precision and intent.\r\nHelp Desk Deception Leads Unlocks 350 GB Breach\r\nTarget: Customer records, proprietary documentation and internal files were hosted in cloud storage. An attacker staged and exfiltrated over 350 GB of data without triggering endpoint or endpoint detection and response (EDR) alerts. Attackers used no malware \u2014 only legitimate credentials and living-off-the-land binaries.\r\n\r\nTechnique: The attacker contacted the organization's help desk, impersonating a locked-out employee. Using a mix of leaked and publicly available details, the adversary passed identity checks and gained the agent\u2019s trust, prompting a reset of MFA credentials. With access secured, the attacker logged in and moved laterally using legitimate administrative tools. Every action mimicked legitimate user behavior, deliberately avoiding triggers that might raise alerts.\r\nRapport-Building To Bypass the Help Desk\r\nTarget: Internal corporate systems. The attacker sought employee-level access to escalate privileges and stage a broader compromise. The intrusion was contained shortly after login.\r\n\r\nTechnique: Over several days, the attacker made repeated low-pressure calls to the help desk, each time impersonating a locked-out employee. They gathered process details and refined their pretext with each attempt. Once the story aligned with internal escalation protocols, they issued a time-sensitive request that prompted an MFA reset to grant access. The attacker logged in successfully but was flagged minutes later due to geographic anomalies.\r\nExecutive MFA Reset Blocked by Conditional Access\r\nTarget: Mid-level executive credentials with broad system permissions. The attacker aimed to use these credentials to access sensitive business data and perform reconnaissance via cloud APIs. The attempt was contained before data exfiltration, thanks to conditional access controls.\r\n\r\nTechnique: After several failed phishing attempts, the attacker called IT support, impersonating the executive and citing travel-related access issues. The pretext was convincing enough to prompt an MFA reset. With fresh credentials, the attacker initiated Graph API queries to enumerate permissions, group memberships and file paths. However, the conditional access policy flagged the session due to an unusual login from an unrecognized device and location, blocking further escalation.\r\n\r\nThese case studies show that high-touch attackers succeed not by breaking systems, but by understanding them. They manipulate processes, personnel and platforms across industries in ways that appear routine until it is too late.\r\nHow AI Is Shaping the Social Engineering Threat Landscape\r\nRecent months have seen a shift in how AI and automation are being used in social engineering campaigns. While conventional techniques remain dominant, some attackers are now experimenting with tools that offer greater speed, realism and scalability.\r\n\r\nUnit 42\u2019s incident response cases point to three distinct layers of AI-enabled tooling:\r\n\r\n \tAutomation: Tools in this category follow predefined rules to accelerate common intrusion steps, such as phishing distribution, spoofed SMS delivery and basic credential testing. These capabilities are not new, but they are increasingly configured to mimic enterprise workflows and bypass common detection measures.\r\n \tGenerative AI (GenAI): Used to produce credible, human-like content across channels including email, voice and live chat. In multiple investigations, threat actors employed GenAI to craft highly personalized lures using public information. Some campaigns went further, using cloned executive voices in callback scams to increase the plausibility of urgent phone requests. In more sustained intrusions AI was used to refine attacker personas, generate tailored phishing follow-ups and draft real-time responses. These adaptive techniques allowed threat actors to maintain engagement across multiple stages of the intrusion, with a level of tone and timing that previously required a live operator.\r\n \tAgentic AI: Refers to role-based, context-aware systems capable of autonomously executing multi-step tasks with minimal human input and learning from feedback. While adoption remains limited to date, we observed Agentic AI\u2019s use in chaining activities such as cross-platform reconnaissance and message distribution. In one case, attackers built multi-layered synthetic identities (including fake CVs and social media profiles) to support fraudulent job applications in a targeted insider campaign.\r\n\r\nThis spectrum of use reflects a hybridization of tactics, where conventional social engineering methods are increasingly supported by AI-enabled components. Adversary adoption of AI remains uneven, but the underlying shift is clear: automation and AI are beginning to reshape the scale, pacing and adaptability of social engineering attacks.\r\nAt-Scale Attacks\r\nAt-Scale Deception: The Rise of ClickFix Campaigns\r\nSocial engineering has evolved into a scalable, automated ecosystem that mimics trusted signals and exploits familiar workflows. One example is ClickFix, a technique using fake browser alerts, fraudulent update prompts and drive-by downloads to initiate compromise. Between May 2024 and May 2025, ClickFix was the initial access vector in at least eight confirmed IR cases.\r\nDelivery Mechanisms: SEO Poisoning, Malvertising and Fraudulent Prompts\r\nClickFix campaigns don\u2019t rely on a single delivery method. Instead, they exploit multiple entry points. We have observed these campaigns using SEO poisoning, malvertising and fraudulent browser alerts to lure users into initiating the attack chain themselves.\r\n\r\nIn one confirmed IR case, the threat actor leveraged SEO poisoning to plant a malicious link high in search engine results. When an employee searched for a software installer, they were redirected to a spoofed landing page that triggered a payload download. Malvertising plays a similar role, delivering fake \u201cclick to fix\u201d banners via ad networks or pop-ups mimicking trusted software brands. Another growing vector is fraudulent system alerts, crafted to mimic legitimate browser or operating system warnings. In one healthcare example, an employee encountered what appeared to be an authentic Microsoft update notification while accessing an internal system from home. The link led to the download of a loader, which executed an infostealer and enabled credential harvesting. These delivery mechanisms share three core attributes:\r\n\r\n \tTrust mimicry\r\n \tUser initiation\r\n \tPlatform agnosticism\r\n\r\nBecause the user initiates the action (by clicking a link, downloading a file or responding to a prompt) the attack often bypasses traditional perimeter defenses and evades early detection by endpoint tools.\r\n\r\n\r\nCommon ClickFix Payloads and Behavioral Patterns\r\nClickFix campaigns follow a consistent behavioral pattern that prioritizes speed, credential theft and staging for monetization. The payloads vary, but their objectives are tightly aligned. Establish a foothold, extract value and avoid detection.\r\n\r\nCredential-harvesting malware is the most common first-stage payload. In many cases, we identified the use of off-the-shelf stealers such as RedLine or Lumma, deployed immediately after a user downloaded a malicious installer or responded to a spoofed update prompt. These tools are configured to extract browser-stored credentials, saved tokens and session cookies.\r\n\r\nLampion, a banking Trojan analyzed by Unit 42 researchers, exemplifies the evolution of these payloads. Initially delivered via email lures, newer Lampion variants use SEO poisoning and compromised websites for distribution, blurring the lines between phishing and ClickFix-style delivery. The retail sector is a common target for these kinds of attacks. An example includes a fake update prompt that led a retail employee to unknowingly install a remote administration Trojan that allowed the attacker to observe and eventually control the victim\u2019s device. Within 90 minutes, the attacker had captured credentials for the organization\u2019s order management system and staged outbound data transfers.\r\nModular, Escalating Toolchains\r\nClickFix attacks often use payloads that escalate in stages, depending on the value of the environment:\r\n\r\n \tBegin with lightweight credential harvesters to gather intelligence.\r\n \tDeploy silent loaders to deliver additional tools only if access to sensitive systems is confirmed.\r\n \tFollow-on payloads can include:\r\n\r\n \tRemote access Trojans (RATs)\r\n \tInfostealers\r\n \tEncryption modules or wipers.\r\n\r\n\r\n\r\nBehavioral consistency is what makes these campaigns effective and repeatable. The payloads are rarely novel, but their delivery and timing are calibrated to minimize risk and maximize utility. In many cases, the attackers relied on well-worn techniques (not zero-days or custom exploits) because the system\u2019s users, not its software, were the point of entry.\r\n\r\nClickFix Isn\u2019t Sophisticated: It's Systematic\r\n\r\nMost ClickFix campaigns used publicly available tools repurposed for credential theft or remote access. What set them apart was not malware sophistication, but precision delivery, legitimate-looking prompts, high-trust delivery paths and low-friction execution.\r\n\r\n\r\nStrategic Takeaways\r\nClickFix is a blueprint for scalable deception. To defend against it, we advise security leaders to:\r\n\r\n \tMonitor for early-stage access patterns, especially credential harvesters and fileless loaders.\r\n \tHarden endpoints against commonly misused malware families, even those considered commodity-grade.\r\n \tTrack and control software installation privileges. Many incidents began with user-approved downloads.\r\n \tExpand detection beyond email to include web-based delivery, SEO manipulation and spoofed system prompts.\r\n \tEstablish credential hygiene practices and limit reuse across systems to prevent chained exposure.\r\n\r\nSocial Engineering by the Numbers\r\nIn this section, we turn from tactics to telemetry to examine the technical and organizational patterns behind that success.\r\n\r\nAs reported in January, phishing accounted for 23% of all intrusions and that number remains constant for the data set used in this report. When isolating only social engineering-driven intrusions, phishing rises to 65% of those cases (see Figure 1).\r\n\r\nOf those phishing-driven social engineering cases:\r\n\r\n \t66% of social engineering attacks targeted privileged accounts.\r\n \t23% involved callback or voice-based techniques.\r\n \t45% used impersonation of internal personnel to build trust.\r\n\r\nOur data reveals six key patterns behind the continued success of social engineering attacks \u2014 and how they\u2019re bypassing defenses.\r\nInitial Access: Social Engineering Remains the Top Tactic\r\nSocial engineering dominated initial access methods in our IR caseload between May 2024 and May 2025 (see Figure 4).\r\nKey Insight\r\nUnlike technical exploits that rely on unpatched systems or zero-days, social engineering succeeds due to weak access controls, a sense of urgency, over-permissioned accounts and misplaced trust. These conditions persist in enterprise environments, even where modern detection tools are deployed.\r\nStrategic Takeaways\r\nSocial engineering remains the most effective attack vector because it exploits human behavior, not technical flaws. It consistently bypasses controls, regardless of an organization\u2019s maturity.\r\n\r\nIt\u2019s time to move beyond user education as the primary defense. Treat social engineering as a systemic vulnerability that demands layered technical controls and strict zero-trust verification.\r\nNovel Vectors Are Gaining Ground\r\nA combined 35% of social engineering cases involved less conventional methods, including SEO poisoning and malvertising, smishing and MFA bombing, and a growing set of other techniques (see Figure 1).\r\n\r\n\r\n\r\nWhile phishing remains the primary delivery mechanism, these emerging methods show that attackers are adapting social engineering to reach users across platforms, devices and workflows where traditional email security is no longer a barrier.\r\nKey Insight\r\nThe dominance of phishing masks a deeper shift; threat actors are broadening their playbook.\r\nStrategic Takeaways\r\nSocial engineering defense must cover more than the inbox. Security leaders should ensure that detection extends to mobile messaging, collaboration tools, browser-based vectors and quick response (QR) interfaces.\r\nMissed Alerts, Weak Controls and Permission Overreach Fuel Social Engineering Intrusions\r\nThese weaknesses span industries, from manufacturing and healthcare to finance, professional services, retail and state and federal government. They also affect organizations of every size, from small businesses to large enterprises, underscoring just how widespread and systemic these issues are.\r\n\r\n \tIn many cases, security alerts went unnoticed because overwhelmed security operations teams were unaware of them. Overburdened teams missed, deprioritized or dismissed malicious logins, privilege escalations or alerts triggered by unusual device access until after compromise was confirmed.\r\n \tExcessive permissions increased the area of impact. In many cases, compromised accounts had access well beyond their operational role.\r\n \tLack of or insufficiently deployed MFA featured in a large share of credential-based intrusions. Attackers were often able to authenticate successfully using harvested credentials without encountering any secondary verification.\r\n\r\n\r\nKey Insight\r\nThreat actors exploited control gaps that could have been closed.\r\nStrategic Takeaways\r\nSocial engineering defense depends on detecting early-stage indicators and limiting access after compromise. Many early indicators are missed, not because they\u2019re ignored, but because they\u2019re misclassified.\r\n\r\n \tPrioritize alert logic to flag abnormal login patterns, MFA abuse and unusual SaaS access as high priority.\r\n \tSecurity teams must ensure that critical alerts are reviewed and escalated.\r\n \tEnforced MFA should be applied to all privileged and external-facing accounts.\r\n \tAccess entitlements must be tightly scoped and regularly reviewed.\r\n\r\nWeak Detection Capability Technology and Under-Resourced Teams Amplify Social Engineering Risks\r\nSocial engineering intrusions rarely hinge on attacker sophistication alone. More often, they succeed when limited detection capabilities intersect with over-burdened or under-trained teams. Our IR data highlights how technical weak points and staffing constraints create fertile ground for compromise.\r\n\r\nThe six most cited contributing factors reflect more than tooling gaps, they reveal systemic strain. Crucially, few organizations had implemented ITDR or user and entity behavior analytics (UEBA), two capabilities that are increasingly vital for detecting social engineering attacks and preventing account takeover. Enterprises relying solely on conventional logging and endpoint telemetry are likely to miss early indicators.\r\n\r\nThese weaknesses surfaced most often in four key areas:\r\n\r\n \tFailure to escalate alerts: In several cases, early warning signs were logged but not acted on, allowing attackers to progress unchecked.\r\n \tFailure to action alerts: In many cases, anomalous behavior was flagged but not escalated due to alert fatigue, unclear ownership, or skill gaps within security teams.\r\n \tInsufficient EDR and endpoint logging: Without clear indicators or behavioral baselines, analysts struggled to distinguish between routine activity and signs of compromise.\r\n \tLack of centralized anti-virus \/EDR and tamper protection exposed unmanaged assets, which threat actors exploited to move laterally or disable controls undetected.\r\n\r\n\r\nKey Insight\r\n\r\nSecurity breakdowns stemmed from weak telemetry, unclear alert ownership, and strained frontline capacity, not attacker sophistication.\r\nStrategic Takeaways \r\n\r\n \tStrengthening the human layer requires strengthening the systems that support it.\r\n \tEnsure alerts are triaged with clear ownership and escalation paths.\r\n \tPrioritize expanding EDR to extended detection and response (XDR), for centralized detection with rich context.\r\n \tAddress tooling gaps that leave unmanaged assets or disablement vectors exposed.\r\n\r\nInitial access is only the beginning. In many cases, what starts as a single deceptive interaction results in wide-ranging data exposure and attackers are optimizing for that exact outcome.\r\nSocial Engineering Leads to Higher Rates of Data Exposure\r\nSocial engineering attacks led to data exposure in 60% of cases, according to our IR data. That is 16 percentage points higher than attacks involving other initial access vectors. This includes direct exfiltration as well as indirect exposure through credential theft, unauthorized access to internal systems or deployment of infostealers and remote access Trojans.\r\n\r\n\r\nKey Insight\r\nRoughly half of social engineering cases were business email compromises (BEC), and almost 60% of all BEC cases saw data exposure, showing that threat actors moved quickly from gaining access to exfiltrating data or harvesting credentials during these kinds of incidents. Additionally, general network intrusions and ransomware were the two other top incident types where data was exposed. Of those incident types, social engineering was in the top two initial access vectors, showing the popularity of this technique across different types of intrusions and actors.\r\n\r\nCredential exposure was also a common precursor to broader data loss. In several cases, attackers reused compromised credentials to access file shares, customer systems or cloud environments. This chained exposure effect amplifies the impact of a single successful lure, turning one compromised identity into broader organizational risk.\r\nStrategic Takeaways\r\nSocial engineering isn\u2019t just an access problem, it\u2019s a significant data-loss risk. To reduce exposure, security teams must improve visibility into user behavior after login, not just focus on preventing initial compromise. Key measures include:\r\n\r\n \tRestricting access to sensitive assets by default.\r\n \tMonitoring lateral movement and file access.\r\n \tImplementing just-in-time provisioning for privileged operations.\r\n \tData loss prevention (DLP) policies and data tagging should also account for the likelihood of social engineering-driven access, especially via compromised accounts.\r\n\r\nProfit Remains the Primary Driver\r\nFinancial gain is the dominant motive across Unit 42 incident response cases, and social engineering attacks are no exception. Nearly all social engineering intrusions between May 2024 and May 2025 were financially motivated (see Figure 5), with threat actors seeking to monetize access through data theft, extortion, ransomware or resale of credentials.\r\n\r\n\r\nKey Insight\r\nSocial engineering remains a preferred access method because it's easy to execute, infrastructure-light and often avoids detection\u2014with attackers reusing proven tools like phishing kits and impersonation scripts to achieve compromise-to-monetization in under 48 hours, delivering superior cost-benefit over technical exploits by leveraging trust-based access to bypass hardened controls.\r\nStrategic Takeaways \r\nAttackers continue to choose social engineering not for its sophistication, but for its ease, speed and reliability. Detection and response must be tuned to spot monetization signals (including data staging, file movement and abnormal system access) even when the intrusion begins with a single deceptive message.\r\n\r\n\r\nRecommendations for Defenders\r\nSocial engineering continues to outperform other access vectors, not through technical sophistication, but by exploiting gaps between people, process and platform. To counter this, defenders must focus on identity resilience, visibility across workflows, and intelligence-led operations. Capabilities such as UEBA and ITDR play an increasingly critical role in identifying abnormal activity and blocking account takeover attempts.\r\n\r\nThe following eight recommendations draw directly from Unit 42\u2019s incident response caseload and threat research:\r\nCorrelate Identity Signals To Detect Abuse Earlier\r\nAttackers often appear legitimate, until anomalous behaviors emerge. Correlating signals across identity, device and session behavior helps security operations teams detect social engineering attacks before they escalate. Platforms such as Cortex XSIAM accelerate this process, enabling faster threat detection and containment without relying on traditional indicators of compromise (IOCs).\r\nEnforce Zero Trust Across Access Pathways\r\nA strong Zero Trust posture limits attacker movement after initial access. Apply conditional access policies that assess device trust, location and login behavior before granting access. Combine least-privilege principles, just-in-time access and network segmentation to restrict lateral movement and reduce blast radius.\r\nStrengthen the Human Layer With Detection and Training\r\nEmployees are part of the detection surface. Harden common abuse points (email, browsers, messaging apps, domain name system (DNS)) with intelligence-led controls. Train frontline teams such as HR and IT support to recognize and report impersonation, voice lures and pretexting. Simulate current attack techniques, including help desk spoofing and MFA manipulation.\r\nStrengthen Detection With Identity and Behavioral Analytics\r\nDetecting social engineering requires visibility beyond traditional indicators. Correlate signals across identity, endpoint, network and SaaS activity to expose escalation attempts early. Capabilities like UEBA and ITDR help surface anomalies such as impersonation, session abuse and credential misuse.\r\nControl and Monitor Illegitimate Use of Native Tools and Business Process Workflows\r\nAttackers can use built-in utilities such as PowerShell or WMI to move undetected. Establish behavioral baselines and alert on anomalies. Map business workflows to uncover where process trust can be exploited, particularly escalation points that rely on fast-track approvals or assumed identity, such as help desk credential resets, finance system overrides or privileged access granted through informal Slack or Teams messages.\r\nBuild Resilience Through Simulation and Playbook Readiness\r\nPreparation narrows response gaps. Run live drills based on current social engineering tactics, such as impersonation or chained credential use. Validate playbooks, involve cross-functional teams and integrate Unit 42 threat intelligence into both blue team exercises and user awareness programs.\r\nEnforce Network-Layer Controls\r\nDeploy Advanced DNS Security and Advanced URL filtering to block access to malicious infrastructure. These controls help detect and prevent social engineering attacks that rely on spoofed domains, typo-squatting, SEO poisoning and link-based credential theft. Visibility at the network layer adds a critical line of defence when endpoint or identity-based detection fails.\r\n\r\nLock Down Identity Recovery Paths\r\n\r\nThreat actors increasingly target IT help desks to reset credentials and bypass MFA. Strengthen controls around account recovery by enforcing strict identity verification protocols, limiting who can initiate resets, and monitor for unusual request patterns. Help desk staff should receive regular training grounded in real-world threat activity.\r\nFinal Thoughts\r\nSocial engineering continues to evolve, but its success still depends on trust. Defenders must think beyond malware and infrastructure controls. The new perimeter is shaped by people, processes and the decisions they make in real time.                 <h2>Executive Summary</h2>
<p>We see social engineering evolving into one of the most reliable, scalable and impactful intrusion methods in 2025 for five key reasons:</p>
<p><strong>First</strong>, social engineering remained the top initial access vector in Unit 42 incident response cases between May 2024 and May 2025. These attacks consistently bypassed technical controls by targeting human workflows, exploiting trust and manipulating identity systems. More than one-third of social engineering incidents involved non-phishing techniques, including search engine optimization (SEO) poisoning, fake system prompts and help desk manipulation.</p>
<p><strong>Second</strong>, high-touch attacks are on the rise. Threat actors such as Muddled Libra bypass multi-factor authentication (MFA) and exploit IT support processes to escalate privileges in minutes, often without malware. In one case, a threat actor moved from access to domain administrator in under 40 minutes using only built-in tools and social pretexts.</p>
<p><strong>Third</strong>, low-detection coverage and alert fatigue remain key enablers. In many cases, social engineering attacks succeeded not through advanced tradecraft, but because organizations missed or misclassified critical signals. This was a particular issue for identity recovery workflows and lateral movement paths.</p>
<p><strong>Fourth,</strong> business disruption resulting from these attacks continues to grow. Over half of social engineering incidents led to sensitive data exposure, while other incidents interrupted critical services or affected overall organizational performance. These high-speed attacks are designed to deliver high financial returns while requiring minimal infrastructure or risk.</p>
<p><strong>Fifth</strong>, artificial intelligence (AI) is accelerating both the scale and realism of social engineering campaigns. Threat actors are now using generative AI to craft personalized lures, clone executive voices in callback scams and maintain live engagement during impersonation campaigns.</p>
<p>Beneath these trends lie three systemic enablers: <strong>over-permissioned access</strong>, <strong>gaps in behavioral visibility</strong> and <strong>unverified user trust in human processes</strong>. Identity systems, help desk protocols and fast-track approvals are routinely exploited by threat actors mimicking routine activity.</p>
<p>To counter this, security leaders must drive a shift beyond traditional user awareness to recognizing social engineering as a systemic, identity-centric threat. This transition requires:</p>
<ul>
<li>Implementing behavioral analytics and identity threat detection and response (ITDR) to proactively detect credentials misuse.</li>
<li>Securing identity recovery processes and enforcing conditional access.</li>
<li>Expanding Zero Trust principles to encompass users, not just network perimeters.</li>
</ul>
<p>Social engineering works not because attackers are sophisticated, but because people still trust too easily, compromising organizational security.</p>
<h2><a id="post-147913-_8ki2xigzp2o0"></a>Introduction</h2>
<p>Social engineering continues to dominate the threat landscape<s>.</s> Over the past year, more than a third of the incident response cases my team handled began with a social engineering tactic. These intrusions didn’t rely on zero-days or sophisticated malware. They exploited trust. Attackers bypassed controls by impersonating employees, manipulating workflows and taking advantage of gaps in how organizations manage identity and human interaction.</p>
<p>What stands out this year is how sharply these attacks are evolving. Unit 42 is tracking two distinct models, both designed to bypass controls by mimicking trusted activity:</p>
<ul>
<li><strong>High-touch compromise</strong> that targets specific individuals in real time. Threat actors impersonate staff, exploit help desks and escalate access without deploying malware, often using voice lures, live pretexts, and stolen identity data.</li>
<li><strong>At-scale deception</strong>, including ClickFix-style campaigns, SEO poisoning, fake browser prompts, and blended lures to trigger user-initiated compromise across multiple devices and platforms.</li>
<li>Our team has seen attackers move from gaining initial access to becoming a domain administrator in under an hour, using only built-in tools and a convincing story.</li>
</ul>
<p>These aren’t edge cases. They’re repeatable, reliable techniques that adversaries are refining week after week. One of the most revealing trends we’ve observed is the rise of non-phishing vectors. This includes voice-based lures, spoofed browser alerts and direct manipulation of support teams. In many environments, these tactics enable attackers to slip through undetected, exfiltrate data and cause significant operational harm.</p>
<p>This report brings together intelligence from Unit 42 threat researchers, telemetry from real-world intrusions, and insight from the front lines of our global incident response (IR) practice. The goal of this report is to help you understand how social engineering actually works in 2025 and what it takes to stop it.</p>
<p>One thing is clear: adversaries aren’t just hacking systems: They’re hacking people.</p>
<h2><a id="post-147913-_67gk1lg6k6fe"></a>High-Touch, High-Impact Compromise</h2>
<h3><a id="post-147913-_92mixvxa75qw"></a><strong>How Attackers Breach Trust</strong></h3>
<p>A growing class of attacks targets individuals using tailored, real-time manipulation, often within large enterprises where identity systems and human workflows are more complex. These organizations present a richer set of access points, from federated identity platforms to distributed support operations, which attackers exploit to escalate access discreetly. Enterprises’ size and complexity make it easier for malicious activity to blend in with routine requests, increasing the time to detection.</p>
<p>High-touch operations are often financially motivated, driven by threat actors who invest time and research to breach identity defenses without triggering alerts. They impersonate employees, exploit trust and escalate quickly from user-level access to privileged control. We have investigated multiple high-impact cases where attackers bypassed MFA and convinced help desk staff to reset credentials. In one recent case, the attacker progressed from gaining initial access to obtaining domain administrator rights in minutes, without deploying malware at all.</p>
<p>Groups such as <a href="https://unit42.paloaltonetworks.com/tag/muddled-libra/">Muddled Libra</a>, a global, financially motivated cybercrime operation, exemplify this model. Instead of phishing broadly, these attackers identify key personnel, build a profile using public data and impersonate them convincingly. As a result, these groups gain deep access, broad system control and the ability to monetize attacks quickly .</p>
<p>Not all high-touch operations are profit-driven. We have also tracked state-aligned actors using similar tactics for espionage and strategic infiltration. Campaigns attributed to state-aligned threat actors such as Iran-affiliated <a href="https://unit42.paloaltonetworks.com/tag/agent-serpens/">Agent Serpens</a> and <a href="https://unit42.paloaltonetworks.com/threat-assessment-north-korean-threat-groups-2024/">threat groups from North Korea</a> have relied on spoofed institutional identities, custom-crafted lures and counterfeit documentation to compromise diplomatic and public sector targets.</p>
<h4><a id="post-147913-_gk4xc7eo5z9"></a>Profile #1: Muddled Libra</h4>
<p>Among financially motivated actors, few have demonstrated the persistence, adaptability and technical fluency of Muddled Libra. Tracked by Unit 42 across multiple IR cases and threat research streams, this group is known for its ability to move from initial access to domain-level control in under 40 minutes, without relying on malware or exploits.</p>
<p>Muddled Libra operates within a broader cluster of actors. These threat actors are financially motivated, bypassing technical controls by exploiting identity systems directly. Muddled Libra stands out, however, due to its persistence, speed and human-led tradecraft.</p>
<p>The group doesn’t just phish for credentials. It impersonates employees in real time, often targeting help desk staff to reset MFA, take over identities and gain access to internal systems. Once inside, Muddled Libra leverages remote monitoring and management (RMM) tools to maintain persistent access and establish control.</p>
<p>Their infrastructure is lightweight and evasive. For command and control (C2), we have observed repeated misuse of tunneling services, allowing the group to operate from outside the network perimeter without raising immediate red flags.</p>
<p>MFA resets, subscriber identity module (SIM) swapping and the misuse of public-facing trust signals are recurring patterns. In several cases, attackers gathered detailed personal data from sources like LinkedIn to build convincing personas, increasing their chances of bypassing identity verification steps.</p>
<h4><img  class="alignnone wp-image-147925 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-388599-147913-1.jpeg" alt="A list titled &quot;Six Key Tools and Techniques Employed by Muddled Libra&quot;, detailing methods such as SIM swapping, remote access tools, credential harvesting, social reconnaissance, and live interaction to intercept messages and control pathways." width="2048" height="569" data-srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-388599-147913-1.jpeg 2048w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-388599-147913-1-786x218.jpeg 786w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-388599-147913-1-1920x533.jpeg 1920w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-388599-147913-1-768x213.jpeg 768w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-388599-147913-1-1536x427.jpeg 1536w" sizes="(max-width: 2048px) 100vw, 2048px" /><br />
<strong>Profile #2: Nation-State Actors</strong></h4>
<p>High-touch social engineering is not exclusive to cybercriminal groups. Nation-state actors have long relied on human-first tradecraft to achieve strategic access, whether for espionage, surveillance or geopolitical disruption. Recent campaigns show that state-aligned intrusions increasingly mirror the same identity-centric methods seen in financially motivated operations.</p>
<p><a href="https://unit42.paloaltonetworks.com/iranian-cyberattacks-2025/"><strong>Iran-affiliated actors</strong></a> are one such group. We tracked Iran-affiliated Agent Serpens campaigns in which the Iran-affiliated group impersonated trusted institutions, often delivering malware via spoofed emails that mimicked legitimate document-sharing workflows.</p>
<p><a href="https://unit42.paloaltonetworks.com/threat-assessment-north-korean-threat-groups-2024/"><strong>North Korean actors</strong></a> have developed a distinct variation of high-touch compromise known as synthetic insiders. These campaigns involve attackers posing as job applicants, using fabricated curriculum vitae (CVs) and professional personas to secure remote employment in target organizations.</p>
<p>From Muddled Libra’s financially driven intrusions to the strategic targeting seen in Agent Serpens and North Korean campaigns, these attackers’ strategies remain consistent. Manipulate people, mimic trust and escalate access without triggering detection.</p>
<h3>Threat Actors and Their Tactics, Techniques and Procedures (TTPs) Tradecraft</h3>
<p><strong><img  class="alignnone wp-image-147936 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-392313-147913-2.jpeg" alt="Table listing three threat actors: Muddled Libra, Agent Serpens, and a North Korean Actor, with their motivations and tactics. <h3>High-Touch Intrusions in the Field</h3>
<p>The following case studies drawn from our IR caseload illustrate how attackers can bypass controls not through exploits, but by navigating human systems with precision and intent.</p>
<h4>Help Desk Deception Leads Unlocks 350 GB Breach</h4>
<p><strong>Target:</strong> Customer records, proprietary documentation and internal files were hosted in cloud storage. An attacker staged and exfiltrated over 350 GB of data without triggering endpoint or endpoint detection and response (EDR) alerts. Attackers used no malware — only legitimate credentials and living-off-the-land binaries.</p>
<p><strong>Technique:</strong> The attacker contacted the organization's help desk, impersonating a locked-out employee. Using a mix of leaked and publicly available details, the adversary passed identity checks and gained the agent’s trust, prompting a reset of MFA credentials. With access secured, the attacker logged in and moved laterally using legitimate administrative tools. Every action mimicked legitimate user behavior, deliberately avoiding triggers that might raise alerts.</p>
<h4>Rapport-Building To Bypass the Help Desk</h4>
<p><strong>Target:</strong> Internal corporate systems. The attacker sought employee-level access to escalate privileges and stage a broader compromise. The intrusion was contained shortly after login.</p>
<p><strong>Technique:</strong> Over several days, the attacker made repeated low-pressure calls to the help desk, each time impersonating a locked-out employee. They gathered process details and refined their pretext with each attempt. Once the story aligned with internal escalation protocols, they issued a time-sensitive request that prompted an MFA reset to grant access. The attacker logged in successfully but was flagged minutes later due to geographic anomalies.</p>
<h4>Executive MFA Reset Blocked by Conditional Access</h4>
<p><strong>Target:</strong> Mid-level executive credentials with broad system permissions. The attacker aimed to use these credentials to access sensitive business data and perform reconnaissance via cloud APIs. The attempt was contained before data exfiltration, thanks to conditional access controls.</p>
<p><strong>Technique:</strong> After several failed phishing attempts, the attacker called IT support, impersonating the executive and citing travel-related access issues. The pretext was convincing enough to prompt an MFA reset. With fresh credentials, the attacker initiated Graph API queries to enumerate permissions, group memberships and file paths. However, the conditional access policy flagged the session due to an unusual login from an unrecognized device and location, blocking further escalation.</p>
<p>These case studies show that high-touch attackers succeed not by breaking systems, but by understanding them. They manipulate processes, personnel and platforms across industries in ways that appear routine until it is too late.</p>
<h3>How AI Is Shaping the Social Engineering Threat Landscape</h3>
<p>Recent months have seen a shift in how AI and automation are being used in social engineering campaigns. While conventional techniques remain dominant, some attackers are now experimenting with tools that offer greater speed, realism and scalability.</p>
<p>Unit 42’s incident response cases point to three distinct layers of AI-enabled tooling:</p>
<ul>
<li><strong>Automation</strong>: Tools in this category follow predefined rules to accelerate common intrusion steps, such as phishing distribution, spoofed SMS delivery and basic credential testing. These capabilities are not new, but they are increasingly configured to mimic enterprise workflows and bypass common detection measures.</li>
<li><strong>Generative AI (GenAI)</strong>: Used to produce credible, human-like content across channels including email, voice and live chat. In multiple investigations, threat actors employed GenAI to craft highly personalized lures using public information. Some campaigns went further, using cloned executive voices in callback scams to increase the plausibility of urgent phone requests. In more sustained intrusions AI was used to refine attacker personas, generate tailored phishing follow-ups and draft real-time responses. These adaptive techniques allowed threat actors to maintain engagement across multiple stages of the intrusion, with a level of tone and timing that previously required a live operator.</li>
<li><strong>Agentic AI</strong>: Refers to role-based, context-aware systems capable of autonomously executing multi-step tasks with minimal human input and learning from feedback. While adoption remains limited to date, we observed Agentic AI’s use in chaining activities such as cross-platform reconnaissance and message distribution. In one case, attackers built multi-layered synthetic identities (including fake CVs and social media profiles) to support fraudulent job applications in a targeted insider campaign.</li>
</ul>
<p>This spectrum of use reflects a hybridization of tactics, where conventional social engineering methods are increasingly supported by AI-enabled components. Adversary adoption of AI remains uneven, but the underlying shift is clear: automation and AI are beginning to reshape the scale, pacing and adaptability of social engineering attacks.</p>
<h2><a id="post-147913-_kqeswqpvryil"></a>At-Scale Attacks</h2>
<h3><a id="post-147913-_sg6edabbsgdz"></a>At-Scale Deception: The Rise of ClickFix Campaigns</h3>
<p>Social engineering has evolved into a scalable, automated ecosystem that mimics trusted signals and exploits familiar workflows. One example is ClickFix, a technique using fake browser alerts, fraudulent update prompts and drive-by downloads to initiate compromise. Between May 2024 and May 2025, ClickFix was the initial access vector in at least eight confirmed IR cases.</p>
<h4><strong>Delivery Mechanisms: SEO Poisoning, Malvertising and Fraudulent Prompts</strong></h4>
<p>ClickFix campaigns don’t rely on a single delivery method. Instead, they exploit multiple entry points. We have observed these campaigns using SEO poisoning, malvertising and fraudulent browser alerts to lure users into initiating the attack chain themselves.</p>
<p>In one confirmed IR case, the threat actor leveraged SEO poisoning to plant a malicious link high in search engine results. When an employee searched for a software installer, they were redirected to a spoofed landing page that triggered a payload download. Malvertising plays a similar role, delivering fake “click to fix” banners via ad networks or pop-ups mimicking trusted software brands. Another growing vector is fraudulent system alerts, crafted to mimic legitimate browser or operating system warnings. In one healthcare example, an employee encountered what appeared to be an authentic Microsoft update notification while accessing an internal system from home. The link led to the download of a loader, which executed an infostealer and enabled credential harvesting. These delivery mechanisms share three core attributes:</p>
<ul>
<li><strong>Trust mimicry</strong></li>
<li><strong>User initiation</strong></li>
<li><strong>Platform agnosticism</strong></li>
</ul>
<p>Because the user initiates the action (by clicking a link, downloading a file or responding to a prompt) the attack often bypasses traditional perimeter defenses and evades early detection by endpoint tools.</p>
<p><img  class="alignnone wp-image-148059 size-full lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/Callout-Box-2--scaled.jpg" alt="Text in an image reads: Of ClickFix attacks reviewed by Unit 42, more than 60% of initial access was initiated via web interaction rather than email. This marks a distinct shift from traditional phishing and underscores the importance of defending beyond the inbox." width="2560" height="427" data-srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/Callout-Box-2--scaled.jpg 2560w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/Callout-Box-2--786x131.jpg 786w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/Callout-Box-2--1920x320.jpg 1920w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/Callout-Box-2--768x128.jpg 768w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/Callout-Box-2--1536x256.jpg 1536w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/Callout-Box-2--2048x341.jpg 2048w" sizes="(max-width: 2560px) 100vw, 2560px" /></p>
<h4><strong>Common ClickFix Payloads and Behavioral Patterns</strong></h4>
<p>ClickFix campaigns follow a consistent behavioral pattern that prioritizes speed, credential theft and staging for monetization. The payloads vary, but their objectives are tightly aligned. Establish a foothold, extract value and avoid detection.</p>
<p>Credential-harvesting malware is the most common first-stage payload. In many cases, we identified the use of off-the-shelf stealers such as <a href="https://unit42.paloaltonetworks.com/malware-configuration-extraction-techniques-guloader-redline-stealer/"><strong>RedLine</strong></a> or Lumma, deployed immediately after a user downloaded a malicious installer or responded to a spoofed update prompt. These tools are configured to extract browser-stored credentials, saved tokens and session cookies.</p>
<p><a href="https://unit42.paloaltonetworks.com/lampion-malware-clickfix-lures/"><strong>Lampion</strong></a>, a banking Trojan analyzed by Unit 42 researchers, exemplifies the evolution of these payloads. Initially delivered via email lures, newer Lampion variants use SEO poisoning and compromised websites for distribution, blurring the lines between phishing and ClickFix-style delivery. The retail sector is a common target for these kinds of attacks. An example includes a fake update prompt that led a retail employee to unknowingly install a remote administration Trojan that allowed the attacker to observe and eventually control the victim’s device. Within 90 minutes, the attacker had captured credentials for the organization’s order management system and staged outbound data transfers.</p>
<h4><strong>Modular, Escalating Toolchains</strong></h4>
<p>ClickFix attacks often use payloads that escalate in stages, depending on the value of the environment:</p>
<ul>
<li>Begin with lightweight<strong> credential harvesters</strong> to gather intelligence.</li>
<li>Deploy silent loaders to deliver additional tools only if access to sensitive systems is confirmed.</li>
<li>Follow-on payloads can include:
<ul>
<li>Remote access Trojans (RATs)</li>
<li>Infostealers</li>
<li>Encryption modules or wipers.</li>
</ul>
</li>
</ul>
<p>Behavioral consistency is what makes these campaigns effective and repeatable. The payloads are rarely novel, but their delivery and timing are calibrated to minimize risk and maximize utility. In many cases, the attackers relied on well-worn techniques (not zero-days or custom exploits) because the system’s users, not its software, were the point of entry.</p>
<p><strong>ClickFix Isn’t Sophisticated: It's Systematic</strong></p>
<p>Most ClickFix campaigns used publicly available tools repurposed for credential theft or remote access. What set them apart was not malware sophistication, but precision delivery, legitimate-looking prompts, high-trust delivery paths and low-friction execution.</p>
<p><strong><img  class="alignnone wp-image-148081 size-full lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/Callout-Box-2-2-1-scaled.jpg" alt="Text in an image reporting that in 75% of ClickFix-related cases, attackers reused credentials harvested during the initial compromise within 48 hours for direct access to cloud services or offered for sale on illicit marketplaces." width="2560" height="427" data-srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/Callout-Box-2-2-1-scaled.jpg 2560w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/Callout-Box-2-2-1-786x131.jpg 786w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/Callout-Box-2-2-1-1920x320.jpg 1920w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/Callout-Box-2-2-1-768x128.jpg 768w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/Callout-Box-2-2-1-1536x256.jpg 1536w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/Callout-Box-2-2-1-2048x341.jpg 2048w" sizes="(max-width: 2560px) 100vw, 2560px" /></strong></p>
<h4><strong>Strategic Takeaways</strong></h4>
<p>ClickFix is a blueprint for scalable deception. To defend against it, we advise security leaders to:</p>
<ul>
<li>Monitor for early-stage access patterns, especially credential harvesters and fileless loaders.</li>
<li>Harden endpoints against commonly misused malware families, even those considered commodity-grade.</li>
<li>Track and control software installation privileges. Many incidents began with user-approved downloads.</li>
<li>Expand detection beyond email to include web-based delivery, SEO manipulation and spoofed system prompts.</li>
<li>Establish credential hygiene practices and limit reuse across systems to prevent chained exposure.</li>
</ul>
<h2><a id="post-147913-_b3r58xh5on9f"></a>Social Engineering by the Numbers</h2>
<p>In this section, we turn from tactics to telemetry to examine the technical and organizational patterns behind that success.</p>
<p>As reported in <a href="https://www.paloaltonetworks.com/resources/research/unit-42-incident-response-report?utm_source=google-jg-amer-unit42-unrc-unpt&amp;utm_medium=paid_search&amp;utm_campaign=google-unit42-unit42_port-amer-multi-lead_gen-en-non_brand&amp;utm_content=7014u000001ZyZbAAK&amp;utm_term=cyber%20threat%20landscape&amp;cq_plac=&amp;cq_net=g&amp;gad_source=1&amp;gad_campaignid=20369915908&amp;gbraid=0AAAAADHVeKloupLzGlReZDzsOSqGnMQ6J&amp;gclid=CjwKCAjw4K3DBhBqEiwAYtG_9DCbZEEQOQXDf8IClfk55q9kAPrmUKmD_ZvZW7zHPGVLspBVHD0FexoC7hYQAvD_BwE">January</a>, phishing accounted for 23% of all intrusions and that number remains constant for the data set used in this report. When isolating only <strong>social engineering-driven intrusions</strong>, phishing rises to 65% of those cases (see Figure 1).</p>
<p>Of those phishing-driven social engineering cases:</p>
<ul>
<li><strong>66%</strong> of social engineering attacks targeted privileged accounts.</li>
<li><strong>23%</strong> involved callback or voice-based techniques.</li>
<li><strong>45%</strong> used impersonation of internal personnel to build trust.</li>
</ul>
<p>Our data reveals six key patterns behind the continued success of social engineering attacks — and how they’re bypassing defenses.</p>
<h3>Initial Access: Social Engineering Remains the Top Tactic</h3>
<p>Social engineering dominated initial access methods in our IR caseload between May 2024 and May 2025 (see Figure 4).</p>
<h4><strong>Key Insight</strong></h4>
<p>Unlike technical exploits that rely on unpatched systems or zero-days, social engineering succeeds due to weak access controls, a sense of urgency, over-permissioned accounts and misplaced trust. These conditions persist in enterprise environments, even where modern detection tools are deployed.</p>
<h4><strong>Strategic Takeaways</strong></h4>
<p>Social engineering remains the most effective attack vector because it exploits human behavior, not technical flaws. It consistently bypasses controls, regardless of an organization’s maturity.</p>
<p>It’s time to move beyond user education as the primary defense. Treat social engineering as a systemic vulnerability that demands layered technical controls and strict zero-trust verification.</p>
<h3>Novel Vectors Are Gaining Ground</h3>
<p>A combined 35% of social engineering cases involved less conventional methods, including SEO poisoning and malvertising, smishing and MFA bombing, and a growing set of other techniques (see Figure 1).</p>
<p><img  class="alignnone wp-image-147980 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-404451-147913-6.jpeg" alt="Pie chart titled &quot;Breakdown of Social Engineering Attack Types&quot; showing distributions: 65% Phishing, 22% Other, 12% SEO Poisoning/Malvertising, and 1% Smishing/MFA Bombing." width="600" height="736" data-srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-404451-147913-6.jpeg 802w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-404451-147913-6-359x440.jpeg 359w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-404451-147913-6-571x700.jpeg 571w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-404451-147913-6-768x942.jpeg 768w" sizes="(max-width: 600px) 100vw, 600px" /></p>
<p>While phishing remains the primary delivery mechanism, these emerging methods show that attackers are adapting social engineering to reach users across platforms, devices and workflows where traditional email security is no longer a barrier.</p>
<h4><strong>Key Insight</strong></h4>
<p>The dominance of phishing masks a deeper shift; threat actors are broadening their playbook.</p>
<h4><strong>Strategic Takeaways</strong></h4>
<p>Social engineering defense must cover more than the inbox. Security leaders should ensure that detection extends to mobile messaging, collaboration tools, browser-based vectors and quick response (QR) interfaces.</p>
<h3>Missed Alerts, Weak Controls and Permission Overreach Fuel Social Engineering Intrusions</h3>
<p>These weaknesses span industries, from manufacturing and healthcare to finance, professional services, retail and state and federal government. They also affect organizations of every size, from small businesses to large enterprises, underscoring just how widespread and systemic these issues are.</p>
<ul>
<li>In many cases, security alerts went unnoticed because overwhelmed security operations teams were unaware of them. Overburdened teams missed, deprioritized or dismissed malicious logins, privilege escalations or alerts triggered by unusual device access until after compromise was confirmed.</li>
<li>Excessive permissions increased the area of impact. In many cases, compromised accounts had access well beyond their operational role.</li>
<li>Lack of or insufficiently deployed MFA featured in a large share of credential-based intrusions. Attackers were often able to authenticate successfully using harvested credentials without encountering any secondary verification.</li>
</ul>
<p><img  class="alignnone wp-image-147991 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-408245-147913-7.jpeg" alt="Bar chart titled &quot;Figure 2: Identity and Access Issues Contributing to Social Engineering Success.&quot; It shows five categories with corresponding percentages: Ignored Security Alerts at 13%, Lack of MFA at 10%, Excessive Permissions at 10%, Insufficient Endpoint Logging at 8%, Weak/Default Passwords at 7%, and No AV/EDR also at 7%." width="1000" height="426" data-srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-408245-147913-7.jpeg 1254w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-408245-147913-7-786x335.jpeg 786w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-408245-147913-7-768x327.jpeg 768w" sizes="(max-width: 1000px) 100vw, 1000px" /></p>
<h4><strong>Key Insight</strong></h4>
<p>Threat actors exploited control gaps that could have been closed.</p>
<h4><strong>Strategic Takeaways</strong></h4>
<p>Social engineering defense depends on detecting early-stage indicators and limiting access after compromise. Many early indicators are missed, not because they’re ignored, but because they’re misclassified.</p>
<ul>
<li>Prioritize alert logic to flag abnormal login patterns, MFA abuse and unusual SaaS access as high priority.</li>
<li>Security teams must ensure that critical alerts are reviewed and escalated.</li>
<li>Enforced MFA should be applied to all privileged and external-facing accounts.</li>
<li>Access entitlements must be tightly scoped and regularly reviewed.</li>
</ul>
<h3>Weak Detection Capability Technology and Under-Resourced Teams Amplify Social Engineering Risks</h3>
<p>Social engineering intrusions rarely hinge on attacker sophistication alone. More often, they succeed when limited detection capabilities intersect with over-burdened or under-trained teams. Our IR data highlights how technical weak points and staffing constraints create fertile ground for compromise.</p>
<p>The six most cited contributing factors reflect more than tooling gaps, they reveal systemic strain. Crucially, few organizations had implemented ITDR or user and entity behavior analytics (UEBA), two capabilities that are increasingly vital for detecting social engineering attacks and preventing account takeover. Enterprises relying solely on conventional logging and endpoint telemetry are likely to miss early indicators.</p>
<p>These weaknesses surfaced most often in four key areas:</p>
<ul>
<li><strong>Failure to escalate alerts</strong>: In several cases, early warning signs were logged but not acted on, allowing attackers to progress unchecked.</li>
<li><strong>Failure to action alerts: </strong>In many cases, anomalous behavior was flagged but not escalated due to alert fatigue, unclear ownership, or skill gaps within security teams.</li>
<li><strong>Insufficient EDR and endpoint logging: </strong>Without clear indicators or behavioral baselines, analysts struggled to distinguish between routine activity and signs of compromise.</li>
<li><strong>Lack of centralized anti-virus /EDR and tamper protection</strong> exposed unmanaged assets, which threat actors exploited to move laterally or disable controls undetected.</li>
</ul>
<p><img  class="alignnone wp-image-148002 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-411046-147913-8.jpeg" alt="Bar chart titled &quot;Figure 3: Tech and Human Capital Weakness Contributing to Social Engineering Success.&quot; The chart lists types of weaknesses: Failed Action Alert at 13%, Insufficient Endpoint Logging at 8%, Insufficient Network Logging and No Central A/V/EDR both at 7%, and Insufficient Tamper Protection at 4%." width="1256" height="534" data-srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-411046-147913-8.jpeg 1256w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-411046-147913-8-786x334.jpeg 786w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-411046-147913-8-768x327.jpeg 768w" sizes="(max-width: 1256px) 100vw, 1256px" /></p>
<h4><strong>Key Insight<br />
</strong></h4>
<p>Security breakdowns stemmed from weak telemetry, unclear alert ownership, and strained frontline capacity, not attacker sophistication.</p>
<h4><strong>Strategic Takeaways </strong></h4>
<ul>
<li>Strengthening the human layer requires strengthening the systems that support it.</li>
<li>Ensure alerts are triaged with clear ownership and escalation paths.</li>
<li>Prioritize expanding EDR to extended detection and response (XDR), for centralized detection with rich context.</li>
<li>Address tooling gaps that leave unmanaged assets or disablement vectors exposed.</li>
</ul>
<p>Initial access is only the beginning. In many cases, what starts as a single deceptive interaction results in wide-ranging data exposure and attackers are optimizing for that exact outcome.</p>
<h3>Social Engineering Leads to Higher Rates of Data Exposure</h3>
<p>Social engineering attacks led to data exposure in 60% of cases, according to our IR data. That is 16 percentage points higher than attacks involving other initial access vectors. This includes direct exfiltration as well as indirect exposure through credential theft, unauthorized access to internal systems or deployment of infostealers and remote access Trojans.</p>
<p><img  class="alignnone wp-image-148013 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-413875-147913-9.jpeg" alt="Bar chart titled &quot;Figure 4. Data Exposure Rates: Social Engineering vs. Other Attacks.&quot; It shows two categories: Social Engineering Attacks with 60% Data Exposure and 28% No Data Exposure, and At Large Attacks with 44% Data Exposure and 38% No Data Exposure. Data exposure rates are depicted in yellow and no data exposure rates in red." width="1228" height="518" data-srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-413875-147913-9.jpeg 1228w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-413875-147913-9-786x332.jpeg 786w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-413875-147913-9-768x324.jpeg 768w" sizes="(max-width: 1228px) 100vw, 1228px" /></p>
<h4><strong>Key Insight</strong></h4>
<p>Roughly half of social engineering cases were business email compromises (BEC), and almost 60% of all BEC cases saw data exposure, showing that threat actors moved quickly from gaining access to exfiltrating data or harvesting credentials during these kinds of incidents. Additionally, general network intrusions and ransomware were the two other top incident types where data was exposed. Of those incident types, social engineering was in the top two initial access vectors, showing the popularity of this technique across different types of intrusions and actors.</p>
<p>Credential exposure was also a common precursor to broader data loss. In several cases, attackers reused compromised credentials to access file shares, customer systems or cloud environments. This chained exposure effect amplifies the impact of a single successful lure, turning one compromised identity into broader organizational risk.</p>
<h4><strong>Strategic Takeaways</strong></h4>
<p>Social engineering isn’t just an access problem, it’s a significant data-loss risk. To reduce exposure, security teams must improve visibility into user behavior after login, not just focus on preventing initial compromise. Key measures include:</p>
<ul>
<li>Restricting access to sensitive assets by default.</li>
<li>Monitoring lateral movement and file access.</li>
<li>Implementing just-in-time provisioning for privileged operations.</li>
<li>Data loss prevention (DLP) policies and data tagging should also account for the likelihood of social engineering-driven access, especially via compromised accounts.</li>
</ul>
<h3>Profit Remains the Primary Driver</h3>
<p>Financial gain is the dominant motive across Unit 42 incident response cases, and social engineering attacks are no exception. Nearly all social engineering intrusions between May 2024 and May 2025 were financially motivated (see Figure 5), with threat actors seeking to monetize access through data theft, extortion, ransomware or resale of credentials.</p>
<p><strong><img  class="alignnone wp-image-148024 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-416579-147913-10.jpeg" alt="Pie chart titled &quot;Figure 5: Threat Actor Motives&quot; shows 93% Financial Gain, 4% Convenience/Expediency, and 3% Other." width="600" height="723" data-srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-416579-147913-10.jpeg 802w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-416579-147913-10-365x440.jpeg 365w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-416579-147913-10-581x700.jpeg 581w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-416579-147913-10-768x925.jpeg 768w" sizes="(max-width: 600px) 100vw, 600px" /></strong></p>
<h4><strong>Key Insight</strong></h4>
<p>Social engineering remains a preferred access method because it's easy to execute, infrastructure-light and often avoids detection—with attackers reusing proven tools like phishing kits and impersonation scripts to achieve compromise-to-monetization in under 48 hours, delivering superior cost-benefit over technical exploits by leveraging trust-based access to bypass hardened controls.</p>
<h4><strong>Strategic Takeaways </strong></h4>
<p>Attackers continue to choose social engineering not for its sophistication, but for its ease, speed and reliability. Detection and response must be tuned to spot monetization signals (including data staging, file movement and abnormal system access) even when the intrusion begins with a single deceptive message.</p>
<p><img  class="alignnone wp-image-148035 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-419434-147913-11.jpeg" alt="Text in an image reads: &quot;Why Social Engineering Works&quot; with bullet points on technology weak points, human error and trust, and privilege pathways in cybersecurity contexts." width="2048" height="569" data-srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-419434-147913-11.jpeg 2048w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-419434-147913-11-786x218.jpeg 786w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-419434-147913-11-1920x533.jpeg 1920w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-419434-147913-11-768x213.jpeg 768w, https://unit42.paloaltonetworks.com/wp-content/uploads/2025/07/word-image-419434-147913-11-1536x427.jpeg 1536w" sizes="(max-width: 2048px) 100vw, 2048px" /></p>
<h2><a id="post-147913-_j90ratamswuz"></a>Recommendations for Defenders</h2>
<p>Social engineering continues to outperform other access vectors, not through technical sophistication, but by exploiting gaps between people, process and platform. To counter this, defenders must focus on identity resilience, visibility across workflows, and intelligence-led operations. Capabilities such as UEBA and ITDR play an increasingly critical role in identifying abnormal activity and blocking account takeover attempts.</p>
<p>The following eight recommendations draw directly from Unit 42’s incident response caseload and threat research:</p>
<h3><a id="post-147913-_wtd9v7w5rykv"></a>Correlate Identity Signals To Detect Abuse Earlier</h3>
<p>Attackers often appear legitimate, until anomalous behaviors emerge. Correlating signals across identity, device and session behavior helps security operations teams detect social engineering attacks before they escalate. Platforms such as <a href="https://www.paloaltonetworks.com/cortex/cortex-xsiam"><strong>Cortex XSIAM</strong></a> accelerate this process, enabling faster threat detection and containment without relying on traditional indicators of compromise (IOCs).</p>
<h3><a id="post-147913-_o568cq8al5jk"></a>Enforce Zero Trust Across Access Pathways</h3>
<p>A strong Zero Trust posture limits attacker movement after initial access. Apply conditional access policies that assess device trust, location and login behavior before granting access. Combine least-privilege principles, just-in-time access and network segmentation to restrict lateral movement and reduce blast radius.</p>
<h3><a id="post-147913-_nyfplspx0sce"></a>Strengthen the Human Layer With Detection and Training</h3>
<p>Employees are part of the detection surface. Harden common abuse points (email, browsers, messaging apps, domain name system (DNS)) with intelligence-led controls. Train frontline teams such as HR and IT support to recognize and report impersonation, voice lures and pretexting. Simulate current attack techniques, including help desk spoofing and MFA manipulation.</p>
<h3><a id="post-147913-_19syv66cns3k"></a>Strengthen Detection With Identity and Behavioral Analytics</h3>
<p>Detecting social engineering requires visibility beyond traditional indicators. Correlate signals across identity, endpoint, network and SaaS activity to expose escalation attempts early. Capabilities like UEBA and ITDR help surface anomalies such as impersonation, session abuse and credential misuse.</p>
<h3><a id="post-147913-_8nn59j3ljo3c"></a>Control and Monitor Illegitimate Use of Native Tools and Business Process Workflows</h3>
<p>Attackers can use built-in utilities such as PowerShell or WMI to move undetected. Establish behavioral baselines and alert on anomalies. Map business workflows to uncover where process trust can be exploited, particularly escalation points that rely on fast-track approvals or assumed identity, such as help desk credential resets, finance system overrides or privileged access granted through informal Slack or Teams messages.</p>
<h3><a id="post-147913-_336czld8d2mi"></a>Build Resilience Through Simulation and Playbook Readiness</h3>
<p>Preparation narrows response gaps. Run live drills based on current social engineering tactics, such as impersonation or chained credential use. Validate playbooks, involve cross-functional teams and integrate Unit 42 threat intelligence into both blue team exercises and user awareness programs.</p>
<h3><a id="post-147913-_s5nqrcvraq6h"></a>Enforce Network-Layer Controls</h3>
<p>Deploy Advanced DNS Security and Advanced URL filtering to block access to malicious infrastructure. These controls help detect and prevent social engineering attacks that rely on spoofed domains, typo-squatting, SEO poisoning and link-based credential theft. Visibility at the network layer adds a critical line of defence when endpoint or identity-based detection fails.</p>
<p><strong>Lock Down Identity Recovery Paths</strong></p>
<p>Threat actors increasingly target IT help desks to reset credentials and bypass MFA. Strengthen controls around account recovery by enforcing strict identity verification protocols, limiting who can initiate resets, and monitor for unusual request patterns. Help desk staff should receive regular training grounded in real-world threat activity.</p>
<h2><a id="post-147913-_fae2lchclye2"></a>Final Thoughts</h2>
<p>Social engineering continues to evolve, but its success still depends on trust. Defenders must think beyond malware and infrastructure controls. The new perimeter is shaped by people, processes and the decisions they make in real time. The recommendations in this section are designed to strengthen those decisions and create an environment where trust cannot be easily exploited.</p>

<h2><a id="post-147913-_w2ca2mcp6wn3"></a>Data and Methodology</h2>
<p>The mission of this report is to provide readers with a strategic understanding of existing and anticipated threat scenarios, enabling them to implement more effective protection strategies.</p>
<p>We sourced data for this report from more than 700 cases Unit 42 responded to between May 2024 and May 2025. Our clients range from small organizations with fewer than 50 personnel to Fortune 500 and Global 2000 companies and government organizations with more than 100,000 employees.</p>
<p>The affected organizations were headquartered in 49 unique countries. About 73% of the targeted organizations in these cases were located in North America. Cases related to organizations based in Europe, the Middle East and Asia-Pacific form the other approximately 27% of the work. Attacks frequently have impact beyond the locations where organizations are headquartered.</p>
<p>Findings may differ from those published in the January 2025 <a href="https://www.paloaltonetworks.com/resources/research/unit-42-incident-response-report?utm_source=google-jg-amer-unit42-unrc-unpt&amp;utm_medium=paid_search&amp;utm_campaign=google-unit42-unit42_port-amer-multi-lead_gen-en-brand&amp;utm_content=7014u000001ZyZbAAK&amp;utm_term=unit%2042%20incident%20response&amp;cq_plac=&amp;cq_net=g&amp;gad_source=1&amp;gad_campaignid=20369915902&amp;gbraid=0AAAAADHVeKl-iJ526vXa9_8M4CQLN7U8s&amp;gclid=CjwKCAjwprjDBhBTEiwA1m1d0jcc13ZR37ezKDbwwclTLp2OGMp0fAgfYH-OGyEcJ4AWxw6n_sdFZxoCSP8QAvD_BwE">Global Incident Response Report</a>, which analyzed IR cases from October 2023 to October 2024. Differences in percentages reflect both the different timeframes and the focused nature of this report, which emphasizes social engineering-specific intrusions across verticals.</p>
<p>We excluded some factors in our data that might compromise our analytical integrity. For example, we supported customers investigating possible effects of CVE-2024-3400, causing this vulnerability to appear with unusual frequency in our data set. Where necessary, we recalibrated the data to address any statistical imbalances.</p>

(function($) {
	// Migrated from the unit42-v5 + Modifications

	var subscription_form = document.getElementById('unit42footerSubscription_form');

	window.captchaComplete = function() {
		subscribeSuccess = true;
		if ($(mail).val() != '' && isEmail($(mail).val())) {
			$(subscription_form_button).removeClass('is-disabled');
		}
		setTimeout(function() {
			$(email).focus();
			$('.g-recaptcha iframe').attr('tabindex', '-1');
		}, 100)
	}
	window.captchaExpires = function() {
		subscribeSuccess = false;
		$(subscription_form_button).addClass('is-disabled', true);
	} 
	$(subscription_form).submit(function(e) {
		e.preventDefault();
		e.stopImmediatePropagation();
		updateEmailMask();
		var success = true;
		var form = $(this);
		var mail = form.find('input[name="Email"]');
		
		if (mail.val() === '') {
			mail.addClass('has-error');
			showError(1);
			success = false;	
		}  else if (!isEmail(mail.val())){
			showError(2);
			success = false;
		}
		else {
			mail.removeClass('has-error');
			$('.error-mail').addClass('d-none');
		}

		if (!subscribeSuccess) {
			$('.error-recaptcha').removeClass('d-none');
		} else {
			$('.error-recaptcha').addClass('d-none');
		}

		if (success && subscribeSuccess) {
			$.ajax({
				type: 'POST',
				url: form.attr('action'),
				data: form.serialize(),
				beforeSend: function() {
                 form.find('button').addClass('is-loading');
    			},
				success: function(msg) {
					form.find('.form-success-message').html('<p class="success-message">You have been successfully subscribed</p>');
					form.find('button').removeClass('is-loading');
					$(email).val('');
					clearError();
				},
				error: function(jqXHR, textStatus, errorThrown) {
					$(subscription_form_button).addClass('is-disabled', true);
					form.find('button').removeClass('is-loading');
					
				}
			});
		}
		return false;
	});
	function showError(error_type){
		if(error_type == 1) {
			$('.error-mail').text("Please enter the email address.").addClass('error-show');
			$(subscription_form_button).addClass('is-disabled');
		} else if(error_type == 2){
			$('.error-mail').text("Please provide a valid e-mail address.").addClass('error-show');
			$(subscription_form_button).addClass('is-disabled');
		}	
		$(subscription_form_button).removeClass('is-loading');
	}
	function clearError(){
		$('.error-mail').text("").removeClass('error-show');;
		$(subscription_form_button).removeClass('is-loading');
		$(subscription_form_button).removeClass('is-disabled');
	}
	
	$(email).on('input', function (event) { 
		var email = $(this).val();
		if (isEmail(email) ) {			
			clearError();
		} else if(email == ""){
			clearError();
		} else{
			showError(2);			  
		}
    });

	function isEmail(email) {
		var re = /^(([^<>()\[\]\\.,;:\s@"]+(\.[^<>()\[\]\\.,;:\s@"]+)*)|(".+"))@((\[[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\])|(([a-zA-Z\-0-9]+\.)+[a-zA-Z]{2,}))$/;
		return re.test(String(email).toLowerCase());
	}

                  script.src = 'https://www.google.com/recaptcha/api.js?hl=en_US';
                  head.appendChild(script);
               }
            });
        }
	
	function updateEmailMask() {
		var email = $("#unit42footerSubscription_form input[name='Email']").val();
		if (email && email.trim() != '') {
			var maskedEmail = maskEmailAddress(email);
			$("#unit42footerSubscription_form input[name='emailFormMask']").val(maskedEmail);
		}
	}

	function maskEmailAddress (emailAddress) {
		function mask(str) {
			var strLen = str.length;
			if (strLen > 4) {
				return str.substr(0, 1) + str.substr(1, strLen - 1).replace(/\w/g, '*') + str.substr(-1,1);
			}
			return str.replace(/\w/g, '*');
		}
		return emailAddress.replace(/([\w.]+)@([\w.]+)(\.[\w.]+)/g, function (m, p1, p2, p3) {
			return mask(p1) + '@' + mask(p2) + p3;
		});

		return emailAddress;
	}
