QR codes allow companies to interact seamlessly with their customer base for payments, enabling customers to join rewards programs and sign up for apps or mailing services. People have grown used to QR codes in daily life, and often scan them without sufficient caution, increasing their susceptibility to attacks.\r\n\r\nThe popularity of QR codes has led to their use by attackers. In our offline web crawlers, we currently find an average of 75,000 detections of QR codes each day, with 15% of these pages containing QR codes leading to malicious links. This represents an average of over 11,000 detections of malicious QR code use each day.\r\nProblem of Evasive QR Code Redirects\r\nWe looked beyond the recognized risks of QR codes. While straightforward QR code web-based attacks remain a threat, our focus shifted to understanding how attackers are leveraging the following trends to remain evasive to both victims and security controls:\r\n\r\n \tQR code shorteners\r\n \tIn-app deep links (special URLs that allow people to open specific content within a mobile app)\r\n \tDirect app file downloads\r\n\r\nThese tactics represent an evolution in QR code-based attacks that security teams need to address.\r\n\r\nPrevious Unit 42 research has covered several key attack vectors for phishing QR codes hosted on documents, which are also relevant when hosted on websites. Attacks through these vectors can be effective for several reasons including:\r\n\r\n \tLower user vigilance\r\n \tSecurity solutions having difficulty extracting URLs embedded in QR codes\r\n \tComplex redirection chains that obscure final destinations\r\n \tWeaker security controls on personal mobile devices\r\n \tHosting on otherwise legitimate-looking pages\r\n\r\nBuilding upon this threat model, in-app deep links allow the attacker to target specific apps and trigger specific behavior (Figure 1).\r\n\r\n[caption id=\"attachment_172818\" align=\"alignnone\" width=\"960\"] Figure 1. QR code threat model.[\/caption]\r\n\r\nQR codes on websites need to be analyzed by security crawlers and other security solutions. To close this security gap, specific QR code detection techniques must be deployed to analyze the various data types stored in QR codes:\r\n\r\n \tStandard HTTPS URLs\r\n \tDeep links\r\n \tNon-URL content (e.g., JSON, plaintext)\r\n\r\nKey Definitions\r\nQR code shorteners are services that combine a URL shortener with a QR code generator to create a shorter, more scannable QR code that links to a long URL. These shorteners offer benefits such as reducing the size of the QR code, allowing attackers to change the destination URL later, and tracking scan data in a single dashboard.\r\n\r\nIn-app deep links are hyperlinks that direct visitors to a specific screen or content within a mobile app. In-app deep links can use both custom URL schemes (i.e., sms:+1234567890:Hello, tg[:]\/\/login?token= ) or standard web URLs (i.e., hxxps[:]\/\/wa[.]me\/settings\/linked_devices#) that the operating system redirects to the app.\r\n\r\nFigure 2 shows an example that displays a phishing site impersonating a job match and training program website that hosts a payment in-app deep link. Deep links are often used to improve user experience by reducing the number of steps to access specific content from external sources like emails, social media, authentication tokens or ads.\r\n\r\n[caption id=\"attachment_172829\" align=\"alignnone\" width=\"1000\"] Figure 2. QR code in-app deep link example.[\/caption]\r\nThe Stealth Factor: QR Code Shorteners\r\nAttackers use QR code shorteners to mask malicious destinations. QR code shorteners convert a static image into a dynamic endpoint. Consequently, the attacker can change the redirect destination at will.\r\n\r\nThe attacker is also able to leverage the good reputation of QR code shortener services to evade detection of malicious activity. Even security-conscious people who check the URL preview before scanning cannot determine the final destination when presented with shortened links. This technique effectively prevents targets from being aware of potential threats until after the malicious payload has been delivered.\r\n\r\nOur previous article has already talked about the risk of URL shorteners more broadly. However, the combination of a QR code and URL shortener is even more likely to bypass scrutiny.\r\nSteady Increase in QR Code Shortener Traffic\r\nWe have seen QR code shortener traffic grow steadily over the past three years (Figure 3).\r\n\r\n[caption id=\"attachment_172840\" align=\"alignnone\" width=\"1000\"] Figure 3. QR code shortener traffic trends, 2023-2025.[\/caption]\r\n\r\nWe see a steady increase of QR code shortener traffic in our telemetry. This includes a 55% increase from the first half of 2023 to the first half of 2024 and a 44% increase from the first half of 2024 to the first half of 2025. This data is based on the following popular QR code shortener services:\r\n\r\n \tqrcc[.]io\r\n \tqrco[.]de\r\n \tme-qr[.]com\r\n \tqr[.]io\r\n \tqrfy[.]com\r\n \tqrfy[.]io\r\n \tget-qr[.]com\r\n \tqr[.]ne, qrs[.]ly\r\n\r\nMost Misused QR Code Shortener Services\r\nOur telemetry reveals that qrco[.]de, me-qr[.]com and qrs[.]ly are the most used QR code shorteners. Compared to the top QR code shorteners mentioned in the Anti-Phishing Working Group (APWG) phishing trends report [PDF], qrs[.]ly is a notable new addition as the QR code shortener used in 7.3% of the malicious URLs observed.\r\nTargeted Industries\r\nFinancial services was the most impacted industry when considering compromised QR code shorteners, accounting for 29% of this type of attack. This is followed by high tech (19%) and wholesale and retail (14%). Significantly, QR code shorteners for financial services make up only 4.8% of this type of traffic as a whole. After a few days, the URL from this QR code no longer worked, illustrating how QR code shorteners are often ephemeral and can quickly cease redirecting to the original malicious endpoint.\r\n\r\n[caption id=\"attachment_172862\" align=\"alignnone\" width=\"1000\"] Figure 5. Malicious QR code shortener example.[\/caption]\r\nIn-App Deep Links Vulnerabilities: More Than Just Web Browsing\r\nModern mobile devices support a wide range of QR code actions beyond simple web browsing. The distribution of in-app deep links in QR codes is an understudied area despite its exploitability. In-app deep links account for about three percent of the QR codes in our telemetry. Attackers can either misuse app functionality (e.g., adding a trusted device, or sending a payment), or push malicious content to those apps (such as, adding malicious links to calendar invites).\r\n\r\nDefenders face a challenge in detecting malicious in-app deep links embedded in QR codes because the activity generated by these links is often invisible to standard web crawlers. Effective detection necessitates a mobile sandbox environment with the specific app installed to properly observe and analyze this activity. Custom in-app deep links lack standardization across applications. This makes identifying malicious signals difficult to generalize, often requiring individualized investigation for each case.\r\n\r\nBoth iOS and Android devices can process QR codes with in-app deep links that have direct app integration. We categorize in-app deep links as those that apply to the following types of apps:\r\n\r\n \tSocial media and communications\r\n \tApp stores\r\n \tPayment\r\n \tSystem utilities (e.g., Wi-Fi, contacts, calendar, telephone, email, SMS, navigation)\r\n\r\nThe three most popular custom app URLs that we found were for Telegram, XHS Discover (RedNote) and Line, which respectively account for 44.7%, 1.8% and 0.8% of in-app deep links. The ones not observed are plausible, but hypothetical scenarios. We will further discuss the scenarios observed in our data collection below.\r\nCurrent Attack Trends and Examples\r\nFinancial Fraud In-App Deep Links\r\nFinancial in-app deep links represent a significant financial risk to potential victims. QR codes are commonly used in legitimate business transactions to facilitate payments, making it straightforward for attackers to misuse this trusted interaction through phishing schemes. We observed legitimate in-app deep links from popular payment apps such as:\r\n\r\n \tWeChat Pay\r\n \tAlipay\r\n \tBitcoin\r\n \tEthereum\r\n \tLitCoin\r\n \tMetamask\r\n \tTrust (wallet)\r\n\r\nThe familiarity and trust people have with payment-related QR codes create an ideal environment for social engineering attacks, where malicious QR codes can closely mimic legitimate payment requests. Phishing campaigns using pressure tactics can manipulate people into making quick payments.\r\n\r\nBelow, we share a few examples where an attacker attempts to trigger a financial transaction using a QR code. Figure 7 includes two examples. The first example is a phishing campaign claiming easy returns on investment, asking for an initial payment through a Bitcoin in-app deep link. The second example is a hacking for hire service advertising and providing easy payment with a WeChat payment in-app deep link.\r\n\r\n[caption id=\"attachment_172884\" align=\"alignnone\" width=\"960\"] Figure 7. Examples of malicious financial in-app deep links.[\/caption]\r\n\r\nFigure 8 illustrates another get-rich-quick phishing scheme that requests an initial payment through a popular cryptocurrency wallet via a QR code with an in-app deep link.\r\n\r\n[caption id=\"attachment_172895\" align=\"alignnone\" width=\"960\"] Figure 8. QR code phishing scheme that uses a popular cryptocurrency wallet.[\/caption]\r\nMessenger Account Takeover Through In-App Deep Links\r\nAccount takeovers through in-app deep links appear to be a significant phishing vector for messaging and social media sites. Telegram, in particular, was the most prominent application identified in our analysis that uses custom in-app deep links. We found over 35,000 QR codes that contain Telegram in-app deep links such as tg[:]\/\/login or tg[:]\/\/resolve and we observed multiple instances where attackers exploited these links to compromise accounts.\r\n\r\nWe saw three kinds of Telegram in-app deep links:\r\n\r\n \tLogin\r\n \tResolve\r\n \tProxy\r\n\r\nLogin accounted for 97% of the Telegram in-app deep links observed. Login grants the QR code creator authorization to access your account.\r\n\r\nPrevious reporting of Telegram in-app deep link scams warns about these account takeover attacks. Roughly one out of every five host pages with a login Telegram in-app deep link is malicious, based on our conservative estimate.\r\n\r\nFigure 9 includes two examples of such Telegram login scams. However, while Telegram is the most popular, attackers are also targeting other popular communication apps.\r\n\r\n[caption id=\"attachment_172906\" align=\"alignnone\" width=\"960\"] Figure 9. Example of a QR code designed to give an attacker full access to the device and account owner\u2019s Telegram.[\/caption]\r\n\r\n Figure 10 shows an example of a QR code containing an in-app deep link that requests authorization to a target's Line account. This would allow attackers to send Line messages under the device and account owner\u2019s name. Of note, Line has since deprecated this in-app deep link, and the link will now result in an error.\r\n\r\n&nbsp;\r\n\r\n[caption id=\"attachment_172917\" align=\"alignnone\" width=\"960\"] Figure 10. Example of a QR code for a Line account takeover.[\/caption]\r\n\r\nFigure 11 shows an example of a QR code containing an in-app deep link that requests authorization to access a target's Signal account.\r\n\r\n[caption id=\"attachment_172928\" align=\"alignnone\" width=\"960\"] Figure 11. Example of a QR code for a Signal account takeover.[\/caption]\r\n\r\nFigure 12 shows an example of a QR code containing an in-app deep link that requests authorization to access a target's WhatsApp account.\r\n\r\n[caption id=\"attachment_172939\" align=\"alignnone\" width=\"960\"] Figure 12. Example of a QR code for a WhatsApp account takeover.[\/caption]\r\n\r\nIn addition to mass phishing campaigns, there's a clear trend toward more focused attacks aimed at stealing Signal credentials. For instance, the Google Threat Intelligence Group (GTIG) has documented increased efforts by Russia state-aligned actors to compromise Signal Messenger accounts. These attacks frequently misuse Signal's feature to link devices with malicious QR codes.\r\n\r\nMany of these campaigns have targeted Ukraine in the context of the Russia-Ukraine war. In July 2024, the CERT-UA reported on several threat groups, such as UAC-0185 (aka UNC4221), that have specifically targeted messenger accounts.\r\n\r\nOur researchers continue to observe new malicious domains targeting Ukrainian Signal users, including snitch.open-group[.]site and similar variations. After linking a new session to Signal accounts, the attackers can exfiltrate message history and other account information. We have reported discovered information to our Ukrainian cybersecurity partners.\r\n\r\nFigure 13 shows a QR code from a campaign targeting Ukraine-based Signal accounts.\r\n\r\n[caption id=\"attachment_172950\" align=\"alignnone\" width=\"960\"] Figure 13. QR code from a campaign targeting Ukraine-based Signal accounts.[\/caption]\r\nBypassing App Store Security: Direct App Downloads\r\nQR codes are widely used for easy downloading of files and applications. Attackers can exploit this convenience to trick victims into downloading malicious content or installing harmful mobile applications.\r\n\r\nMajor app stores impose strict security and compliance guidelines to limit the distribution of harmful apps. However, attackers may circumvent these security measures by distributing links to unreviewed Android Package Kit (APK) files hosted on their own servers via QR codes.\r\n\r\nOur investigation identified 59,000 detections of host pages distributing a total of 1,457 distinct APK files directly through QR codes, without going through any app store. Notable examples of these distributed APKs are listed below.\r\nGambling\/Casino App Downloads\r\nGambling and casino games websites are distributing their apps through APK files in QR codes.\r\n\r\nFigures 14-16 illustrate some examples of such host pages. They are all hosted by many different domains and request certain Android permissions that could be concerning to people.\r\n\r\nFigure 14 shows an ad for a popular game that includes a QR code, which redirects the victim to another QR code to download a game app named yicai.apk from \u200b\u200bf9999[.]app. This QR code is hosted on 10,022 unique URLs.\r\n\r\nThe app requests read and write permissions to the device's external storage and camera.                 <h2><a id="post-172806-_4lt92rr5muov"></a>Executive Summary</h2>
<p>This article explores the misuse of QR codes in today's threat landscape, covering three areas of concern:</p>
<ul>
<li>QR codes using URL shorteners to disguise malicious destinations</li>
<li>QR codes using in-app deep links to steal account credentials and take control of a victim's apps</li>
<li>QR codes attempting to bypass app store security by linking to direct downloads of malicious apps</li>
</ul>
<p>With QR codes a notable presence in our everyday lives, some people instinctively scan them without hesitation. But QR codes are also a vector for attack. QR codes enable attackers to bypass organizational security by exploiting the weaker controls of personal mobile devices. By doing this, they can trick users into scanning codes and interacting with malicious destinations outside the corporate security perimeter.</p>
<p>Over the past several months, we have tracked campaigns that used QR codes for phishing (known as quishing) and scams. Our telemetry reveals an average of over 11,000 detections of malicious QR codes each day. Investigating these detections, we found that attackers are leveraging QR code shorteners, in-app deep links and direct downloads to bypass people’s awareness and security controls.</p>
<p>In addition to mass campaigns, we see attackers using QR codes for highly targeted messenger app phishing, such as <a href="https://cloud.google.com/blog/topics/threat-intelligence/russia-targeting-signal-messenger" target="_blank" rel="noopener">targeting Ukrainian Signal users</a> in the context of the Russia-Ukraine war. These findings necessitate further analysis of deep links and QR code data.</p>
<p>Palo Alto Networks customers are better protected from the threats described in this article through the following products and services:</p>
<ul>
<li><a href="https://docs.paloaltonetworks.com/advanced-url-filtering/administration" target="_blank" rel="noopener">Advanced URL Filtering</a></li>
<li><a href="https://docs.paloaltonetworks.com/prisma-access-browser" target="_blank" rel="noopener">Prisma Browser</a></li>
</ul>

<h2><a id="post-172806-_40n5dkvznd8t"></a>Phishing QR Codes Not New, but a Growing Threat</h2>
<p>QR codes are not a new technology, but <a href="https://codebroker.com/the-rise-of-qr-codes-in-the-united-states-a-trend-driven-transformation/" target="_blank" rel="noopener">their prevalence</a> has increased with the push for contactless interactions, especially <a href="https://www.lithiosapps.com/blog/rise-and-fall-of-qr-codes-and-how-a-pandemic-revived-them" target="_blank" rel="noopener">during the initial emergency phase of the coronavirus pandemic</a>. QR codes allow companies to interact seamlessly with their customer base for payments, enabling customers to join rewards programs and sign up for apps or mailing services. People have grown used to QR codes in daily life, and often scan them without sufficient caution, increasing their susceptibility to attacks.</p>
<p>The popularity of QR codes has led to their use by attackers. In our offline web crawlers, we currently find an average of 75,000 detections of QR codes each day, with 15% of these pages containing QR codes leading to malicious links. This represents an average of over 11,000 detections of malicious QR code use each day.</p>
<h3><a id="post-172806-_fu273x3wlmtq"></a><strong>Problem of Evasive QR Code Redirects</strong></h3>
<p>We looked beyond the recognized risks of QR codes. While straightforward QR code web-based attacks remain a threat, our focus shifted to understanding how attackers are leveraging the following trends to remain evasive to both victims and security controls:</p>
<ul>
<li><a href="#post-172806-_n4f31vojng4h">QR code shorteners</a></li>
<li><a href="#post-172806-_61lur7d6vrni">In-app deep links</a> (special URLs that allow people to open specific content within a mobile app)</li>
<li><a href="#post-172806-_10nvxr8wfwtr">Direct app file downloads</a></li>
</ul>
<p>These tactics represent an evolution in QR code-based attacks that security teams need to address.</p>
<p>Previous <a href="https://unit42.paloaltonetworks.com/qr-code-phishing/" target="_blank" rel="noopener">Unit 42 research</a> has covered several key attack vectors for phishing QR codes hosted on documents, which are also relevant when hosted on websites. Attacks through these vectors can be effective for several reasons including:</p>
<ul>
<li>Lower user vigilance</li>
<li>Security solutions having difficulty extracting URLs embedded in QR codes</li>
<li>Complex redirection chains that obscure final destinations</li>
<li>Weaker security controls on personal mobile devices</li>
<li>Hosting on otherwise legitimate-looking pages</li>
</ul>
<p>Building upon this threat model, in-app deep links allow the attacker to target specific apps and trigger specific behavior (Figure 1).</p>
<figure id="attachment_172818" aria-describedby="caption-attachment-172818" style="width: 960px" class="wp-caption alignnone"><img  class="wp-image-172818 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2026/02/word-image-525372-172806-1.png" alt="QR Code Threat Model illustrating a threat scenario. QR code threat model.</figcaption></figure>
<p>QR codes on websites need to be analyzed by security crawlers and other security solutions. To close this security gap, specific QR code detection techniques must be deployed to analyze the various data types stored in QR codes:</p>
<ul>
<li>Standard HTTPS URLs</li>
<li>Deep links</li>
<li>Non-URL content (e.g., JSON, plaintext)</li>
</ul>
<h3><a id="post-172806-_uopm7e11pthh"></a><strong>Key Definitions</strong></h3>
<p><strong>QR code shorteners </strong>are services that combine a URL shortener with a QR code generator to create a shorter, more scannable QR code that links to a long URL. These shorteners offer benefits such as reducing the size of the QR code, allowing attackers to change the destination URL later, and tracking scan data in a single dashboard.</p>
<p><strong>In-app deep links </strong>are <a href="https://developer.android.com/training/app-links/create-deeplinks" target="_blank" rel="noopener">hyperlinks that direct visitors to a specific screen or content within a mobile app</a>. In-app deep links can use both custom URL schemes (i.e., <span style="font-family: 'courier new', courier, monospace;">sms:+1234567890:Hello, tg[:]//login?token=</span> ) or standard web URLs (i.e., <span style="font-family: 'courier new', courier, monospace;">hxxps[:]//wa[.]me/settings/linked_devices#</span>) that the operating system redirects to the app.</p>
<p>Figure 2 shows an example that displays a phishing site impersonating a job match and training program website that hosts a payment in-app deep link. Deep links are often used to improve user experience by reducing the number of steps to access specific content from external sources like emails, social media, authentication tokens or ads.</p>
<figure id="attachment_172829" aria-describedby="caption-attachment-172829" style="width: 1000px" class="wp-caption alignnone"><img  class="wp-image-172829 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2026/02/word-image-527541-172806-2.png" alt="Phishing Host Page - Job Match &amp; Training Program. The background is blue with a navigation menu listing &quot;About Us,&quot; &quot;Our Services,&quot; and &quot;Contact Us.&quot; There is a QR code on the right labeled &quot;Payment QR Code Landing Page&quot; with a URL highlighted." width="1000" height="554" data-srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2026/02/word-image-527541-172806-2.png 1670w, https://unit42.paloaltonetworks.com/wp-content/uploads/2026/02/word-image-527541-172806-2-786x436.png 786w, https://unit42.paloaltonetworks.com/wp-content/uploads/2026/02/word-image-527541-172806-2-1262x700.png 1262w, https://unit42.paloaltonetworks.com/wp-content/uploads/2026/02/word-image-527541-172806-2-768x426.png 768w, https://unit42.paloaltonetworks.com/wp-content/uploads/2026/02/word-image-527541-172806-2-1536x852.png 1536w" sizes="(max-width: 1000px) 100vw, 1000px" /><figcaption id="caption-attachment-172829" class="wp-caption-text">Figure 2. QR code in-app deep link example.</figcaption></figure>
<h2><a id="post-172806-_n4f31vojng4h"></a>The Stealth Factor: QR Code Shorteners</h2>
<p>Attackers use QR code shorteners to mask malicious destinations. QR code shorteners convert a static image into a dynamic endpoint. Consequently, the attacker can change the redirect destination at will.</p>
<p>The attacker is also able to leverage the good reputation of QR code shortener services to evade detection of malicious activity. Even security-conscious people who check the URL preview before scanning cannot determine the final destination when presented with shortened links. This technique effectively prevents targets from being aware of potential threats until after the malicious payload has been delivered.</p>
<p>Our previous article has already talked about the <a href="https://unit42.paloaltonetworks.com/why-innocent-clicks-dont-exist-in-cybersecurity/" target="_blank" rel="noopener">risk of URL shorteners</a> more broadly. However, the combination of a QR code and URL shortener is even more likely to bypass scrutiny.</p>
<h3><a id="post-172806-_xor037sexhdm"></a><strong>Steady Increase in QR Code Shortener Traffic</strong></h3>

<h2><a id="post-172806-_61lur7d6vrni"></a>In-App Deep Links Vulnerabilities: More Than Just Web Browsing</h2>
<p>Modern mobile devices support a wide range of QR code actions beyond simple web browsing. The distribution of in-app deep links in QR codes is an understudied area despite its exploitability. In-app deep links account for about three percent of the QR codes in our telemetry. Attackers can either misuse app functionality (e.g., adding a trusted device, or sending a payment), or push malicious content to those apps (such as, adding malicious links to calendar invites).</p>
<p>Defenders face a challenge in detecting malicious in-app deep links embedded in QR codes because the activity generated by these links is often invisible to standard web crawlers. Effective detection necessitates a mobile sandbox environment with the specific app installed to properly observe and analyze this activity. Custom in-app deep links lack standardization across applications. This makes identifying malicious signals difficult to generalize, often requiring individualized investigation for each case.</p>
<p>Both iOS and Android devices can process QR codes with in-app deep links that have direct app integration. We categorize in-app deep links as those that apply to the following types of apps:</p>
<ul>
<li>Social media and communications</li>
<li>App stores</li>
<li>Payment</li>
<li>System utilities (e.g., Wi-Fi, contacts, calendar, telephone, email, SMS, navigation)</li>
</ul>
<p>The three most popular custom app URLs that we found were for Telegram, XHS Discover (RedNote) and Line, which respectively account for 44.7%, 1.8% and 0.8% of in-app deep links. As we discuss later, attackers commonly misuse Telegram and Line.</p>

<h3><a id="post-172806-_kwp0x6d8zqle"></a>Current Attack Trends and Examples</h3>
<h4><a id="post-172806-_gluzx79vg6ta"></a>Financial Fraud In-App Deep Links</h4>
<p>Financial in-app deep links represent a significant financial risk to potential victims. QR codes are commonly used in legitimate business transactions to facilitate payments, making it straightforward for attackers to misuse this trusted interaction through phishing schemes. We observed legitimate in-app deep links from popular payment apps such as:</p>
<ul>
<li>WeChat Pay</li>
<li>Alipay</li>
<li>Bitcoin</li>
<li>Ethereum</li>
<li>LitCoin</li>
<li>Metamask</li>
<li>Trust (wallet)</li>
</ul>
<p>The familiarity and trust people have with payment-related QR codes create an ideal environment for social engineering attacks, where malicious QR codes can closely mimic legitimate payment requests. Phishing campaigns using pressure tactics can manipulate people into making quick payments.</p>

<h4><a id="post-172806-_9drrlcr4bkwg"></a>Messenger Account Takeover Through In-App Deep Links</h4>
<p>Account takeovers through in-app deep links appear to be a significant phishing vector for messaging and social media sites. Telegram, in particular, was the most prominent application identified in our analysis that uses custom in-app deep links. We found over 35,000 QR codes that contain Telegram in-app deep links such as tg[:]//login or tg[:]//resolve and we observed multiple instances where attackers exploited these links to compromise accounts.</p>
<p>We saw three kinds of Telegram in-app deep links:</p>
<ul>
<li>Login</li>
<li>Resolve</li>
<li>Proxy</li>
</ul>
<p>Login accounted for 97% of the Telegram in-app deep links observed. Login grants the QR code creator authorization to access your account.</p>
<p>Previous reporting of <a href="https://www.criminalip.io/knowledge-hub/blog/29770">Telegram in-app deep link scams</a> warns about these account takeover attacks. Roughly one out of every five host pages with a login Telegram in-app deep link is malicious, based on our conservative estimate.</p>

<h2><a id="post-172806-_10nvxr8wfwtr"></a>Bypassing App Store Security: Direct App Downloads</h2>
<p>QR codes are widely used for easy downloading of files and applications. Attackers can exploit this convenience to trick victims into downloading malicious content or installing harmful mobile applications.</p>
<p>Major app stores impose strict security and compliance guidelines to limit the distribution of harmful apps. However, attackers may circumvent these security measures by distributing links to unreviewed Android Package Kit (APK) files hosted on their own servers via QR codes.</p>
<p>Our investigation identified 59,000 detections of host pages distributing a total of 1,457 distinct APK files directly through QR codes, without going through any app store. Notable examples of these distributed APKs are listed below.</p>
<h3><a id="post-172806-_f5wxm8mkyqyw"></a>Gambling/Casino App Downloads</h3>

<h2><a id="post-172806-_j094b7unecbm"></a>Conclusion</h2>
<p>The attack scenarios and variety of examples we've discovered illustrate the extensive potential and existing prevalence of QR code misuse. The fundamental challenges of this type of misuse are user awareness and lack of visibility from current detection systems.</p>
<p>Most people scanning QR codes don't anticipate the broad range of device functions that can be triggered from in-app deep links or unexpected endpoints from QR code shorteners. This expectation mismatch creates a significant security weak spot that attackers can actively exploit.</p>
<p>User education remains critical — people need to understand that QR codes can do much more than simply open webpages.</p>

	function maskEmailAddress (emailAddress) {
		function mask(str) {
			var strLen = str.length;
			if (strLen > 4) {
				return str.substr(0, 1) + str.substr(1, strLen - 1).replace(/\w/g, '*') + str.substr(-1,1);
			}
			return str.replace(/\w/g, '*');
		}

			return mask(p1) + '@' + mask(p2) + p3;
		});

	}
