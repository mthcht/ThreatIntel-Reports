<!--more-->
<p>&nbsp;</p>
<p>Tox is a peer-to-peer serverless messaging system that uses NaCl for encryption and decryption. Since it’s serverless, it uses UDP and the DHT to find online peers, <a href="https://github.com/irungentoo/toxcore/blob/master/docs/updates/DHT.md"><span>similar to what BitTorrent does</span></a>. It is also meant to be anonymous, which means each user gets a public key that also acts as their ID within the system.</p>
<p>&nbsp;</p>
<p>Tox has been used <a href="https://unit42.paloaltonetworks.com/helloxd-ransomware/"><span>before</span></a> by threat actors as a contact method, but in this case, Tox is being used for remote administration<span style="font-size: 12px;">. </span>The Uptycs <a href="/products/use-cases/threat-hunting" rel="noopener" target="_blank">threat research</a> team recently found an ELF sample that acts as a bot and can run scripts on the victim machine using the Tox protocol.</p>
<p>&nbsp;</p>
<h2>Technical Overview</h2>
<p>The binary found in the wild is a stripped but dynamic executable, making decompilation easier. The entire binary appears to be written in C, and has only statically linked the <a href="https://github.com/TokTok/c-toxcore"><span>c-toxcore</span></a> library.</p>
<p>&nbsp;</p>
<p>Figure 1 shows the decompiled main function of the sample or starting point:</p>
<p style="text-align: center;"><a href="https://www.uptycs.com/hubfs/Figure1tox.png" rel="noopener" target="_blank"><img src="https://www.uptycs.com/hs-fs/hubfs/Figure1tox.png?width=921&amp;height=628&amp;name=Figure1tox.png" alt="Figure1tox" width="921" height="628" loading="lazy" style="width: 921px; margin-left: auto; margin-right: auto; display: block; height: auto; max-width: 100%;" srcset="https://www.uptycs.com/hs-fs/hubfs/Figure1tox.png?width=461&amp;height=314&amp;name=Figure1tox.png 461w, https://www.uptycs.com/hs-fs/hubfs/Figure1tox.png?width=921&amp;height=628&amp;name=Figure1tox.png 921w, https://www.uptycs.com/hs-fs/hubfs/Figure1tox.png?width=1382&amp;height=942&amp;name=Figure1tox.png 1382w, https://www.uptycs.com/hs-fs/hubfs/Figure1tox.png?width=1842&amp;height=1256&amp;name=Figure1tox.png 1842w, https://www.uptycs.com/hs-fs/hubfs/Figure1tox.png?width=2303&amp;height=1570&amp;name=Figure1tox.png 2303w, https://www.uptycs.com/hs-fs/hubfs/Figure1tox.png?width=2763&amp;height=1884&amp;name=Figure1tox.png 2763w" sizes="(max-width: 921px) 100vw, 921px"></a><em><span style="font-size: 16px;">Figure 1: main function</span></em></p>
<p>&nbsp;</p>
<p>The shell_script variable itself is suspicious, but it only kills certain programs that are known to infect linux servers. It also deletes the crontab, which is something frequently used for persistence. Highlighted (see figure 1) is a function called start_routine1 (decompilation in figure 2), which opens a file with a random filename in /var/tmp/ (Figure 3) and dumps the contents of shell_script in there and later executes it.</p>
<p style="text-align: center;"><a href="https://www.uptycs.com/hubfs/Figure2tox.png" rel="noopener" target="_blank"><img src="https://www.uptycs.com/hs-fs/hubfs/Figure2tox.png?width=923&amp;height=625&amp;name=Figure2tox.png" alt="Figure2tox" width="923" height="625" loading="lazy" style="margin-left: auto; margin-right: auto; display: block; width: 923px; height: auto; max-width: 100%;" srcset="https://www.uptycs.com/hs-fs/hubfs/Figure2tox.png?width=462&amp;height=313&amp;name=Figure2tox.png 462w, https://www.uptycs.com/hs-fs/hubfs/Figure2tox.png?width=923&amp;height=625&amp;name=Figure2tox.png 923w, https://www.uptycs.com/hs-fs/hubfs/Figure2tox.png?width=1385&amp;height=938&amp;name=Figure2tox.png 1385w, https://www.uptycs.com/hs-fs/hubfs/Figure2tox.png?width=1846&amp;height=1250&amp;name=Figure2tox.png 1846w, https://www.uptycs.com/hs-fs/hubfs/Figure2tox.png?width=2308&amp;height=1563&amp;name=Figure2tox.png 2308w, https://www.uptycs.com/hs-fs/hubfs/Figure2tox.png?width=2769&amp;height=1875&amp;name=Figure2tox.png 2769w" sizes="(max-width: 923px) 100vw, 923px"></a><em><span style="font-size: 16px;">Figure 2: start_routine1</span></em></p>
<p>&nbsp;</p>
<p>&nbsp;</p>
<p>&nbsp;</p>
<p style="text-align: center;"><a href="https://www.uptycs.com/hubfs/Figure%203tox.png" rel="noopener" target="_blank"><img src="https://www.uptycs.com/hs-fs/hubfs/Figure%203tox.png?width=941&amp;height=373&amp;name=Figure%203tox.png" alt="Figure 3tox" width="941" height="373" loading="lazy" style="width: 941px; height: auto; max-width: 100%; margin-left: auto; margin-right: auto; display: block;" srcset="https://www.uptycs.com/hs-fs/hubfs/Figure%203tox.png?width=471&amp;height=187&amp;name=Figure%203tox.png 471w, https://www.uptycs.com/hs-fs/hubfs/Figure%203tox.png?width=941&amp;height=373&amp;name=Figure%203tox.png 941w, https://www.uptycs.com/hs-fs/hubfs/Figure%203tox.png?width=1412&amp;height=560&amp;name=Figure%203tox.png 1412w, https://www.uptycs.com/hs-fs/hubfs/Figure%203tox.png?width=1882&amp;height=746&amp;name=Figure%203tox.png 1882w, https://www.uptycs.com/hs-fs/hubfs/Figure%203tox.png?width=2353&amp;height=933&amp;name=Figure%203tox.png 2353w, https://www.uptycs.com/hs-fs/hubfs/Figure%203tox.png?width=2823&amp;height=1119&amp;name=Figure%203tox.png 2823w" sizes="(max-width: 941px) 100vw, 941px"></a><br><em><span style="font-size: 16px;">Figure 3: The script that is dropped into /var/tmp/</span></em></p>
<p>&nbsp;</p>
<p>The dropped shell script contains commands to kill cryptominer related processes.</p>
<p>start_routine2 gets called via pthread_create in the main function, which appears to send the output of every command over UDP to the Tox recipient.</p>
<p style="text-align: center;"><a href="https://www.uptycs.com/hubfs/Figure4tox.png" rel="noopener" target="_blank"><img src="https://www.uptycs.com/hs-fs/hubfs/Figure4tox.png?width=923&amp;height=627&amp;name=Figure4tox.png" alt="Figure4tox" width="923" height="627" loading="lazy" style="width: 923px; margin-left: auto; margin-right: auto; display: block; height: auto; max-width: 100%;" srcset="https://www.uptycs.com/hs-fs/hubfs/Figure4tox.png?width=462&amp;height=314&amp;name=Figure4tox.png 462w, https://www.uptycs.com/hs-fs/hubfs/Figure4tox.png?width=923&amp;height=627&amp;name=Figure4tox.png 923w, https://www.uptycs.com/hs-fs/hubfs/Figure4tox.png?width=1385&amp;height=941&amp;name=Figure4tox.png 1385w, https://www.uptycs.com/hs-fs/hubfs/Figure4tox.png?width=1846&amp;height=1254&amp;name=Figure4tox.png 1846w, https://www.uptycs.com/hs-fs/hubfs/Figure4tox.png?width=2308&amp;height=1568&amp;name=Figure4tox.png 2308w, https://www.uptycs.com/hs-fs/hubfs/Figure4tox.png?width=2769&amp;height=1881&amp;name=Figure4tox.png 2769w" sizes="(max-width: 923px) 100vw, 923px"></a><span style="font-size: 16px;"><em>Figure 4: start_routine2</em></span></p>
<p>&nbsp;</p>
<p>There are some bash commands (see Figure 5) that warrant attention. The dig command attempts to use resolver4.opendns.com as a DNS server and looks up myip.opendns.com, something similar to <a href="https://www.ptsecurity.com/ru-ru/research/pt-esc-threat-intelligence/space-pirates-tools-and-connections/"><span>this</span></a>.</p>
<p>&nbsp;</p>
<p>Using curl -s -m 20 ifconfig.me, the IP address of the machine is saved into a variable named name_var and&nbsp; `cat /var/lib/dbus/machine-id` gives the hardware ID of the machine, which is also stored into the same variable and further used in tox_self_set_name to set the name of the user. Later, `nproc`, `uname -a` and `whoami` commands are run, which are then stored into status_var, used in tox_self_set_status_message to set the status message of the user.</p>
<p style="text-align: center;"><a href="https://www.uptycs.com/hubfs/Figure%206%20Tox.png" rel="noopener" target="_blank"><img src="https://www.uptycs.com/hs-fs/hubfs/Figure%206%20Tox.png?width=920&amp;height=628&amp;name=Figure%206%20Tox.png" alt="Figure 6 Tox" width="920" height="628" loading="lazy" style="width: 920px; margin-left: auto; margin-right: auto; display: block; height: auto; max-width: 100%;" srcset="https://www.uptycs.com/hs-fs/hubfs/Figure%206%20Tox.png?width=460&amp;height=314&amp;name=Figure%206%20Tox.png 460w, https://www.uptycs.com/hs-fs/hubfs/Figure%206%20Tox.png?width=920&amp;height=628&amp;name=Figure%206%20Tox.png 920w, https://www.uptycs.com/hs-fs/hubfs/Figure%206%20Tox.png?width=1380&amp;height=942&amp;name=Figure%206%20Tox.png 1380w, https://www.uptycs.com/hs-fs/hubfs/Figure%206%20Tox.png?width=1840&amp;height=1256&amp;name=Figure%206%20Tox.png 1840w, https://www.uptycs.com/hs-fs/hubfs/Figure%206%20Tox.png?width=2300&amp;height=1570&amp;name=Figure%206%20Tox.png 2300w, https://www.uptycs.com/hs-fs/hubfs/Figure%206%20Tox.png?width=2760&amp;height=1884&amp;name=Figure%206%20Tox.png 2760w" sizes="(max-width: 920px) 100vw, 920px"></a><em><span style="font-size: 16px;">Figure 5: some of the main function</span></em></p>
<p>&nbsp;</p>
<p style="text-align: center;"><img src="https://www.uptycs.com/hs-fs/hubfs/Figure%206%20Tox.png?width=920&amp;height=628&amp;name=Figure%206%20Tox.png" alt="Figure 6 Tox" width="920" height="628" loading="lazy" style="margin-left: auto; margin-right: auto; display: block; width: 920px; height: auto; max-width: 100%;" srcset="https://www.uptycs.com/hs-fs/hubfs/Figure%206%20Tox.png?width=460&amp;height=314&amp;name=Figure%206%20Tox.png 460w, https://www.uptycs.com/hs-fs/hubfs/Figure%206%20Tox.png?width=920&amp;height=628&amp;name=Figure%206%20Tox.png 920w, https://www.uptycs.com/hs-fs/hubfs/Figure%206%20Tox.png?width=1380&amp;height=942&amp;name=Figure%206%20Tox.png 1380w, https://www.uptycs.com/hs-fs/hubfs/Figure%206%20Tox.png?width=1840&amp;height=1256&amp;name=Figure%206%20Tox.png 1840w, https://www.uptycs.com/hs-fs/hubfs/Figure%206%20Tox.png?width=2300&amp;height=1570&amp;name=Figure%206%20Tox.png 2300w, https://www.uptycs.com/hs-fs/hubfs/Figure%206%20Tox.png?width=2760&amp;height=1884&amp;name=Figure%206%20Tox.png 2760w" sizes="(max-width: 920px) 100vw, 920px"><span style="font-size: 16px;"><em>Figure 6: rest of the main function</em></span></p>
<p>&nbsp;</p>
<p>Moving on in the main function, we can see tox related functions tox_new, tox_self_set_name, and tox_self_set_status_message which are most likely used for tox setup on the victim machine.</p>
<p>&nbsp;</p>
<p>In Figure 6, tox_callback_friend_message, gets called which looks at the previous message from a friend, and decides what to do based on the message received.</p>
<p>&nbsp;</p>
<p>There are three commands that are a part of the callback function(passed as arg to tox_callback_friend_message), as shown in Figure 7.</p>
<p>&nbsp;</p>
<p>`updatekilllist` updates the script executed in start_routine1, `execscript` runs the script on-demand, `getinfo` prints information, and `exit` quits the tox connection.</p>
<p style="text-align: center;"><a href="https://www.uptycs.com/hubfs/Figure%207%20Tox.png" rel="noopener" target="_blank"><img src="https://www.uptycs.com/hs-fs/hubfs/Figure%207%20Tox.png?width=757&amp;height=700&amp;name=Figure%207%20Tox.png" alt="Figure 7 Tox" width="757" height="700" loading="lazy" style="width: 757px; margin-left: auto; margin-right: auto; display: block; height: auto; max-width: 100%;" srcset="https://www.uptycs.com/hs-fs/hubfs/Figure%207%20Tox.png?width=379&amp;height=350&amp;name=Figure%207%20Tox.png 379w, https://www.uptycs.com/hs-fs/hubfs/Figure%207%20Tox.png?width=757&amp;height=700&amp;name=Figure%207%20Tox.png 757w, https://www.uptycs.com/hs-fs/hubfs/Figure%207%20Tox.png?width=1136&amp;height=1050&amp;name=Figure%207%20Tox.png 1136w, https://www.uptycs.com/hs-fs/hubfs/Figure%207%20Tox.png?width=1514&amp;height=1400&amp;name=Figure%207%20Tox.png 1514w, https://www.uptycs.com/hs-fs/hubfs/Figure%207%20Tox.png?width=1893&amp;height=1750&amp;name=Figure%207%20Tox.png 1893w, https://www.uptycs.com/hs-fs/hubfs/Figure%207%20Tox.png?width=2271&amp;height=2100&amp;name=Figure%207%20Tox.png 2271w" sizes="(max-width: 757px) 100vw, 757px"></a><span style="font-size: 16px;"></span></p>
<p style="text-align: center;"><em><span style="font-size: 16px;">Figure 7: callback_func</span></em></p>
<p>&nbsp;</p>
<h2>Conclusion</h2>
<p>While the discussed sample does not do anything explicitly malicious, we feel that it might be a component of a coinminer campaign. We are observing it for the first time where Tox protocol is used to run scripts onto the machine.</p>
<p>&nbsp;</p>
<p>We have seen attackers using Tox as a communication mechanism in the past, like in HelloXD ransomware, where the attacker used Tox and onion-based messengers. Therefore, it becomes important to monitor the network components involved in the attack chains.</p>
<p>&nbsp;</p>
