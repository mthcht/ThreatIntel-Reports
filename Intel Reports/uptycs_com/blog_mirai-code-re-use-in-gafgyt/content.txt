            <div class="blog-post__body">
              <span id="hs_cos_wrapper_post_body" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_rich_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="rich_text"><p>Research by <a href="/blog/author/siddharth-sharma" rel="noopener" target="_blank">Siddharth Sharma</a></p>
<p>&nbsp;</p>
<p>Uptycs' threat research team recently detected several variants of the Linux-based botnet malware family, “<strong>Gafgyt</strong>”, via threat intelligence systems and our in-house osquery-based sandbox. Upon analysis, we identified several codes, techniques and implementations of Gafgyt, re-used from the infamous <a href="https://github.com/jgamblin/Mirai-Source-Code" rel="noopener" target="_blank"><span>Mirai botnet</span></a>.</p>
<p>&nbsp;</p>
<!--more-->
<p>In this blog, we’ll take a look at some of the re-used Mirai modules, their functionality, and the Uptycs <a href="/products/attack-surfaces/endpoints" rel="noopener" target="_blank">EDR</a> detection capabilities of Gafgyt.</p>
<p>&nbsp;</p>
<h2>Gafgyt</h2>
<p><a href="https://en.wikipedia.org/wiki/BASHLITE" rel="noopener" target="_blank"><span>Gafgyt</span></a> (also known as Bashlite) is a prominent malware family for *nix systems, which mainly target vulnerable IoT devices like Huawei routers, Realtek routers and ASUS devices. Gafgyt also uses some of the existing exploits (CVE-2017-17215, CVE-2018-10561) to download the next stage payloads, which we will discuss further on.</p>
<p>&nbsp;</p>
<p>Gafgyt malware variants have very similar functionality to Mirai, as a majority of the code was copied.</p>
<p>&nbsp;</p>
<h2>Technical Analysis: Gafgyt; Re-used Mirai Modules&nbsp;</h2>
<p>During our analysis of Gafgyt, we identified several recent variants that have re-used some code modules from the Mirai source code. The modules are:&nbsp;</p>
<ol>
<li><strong>HTTP Flooding</strong></li>
<li><strong>UDP Flooding</strong></li>
<li><strong>TCP Flooding</strong></li>
<li><strong>STD Module</strong></li>
<li><strong>Telnet Bruteforce</strong></li>
</ol>
<p>We will provide details of these modules and their functionality, but for the purpose of this blog we are using the hashes (<strong>da20bf020c083eb080bf75879c84f8885b11b6d3d67aa35e345ce1a3ee762444</strong> and <strong>1b3bb39a3d1eea8923ceb86528c8c38ecf9398da1bdf8b154e6b4d0d8798be49</strong>) and the Mirai leaked source code.</p>
<p>&nbsp;</p>
<h3>1. HTTP Flooding Module</h3>
<p>HTTP flooding is a kind of DDoS attack in which the attacker sends a large number of HTTP requests to the targeted server to overwhelm it. The creators of Gafgyt have re-used this code from the leaked Mirai source code.</p>
<p>&nbsp;</p>
<p style="text-align: left;">The below figure (Figure 1) shows the comparison of the Gafgyt and Mirai HTTP flooding module.&nbsp;</p>

<p>&nbsp;</p>
<p>In the above image, the left is the Gafgyt decompiled code, which matches the Mirai source code on the right.</p>
<p>&nbsp;</p>
<h3>2. UDP Flood Module</h3>
<p>UDP flooding is a type of DDoS attack in which an attacker sends several UDP packets to the victim server as a means of exhausting it. Gafgyt contained this same functionality of UDP flooding, copied from the leaked Mirai source code (see Figure 2).</p>
