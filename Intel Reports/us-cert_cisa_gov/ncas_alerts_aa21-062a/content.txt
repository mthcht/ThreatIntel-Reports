                <h1 class="c-page-title__title">
<span>Mitigate Microsoft Exchange Server Vulnerabilities</span>
</h1>
                                                          <div class="c-page-title__fields">  




<div  class="c-field c-field--name-field-last-updated c-field--type-datetime c-field--label-above">
  <div  class="c-field__label">Last Revised</div><div class="c-field__content"><time datetime="2021-07-19T12:00:00Z">July 19, 2021</time></div></div>

  




<div  class="c-field c-field--name-field-alert-code c-field--type-string c-field--label-above">
  <div  class="c-field__label">Alert Code</div><div class="c-field__content">AA21-062A</div></div>

</div>
                        
        
      </div>
          </div>
    <div class="c-page-title__decoration"></div>
  </div>

    </div>

<div  class="l-page-section l-page-section--rich-text">

  <div class="l-page-section__content">
          <div>
<h3>Summary</h3>
</div>
<p><em>Updated July 19, 2021: The U.S. Government attributes this activity to malicious cyber actors affiliated with the People's Republic&nbsp;of China (PRC) Ministry of State Security (MSS). Additional information may be found in a <a href="https://www.whitehouse.gov/briefing-room/statements-releases/2021/07/19/the-united-states-joined-by-allies-and-partners-attributes-malicious-cyber-activity-and-irresponsible-state-behavior-to-the-peoples-republic-of-china/">statement from the White House</a>. For more information on Chinese malicious cyber activity, refer to <a href="https://us-cert.cisa.gov/china">us-cert.cisa.gov/China</a>.</em></p>
<p><em><strong>Note:</strong> This Alert was updated April 13, 2021, to provide further guidance.&nbsp;</em></p>
<p>Cybersecurity and Infrastructure Security Agency (CISA) partners have observed active exploitation of vulnerabilities in Microsoft Exchange Server products. Successful exploitation of these vulnerabilities allows an unauthenticated attacker to execute arbitrary code on vulnerable Exchange Servers, enabling the attacker to gain persistent system access, as well as access to files and mailboxes on the server and to credentials stored on that system. Successful exploitation may additionally enable the attacker to compromise trust and identity in a vulnerable network. Microsoft released out-of-band patches to address vulnerabilities in Microsoft Exchange Server. The vulnerabilities impact on-premises Microsoft Exchange Servers and are not known to impact Exchange Online or Microsoft 365 (formerly O365) cloud email services.</p>
<p>This Alert includes both tactics, techniques and procedures (TTPs) and the indicators of compromise (IOCs) associated with this malicious activity. To secure against this threat, CISA recommends organizations examine their systems for the TTPs and use the IOCs to detect any malicious activity. If an organization discovers exploitation activity, they should assume network identity compromise and follow incident response procedures. If an organization finds no activity, they should apply available patches immediately and implement the mitigations in this Alert.</p>
<p><a href="/sites/default/files/publications/AA21-062A.stix.xml">Click here for IOCs in STIX format.</a></p>
<div>
<h3>Technical Details</h3>
</div>
<p><em>(Updated April 14, 2021)</em>: <a href="https://msrc.microsoft.com/update-guide/releaseNote/2021-Apr">Microsoft's April 2021 Security Update</a> newly discloses and mitigates significant vulnerabilities affecting on-premises Exchange Server 2013, 2016, and 2019.</p>
<p>Microsoft has released out-of-band security updates to address four vulnerabilities in Exchange Server:</p>
<ul>
<li><a href="https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2021-26855">CVE-2021-26855</a> allows an unauthenticated attacker to send arbitrary HTTP requests and authenticate as the Exchange Server. The vulnerability exploits the Exchange Control Panel (ECP) via a Server-Side Request Forgery (SSRF). This would also allow the attacker to gain access to mailboxes and read sensitive information.</li>
<li><a href="https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2021-26857">CVE-2021-26857</a>, <a href="https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2021-26858">CVE-2021-26858</a>, and <a href="https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2021-27065">CVE-2021-27065</a> allow for remote code execution. &nbsp;
<ul>
<li>
<p>CVE-2021-26858 and CVE-2021-27065 are similar post-authentication arbitrary write file vulnerabilities in Exchange. An attacker, authenticated either by using CVE-2021-26855 or via stolen admin credentials, could write a file to any path on the server.</p>
</li>
<li>
<p>CVE-2021-26857 is an insecure deserialization vulnerability in the Unified Messaging service. An attacker, authenticated either by using CVE-2021-26855 or via stolen admin credentials, could execute arbitrary code as <code>SYSTEM </code>on the Exchange Server.</p>
</li>
</ul>
</li>
<li>
<p>To locate a possible compromise of these CVEs, CISA encourages organizations&nbsp;read the <a href="https://www.microsoft.com/security/blog/2021/03/02/hafnium-targeting-exchange-servers/">Microsoft Advisory</a>.</p>
</li>
</ul>
<p>It is possible for an attacker, once authenticated to the Exchange server, to gain access to the Active Directory environment and download the Active Directory Database.</p>
<p><em>(Updated March 12, 2021):</em> Microsoft Security Intelligence has released a <a href="https://twitter.com/MsftSecIntel/status/1370236539427459076">tweet</a> on <a href="https://www.bleepingcomputer.com/news/security/ransomware-now-attacks-microsoft-exchange-servers-with-proxylogon-exploits/">DearCry</a> ransomware being used to exploit compromised on-premises Exchange Servers. Ransomware infections can have negative consequences to an affected organization, including:</p>
<ul>
<li>temporary or permanent loss of sensitive or proprietary information,</li>
<li>disruption to regular operations,</li>
<li>financial losses incurred to restore systems and files, and</li>
<li>potential harm to an organization’s reputation.</li>
</ul>
<p>(<em>Updated April 12, 2021</em>): CISA recommends organizations review Malware Analysis Report (MAR) <a href="https://us-cert.cisa.gov/ncas/analysis-reports/ar21-102b">MAR-10330097-1.v1 – DearCry Ransomware</a> for detailed analysis, along with TTPs and IOCs.</p>
<p><em>(Updated March 12, 2021): </em>CISA encourages organizations to review CISA’s <a href="https://www.cisa.gov/ransomware">Ransomware web page</a> for guidance and resources. Victims of ransomware should report it immediately to CISA at <a href="https://www.us-cert.gov/report">www.us-cert.gov/report</a>, a local<a href="https://www.fbi.gov/contact-us/field-offices"> FBI Field Office</a>, or <a href="http://www.secretservice.gov/contact/field-offices/">Secret Service Field Office</a>.</p>
<h3>Tactics, Techniques and Procedures</h3>
<p><em>(Updated March 10, 2021):</em> Microsoft has released a script that scans Exchange log files for IOCs. CISA strongly encourages organizations to run the <a href="https://github.com/microsoft/CSS-Exchange/tree/main/Security">Test-ProxyLogon.ps1 script</a>—as soon as possible—to help determine whether their systems are compromised.</p>
<p><em>(Updated March 16, 2021): </em><strong>Note:</strong> Microsoft has released the <a href="https://msrc-blog.microsoft.com/2021/03/15/one-click-microsoft-exchange-on-premises-mitigation-tool-march-2021/">Exchange On-premises Mitigation Tool (EOMT.ps1)</a> that can automate portions of both the detection and patching process. Microsoft stated the following along with the release: "[the tool is intended] to help customers who do not have dedicated security or IT teams to apply these security updates. We have tested this tool across Exchange Server 2013, 2016, and 2019 deployments. This new tool is designed as an interim mitigation for customers who are unfamiliar with the patch/update process or who have not yet applied the on-premises Exchange security update.” Review the <a href="https://msrc-blog.microsoft.com/2021/03/15/one-click-microsoft-exchange-on-premises-mitigation-tool-march-2021/">EOMT.ps1 blog post</a> for directions on using the tool.</p>
<p><em>(Updated March 10, 2021):</em> CISA recommends investigating for signs of a compromise from at least January 1, 2021&nbsp;through present.</p>
<p><em>(Updated April 12, 2021): </em>CISA has identified 10&nbsp;webshells associated with this activity. This is not an all-inclusive list of webshells that are being leveraged by actors. CISA recommends organizations review the following MARs&nbsp;for detailed analysis of the 10&nbsp;webshells, along with TTPs and IOCs. These MARs include&nbsp;CISA-developed YARA rules to help network defenders detect associated malware.</p>
<ol>
<li>AR21-072A: <a href="https://us-cert.cisa.gov/ncas/analysis-reports/ar21-072a">MAR-10328877.r1.v1: China Chopper Webshell</a></li>
<li>AR21-072B: <a href="https://us-cert.cisa.gov/ncas/analysis-reports/ar21-072b">MAR-10328923.r1.v1: China Chopper Webshell</a></li>
<li>AR21-072C: <a href="https://us-cert.cisa.gov/ncas/analysis-reports/ar21-072c">MAR-10329107.r1.v1: China Chopper Webshell</a></li>
<li>AR21-072D: <a href="https://us-cert.cisa.gov/ncas/analysis-reports/ar21-072d">MAR-10329297.r1.v1: China Chopper Webshell</a></li>
<li>AR21-072E: <a href="https://us-cert.cisa.gov/ncas/analysis-reports/ar21-072e">MAR-10329298.r1.v1: China Chopper Webshell</a></li>
<li>AR21-072F: <a href="https://us-cert.cisa.gov/ncas/analysis-reports/ar21-072f">MAR-10329301.r1.v1: China Chopper Webshell</a></li>
<li>AR21-072G: <a href="https://us-cert.cisa.gov/ncas/analysis-reports/ar21-072g">MAR-10329494.r1.v1: China Chopper Webshell</a></li>
<li>AR21-084A: <a href="https://us-cert.cisa.gov/ncas/analysis-reports/ar21-084a">MAR-10329496-1.v1: China Chopper Webshell</a></li>
<li>AR21-084B: <a href="https://us-cert.cisa.gov/ncas/analysis-reports/ar21-084b">MAR-10329499-1.v1: China Chopper Webshell </a></li>
<li>AR21-102A: <a href="https://us-cert.cisa.gov/ncas/analysis-reports/ar21-102a">MAR-10331466-1.v1: China Chopper Webshell</a></li>
</ol>
<p><em>(Updated March 13, 2021):</em> A webshell is a script that can be uploaded to a compromised Microsoft Exchange Server to enable remote administration of the machine. Webshells are utilized for the following purposes:</p>
<ul>
<li>To harvest and exfiltrate sensitive data and credentials;</li>
<li>To upload additional malware for the potential of creating, for example, a watering hole for infection and scanning of further victims;</li>
<li>To use as a relay point to issue commands to hosts inside the network without direct internet access;</li>
<li>To use as command-and-control infrastructure, potentially in the form of a bot in a botnet or in support of compromises to additional external networks. This could occur if the adversary intends to maintain long-term persistence.</li>
</ul>
<p><em>(Updated March 13, 2021): </em>For more information, see <a href="https://us-cert.cisa.gov/ncas/alerts/TA15-314A">TA15-314A Compromised Web Servers and Web Shells - Threat Awareness and Guidance</a>.</p>
<p>The majority of the TTPs in this section are sourced from a <a href="https://www.volexity.com/blog/2021/03/02/active-exploitation-of-microsoft-exchange-zero-day-vulnerabilities/">blog post from Volexity</a>, a third-party cybersecurity firm. <strong>Note: </strong>the United States Government does not endorse any commercial product or service, including any subjects of analysis. Any reference to specific commercial products, processes, or services by service mark, trademark, manufacturer, or otherwise, does not constitute or imply their endorsement, recommendation, or favoring by the United States Government.</p>
<p>Volexity has observed the following files as targets of <code>HTTP POST</code> requests:</p>
<ul>
<li><code>/owa/auth/Current/themes/resources/logon.css</code></li>
<li><code>/owa/auth/Current/themes/resources/owafont_ja.css</code></li>
<li><code>/owa/auth/Current/themes/resources/lgnbotl.gif</code></li>
<li><code>/owa/auth/Current/themes/resources/owafont_ko.css</code></li>
<li><code>/owa/auth/Current/themes/resources/SegoeUI-SemiBold.eot</code></li>
<li><code>/owa/auth/Current/themes/resources/SegoeUI-SemiLight.ttf</code></li>
<li><code>/owa/auth/Current/themes/resources/lgnbotl.gif</code></li>
</ul>
<p>Administrators should search the ECP server logs for the following string (or something similar):</p>
<p><code>S:CMD=Set-OabVirtualDirectory.ExternalUrl='</code></p>
<p>The logs can be found at <code>&lt;exchange install path&gt;\Logging\ECP\Server\</code>.</p>
<p>To determine possible webshell activity, administrators should search for <code>aspx</code> files in the following paths:</p>
<ul>
<li><code>\inetpub\wwwroot\aspnet_client\ </code>(any <code>.aspx</code> file under this folder or sub folders)</li>
<li><code>\&lt;exchange install path&gt;\FrontEnd\HttpProxy\ecp\auth\ </code>(any file besides <code>TimeoutLogoff.aspx</code>)</li>
<li><code>\&lt;exchange install path&gt;\FrontEnd\HttpProxy\owa\auth\ </code>(any file or modified file that is not part of a standard install)</li>
<li><code>\&lt;exchange install path&gt;\FrontEnd\HttpProxy\owa\auth\Current\ </code>(any <code>aspx </code>file in this folder or subfolders)</li>

<h3>Conduct Forensic Analysis</h3>
<p>Should your organization see evidence of compromise, your incident response should begin with conducting forensic analysis to collect artifacts and perform triage. Please see the following list of recommendations on how to conduct forensic analysis using various tools.</p>
<p>Although the following free tools are not endorsed by the Federal Government, incident responders commonly use them to perform forensics.</p>
<p>While collecting artifacts to perform triage, use processes and tools that minimize the alteration of the data being collected and that minimize impact to the operating system itself.</p>
<p>Ideally, during data collection, store the data on removable/external media and, when possible, run the artifact collection tools from the same media.</p>
<p>Key artifacts for triage that should be collected:</p>
<ul>
<li>Memory</li>
<li>All registry hives</li>
<li>All windows event logs</li>
<li>All web logs</li>
</ul>
<p>Memory can be collected with a variety of open source tools (e.g., FTK Imager by AccessData, Ram Capture by Belkasoft).</p>
<p>Registry and Windows Event logs can be collected with a variety of open source tools as well (e.g., FTK_Imager, Kroll Artifact Parser And Extractor [KAPE]).</p>
<p>Web logs can also be collected with a variety of open source tools (e.g., FTK Imager).</p>
<h4><strong>Windows Artifact Collection Guide</strong></h4>
<p>Execute the following steps in order.</p>
<p><strong>1) Download the latest FTK Imager</strong> from&nbsp;<a href="https://accessdata.com/product-download/">https://accessdata.com/product-download/</a>.</p>
<ul>
<li><strong>Note:</strong> Ensure your review of and compliance with the applicable license associated with the product referenced, which can be found in the product’s User Guide. The United States Government does not endorse any commercial product or service, including any subjects of analysis. Any reference to specific commercial products, processes, or services by service mark, trademark, manufacturer, or otherwise, does not constitute or imply their endorsement, recommendation, or favoring by the United States Government.</li>
</ul>
<p><strong>2) Collect memory from live system using FTK Imager.</strong> See Memory Capture with FTK Imager.pdf for instructions. Note: Download and copy “FTK Imager” folder to an external drive. Run FTK Imager.exe from the FTK Imager folder from external drive. Wait until memory collect is complete before proceeding to step 2.</p>
<p><strong>3) Collect important system artifacts using KAPE.</strong> See KAPE Collection Procedure. Note: Download KAPE from a separate system; do not download KAPE to the target system. Run KAPE from external drive.</p>
<p><strong>4) Collect disk image using FTK Imager. </strong>See Live Image with FTK Imager.pdf for instructions. <strong>Note:</strong> Run FTK Imager.exe from the “FTK Imager” folder from external drive.</p>

<h3>Mitigations</h3>
</div>
<p>CISA strongly recommends organizations read <a href="https://msrc-blog.microsoft.com/2021/03/02/multiple-security-updates-released-for-exchange-server/">Microsoft’s advisory</a> and <a href="https://www.microsoft.com/security/blog/2021/03/02/hafnium-targeting-exchange-servers/">security blog post</a> for more information on how to look for this malicious activity and to apply critical patches as soon as possible.</p>
<p><em>(Updated March 4, 2021):</em> CISA is aware of threat actors using open source tools to search for vulnerable Microsoft Exchange Servers. This particular type of attack is scriptable, allowing attackers to easily exploit vulnerabilities through automated mechanisms. CISA advises all entities to patch as soon as possible to avoid being compromised. &nbsp;</p>
<p><em>(Updated March 4, 2021):</em> From <a href="https://techcommunity.microsoft.com/t5/exchange-team-blog/released-march-2021-exchange-server-security-updates/ba-p/2175901">Microsoft's patch release</a>, the security updates are available for the following operating systems:</p>
<ul>
<li>Exchange Server 2010 (update requires SP 3 or any SP 3 RU – this is a Defense in Depth update)</li>
<li>Exchange Server 2013 (update requires CU 23)</li>
<li>Exchange Server 2016 (update requires CU 19 or CU 18)</li>
<li>Exchange Server 2019 (update requires CU 8 or CU 7)</li>
</ul>
<p><em>(Updated March 4, 2021):</em> If you are running an older CU then what the patch will accept, you must upgrade to at least the required CU as stated above then apply the patch.&nbsp;</p>
<p><em>(Updated March 4, 2021):</em> All patches must be applied using administrator privileges.<br><br>
&nbsp;</p>
<p><em>(Updated March 5, 2021)</em>: If patching is not an immediate option, CISA strongly recommends following alternative mitigations found in <a href="https://msrc-blog.microsoft.com/2021/03/05/microsoft-exchange-server-vulnerabilities-mitigations-march-2021/">Microsoft’s blog on Exchange Server Vulnerabilities Mitigations</a>. However, these options should only be used as a temporary solution, not a replacement for patching. Additionally, there are other mitigation options available. CISA recommends limiting or blocking external access to internet-facing Exchange Servers via the following:</p>
<ul>
<li>Restrict untrusted connections to port 443, or set up a VPN to separate the Exchange Server from external access; note that this will not prevent an adversary from exploiting the vulnerability if the attacker is already in your network.</li>
<li>Block external access to on-premises Exchange:
<ul>
<li>Restrict external access to OWA URL: <code>/owa/</code>.&nbsp;</li>
<li>
<p>Restrict external access to Exchange Admin Center (EAC) aka Exchange Control Panel (ECP) URL:<code>&nbsp;/ecp/</code>.</p>
</li>
</ul>
</li>
<li>
<p><em>(Updated March 4, 2021):</em> Disconnect vulnerable Exchange servers from the internet until a patch can be applied.</p>
</li>
</ul>
