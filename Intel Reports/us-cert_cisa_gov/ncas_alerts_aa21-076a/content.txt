                <h1 class="c-page-title__title">
<span>TrickBot Malware</span>
</h1>
                                                          <div class="c-page-title__fields">  




<div  class="c-field c-field--name-field-last-updated c-field--type-datetime c-field--label-above">
  <div  class="c-field__label">Last Revised</div><div class="c-field__content"><time datetime="2021-05-20T12:00:00Z">May 20, 2021</time></div></div>

  




<div  class="c-field c-field--name-field-alert-code c-field--type-string c-field--label-above">

          <div>
<h3>Summary</h3>
</div>
<p><em>This Joint Cybersecurity Advisory uses the MITRE Adversarial Tactics, Techniques, and Common Knowledge (ATT&amp;CK®) framework, Version 8. See the <a href="https://attack.mitre.org/versions/v8/techniques/enterprise/">ATT&amp;CK for Enterprise</a> for all referenced threat actor tactics and techniques.</em></p>
<p>The Cybersecurity and Infrastructure Security Agency (CISA) and Federal Bureau of Investigation (FBI) have observed continued targeting through spearphishing campaigns using TrickBot malware&nbsp;in North America. A sophisticated group of cybercrime actors is luring victims, via phishing emails, with a traffic infringement phishing scheme to download TrickBot.</p>
<p>TrickBot—first identified in 2016—is a Trojan developed and operated by a sophisticated group of cybercrime actors. Originally designed as a banking Trojan to steal financial data, TrickBot has evolved into highly modular, multi-stage malware that provides its operators a full suite of tools to conduct a myriad of illegal cyber activities.</p>
<p>To secure against TrickBot, CISA and FBI recommend implementing the mitigation measures described in this Joint Cybersecurity Advisory, which include blocking suspicious Internet Protocol addresses, using antivirus software, and providing social engineering and phishing training to employees.</p>
<p><a href="/sites/default/files/publications/AA21-076A-TrickBot_Malware_508_0.pdf">Click here</a> for a PDF version of this report.</p>
<div>
<h3>Technical Details</h3>
</div>
<p>TrickBot is an advanced Trojan that malicious actors spread primarily by spearphishing campaigns using tailored emails that contain malicious attachments or links, which—if enabled—execute malware (<em>Phishing:</em> <em>Spearphishing Attachment </em>[<a href="https://attack.mitre.org/versions/v8/techniques/T1566/001/">T1566.001</a>], <em>Phishing: Spearphishing Link</em> [<a href="https://attack.mitre.org/versions/v8/techniques/T1566/002">T1566.002</a>]). CISA and FBI are aware of recent attacks that use phishing emails, claiming to contain proof of a traffic violation, to steal sensitive information. The phishing emails contain links that redirect to a website hosted on a compromised server that prompts the victim to click on photo proof of their traffic violation. (<em>User Execution: Malicious Link</em> [<a href="https://attack.mitre.org/versions/v8/techniques/T1204/001/">T1204.001</a>], <em>User Execution: Malicious File</em> [<a href="https://attack.mitre.org/versions/v8/techniques/T1204/002/">T1204.002</a>]). In clicking the photo, the victim unknowingly downloads a malicious JavaScript file that, when opened, automatically communicates with the malicious actor’s command and control (C2) server to download TrickBot to the victim’s system (<em>Command and Scripting Interpreter: JavaScript</em> [<a href="https://attack.mitre.org/versions/v8/techniques/T1059/007/">T1059.007</a>]).</p>
<p>Attackers can use TrickBot to:</p>
<ul>
<li>Drop other malware, such as Ryuk and Conti ransomware, or</li>
<li>Serve as an Emotet downloader (<em>Ingress Tool Transfer</em> [<a href="https://attack.mitre.org/versions/v8/techniques/T1105/">T1105</a>]).[<a href="https://www.fireeye.com/blog/threat-research/2019/01/a-nasty-trick-from-credential-theft-malware-to-business-disruption.html">1</a>]</li>
</ul>
<p>TrickBot uses person-in-the-browser attacks to steal information, such as login credentials (<em>Man in the Browser</em> [<a href="https://attack.mitre.org/versions/v8/techniques/T1185/">T1185</a>]). Additionally, some of TrickBot’s modules spread the malware laterally across a network by abusing the Server Message Block (SMB) Protocol. TrickBot operators have a toolset capable of spanning the entirety of the MITRE ATT&amp;CK framework, from actively or passively gathering information that can be used to support targeting (<em>Reconnaissance </em>[<a href="https://attack.mitre.org/tactics/TA0043/">TA0043</a>]), to trying to manipulate, interrupt, or destroy systems and data (<em>Impact </em>[<a href="https://attack.mitre.org/tactics/TA0040/">TA0040</a>]).</p>
<p>TrickBot is capable of data exfiltration over a hardcoded C2 server, cryptomining, and host enumeration (e.g., reconnaissance of Unified Extensible Firmware Interface or Basic Input/Output System [UEFI/BIOS] firmware) (<em>Exfiltration Over C2 Channel </em>[<a href="https://attack.mitre.org/versions/v8/techniques/T1041/">T1041</a>], <em>Resource Hijacking</em> [<a href="https://attack.mitre.org/versions/v8/techniques/T1496">T1496</a>], System Information Discovery.[<a href="https://eclypsium.com/2020/12/03/trickbot-now-offers-trickboot-persist-brick-profit/#background">2</a>] For host enumeration, operators deliver TrickBot in modules containing a configuration file with specific tasks.</p>
<p>Figure 1 lays out TrickBot’s use of enterprise techniques.</p>
<p><img class="align-center"></p>
<p><em>Figure 1: MITRE ATT&amp;CK enterprise techniques used by TrickBot </em></p>

<h3>Mitigations</h3>
</div>
<p>CISA and FBI recommend that network defenders—in federal, state, local, tribal, territorial governments, and the private sector—consider applying the following best practices to strengthen the security posture of their organization's systems. System owners and administrators should review any configuration changes prior to implementation to avoid negative impacts.</p>
<ul>
<li>Provide social engineering and phishing training to employees.</li>
<li>Consider drafting or updating a policy addressing suspicious emails&nbsp; that specifies users must report all suspicious emails to the security and/or IT departments.</li>
<li>Mark external emails with a banner denoting the email is from an external source to assist users in detecting spoofed emails.</li>
<li>Implement Group Policy Object and firewall rules.</li>
<li>Implement an antivirus program and a formalized patch management process.</li>
<li>Implement filters at the email gateway and block suspicious IP addresses at the firewall.</li>
<li>Adhere to the principle of least privilege.</li>
<li>Implement a Domain-Based Message Authentication, Reporting &amp; Conformance validation system.</li>
<li>Segment and segregate networks and functions.</li>
<li>Limit unnecessary lateral communications between network hoses, segments, and devices.</li>
<li>Consider using application allowlisting technology on all assets to ensure that only authorized software executes, and all unauthorized software is blocked from executing on assets. Ensure that such technology only allows authorized, digitally signed scripts to run on a system.</li>
<li>Enforce multi-factor authentication.</li>
<li>Enable a firewall on agency workstations configured to deny unsolicited connection requests.</li>
<li>Disable unnecessary services on agency workstations and servers.</li>
<li>Implement an Intrusion Detection System, if not already used, to detect C2 activity and other potentially malicious network activity</li>
<li>Monitor web traffic. Restrict user access to suspicious or risky sites.</li>
<li>Maintain situational awareness of the latest threats and implement appropriate access control lists.</li>
<li>Disable the use of SMBv1 across the network and require at least SMBv2 to harden systems against network propagation modules used by TrickBot.</li>
<li>Visit the MITRE ATT&amp;CK Techniques pages (linked in table 1 above) for additional mitigation and detection strategies.</li>
<li>See CISA’s Alert on <a href="https://us-cert.cisa.gov/ncas/alerts/aa20-245a">Technical Approaches to Uncovering and Remediating Malicious Activity</a> for more information on addressing potential incidents and applying best practice incident response procedures.</li>
</ul>
<p>For additional information on malware incident prevention and handling, see the National Institute of Standards and Technology Special Publication 800-83, <a href="https://csrc.nist.gov/publications/detail/sp/800-83/rev-1/final">Guide to Malware Incident Prevention and Handling for Desktops and Laptops</a>.</p>
