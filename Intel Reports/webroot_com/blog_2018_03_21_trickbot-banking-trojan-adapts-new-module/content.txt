			</div><p>Since inception in late 2016, the TrickBot banking trojan has continually undergone updates and changes in attempts to stay one step ahead of defenders and <a href="https://www.webroot.com/us/en/home/products/isp">internet security providers</a>. While TrickBot has not always been the stealthiest trojan, its authors have remained consistent in the use of new distribution vectors and development of new features for their product. On March 15, 2018, Webroot observed a module (tabDll32 / tabDll64) being downloaded by TrickBot that has not been seen in the wild before this time.</p>
<p>It appears that the TrickBot authors are still attempting to leverage MS17-010 and other lateral movement methods coupled with this module in an attempt to create a new monetization scheme for the group.</p>
<h2>You can teach an old bot older tricks</h2>

<h3>Behavioral Analysis</h3>
<p>Upon execution, this iteration of TrickBot will install itself into the %APPDATA%\TeamViewer\ directory. If the bot has not been executed from its installation directory, it will restart itself from this directory and continue operation. Once running from its installation directory, TrickBot will write to the usual group_tag and client_id files along with creating a &#8220;Modules&#8221; folder used to store the encrypted plug and play modules and configuration files for the bot.</p>
<p><img decoding="async" class="alignnone size-full wp-image-24084" src="https://blog-en.webroot.com/wp-content/uploads/2018/03/Trickbot-image1.png" alt="" width="979" height="504" /><br />
<em>Image 1: TrickBot’s plug and play modules used to extend the bots functionality</em></p>
<p>Many of the modules shown above have been previously documented. The systeminfo and injectDll module have been coupled with the bot since its inception. The <a href="https://devcentral.f5.com/articles/is-xmaker-the-new-trickloader-24372)">mailsearcher module</a> was added in December 2016 and the <a href="https://www.flashpoint-intel.com/blog/new-version-trickbot-adds-worm-propagation-module/">worm module</a> was discovered in late July 2017. The module of interest here is tabDll32 as this module has been previously undocumented. Internally, the module is named spreader_x86.dll and exports four functions similar to the other TrickBot modules.</p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-24085" src="https://blog-en.webroot.com/wp-content/uploads/2018/03/Trickbot-image2.png" alt="" width="777" height="595" /><br />
<em>Image 2a: Peering inside tabDll.dll</em></p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-24086" src="https://blog-en.webroot.com/wp-content/uploads/2018/03/Trickbot-image2b.png" alt="" width="743" height="258" /><br />
<em>Image 2b: Abnormally large .rdata section</em></p>
<p>The file has an abnormally large rdata section which proves to be quite interesting because it contains two additional files intended to be used by spreader_x86.dll. The spreader module contains an additional executable SsExecutor_x86.exe and an additional module screenLocker_x86.dll. Each module will be described in more detail in its respective section below.</p>
<p><strong>Spreader_x86.dll</strong></p>
<p>When loading the new TrickBot module in IDA, you are presented with the option of loading the debug symbol filename.</p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-24087" src="https://blog-en.webroot.com/wp-content/uploads/2018/03/Trickbot-image3.png" alt="" width="720" height="70" /><br />
<em>Image 3: Debug symbol filename of the downloaded module tabDll.dll</em></p>
<p>This gives us a preview of how the TrickBot developers structure new modules that are currently under development. When digging deeper into the module, it becomes evident that this module is used to spread laterally through an infected network making use of MS17-010.</p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-24088" src="https://blog-en.webroot.com/wp-content/uploads/2018/03/Trickbot-image4.png" alt="" width="1118" height="220" /><em>Image 4: String references to EternalRomance exploit used for lateral movement</em></p>
<p>This module appears to make use of lateral movement in an attempt to set up the embedded executable as a service on the exploited system. Additionally, the TrickBot authors appear to be still developing this module as parts of the modules reflective dll injection mechanism are stolen from GitHub.</p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-24089" src="https://blog-en.webroot.com/wp-content/uploads/2018/03/Trickbot-image5.png" alt="" width="777" height="441" /><br />
<em>Image 5: Copied code from <a href="https://github.com/dismantl/ImprovedReflectiveDLLInjection/blob/master/inject/src/LoadLibraryR.c">ImprovedReflectiveDLLInjection</a></em></p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-24090" src="https://blog-en.webroot.com/wp-content/uploads/2018/03/Trickbot-image6.png" alt="" width="777" height="408" /><br />
<em>Image 6: Printf statements from the <a href="https://github.com/dismantl/ImprovedReflectiveDLLInjection/blob/master/inject/src/LoadLibraryR.c">copied project on GitHub</a></em></p>
<p><strong>SsExecutor_x86.exe</strong><em> </em></p>
<p>The second phase of the new module comes in the form of an executable meant to run after post exploitation. Again, it was very nice of the TrickBot authors to give us a look at the debug symbols file path.</p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-24091" src="https://blog-en.webroot.com/wp-content/uploads/2018/03/Trickbot-image7.png" alt="" width="722" height="70" /><br />
<em>Image 7: Debug symbol filename of the embedded PE file.</em></p>
<p>When run, this executable will iterate over the use profiles in registry and goes to each profile to add a link to the copied binary to the start up path. This occurs after lateral movement takes place.</p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-24092" src="https://blog-en.webroot.com/wp-content/uploads/2018/03/Trickbot-image8.png" alt="" width="750" height="712" />                        <em>Image 8: Iterate over user profiles and create</em></p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-24093" src="https://blog-en.webroot.com/wp-content/uploads/2018/03/Trickbot-image9.png" alt="" width="750" height="337" /><br />
<em>Image 9: Execution of the copied binary</em></p>
<p><strong>ScreenLocker_x86.dll</strong></p>
<p>Similarly, to the other TrickBot modules, this module was written in Delphi. This is the first time TrickBot has shown any attempt at &#8220;locking&#8221; the victims machine.</p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-24094" src="https://blog-en.webroot.com/wp-content/uploads/2018/03/Trickbot-image10.png" alt="" width="691" height="487" /><br />
<em>Image 10: Peering inside screenLocker_x86.dll </em></p>
<p>This Module exports two functions, &#8220;MyFunction&#8221; and a reflective DLL loading function. &#8220;MyFunction&#8221; appears to be the work in progress:</p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-24095" src="https://blog-en.webroot.com/wp-content/uploads/2018/03/Trickbot-image11.png" alt="" width="635" height="841" /><br />
<em>Image 11: Peering inside &#8220;MyFunction&#8221;</em></p>
<p><img decoding="async" loading="lazy" class="alignnone size-full wp-image-24096" src="https://blog-en.webroot.com/wp-content/uploads/2018/03/Trickbot-image12.png" alt="" width="643" height="556" /><br />
<em>Image 12: Creation of the Locker Window</em></p>
<p>If the TrickBot developers are attempting to complete this locking functionality, this generates interesting speculation around the group&#8217;s business model. Locking a victim&#8217;s computer before you are able to steal their banking credentials alerts the victim that they are infected, thus limiting the potential for credit card or bank theft. However, extorting victims to unlock their computer is a much simpler monetization scheme.</p>
<p>It is notable that this locking functionality is only deployed after lateral movement, meaning that it would be used to primarily target unpatched corporate networks. In a corporate setting (with unpatched machines) it is highly likely that backups would not exist as well. The authors appear to be getting to know their target audience and how to best extract money from them. On a corporate network, where users are unlikely to be regularly visiting targeted banking URLs, exfiltrating banking credentials is a less successful money-making model compared to the locking of potentially hundreds of machines.<strong> </strong></p>
<p>The TrickBot authors continue to target various financial institutions across the world, using MS17-010 exploits in an attempt to successfully laterally move throughout a victim&#8217;s network. This is being coupled with an unfinished &#8220;screenLocker&#8221; module in a new possible attempt to extort money from victims. The TrickBot banking trojan remains under continual development and testing in a constant effort by its developers to stay one step ahead of <a href="https://www.webroot.com/us/en/business">cybersecurity</a> professionals.</p>
