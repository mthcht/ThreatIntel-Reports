    <div class="article-tags mb-2 dark big"><p class="article-tag text-capitalize">ESET Research</p></div>    <h1 class="page-headline">Janeleiro, the time traveler: A new old banking trojan in Brazil</h1>    <p class="sub-title">ESET Research uncovers a new threat that targets organizations operating in various sectors in Brazil</p>
    
    <div class="article-authors d-flex flex-wrap"><div class="article-author d-flex"><a href="/en/our-experts/facundo-munoz/" title="Facundo Mu√±oz"><picture><source srcset="https://web-assets.esetstatic.com/tn/-x45/wls/2021/02/facundo-munoz.jpeg" media="(max-width: 768px)" /><img class="author-image me-3"  src="https://web-assets.esetstatic.com/tn/-x45/wls/2021/02/facundo-munoz.jpeg" alt="Facundo Mu√±oz"  /></picture></a><div class="author-text"><p><a href="/en/our-experts/facundo-munoz/" title="Facundo Mu√±oz"><b>Facundo Mu√±oz</b></a></p></div></div><div class="article-author d-flex"><a href="/en/our-experts/matias-porolli/" title="Mat√≠as Porolli"><picture><source srcset="https://web-assets.esetstatic.com/tn/-x45/wls/2023/07/matias-porolli-eset.jpeg" media="(max-width: 768px)" /><img class="author-image me-3"  src="https://web-assets.esetstatic.com/tn/-x45/wls/2023/07/matias-porolli-eset.jpeg" alt="Mat√≠as Porolli"  /></picture></a><div class="author-text"><p><a href="/en/our-experts/matias-porolli/" title="Mat√≠as Porolli"><b>Mat√≠as Porolli</b></a></p></div></div></div>

                    <div class="article-body">
    <span class="update-block"><p><em>UPDATE (April 6<sup>th</sup>, 2021):<br />
</em>
<br /><em>Although we have not received any official response from GitHub, when we checked April 6<sup>th</sup> at around 18:00 UTC, the malicious repositories used by Janeleiro had been taken down.</em> </p></span><p>ESET Research has been tracking a newly discovered banking trojan that has been targeting corporate users in Brazil since 2019 across many verticals affecting sectors such as engineering, healthcare, retail, manufacturing, finance, transportation, and government.</p>
<p>This new threat, which we've named Janeleiro, attempts to deceive its victims with pop-up windows designed to look like the websites of some of the biggest banks in Brazil. These pop-ups contain fake forms, aiming to trick the malware‚Äôs victims into entering their banking credentials and personal information that the malware captures and exfiltrates to its C&amp;C servers. Janeleiro follows exactly the same blueprint for the core implementation of this technique as some of the most prominent malware families targeting the region: <a href="https://www.welivesecurity.com/2019/10/03/casbaneiro-trojan-dangerous-cooking/" target="_blank" rel="noopener">Casbaneiro</a>, <a href="https://www.welivesecurity.com/2020/04/28/grandoreiro-how-engorged-can-exe-get/" target="_blank" rel="noopener">Grandoreiro</a>, <a href="https://www.welivesecurity.com/2020/08/13/mekotio-these-arent-the-security-updates-youre-looking-for/" target="_blank" rel="noopener">Mekotio</a>, <a href="https://www.welivesecurity.com/2019/08/01/banking-trojans-amavaldo/" target="_blank" rel="noopener">Amavaldo</a>, and <a href="https://www.welivesecurity.com/2021/01/21/vadokrist-wolf-sheeps-clothing/" target="_blank" rel="noopener">Vadokrist</a>, among others.</p>
<p>In contrast to those well-known malware families, Janeleiro is written in Visual Basic .NET, a big deviation from the favored Delphi programming language that threat actors in the region have been using for years. Janeleiro has been evolving towards the objective of giving more control to the operators to manipulate and adjust its fake pop-up windows based on what they need to pull off the attack, send mouse clicks and keystrokes, and recording user input and the screen in real time. The nature of these types of attack is not characterized by their automation capabilities, but rather by the hands-on approach: in many cases the operator must adjust the windows via commands in real time.</p>
<p>The operators seem comfortable using GitHub to store their modules, administering their organization page, and uploading new repositories every day where they store the files with the lists of C&amp;C servers that the trojans retrieve to connect to their operators. Having your malware depend on a single source is an interesting move - but what if we told you that the newest version of Janeleiro only lives for one day?</p>
<h2>Target: Brazil</h2>
<p>Based on our telemetry data, we can affirm that this malware targets only corporate users. Malicious emails are sent to companies in Brazil and, even though we do not think these are targeted attacks, they seem to be sent in small batches. According to our telemetry, the affected sectors are engineering, healthcare, retail, manufacturing, finance, transportation and government.</p>
<p>An example of a phishing email is shown in Figure 1: a false notification regarding an unpaid invoice. It contains a link that leads to a compromised server. The retrieved page simply redirects to the download of a ZIP archive hosted in Azure. Some other emails sent by these attackers don‚Äôt have a redirection via a compromised server but lead directly to the ZIP archive.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2021/04/Figure-1.-Example-malicious-email.png"><img class="wp-image-149320" src="https://web-assets.esetstatic.com/wls/2021/04/Figure-1.-Example-malicious-email.png" alt="" width="800" height="585" /></a><p class="caption-text"><em>Figure 1. Example of a malicious email</em></p></div></p>
<p>The servers that host these ZIP archives with Janeleiro have URLs that follow the same convention as other URLs that we saw delivering other banking trojan families (see the <em>Indicators of Compromise</em> section). In some cases, these URLs have distributed both Janeleiro and other Delphi bankers at different times. This suggests that either the various criminal groups share the same provider for sending spam emails and for hosting their malware, or that they are the same group. We have not yet determined which hypothesis is correct.</p>
<p>An overview of the attack is shown in Figure 2.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2021/04/Figure-2.-Janeleiro-attack-overview-simplified-1.png"><img class="wp-image-149475" src="https://web-assets.esetstatic.com/wls/2021/04/Figure-2.-Janeleiro-attack-overview-simplified-1.png" alt="" width="800" height="524" /></a><p class="caption-text"><em>Figure 2. Janeleiro attack overview (simplified)</em></p></div></p>
<p>The ZIP archive contains an MSI installer that loads the main trojan DLL. Using an MSI installer is a favored technique of several malware families in the region. Janeleiro retrieves the computer‚Äôs public IP address and uses a web service to attempt to geolocate it. If the returned country code value does not match <span style="font-family: courier new, courier, monospace;">BR</span>, the malware exits. If the geolocation check passes, Janeleiro gathers information of the compromised machine, including:</p>
<ul>
<li>Current date and time</li>
<li>Machine name and username</li>
<li>OS full name and architecture</li>
<li>Malware version</li>
<li>Region name obtained when geolocating the computer</li>
</ul>
<p>The information is uploaded to a website with the purpose of tracking successful attacks. After that, Janeleiro retrieves the IP addresses of the C&amp;C servers from a GitHub organization page apparently created by the criminals. Then it is ready to start its core functionality and wait for commands from an operator.</p>
<p>In 2020 ESET published a <a href="https://www.welivesecurity.com/2020/10/01/latam-financial-cybercrime-competitors-crime-sharing-ttps/" target="_blank" rel="noopener">white paper</a> detailing findings about interconnectivity of the most prominent Latin American families of banking trojans including <a href="https://www.welivesecurity.com/2019/10/03/casbaneiro-trojan-dangerous-cooking/" target="_blank" rel="noopener">Casbaneiro</a>, <a href="https://www.welivesecurity.com/2020/04/28/grandoreiro-how-engorged-can-exe-get/" target="_blank" rel="noopener">Grandoreiro</a>, <a href="https://www.welivesecurity.com/2019/08/01/banking-trojans-amavaldo/" target="_blank" rel="noopener">Amavaldo</a> among others.¬† The similarities described in that paper are in the implementation of the trojan‚Äôs core: notifying the operator when there is an active window with an interesting name or title based on a predefined keyword list, and using a fake pop-up window to trick potential victims into thinking they are entering sensitive information on a legitimate website. This process is illustrated by the flowchart in Figure 3.</p>

<h2><span lang="EN-US">Janeleiro in action</span></h2>
<p>Janeleiro begins enumerating windows and checking their titles to find interesting keywords (as shown in Figure¬†5) that would indicate that the user is visiting the website of a banking entity of interest, especially those that are supported by its implementation of fake pop-up windows.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2021/04/Figure-5.-List-of-keywords-that-Janeleiro-searches-for-in-window-titles.png"><img class="wp-image-149324 size-full" src="https://web-assets.esetstatic.com/wls/2021/04/Figure-5.-List-of-keywords-that-Janeleiro-searches-for-in-window-titles.png" alt="" width="354" height="483" /></a><p class="caption-text"><em>Figure 5. List of keywords that Janeleiro searches for in window titles</em></p></div></p>
<p>When one of the keywords is found, Janeleiro immediately attempts to retrieve the addresses of its C&amp;C servers from GitHub and connects to them. These fake pop-up windows are dynamically created on demand and controlled by the attacker via commands to the malware, as they go through several stages to trick the user while the attacker, in real time, receives screen captures, the logged keystrokes and information that is entered in the fake forms.</p>
<p>The fact that threat actors abuse GitHub is nothing new; however, Janeleiro does it in quite interesting ways: the operators have created a GitHub organization page that they rename every day in the form <span style="font-family: courier new, courier, monospace;">SLK&lt;dd/mm/yyyy&gt;</span> where <span style="font-family: courier new, courier, monospace;">&lt;dd/mm/yyyy&gt;</span> is the current date.</p>
<p>A screenshot of the GitHub organization page as it looked on 15 March 2021 is shown in Figure 6.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2021/04/Figure-6.-GitHub-organization-page-with-repositories-used-by-the-operators-of-Janeleiro.png"><img class="wp-image-149325" src="https://web-assets.esetstatic.com/wls/2021/04/Figure-6.-GitHub-organization-page-with-repositories-used-by-the-operators-of-Janeleiro.png" alt="" width="800" height="462" /></a><p class="caption-text"><em>Figure 6. The purpose of the repository is to contain a file that has the list of IP addresses for Janeleiro‚Äôs C&amp;C servers where it connects to report to its operators, to receive commands and to exfiltrate information in real time.</p>
<p>A screenshot showing one of the repositories in the GitHub organization page attributed to Janeleiro‚Äôs operators is shown in Figure 7, including the username of the account that does the commits.</p>

<p>In the newest version of Janeleiro, version 0.0.3, the developers introduced an interesting encryption/decryption feature using an open-source library called EncryptDecryptUtils. The new procedure for decryption is shown in Figure 9.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2021/04/Figure-9.-Procedure-for-decryption-implemented-by-Janeleiro-version-0.0.3.png"><img class="wp-image-149328 size-full" src="https://web-assets.esetstatic.com/wls/2021/04/Figure-9.-Procedure-for-decryption-implemented-by-Janeleiro-version-0.0.3.png" alt="" width="458" height="215" /></a><p class="caption-text"><em>Figure 9. Procedure for decryption implemented by Janeleiro version 0.0.3</em></p></div></p>
<p>To decrypt a string, Janeleiro encrypts the string resulting from <em>the current date</em> and the result is then used as a passphrase and salt value to create a new key for decryption. This has an extremely important effect: <strong>the newest version of Janeleiro can only decrypt its strings on one intended day.</strong> That could be the same day the strings were encrypted or one day in the future; on any other day, the decryption fails.</p>
<p>This is also true for the contents of the <span style="font-family: courier new, courier, monospace;">SLK</span> file in the main branch: the encrypted and base 64 encoded list of C&amp;C servers as shown in shown in Figure 10.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2021/04/Figure-10.-Contents-of-the-SLK-file-in-the-main-branch..png"><img class="wp-image-149329" src="https://web-assets.esetstatic.com/wls/2021/04/Figure-10.-Contents-of-the-SLK-file-in-the-main-branch..png" alt="" width="800" height="151" /></a><p class="caption-text"><em>Figure 10. Contents of the <span style="font-family: courier new, courier, monospace;">SLK </span>file in the main branch.</em></p></div></p>
<p>The contents are encrypted with the same procedure: when Janeleiro decrypts the contents of the file it must be on one specific date ‚Äì the current date ‚Äì to work as intended.</p>
<h2>Evolution of Janeleiro</h2>
<p>Janeleiro has an internal version value (as shown in Figure 11) that can be used by the attackers to identify which version of their malware successfully compromised a machine. As of March 2021, we have identified four versions, but with two of them sharing the same internal version number.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2021/04/Figure-11.-Configuration-values-used-by-version-0.0.2A-from-2020.png"><img class="wp-image-149330 size-full" src="https://web-assets.esetstatic.com/wls/2021/04/Figure-11.-Configuration-values-used-by-version-0.0.2A-from-2020.png" alt="" width="548" height="230" /></a><p class="caption-text"><em>Figure 11. Configuration values used by version 0.0.2A from 2020</em></p></div></p>
<p>While in 2021 we have seen versions 0.0.2 and 0.0.3, we were interested in finding a missing key piece in the evolution of Janeleiro: version 0.0.1, which should have been in existence in late 2019 or early 2020. To our surprise we found version 0.0.4 samples instead dating to 2019. These new samples of the trojan were deployed by a DLL loader component in tandem with a password stealer, which means the group behind Janeleiro has other tools in their arsenal.</p>
<p>An overview of Janeleiro‚Äôs versions from 2019 through 2021 is shown in Figure 12.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2021/04/Janeleiro_timeline.png"><img class="wp-image-149497" src="https://web-assets.esetstatic.com/wls/2021/04/Janeleiro_timeline.png" alt="" width="800" height="220" /></a><p class="caption-text"><em>Figure 12. Janeleiro‚Äôs strange evolution timeline, based in the internal version of the malware</em></p></div></p>
<p>The inconsistency in the timeline and internal versioning of the malware suggests that it was under development as far back as 2018, and in 2020 they decided to switch to a previous version of their code and to improve that and refine its command processing for the operator to have better control of the trojan during the attack.</p>
<h2>Breaker and keeper of traditions</h2>
<p>While Janeleiro follows the same blueprint for the core implementation of its fake pop-up windows, along with other malware families that ESET has <a href="https://www.welivesecurity.com/2020/10/01/latam-financial-cybercrime-competitors-crime-sharing-ttps/" target="_blank" rel="noopener">documented</a> in the region, it sets itself apart from those malware families in several ways:</p>
<ul>
<li><strong>It is written in Visual Basic .NET:</strong> The curious case of Brazil is that it is mostly targeted by banking trojans developed in Delphi ‚Äì the programming language of choice for several threat actors that are apparently working together sharing tools and infrastructure. Janeleiro‚Äôs preference for VB.NET is a notable deviation from what appears to be the norm for the region.</li>
<li><strong>No binary obfuscation: </strong>While Janeleiro does make use of light obfuscation by generating random names for its classes, modules, method names, parameters, and string encryption, it does not employ packers to make detection and analysis harder. Other trojans such as Grandoreiro, Mekotio, Ousaban, Vadokrist and Guildma make heavy use of Themida and binary padding techniques.</li>
<li><strong>No custom encryption algorithms: </strong>Janeleiro‚Äôs developers rely on cryptographic functions provided by the .NET Framework as well as open-source projects for string encryption/decryption, with a preference for AES and RSA algorithms. Trojans such as Casbaneiro, Grandoreiro, Amavaldo, Mispadu, and Guildma, among others, use custom encryption algorithms, including obfuscation techniques using string tables.</li>
<li><strong>Simple method of execution: </strong>The MSI installer does not deploy other components besides the main trojan DLL or execute further instructions other than load and execute one of the exports of the DLL that installs itself in the system. We have found no samples of an MSI installer executing obfuscated scripts, unpacking support tools, or components for DLL side-loading, which is popular with other malware families in the region.</li>
<li><strong>No defense against security software: </strong>Some of the biggest banks in Brazil require a security module to be installed by their customers before allowing them access to their bank accounts online; for example, Warsaw anti-fraud software. It‚Äôs often the case that LATAM banking trojans try to find out if such software is installed in the compromised machine and report it to the attackers. Some malware families such as Grandoreiro and Guildma attempt to disable it in Windows Firewall or disable its driver.</li>
<li><strong>Uses code from NjRAT:</strong> Janeleiro is far from being another incarnation of the well-known NjRAT, but it does use NjRAT‚Äôs <span style="font-family: courier new, courier, monospace;">SocketClient</span> and Remote Desktop capture functions, as well as other miscellaneous functions. NjRAT is not commonly used ‚Äì at least by LATAM baking trojans ‚Äì perhaps because of their preference to use custom-made trojans in Delphi. However, among other malware, NjRAT has been used in <a href="https://www.welivesecurity.com/2021/01/12/operation-spalax-targeted-malware-attacks-colombia/" target="_blank" rel="noopener">Operation Spalax</a>, a campaign that targets Colombia specifically.</li>
</ul>
<h2>Commands</h2>
<p>Commands with parameters are received from the C&amp;C server in encrypted form with the same algorithm used to encrypt strings (see section <em>Appendix A</em>). A typical command format is like this: <span style="font-family: courier new, courier, monospace;">%CommandName%%PredefinedSeparatorKeyword%%Parameters%</span>.</p>
<p>After decryption the command is split into an array of strings; each part of the command is separated by a predefined keyword hardcoded in the malware‚Äôs configuration ‚Äì all versions we analyzed use <span style="font-family: courier new, courier, monospace;">|'meio'|</span>, which separates the command name and each parameter.</p>
<p>Figure 13 shows how Janeleiro checks the name of the command and executes the requested action.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2021/04/Figure-13.-Example-of-version-0.0.2B-processing-command-startinfo.png"><img class="wp-image-149332" src="https://web-assets.esetstatic.com/wls/2021/04/Figure-13.-Example-of-version-0.0.2B-processing-command-startinfo.png" alt="" width="800" height="251" /></a><p class="caption-text"><em>Figure 13. Example of Version 0.0.2B processing command <span style="font-family: courier new, courier, monospace;">startinfo</span></em></p></div></p>
<p>When Janeleiro sends data back to the operator, it does it in a similar format: <span style="font-family: courier new, courier, monospace;">%CommandName%%PredefinedSeparatorKeyword%%Encoded data%</span>.</p>
<p>The majority of Janeleiro‚Äôs commands are for controlling windows, the mouse and keyboard, and its fake pop-up windows. As the development evolved from Version 0.0.2A to 0.0.3, more commands were added that offered the operator a more refined control:</p>
<ul>
<li>Commands to control specific window</li>
<li>Enumerate and send information about windows (title, class, handle)</li>
<li>Adjust specific window size, minimize, maximize</li>
<li>Dimensions of the screen</li>
<li>Kill all <span style="font-family: courier new, courier, monospace;">chrome.exe</span> processes, and restart <span style="font-family: courier new, courier, monospace;">chrome.exe</span> with arguments <span style="font-family: courier new, courier, monospace;">--disable-gpu</span></li>
<li>Capture the screen in real time</li>
<li>Keylogging in real time</li>
<li>Send keys and mouse clicks</li>
<li>Display or close a specific fake pop-up window</li>
<li>Show or close a specific fake pop-up window</li>
<li>Miscellaneous commands such as: send date and time, disconnect socket, terminate own process</li>
</ul>
<h2>Conclusion</h2>
<p>The experimental nature of Janeleiro, going back and forth between different versions, tell us about an actor who is still trying to find the right way to do it but is no less experienced than the competition: Janeleiro follows the unique blueprint for the core implementation of the fake pop-up windows as many LATAM banking trojans, this does not seem to be a coincidence or inspiration: this actor employs and distributes Janeleiro sharing the same infrastructure as some of the most prominent of these active malware families. As we continue to track the activities of this actor, time will tell what new developments they will come up with in the future.</p>

<h2>Appendix A: Overview of Janeleiro‚Äôs malware family</h2>
<p>Here is each incarnation we have found of Janeleiro from 2019 until March 2021.</p>
<h3>Version 0.0.4</h3>
<ul>
<li>Period of activity: 2019 ‚Äì Possibly still active.</li>
<li>The first version of Janeleiro - that we know of - came in the form of an MSI installer and at least two variants:
<ul>
<li style="text-align: left;">Variant 1: MSI installer loads a DLL called <span style="font-family: courier new, courier, monospace;">LoadDllMSI.dll</span> internally</li>
<li style="text-align: left;">Variant 2: MSI installer executes <span style="font-family: courier new, courier, monospace;">Console.exe</span>, which checks privileges and loads an embedded DLL assembly called <span style="font-family: courier new, courier, monospace;">LoadSystem.dll</span>.</li>
</ul>
</li>
</ul>
<p>Both <span style="font-family: courier new, courier, monospace;">LoadDllMSI.dll</span> and <span style="font-family: courier new, courier, monospace;">LoadSystem.dll</span> perform the same tasks:</p>
<ul>
<li>Create an installation folder</li>
<li>Download and store two modules: <span style="font-family: courier new, courier, monospace;">Logins.Initial.dll</span> and <span style="font-family: courier new, courier, monospace;">System.Modules.Initial.dll</span>. The two modules are downloaded from a GitHub account that, at the time of writing, has been closed.</li>
<li>Create several Shortcuts in strategic places</li>
<li>Log the successful compromise of the system to a tracking website</li>
</ul>
<p><strong>System.Logins:</strong> It is a password stealer for Google Chrome, FileZilla, Mozilla Firefox, Opera, Pidgin, and Mozilla Thunderbird. Additionally, it harvests email information from Gmail. All the information is exfiltrated to two websites. Version 0.0.4 is the only one that is deployed with this malicious tool.</p>
<p><strong>System.Modules:</strong> Janeleiro‚Äôs main trojan, implemented as a Windows Forms application compiled as DLL. This version had the capacity to dynamically create fake pop-up windows using several Forms for several banking entities, including banks operating in Mexico, but it is unknown if this version was distributed in Mexico at any point.</p>
<p>This version used two GitHub organization pages to download the IP addresses of its C&amp;C servers: the names of the pages are generated by encrypting the current date with <span style="font-family: courier new, courier, monospace;">SLK</span> as suffix as shown in Figure 14.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2021/04/Figure-14.-Version-0.0.4-attempts-to-read-file-in-a-GitHub-repository-that-contains-the-encrypted-list-of-CC-servers.png"><img class="wp-image-149333" src="https://web-assets.esetstatic.com/wls/2021/04/Figure-14.-Version-0.0.4-attempts-to-read-file-in-a-GitHub-repository-that-contains-the-encrypted-list-of-CC-servers.png" alt="" width="800" height="583" /></a><p class="caption-text"><em>Figure 14. Version 0.0.4 attempts to read file in a GitHub repository that contains the encrypted list of C&amp;C servers</em></p></div></p>
<p>At the time of writing, we believe that the operators have abandoned this version of the malware. We couldn‚Äôt find any active GitHub pages by following the name generation algorithm used by Janeleiro.</p>
<p>Many commands for the trojan were left unimplemented, some were implemented and other discarded in newer versions used in 2020 and 2021.</p>
<h3>Version 0.0.2A</h3>
<ul>
<li>Period of activity: 2020 ‚Äì Unknown.</li>
<li>Internal Malware Version: 0.0.2</li>
</ul>
<p>The MSI installer loads a DLL that borrows from <span style="font-family: courier new, courier, monospace;">LoadSystem</span> installation and persistence procedures but unpacks the embedded main trojan DLL from its resources. The main trojan was implemented as a Windows Forms application compiled as DLL.</p>
<p>This version of Janeleiro only uses one Form to create the fake pop-up windows with more commands supported by the operator but with fewer targets: Mexican banking entities were discarded. All of the images used to cover the screen and trick the user are for Brazilian banks.</p>
<p>This version also appears to have been abandoned and cannot contact its C&amp;C servers by retrieving the IP lists from a GitHub page. It uses the same algorithm as Version 0.0.4 with the same key <span style="font-family: courier new, courier, monospace;">vhpjzqqtpo</span>, suggesting that the operators where using the same GitHub page as for Version 0.0.4. Figure 15 shows the code that attempts to retrieve the list from GitHub.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2021/04/Figure-15.-Version-0.0.2A-attempts-to-download-a-new-list-of-CC-servers-from-a-repository-on-a-GitHub-organization-page.png"><img class="wp-image-149334" src="https://web-assets.esetstatic.com/wls/2021/04/Figure-15.-Version-0.0.2A-attempts-to-download-a-new-list-of-CC-servers-from-a-repository-on-a-GitHub-organization-page.png" alt="" width="800" height="300" /></a><p class="caption-text"><em>Figure 15. Version 0.0.2A attempts to download a new list of C&amp;C servers from a repository on a GitHub organization page</em></p></div></p>
<h3>Version 0.0.2B</h3>
<ul>
<li>Period of activity: 2021 ‚Äì Still active.</li>
<li>Internal Malware Version: 0.0.2</li>
</ul>
<p>New characteristics of this version:</p>
<ul>
<li>Implemented as a Windows Presentation Foundation application</li>
<li>Major restructuration of the code combining the loader code with the main trojan</li>
<li>Geolocation of the compromised machine</li>
<li>Implementation of clipboard hijacking to replace bitcoin addresses</li>
<li>Expanded set of supported commands</li>
<li>Strings encrypted/decrypted with the RSA algorithm</li>
</ul>
<p>Figure 16 shows the implementation of clipboard hijacking by Janeleiro; when a bitcoin address is found, it randomly picks one from its own list of bitcoin addresses and replaces it.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2021/04/Figure-16.-Janeleiros-implementation-of-clipboard-hijacking.png"><img class="wp-image-149335 size-full" src="https://web-assets.esetstatic.com/wls/2021/04/Figure-16.-Janeleiros-implementation-of-clipboard-hijacking.png" alt="" width="634" height="448" /></a><p class="caption-text"><em>Figure 16. Janeleiro's implementation of clipboard hijacking</em></p></div></p>
<p>In this version a simplified procedure was implemented to retrieve the addresses of its C&amp;C servers from a GitHub organization page; the name scheme this time is a simple concatenation of <span style="font-family: courier new, courier, monospace;">SLK</span> with the current date time without the slashes, as shown in Figure 17.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2021/04/Figure-17.-Version-0.0.2B-procedure-to-retrieve-its-list-of-CC-servers.-We-have-decrypted-some-strings-for-clarity..png"><img class="wp-image-149336" src="https://web-assets.esetstatic.com/wls/2021/04/Figure-17.-Version-0.0.2B-procedure-to-retrieve-its-list-of-CC-servers.-We-have-decrypted-some-strings-for-clarity..png" alt="" width="800" height="116" /></a><p class="caption-text"><em>Figure 17. Version 0.0.2B procedure to retrieve its list of C&amp;C servers. We have decrypted some strings for clarity.</em></p></div></p>
<p>The code attempts to download the contents of a file in a secondary branch. The file contains, in plaintext, the list of the C&amp;C IP addresses and ports. At the time of writing, the GitHub organization pages can be found using the procedure as they continue to operate with this recent version of Janeleiro.</p>
<h3>Version 0.0.3</h3>
<ul>
<li>Period of activity: Since March 2021 ‚Äì Still active.</li>
<li>Internal Malware Version: 0.0.3</li>
</ul>
<p>New characteristics of this version:</p>
<ul>
<li>Implemented as a Windows Forms application</li>
<li>A recombination of Version 0.0.2A and 0.0.2B code and technique implementations</li>
<li>New persistence method using Windows Registry Run Key</li>
<li>Expanded set of supported commands</li>
<li>Uses AES algorithm to encrypt/decrypt its strings</li>
</ul>
<p>This version uses the same procedure as Version 0.0.2B to get the C&amp;C servers from the GitHub organization page, with the difference that it uses the main branch within the repository and the list is encrypted and encoded with base64 as shown in Figure 18.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2021/04/Figure-18.-Main-repository-containing-an-encrypted-list-of-CC-servers.png"><img class="wp-image-149337" src="https://web-assets.esetstatic.com/wls/2021/04/Figure-18.-Main-repository-containing-an-encrypted-list-of-CC-servers.png" alt="" width="800" height="162" /></a><p class="caption-text">Figure 18. Main repository containing an encrypted list of C&amp;C servers</p></div></p>
<p>This procedure is also used when decrypting the list of C&amp;C servers, therefore there must exist a repository containing the file in the main branch, with the encrypted list intended for that day. Otherwise this version cannot contact the operators as decryption will fail.</p>
