    <div class="article-tags mb-2 dark big"><p class="article-tag text-capitalize">ESET Research</p></div>    <h1 class="page-headline">Mustang Panda’s Hodur: Old tricks, new Korplug variant</h1>    <p class="sub-title">ESET researchers have discovered Hodur, a previously undocumented Korplug variant spread by Mustang Panda, that uses phishing lures referencing current events in Europe, including the invasion of Ukraine</p>

                    <div class="article-body">
    <p>ESET researchers discovered a still-ongoing campaign using a previously undocumented Korplug variant, which they named Hodur due to its resemblance to the <em>THOR variant</em> previously documented by <a href="https://unit42.paloaltonetworks.com/thor-plugx-variant/" target="_blank" rel="noopener"><em>Unit 42</em></a> in 2020. In Norse mythology, Hodur is Thor’s blind half-brother, who is tricked by Loki into killing their half-brother Baldr.</p>
<p><strong>Key findings in this blogpost:</strong></p>
<ul>
<li>As of March 2022, this campaign is still ongoing and goes back to at least August 2021.</li>
<li>Known victims include research entities, internet service providers, and European diplomatic missions.</li>
<li>The compromise chain includes decoy documents that are frequently updated and relate to events in Europe.</li>
<li>The campaign uses a custom loader to execute a new Korplug variant.</li>
<li>Every stage of the deployment process utilizes anti-analysis techniques and control-flow obfuscation, which sets it apart from other campaigns.</li>
<li>ESET researchers provide an in-depth analysis of the capabilities and commands of this new variant.</li>
</ul>
<p>Victims of this campaign are likely lured with phishing documents abusing the latest events in Europe such as Russia’s invasion of Ukraine. This resulted in more than <a href="https://data2.unhcr.org/en/situations/ukraine" target="_blank" rel="noopener"><em>three million residents</em></a> fleeing the war to neighboring countries, leading to an unprecedented crisis on Ukraine’s borders. One of the filenames related to this campaign is <span style="font-family: 'courier new', courier, monospace;">Situation at the EU borders with Ukraine.exe</span>.</p>
<p>Other phishing lures mention updated COVID-19 travel restrictions, an approved regional aid map for Greece, and a Regulation of the European Parliament and of the Council. The last one is a real document available on the European Council’s website. This shows that the APT group behind this campaign is following current affairs and is able to successfully and swiftly react to them.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2022/03/Figure-1.-Countries-affected-by-Mustang-Panda-in-this-campaign-1.png"><img class="wp-image-160405" src="https://web-assets.esetstatic.com/wls/2022/03/Figure-1.-Countries-affected-by-Mustang-Panda-in-this-campaign-1.png" alt="" width="800" height="425" /></a><p class="caption-text"><em>Figure 1. Countries affected by Mustang Panda in this campaign</em></p></div></p>
<h3>Affected countries:</h3>
<ul>
<li>Mongolia</li>
<li>Vietnam</li>
<li>Myanmar</li>
<li>Greece</li>
<li>Russia</li>
<li>Cyprus</li>
<li>South Sudan</li>
<li>South Africa</li>
</ul>
<h3>Affected verticals:</h3>
<ul>
<li>Diplomatic missions</li>
<li>Research entities</li>
<li>Internet service providers (ISPs)</li>
</ul>
<h2>Analysis</h2>
<p>Based on code similarities and the many commonalities in Tactics, Techniques, and Procedures (TTPs), ESET researchers attribute this campaign with <strong>high</strong> confidence to Mustang Panda (also known as TA416, RedDelta, or PKPLUG). It is a cyberespionage group mainly targeting governmental entities and NGOs. Its victims are mostly, but not exclusively, located in East and Southeast Asia with a focus on Mongolia. The group is also known for its <a href="https://www.recordedfuture.com/reddelta-targets-catholic-organizations/" target="_blank" rel="noopener"><em>campaign targeting the Vatican in 2020</em></a>.</p>
<p>While we haven’t been able to identify the verticals of all victims, this campaign seems to have the same targeting objectives as other Mustang Panda campaigns. Following the APT’s typical victimology, most victims are located in East and Southeast Asia, along with some in European and African countries. According to ESET telemetry, the vast majority of targets are located in Mongolia and Vietnam, followed by Myanmar, with only a few in the other affected countries.</p>
<p>Mustang Panda’s campaigns frequently use custom loaders for shared malware including Cobalt Strike, Poison Ivy, and Korplug (also known as PlugX). The group has also been known to create its own Korplug variants. Compared to other campaigns using Korplug, every stage of the deployment process utilizes anti-analysis techniques and control-flow obfuscation.</p>
<p>This blogpost contains a detailed analysis of this previously unseen Korplug variant used in this campaign. This activity is part of the same campaign <a href="https://www.proofpoint.com/us/blog/threat-insight/good-bad-and-web-bug-ta416-increases-operational-tempo-against-european" target="_blank" rel="noopener"><em>recently covered by Proofpoint</em></a>, but we provide additional historical and targeting information.</p>
<h2>Toolset</h2>
<p>Mustang Panda is known for its elaborate custom loaders and Korplug variants, and the samples used in this campaign showcase this perfectly.</p>
<p>Compromise chains seen in this campaign follow the typical Korplug pattern: a legitimate, validly signed, executable vulnerable to DLL search-order hijacking, a malicious DLL, and an encrypted Korplug file are deployed on the target machine. The executable is abused to load the module, which then decrypts and executes the Korplug RAT. In some cases, a downloader is used first to deploy these files along with a decoy document. This process is illustrated in Figure 2.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2022/03/Figure-2.-Overview-of-the-deployment-process-for-the-Hodur-Korplug-variant..png"><img class="wp-image-160393" src="https://web-assets.esetstatic.com/wls/2022/03/Figure-2.-Overview-of-the-deployment-process-for-the-Hodur-Korplug-variant..png" alt="" width="600" height="937" /></a><p class="caption-text"><em>Figure 2. Overview of the deployment process for the Hodur Korplug variant.</em></p></div></p>
<p>What sets this campaign apart is the heavy use of control-flow obfuscation and anti-analysis techniques at every stage of the deployment process. The following sections describe the behavior of each stage and take a deeper look at the defense evasion techniques used in each of them.</p>
<h3>Initial access</h3>
<p>We haven’t been able to observe the initial deployment vector, but our analysis points to phishing and watering hole attacks as likely vectors. In instances where we saw a downloader, the filenames used suggest a document with an interesting subject for the target. Such examples include:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">COVID-19 travel restrictions EU reviews list of third countries.exe</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">State_aid__Commission_approves_2022-2027_regional_aid_map_for_Greece.exe</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">REGULATION OF THE EUROPEAN PARLIAMENT AND OF THE COUNCIL.exe</span></li>

The contents of these decoys accurately reflect the filename. As shown in Figure 3, at least one of them is a publicly accessible legitimate document from the European Parliament.</p>

<h3>Downloader</h3>
<p>Although its complexity has increased over the course of the campaign, the downloader is fairly straightforward. This increase in complexity comes from additional anti-analysis techniques, which we cover later in this section.</p>
<p>It first downloads four files over HTTPS: a decoy document, a legitimate executable, a malicious module, and an encrypted Korplug file. The combination of those last three components to execute a payload via DLL side-loading is sometimes referred to as a <a href="https://www.welivesecurity.com/2020/12/10/luckymouse-ta428-compromise-able-desktop/" target="_blank" rel="noopener"><em>trident</em></a> and is a technique commonly used by Mustang Panda, and with Korplug loaders in general. Both the server addresses and file paths are hardcoded in the downloader executable. Once everything is downloaded, and the decoy document opened to distract the victim, the downloader uses the following command line to launch the legitimate executable:</p>
<p style="margin-left: 10%; margin-right: 10%;"><span style="font-family: 'courier new', courier, monospace;">cmd /c ping 8.8.8.8 -n 70&amp;&amp;"%temp%\&lt;legitimate executable&gt;"</span></p>
<p>This <span style="font-family: 'courier new', courier, monospace;">ping</span> command both checks internet connectivity and introduces a delay (through the <span style="font-family: 'courier new', courier, monospace;">-n 70</span> option) before executing the downloaded, legitimate executable.</p>
<p>The downloader uses multiple anti-analysis techniques, many of which are also used in the loader and final payload. Additional obfuscation has been added to new versions over the course of the campaign without otherwise changing their goal.</p>
<p>In early versions of the downloader, junk code and opaque predicates were used to hinder analysis, as shown in Figure 4, but the server and filenames are plainly visible in cleartext.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2022/03/Figure-4.-Control-flow-obfuscation-in-early-versions-of-the-downloader.png"><img class="wp-image-160395" src="https://web-assets.esetstatic.com/wls/2022/03/Figure-4.-Control-flow-obfuscation-in-early-versions-of-the-downloader.png" alt="" width="700" height="223" /></a><p class="caption-text"><em>Figure 4. Control flow obfuscation in early versions of the downloader</em></p></div></p>
<p>In later versions, the files on the server are RC4 encrypted, using the base 10 string representation of the file size as the key, and then hex-encoded. This process is illustrated in the Python snippet below. The opposite operations are performed client-side by the downloader to recover the plaintext files. This is likely done to bypass network-level protections.</p>
<p style="margin-left: 10%; margin-right: 10%;"><span style="font-family: 'courier new', courier, monospace;">from Crypto.cipher import ARC4</span><br />
<span style="font-family: 'courier new', courier, monospace;">key = "%d" % len(plaintext)</span><br />
<span style="font-family: 'courier new', courier, monospace;">rc4 = ARC4.new(key)</span><br />
<span style="font-family: 'courier new', courier, monospace;">cipher_content = rc4.encrypt(plaintext).hex().upper()</span></p>
<p>These versions replace the use of cleartext strings with encrypted stack strings. They are still hardcoded in the file, but the obfuscation surrounding them, and the use of different keys, makes it hard to decrypt them statically in an automated manner. This same technique is used heavily in the subsequent stages. Encrypted stack strings are also used to obfuscate calls to Windows API functions.</p>
<p>First, the name of the target function is decrypted and passed to a function. This function obtains a pointer to the <span style="font-family: 'courier new', courier, monospace;">InMemoryOrderModuleList</span> field of the <span style="font-family: 'courier new', courier, monospace;">PEB</span> (<a href="https://docs.microsoft.com/en-us/windows/win32/api/winternl/ns-winternl-peb" target="_blank" rel="noopener"><em>Process Environment Block</em></a>). It then iterates over the loaded modules, passing each handle to <span style="font-family: 'courier new', courier, monospace;">GetProcAddress</span> along with the function name until the target function is successfully resolved. Part of this process can be seen in Figure 5.</p>

<h3>Loader</h3>
<p>As is common with Korplug, the loader is a DLL that exploits a side-loading vulnerability in a legitimate, signed executable. We have observed many different applications being abused in this campaign, for instance a vulnerable SmadAV executable previously seen by Qurium <a href="https://www.qurium.org/alerts/targeted-malware-against-crph/"><em>in a campaign attributed to Mustang Panda</em></a> that targeted Myanmar.</p>
<p>The loader exports multiple functions. The exact list varies depending on the abused application, but in all cases, only one of them does anything of consequence. In all of the loaders we observed, this is the exported function with the highest load address. All the other exports, and the library’s entry point, either return immediately or execute some do-nothing junk code. Many of these exports have names that consist of random lowercase letters and point to the same address as shown in Table 1.</p>

<p>The loader function obtains the directory from which the DLL is running using <span style="font-family: 'courier new', courier, monospace;">GetModuleFileNameA</span> and tries to open the encrypted Korplug file it contains. That filename is hardcoded in the loader. It reads the file’s contents into a locally allocated buffer and decrypts it. The loader makes this buffer executable using <span style="font-family: 'courier new', courier, monospace;">VirtualProtect</span> before calling into it at offset 0x00.</p>
<p>Windows API function calls are obfuscated with a different technique than that used in the downloader. Unlike the loader, which contains the <em>names</em> of its functions (as shown in Table 1 above), only the 64-bit <em>hashes</em> of the Windows API function calls are present in the binary. To resolve those functions, the loader traverses the export lists of all loaded libraries via the <span style="font-family: 'courier new', courier, monospace;">InMemoryOrderModuleList</span> of the <span style="font-family: 'courier new', courier, monospace;">PEB</span>. Each export’s name is hashed, then compared to the expected value. The FNV-1a hash algorithm, recently brought back into the mainstream by the <a href="https://www.mandiant.com/resources/sunburst-additional-technical-details" target="_blank" rel="noopener"><em>Sunburst backdoor</em></a>, has previously been used by Mustang Panda, in <a href="https://blog.xorhex.com/blog/reddeltaplugxchangeup/" target="_blank" rel="noopener"><em>Korplug loaders</em></a> documented by XORHEX, to resolve <span style="font-family: 'courier new', courier, monospace;">GetProcAddress</span> and <span style="font-family: 'courier new', courier, monospace;">LoadLibraryA</span>, although it was not identified by name in that analysis. In this version, however, it is used for all API functions.</p>
<h3>Korplug backdoor</h3>
<p>Korplug (also known as PlugX) is a RAT used by multiple APT groups. In spite of it being so widely used, or perhaps because of it, few reports extensively describe its commands and the data it exfiltrates. Its functionality is not constant between variants, but there does seem to exist a significant overlap in the list of commands between the version we analyzed and other sources such as the <a href="https://www.avira.com/en/blog/new-wave-of-plugx-targets-hong-kong" target="_blank" rel="noopener"><em>Avira report from January 2020</em></a> and the <a href="https://github.com/kcreyts/plugxdecoder" target="_blank" rel="noopener"><em>plugxdecoder</em></a> project on GitHub.</p>
<p>As previously mentioned, the variant used in this campaign bears many similarities to the THOR variant, which is why we have named it Hodur. The similarities include the use of the <span style="font-family: 'courier new', courier, monospace;">Software\CLASSES\ms-pu</span> registry key, the same format for C&amp;C servers in the configuration, and use of the <span style="font-family: 'courier new', courier, monospace;">Static</span> window class.</p>
<p>As expected for Korplug payloads, this stage is only ever decrypted in memory by the loader. Only the encrypted version is written to disk in a file with a <span style="font-family: 'courier new', courier, monospace;">.dat</span> extension.</p>
<p>Unless stated otherwise, all hardcoded strings discussed in this section are stored as encrypted stack strings.</p>
<p>In this module, Windows API functions are obfuscated through a combination of the methods used in previous stages. <span style="font-family: 'courier new', courier, monospace;">LoadLibraryA</span> and <span style="font-family: 'courier new', courier, monospace;">GetProcAddress</span> are resolved via the FNV-1a hashing technique and stack strings are decrypted and passed to them to obtain the target function.</p>
<h4><strong><em>Loading</em></strong></h4>
<p>Once decrypted, the payload is a valid DLL that exports a single function. In almost all observed samples from this campaign, this function is named <span style="font-family: 'courier new', courier, monospace;">StartProtect</span>. However, launching it directly via this export or its entry point will not execute the main payload and the loading process is quite intricate.</p>
<p>As explained in the previous section, the file is decrypted in memory as a continuous blob by the loader and the execution starts at offset 0x00. The PE header contains shellcode, shown in Figure 6, that calls a specific offset that corresponds to the module’s single export.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2022/03/Figure-6.-Shellcode-in-the-PE-header-that-calls-the-exported-function.png"><img class="wp-image-160399" src="https://web-assets.esetstatic.com/wls/2022/03/Figure-6.-Shellcode-in-the-PE-header-that-calls-the-exported-function.png" alt="" width="700" height="259" /></a><p class="caption-text"><em>Figure 6. Shellcode in the PE header that calls the exported function</em></p></div></p>
<p>This function parses the PE blob in memory and manually maps it as a library into a newly allocated buffer. This includes mapping the various sections, resolving imports and, finally, using <span style="font-family: 'courier new', courier, monospace;">DLL_PROCESS_ATTACH</span> to call the DLL entry point. Once again, opaque predicates and junk code are used to obfuscate the purpose of this function.</p>
<p>The entry point of the properly loaded library is then called with the non-standard value of <span style="font-family: 'courier new', courier, monospace;">0x04</span> for the <span style="font-family: 'courier new', courier, monospace;">fdwReason</span> parameter (only values from <span style="font-family: 'courier new', courier, monospace;">0x00</span> to <span style="font-family: 'courier new', courier, monospace;">0x03</span> are <a href="https://docs.microsoft.com/en-us/windows/win32/dlls/dllmain" target="_blank" rel="noopener"><em>currently defined</em></a>). This special value is required to get it to execute its main payload. This simple check prevents the RAT from being trivially executed directly with a generic tool like <span style="font-family: 'courier new', courier, monospace;">rundll32.exe</span>.</p>
<p>The backdoor first decrypts its configuration using the string <span style="font-family: 'courier new', courier, monospace;">123456789</span> as a repeating XOR key. Once decrypted, the configuration block starts with <span style="font-family: 'courier new', courier, monospace;">########</span>. The layout of the configuration varies slightly between samples, but they all contain at least the following fields:</p>
<ul>
<li>Installation directory name. Also used as the name of the registry key created for persistence. This value roughly corresponds to the name of the abused application with three random letters appended (e.g., <span style="font-family: 'courier new', courier, monospace;">FontEDLZeP</span> or <span style="font-family: 'courier new', courier, monospace;">AdobePhotosGQp</span>)</li>
<li>Mutex name</li>
<li>A value that is either a version or ID string</li>
<li>List of C&amp;C servers. Each entry includes IP address, port number, and a number indicating the protocol to use with that C&amp;C</li>
</ul>
<p>The backdoor then checks the path from which it is running using <span style="font-family: 'courier new', courier, monospace;">GetModuleFileNameW</span>. If this matches <span style="font-family: 'courier new', courier, monospace;">%userprofile%\&lt;installation directory&gt;</span> or <span style="font-family: 'courier new', courier, monospace;">%allusersprofile%\&lt;installation directory&gt;</span>, the RAT functionality will be executed. Otherwise, it will go through the installation process.</p>
<h4><strong><em>Installation</em></strong></h4>
<p>To install itself, the malware creates the aforementioned directory under <span style="font-family: 'courier new', courier, monospace;">%allusersprofile%</span>. Using <span style="font-family: 'courier new', courier, monospace;">SetFileAttributesW</span>, it is then marked as <span style="font-family: 'courier new', courier, monospace;">hidden</span> and <span style="font-family: 'courier new', courier, monospace;">system</span>. The vulnerable executable, loader module, and encrypted Korplug files are copied to the new directory.</p>
<p>Next, persistence is established. Earlier samples achieved this by creating a scheduled task to be run at boot via <span style="font-family: 'courier new', courier, monospace;">schtasks.exe</span>. Newer samples add a registry entry to <span style="font-family: 'courier new', courier, monospace;">Software\Microsoft\Windows\CurrentVersion\Run</span>, trying the <span style="font-family: 'courier new', courier, monospace;">HKLM</span> hive first, then <span style="font-family: 'courier new', courier, monospace;">HKCU</span>. This entry has the same name as the installation directory with its value set to the newly copied executable’s path.</p>
<p>Once persistence has been set up, the malware launches the executable from its new location and exits.</p>
<h4><em><strong>RAT</strong></em></h4>
<p>The RAT functionality of the Hodur variant used in this campaign mostly lines up with other Korplug variants, with some additional commands and characteristics. As we have previously stated, though, detailed analyses of Korplug commands are few and far between, so we aim to provide such an analysis in the hopes of aiding future analysts.</p>
<p>When in this mode, the backdoor iterates through the list of C&amp;C servers in its configuration until it reaches the end or receives an Uninstall command. For each of those servers, it processes commands until it receives a Stop command or encounters an error.</p>
<p>Hodur’s initial handshake can be done over HTTPS or TCP. This is determined by a value in the configuration for that particular C&amp;C server. Subsequent communication is always done over TCP using a custom protocol that we describe in this section, along with the commands that can be issued. Hodur uses sockets from the Windows Sockets API (Winsock) that support <a href="https://docs.microsoft.com/en-us/windows/win32/winsock/overlapped-i-o-2"><em>overlapped I/O</em></a>.</p>
<p>Following the initial handshake, Hodur’s communications involve TCP messages that consist of a header, with the structure described in Table 2, followed by a message body that is usually compressed using LZNT1 and always encrypted with RC4. Messages whose Command number header field have the <span style="font-family: 'courier new', courier, monospace;">0x10000000</span> bit set (those that contain file contents for the <span style="font-family: 'courier new', courier, monospace;">ReadFile</span> and <span style="font-family: 'courier new', courier, monospace;">WriteFile</span> commands, described in Table 3) have encrypted but not compressed message bodies. All encrypted message bodies use the hardcoded key <span style="font-family: 'courier new', courier, monospace;">sV!e@T#L$PH%</span> with a four-byte random nonce (the value at offset 0x00 in the header) appended to it.</p>

<h2>Conclusion</h2>
<p>The decoys used in this campaign show once more how quickly Mustang Panda is able to react to world events. For example, an EU regulation on COVID-19 was used as a decoy only two weeks after it came out, and documents about the war in Ukraine started being used in the days following the beginning of the launch of the invasion. This group also demonstrates an ability to iteratively improve its tools, including its signature use of trident downloaders to deploy Korplug.</p>
