    <div class="article-tags mb-2 dark big"><p class="article-tag text-capitalize">ESET Research</p></div>    <h1 class="page-headline">Fantasy – a new Agrius wiper deployed through a supply-chain attack</h1>    <p class="sub-title">ESET researchers analyzed a supply-chain attack abusing an Israeli software developer to deploy Fantasy, Agrius’s new wiper, with victims including the diamond industry</p>

                    <div class="article-body">
    <p>ESET researchers discovered a new wiper and its execution tool, both attributed to the Agrius APT group, while analyzing a supply-chain attack abusing an Israeli software developer. The group is known for its destructive operations.</p>
<p>In February 2022, Agrius began targeting Israeli HR and IT consulting firms, and users of an Israeli software suite used in the diamond industry. We believe that Agrius operators conducted a supply-chain attack abusing the Israeli software developer to deploy their new wiper, Fantasy, and a new lateral movement and Fantasy execution tool, Sandals.</p>
<p>The Fantasy wiper is built on the foundations of the previously reported <a href="https://www.sentinelone.com/labs/new-version-of-apostle-ransomware-reemerges-in-targeted-attack-on-higher-education/" target="_blank" rel="noopener">Apostle wiper</a> but does not attempt to masquerade as ransomware, as Apostle originally did. Instead, it goes right to work wiping data. Victims were observed in South Africa – where reconnaissance began several weeks before Fantasy was deployed – Israel, and Hong Kong.</p>
<span class="update-block"><p><strong>Key points of this blogpost:</strong>
<ul>
<li>Agrius conducted a supply-chain attack abusing an Israeli software suite used in the diamond industry.</li>
<li>The group then deployed a new wiper we named Fantasy. Most of its code base comes from Apostle, Agrius’s previous wiper.</li>
<li>Along with Fantasy, Agrius also deployed a new lateral movement and Fantasy execution tool that we have named Sandals.</li>
<li>Victims include Israeli HR firms, IT consulting companies, and a diamond wholesaler; a South African organization working in the diamond industry; and a jeweler in Hong Kong.</li></ul></p></span>
<h2>Group overview</h2>
<p>Agrius is a newer <a href="https://www.sentinelone.com/labs/new-version-of-apostle-ransomware-reemerges-in-targeted-attack-on-higher-education/" target="_blank" rel="noopener">Iran-aligned</a> group targeting victims in Israel and the United Arab Emirates <a href="https://www.sentinelone.com/labs/new-version-of-apostle-ransomware-reemerges-in-targeted-attack-on-higher-education/" target="_blank" rel="noopener">since 2020</a>. The group initially deployed a wiper, Apostle, disguised as ransomware, but later <a href="https://assets.sentinelone.com/sentinellabs/evol-agrius" target="_blank" rel="noopener">modified Apostle</a> into fully fledged ransomware. Agrius exploits known vulnerabilities in internet-facing applications to install webshells, then conducts internal reconnaissance before moving laterally and then deploying its malicious payloads.</p>
<h2>Campaign overview</h2>
<p>On February 20<sup>th</sup>, 2022 at an organization in the diamond industry in South Africa, Agrius deployed credential harvesting tools, probably in preparation for this campaign. Then, on March 12<sup>th</sup>, 2022, Agrius launched the wiping attack by deploying Fantasy and Sandals, first to the victim in South Africa and then to victims in Israel and lastly to a victim in Hong Kong.</p>
<p>Victims in Israel include an IT support services company, a diamond wholesaler, and an HR consulting firm. South African victims are from a single organization in the diamond industry, with the Hong Kong victim being a jeweler.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2022/12/F1.png"><img class="wp-image-167617 size-large" src="https://web-assets.esetstatic.com/wls/2022/12/F1.png" alt="" width="1024" height="549" /></a><p class="caption-text"><em>Figure 1. Victim timeline and locations</em></p></div></p>
<p>The campaign lasted less than three hours and within that timeframe ESET customers were already protected with detections identifying Fantasy as a wiper, and blocking its execution. We observed the software developer pushing out clean updates within a matter of hours of the attack. We reached out to the software developer to notify them about a potential compromise, but our enquiries went unanswered.</p>
<h2>Preparing for departure</h2>
<p>The first tools deployed by Agrius operators to victim systems, through means unknown, were:</p>
<ul>
<li><a href="https://github.com/cube0x0/MiniDump" target="_blank" rel="noopener">MiniDump</a>, “<em>a C# implementation of mimikatz/pypykatz minidump functionality to get credentials from LSASS dumps</em>”</li>
<li><a href="https://github.com/bb00/zer0dump/blob/master/secretsdump.py" target="_blank" rel="noopener">SecretsDump</a>, a Python script that “<em>performs various techniques to dump hashes from [a] remote machine without executing any agent there</em>”</li>
<li>Host2IP, a custom C#/.NET tool that resolves a hostname to an IP address.</li>
</ul>
<p>Usernames, passwords, and hostnames collected by these tools are required for Sandals to successfully spread and execute the Fantasy wiper. Agrius operators deployed MiniDump and SecretsDump to this campaign’s first victim on February 20<sup>th</sup>, 2022, but waited until March 12<sup>th</sup>, 2022 to deploy Host2IP, Fantasy, and Sandals (consecutively).</p>
<h2>Sandals: Igniting the Fantasy (wiper)</h2>
<p>Sandals is a 32-bit Windows executable written in C#/.NET. We chose the name because Sandals is an anagram of some of the command line arguments it accepts. It is used to connect to systems in the same network via SMB, to write a batch file to disk that executes the Fantasy wiper, and then run that batch file via <a href="https://docs.microsoft.com/en-us/sysinternals/downloads/psexec">PsExec</a> with this command line string:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">PsExec.exe /accepteula -d -u "&lt;username&gt;" -p "&lt;password&gt;" -s "C:\&lt;path&gt;\&lt;GUID&gt;.bat"</span></li>
</ul>
<p>The PsExec options have the following meanings:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">-d</span> – Don't wait for process to terminate (non-interactive).</li>
<li><span style="font-family: 'courier new', courier, monospace;">/accepteula</span> – Suppress display of the license dialog.</li>
<li><span style="font-family: 'courier new', courier, monospace;">-s</span> – Run the remote process in the <span style="font-family: 'courier new', courier, monospace;">SYSTEM</span> account.</li>
</ul>
<p>Sandals does not write the Fantasy wiper to remote systems. We believe that the Fantasy wiper is deployed via a supply-chain attack using the software developer’s software update mechanism. This assessment is based on several factors:</p>
<ul>
<li>all victims were customers of the affected software developer;</li>
<li>the Fantasy wiper was named in a similar fashion to legitimate versions of the software;</li>
<li>all victims executed the Fantasy wiper within a 2.5 hour timeframe, where victims in South Africa were targeted first, then victims in Israel, and finally victims in Hong Kong (we attribute the delay in targeting to time zone differences and a hardcoded check-in time within the legitimate software); and,</li>
<li>lastly, the Fantasy wiper was written to, and executed from, <span style="font-family: 'courier new', courier, monospace;">%SYSTEM%\Windows\Temp</span>, the default temp directory for Windows systems.</li>
</ul>
<p>Additionally, we believe the victims were already using PsExec, and Agrius operators chose to use PsExec to blend into typical administrative activity on the victims’ machines, and for ease of batch file execution. Table 1 lists the command line arguments accepted by Sandals.</p>

<h2>Fantasy wiper</h2>
<p>The Fantasy wiper is also a 32-bit Windows executable written in C#/.NET, so named for its filenames: <span style="font-family: 'courier new', courier, monospace;">fantasy45.exe</span> and <span style="font-family: 'courier new', courier, monospace;">fantasy35.exe</span>, respectively. Figure 3 depicts the execution flow of the Fantasy wiper.</p>
<p><div class="caption center"><a href="https://web-assets.esetstatic.com/wls/2022/12/F3.png"><img class="wp-image-167619" src="https://web-assets.esetstatic.com/wls/2022/12/F3.png" alt="" width="750" height="1736" /></a><p class="caption-text"><em>Figure 3. Fantasy wiper execution flow</em></p></div></p>
<p>Initially, Fantasy creates a mutex to ensure that only one instance is running. It collects a list of fixed drives but excludes the drive where the <span style="font-family: 'courier new', courier, monospace;">%WINDOWS%</span> directory exists. Then it enters a for loop iterating over the drive list to build a recursive directory listing, and uses the <span style="font-family: 'courier new', courier, monospace;">RNGCryptoServiceProvider.GetBytes</span> method to create <a href="https://docs.microsoft.com/en-us/dotnet/api/system.security.cryptography.rngcryptoserviceprovider.getbytes?view=net-6.0" target="_blank" rel="noopener">a cryptographically strong sequence of random values</a> in a 4096-byte array. If a runtime argument of <span style="font-family: 'courier new', courier, monospace;">411</span> is supplied to the wiper, the <span style="font-family: 'courier new', courier, monospace;">for</span> loop overwrites the contents of every file with the aforementioned byte array using a nested <span style="font-family: 'courier new', courier, monospace;">while</span> loop. Otherwise, the <span style="font-family: 'courier new', courier, monospace;">for</span> loop only overwrites files with a file extension listed in the Appendix.</p>
<p>Fantasy then assigns a specific timestamp (<span style="font-family: 'courier new', courier, monospace;">2037-01-01 00:00:00</span>) to these file timestamp properties:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">CreationTime</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">LastAccessTime</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">LastWriteTime</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">CreationTimeUtc</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">SetLastAccessTimeUtc</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">LastWriteTimeUtc</span></li>
</ul>
<p>and then deletes the file. This is presumably done to make recovery and forensic analysis more difficult.</p>
<p>During the <span style="font-family: 'courier new', courier, monospace;">for</span> loop, the Fantasy wiper counts errors within the current directory when attempting to overwrite files. If the number of errors exceeds 50, it writes a batch file, <span style="font-family: 'courier new', courier, monospace;">%WINDOWS%\Temp\&lt;GUID&gt;.bat</span>, that deletes the directory with the files causing the errors, and then self-deletes. File wiping then resumes in the next directory in the target list.</p>
<p>Once the <span style="font-family: 'courier new', courier, monospace;">for</span> loop completes, the Fantasy wiper creates a batch file in <span style="font-family: 'courier new', courier, monospace;">%WINDOWS%\Temp</span> called <span style="font-family: 'courier new', courier, monospace;">registry.bat</span>. The batch file deletes the following registry keys:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">HKCR\.EXE</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">HKCR\.dll</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">HKCR\*</span></li>
</ul>
<p>Then it runs the following to attempt to clear file system cache memory:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">%windir%\system32\rundll32.exe advapi32.dll,ProcessIdleTasks</span></li>
</ul>
<p>Lastly, <span style="font-family: 'courier new', courier, monospace;">registry.bat</span> deletes itself (<span style="font-family: 'courier new', courier, monospace;">del %0</span>).</p>
<p>Next, the Fantasy wiper clears all Windows event logs and creates another batch file, <span style="font-family: 'courier new', courier, monospace;">system.bat</span>, in <span style="font-family: 'courier new', courier, monospace;">%WINDOWS%\Temp</span>, that recursively deletes all files on <span style="font-family: 'courier new', courier, monospace;">%SYSTEMDRIVE%</span>, attempts to clear file system cache memory, and self-deletes. Then Fantasy sleeps for two minutes before overwriting the system’s <a href="https://www.virusradar.com/en/glossary/master-boot-record" target="_blank" rel="noopener">Master Boot Record</a>.</p>
<p>Fantasy then writes another batch file, <span style="font-family: 'courier new', courier, monospace;">%WINDOWS%\Temp\remover.bat</span>, that deletes the Fantasy wiper from disk and then deletes itself. Then Fantasy wiper sleeps for 30 seconds before rebooting the system with reason code <a href="https://docs.microsoft.com/en-us/windows/win32/shutdown/system-shutdown-reason-codes" target="_blank" rel="noopener">SHTDN_REASON_MAJOR_OTHER (0x00000000) -- Other issue</a>.</p>
<p>It is likely that <span style="font-family: 'courier new', courier, monospace;">%SYSTEMDRIVE%</span> recovery is possible. Victims were observed to be back up and running within a matter of hours.</p>
<h2>Attribution to Agrius</h2>
<p>Much of the code base from Apostle, initially a wiper masquerading as ransomware then updated to actual ransomware, was directly copied to Fantasy and many other functions in Fantasy were only slightly modified from Apostle, a <a href="https://www.sentinelone.com/labs/new-version-of-apostle-ransomware-reemerges-in-targeted-attack-on-higher-education/">known Agrius tool</a>. However, the overall functionality of Fantasy is that of a wiper without any attempt to masquerade as ransomware. Figure 4 shows the file deletion functions in Fantasy and Apostle, respectively. There are only a few small tweaks between the original function in Apostle and the Fantasy implementation.</p>

<p>In addition to the code reuse, we can see remnants of the Apostle execution flow in Fantasy. In the original analysis of Apostle, <a href="https://assets.sentinelone.com/sentinellabs/evol-agrius">SentinelOne</a> notes that “<em>Proper execution of the ransomware version requires supplying it with a base64 encoded argument containing an XML of an ‘RSAParameters’ object. This argument is passed on and saved as the Public Key used for the encryption process and is most likely generated on a machine owned by the threat actor.</em>” We can see in the batch file in Figure 7, which Sandals creates on remote systems to launch Fantasy, that the same base64-encoded argument containing an XML of an <span style="font-family: 'courier new', courier, monospace;">RSAParameters</span> object is passed to Fantasy at runtime. Fantasy, however, does not use this runtime argument.</p>

<h2>Conclusion</h2>
<p>Since its discovery in 2021, Agrius has been solely focused on destructive operations. To that end, Agrius operators probably executed a supply-chain attack by targeting an Israeli software company’s software updating mechanisms to deploy Fantasy, its newest wiper, to victims in Israel, Hong Kong, and South Africa. Fantasy is similar in many respects to the previous Agrius wiper, Apostle, that initially masqueraded as ransomware before being rewritten to be actual ransomware. Fantasy makes no effort to disguise itself as ransomware. Agrius operators used a new tool, Sandals, to connect remotely to systems and execute Fantasy.</p>
