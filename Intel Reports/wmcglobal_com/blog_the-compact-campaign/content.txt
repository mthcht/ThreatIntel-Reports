        "text": "SuMMARY Phishing campaigns continue to utilize the disruption of the pandemic to target victims, and a new campaign takes advantage of Zoom's rising popularity. Since December, the \"Compact\" Campaign has been targeting thousands of users by impersonating a Zoom invite and is estimated to have collected over 400,000 Outlook Web Access and Office 365 credentials. This campaign is unique in its use of trusted domains to ensure delivery of phishing emails and preventing phishing pages from being blocked. This is especially worrisome for organizations who will struggle to defend against this attack. The WMC Global Threat Intelligence team has analyzed the entire attack, including PHP code used on the website, and has recovered stolen credentials harvested by the threat actors. The analysis has linked email addresses to the responsible threat actors and found a history of these threat actors performing attacks since the start of 2020. IN DEPTH analysis Email This phishing lure delivery method was via email and appeared to be using an extensive email address spam list to target victims. It's estimated that as many as 11% of users fall for phishing campaigns and given the size of this attack it is logical that a large number of users fell victim to the phishing site. Figure 1: Initial Email Lure The team reviewed exposed credential logs and estimated that approximately each campaign collected 3,700 unique credentials. With multiple campaigns identified, the threat actor could have upwards of 400,000 unique username password values. The send field was spoofed to show it has come from Zoom but using noreply@.tld. For example, from WMC Global, we would expect to see a from address, such as: From: Zoom DELIVERY Many of the emails were sent using compromised SendGrid accounts. WMC Global worked closely with SendGrid to terminate the sending accounts which were then restored back to their rightful owners. We saw in later campaigns the threat actors pivoting away from SendGrid and moving to MailGun to send the emails. We suspect this is due to the efficiency of us sending accounts to SendGrid for termination. Phishing site The phishing site has distinctive fingerprints within the code. As a result, the WMC Global Threat Intelligence team monitored and detected the phishing sites as soon as the landing page infrastructure was available using KITIntel. Figure 2: Initial HTML Code There were two phishing landing sites in use in the December 2020 and January 2021 phishing campaigns. In December, the landing page impersonated the Outlook Web App brand to trick targets into entering their credentials. In January, the attacks changed to mimic Office 365 brand, likely to capture more employee credentials. Figure 3: OWA Lure The source code contains several layers of encoding from the escaped source. When first decoded, it reveals the headers, although there is a second encoding layer for HTML and CSS elements. The header contains key information to assist in tracking the threat further. While these campaigns always have the same double encoded content, the top header variables change as the campaigns evolve or websites are taken down or detected. Figure 4: Exfiltration Locations HTML There are three primary inputs into the code: The media host store - Which collecting images for the phishing page: media = 'hxxps:\/\/applink[.]host\/media\/mediarounds\/'; The exfiltration POST script \u2013 To POST credentials to and always stored on a different domain: b = 'hxxps:\/\/kedai27[.]com\/\/appspy\/gle21\/off2\/compact.php'; The redirect domain redirects the user once they have completed entering their credentials: rdr = 'hxxps:\/\/login.microsoftonline[.]com'; The URLs above have been defanged for safety in this blog. Figure 5: Decoded Source The team tracked and monitored the website source code for this header, extracted the indicators of compromise (IoCs), and then recovered victims from the exfiltration locations. Exfiltration &amp; Logs Exfiltration is the method attackers utilize to move compromised credentials away from the initial phishing website to locations where they can then access the credentials without revealing information that could compromise identity. This attack used numerous legitimate compromised websites to host the exfiltration code. This campaign used static .txt files for Exfiltration logs. Figure 6: Victim File The controlling threat actors periodically wiped and removed the credential file logs to prevent other threat actors or security teams from discovering the phishing attack victims. The WMC Global Threat Intelligence team monitored credential logs and identified multiple victims from large and notable companies. In mid-January 2021, the team tracked a pattern that emerged in a small number of logs submitted and concluded a credential poisoning operation was being conducted to dilute the legitimate credentials harvested by the threat actors. Some cybersecurity companies use credential poisoning to hide legitimate credentials in pools of fake logins, causing the threat actor difficulty detecting the authentic logins. Some threat actors may also poison the results of their competitors. Many threat actors are cognizant of credential poisoning operations and frequently validate credentials automatically upon ingestion flagging inauthentic credentials. Exfiltration Code This threat actor made a mistake and left a misconfigured exfiltration script, allowing WMC Global to analyze the source code of the PHP file. At a later date, the threat actors left a web shell exposed on the website enabling download of further copies of the exfiltration code. Figure 7: Webshell #1 Figure 8: Webshell #2 Reviewing the script gave the team a complete understanding of how the threat actors handle the POST data. The workflow is simple, but the code shows unusual complexities for credential phishing actors, allowing WMC Global to assert that the threat actors possess a higher technical aptitude. Initially, the POST information checks to ensure data is being POST to the page. A 404-status page can occur when the page receives a GET request or an incorrect POST format. If the data is in the correct format, the username, password, and \"desc\" inputs are assigned variables, and the username and password are then automatically validated. Figure 9: Message Write The validation occurs by the threat actors attempting to log in to the user's IMAP email account. The login attempt occurs within the code, and establishes an initial connection to a Microsoft email server on port 993. The code then uses the built-in PHP function imap_open to open a connection stream to the mailbox. If the connection succeeds, the email stream returns; otherwise, an error occurs. This technique allows the threat actors to know which compromised users' credentials are valid and have access to their email accounts externally. The result of the login test write to the log file. As a result, threat actors will use VPNs and proxies to ensure they are in the same geographical region according to their IP address when they attempt to access a user's account. Using the gathered metadata and checked credentials, the data writes to the host's log file and emails a copy to the controlling threat actors.   </div><div class="pwr-post-content">
    <span id="hs_cos_wrapper_post_body" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_rich_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="rich_text"><h3><span>SuMMARY</span></h3>
<p>Phishing campaigns continue to utilize the disruption of the&nbsp;<a href="https://www.wmcglobal.com/blog/covid-update" target="_blank" rel="noopener noreferrer" data-token-index="1" data-reactroot="" style="font-weight: bold;"><span>pandemic to target victims, </span></a>and a new campaign takes advantage of Zoom's rising popularity. Since December, the "Compact" Campaign has been targeting thousands of users by impersonating a Zoom invite and is estimated to have collected over 400,000 Outlook Web Access and Office 365 credentials. <span style="font-size: 18px;">This campaign is unique in its use of trusted domains to ensure delivery of phishing emails and preventing phishing pages from being blocked. This is especially worrisome for organizations who will struggle to defend against this attack.</span></p>
<!--more--><br>
<p>The WMC Global Threat Intelligence team has analyzed the entire attack, including PHP code used on the website, and has recovered stolen credentials harvested by the threat actors. The analysis has linked email addresses to the responsible threat actors and found a history of these threat actors performing attacks since the start of 2020.<span></span></p>
<p>&nbsp;</p>
<h3 style="font-weight: bold;"><span style="text-decoration: underline;">IN DEPTH analysis</span></h3>
<h3><span>Email <br></span></h3>
<p style="font-size: 18px;"><span>This phishing lure delivery method was via email and appeared to be using an extensive email address spam list to target victims. It's estimated that as many as 11% of users fall for phishing campaigns and given the size of this attack it is logical that a large number of users fell victim to the phishing site.</span><span></span></p>
<p style="text-align: center;"><span style="color: #c9211e;"><img src="https://www.wmcglobal.com/hs-fs/hubfs/InitalEmailLure.png?width=1352&amp;name=InitalEmailLure.png" alt="InitalEmailLure" loading="lazy" style="width: 1352px;" width="1352" srcset="https://www.wmcglobal.com/hs-fs/hubfs/InitalEmailLure.png?width=676&amp;name=InitalEmailLure.png 676w, https://www.wmcglobal.com/hs-fs/hubfs/InitalEmailLure.png?width=1352&amp;name=InitalEmailLure.png 1352w, https://www.wmcglobal.com/hs-fs/hubfs/InitalEmailLure.png?width=2028&amp;name=InitalEmailLure.png 2028w, https://www.wmcglobal.com/hs-fs/hubfs/InitalEmailLure.png?width=2704&amp;name=InitalEmailLure.png 2704w, https://www.wmcglobal.com/hs-fs/hubfs/InitalEmailLure.png?width=3380&amp;name=InitalEmailLure.png 3380w, https://www.wmcglobal.com/hs-fs/hubfs/InitalEmailLure.png?width=4056&amp;name=InitalEmailLure.png 4056w" sizes="(max-width: 1352px) 100vw, 1352px"></span><em><span style="font-size: 15px; font-weight: bold;">Figure 1: Initial Email Lure</span></em></p>
<p>The team reviewed exposed credential logs and estimated that approximately each campaign collected 3,700 unique credentials. With multiple campaigns identified, the threat actor could have upwards of 400,000 unique username password values.</p>
<p>The send field was spoofed to show it has come from Zoom but using noreply@&lt;companyName&gt;.tld.</p>
<p>For example, from WMC Global, we would expect to see a from address, such as:<span></span></p>
<p style="padding-left: 35.45pt;"><span style="color: black;">From: Zoom &lt;noreply@wmcglobal.com&gt;</span></p>
<h3>&nbsp;</h3>
<h3><span>DELIVERY<br></span></h3>
<p><span style="color: #444444;">Many of the emails were sent using compromised SendGrid accounts. WMC Global worked closely with SendGrid to terminate the sending accounts which were then restored back to their rightful owners. We saw in later campaigns the threat actors pivoting away from SendGrid and moving to MailGun to send the emails. We suspect this is due to the efficiency of us sending accounts to SendGrid for termination.</span></p>
<h3>&nbsp;</h3>
<h3><span>Phishing site</span></h3>
<p><span>The phishing site has distinctive fingerprints within the code. As a result, the WMC Global Threat Intelligence team monitored and detected the phishing sites as soon as the landing page infrastructure was available using <a href="/phishing-kit-intel" rel="noopener" target="_blank">KITIntel</a>.</span></p>
<p style="text-align: center;"><img src="https://www.wmcglobal.com/hs-fs/hubfs/InitalHTMLCode.png?width=4096&amp;name=InitalHTMLCode.png" alt="InitalHTMLCode" loading="lazy" style="width: 4096px;" width="4096" srcset="https://www.wmcglobal.com/hs-fs/hubfs/InitalHTMLCode.png?width=2048&amp;name=InitalHTMLCode.png 2048w, https://www.wmcglobal.com/hs-fs/hubfs/InitalHTMLCode.png?width=4096&amp;name=InitalHTMLCode.png 4096w, https://www.wmcglobal.com/hs-fs/hubfs/InitalHTMLCode.png?width=6144&amp;name=InitalHTMLCode.png 6144w, https://www.wmcglobal.com/hs-fs/hubfs/InitalHTMLCode.png?width=8192&amp;name=InitalHTMLCode.png 8192w, https://www.wmcglobal.com/hs-fs/hubfs/InitalHTMLCode.png?width=10240&amp;name=InitalHTMLCode.png 10240w, https://www.wmcglobal.com/hs-fs/hubfs/InitalHTMLCode.png?width=12288&amp;name=InitalHTMLCode.png 12288w" sizes="(max-width: 4096px) 100vw, 4096px"><span style="color: #c9211e;"></span><em><span style="font-size: 15px; font-weight: bold;">Figure 2: Initial HTML Code</span></em></p>
<p><span>There were two phishing landing sites in use in the December 2020 and January 2021 phishing campaigns. In December, the landing page impersonated the Outlook Web App brand to trick targets into entering their credentials. In January, the attacks changed to mimic Office 365 brand, likely to capture more employee credentials.</span></p>
<p style="text-align: center;"><img src="https://www.wmcglobal.com/hs-fs/hubfs/OWALure.png?width=1600&amp;name=OWALure.png" alt="OWALure" loading="lazy" style="width: 1600px;" width="1600" srcset="https://www.wmcglobal.com/hs-fs/hubfs/OWALure.png?width=800&amp;name=OWALure.png 800w, https://www.wmcglobal.com/hs-fs/hubfs/OWALure.png?width=1600&amp;name=OWALure.png 1600w, https://www.wmcglobal.com/hs-fs/hubfs/OWALure.png?width=2400&amp;name=OWALure.png 2400w, https://www.wmcglobal.com/hs-fs/hubfs/OWALure.png?width=3200&amp;name=OWALure.png 3200w, https://www.wmcglobal.com/hs-fs/hubfs/OWALure.png?width=4000&amp;name=OWALure.png 4000w, https://www.wmcglobal.com/hs-fs/hubfs/OWALure.png?width=4800&amp;name=OWALure.png 4800w" sizes="(max-width: 1600px) 100vw, 1600px"><span><em><span style="font-size: 15px; font-weight: bold;">Figure 3: OWA Lure</span></em></span></p>
<p>The source code contains several layers of encoding from the escaped source. When first decoded, it reveals the headers, although there is a second encoding layer for HTML and CSS elements. The header contains key information to assist in tracking the threat further.&nbsp;</p>
<p>While these campaigns always have the same double encoded content, the top header variables change as the campaigns evolve or websites are taken down or detected.<span><span style="background-color: #ffff04;"></span></span></p>
<p style="text-align: center;"><img src="https://www.wmcglobal.com/hs-fs/hubfs/ExfilLocationsHTML.png?width=2264&amp;name=ExfilLocationsHTML.png" alt="ExfilLocationsHTML" loading="lazy" style="width: 2264px;" width="2264" srcset="https://www.wmcglobal.com/hs-fs/hubfs/ExfilLocationsHTML.png?width=1132&amp;name=ExfilLocationsHTML.png 1132w, https://www.wmcglobal.com/hs-fs/hubfs/ExfilLocationsHTML.png?width=2264&amp;name=ExfilLocationsHTML.png 2264w, https://www.wmcglobal.com/hs-fs/hubfs/ExfilLocationsHTML.png?width=3396&amp;name=ExfilLocationsHTML.png 3396w, https://www.wmcglobal.com/hs-fs/hubfs/ExfilLocationsHTML.png?width=4528&amp;name=ExfilLocationsHTML.png 4528w, https://www.wmcglobal.com/hs-fs/hubfs/ExfilLocationsHTML.png?width=5660&amp;name=ExfilLocationsHTML.png 5660w, https://www.wmcglobal.com/hs-fs/hubfs/ExfilLocationsHTML.png?width=6792&amp;name=ExfilLocationsHTML.png 6792w" sizes="(max-width: 2264px) 100vw, 2264px"><span><em><span style="font-size: 15px; font-weight: bold;">Figure 4: Exfiltration Locations HTML</span></em></span></p>
<p>There are three primary inputs into the code:</p>
<ol>
<li>The media host store - Which collecting images for the phishing page:</li>
</ol>
<p style="padding-left: 40px;">media = 'hxxps://applink[.]host/media/mediarounds/';</p>
<ol start="2">
<li>The exfiltration POST script – To POST credentials to and always stored on a different domain:</li>
</ol>
<p style="padding-left: 40px;">b = 'hxxps://kedai27[.]com//appspy/gle21/off2/compact.php';</p>
<ol start="3">
<li>The redirect domain redirects the user once they have completed entering their credentials:</li>
</ol>
<p style="padding-left: 40px;">rdr = 'hxxps://login.microsoftonline[.]com';</p>
<p><em><span style="font-size: 15px;">The URLs above have been defanged for safety in this blog.</span> <br></em></p>
<p style="text-align: center;"><img src="https://www.wmcglobal.com/hs-fs/hubfs/DecodedSource.png?width=4096&amp;name=DecodedSource.png" alt="DecodedSource" loading="lazy" style="width: 4096px;" width="4096" srcset="https://www.wmcglobal.com/hs-fs/hubfs/DecodedSource.png?width=2048&amp;name=DecodedSource.png 2048w, https://www.wmcglobal.com/hs-fs/hubfs/DecodedSource.png?width=4096&amp;name=DecodedSource.png 4096w, https://www.wmcglobal.com/hs-fs/hubfs/DecodedSource.png?width=6144&amp;name=DecodedSource.png 6144w, https://www.wmcglobal.com/hs-fs/hubfs/DecodedSource.png?width=8192&amp;name=DecodedSource.png 8192w, https://www.wmcglobal.com/hs-fs/hubfs/DecodedSource.png?width=10240&amp;name=DecodedSource.png 10240w, https://www.wmcglobal.com/hs-fs/hubfs/DecodedSource.png?width=12288&amp;name=DecodedSource.png 12288w" sizes="(max-width: 4096px) 100vw, 4096px"><span><em><span style="font-size: 15px; font-weight: bold;">Figure 5: Decoded Source</span></em></span></p>
<p><span>The team tracked and monitored the website source code for this header, extracted the indicators of compromise (IoCs), and then recovered victims from the exfiltration locations.</span></p>
<h3>&nbsp;</h3>
<h3><span>Exfiltration &amp; Logs</span></h3>
<p>Exfiltration is the method attackers utilize to move compromised credentials away from the initial phishing website to locations where they can then access the credentials without revealing information that could compromise identity. This attack used numerous legitimate compromised websites to host the exfiltration code. This campaign used static .txt files for Exfiltration logs.</p>
<p style="text-align: center;"><img src="https://www.wmcglobal.com/hs-fs/hubfs/vicFile.png?width=451&amp;name=vicFile.png" alt="vicFile" loading="lazy" style="width: 451px; margin-left: auto; margin-right: auto; display: block;" width="451" srcset="https://www.wmcglobal.com/hs-fs/hubfs/vicFile.png?width=226&amp;name=vicFile.png 226w, https://www.wmcglobal.com/hs-fs/hubfs/vicFile.png?width=451&amp;name=vicFile.png 451w, https://www.wmcglobal.com/hs-fs/hubfs/vicFile.png?width=677&amp;name=vicFile.png 677w, https://www.wmcglobal.com/hs-fs/hubfs/vicFile.png?width=902&amp;name=vicFile.png 902w, https://www.wmcglobal.com/hs-fs/hubfs/vicFile.png?width=1128&amp;name=vicFile.png 1128w, https://www.wmcglobal.com/hs-fs/hubfs/vicFile.png?width=1353&amp;name=vicFile.png 1353w" sizes="(max-width: 451px) 100vw, 451px"><span><em><span style="font-size: 15px; font-weight: bold;">Figure 6: Victim File</span></em></span></p>
<p>The controlling threat actors periodically wiped and removed the credential file logs to prevent other threat actors or security teams from discovering the phishing attack victims. The WMC Global Threat Intelligence team monitored credential logs and identified multiple victims from large and notable companies.</p>
<p>In mid-January 2021, the team tracked a pattern that emerged in a small number of logs submitted&nbsp; and <span style="font-size: 18px;">concluded a credential poisoning operation was being conducted to dilute the legitimate credentials harvested by the threat actors.</span> Some cybersecurity companies use credential poisoning to hide legitimate credentials in pools of fake logins, causing the threat actor difficulty detecting the authentic logins. Some threat actors may also poison the results of their competitors. Many threat actors are cognizant of credential poisoning operations and frequently validate credentials automatically upon ingestion flagging inauthentic credentials.</p>
<h3>&nbsp;</h3>
<h3><span>Exfiltration Code</span></h3>
<p><span>This threat actor made a mistake and left a misconfigured exfiltration script, allowing WMC Global to analyze the source code of the PHP file. At a later date, the threat actors left a web shell exposed on the website enabling download of further copies of the exfiltration code.</span></p>
<p style="text-align: center;"><span style="color: #444444;"><em><span style="font-size: 15px; font-weight: bold;"><img src="https://www.wmcglobal.com/hs-fs/hubfs/webshell%231.png?width=862&amp;name=webshell%231.png" alt="webshell#1" loading="lazy" style="width: 862px;" width="862" srcset="https://www.wmcglobal.com/hs-fs/hubfs/webshell%231.png?width=431&amp;name=webshell%231.png 431w, https://www.wmcglobal.com/hs-fs/hubfs/webshell%231.png?width=862&amp;name=webshell%231.png 862w, https://www.wmcglobal.com/hs-fs/hubfs/webshell%231.png?width=1293&amp;name=webshell%231.png 1293w, https://www.wmcglobal.com/hs-fs/hubfs/webshell%231.png?width=1724&amp;name=webshell%231.png 1724w, https://www.wmcglobal.com/hs-fs/hubfs/webshell%231.png?width=2155&amp;name=webshell%231.png 2155w, https://www.wmcglobal.com/hs-fs/hubfs/webshell%231.png?width=2586&amp;name=webshell%231.png 2586w" sizes="(max-width: 862px) 100vw, 862px"></span></em></span><span style="color: #444444;"><em><span style="font-size: 15px; font-weight: bold;">Figure 7: Webshell #1</span></em></span></p>

<p>Reviewing the script gave the team a complete understanding of how the threat actors handle the POST data. The workflow is simple, but the code shows unusual complexities for credential phishing actors, allowing WMC Global to assert that the threat actors possess a higher technical aptitude. Initially, the POST information checks to ensure data is being POST to the page. <span style="font-size: 18px;">A 404-status page can occur when the page receives a GET request or an incorrect POST format.</span>&nbsp; If the data is in the correct format, the username, password, and "desc" inputs are assigned variables, and the username and password are then automatically validated.</p>

<p><span>The validation occurs by the threat actors attempting to log in to the user's IMAP email account. The result of the login test write to the log file.</span></p>

<p>The attacker uses the geoPlugin service for IP enrichment. IP enrichment is a popular tool used by threat actors to get additional metadata on an IP. The reason this data is required is due to many large companies performing IP checking for login attempts. As a result, threat actors will use VPNs and proxies to ensure they are in the same geographical region according to their IP address when they attempt to access a user's account. Using the gathered metadata and checked credentials, the data writes to the host's log file and emails a copy to the controlling threat actors. The threat actors' emails are hardcoded and are referred to in the code.</p>
<p><em>$config['adminEmail'] = 'spyblessings2021@gmail.com,postmaster@spy224.host';</em></p>
<p>Above are two email addresses linked to the threat actors: SpyBlessings2021 and postmaster. Because there is no digital footprint, WMC Global asserts these emails were registered solely to receive the phishing logs. Three of the scripts were identified as used by the threat actors but are only different in the email subjects. So far, the team has observed the following email subjects being used:</p>
<ul>
<li>"Off 365- Scrt@hed !nvesments"</li>
<li>"365-Inves!ment"</li>
<li>"Off 365- !nvesm3nt!s Cont@cts"</li>
