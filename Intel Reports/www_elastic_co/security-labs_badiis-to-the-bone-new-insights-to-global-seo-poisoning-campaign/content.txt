id="introduction" class="absolute -top-32"></span>Introduction</h2>
<p>In November 2025, Elastic Security Labs observed an intrusion affecting a multinational organization based in Southeast Asia. During the analysis of this activity, our team observed various post-compromise techniques and tooling used to deploy BADIIS malware onto a Windows web server.  These observations align with previous reporting from <a href="https://blog.talosintelligence.com/uat-8099-chinese-speaking-cybercrime-group-seo-fraud/">Cisco Talos</a> and <a href="https://www.trendmicro.com/en_us/research/25/b/chinese-speaking-group-manipulates-seo-with-badiis.html">Trend Micro</a> from last year.</p>
<p>This threat group has amassed more victims and is coordinating a large-scale SEO poisoning operation from countries across the globe. Our visibility into the campaign indicates a complex, geotargeted infrastructure designed to monetize compromised servers by redirecting users to a broad network of illicit websites such as online gambling platforms and cryptocurrency schemes.</p>
<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="key-takeaways" class="absolute -top-32"></span>Key takeaways</h3>
<ul>
<li>Elastic Security Labs observes large-scale SEO poisoning campaigns targeting IIS servers with BADIIS malware globally, impacting over 1,800 Windows servers</li>
<li>Compromised servers are monetized through a web of infrastructure used to target users with gambling advertisements and other illicit websites</li>
<li>Victim infrastructure includes governments, various corporate organizations, and educational institutions from Australia, Bangladesh, Brazil, China, India, Japan, Korea, Lithuania, Nepal, and Vietnam</li>
<li>This activity corresponds with the threat group, UAT-8099, identified by Cisco Talos last October, and is consistent with prior reporting from Trend Micro</li>
</ul>
<h2 class="font-bold text-2xl md:text-4xl relative"><span id="campaign-overview" class="absolute -top-32"></span>Campaign Overview</h2>
<p>REF4033 is a Chinese-speaking cybercrime group responsible for a massive, coordinated SEO poisoning campaign that has compromised more than 1,800 Windows web servers worldwide using a malicious IIS module called BADIIS.</p>
<p>The campaign operates through a two-phase process:</p>
<ul>
<li>First, it serves keyword-stuffed HTML to search engine crawlers to poison search results, and</li>
<li>Next, it redirects victims to a sprawling &quot;vice economy&quot; of illicit gambling platforms, pornography, and sophisticated cryptocurrency phishing sites, such as a fraudulent clone of the Upbit exchange.</li>
</ul>
<p>By deploying the BADIIS malware, a malicious IIS module that integrates directly into a web server&#x27;s request processing pipeline, the group hijacks the web servers for legitimate government, educational, and corporate domains. This high-reputation infrastructure is used to manipulate search engine rankings, thereby allowing attackers to intercept web traffic and facilitate widespread financial fraud.</p>
<h2 class="font-bold text-2xl md:text-4xl relative"><span id="intrusion-activity" class="absolute -top-32"></span>Intrusion activity</h2>
<p>In November 2025, Elastic Security Labs observed post-compromise activity from a Windows IIS server from an unknown attack vector. This threat actor moved quickly, progressing from initial access to IIS module deployment in less than 17 minutes. The initial enumeration was performed via a webshell running under the IIS worker process (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">w3wp.exe</code>). The attacker conducted initial discovery and then created a new user account.</p>
<p></p>
<p>Shortly after the account was created and added to the Administrators group, <a href="https://www.elastic.co/security">Elastic Defend</a> generated several alerts related to a newly created Windows service, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">WalletServiceInfo</code>. The service loaded an unsigned ServiceDLL  (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">C:\ProgramData\Microsoft\Windows\Ringtones\CbsMsgApi.dll</code>)  and subsequently executed direct syscalls from the module.</p>
<p></p>
<p>Next, we saw the threat actor harden their access by using a program called <a href="https://www.d99net.net/">D-Shield Firewall</a>. This software provides additional security features for IIS servers, including preventive protections and capabilities to add network restrictions. To proceed with the investigation, we used the observed imphash (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">1e4b23eee1b96b0cc705da1e7fb9e2f3</code>) of the loader (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">C:\ProgramData\Microsoft\Windows\Ringtones\CbsMsgApi.exe</code>) to obtain a loader <a href="https://www.virustotal.com/gui/file/055bdcaa0b69a1e205c931547ef863531e9fdfdaac93aaea29fb701c7b468294">sample</a> from VirusTotal for our analysis.</p>
<p></p>
<p>To collect a sample of the malicious DLL used by this loader, we performed a VirusTotal <a href="https://www.virustotal.com/gui/search/name%253ACbsMsgApi%252Edll?type=files">search</a> on the name (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">CbsMsgApi.dll</code>). We found 7 samples submitted using the same filename. The group behind this appears to have been using a similar codebase since September 2024. Most of these samples employ <a href="https://vmpsoft.com/">VMProtect</a>, a commercial code-obfuscation framework, to hinder static and dynamic analysis. Fortunately, we used an older, non-protected <a href="https://www.virustotal.com/gui/file/2340f152e8cb4cc7d5d15f384517d756a098283aef239f8cbfe3d91f8722800a">sample</a> to gain additional insight into this attack chain.</p>
<p></p>
<h2 class="font-bold text-2xl md:text-4xl relative"><span id="code-analysis---cbsmsgapiexe" class="absolute -top-32"></span>Code analysis - CbsMsgApi.exe</h2>
<p>The group employs an attack workflow that requires several files staged by the attacker to deploy the malicious IIS module. The execution chain begins with the PE executable, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">CbsMsgApi.exe</code>. This file contains Chinese Simplified strings, including the PDB string (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">C:\Users\Administrator\Desktop\替换配置文件\w3wpservice-svchost\x64\Release\CbsMsgApi.pdb</code>).</p>
<p>After launch, this program creates a Windows service, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">WalletServiceinfo,</code> which configures a ServiceDLL (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">CbsMsgApi.dll</code>) that runs under <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">svchost.exe</code>, similar to this <a href="https://www.ired.team/offensive-security/persistence/persisting-in-svchost.exe-with-a-service-dll-servicemain">persistence technique</a>.</p>
<p></p>
<p>This newly created service focuses on stealth and anti-tampering by modifying the security descriptor of the service with the following command-line:</p>
<pre><code>sc sdset &quot;WalletServiceInfo&quot; &quot;D:(D;;DCLCWPDTSD;;;IU)(D;;DCLCWPDTSD;;;SU)(D;;DCLCWPDTSD;;;BA)(A;;CCLCSWLOCRRC;;;IU)(A;;CCLCSWLOCRRC;;;SU)(A;;CCLCSWRPWPDTLOCRRC;;;SY)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;BA)S:(AU;FA;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;WD)&quot;</code></pre>
<h2 class="font-bold text-2xl md:text-4xl relative"><span id="code-analysis---cbsmsgapidll" class="absolute -top-32"></span>Code analysis - CbsMsgApi.dll</h2>
<p>The main component of this attack sequence is the ServiceDLL (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">CbsMsgApi.dll</code>). The malicious DLL stages the BADIIS IIS native modules and alters the IIS configuration to load them into the request pipeline of the DefaultAppPool.</p>
<p></p>
<p>During this attack, the threat actor stages three files masquerading within the <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">System32\drivers</code> folder:</p>
<ul>
<li><code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">C:\Windows\System32\drivers\WUDFPfprot.sys</code></li>
<li><code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">C:\Windows\System32\drivers\WppRecorderpo.sys</code></li>
<li><code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">C:\Windows\System32\drivers\WppRecorderrt.sys</code></li>
</ul>
<p>Two of these files (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">WppRecorderrt.sys</code>, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">WppRecorderpo.sys</code>) represent the malicious 32-bit / 64-bit BADIIS modules. The other file (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">WUDFPfprot.sys</code>) represents configuration elements that will be injected into the IIS’s existing configuration. <pre><code>&lt;globalModules&gt;

<p></p>
<p>Next, the malware parses the <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">DefaultAppPool.config</code> file, examining each node to update the <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">&lt;globalModules&gt;</code> and <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">&lt;modules&gt;</code> nodes. The module will inject configuration content from the previously masqueraded file (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">WUDFPfprot.sys</code>), updating the IIS configuration via a series of append operations.</p>

<p></p>
<h2 class="font-bold text-2xl md:text-4xl relative"><span id="iis-modules-analysis-wsmres32dll--wsmres64dll" class="absolute -top-32"></span>IIS Modules Analysis (WsmRes32.dll / WsmRes64.dll)</h2>
<p>The following section will describe the functionality of BADIIS modules. These modules facilitate the conditional injection or redirection of malicious SEO content based on criteria such as the User-Agent or Referer header value. This technique ensures that malicious content remains hidden during normal use, thereby allowing the modules to remain undetected for as long as possible.</p>
<p>Upon initialization, the module downloads content from URLs defined in its configuration. These URLs are stored in an encrypted format and decrypted using the <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">SM4 algorithm</code> (a Chinese national standard block cipher) in ECB mode with the key “<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">1111111122222222”</code>. In older samples, the AES-128 ECB algorithm was used instead.</p>
<p></p>
<p>Each URL in the configuration points to a static <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">.txt</code> file that contains a second-stage resource. The list below details these source files and their specific roles:</p>

<p>These URLs point to region-specific files, prefixed with the corresponding country code. While the examples above focus on Korea (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">hxxp://kr.domain.com</code>), equivalent files exist for other regions, such as Vietnam (VN), where filenames are prefixed with <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">&quot;vn&quot;</code> rather than &quot;<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">kr&quot;</code>(<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">hxxp://vn.domain.com</code>).</p>
<p></p>
<p>The BADIIS module registers within the request processing pipeline, positioning itself as both the first and the last handler. For each request, the module verifies specific properties and selects an injection or redirection strategy based on the results. We have three types of injection:</p>

<p>To distinguish between bot and human traffic, the module checks against the following list of Referers and User-Agents.</p>
<p><strong>Referers</strong>: <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">bing</code>, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">google</code>, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">naver</code>, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">daum</code><br/>
<strong>User agents</strong>: <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">bingbot</code>, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">Googlebot</code>, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">Yeti</code>, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">Daum</code></p>
<p>The default injection strategy targets search engine crawlers accessing a legitimate page on the compromised site.</p>
<p></p>
<p>In this scenario, the SEO backlinks are retrieved from the secondary link and injected into the page to be crawled by search engine bots. The downloaded backlinks that are injected into the infected page primarily target other local pages within the domain, whereas the remainder point to pages on other infected domains. This is a key aspect of the link-farming strategy, which involves creating large networks of sites that link to one another and manipulate search rankings.</p>
<p></p>
<p>The local pages linked by the backlinks do not exist on the infected domain, so visiting them results in a 404 error. However, when the request is intercepted, the malware checks two conditions: whether the status code is not 200 or 3xx, and whether the browser&#x27;s User-Agent matches a crawler bot. If so, it downloads content from an SEO page hosted on its infrastructure (via a link in the <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">*fmldz.txt</code> file from its configuration URL) and returns a 200 response code to the bot. This target URL is built using &#x27;domain&#x27; and &#x27;uri&#x27; parameters that contain the infected domain name and the resource the crawler originally attempted to access.</p>
<p></p>
<p>Finally, if a user requests a page that does not exist and arrives with a Referer header value listed by the malware, the page is replaced by a third type of content: a landing page with a loading bar. This page uses JavaScript to redirect the user to the link contained in the <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">*fmltz.txt</code> file, which is obtained via its configuration link.</p>
<p></p>
<p></p>
<p>Optionally, if enabled, the request is executed only when the User-Agent matches a mobile phone, ensuring the server targets mobile users exclusively.</p>
<p></p>
<p>The list of mobile User-Agents is listed below:</p>

<p>At its core, the recent campaign monetizes compromised servers by redirecting users to a vast network of illicit websites. The campaign is heavily invested in the vice economy, targeting Asian audiences, such as unregulated online casinos, pornography streaming, and explicit advertisements for prostitution services.</p>
<p></p>
<p>It also poses a direct financial threat. One example was a fraudulent cryptocurrency staking platform hosted at <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">uupbit[.]top</code>, impersonating Upbit, South Korea’s largest cryptocurrency exchange.</p>
<p></p>
<p>The campaign’s targeting logic largely mirrors the compromised infrastructure&#x27;s geography, establishing a correlation between the server’s location and the user’s redirection target. For instance, compromised servers in China funnel traffic to local gambling sites, while those in South Korea redirect to the fraudulent Upbit phishing site. The exception to this pattern involved compromised infrastructure in Bangladesh, which the actors configured HTTP redirects to point to non-local, Vietnamese gambling sites.</p>

<p>Within the scope of REF4033, <strong>more than 1800 servers were impacted globally</strong>, and the campaign demonstrates a clear geographic focus on the APAC region, with China and Vietnam accounting for approximately 82% of all observed compromised servers (46.1% and 35.8%, respectively). Secondary concentrations are observed in India (3.9%), Brazil (3.8%), and South Korea (3.4%), although these represent a minority of the overall campaign footprint. Notably, approximately 30% of compromised servers reside on major cloud platforms, including Amazon Web Services, Microsoft Azure, Alibaba Cloud, and Tencent Cloud. The remaining 70% of victims are distributed across regional telecommunications providers.</p>

To proceed with the investigation, we used the observed imphash (`1e4b23eee1b96b0cc705da1e7fb9e2f3`) of the loader (`C:\\ProgramData\\Microsoft\\Windows\\Ringtones\\CbsMsgApi.exe`) to obtain a loader [sample](https://www.virustotal.com/gui/file/055bdcaa0b69a1e205c931547ef863531e9fdfdaac93aaea29fb701c7b468294) from VirusTotal for our analysis.\n\n![CbsMsgApi - imphash](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image14.png)\n\nTo collect a sample of the malicious DLL used by this loader, we performed a VirusTotal [search](https://www.virustotal.com/gui/search/name%253ACbsMsgApi%252Edll?type=files) on the name (`CbsMsgApi.dll`). We found 7 samples submitted using the same filename. The group behind this appears to have been using a similar codebase since September 2024\\. Most of these samples employ [VMProtect](https://vmpsoft.com/), a commercial code-obfuscation framework, to hinder static and dynamic analysis. Fortunately, we used an older, non-protected [sample](https://www.virustotal.com/gui/file/2340f152e8cb4cc7d5d15f384517d756a098283aef239f8cbfe3d91f8722800a) to gain additional insight into this attack chain.\n\n![CbsMsgApi.dll sample listing in VirusTotal](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image12.png)\n\n## Code analysis - CbsMsgApi.exe\n\nThe group employs an attack workflow that requires several files staged by the attacker to deploy the malicious IIS module. The execution chain begins with the PE executable, `CbsMsgApi.exe`. This file contains Chinese Simplified strings, including the PDB string (`C:\\Users\\Administrator\\Desktop\\替换配置文件\\w3wpservice-svchost\\x64\\Release\\CbsMsgApi.pdb`).\n\nAfter launch, this program creates a Windows service, `WalletServiceinfo,` which configures a ServiceDLL (`CbsMsgApi.dll`) that runs under `svchost.exe`, similar to this [persistence technique](https://www.ired.team/offensive-security/persistence/persisting-in-svchost.exe-with-a-service-dll-servicemain). \n\n![Console output from Cbs.exe](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image26.png)\n\nThis newly created service focuses on stealth and anti-tampering by modifying the security descriptor of the service with the following command-line:\n\n```\nsc sdset \"WalletServiceInfo\" \"D:(D;;DCLCWPDTSD;;;IU)(D;;DCLCWPDTSD;;;SU)(D;;DCLCWPDTSD;;;BA)(A;;CCLCSWLOCRRC;;;IU)(A;;CCLCSWLOCRRC;;;SU)(A;;CCLCSWRPWPDTLOCRRC;;;SY)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;BA)S:(AU;FA;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;WD)\"\n```\n\n## Code analysis - CbsMsgApi.dll\n\nThe main component of this attack sequence is the ServiceDLL (`CbsMsgApi.dll`). The malicious DLL stages the BADIIS IIS native modules and alters the IIS configuration to load them into the request pipeline of the DefaultAppPool. \n\n![ServiceMain functionality within CbsMsgApi.dll](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image4.png)\n\nDuring this attack, the threat actor stages three files masquerading within the `System32\\drivers` folder:\n\n- `C:\\Windows\\System32\\drivers\\WUDFPfprot.sys`  \n- `C:\\Windows\\System32\\drivers\\WppRecorderpo.sys`  \n- `C:\\Windows\\System32\\drivers\\WppRecorderrt.sys`\n\nTwo of these files (`WppRecorderrt.sys`, `WppRecorderpo.sys`) represent the malicious 32-bit / 64-bit BADIIS modules. The other file (`WUDFPfprot.sys`) represents configuration elements that will be injected into the IIS’s existing configuration. Below is an example configuration used during our analysis. Of note is the module name `WsmRes64` (more information on this DLL is detailed in the IIS Modules Analysis (`WsmRes32.dll` / `WsmRes64.dll`) section below):\n\n```\n\u003cglobalModules\u003e\n    \t\u003cadd name=\"WsmRes64\" image=\"C:\\Windows\\Microsoft.NET\\Framework\\WsmRes64.dll\" preCondition=\"bitness64\" /\u003e\n\u003c/globalModules\u003e\n\u003cmodules\u003e\n    \u003cadd name=\"WsmRes64\" preCondition=\"bitness64\" /\u003e\n\u003c/modules\u003e\n```\n\nThe malware uses the `CopyFileA` function to move the contents from the masqueraded files into the .NET directory (`C:\\Windows\\Microsoft.NET\\Framework`). \n\n![Copying the IIS module into the .NET Framework directory](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image7.png)\n\nNext, the malware parses the `DefaultAppPool.config` file, examining each node to update the `\u003cglobalModules\u003e` and `\u003cmodules\u003e` nodes. The module will inject configuration content from the previously masqueraded file (`WUDFPfprot.sys`), updating the IIS configuration via a series of append operations. \n\n![Procmon output showing DefaultAppPool modification](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image8.png)\n\nBelow is an example of the newly added global module entry that references the BADIIS DLL.\n\n![Newly added global module](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image6.png)\n\nUpon successful execution, the BADIIS module is installed on the IIS server and becomes visible as a loaded module in the `w3wp.exe` worker process.\n\n![WsmRes64.dll loaded under w3wp.exe](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image20.png)\n\n## IIS Modules Analysis (WsmRes32.dll / WsmRes64.dll)\n\nThe following section will describe the functionality of BADIIS modules. These modules facilitate the conditional injection or redirection of malicious SEO content based on criteria such as the User-Agent or Referer header value. This technique ensures that malicious content remains hidden during normal use, thereby allowing the modules to remain undetected for as long as possible.\n\nUpon initialization, the module downloads content from URLs defined in its configuration. These URLs are stored in an encrypted format and decrypted using the `SM4 algorithm` (a Chinese national standard block cipher) in ECB mode with the key “`1111111122222222”`. In older samples, the AES-128 ECB algorithm was used instead.\n\n![Configuration decryption function](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image3.png)\n\nEach URL in the configuration points to a static `.txt` file that contains a second-stage resource. The list below details these source files and their specific roles:\n\n| Example configuration URL | File Name | Content Description |\n| :---- | :---- | :---- |\n| `hxxp://kr.gotz003[.]com/krfml/krfmlip.txt` | `*fmlip.txt` | Contains a URL pointing to a fake CSS file, `google.css`, that lists subnets used for filtering requests. |\n| `hxxp://kr.gotz003[.]com/krfml/krfmltz.txt` | `*fmltz.txt` | Contains a link to the target URL used for user redirections. |\n| `hxxp://kr.gotz003[.]com/krfml/krfmllj.txt` | `*fmllj.txt` | Contains a link to the malicious SEO backlinks intended for injection. |\n| `hxxp://kr.gotz003[.]com/krfml/krfmldz.txt` | `*fmldz.txt` | Contains the link to the SEO content generator. |\n\nThese URLs point to region-specific files, prefixed with the corresponding country code. While the examples above focus on Korea (`hxxp://kr.domain.com`), equivalent files exist for other regions, such as Vietnam (VN), where filenames are prefixed with `\"vn\"` rather than \"`kr\"`(`hxxp://vn.domain.com`).\n\n![Decrypted configuration URL using the SM4 algorithm](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image3.png)\n\nThe BADIIS module registers within the request processing pipeline, positioning itself as both the first and the last handler. For each request, the module verifies specific properties and selects an injection or redirection strategy based on the results. We have three types of injection:\n\n| Source | Injection Method | Description |\n| :---- | :---- | :---- |\n| `*fmltz.txt` | Full page replacement | HTML loader with progress bar \\+ auto-redirect \\+ Google Analytics tracking |\n| `*fmldz.txt` | Full page replacement | Direct link to SEO content, built with `index.php?domain=\u003chost\u003e\u0026uri=\u003coriginal_link\u003e` |\n| `*fmllj.txt` | Inline injection | SEO backlinks injected after `\u003cbody\u003e` or `\u003chtml\u003e` tag in existing response |\n\nTo distinguish between bot and human traffic, the module checks against the following list of Referers and User-Agents.\n\n**Referers**: `bing`, `google`, `naver`, `daum`  \n**User agents**: `bingbot`, `Googlebot`, `Yeti`, `Daum`\n\nThe default injection strategy targets search engine crawlers accessing a legitimate page on the compromised site.\n\n![Download and injection of SEO backlinks](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image19.png)\n\nIn this scenario, the SEO backlinks are retrieved from the secondary link and injected into the page to be crawled by search engine bots. The downloaded backlinks that are injected into the infected page primarily target other local pages within the domain, whereas the remainder point to pages on other infected domains. This is a key aspect of the link-farming strategy, which involves creating large networks of sites that link to one another and manipulate search rankings.\n\n![Inlined SEO backlinks on the infected page](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image22.png)\n\nThe local pages linked by the backlinks do not exist on the infected domain, so visiting them results in a 404 error. However, when the request is intercepted, the malware checks two conditions: whether the status code is not 200 or 3xx, and whether the browser's User-Agent matches a crawler bot. If so, it downloads content from an SEO page hosted on its infrastructure (via a link in the `*fmldz.txt` file from its configuration URL) and returns a 200 response code to the bot. This target URL is built using 'domain' and 'uri' parameters that contain the infected domain name and the resource the crawler originally attempted to access.\n\n![404 page replacement strategy with User-Agent](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image1.png)\n\nFinally, if a user requests a page that does not exist and arrives with a Referer header value listed by the malware, the page is replaced by a third type of content: a landing page with a loading bar. This page uses JavaScript to redirect the user to the link contained in the `*fmltz.txt` file, which is obtained via its configuration link.\n\n![404 landing/redirection page with Referer](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image23.png)\n\n![404 redirection target with Referer](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image5.png)\n\nOptionally, if enabled, the request is executed only when the User-Agent matches a mobile phone, ensuring the server targets mobile users exclusively.\n\n![Requests are optionally filtered on server IP and mobile’s User-Agents](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image17.png)\n\nThe list of mobile User-Agents is listed below:\n\n**Devices:** `iPhone`, `iPad`, `iPod`, `iOS`, `Android`, `uc (UC Browser)`, `BlackBerry`, `HUAWEI`\n\nIf the option is enabled and the infected server's IP address matches the subnet list in the `google.css` file downloaded from the `*fmlip.txt` file, the server serves the standard SEO content instead of the landing/redirection page. \n\n![Subnet filter list](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image21.png)\n\nThe landing page includes JavaScript code containing an analytics tag—either Google Analytics or Baidu Tongji, depending on the target region—to monitor redirections. While the exact reason for using server IP filtering to restrict traffic remains unclear, we speculate that it is linked to these analytics and implemented for SEO purposes.\n\n![Google Analytics in landing pages](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image24.png)\n\nWe identified the following Google tags across the campaign:\n\n- `G-2FK43E86ZM`  \n- `G-R0KHSLRZ7N`\n\nAs well as a Baidu Tongji tag:\n\n- `B59ff1638e92ab1127b7bc76c7922245`\n\n## Campaign Analysis\n\nBased on similarity in URL patterns (`\u003ccountry_code\u003efml__.txt`, `\u003ccountry_code\u003efml/index.php`), we discovered an older campaign dating back to mid-2023 that was using the following domains as the configuration server.\n\n- `tz123[.]app`  \n- `tz789[.]app`\n\n`tz123[.]app` was disclosed in a Trend Micro BADIIS [campaign summary](https://www.trendmicro.com/en_us/research/25/b/chinese-speaking-group-manipulates-seo-with-badiis.html) [IOC list](https://documents.trendmicro.com/assets/txt/badiis-IOCspbJhGdi.txt), published in 2024 . Based on the first submission dates on VT for samples named `ul_cache.dll` and communicating with `hxxp://tz789[.]app/brfmljs[.]txt`, some are also submitted under the filename `WsmRes64.dll`. This naming convention is consistent with the BADIIS loader component analyzed in the prior section. The earliest [sample](https://www.virustotal.com/gui/file/ec5a69bc62e66a2677ab91da16c247b780d109baa24c46b4b99233768455f558) we discovered on VT was first submitted on 2023-12-12.\n\nFor REF4033, the infrastructure is split between two primary configuration servers.\n\n- Recent campaigns (`gotz003[.]com`): Currently serves as the primary configuration hub. Further analysis has identified 5 active subdomains categorized by country codes:  \n  - `kr.gotz003[.]com` (South Korea)  \n  - `vn.gotz003[.]com` (Vietnam)  \n  - `cn.gotz003[.]com` (China)  \n  - `cnse.gotz003[.]com` (China)  \n  - `bd.gotz003[.]com` (Bangladesh)  \n- Legacy infrastructure (`jbtz003[.]com`): Used in older campaign iterations, though several subdomains remain operational:  \n- `br.jbtz003[.]com` (Brazil)  \n- `vn.jbtz003[.]com` (Vietnam)  \n- `vnbtc.jbtz003[.]com` (Vietnam)  \n- `in.jbtz003[.]com` (India)  \n- `cn.jbtz003[.]com` (China)  \n- `jp.jbtz003[.]com` (Japan)  \n- `pk.jbtz003[.]com` (Pakistan)\n\nAt its core, the recent campaign monetizes compromised servers by redirecting users to a vast network of illicit websites. The campaign is heavily invested in the vice economy, targeting Asian audiences, such as unregulated online casinos, pornography streaming, and explicit advertisements for prostitution services.\n\n![Redirected sites for users](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image10.png)\n\nIt also poses a direct financial threat. One example was a fraudulent cryptocurrency staking platform hosted at `uupbit[.]top`, impersonating Upbit, South Korea’s largest cryptocurrency exchange. \n\n![www.uupbit[.]top - from urlscan.io](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image11.png)\n\nThe campaign’s targeting logic largely mirrors the compromised infrastructure's geography, establishing a correlation between the server’s location and the user’s redirection target. For instance, compromised servers in China funnel traffic to local gambling sites, while those in South Korea redirect to the fraudulent Upbit phishing site. The exception to this pattern involved compromised infrastructure in Bangladesh, which the actors configured HTTP redirects to point to non-local, Vietnamese gambling sites.\n\n![User redirection link - infected servers located in Bangladesh](/assets/images/badiis-to-the-bone-new-insights-to-global-seo-poisoning-campaign/image16.png)\n\nWe have observed several different redirection loading pages throughout the clusters.
