                    <div class="article-body">
    <p>In this blogpost, ESET researchers provide an analysis of Spellbinder, a lateral movement tool for performing adversary-in-the-middle attacks, used by the China-aligned threat actor that we have named TheWizards. Spellbinder enables adversary-in-the-middle (AitM) attacks, through IPv6 stateless address autoconfiguration (SLAAC) spoofing, to move laterally in the compromised network, intercepting packets and redirecting the traffic of legitimate Chinese software so that it downloads malicious updates from a server controlled by the attackers.&nbsp;</p>
<blockquote>
<div><strong>Key points in this blogpost:</strong></div>
<ul>
<li>We discovered a malicious downloader being deployed, by legitimate Chinese software update mechanisms, onto victims&rsquo; machines.</li>
<li>The downloader seeks to deploy a modular backdoor that we have named WizardNet.</li>
<li>We analyzed Spellbinder: the tool the attackers use to conduct local adversary-in-the-middle attacks and to redirect traffic to an attacker-controlled server to deliver the group&rsquo;s signature backdoor WizardNet.</li>
<li>We provide details abouts links between TheWizards and the Chinese company Dianke Network Security Technology, also known as UPSEC.</li>
</ul>
</blockquote>
<h2>Overview</h2>
<p>In 2022, we noticed that a suspicious DLL had been downloaded by the popular Chinese <a href="https://en.wikipedia.org/wiki/Chinese_input_methods_for_computers" target="_blank" rel="noopener">input method</a> software application known as <a href="https://en.wikipedia.org/wiki/Sogou_Pinyin" target="_blank" rel="noopener">Sogou Pinyin</a>. The DLL, named after a legitimate component of that software, was a dropper for a downloader that retrieved an encrypted blob from a remote server. The blob contained shellcode that loads the backdoor we have named WizardNet.</p>
<p>Our research led to the discovery of a tool, used by the attackers, that is designed to perform adversary-in-the-middle attacks using IPv6 SLAAC spoofing to intercept and reply to packets in a network, allowing the attackers to redirect traffic and serve malicious updates targeting legitimate Chinese software.</p>
<h2>Victimology</h2>
<p>TheWizards has been constantly active since at least 2022 up to the time of writing. According to ESET telemetry, TheWizards targets individuals, gambling companies, and unknown entities in the Philippines, Cambodia, the United Arab Emirates, mainland China, and Hong Kong. Its geographical distribution is shown in Figure 1.</p>
<figure class="image"><img title="Figure 1. <h2>Spellbinder, the IPv6 AitM tool</h2>
<p>We initially discovered and analyzed this tool in 2022, and observed a new version with a few changes that was deployed to compromised machines in 2023 and 2024. Once the attackers gain access to a machine in a targeted network, they deploy an archive called <span style="font-family: courier new, courier, monospace;">AVGApplicationFrameHostS.zip</span>, and extract its components into <span style="font-family: courier new, courier, monospace; white-space: nowrap;">%PROGRAMFILES%\AVG Technologies</span>. The files include:</p>
<ul>
<li><span style="font-family: courier new, courier, monospace;">AVGApplicationFrameHost.exe</span></li>
<li><span style="font-family: courier new, courier, monospace;">wsc.dll</span></li>
<li><span style="font-family: courier new, courier, monospace;">log.dat</span></li>
<li><span style="font-family: courier new, courier, monospace;">winpcap.exe</span></li>
</ul>
<p>Next, the attackers install winpcap.exe and run <span style="font-family: courier new, courier, monospace;">AVGApplicationFrameHost.exe</span>. The latter, originally named <span style="font-family: courier new, courier, monospace;">wsc_proxy.exe</span>, is a legitimate software component from AVG that is abused to side-load <span style="font-family: courier new, courier, monospace;">wsc.dll</span>; this DLL simply reads the shellcode from the file <span style="font-family: courier new, courier, monospace;">log.dat</span> and executes it in memory. The shellcode decompresses and loads Spellbinder in memory.</p>
<p>Spellbinder uses the <a href="https://www.winpcap.org/">WinPcap</a> library to capture packets and to reply to packets when needed. The first task is to select or find an adapter with which to perform the packet capture. The code uses the WinPcap API <span style="font-family: courier new, courier, monospace;">pcap_findalldevs</span> to get all available adapter devices. The devices are itemized in a numbered list for the attacker. Optionally, Spellbinder accepts, as an argument, an index that can be used to pick one adapter from this list. If a device is not supplied, Spellbinder uses the Windows APIs <span style="font-family: courier new, courier, monospace;">GetBestInterface</span> and <span style="font-family: courier new, courier, monospace;">GetAdapterInfo</span> to find a suitable adapter, and prints its information on screen.</p>
<p>Figure 2 shows the output of Spellbinder when no item number is supplied. In that case, the tool finds the most suitable adapter by itself.</p>
<figure class="image"><img title="Figure 2. Spellbinder&rsquo;s output during its initialization phase" src="https://web-assets.esetstatic.com/wls/2025/04-25/thewizards/figure-2.png" alt="Figure 2. Spellbinder&rsquo;s output during its initialization phase" width="" height="" />
<figcaption><em>Figure 2. Spellbinder&rsquo;s output during its initialization phase</em></figcaption>
</figure>
<p>As shown in Figure 3, once an adapter is found, Spellbinder uses the <span style="font-family: courier new, courier, monospace;">WinPcap pcap_open_live</span> API to start capturing packets, and creates two threads: one to send ICMPv6 Router Advertisement packets (explained in the next section), and a thread to monitor network changes. The WinPcap <span style="font-family: courier new, courier, monospace;">pcap_loop</span> API does the job of invoking a callback function from Spellbinder every time a new packet is captured.</p>

<h3>Router Advertisement thread</h3>
<p>This attack vector was <a href="https://datatracker.ietf.org/doc/html/draft-chown-v6ops-rogue-ra-02" target="_blank" rel="noopener">discussed by the IETF</a> as early as 2008 and is caused by a commonly overlooked network misconfiguration of IPv4 and IPv6 coexistence. It was then thoroughly <a href="https://wirewatcher.wordpress.com/2011/04/04/the-slaac-attack-using-ipv6-as-a-weapon-against-ipv4/" target="_blank" rel="noopener">detailed</a> in 2011 by Alec Waters, who dubbed it the SLAAC Attack. It takes advantage of IPv6&rsquo;s Network Discovery Protocol in which ICMPv6 Router Advertisement (RA) messages advertise that an IPv6-capable router is present in the network so that hosts that support IPv6, or are soliciting an IPv6-capable router, can adopt the advertising device as their default gateway.</p>
<p>Spellbinder sends a multicast RA packet every 200 ms to <span style="font-family: courier new, courier, monospace;">ff02::1</span> (&ldquo;all nodes&rdquo;); Windows machines in the network with IPv6 enabled will autoconfigure via <a href="https://www.rfc-editor.org/rfc/rfc4862" target="_blank" rel="noopener">stateless address autoconfiguration</a> (SLAAC) using information provided in the RA message, and begin sending IPv6 traffic to the machine running Spellbinder, where packets will be intercepted, analyzed, and replied to where applicable. Figure 4 illustrates the first stage of the attack.</p>
<figure class="image"><img title="Figure 4. Illustration of the SLAAC attack carried out by Spellbinder" src="https://web-assets.esetstatic.com/wls/2025/04-25/thewizards/figure-4.png" alt="Figure 4. Illustration of the SLAAC attack carried out by Spellbinder" width="" height="" />
<figcaption><em>Figure 4. Illustration of the SLAAC attack carried out by Spellbinder</em></figcaption>
</figure>
<p>The RA packet built by Spellbinder consists of four major parts:</p>
<ul>
<li>RA Flags: has the &ldquo;managed address configuration&rdquo; flag set to <span style="font-family: courier new, courier, monospace;">0</span>, indicating to hosts that SLAAC should be used.</li>
<li>The prefix option that indicates to the host to use the <span style="font-family: courier new, courier, monospace;">2001:db8::/64</span> prefix to generate its IPv6 address, which is not an internet-routable subnet, but rather a subnet reserved for documentation.</li>
<li>The recursive DNS server (RDNSS) option that provides the host with the addresses of two DNS servers: <span style="font-family: courier new, courier, monospace;">240e:56:4000:8000::11</span> and <span style="font-family: courier new, courier, monospace;">240e:56:4000:8000::22</span>. Both addresses are part of AS4134 from China Telecom Backbone, but do not seem to be responding to DNS requests from the Internet. We have not found any evidence indicating that either is a legitimate DNS server.</li>
<li>The source link-layer option, which provides the MAC address of the machine running Spellbinder as the router to use in the local network segment.</li>
</ul>

<h2>Hijacking of updates</h2>
<p>For this blogpost we have focused on one of the latest cases in 2024, in which the update of Tencent QQ software was hijacked. The malicious server that issues the update instructions was still active at the time of writing. Figure 11 illustrates the observed chain.</p>
<figure class="image"><img title="Figure 11. Compromise chain" src="https://web-assets.esetstatic.com/wls/2025/04-25/thewizards/figure-11.png" alt="Figure 11. Compromise chain" width="" height="" />

<p>The legitimate software component <span style="font-family: courier new, courier, monospace; white-space: nowrap;">QQ.exe</span> sends an HTTP request to <span style="font-family: courier new, courier, monospace; white-space: nowrap;">update.browser.qq.com</span>. The Spellbinder tool intercepts the DNS query for that domain name and issues a DNS answer with the IP address of an attacker-controlled server used for hijacking, for example, <span style="font-family: courier new, courier, monospace;">43.155.62[.]54</span>, that at the time of writing was still serving malicious updates.</p>

<h2>Execution chain after a successful AitM attack</h2>
<p>The execution of the malware on a compromised machine begins with the malicious <span style="font-family: courier new, courier, monospace;">minibrowser_shell.dll</span> downloader. This DLL has three export functions and the execution of any of them triggers its main functionality but only if the name of the current process contains <span style="font-family: courier new, courier, monospace;">QQ</span> &mdash; for example, <span style="font-family: courier new, courier, monospace;">QQ.exe</span> would be valid.</p>
<p>It uses the WinSock API to connect via TCP to an attacker-controlled server, from where it obtains an encrypted blob containing position-independent loader code and the WizardNet backdoor.</p>
<h3>Loader shellcode</h3>
<p>The loader begins by attempting to use a well-known <a href="https://rastamouse.me/memory-patching-amsi-bypass/">bypass for AMSI</a> that patches the first bytes of the <span style="font-family: courier new, courier, monospace;">AmsiScanBuffer</span> function to return an error code, thus bypassing the mechanism that scans memory for malicious artifacts. Then, it patches the entry point of the <span style="font-family: courier new, courier, monospace;">EtwEventWrite</span> function with a <span style="font-family: courier new, courier, monospace;">RETN 0x14</span> instruction; this has the effect of <a href="https://www.bordergate.co.uk/unhooking-event-tracing-for-windows/">disabling Event Logging</a>.</p>
<p>To execute the payload in memory, the loader initializes the .NET runtime, as shown in Figure 12, using the <span style="font-family: courier new, courier, monospace;">ICLRMetaHost</span>, <span style="font-family: courier new, courier, monospace;">ICLRRuntimeInfo</span>, and <span style="font-family: courier new, courier, monospace;">ICorRuntimeHost</span> interfaces, requiring a runtime version of either v2.0.50727 or v4.0.30319.</p>
<figure class="image"><img title="Figure 12. Decompiled code that initializes the .NET runtime to execute WizardNet in memory" src="https://web-assets.esetstatic.com/wls/2025/04-25/thewizards/figure-12.png" alt="Figure 12. Decompiled code that initializes the .NET runtime to execute WizardNet in memory" width="" height="" />
<figcaption><em>Figure 12. Decompiled code that initializes the .NET runtime to execute WizardNet in memory</em></figcaption>
</figure>
<p>Then the payload is decrypted using a simple combination of <span style="font-family: courier new, courier, monospace;">ADD</span> and <span style="font-family: courier new, courier, monospace;">XOR</span>. The payload is loaded into memory using the .NET runtime, then its entry point is executed.</p>
<h3>WizardNet</h3>
<p>The final payload is a backdoor that we named WizardNet &ndash; a modular implant that connects to a remote controller to receive and execute .NET modules on the compromised machine. During its initialization it creates a mutex named <span style="font-family: courier new, courier, monospace;">Global\&lt;MD5(computer_name)&gt;</span> and reads shellcode from a file called <span style="font-family: courier new, courier, monospace;">ppxml.db</span> in the current working directory or the value from the key <span style="font-family: courier new, courier, monospace;">HKCU\000000</span>, and attempts to inject it into a new process of <span style="font-family: courier new, courier, monospace;">explorer.exe</span> or <span style="font-family: courier new, courier, monospace; white-space: nowrap;">%ProgramFiles%\Windows Photo Viewer\ImagingDevices.exe</span>.</p>
<p>The last step of the initialization phase is to create a unique identifier for the computer, referred to as the <span style="font-family: courier new, courier, monospace;">SessionKey</span>. It is the result of the MD5 hash of the computer name concatenated with the installation time of the backdoor and the serial number of the disk drive, with each hex-encoded byte of the hash value separated by <span style="font-family: courier new, courier, monospace;">@</span>. The <span style="font-family: courier new, courier, monospace;">SessionKey</span> is stored under the registry path <span style="font-family: courier new, courier, monospace; white-space: nowrap;">HKCU\Software\&lt;MD5(computer_name)&gt;\&lt;MD5(computer_name)&gt;mid</span>.</p>
<p>Depending on its configuration, WizardNet can then create a TCP or UDP socket to communicate with its C&amp;C server, and the messages exchanged are padded using the PKCS7 algorithm and encrypted with AES-ECB; the <span style="font-family: courier new, courier, monospace;">SessionKey</span> is used as the key for encryption and decryption and the IV is randomly generated for each packet and placed before the encrypted data.</p>

<h2>Links to Sichuan Dianke Network Security</h2>
<p>In December 2024, Trend Micro researchers published an <a href="https://www.trendmicro.com/en_gb/research/24/l/earth-minotaur.html" target="_blank" rel="noopener">analysis</a> of the <a href="https://citizenlab.ca/2019/09/poison-carp-tibetan-groups-targeted-with-1-click-mobile-exploits/" target="_blank" rel="noopener">MOONSHINE</a> exploit kit and the DarkNimbus malware for Android devices. The toolset is used by a group Trend Micro tracks as Earth Minotaur and that targets primarily Tibetan and Uyghur communities. In January 2025, <a href="https://www.intelligenceonline.com/surveillance--interception/2025/01/29/chinese-firm-behind-hacking-operations-against-uyghurs-and-tibetans-unveiled,110368855-evg" target="_blank" rel="noopener">Intelligence Online</a> identified the Chinese company Sichuan Dianke Network Security Technology Co., Ltd., also known as UPSEC (Figure 13), as the supplier of the DarkNimbus malware.</p>
<figure class="image"><img title="Figure 13. UPSEC&rsquo;s website" src="https://web-assets.esetstatic.com/wls/2025/04-25/thewizards/figure-13.png" alt="Figure 13. UPSEC&rsquo;s website" width="" height="" />
<figcaption><em>Figure 13. UPSEC&rsquo;s website</em></figcaption>
</figure>
<p>ESET tracks the malware that Trend Micro named DarkNimbus as DarkNights (both for Windows and Android); amusingly, Trend Micro named the malware after the string <span style="font-family: courier new, courier, monospace; white-space: nowrap;">DKNS</span> present in the malware&rsquo;s function names, and we did the same (<strong>D</strong>ar<strong>kN</strong>ight<strong>s</strong>) when we discovered the malware. In April 2025, NCSC UK published an <a href="https://www.ncsc.gov.uk/news/advisory-badbazaar-moonshine">advisory</a> about the BADBAZAAR malware and MOONSHINE, also mentioning UPSEC in relation to Trend Micro&rsquo;s research on Earth Minotaur.</p>
<p>While TheWizards uses a different backdoor for Windows (WizardNet), the hijacking server is configured to serve DarkNights to updating applications running on Android devices. While we have not seen any victims in ESET telemetry, we managed to obtain a malicious update instruction for the Android version of Tencent QQ:</p>

<p>The file <span style="font-family: courier new, courier, monospace;">plugin-audiofirstpiece.ml</span> is a ZIP archive that only contains a <span style="font-family: courier new, courier, monospace;">classes.dex</span> file, which is DarkNights for Android. This indicates that Dianke Network Security is a digital quartermaster to TheWizards APT group.</p>
<p>ESET continues tracking TheWizards independently of Earth Minotaur. While both threat actors use DarkNights/DarkNimbus, according to ESET telemetry TheWizards has focused on different targets and uses infrastructure and additional tools (for example, Spellbinder and WizardNet) not observed to be used by Earth Minotaur.</p>
<h2>Conclusion</h2>
<p>In 2022, we discovered the activity of a China-aligned APT group that we have named TheWizards. We analyzed the custom malware and tools developed and used by TheWizards: the IPv6 AitM tool we&rsquo;ve named Spellbinder, which allows the attackers to redirect the update protocols of legitimate Chinese software to malicious servers, where the software is tricked into downloading and executing fake updates on victims&rsquo; machines, and the malicious components that launch the backdoor that we have named WizardNet.</p>
