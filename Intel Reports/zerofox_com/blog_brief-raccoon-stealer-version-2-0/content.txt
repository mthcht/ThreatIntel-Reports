has observed the following information as of June 28, 2022, and has released the following.</p><h2 id="executive-summary" class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left">Executive Summary</h2><p class="  ">On June 4, 2022, ZeroFox Intelligence discovered a then-unknown information stealer being distributed by ProCrackerz, a website distributing fake software cracks and key generators (keygens). The earliest known instance of this information stealer observed by ZeroFox Intelligence was a sample uploaded to VirusTotal on April 19, 2022. Twitter user @James_inthe_box <a href="https://twitter.com/James_inthe_box/status/1534593736356663296" target="_blank" rel="noreferrer noopener">suggested</a> the name “Recordbreaker” for it based on the use of “record” as the User-Agent string in each sample. In May 2022, logs for sale with “Raccoon Stealer V2.0” branding were <a href="https://medium.com/s2wblog/raccoon-stealer-is-back-with-a-new-version-5f436e04b20d" target="_blank" rel="noreferrer noopener">discovered</a> that matched what ZeroFox Intelligence was observing with Recordbreaker. Due to this and multiple other private confirmations, ZeroFox Intelligence asserts with MEDIUM confidence that Raccoon Stealer has returned and that Recordbreaker is actually Raccoon Stealer version 2.0.</p><h2 id="details" class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left">Details</h2><p class="  ">Raccoon Stealer version 2.0 is capable of targeting Chromium and Mozilla-based browsers by looking for well-known file names in specific directories. For Chromium-based browsers, threat actors using Raccoon Stealer 2.0 have the ability to specify a list of Chrome extension IDs and associated files as well. In ZeroFox Intelligence’s observations, these consisted entirely of cryptocurrency extensions. Raccoon Stealer 2.0 attempts to collect credentials, cookies, autofill data, credit cards, and data associated with specified Chrome extensions. For Mozilla browsers such as Firefox, only credentials, cookies, and autofill data were targeted. Other applications like Telegram and specific cryptocurrency applications can be targeted as well. To ensure that all cryptocurrency wallets are collected, a separate function exists solely to collect “wallet.dat” files. For any applications or files without specific support, a generic “grbr_” function exists to allow actors to specify files by path and a name or pattern.</p><h2 id="technical-analysis" class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left">Technical Analysis</h2><p class="  ">ZeroFox Intelligence first discovered Raccoon Stealer version 2.0 disguised as a crack for Microsoft Office on the ProCrackerz website. Clicking on any of the download links redirected the viewer through various advertisements and click trackers until they were eventually shown a set of directions and a Discord CDN link hosting the fake crack. The download links on ProCrackerz change regularly as the Discord links are removed.</p><figure></figure><figure></figure><p class="  ">The compressed files are small in size but inflate to hundreds of megabytes when decompressed. This is due to the samples being padded with large amounts of repeating bytes.</p><p class="  ">Samples ZeroFox Intelligence observed distributed in this way were obfuscated or packed; the unique string “edinayarossiya” was visible and used to pivot to other samples uploaded to VirusTotal. This allowed ZeroFox Intelligence to download much smaller (~56KB) unprotected samples and greatly sped up our analysis. Translated, Edinaya Rossiya means “United Russia,” which is currently the <a href="https://en.wikipedia.org/wiki/United_Russia" target="_blank" rel="noreferrer noopener">largest political party</a> in Russia. Later analysis of the string decryption routine determined this is an encryption key for the protected strings used by the stealer.</p><p class="  ">Raccoon Stealer version 2.0 begins by importing all of the Windows API calls it needs (and some it does not). Importing API calls at runtime is a common tactic used by malware to avoid adding them to the import table to be used as a signature.</p><figure></figure><p class="  ">Afterwards, all protected strings are base64 decoded and RC4 decrypted as shown in <strong>Figure 4</strong>.</p><figure></figure><p class="  ">The RC4 key “edinayarossiya” was consistent across most samples, though some also used “credit19” instead. ZeroFox Intelligence is currently unsure if this is specified by each actor deploying the stealer or if this is decided by the authors for each build.</p><p class="  ">Unlike the RC4 key used to decrypt strings, the RC4 key used to decrypt command and control (C2) servers is a fixed length and changes with every sample. Aside from this, C2 servers are protected in much the same way the other strings are. Up to five C2s can be configured per sample, with each C2 slot hardcoded to be 65 bytes long. Addresses that are shorter than 65 bytes after being encrypted and base64 encoded are padded with spaces.</p><figure></figure><p class="  ">The locale on the victim’s machine is checked against two locales that can be hard-coded in the binary. ZeroFox did observe a check for a “ru” locale, but the language check does not affect the execution in any way. A second locale was not configured in the samples we observed.</p><figure></figure><p class="  ">Raccoon Stealer 2.0 also ensures that only one instance is running at a time by checking and creating a mutex. ZeroFox Intelligence observed this to be “8724643052” with every sample obtained. If it cannot open a handle to the mutex, Raccoon Stealer 2.0 will exit with Error Code 2.</p><figure></figure><p class="  ">The victim’s security identifier (SID) is checked against the value “S-1-5-18” to determine if the process happens to be running as the SYSTEM or LOCAL SYSTEM user. If so, Raccoon Stealer 2.0 will enumerate the list of running processes on the infected machine.</p><figure></figure><p class="  ">The first real action Raccoon Stealer 2.0 takes is to get the machine GUID and username, which are then sent as an HTTP POST request to the C2. As seen in <strong>Figure 9</strong> below, the GUID and username are sent together in the URL parameter “machineId” separated by a pipe character. The “configId” parameter shown is the RC4 key used to decrypt C2 addresses.</p><figure></figure><p class="  ">If the C2 is still available, the server will respond with a simple, newline-separated configuration. If no C2 is available, Raccoon Stealer 2.0 simply exits.</p><figure></figure><p class="  ">There are currently nine options that can be processed from the settings shown in <strong>Figure 10</strong>. A sample configuration returned by one of the C2 servers can be found <a href="https://gist.github.com/FirehaK/bb4f0c17c77d284bd5ad54187bbd199a" target="_blank" rel="noreferrer noopener">here</a>.</p><div class="rich-text max-w-full overflow-x-auto"><figure class="wp-block-table is-style-stripes"><table><thead><tr><th scope="col">Option</th><th class="has-text-align-left" data-align="left" scope="col">Description</th></tr></thead><tbody><tr><td>ews_</td><td class="has-text-align-left" data-align="left">Targeted Chrome browser extensions</td></tr><tr><td>grbr_</td><td class="has-text-align-left" data-align="left">Targeted files to steal</td></tr><tr><td>ldr_</td><td class="has-text-align-left" data-align="left">A command, DLL, or executable to run</td></tr><tr><td>libs_</td><td class="has-text-align-left" data-align="left">DLLs to download</td></tr><tr><td>scrnsht_</td><td class="has-text-align-left" data-align="left">Screenshot file name</td></tr><tr><td>sstmnfo_</td><td class="has-text-align-left" data-align="left">Send system information to the C2 with this file name and add this template text</td></tr><tr><td>tlgrm_</td><td class="has-text-align-left" data-align="left">Telegram-specific files and folders to target</td></tr><tr><td>token</td><td class="has-text-align-left" data-align="left">URL path to POST stolen data</td></tr><tr><td>wlts_</td><td class="has-text-align-left" data-align="left">Cryptocurrency wallets and associated files and folders to target</td></tr></tbody></table></figure></div><p class="  ">DLL files downloaded using the “libs_” option are saved to the AppData\LocalLow directory. Raccoon Stealer 2.0 attempts to add this directory to the PATH environment variable but does not verify if it was successful. In our observations, this actually failed, and the sample continued to run without issue.</p><p class="  ">The following system information is collected during a run:</p><ul class=" rich-text space-y-2"><li>User locale</li><li>System time zone</li><li>Operating system</li><li>System architecture (32-bit or 64-bit)</li><li>CPU core count</li><li>Installed RAM</li><li>Screen resolution</li><li>All display devices (GPUs)</li><li>Installed software and versions</li></ul><p class="  ">Once each of these functions has run, another POST request is made to /&lt;token&gt;.</p><figure></figure><p class="  ">Rather than look for specific browsers, Raccoon Stealer version 2.0 targets any Chromium or Mozilla-based browsers by the name of the directories in which each respective browser stores its data. For Chromium, this is “User Data” while Mozilla/Gecko uses “Profiles.”</p><figure></figure><p class="  ">In order for Raccoon Stealer 2.0 to be able to read the data threat actors are interested in, it must load the DLLs from the “libs_” options earlier.</p><figure></figure><p class="  ">Data targeted from Chromium-based browsers includes:</p><ul class=" rich-text space-y-2"><li>Credentials</li><li>Cookies</li><li>Autofill data</li><li>Credit cards</li><li>Extensions listed in the configuration retrieved from the C2</li></ul><p class="  ">Data targeted from Mozilla/Gecko-based browsers includes:</p><ul class=" rich-text space-y-2"><li>Credentials</li><li>Cookies</li><li>Autofill data</li></ul><p class="  ">Although the “ews_” option is not necessarily limited to only cryptocurrency-related browser extensions, ZeroFox Intelligence has only observed this to be the use case. <strong>Figure 14</strong> below shows two more functions dedicated to stealing cryptocurrency wallets. The first, “wlts_”, exfiltrates files based on the configuration option of the same name. Other cryptocurrency wallets may still be stolen by the next function, which looks for “wallet.dat” files.</p><figure></figure><p class="  ">The “grbr_” function uses directory paths, file names or patterns, and other options such as file size specified in the configuration to decide which files it should exfiltrate.</p><p class="  ">The “tlgrm_” function is similar to “grbr_” but has fewer options. It is meant to target&nbsp; Telegram data, though the same functionality could have been achieved with the generic file grabber.</p><p class="  ">Taking a screenshot is separated into two functions. The first, “scrnsht_” checks to see if the configuration wants it to take one, and the second function actually takes and sends the screenshot.</p><p class="  ">Finally, the “ldr_” function is capable of allowing Raccoon Stealer version 2.0 to act as a loader for secondary payloads or execute commands. Each “ldr_” option contains multiple parts. It could contain a command to execute or the URL of a file to download, and if a URL is given a directory is specified to which the file should be downloaded. The last part specifies which action should be taken (e.g., execute a command, run EXE or DLL).</p><figure></figure><h2 id="recommendations" class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left">Recommendations</h2><p class="  ">ZeroFox Intelligence highly discourages seeking out pirated software of any kind. As in this case, such downloads are often completely fake and will not install the software the victim wanted. In some cases, the download may contain the actual software—as well as a hidden malicious component to infect the victim.</p><p class="  ">ZeroFox Intelligence also highly recommends that organizations take reports of pirated software on corporate machines seriously. With each download, the risk of infection increases.&nbsp;</p><h2 id="yara" class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left">YARA</h2><p class="  ">ZeroFox Intelligence has created a public YARA rule that can be <a href="https://github.com/FirehaK/YARA/blob/master/RaccoonV2/RaccoonV2.yar" target="_blank" rel="noreferrer noopener">found on GitHub</a>.</p><h2
