class="prose_prose__BF0Lc"><p class="  "><em><strong>February 22, 2022&nbsp;Editor’s Note: Since conducting his initial research, ZeroFox Intelligence Researcher Stephan Simon has uncovered additional details about the operators and the botnet. Updates have been published <a href="/blog/quick-update-kraken-completes-its-rebrand-to-anubis/">here</a>. </strong></em></p><p class="  ">In late October 2021, ZeroFox Intelligence discovered a previously unknown botnet called Kraken. Though still under active development, Kraken already features the ability to download and execute secondary payloads, run shell commands, and take screenshots of the victim’s system. It currently makes use of SmokeLoader—a piece of malware used to install other malicious software—to spread, quickly gaining hundreds of bots each time a new command and control server is deployed. Despite having the same name, it should not be confused with the Kraken botnet from 2008 as they have little else in common.</p><h2 class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left undefined ">Details</h2><p class="  ">Since October 2021, ZeroFox Intelligence has been tracking Kraken – a previously unknown botnet targeting Windows that is currently under active development. Although the bot is simple in functionality, the author has been experimenting with new features while altering others. Current iterations of Kraken feature the ability to:</p><ul class="rich-text space-y-2"><li>Maintain persistence</li><li>Collect information about the host for registration (varies per version)</li><li>Download and execute files</li><li>Run shell commands</li><li>Steal various cryptocurrency wallets</li><li>Take screenshots</li></ul><h2 class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left undefined ">“Open Source” Beginnings</h2><p class="  ">Early versions of Kraken were based on code uploaded to GitHub on October 10, 2021. The project only had two commits, and the source code pre-dated any binaries ZeroFox observed in the wild. It is not currently known if the GitHub profile belongs to the botnet’s operator or if the operator simply used the code to kickstart their development.</p><figure class="my-8"><img alt="" data-href="true" width="1600" height="616" decoding="async" data-nimg="1" class="mx-auto" style="color:transparent" src=""/><figcaption class="text-center text-xs text-dark-slate"></figcaption></figure><h2 class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left undefined ">Observed Infection Vector</h2><p class="  ">Thanks to a tip by <a href="https://twitter.com/abuse_ch">@abuse_ch</a>, ZeroFox learned that Kraken originally spread in self-extracting RAR SFX files downloaded by SmokeLoader. These SFX files contained a UPX-packed version of Kraken, RedLine Stealer, and another binary used to delete Kraken. Current versions of Kraken are now downloaded by SmokeLoader directly. Kraken binaries are still UPX-packed but are now further protected by the Themida packer as well.</p><figure class="my-8"><img alt="" data-href="true" width="688" height="144" decoding="async" data-nimg="1" class="mx-auto" style="color:transparent" src=""/><figcaption class="text-center text-xs text-dark-slate"></figcaption></figure><h2 class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left undefined ">Installation and Persistence</h2><p class="  ">During Kraken’s installation phase, it attempts to move itself into %AppData%\Microsoft. The file name is hardcoded, though the author has changed it a few times. ZeroFox has observed file names such as taskhost.exe, Registry.exe, and Windows Defender GEO.exe.</p><p class="  ">To stay hidden, Kraken runs the following two commands:</p><ol class="rich-text space-y-2"></ol><p class="  ">The PowerShell command tells Microsoft Defender not to scan Kraken's installation directory, while the attrib command is used to hide the copied EXE file from an Explorer window that has not enabled the “Show hidden files, folders, and drives” option.</p><p class="  ">Kraken also makes use of the Windows Run registry key to ensure it starts every time the victim logs in.</p><figure class="my-8"><img alt="" data-href="true" width="1076" height="200" decoding="async" data-nimg="1" class="mx-auto" style="color:transparent" src=""/><figcaption class="text-center text-xs text-dark-slate"></figcaption></figure><p class="  ">A mix of fake and real information is stored in a new registry key under HKEY_CURRENT_USER\Software; it is all currently unused after saving it during the installation. The name of the key is another hardcoded value, though it has also changed occasionally. Early versions of Kraken observed by ZeroFox created a key with the name “Networking Service” or some slight variation, such as “Networking5 Servic1e” and “Netrworking5r Servirc1er”.</p><figure class="my-8"><img alt="" data-href="true" width="1029" height="244" decoding="async" data-nimg="1" class="mx-auto" style="color:transparent" src=""/><figcaption class="text-center text-xs text-dark-slate"></figcaption></figure><p class="  ">Aside from the hard-coded name for the registry key, the following information shown in <strong>Figure 4</strong> has remained the same in every version ZeroFox has encountered in the wild:&nbsp;</p><ul class="rich-text space-y-2"><li>ID - obfuscated UUID</li><li>INSTALL - installation timestamp</li><li>LAST - empty</li><li>NAME - obfuscated binary and Run key name (minus file extension)</li><li>REMASTER - always “nil”</li><li>VERSION - always “0.5.6”</li></ul><h2 class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left undefined ">Features</h2><p class="  ">Kraken’s feature set is simplistic for a botnet. Although not present in earlier builds, the bot is capable of collecting information about the infected host and sending it back to the command and control (C2) server during registration. The information collected seems to vary from build to build, though ZeroFox has observed the following being collected:<br></p><ul class="rich-text space-y-2"><li>Hostname</li><li>Username</li><li>Build ID (TEST_BUILD_ + the timestamp of the first run)</li><li>CPU details</li><li>GPU details</li><li>Operating system and version</li></ul><p class="  ">The botnet also features the ability to download and execute files. Originally, Kraken contained separate but similar functions for downloading files for different situations, such as updating the bot itself, executing secondary payloads, and receiving files through direct socket connection. These functions have since been combined into one, while the redundant functions were removed.</p><p class="  ">Kraken’s operators are able to run shell commands on infected hosts from the dashboard as well, returning the results back to the C2 server.</p><p class="  ">SSH brute-forcing functionality was added to some builds but was quickly removed. This function was hardcoded to attempt logging in as the root user of a given target and assumed a server would be listening on the default port. ZeroFox did not see any evidence of this feature being used, likely explaining its quick removal.</p><p class="  ">Upon execution, Kraken immediately takes a screenshot to send to the C2. A “ScreenShot” command also exists if the operator decides to take screenshots of the victim’s system on demand.</p><p class="  ">The most recent feature addition is the ability to steal various cryptocurrency wallets from the following locations:</p><ul class="rich-text space-y-2"><li>%AppData%\Zcash</li><li>%AppData%\Armory</li><li>%AppData%\bytecoin</li><li>%AppData%\Electrum\wallets</li><li>%AppData%\Ethereum\keystore</li><li>%AppData%\Exodus\exodus.wallet</li><li>%AppData%\Guarda\Local Storage\leveldb</li><li>%AppData%\atomic\Local Storage\leveldb</li><li>%AppData%\com.liberty.jaxx\IndexedDB\file__0.indexeddb.leveldb</li></ul><p class="  ">Currently supported commands are:</p><ul class="rich-text space-y-2"><li>Position - Unknown</li><li>ScreenShot - take a screenshot</li><li>SHELL - run a Windows shell command with cmd</li><li>UPLOAD - download and execute an EXE</li></ul><figure class="my-8"><img alt="" data-href="true" width="910" height="862" decoding="async" data-nimg="1" class="mx-auto" style="color:transparent" src=""/><figcaption class="text-center text-xs text-dark-slate"></figcaption></figure><h2 class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left undefined ">Dashboards</h2><p class="  ">Multiple versions of the administration panel or dashboard have been created since October 2021. While the original code found on GitHub did include a server, it did not have a web-based interface for interacting with the botnet.</p><h2 class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left undefined ">Kraken Panel</h2><p class="  ">The initial panel, aptly named “Kraken Panel,” was simple in terms of features. It offered basic statistics, links to download payloads, an option to upload new payloads, and a way to interact with a specific number of bots. This version did not appear to allow the operator(s) to choose which victims to interact with.</p><figure class="my-8"><img alt="" data-href="true" width="1536" height="1387" decoding="async" data-nimg="1" class="mx-auto" style="color:transparent" src=""/><figcaption class="text-center text-xs text-dark-slate"></figcaption></figure><h2 class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left undefined ">Anubis Panel</h2><p class="  ">The current version of the C2 has undergone a total redesign—complete with a new name,&nbsp;Anubis. The Anubis Panel provides far more information to the operator(s) than the original Kraken Panel. In addition to the previously provided statistics, it is now possible to view command history and information about the victim.</p><figure class="my-8"><img alt="" data-href="true" width="1600" height="530" decoding="async" data-nimg="1" class="mx-auto" style="color:transparent" src=""/><figcaption class="text-center text-xs text-dark-slate"></figcaption></figure><p class="  ">A later update to the Anubis Panel added the ability for the operator(s) to be more selective when choosing targets for commands. In previous versions, the operator(s) could only choose the number of victims to target with the command. With this update, targets can be chosen individually or by group using their external IP or geographic location.</p><figure class="my-8"><img alt="" data-href="true" width="1074" height="519" decoding="async" data-nimg="1" class="mx-auto" style="color:transparent" src=""/><figcaption class="text-center text-xs text-dark-slate"></figcaption></figure><p class="  ">The Anubis Panel also allows the operator(s) to view task and command history via the dashboard and TASK page. The TASK page shows information such as the ID generated for the task, the command being sent, how many victims the command should be sent to, the targeted geolocation, and a timestamp of when the task was initiated.</p><p class="  ">Initially, every task investigated by ZeroFox resulted in a version of RedLine Stealer being downloaded and executed on the victim’s machine. Some shell commands were observed as well, though these were only used to download more RedLine payloads using curl.</p><figure class="my-8"><img alt="" data-href="true" width="1600" height="474" decoding="async" data-nimg="1" class="mx-auto" style="color:transparent" src=""/><figcaption class="text-center text-xs text-dark-slate"></figcaption></figure><p class="  ">As the operator(s) behind Kraken continued to expand and gather more victims, ZeroFox began observing other generic information stealers and cryptocurrency miners being deployed. As of this writing, the botnet appears to be collecting around USD 3,000 every month.</p><figure class="my-8"><img alt="" data-href="true" width="1600" height="477" decoding="async" data-nimg="1" class="mx-auto" style="color:transparent" src=""/><figcaption class="text-center text-xs text-dark-slate"></figcaption></figure><h2 class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left undefined ">Recommendations</h2><ul class="rich-text space-y-2"><li>Ensure antivirus and intrusion detection software is up to date with all patches and rule sets.</li><li>Enable two-factor authentication for all organizational accounts to help mitigate phishing and credential stuffing attacks.</li><li>Maintain regularly scheduled backup routines, including off-site storage and integrity checks.</li><li>Avoid opening unsolicited attachments and never click suspicious links.</li><li>Log and monitor all administrative actions as much as possible. Alert on any suspicious activity.</li><li>Review network logs for potential signs of compromise and data egress.</li></ul><h2 class="my-5 text-2xl md:text-5xl lg:text-6xl font-semibold mb-3 text-left undefined ">Conclusion</h2><p class="  ">While in development, Kraken C2s seem to disappear often. ZeroFox has observed dwindling activity for a server on multiple occasions, only for another to appear a short time later using either a new port or a completely new IP. By using SmokeLoader to spread, Kraken quickly gains hundreds of new bots each time the operator changes the C2. Monitoring commands sent to Kraken victims from October 2021 through December 2021 revealed that the operator had focused entirely on pushing information stealers – specifically RedLine Stealer. It is currently unknown what the operator intends to do with the stolen credentials that have been collected or what the end goal is for creating this new botnet.</p><h2 class="my-5 text-2xl
